# Robot Learning（机器人学习）

*Robot learning 弥合了算法与物理动作之间的鸿沟。本文件涵盖运动学、动力学、经典 control、imitation learning（模仿学习）、sim-to-real（仿真到现实）迁移、manipulation（操作）、locomotion（移动）和安全性，这些技术赋予机器人移动、抓取、行走和与现实世界交互的能力。*

- 在前几章中，我们研究了如何感知世界（第 8 章、第 11 章第 1 个文件）以及如何从数据中学习（第 6 章）。但 perception 和学习还不够。机器人必须**行动**：移动手臂去抓取杯子、走过不平的地形、或在仓库中导航。这就是 robot learning 的用武之地。

- 核心挑战在于物理世界是连续的、高维的、充满接触的、且不容错误的。图像识别中的分类错误只是一个错误标签。机器人 control 中的错误则意味着机器人损坏或物体掉落。代价截然不同。

## 机器人运动学

- **Kinematics（运动学）**在不考虑力的情况下描述运动的几何。机械臂是一系列由关节连接的刚性连杆。每个关节有一个自由度（DoF）：要么旋转（旋转关节），要么滑动（移动关节）。

- 机器人的 **configuration（位形）**是所有关节角度（或位移）的集合 $\mathbf{q} = [q_1, q_2, \ldots, q_n]^T$。该向量位于 **joint space（关节空间）**（或位形空间）中，这是一个 $n$ 维空间，每个轴对应一个关节。一个 6-DoF 机械臂有 6D 位形空间。

![二连杆机械臂：关节角 q1 和 q2 通过 forward kinematics 决定末端执行器位置](../images/robot_arm_fk.svg)

- **Forward kinematics (FK)（正向运动学）**在给定关节角的情况下计算末端执行器（"手"）的位置和姿态。这是一个函数 $\mathbf{x} = f(\mathbf{q})$，把关节空间映射到 **task space（任务空间）**（末端执行器的 3D 位置和姿态，又称笛卡尔空间）。

- 每个关节由一个 $4 \times 4$ 齐次变换矩阵描述（回顾第 2 章的仿射变换）。**Denavit-Hartenberg (DH) 约定**用四个参数来参数化每个关节：连杆长度 $a$、连杆扭角 $\alpha$、连杆偏置 $d$ 和关节角 $\theta$。关节 $i$ 的变换为：

$$T_i = \begin{bmatrix} \cos\theta_i & -\sin\theta_i \cos\alpha_i & \sin\theta_i \sin\alpha_i & a_i \cos\theta_i \\ \sin\theta_i & \cos\theta_i \cos\alpha_i & -\cos\theta_i \sin\alpha_i & a_i \sin\theta_i \\ 0 & \sin\alpha_i & \cos\alpha_i & d_i \\ 0 & 0 & 0 & 1 \end{bmatrix}$$

- 完整的 forward kinematics 是所有关节变换的乘积：$T_{0 \to n} = T_1 T_2 \cdots T_n$。这是矩阵乘法串联变换（第 2 章）：每个关节的变换依次应用，从基座到末端执行器依次旋转和平移坐标系。

- **Inverse kinematics (IK)（逆向运动学）**是反向问题：给定期望的末端执行器位姿 $\mathbf{x}^*$，找到关节角 $\mathbf{q}$ 使得 $f(\mathbf{q}) = \mathbf{x}^*$。这要困难得多，因为：

    - 该映射是非线性的（涉及正弦和余弦）。
    - 可能存在多个解（不同的机械臂位形到达同一点）。
    - 可能无解（目标超出可达范围）。

- 解析解仅存在于特定机器人几何结构中。对于一般机器人，IK 使用 **Jacobian** 迭代求解。Jacobian $J(\mathbf{q})$ 把关节角的微小变化与末端执行器位置的微小变化联系起来（回顾第 3 章的 Jacobian）：

$$\dot{\mathbf{x}} = J(\mathbf{q}) \dot{\mathbf{q}}$$

- 要使末端执行器移动一个微小量 $\Delta \mathbf{x}$，我们需要 $\Delta \mathbf{q} = J^{-1} \Delta \mathbf{x}$（或当 $J$ 不是方阵时使用伪逆 $J^+ \Delta \mathbf{x}$）。这反复迭代直到末端执行器到达目标，本质上就是应用于运动学方程的牛顿法（第 3 章）。

- 在 **singularity（奇异点）**附近，Jacobian 会丢失秩（某些列变为线性相关，如我们在第 2 章所学的）。在物理上，这意味着机器人丢失一个自由度：无论关节运动多快，末端执行器都无法朝某些方向移动。伪逆在奇异点附近会爆炸，因此改用阻尼最小二乘（添加正则化项 $\lambda^2 I$）：

$$\Delta \mathbf{q} = J^T(JJ^T + \lambda^2 I)^{-1} \Delta \mathbf{x}$$

## 动力学与 Control

- **Dynamics（动力学）**把力纳入图景。机械臂的运动方程遵循 **manipulator equation（机械臂方程）**：

$$M(\mathbf{q})\ddot{\mathbf{q}} + C(\mathbf{q}, \dot{\mathbf{q}})\dot{\mathbf{q}} + \mathbf{g}(\mathbf{q}) = \boldsymbol{\tau}$$

- 其中 $M(\mathbf{q})$ 是质量（惯性）矩阵，$C(\mathbf{q}, \dot{\mathbf{q}})$ 捕获科氏力和离心力效应，$\mathbf{g}(\mathbf{q})$ 是重力向量，$\boldsymbol{\tau}$ 是关节力矩向量（即 control 输入）。这是一个二阶微分方程组，每个关节一个。

- 质量矩阵 $M$ 总是对称正定的（回顾第 2 章，正定矩阵保证唯一最小值，这里它确保系统对施加的力矩响应可预测）。

- **PID control** 是机器人学中最广泛使用的控制器。对于每个关节，它基于误差 $e(t) = q_{\text{desired}}(t) - q_{\text{actual}}(t)$ 计算力矩：

$$\tau(t) = K_p e(t) + K_i \int_0^t e(s) \, ds + K_d \dot{e}(t)$$

- 三项各有直观作用：
    - **Proportional（比例项）**（$K_p$）：与当前误差成正比地修正。误差越大 → 修正越大。就像把关节拉向目标的弹簧。
    - **Integral（积分项）**（$K_i$）：累积过去的误差以消除稳态偏差。如果关节持续欠调，积分项会累积并提供额外推力。
    - **Derivative（微分项）**（$K_d$）：对误差的变化率做出反应，提供阻尼。它减缓误差减小过程中的响应，防止超调和振荡。

![PID 控制器调参：高 Kp 会振荡，高 Kd 反应迟缓，调好的 PID 快速到达目标](../images/pid_response.svg)

- 调参 $K_p, K_i, K_d$ 是一种平衡：$K_p$ 过大引起振荡，$K_d$ 过大让系统迟钝，$K_i$ 过大引起 wind-up（积分在持续误差期间无限增长）。

- **Model Predictive Control (MPC)（模型预测控制）**向前看。在每个时间步，它求解一个优化问题：在动力学模型和约束下，找到在未来有限时域内最小化代价函数（如跟踪误差 + control 努力消耗）的未来 control 序列。只应用第一个 control，然后在下一个时间步重复该过程。

$$\min_{\mathbf{u}_{0:T}} \sum_{t=0}^{T} \left[ \|\mathbf{x}_t - \mathbf{x}_t^*\|_Q^2 + \|\mathbf{u}_t\|_R^2 \right] \quad \text{subject to} \quad \mathbf{x}_{t+1} = f(\mathbf{x}_t, \mathbf{u}_t)$$

- 这里 $\|\mathbf{x}\|_Q^2 = \mathbf{x}^T Q \mathbf{x}$ 是使用正定矩阵 $Q$ 的加权范数（第 2 章），它让你可以不同地惩罚不同的状态误差。MPC 自然地处理约束（关节限位、力矩限位、避障），因为它们被显式地纳入优化。

- **Impedance control（阻抗控制）**调节力与运动之间的关系，而非跟踪刚性轨迹。它不是命令"去到位置 $x$"，而是命令"表现得像一个以 $x$ 为中心的弹簧-阻尼系统"：

$$F = K_s(\mathbf{x}^* - \mathbf{x}) + D(\dot{\mathbf{x}}^* - \dot{\mathbf{x}})$$

- 其中 $K_s$ 是刚度矩阵，$D$ 是阻尼矩阵。这使机器人变得柔顺：如果它接触障碍物，会退让而非强行穿过。Impedance control 对接触密集型任务至关重要，如把销钉插入孔中或向人递物。

## Imitation Learning（模仿学习）

- 与其手工设计控制器，我们可以从示范中学习 control 策略。人类执行任务，机器人观察，学习算法提取策略。这就是 **imitation learning**（或从示范中学习）。

- **Behavioural cloning (BC)（行为克隆）**是最简单的方法：把示范当作有监督学习数据集。给定来自专家的观测-动作对 $\{(\mathbf{o}_t, \mathbf{a}_t)\}$，训练策略 $\pi_\theta(\mathbf{a} \mid \mathbf{o})$ 从观测预测专家的动作。这是标准的有监督学习（第 6 章）：最小化损失：

$$\mathcal{L}(\theta) = \mathbb{E}_{(\mathbf{o}, \mathbf{a}) \sim \mathcal{D}} \left[ \| \pi_\theta(\mathbf{o}) - \mathbf{a} \|^2 \right]$$

![行为克隆中的分布偏移：小误差累积，导致学习到的策略偏离专家轨迹](../images/distribution_shift_bc.svg)

- 问题在于 **distribution shift（分布偏移）**（也称为 **compounding error problem，复合误差问题**）。训练时，策略看到的是专家所处的状态。部署时，策略自身的小误差把它推入专家从未访问过的状态。这些陌生状态导致更差的动作，进而导致更陌生的状态，误差迅速累积。

- 想象通过观看完美驾驶员来学开车。你从未见过小幅偏转之后会发生什么，因为专家从不偏转。第一次你稍微偏离时，你不知道如何恢复。

- **DAgger**（Dataset Aggregation，数据集聚合）通过迭代解决这个问题：
    1. 在当前数据上训练策略。
    2. 在环境中运行策略，收集新状态。
    3. 请专家为这些新状态标注正确动作。
    4. 把新数据加入数据集并重新训练。

- 经过多次迭代，数据集覆盖学习策略实际访问的状态，而不仅仅是专家的轨迹。策略得以改进，因为它见过自己的错误并学会了如何恢复。

- **Action Chunking with Transformers (ACT)** 是一种现代方法，策略预测未来动作序列（一个"chunk"）而非一次一个动作。这通过带 transformer 主干的条件 VAE 实现。预测动作 chunk 更鲁棒，因为它捕获了时间相关性：伸手动作的平滑性被编码在 chunk 中，而非依赖可能漂移的自回归单步预测。

- **Diffusion Policy** 把 diffusion model（第 8 章）应用于动作生成。它不预测单个动作，而是建模以观测为条件的可能动作的完整分布。从噪声开始，它迭代去噪以产生动作序列。这自然地处理 **multimodality（多模态性）**：当完成任务存在多种有效方式时（从左边还是从右边接近），diffusion model 可以表示两种模态，而回归策略会把它们平均（到达中间某处，可能既非有效）。

## Sim-to-Real 迁移

- 在现实世界中训练机器人昂贵、缓慢且危险。一个通过试错学习抓取的机器人可能需要数千次尝试，过程中会损坏物体和自身。**Simulation（仿真）**提供无限、安全、快速的体验。但仿真器并不完美：物理是近似的，视觉是合成的，接触是简化的。

- **Sim-to-real gap（仿真到现实差距）**是仿真性能与现实性能之间的差异。在仿真中完美工作的策略可能在真实机器人上彻底失败，因为它过拟合了仿真器特有的细节。

![通过 domain randomisation 实现 sim-to-real：在许多随机化仿真上训练，使现实世界只是另一种变体](../images/sim_to_real.svg)

- **Domain randomisation（域随机化）**通过在广泛的仿真器设置范围内训练来对抗这一问题。不用一种仿真，而是用数千种随机化设置：
    - 物理：摩擦系数、质量、阻尼
    - 视觉：光照、纹理、颜色、相机位置
    - 动力学：电机延迟、噪声水平

- 思路是：如果策略在所有这些变体上都能工作，那么现实世界只是分布内的"另一种变体"。策略学习对随机化属性不变的特征，这些不变特征能够迁移。

- **System identification（系统辨识）**采取相反的方法：与其随机化一切，不如仔细测量真实系统的物理参数并调整仿真器以匹配。这给出更准确的仿真，但很脆弱（任何未建模的效应都会产生差距）。

- 实践中，最佳结果结合两者：用 system identification 让仿真器相当接近，然后用 domain randomisation 覆盖剩余的不确定性。

- **通过微调实现 sim-to-real**主要在仿真中训练，然后进行少量现实世界微调。仿真提供良好的初始化，现实世界数据纠正仿真器特有的偏差。这比从零开始训练需要的现实世界数据少得多。

## 机器人的世界模型

- 上述所有 RL 和 imitation learning 方法都是 **model-free** 的：策略通过直接交互（或示范）学习行动，而不显式建模世界如何运作。另一种选择是 **model-based** 学习：先学习环境动力学的模型，然后用该模型来规划或生成合成经验。

- **World model（世界模型）**学习转移函数 $p(s_{t+1} \mid s_t, a_t)$：给定当前状态和动作，预测下一个状态（如第 10 章所介绍）。在机器人学中，这意味着预测机器人采取某个动作后会发生什么："如果我把这块方块向左推，它会滑动 3cm，它后面的杯子会倒下。"

- 其吸引力在于 **sample efficiency（样本效率）**。现实世界机器人交互很昂贵。如果机器人能从适量真实数据中学到世界模型，它就可以在脑中 rollout 模型来"想象"数千条轨迹，在不触碰物理世界的情况下规划和改进策略。这类似于棋手通过在脑中模拟棋步来前瞻。

- **DreamerV3** 是一个通用的 model-based RL 智能体。它联合学习三个组件：
    - **Representation model（表示模型）**：把观测编码为紧凑的潜状态。
    - **Transition model（转移模型）**（即世界模型）：给定当前状态和动作，预测下一个潜状态。
    - **Reward model（奖励模型）**：从潜状态预测奖励。

- 然后智能体通过在潜空间中 rollout 转移模型多步来"做梦"，在这些想象的轨迹上训练策略，并把策略迁移到真实环境。关键创新在于所有想象都发生在潜空间（紧凑的学习型表示）而非像素空间，使其在计算上可行。

$$\hat{s}_{t+1} = f_\theta(s_t, a_t), \quad \hat{r}_t = g_\theta(s_t)$$

- 转移模型 $f_\theta$ 和奖励模型 $g_\theta$ 在真实经验上训练，策略在想象的 rollout 上训练。这把数据收集与策略优化解耦。

- 对于机器人 manipulation，世界模型使 **mental rehearsal（心理预演）**成为可能。在尝试抓取之前，机器人可以在其学习模型中模拟若干种方法并选择最可能成功的一种。这对接触密集型任务尤其有价值，因为现实世界试错既慢又危险。

- 世界模型还自然地连接到 **sim-to-real**：在真实数据上训练的世界模型实际上是一个学习型仿真器，自动捕获现实世界物理，完全绕过 sim-to-real 差距。该模型对于已充分理解的场景可能不如手工仿真器准确，但它捕获了手工仿真器常常搞错的效应（摩擦、变形、接触动力学）。

- **JEPA**（Joint Embedding Predictive Architecture，联合嵌入预测架构，第 10 章介绍）为像素级预测提供了替代方案。JEPA 不预测精确的未来观测，而是在 embedding 空间中预测："下一状态的潜表示将接近这个向量。"这避免了预测像素完美未来的困难（这既无必要又浪费计算），而专注于预测对决策有意义的未来方面。

- 世界模型的局限在于 **compounding prediction error（复合预测误差）**。转移模型中的小误差在长 rollout 中累积，导致想象轨迹偏离现实。缓解措施包括短想象时域、集成模型（用不确定性检测预测何时不可靠），以及周期性地用新鲜的真实世界数据为模型接地。

## Manipulation（操作）

- **Manipulation** 是使用机器人末端执行器与物体交互的艺术：拾取、放置、推动、插入、组装。

- **Grasping（抓取）**是基础的 manipulation 技能。目标是找到稳定的抓取位姿：夹爪的位置和姿态，使其能稳固地握住物体。

- **Analytical grasp planning（解析式抓取规划）**使用物理。如果接触力能抵抗外部 wrench（力和力矩），则抓取是稳定的。对于平行爪夹爪，最简单的准则是 **force closure（力闭合）**条件：接触法线必须跨越所有力方向，使抓取能抵抗任何扰动。这涉及检查抓取 wrench 矩阵的秩，这是第 2 章秩概念的直接应用。

- **Data-driven grasping（数据驱动抓取）**从感觉输入预测抓取成功率。给定桌上物体的深度图，网络为每个候选夹爪位姿预测抓取质量分数。**GraspNet** 和类似架构使用点云编码器（PointNet 风格，第 8 章）预测 6-DoF 抓取位姿（位置 + 姿态）及置信度分数。

- **Dexterous manipulation（灵巧操作）**超越简单的拾取和放置。多指手有 20+ DoF，能执行诸如手中旋转（在手指间转动笔）、使用工具和精细装配等任务。状态空间巨大且接触复杂，使这成为机器人学中最困难的问题之一。

- 学习灵巧操作常使用强化学习（第 6 章），在仿真中配合大量 domain randomisation。OpenAI 用 Shadow 手解魔方的工作在仿真中用随机化物理训练 PPO 策略，并实现了向真实机器人手的迁移。

- **Contact-rich task（接触密集型任务）**如销孔装配或擦拭表面要求机器人与环境保持受控接触。这些任务需要力感知和柔顺 control（impedance control），且难以精确仿真，因为接触物理是出了名地难以建模。

## Locomotion（移动）

- Locomotion 是机器人在世界中移动其身体：行走、奔跑、攀爬、游泳。与 manipulation 的关键区别在于机器人必须在移动中保持平衡，且与地面的接触点随时间变化。

- **Legged locomotion（足式移动）**具有挑战性，因为它本质上不稳定。双足机器人（人形）在迈步时单腿站立就像一个倒立摆。质心必须保持在支撑多边形（与地面接触的脚的凸包）上方，否则机器人会摔倒。

- **Zero Moment Point (ZMP)（零矩点）**是地面上重力和惯性力净力矩为零的点。如果 ZMP 保持在支撑多边形内，机器人就不会翻倒。传统人形控制器（如本田的 ASIMO）规划保持 ZMP 在范围内的轨迹。

- **Central Pattern Generator (CPG)（中央模式发生器）**是受生物启发的基于振荡器的控制器。动物利用脊髓中的神经回路产生节律性 locomotion 模式（行走、小跑、奔驰），而无需大脑持续参与。CPG 模型使用耦合微分方程：

$$\dot{\phi}_i = \omega_i + \sum_j w_{ij} \sin(\phi_j - \phi_i - \psi_{ij})$$

- 其中 $\phi_i$ 是振荡器 $i$ 的相位，$\omega_i$ 是固有频率，$w_{ij}$ 是耦合强度，$\psi_{ij}$ 是期望相位偏移。不同的相位关系产生不同的步态：所有腿同步（bound）、交替对（trot）、顺序（walk）。正弦耦合自然地同步振荡器，类似于 Fourier 级数（第 3 章）把运动分解为频率分量。

- **用于 locomotion 的强化学习**已成为敏捷四足和人形机器人的主流方法。机器人通过在仿真中试错学习策略 $\pi(\mathbf{a} \mid \mathbf{o})$（第 6 章），奖励包括前进速度、稳定性和能效，惩罚包括摔倒、关节超限和抖动。

- 近期工作（如 Agility Robotics、Boston Dynamics 及学术实验室）的关键洞察是：RL 训练的 locomotion 策略远比手工设计的控制器鲁棒。它们自然学会了从推搡中恢复、适应地形变化、应对工程师未曾预料的情况。训练通常使用 PPO（第 6 章）配合 domain randomisation。

- **Quadruped robot（四足机器人）**（如 Boston Dynamics Spot 或 Unitree Go2）已成为足式机器人的主力。四条腿提供固有稳定性（三条腿的三角架总能支撑身体，同时一条腿移动）。四足的 RL 策略取得了令人印象深刻的结果：以 3+ m/s 奔跑、爬楼梯、在岩石地形中导航、从被踢中恢复。

- **Humanoid locomotion（人形移动）**更难，因为双足的支撑多边形更小且质心更高。近期进展（Tesla Optimus、Figure、Unitree H1）使用在仿真中训练的 RL 配合精心的奖励塑造。人形不仅必须学会行走，还要协调手臂摆动以保持平衡、在不平表面上导航、从扰动中恢复。

## 机器人学习中的安全

- 随机探索以学习的机器人（如 RL 中）可能损坏自身、环境或附近的人。**Safe robot learning（安全机器人学习）**约束探索以避免灾难性后果。

- **Constrained RL（约束强化学习）**在 MDP 中加入安全约束（第 6 章）。目标变为：在 $J_c(\pi) \leq d$ 的约束下最大化奖励，其中 $J_c$ 是期望累积代价（如碰撞事件），$d$ 是最大允许代价。Constrained Policy Optimisation (CPO) 等算法扩展 PPO 以处理这些约束。

- **Safety envelope（安全包络）**定义机器人绝不可逾越的硬边界，不论学习策略怎么说。安全控制器监视机器人状态，在约束即将被违反时（如接近关节限位、在人附近移动过快、超过力阈值）覆盖学习策略。这是一种分层架构：学习算法负责性能，安全层负责约束。

- **Risk-aware planning（风险感知规划）**显式地建模环境不确定性和机器人自身状态估计的不确定性。它不规划最可能的结果，而是规划置信界限内的最坏情况。这与第 2 章的条件数概念相连：条件良好的系统对扰动鲁棒，risk-aware planning 寻求在扰动下仍安全的 control 策略。

1. 为简单的二连杆平面机械臂实现 forward kinematics。计算并可视化不同关节角下的末端执行器位置。
```python
import jax.numpy as jnp
import matplotlib.pyplot as plt

def forward_kinematics(q1, q2, l1=1.0, l2=0.8):
    """Compute joint and end-effector positions for a 2-link arm."""
    x1 = l1 * jnp.cos(q1)
    y1 = l1 * jnp.sin(q1)
    x2 = x1 + l2 * jnp.cos(q1 + q2)
    y2 = y1 + l2 * jnp.sin(q1 + q2)
    return jnp.array([0, x1, x2]), jnp.array([0, y1, y2])

fig, ax = plt.subplots(figsize=(6, 6))
configs = [(0.5, 0.3), (1.0, -0.5), (1.5, 1.0), (2.0, -1.5)]
colors = ["#e74c3c", "#3498db", "#27ae60", "#9b59b6"]

for (q1, q2), c in zip(configs, colors):
    xs, ys = forward_kinematics(q1, q2)
    ax.plot(xs, ys, "o-", color=c, linewidth=2, markersize=6,
            label=f"q=({q1:.1f}, {q2:.1f})")

ax.set_xlim(-2, 2); ax.set_ylim(-2, 2)
ax.set_aspect("equal"); ax.grid(True); ax.legend()
ax.set_title("2-Link Robot Arm: Forward Kinematics")
plt.show()
```

2. 使用 Jacobian 伪逆实现 inverse kinematics。从随机位形出发，迭代地把末端执行器移向目标。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

l1, l2 = 1.0, 0.8

def end_effector(q):
    x = l1 * jnp.cos(q[0]) + l2 * jnp.cos(q[0] + q[1])
    y = l1 * jnp.sin(q[0]) + l2 * jnp.sin(q[0] + q[1])
    return jnp.array([x, y])

jacobian_fn = jax.jacobian(end_effector)

target = jnp.array([0.5, 1.2])
q = jnp.array([0.1, 0.1])
trajectory = [end_effector(q)]

for _ in range(50):
    pos = end_effector(q)
    error = target - pos
    if jnp.linalg.norm(error) < 1e-4:
        break
    J = jacobian_fn(q)
    # Damped pseudo-inverse to handle near-singularities
    dq = J.T @ jnp.linalg.solve(J @ J.T + 0.01 * jnp.eye(2), error)
    q = q + dq
    trajectory.append(end_effector(q))

traj = jnp.stack(trajectory)
plt.plot(traj[:, 0], traj[:, 1], "b.-", label="end-effector path")
plt.plot(*target, "r*", markersize=15, label="target")
plt.gca().set_aspect("equal"); plt.grid(True); plt.legend()
plt.title(f"IK converged in {len(trajectory)-1} steps")
plt.show()
```

3. 模拟一个简单的 PID 控制器跟踪期望的关节轨迹。观察调参增益的效果。
```python
import jax.numpy as jnp
import matplotlib.pyplot as plt

# Desired trajectory: smooth sinusoidal motion
dt = 0.01
t = jnp.arange(0, 5, dt)
q_desired = jnp.sin(2 * t)

# Simulate second-order dynamics: m * q_ddot + b * q_dot = tau
m, b_damp = 1.0, 0.5

for Kp, Kd, Ki, label in [(10, 5, 0, "PD only"), (10, 5, 2, "PID"), (50, 10, 2, "Aggressive PID")]:
    q, q_dot, integral = 0.0, 0.0, 0.0
    qs = []
    for i in range(len(t)):
        error = q_desired[i] - q
        integral += error * dt
        d_error = -q_dot  # derivative of error (desired velocity is known but simplified here)
        tau = Kp * error + Kd * d_error + Ki * integral
        q_ddot = (tau - b_damp * q_dot) / m
        q_dot += q_ddot * dt
        q += q_dot * dt
        qs.append(float(q))

    plt.plot(t, qs, label=label)

plt.plot(t, q_desired, "k--", label="desired", linewidth=2)
plt.xlabel("Time (s)"); plt.ylabel("Joint angle")
plt.legend(); plt.title("PID Controller Tracking")
plt.show()
```
