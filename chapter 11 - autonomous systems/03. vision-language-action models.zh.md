# Vision-Language-Action Model（视觉-语言-动作模型）

*Vision-Language-Action model（VLA）把看、理解语言和行动统一到单个神经网络中。本文件涵盖 VLA 架构、动作 tokenisation（动作分词）、RT-2、Octo、OpenVLA、预训练策略、泛化、与具体形态无关的模型以及基准测试。*

- 在前几个文件中，我们涵盖了 perception（感知世界）和 robot learning（控制身体）。传统上，这些是分开的流程：perception 模块检测物体，语言模块解释命令，control 模块生成动作。每个模块独立设计、训练和调试。

- **Vision-Language-Action model（VLA）**把这一流程折叠为单个神经网络。模型接收图像（vision）、自然语言指令（language），输出电机命令（action）。一个模型，端到端。

- 这遵循我们在第 10 章看到的统一趋势：正如多模态模型把视觉和语言理解合并到一个架构中，VLA 把这种统一扩展到物理动作。其洞察是：语言为指定任务（"拿起红色的杯子并放到架子上"）提供了自然、灵活的接口，而大型预训练 vision-language model 已经同时理解图像和指令。

## 从 Vision-Language 到 Action

- 回顾第 10 章，**Vision-Language Model (VLM)** 如 LLaVA 和 Flamingo 接收图像和文本作为输入，产生文本作为输出。它们以语言理解场景、回答问题、遵循指令。

- VLA 提问：如果输出不是文本而是**机器人动作**会怎样？模型不生成"红色的杯子在桌子左侧"，而是生成一串把机械臂移向并抓取该杯子的电机命令。

- 关键架构洞察是：动作可以像单词一样表示为 token。如果 VLM 通过 next-token prediction 逐 token 生成语言，VLA 以相同方式生成动作 token。transformer 在本质上并不关心输出 token 表示的是"杯子"还是"夹爪前移 2cm"。

- 这把机器人 control 重新构造为一个序列建模问题，而 transformer 在这方面表现出色（第 7 章）。模型学习映射：(图像观测, 语言指令) $\to$ (动作 token 序列)。

## VLA 架构

![VLA 架构：camera 图像和语言指令被编码为 token，由 LLM 主干处理，并解码为机器人动作](../images/vla_architecture.svg)

- 一个典型的 VLA 有三个组件：

    - **Vision encoder（视觉编码器）**：把 camera 图像处理为视觉 token。通常是预训练的 ViT（第 8 章）或 SigLIP 编码器（第 10 章）。图像被分成 patch，每个 patch 作为 token 嵌入，与标准 vision transformer 完全一致。

    - **Language model backbone（语言模型主干）**：一个预训练 LLM（如 LLaMA、PaLM），处理视觉 token 和语言 token 的交错序列。推理就在这里发生：模型通过同时关注指令和视觉特征来理解"拿起**红色的**杯子"。

    - **Action head（动作头）**：把 LLM 的输出映射到机器人动作。这可以是把最后隐状态映射到连续动作值的简单 MLP，也可以是把动作转换为 LLM 现有词表预测的离散 token 的 tokenisation 方案。

- 架构看起来是：

$$\text{Image} \xrightarrow{\text{ViT}} \text{visual tokens} \quad + \quad \text{Instruction} \xrightarrow{\text{tokeniser}} \text{language tokens} \quad \xrightarrow{\text{LLM}} \quad \text{action tokens}$$

- 视觉 token 和语言 token 被拼接（或交错）并送入 transformer 主干，由其自回归地产生动作 token。这与 VLM 架构（第 10 章）相同，只是输出模态是动作而非文本。

## Action Tokenisation（动作分词）

- 机器人动作是连续的：关节速度、末端执行器位置、夹爪宽度。这些必须转换为离散 token 才能让 LLM 生成。

![Action tokenisation：连续动作值被分箱为 LLM 作为 token 生成的离散索引](../images/action_tokenisation.svg)

- 最简单的方法是 **uniform discretisation（均匀离散化）**。每个动作维度被划分为覆盖有效值范围的 $N$ 个箱。例如，如果 x 速度范围为 -0.1 到 0.1 m/s，用 256 个箱，每个箱代表 $\frac{0.2}{256} \approx 0.8$ mm/s。一个动作值被映射到最近的箱索引，这个索引成为一个 token。

- 有 7 个动作维度（6 DoF + 夹爪），每个 256 个箱，动作词表有 $7 \times 256 = 1792$ 个 token。这些被加到 LLM 现有的文本词表中。模型对每个维度生成一个动作 token，自回归地生成，就像生成单词一样。

- **Action chunking（动作分块）**一次预测多个未来时间步而非单个动作。如果 chunk 大小为 $H$，模型输出 $H \times d$ 个 token（其中 $d$ 是动作维度数）。这对平滑、时间连贯的运动至关重要。一次预测一步可能产生抖动行为，因为每次预测是独立的。Chunking 强制模型规划一条短轨迹，捕获时间结构。

- 更复杂的方法通过 VQ-VAE（第 10 章）使用**学习型 tokenisation**。VQ-VAE 编码器把连续动作序列映射为离散码本索引序列，解码器从这些索引重建连续动作。LLM 随后生成码本索引而非均匀分箱的值。这类似于图像 tokeniser（第 10 章）把视觉信息压缩为紧凑的离散码。

## 关键 VLA 模型

- **RT-2**（Robotic Transformer 2，Google DeepMind）是首个大规模 VLA。它取一个预训练 VLM（PaLM-E 或 PaLI-X，高达 55B 参数）并在机器人示范数据上微调。动作表示为文本字符串：token 序列"1 128 91 241 5 101 127"编码一个 7 维动作（每个数字是一个箱索引）。

- RT-2 展示了一个显著特性：来自 VLM 主干的 **emergent capability（涌现能力）**迁移到机器人学。模型能跟随涉及机器人数据中从未见过的概念的指令（如"把香蕉移到以 A 开头的国家"需要视觉物体识别 + 世界知识 + 动作）。VLM 的语言理解和视觉推理是"免费附送"的。

- RT-2 的局限在于它是在单一机器人具体形态（特定臂配特定夹爪）的数据上训练的。它不能泛化到不同机器人。

- **Octo**（UC Berkeley）是一个开源、**与具体形态无关（embodiment-agnostic）**的 VLA，设计为跨不同机器人平台工作。关键创新有：

    - 用 **diffusion action head** 代替自回归 token 预测。动作头接收 transformer 的输出，通过去噪 diffusion 过程产生动作（第 8 章）。这自然地处理多模态动作分布（见下图），即完成任务存在多种有效方式的情况。

![多模态动作分布：回归把两条有效路径平均为穿过障碍物的无效路径](../images/multimodal_action_distribution.svg)

    - **灵活的观测和动作空间**：Octo 为不同机器人配置使用任务特定的 tokeniser。它在 Open X-Embodiment 数据集上预训练，该数据集包含来自 22 种不同机器人形态的示范。

    - **高效微调**：Octo 可以用少至 100 个示范微调到新机器人，使其对数据有限的实验室切实可行。

- **OpenVLA**（Stanford、UC Berkeley）采取微调现有开源 VLM（基于 Llama）用于机器人学的方法。它使用 7B 参数主干、均匀 action tokenisation（每维 256 个箱），并在 Open X-Embodiment 数据上训练。其优势在于简单：架构是标准的 VLM，把动作 token 追加到词表，使其易于用现有 LLM 基础设施训练和部署。

- **$\pi_0$**（Physical Intelligence）代表了当前最优水平。它使用预训练 VLM 主干，配以 **flow matching** 动作头（第 8 章）。Flow matching 通过学习把噪声传输到动作分布的速度场来生成动作，产生平滑、时间连贯的动作轨迹。$\pi_0$ 展示了显著的通用性，在多种机器人形态上执行任务，包括双手 manipulation 和灵巧手控制。

## 预训练配方

- VLA 从预训练 VLM 主干中获益巨大，这些主干已经理解视觉场景和语言。训练流程通常遵循以下阶段：

    1. **VLM 预训练**：在来自互联网的数十亿图文对上训练（或使用现成的）vision-language model（CLIP、SigLIP、LLaVA 风格的训练，如第 10 章所述）。

    2. **机器人数据协同训练**：在互联网数据和机器人示范数据的混合上微调 VLM。互联网数据防止对视觉和语言理解的灾难性遗忘，机器人数据则教授动作生成。混合比例很重要：机器人数据过多会降低语言理解，过少则学不会动作。

    3. **任务特定微调**：可选，针对特定任务或机器人的示范微调，常用 LoRA（第 10 章）以保持可训练参数数量较少。

- 机器人数据量比互联网数据小几个数量级。VLM 可能在数十亿图像上预训练，但最大的机器人数据集（Open X-Embodiment）在所有形态下只包含数百万帧。这种数据稀缺是为什么从预训练 VLM 起步至关重要的原因：视觉和语言表示能够迁移，只有动作映射需要从有限的机器人数据中学习。

## 泛化

- VLA 的承诺是**泛化**：执行训练中未见过的任务、使用未见过的物体、在未见过的环境中、遵循未见过的指令。

- VLA 沿几个轴泛化：

    - **新物体**：VLM 主干从互联网预训练中识别物体。如果模型从网络图像中知道"螺丝刀"长什么样，它就能操纵一把，即使没有机器人示范包含过螺丝刀。

    - **新指令**：组合性语言理解让模型能跟随已知概念的新组合。"把蓝色方块叠到绿色方块上"即使训练中只展示了叠红色方块也能工作，因为模型从语言预训练中理解颜色形容词。

    - **新环境**：在一定程度上，VLA 能跨视觉域迁移（不同的桌子、光照、背景），因为视觉编码器在多样化的网络图像上预训练。但这有局限：在实验室中训练的机器人在杂乱厨房中可能失效。

    - **新具体形态**：这是最难的一个轴。不同机器人有不同的动作空间（关节角 vs. 末端执行器速度）、不同 sensor（腕部相机 vs. 顶部相机）、不同物理能力。Embodiment-agnostic 模型如 Octo 和 $\pi_0$ 通过灵活的 tokeniser和跨多种机器人类型的预训练来解决这个问题。

- 泛化在**留出任务**上评估：让机器人执行它从未训练过的任务。在新任务上 50–80% 的成功率被视为强结果，而在分布内任务上则超过 90%。随着模型规模扩大和机器人数据集增长，这一差距正在缩小。

## 与具体形态无关的模型

- 该领域正迈向**一个模型，多种机器人**。不再为每个机器人训练单独的策略，单个 VLA 处理多种形态。

- 这需要解决**动作空间不匹配**问题。配平行夹爪的 7-DoF 机械臂有 7 个动作维度。双手配置有 14 个。四足有 12 个。人形有 30+ 个。Action tokenisation 必须足够灵活以处理所有这些。

- 解决方案包括：
    - **填充动作向量**：使用最大的动作空间，较小的用零填充。
    - **每种形态一个动作头**：共享的 transformer 主干配每种机器人类型的小型独立 MLP。
    - **归一化动作表示**：在统一坐标系中表示所有动作（如世界坐标系中的末端执行器速度），使产生相似末端执行器运动的不同机器人共享相同的动作 token。

- 共享主干学习通用的视觉和语言理解，以及常见的 manipulation 策略（从上方接近、与物体对齐、闭合夹爪）。形态特定组件只需要把这些高层策略翻译为具体的电机命令。

## 基准测试与评估

- 评估 VLA 独具挑战，因为它需要物理机器人实验（或高保真仿真）。

- **SIMPLER**（Simulated Manipulation Policy Evaluation for Robot learning）提供标准化的仿真环境，无需物理硬件即可比较 VLA 性能。它与现实世界成功率相关性良好，并实现可复现的基准测试。

- **现实世界评估**仍是黄金标准。典型协议：
    1. 定义一组有明确成功标准的任务（物体到达目标位置、选中正确物体、任务在时限内完成）。
    2. 每个任务运行 $N$ 次试验（通常 10–50）。
    3. 报告带置信区间的成功率。
    4. 包含留出（从未训练过的）任务以衡量泛化。

- **Open X-Embodiment** 数据集和基准汇集了来自 22 家机构、跨多种机器人平台的机器人数据。它提供共享示范的标准化格式和跨形态迁移的通用评估套件。

1. 实现 action tokenisation：把连续动作离散化为箱并重建它们。观察量化误差随箱数的变化。
```python
import jax.numpy as jnp

# Continuous action: 7 dimensions (6 DoF + gripper)
action_true = jnp.array([0.023, -0.051, 0.012, 0.1, -0.03, 0.005, 0.8])
action_min = jnp.array([-0.1, -0.1, -0.1, -0.5, -0.5, -0.5, 0.0])
action_max = jnp.array([ 0.1,  0.1,  0.1,  0.5,  0.5,  0.5, 1.0])

for n_bins in [16, 64, 256, 1024]:
    # Tokenise: map continuous value to bin index
    normalised = (action_true - action_min) / (action_max - action_min)
    tokens = jnp.clip((normalised * n_bins).astype(int), 0, n_bins - 1)

    # Detokenise: map bin index back to continuous value
    reconstructed = (tokens + 0.5) / n_bins * (action_max - action_min) + action_min

    error = jnp.linalg.norm(action_true - reconstructed)
    print(f"bins={n_bins:4d}  tokens={tokens}  error={error:.6f}")
```

2. 模拟 action chunking 与单步预测。生成平滑轨迹，给单步预测加噪声，并与基于 chunk 的预测比较。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

# Ground truth smooth trajectory (e.g., reaching motion)
t = jnp.linspace(0, 2 * jnp.pi, 100)
gt_x = jnp.sin(t)
gt_y = 1 - jnp.cos(t)

# Single-step: each prediction has independent noise
rng = jax.random.PRNGKey(42)
noise_ss = jax.random.normal(rng, (100, 2)) * 0.05
single_step = jnp.stack([gt_x, gt_y], axis=1) + noise_ss
# Cumulative drift from single-step errors
single_step_cumulative = jnp.cumsum(noise_ss, axis=0) * 0.3 + jnp.stack([gt_x, gt_y], axis=1)

# Chunked (chunk_size=10): noise is correlated within chunks, smoother
chunk_size = 10
rng2 = jax.random.PRNGKey(7)
chunks = []
for i in range(0, 100, chunk_size):
    chunk_noise = jax.random.normal(jax.random.fold_in(rng2, i), (2,)) * 0.05
    chunk = jnp.stack([gt_x[i:i+chunk_size], gt_y[i:i+chunk_size]], axis=1)
    chunks.append(chunk + chunk_noise)
chunked = jnp.concatenate(chunks, axis=0)

plt.figure(figsize=(8, 4))
plt.plot(gt_x, gt_y, "k-", linewidth=2, label="Ground truth")
plt.plot(single_step_cumulative[:, 0], single_step_cumulative[:, 1],
         "r-", alpha=0.7, label="Single-step (drifts)")
plt.plot(chunked[:, 0], chunked[:, 1], "b-", alpha=0.7, label="Chunked (stable)")
plt.legend(); plt.axis("equal"); plt.grid(True)
plt.title("Action Chunking vs Single-Step Prediction")
plt.show()
```

3. 可视化 VLA 的动作分布如何呈现多模态。用简单的 2D 高斯混合说明为什么 diffusion/flow-matching 动作头优于回归。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

# Two valid ways to reach around an obstacle: left or right
rng = jax.random.PRNGKey(0)
k1, k2 = jax.random.split(rng)

mode1 = jax.random.normal(k1, (200, 2)) * 0.15 + jnp.array([-1.0, 0.5])
mode2 = jax.random.normal(k2, (200, 2)) * 0.15 + jnp.array([ 1.0, 0.5])
samples = jnp.concatenate([mode1, mode2])

# Regression predicts the mean = average of modes (invalid!)
mean_pred = samples.mean(axis=0)

plt.figure(figsize=(6, 5))
plt.scatter(samples[:, 0], samples[:, 1], s=5, alpha=0.5, label="True action distribution")
plt.plot(*mean_pred, "rx", markersize=15, markeredgewidth=3, label="Regression mean (invalid!)")
plt.plot(-1, 0.5, "g^", markersize=12, label="Mode 1 (go left)")
plt.plot(1, 0.5, "b^", markersize=12, label="Mode 2 (go right)")
plt.legend(); plt.grid(True)
plt.title("Multimodal Actions: Why Regression Fails")
plt.xlabel("Action dim 1"); plt.ylabel("Action dim 2")
plt.show()
```
