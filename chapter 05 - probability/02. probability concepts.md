# Концепции теории вероятностей

*Теория вероятностей формализует неопределенность и предоставляет правила для рассуждений в условиях этой неопределенности. В этом файле рассматриваются пространства элементарных исходов, события, аксиомы теории вероятностей, условная вероятность, независимость, теорема Байеса, а также частотный и байесовский подходы — математический фундамент, лежащий в основе каждой генеративной и дискриминативной модели в машинном обучении.*

- Вероятность присваивает событию число от 0 до 1, измеряя степень его ожидаемости.

- Вероятность 0 означает невозможность, 1 — достоверность, а 0.5 — подбрасывание монеты.

- Существует две основные интерпретации. **Частотный** (frequentist) подход гласит, что вероятность — это относительная частота в долгосрочной перспективе: если подбросить честную монету 10 000 раз, орел выпадет примерно в 50% случаев.

- **Байесовский** (Bayesian) подход гласит, что вероятность — это степень уверенности: можно сказать, что вероятность дождя завтра составляет 70%, даже если завтрашний день наступит только один раз.

- Обе интерпретации используют одни и те же математические правила. Различие носит философский характер, но оно важно в машинном обучении. Частотные методы дают точечные оценки. Байесовские методы дают полные распределения параметров.

- **Пространство элементарных исходов** $S$ — это множество всех возможных результатов эксперимента. Подбрасывание монеты: $S = \{H, T\}$. Бросок игральной кости: $S = \{1, 2, 3, 4, 5, 6\}$.

- **Событие** — это любое подмножество пространства элементарных исходов. «Выпадение четного числа» — это событие $A = \{2, 4, 6\}$, которое является подмножеством $S$.

- Вероятность события, когда все исходы равновероятны, вычисляется простым подсчетом (из файла 01):

$$P(A) = \frac{|A|}{|S|} = \frac{\text{favourable outcomes}}{\text{total outcomes}}$$

- Для примера с четным числом: $P(\text{even}) = \frac{3}{6} = 0.5$.

![Диаграмма Венна, показывающая события A и B внутри пространства элементарных исходов S, с пересечением и дополнением](../images/venn_diagram.svg)

- **Дополнение** события $A$, обозначаемое как $A'$ или $A^c$, — это все элементы в $S$, которые не входят в $A$. Поскольку любой исход либо принадлежит $A$, либо нет:

$$P(A') = 1 - P(A)$$

- Часто проще работать с дополнениями. Вместо того чтобы подсчитывать все способы получить хотя бы одного орла при 5 подбрасываниях монеты, подсчитайте единственный способ не получить ни одного орла и вычтите его: $P(\text{at least one head}) = 1 - P(\text{all tails}) = 1 - (0.5)^5 = 0.969$.

- Два события являются **взаимоисключающими** (несовместными), если они не могут произойти одновременно: $A \cap B = \emptyset$. Выпадение 2 и выпадение 5 при одном броске кости — это взаимоисключающие события.

- **Правило сложения для взаимоисключающих событий** выглядит просто:

$$P(A \cup B) = P(A) + P(B) \quad \text{(if } A \cap B = \emptyset\text{)}$$

- Когда события могут пересекаться, необходимо использовать **общее правило сложения**, чтобы избежать двойного учета пересечения:

$$P(A \cup B) = P(A) + P(B) - P(A \cap B)$$

- Это отражает принцип включения-исключения из комбинаторики. Приведенная выше диаграмма Венна показывает почему: фиолетовая область (пересечение) учитывается один раз в $P(A)$ и еще раз в $P(B)$, поэтому мы вычитаем ее один раз.

- **Совместная вероятность** $P(A \cap B)$ — это вероятность того, что произойдут и $A$, и $B$. В колоде карт $P(\text{red} \cap \text{king}) = \frac{2}{52}$, так как существует 2 красных короля.

- **Маргинальная вероятность** — это вероятность отдельного события независимо от других. $P(\text{red}) = \frac{26}{52} = 0.5$ — это маргинальная вероятность. Если у вас есть совместное распределение двух переменных, маргинальное распределение получается путем суммирования (или интегрирования) по другой переменной.

- **Условная вероятность** отвечает на вопрос: при условии, что $B$ уже произошло, какова вероятность $A$? Мы сужаем пространство элементарных исходов с $S$ до $B$ и спрашиваем, какая часть $B$ также принадлежит $A$:

$$P(A | B) = \frac{P(A \cap B)}{P(B)}, \quad P(B) > 0$$

![Условная вероятность как сужение пространства элементарных исходов с S до B](../images/conditional_probability.svg)

- Пример: вы вытягиваете карту, и кто-то говорит вам, что она красная. Какова вероятность того, что это король? Всего 26 красных карт, и 2 из них — короли, поэтому $P(\text{king} | \text{red}) = \frac{2}{26} = \frac{1}{13}$. Используя формулу: $P(\text{king} \cap \text{red}) / P(\text{red}) = \frac{2/52}{26/52} = \frac{1}{13}$.

- Два события являются **независимыми**, если знание о том, что одно из них произошло, ничего не говорит о другом. Формально:

$$P(A \cap B) = P(A) \cdot P(B)$$

- Эквивалентно, $P(A | B) = P(A)$. Подбрасывание двух разных монет — это независимые события. Вытягивание двух карт без возвращения — события зависимые (первая карта меняет состав оставшихся).

- Независимость значительно упрощает расчеты. Для независимых событий совместные вероятности распадаются на произведения, что делает вычисления доступными. Многие модели машинного обучения предполагают независимость между признаками (например, наивный байесовский классификатор) именно из-за этого упрощения.

- **Правило умножения** для любых двух событий является переформулировкой формулы условной вероятности:

$$P(A \cap B) = P(A | B) \cdot P(B) = P(B | A) \cdot P(A)$$

- Для независимых событий это упрощается до $P(A \cap B) = P(A) \cdot P(B)$, так как условная вероятность равна маргинальной.

- **Теорема Байеса** — один из важнейших результатов в теории вероятностей и основа байесовского машинного обучения. Она позволяет обратить направление условной вероятности:

$$P(A | B) = \frac{P(B | A) \cdot P(A)}{P(B)}$$

- Теорема непосредственно следует из записи $P(A \cap B)$ двумя способами: $P(B|A) \cdot P(A) = P(A|B) \cdot P(B)$, с последующим выражением $P(A|B)$.

![Компоненты теоремы Байеса: апостериорное распределение, правдоподобие, априорное распределение и свидетельство](../images/bayes_components.svg)

- Каждый компонент имеет свое название:
    - **Априорное распределение** $P(A)$: ваше начальное убеждение до получения свидетельств
    - **Правдоподобие** $P(B|A)$: насколько вероятно свидетельство при условии, что $A$ истинно
    - **Свидетельство** $P(B)$: полная вероятность получения свидетельства, выступает в роли нормализатора
    - **Апостериорное распределение** $P(A|B)$: ваше обновленное убеждение после получения свидетельства

- Давайте разберем классический пример медицинской диагностики. Предположим, болезнь поражает 1% населения. Тест на болезнь точен на 95%: он правильно определяет 95% больных людей (чувствительность) и правильно определяет 90% здоровых людей (специфичность).

- Ваш тест оказался положительным. Какова вероятность того, что вы действительно больны?

- Пусть $D$ = наличие заболевания, $+$ = положительный результат теста.
    - Априорное распределение: $P(D) = 0.01$
    - Правдоподобие: $P(+ | D) = 0.95$
    - Доля ложноположительных результатов: $P(+ | D') = 0.10$

- Нам нужно найти $P(+)$. Согласно формуле полной вероятности:

$$P(+) = P(+ | D) \cdot P(D) + P(+ | D') \cdot P(D')$$
$$= 0.95 \times 0.01 + 0.10 \times 0.99 = 0.0095 + 0.099 = 0.1085$$

- Теперь применим теорему Байеса:

$$P(D | +) = \frac{P(+ | D) \cdot P(D)}{P(+)} = \frac{0.95 \times 0.01}{0.1085} \approx 0.088$$

- Несмотря на то, что тест «точен на 95%», положительный результат дает лишь около 8,8% вероятности наличия заболевания. Априорное распределение имеет огромное значение. Поскольку заболевание встречается редко, большинство положительных результатов являются ложноположительными. Это важнейший вывод для любой задачи классификации в машинном обучении: когда классы несбалансированы, одна лишь точность (accuracy) вводит в заблуждение.

- **Формула полной вероятности** разбивает пространство элементарных исходов на непересекающиеся и исчерпывающие события $B_1, B_2, \ldots, B_n$ и выражает любое событие $A$ как:

$$P(A) = \sum_{i=1}^{n} P(A | B_i) \cdot P(B_i)$$

- Это именно то, что мы использовали для вычисления $P(+)$ в медицинском примере: мы разделили популяцию на «имеющих заболевание» и «не имеющих заболевания».

- **Цепное правило теории вероятностей** обобщает правило умножения на любое количество событий:

$$P(A_1 \cap A_2 \cap \cdots \cap A_n) = P(A_1) \cdot P(A_2 | A_1) \cdot P(A_3 | A_1 \cap A_2) \cdots P(A_n | A_1 \cap \cdots \cap A_{n-1})$$

- Каждый множитель обусловлен всем, что произошло ранее. Это основа авторегрессионных языковых моделей: вероятность предложения равна произведению вероятностей каждого слова при условии всех предыдущих слов.

- **Условная независимость** означает, что два события независимы при условии третьего. $A$ и $B$ условно независимы при условии $C$, если:

$$P(A \cap B | C) = P(A | C) \cdot P(B | C)$$

- События могут быть зависимы маргинально, но условно независимы, и наоборот. Например, результаты экзаменов двух студентов могут быть коррелированы (оба зависят от сложности экзамена), но при фиксированной сложности экзамена их результаты независимы.

- Условная независимость — ключевое допущение графических моделей, таких как байесовские сети. Оно позволяет факторизовать сложные совместные распределения на управляемые части, делая инференс вычислительно осуществимым.

1. Смоделируйте задачу медицинской диагностики. Создайте популяцию из 100 000 человек, примените показатели распространенности заболевания и точности теста, а затем проверьте, что теорема Байеса дает правильное апостериорное распределение.
```python
import jax
import jax.numpy as jnp

key = jax.random.PRNGKey(42)
n = 100_000

# Generate population
k1, k2 = jax.random.split(key)
has_disease = jax.random.bernoulli(k1, p=0.01, shape=(n,))

# Generate test results
k3, k4 = jax.random.split(k2)
# Sensitivity: P(+|D) = 0.95, Specificity: P(-|D') = 0.90
test_positive = jnp.where(
    has_disease,
    jax.random.bernoulli(k3, p=0.95, shape=(n,)),
    jax.random.bernoulli(k4, p=0.10, shape=(n,))
)

# Among those who tested positive, what fraction actually has the disease?
positives = test_positive.astype(bool)
true_positives = (has_disease & positives).sum()
total_positives = positives.sum()

print(f"Total positive tests: {total_positives}")
print(f"True positives: {true_positives}")
print(f"P(Disease | Positive) = {true_positives / total_positives:.4f}")
print(f"Bayes' formula:         {0.95 * 0.01 / 0.1085:.4f}")
```

2. Проверьте правило сложения с помощью симуляции. Сгенерируйте случайные события A и B с известными вероятностями и пересечением, затем проверьте, что $P(A \cup B) = P(A) + P(B) - P(A \cap B)$.
```python
import jax
import jax.numpy as jnp

key = jax.random.PRNGKey(0)
n = 200_000
k1, k2 = jax.random.split(key)

# Events: A = value < 0.4, B = value < 0.6 (overlap at < 0.4)
vals_a = jax.random.uniform(k1, shape=(n,))
vals_b = jax.random.uniform(k2, shape=(n,))

A = vals_a < 0.4
B = vals_b < 0.6

p_a = A.mean()
p_b = B.mean()
p_a_and_b = (A & B).mean()
p_a_or_b = (A | B).mean()

print(f"P(A) = {p_a:.4f}")
print(f"P(B) = {p_b:.4f}")
print(f"P(A ∩ B) = {p_a_and_b:.4f}")
print(f"P(A ∪ B) simulated = {p_a_or_b:.4f}")
print(f"P(A) + P(B) - P(A∩B) = {p_a + p_b - p_a_and_b:.4f}")
```

3. Продемонстрируйте, что условная вероятность меняется при наличии свидетельств. Смоделируйте бросок двух игральных костей и вычислите $P(\text{sum} = 7)$, а затем $P(\text{sum} = 7 | \text{first die} = 3)$.
```python
import jax
import jax.numpy as jnp

key = jax.random.PRNGKey(1)
n = 500_000
k1, k2 = jax.random.split(key)

d1 = jax.random.randint(k1, shape=(n,), minval=1, maxval=7)
d2 = jax.random.randint(k2, shape=(n,), minval=1, maxval=7)
total = d1 + d2

# Unconditional
p_sum7 = (total == 7).mean()
print(f"P(sum=7) = {p_sum7:.4f} (exact: {6/36:.4f})")

# Conditional on first die = 3
mask = d1 == 3
p_sum7_given_d1_3 = (total[mask] == 7).mean()
print(f"P(sum=7 | d1=3) = {p_sum7_given_d1_3:.4f} (exact: {1/6:.4f})")
```

4. Реализуйте теорему Байеса как функцию и используйте ее для итеративного обновления убеждений. Начните с равномерного априорного распределения смещения монеты и обновляйте его после наблюдения каждого подбрасывания.
```python
import jax.numpy as jnp
import matplotlib.pyplot as plt

def bayes_update(prior, likelihood):
    """Multiply prior by likelihood and normalise."""
    posterior = prior * likelihood
    return posterior / posterior.sum()

# Discretise possible bias values
theta = jnp.linspace(0, 1, 200)
prior = jnp.ones_like(theta)  # uniform prior
prior = prior / prior.sum()

# Observed flips: 1=heads, 0=tails
flips = [1, 1, 0, 1, 1, 1, 0, 1, 0, 1]

plt.figure(figsize=(10, 5))
plt.plot(theta, prior, "--", color="#999", label="prior")

for i, flip in enumerate(flips):
    likelihood = theta if flip == 1 else (1 - theta)
    prior = bayes_update(prior, likelihood)
    if i in [0, 2, 4, 9]:
        plt.plot(theta, prior, label=f"after {i+1} flips", linewidth=2)

plt.xlabel("Coin bias θ")
plt.ylabel("Belief (normalised)")
plt.title("Bayesian updating: belief about coin bias")
plt.legend()
plt.grid(alpha=0.3)
plt.show()
```
