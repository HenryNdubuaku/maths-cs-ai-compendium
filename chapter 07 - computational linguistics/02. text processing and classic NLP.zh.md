# 文本处理与经典 NLP

*文本处理将原始字符转化为模型可消费的结构化表示。本文件涵盖分词（词级、子词级、BPE、WordPiece）、文本归一化、编辑距离、TF-IDF、n-gram 语言模型、词性标注、NER 和情感分析——这条经典 NLP 流水线至今仍是现代系统的基石。*

- 原始文本是杂乱的。在任何 NLP 模型能够处理语言之前，文本必须被清洗、归一化，并转化为结构化表示。本文件涵盖从原始字符到模型可消费特征的流水线，以及深度学习之前占据主导地位的经典 NLP 算法。

- **文本归一化（text normalisation）**将原始文本转化为规范形式。目标是减少无关的变体，使 "Hello"、"hello"、"HELLO" 和 "héllo" 得到恰当处理。

- **大小写折叠（case folding）**将文本转为小写。它把 "The" 和 "the" 合并为一个 token。这对多数任务有帮助，但在某些情况下会破坏有用信息："US"（国家）与 "us"（代词），或 "Apple"（公司）与 "apple"（水果）。

- **Unicode 归一化**处理同一字符可有多种编码方式这一事实。字符 "é" 可以是单个码位（U+00E9），也可以是基础字符 "e" 加组合重音符（U+0065 + U+0301）。NFC 归一化将它们组合为一个码位；NFD 将它们分解。若不归一化，两个外观相同的字符串可能无法匹配。

- **编辑距离（edit distance）**衡量两个字符串的差异程度。**Levenshtein 距离**计算把一个字符串变成另一个所需的最少单字符插入、删除和替换次数。"kitten" → "sitting" 的编辑距离为 3（k→s、e→i、插入 g）。

- 编辑距离用动态规划计算（我们在算法章节复习）。定义 $D[i][j]$ 为字符串 $s$ 的前 $i$ 个字符与字符串 $t$ 的前 $j$ 个字符之间的距离：

```math
D[i][j] = \begin{cases} j & \text{if } i = 0 \\ i & \text{if } j = 0 \\ D[i{-}1][j{-}1] & \text{if } s[i] = t[j] \\ 1 + \min(D[i{-}1][j], \; D[i][j{-}1], \; D[i{-}1][j{-}1]) & \text{otherwise} \end{cases}
```

- 编辑距离支撑拼写纠错、模糊匹配和 DNA 序列比对。在 NLP 中，它用于处理拼写错误并查找相似词。

- **分词（tokenisation）**将文本切分为模型可处理的离散单位（token）。这是第一个、也可以说是最重要的预处理步骤。分词策略的选择深刻影响模型行为。

- **空白分词（whitespace tokenisation）**按空格切分。简单但天真："New York" 会变成两个 token，"don't" 是一个 token（或根据切分器被切为 "don" 和 "'t"），而中文和日文等语言词之间根本没有空格。

- **基于规则的分词（rule-based tokenisation）**使用手工设计的模式（正则表达式）来处理缩略、标点和特殊情形。"I'm" → "I" + "'m"，"U.S.A." 保持为一个 token。每种语言都需要自己的规则，这很费人力。

- **子词分词（subword tokenisation）**是现代的解决方案。它不在词边界处切分，而是从数据中学习频繁子词单位的词表。它优雅地处理未登录词：如果 "unhappiness" 不在词表中，它可能被切为 "un" + "happi" + "ness"，保留形态结构。

![词级、字符级和子词级分词对 "unhappiness" 和 "transformers" 的比较](../images/tokenisation_comparison.svg)

- **Byte-Pair Encoding（BPE）**以单个字符作为初始词表。它反复找到最频繁的相邻字符对并将它们合并为新 token。经过足够多次合并后，常见词成为单个 token，罕见词被切为频繁出现的子词片段。

- BPE 算法：
    1. 用训练语料中所有单个字符初始化词表
    2. 统计每个相邻 token 对的频率
    3. 将最频繁的对合并为一个新 token
    4. 重复步骤 2-3 直到达到期望的合并次数（词表大小）

- 例如，从 "l o w"（出现 5 次）、"l o w e r"（2 次）、"n e w e s t"（6 次）开始：最频繁的对可能是 "e s" → 合并为 "es"。然后 "es t" → "est"。然后 "n e w" → "new"。最终词表中既有完整词也有子词片段。

- **WordPiece**（BERT 使用）类似 BPE，但基于似然而非频率选择合并。它合并能最大化训练数据语言模型似然的那一对。不是词首的子词 token 以 "##" 为前缀（如 "playing" → "play" + "##ing"）。

- **Unigram**（SentencePiece 使用）采取相反的方法：从一个大词表开始，迭代地删除那些被移除后对训练数据似然损害最小的 token。最终词表是能最好解释语料的子词单位集合。

- **SentencePiece** 是一个与语言无关的分词库，把输入当作原始字节流处理（不基于空格预分词）。这使其适用于任何语言，包括没有空格的语言。它实现了 BPE 和 Unigram 两种算法。

- 词表大小是一个关键超参数。典型选择在 30,000 到 100,000 个 token 之间。更大的词表意味着每个序列的 token 更少（更高效），但 embedding 表更大。更小的词表意味着更多的子词切分和更长的序列。

- 两种技术都将词还原为基础形式，但方法不同。

- **词干提取（stemming）**用粗糙的规则砍掉后缀。Porter 词干器把 "running" 还原为 "run"，"happiness" 还原为 "happi"，"studies" 还原为 "studi"。它快但不精确："university" 和 "universe" 都被还原为 "univers"，尽管二者无关。

- **词形还原（lemmatisation）**使用词汇表和形态分析找到真正的字典形式（lemma）。"Running" → "run"，"better" → "good"，"mice" → "mouse"。它需要知道词性："saw" 作为动词还原为 "see"，但作为名词仍是 "saw"。

- 现代子词分词在神经 NLP 中已基本取代词干提取和词形还原，但它们在信息检索以及使用较小模型或有限数据时仍然有用。

- **词性标注（Part-of-Speech, POS tagging）**为每个词赋予一个语法类别：名词、动词、形容词、限定词等。这是最古老的 NLP 任务之一，是句法分析的基础。

- Penn Treebank 标签集是英语最常用的，共 36 个标签（NN 为单数名词，NNS 为复数名词，VB 为动词原形，VBD 为过去时，JJ 为形容词等）。

- 词性标注棘手，因为许多词有歧义。"Book" 可以是名词（"the book"）或动词（"book a flight"）。"Run" 跨越多种词性有数十种意义。语境至关重要。

- 早期标注器使用第 05 章的**隐马尔可夫模型（HMM）**。隐藏状态是词性标签，观测是词。转移概率刻画标签序列（限定词后很可能跟名词或形容词），发射概率刻画哪些词以哪些标签出现。Viterbi 算法找到最可能的标签序列。

- 词性标注的 HMM 模型：

$$\hat{t}_{1:n} = \arg\max_{t_{1:n}} \prod_{i=1}^{n} P(w_i \mid t_i) \cdot P(t_i \mid t_{i-1})$$

- 现代词性标注器使用神经网络（双向 LSTM 或 transformer）并在英语上达到超过 97% 的准确率，接近人类水平。

- **命名实体识别（Named Entity Recognition, NER）**识别并分类文本中的专有名词和其他特定实体：人名、组织、地点、日期、货币金额等。

- 在 "Apple CEO Tim Cook announced the event in Cupertino on Monday," 中，NER 系统应识别：Apple（ORG）、Tim Cook（PER）、Cupertino（LOC）、Monday（DATE）。

- NER 通常被构建为使用 **BIO 标注**（又称 IOB 标注）的**序列标注（sequence labelling）**。每个 token 获得一个标签：
    - **B-TYPE**：类型为 TYPE 的实体的开始
    - **I-TYPE**：类型为 TYPE 的实体的内部（延续）
    - **O**：不在任何实体内

- "Tim Cook visited New York" 变为：Tim/B-PER Cook/I-PER visited/O New/B-LOC York/I-LOC。B 标签标记新实体的开始，这在两个同类实体相邻时很重要。

![带颜色编码的 BIO 标签的句子：B-PER（红色）、I-PER（红色）、O（灰色）、B-LOC（蓝色）、I-LOC（蓝色）](../images/bio_tagging.svg)

- 经典 NER 使用第 05 章的**条件随机场（Conditional Random Fields, CRF）**，它对给定输入下整个标签序列的条件概率进行建模。与生成式的 HMM（$P(x, y)$）不同，CRF 是判别式的，直接建模 $P(y \mid x)$。线性链 CRF 定义为：

$$P(y_{1:n} \mid x_{1:n}) = \frac{1}{Z(x)} \exp\!\left(\sum_{i=1}^{n} \left[\sum_k \lambda_k f_k(y_i, x, i) + \sum_j \mu_j g_j(y_i, y_{i-1}, x, i)\right]\right)$$

- 这里 $f_k$ 是**发射特征**（在位置 $i$ 给定输入下标签 $y_i$ 的可能性），$g_j$ 是**转移特征**（给定前一标签 $y_{i-1}$ 下标签 $y_i$ 的可能性）。

- 配分函数 $Z(x) = \sum_{y'} \exp(\ldots)$ 对所有可能的标签序列求和以归一化分布。训练最大化条件对数似然，这需要用前向算法高效地计算 $Z(x)$（第 05 章）。

- 相对独立分类每个 token 的关键优势：CRF 的转移特征强制结构约束（如 I-PER 只能跟在 B-PER 或 I-PER 后，绝不能出现在 O 之后）。

- 现代 NER 在神经编码器（BiLSTM-CRF 或 BERT-CRF）之上堆叠一个 CRF，其中神经网络产生发射分数，CRF 层学习转移结构。

- **句法分析（syntactic parsing）**将句子转化为它的句法结构，要么是成分树，要么是依存树（均来自文件 01）。

- **CYK 算法**（Cocke-Younger-Kasami）用动态规划对上下文无关文法进行句法分析。

- 它要求文法处于 **Chomsky 范式**（每条规则的右侧要么是两个非终结符，要么是一个终结符）。它自底向上地填充一个三角形表：单元格代表句子的跨度，每个单元格存储可以生成该跨度的非终结符。

- CYK 的运行时间为 $O(n^3 \cdot |G|)$，其中 $n$ 是句子长度，$|G|$ 是文法规模。这是精确的但对大文法来说较慢。

- **移进-归约分析（shift-reduce parsing）**从左到右处理句子，维护一个栈。每一步，它要么**移进**（把下一个词压入栈中），要么**归约**（从栈中弹出若干元素并用一个短语替换）。一个训练好的分类器在每步决定动作。其运行时间为 $O(n)$，比 CYK 快得多。

- **依存分析（dependency parsing）**在实践中现在比成分分析更常见。基于转移的依存分析器（如移进-归约）和基于图的分析器（对所有可能的边打分并寻找最大生成树）是两种主要方法。使用 BiLSTM 或 transformer 的神经依存分析器取得了最先进的结果。

- 在 embedding 出现之前，NLP 用简单的计数方法将文档表示为向量。

- **词袋模型（bag-of-words, BoW）**将文档表示为词计数向量，完全忽略词序。如果词表有 $V$ 个词，每个文档是 $\mathbb{R}^V$ 中的一个向量（联系回第 01 章的向量空间）。词 $w$ 对应的分量是 $w$ 在文档中出现的次数。

![词袋模型：文档被转换为词计数表，然后转换为 R^V 中每个词表词一项的稀疏向量](../images/bag_of_words.svg)

- BoW 简单但对文档分类和垃圾邮件过滤等任务惊人地有效。它的主要弱点是把每个词视为同等重要："the" 和 "revolutionary" 得到相同权重。

- **TF-IDF**（Term Frequency-Inverse Document Frequency，词频-逆文档频率）通过根据词的信息量加权来修复这一点。在一个文档中频繁出现但在整个语料中罕见的词，可能对该文档很重要。

$$\text{TF-IDF}(t, d) = \text{TF}(t, d) \times \text{IDF}(t)$$

- **词频** $\text{TF}(t, d)$ 通常是词 $t$ 在文档 $d$ 中的原始计数（或其对数：$1 + \log(\text{count})$）。

- **逆文档频率** $\text{IDF}(t) = \log\frac{N}{|\{d : t \in d\}|}$，其中 $N$ 是文档总数。在每个文档中都出现的词（如 "the"）的 IDF 接近 0。罕见词的 IDF 高。

- TF-IDF 向量可以用余弦相似度（来自第 01 章）比较以衡量文档相似度。这是经典信息检索和搜索引擎的基础。

- **语言模型（language model）**为一个词序列赋予概率。它回答：这句话有多可能？语言模型是机器翻译、语音识别、拼写纠错和文本生成的核心。

- 一个句子 $w_1, w_2, \ldots, w_n$ 的概率，由概率的链式法则（第 05 章）给出：

$$P(w_1, w_2, \ldots, w_n) = \prod_{i=1}^{n} P(w_i \mid w_1, \ldots, w_{i-1})$$

- 这是精确的但不切实际：你需要为每一种可能的历史存储概率。**马尔可夫假设（Markov assumption，第 05 章）**把历史截断为最后 $k-1$ 个词，得到 **n-gram 模型**（其中 $n = k$）。

- **bigram 模型**（$n = 2$）只以前一个词为条件：

$$P(w_i \mid w_1, \ldots, w_{i-1}) \approx P(w_i \mid w_{i-1})$$

- **trigram 模型**（$n = 3$）以前两个词为条件。N-gram 概率通过在语料中计数来估计：

$$P(w_i \mid w_{i-1}) = \frac{\text{count}(w_{i-1}, w_i)}{\text{count}(w_{i-1})}$$

- **困惑度（perplexity）**衡量语言模型对测试集的预测好坏。它是测试集概率的倒数，按词数归一化：

$$\text{PPL} = P(w_1, \ldots, w_N)^{-1/N} = \exp\!\left(-\frac{1}{N} \sum_{i=1}^{N} \log P(w_i \mid w_{<i})\right)$$

- 困惑度越低意味着模型对测试数据越不"惊讶"，因此越好。一个在 10,000 词词表上赋予均匀概率的模型困惑度为 10,000。一个好的 bigram 模型困惑度约为 200。现代神经语言模型的困惑度低于 20。

- 注意困惑度就是交叉熵（第 05 章信息论）取指数。训练中最小化交叉熵损失直接最小化了困惑度。

- **平滑（smoothing）**处理零概率问题：如果某个 n-gram 从未在训练中出现，模型赋予它概率 0，这使整个句子的概率变为 0。**Laplace 平滑（加 1 平滑）**给每个 n-gram 加一个小计数：

$$P_{\text{Laplace}}(w_i \mid w_{i-1}) = \frac{\text{count}(w_{i-1}, w_i) + 1}{\text{count}(w_{i-1}) + V}$$

- 这对大词表来说过于激进（它从已观测 n-gram 中偷走太多概率）。**Kneser-Ney 平滑**是 n-gram 模型的黄金标准。它结合两个想法：绝对折扣和用于回退的延续概率。

- 首先，**绝对折扣（absolute discounting）**从每个已观测计数中减去一个固定折扣 $d$（通常 $d \approx 0.75$），而不是加伪计数。释放出的概率质量被重新分配给未见的 n-gram。插值形式为：

$$P_{\text{KN}}(w_i \mid w_{i-1}) = \frac{\max(\text{count}(w_{i-1}, w_i) - d, \; 0)}{\text{count}(w_{i-1})} + \lambda(w_{i-1}) \cdot P_{\text{cont}}(w_i)$$

- 其中 $\lambda(w_{i-1})$ 是分配被折扣质量的归一化常数。关键创新是**延续概率（continuation probability）** $P_{\text{cont}}(w_i)$，它衡量 $w_i$ 出现在多少个不同语境中，而不是它总体出现得多频繁：

$$P_{\text{cont}}(w_i) = \frac{|\{w' : \text{count}(w', w_i) > 0\}|}{|\{(w', w'') : \text{count}(w', w'') > 0\}|}$$

- 分子计算语料中有多少个不同的词出现在 $w_i$ 之前。像 "Francisco" 这样的词出现在少数语境中（几乎总是跟在 "San" 后），所以即使 "San Francisco" 非常频繁，"Francisco" 的延续概率也很低，不会在其他语境中被错误地预测。

- 反之，像 "the" 这样的常见词出现在许多不同的词之后，延续概率高。这捕捉了这样一个直觉：一个词的多用性比它的原始频率对回退估计更重要。

- N-gram 模型数十年来都是最先进的方法。它们快速、可解释、无需训练（只需计数）。但它们难以处理长距离依赖（"The keys that I left on the table **are** missing" 需要知道主语 "keys" 是复数，而这离动词很远）。神经语言模型从 RNN 开始，最终以 transformer 达到顶峰，解决了这一局限。

1. 用动态规划实现 Levenshtein 编辑距离。在词对上测试，并用于简单的拼写纠错。
```python
import jax.numpy as jnp

def edit_distance(s, t):
    """Compute Levenshtein edit distance using DP."""
    m, n = len(s), len(t)
    D = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(m + 1):
        D[i][0] = i
    for j in range(n + 1):
        D[0][j] = j

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if s[i-1] == t[j-1]:
                D[i][j] = D[i-1][j-1]
            else:
                D[i][j] = 1 + min(D[i-1][j], D[i][j-1], D[i-1][j-1])

    return D[m][n]

# Test
pairs = [("kitten", "sitting"), ("sunday", "saturday"), ("hello", "hallo")]
for s, t in pairs:
    print(f"d('{s}', '{t}') = {edit_distance(s, t)}")

# Simple spelling correction
dictionary = ["the", "their", "there", "then", "than", "this", "that", "these", "those"]
misspelled = "thier"
corrections = sorted(dictionary, key=lambda w: edit_distance(misspelled, w))
print(f"\nClosest to '{misspelled}': {corrections[:3]}")
```

2. 从头实现 BPE 分词。从字符级 token 开始，迭代地合并最频繁的对。
```python
from collections import Counter

def get_pairs(corpus):
    """Count adjacent token pairs across all words."""
    pairs = Counter()
    for word, freq in corpus.items():
        symbols = word.split()
        for i in range(len(symbols) - 1):
            pairs[(symbols[i], symbols[i+1])] += freq
    return pairs

def merge_pair(pair, corpus):
    """Merge all occurrences of a pair in the corpus."""
    new_corpus = {}
    bigram = ' '.join(pair)
    replacement = ''.join(pair)
    for word, freq in corpus.items():
        new_word = word.replace(bigram, replacement)
        new_corpus[new_word] = freq
    return new_corpus

# Training corpus with word frequencies
text = "low low low low low lower lower newest newest newest newest newest newest"
word_freqs = Counter(text.split())
# Initialise: split each word into characters with end-of-word marker
corpus = {' '.join(word) + ' _': freq for word, freq in word_freqs.items()}

print("Initial corpus:")
for word, freq in corpus.items():
    print(f"  {word}: {freq}")

# Run BPE for 10 merges
for i in range(10):
    pairs = get_pairs(corpus)
    if not pairs:
        break
    best_pair = max(pairs, key=pairs.get)
    corpus = merge_pair(best_pair, corpus)
    print(f"\nMerge {i+1}: {best_pair} (freq={pairs[best_pair]})")
    for word, freq in corpus.items():
        print(f"  {word}: {freq}")
```

3. 构建一个 bigram 语言模型并计算测试句子的困惑度。用 Laplace 平滑做实验。
```python
from collections import Counter, defaultdict
import math

# Training corpus
train = """the cat sat on the mat . the dog chased the cat .
the cat ran from the dog . a dog sat on a mat .""".split()

# Count bigrams and unigrams
bigrams = Counter(zip(train[:-1], train[1:]))
unigrams = Counter(train)
vocab_size = len(set(train))

def bigram_prob(w2, w1, alpha=0):
    """P(w2 | w1) with optional Laplace smoothing."""
    return (bigrams[(w1, w2)] + alpha) / (unigrams[w1] + alpha * vocab_size)

# Compute perplexity
test = "the cat sat on a mat .".split()

for alpha in [0, 1, 0.1]:
    log_prob = 0
    for w1, w2 in zip(test[:-1], test[1:]):
        p = bigram_prob(w2, w1, alpha=alpha)
        if p > 0:
            log_prob += math.log(p)
        else:
            log_prob += float('-inf')

    ppl = math.exp(-log_prob / (len(test) - 1)) if log_prob > float('-inf') else float('inf')
    print(f"Smoothing α={alpha}: perplexity = {ppl:.2f}")
```

4. 从头实现 TF-IDF，并用余弦相似度找出与查询最相似的文档。
```python
import jax.numpy as jnp
import math
from collections import Counter

documents = [
    "the cat sat on the mat",
    "the dog chased the cat around the park",
    "a mat was placed on the floor by the door",
    "the quick brown fox jumped over the lazy dog",
]

# Build vocabulary
vocab = sorted(set(word for doc in documents for word in doc.split()))
word_to_idx = {w: i for i, w in enumerate(vocab)}
V = len(vocab)
N = len(documents)

# Compute TF-IDF matrix
doc_freq = Counter()
for doc in documents:
    for word in set(doc.split()):
        doc_freq[word] += 1

tfidf_matrix = jnp.zeros((N, V))
for i, doc in enumerate(documents):
    word_counts = Counter(doc.split())
    for word, count in word_counts.items():
        tf = 1 + math.log(count)
        idf = math.log(N / doc_freq[word])
        j = word_to_idx[word]
        tfidf_matrix = tfidf_matrix.at[i, j].set(tf * idf)

# Query
query = "cat on the mat"
query_vec = jnp.zeros(V)
query_counts = Counter(query.split())
for word, count in query_counts.items():
    if word in word_to_idx:
        tf = 1 + math.log(count)
        idf = math.log(N / doc_freq.get(word, 1))
        query_vec = query_vec.at[word_to_idx[word]].set(tf * idf)

# Cosine similarity (from chapter 01)
def cosine_sim(a, b):
    return jnp.dot(a, b) / (jnp.linalg.norm(a) * jnp.linalg.norm(b) + 1e-8)

print(f"Query: '{query}'\n")
for i, doc in enumerate(documents):
    sim = cosine_sim(query_vec, tfidf_matrix[i])
    print(f"  Doc {i} (sim={sim:.3f}): '{doc}'")
```
