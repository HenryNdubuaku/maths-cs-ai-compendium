# 语言学基础

*语言学为 NLP 系统提供了其隐式学习并加以利用的结构化词汇。本文件涵盖形态学、句法学、语义学、语用学、音系学、成分分析与依存分析，以及分布式假设——这门关于人类语言的科学为 AI 中的分词、语法和意义奠定了基础。*

- 在构建能够理解或生成语言的系统之前，我们需要先理解语言本身是如何运作的。

- 语言学是对语言的科学研究，它提供了 NLP 不断借用的概念词汇。

- 即使是从原始数据中学习语言的现代神经模型，也会隐式地重新发现语言学家们数十年来所记录的许多结构。

- 语言在每一个层级上都有结构：构成词的语音、构成词的语素、将词组合成句的规则、这些句子所承载的意义，以及语境如何塑造理解。我们将自底向上逐一梳理每个层级。

- **形态学（morphology）**研究词的内部结构。词并不是原子单位，它们由更小的、有意义的单位——**语素（morpheme）**——构成。

- 单词 "unhappiness" 包含三个语素："un-"（意为"not" 的前缀）、"happy"（词根）和 "-ness"（将形容词转化为名词的后缀）。每个语素都对意义有所贡献。

- **词根（root，或词干 stem）**是承载核心意义的核心语素。"Happy"、"run"、"compute" 都是词根。

- **词缀（affix）**是附着在词根上以修饰其意义的语素。

- 英语有**前缀（prefix）**（位于词根之前：un-、re-、pre-）和**后缀（suffix）**（位于词根之后：-ing、-ed、-tion）。某些语言还有中缀（插入词根内部）和环缀（包裹词根）。

![语素树："unhappiness" 被拆分为前缀 "un"、词根 "happy"、后缀 "ness"](../images/morpheme_tree.svg)

- 形态学过程有两种。**屈折（inflection）**在不改变核心意义或词性的情况下改变词的语法属性："run" 变为 "runs"（第三人称）、"running"（进行时）、"ran"（过去时）。这个词仍然是动词，意义相同。

- **派生（derivation）**创造新词，常常改变词性："happy"（形容词）变为 "happiness"（名词），"compute"（动词）变为 "computation"（名词）再变为 "computational"（形容词）。每次派生都会改变意义和语法类别。

- 各种语言的形态复杂度差异巨大。英语相对而言是**分析语（analytic）**（每个词的语素较少，依赖词序）。

- 土耳其语和芬兰语是**黏着语（agglutinative）**（词中可以串接许多语素）。阿拉伯语和希伯来语使用**模板式（templatic）**形态学（词根是辅音骨架，如 k-t-b 表示 "write"，通过插入不同的元音模式创造不同的词：kitab "book"、kataba "he wrote"、maktub "written"）。

- 形态学对 NLP 至关重要，因为它影响分词。词级分词器会把 "run"、"runs"、"running" 和 "ran" 当作四个互不相关的符号。

- 一个具备形态学意识的系统能识别它们共享同一个词根。子词分词（BPE、WordPiece，将在文件 02 中讲解）是形态分析的统计近似。

- **句法学（syntax）**研究词如何组合成短语和句子。每种语言都有支配词序和结构的规则；违反这些规则就会产生无意义的话。

- "The cat sat on the mat" 是合乎语法的英语；"Mat the on sat cat the" 则不是。

- 描述句法结构主要有两种框架。

- **短语结构语法（phrase structure grammar，又称成分语法 constituency grammar）**认为句子由短语嵌套短语构成。一个句子（S）由名词短语（NP）和动词短语（VP）组成。

- 名词短语可以是限定词（Det）加上名词（N）。动词短语可以是动词（V）加上名词短语。这些规则构成一棵树：

!["the cat sat on the mat" 的成分树：S 分为 NP 和 VP，NP 分为 Det "the" 和 N "cat"，VP 分为 V "sat" 和 PP，PP 分为 P "on" 和 NP](../images/constituency_tree.svg)

- 这棵树称为**成分树（constituency tree，或分析树 parse tree）**。每个内部节点是一个短语类型，每个叶子是一个词。树刻画了层级分组："on the mat" 是一个单位（介词短语），"sat on the mat" 是一个单位（动词短语），而整体是一个句子。

- **上下文无关文法（CFG）**将这些规则形式化。它由一组产生式规则组成，每条规则形如 $A \to \alpha$，其中 $A$ 是非终结符（如 NP 或 VP 这样的短语类型），$\alpha$ 是终结符（词）和非终结符的序列。例如：

```
S  → NP VP
NP → Det N
NP → Det N PP
VP → V NP
VP → V PP
PP → P NP
Det → "the" | "a"
N  → "cat" | "mat" | "dog"
V  → "sat" | "chased"
P  → "on" | "under"
```

- 从 S 出发反复应用规则，可以生成该文法允许的所有句子。分析是逆向过程：给定一个句子，找出产生它的分析树（或树集）。一个句子若有多种合法的分析树，就是**句法歧义（syntactically ambiguous）**的。"I saw the man with the telescope" 有两种分析：我用望远镜看到了那个男人，或我看到了一个拿着望远镜的男人。

- **依存语法（dependency grammar）**采取不同视角。它不关注短语嵌套，而是描述词与词之间的直接关系。句子中每个词都依存于唯一一个其他词（它的**中心词 head**），但句子的根除外。结果是一棵**依存树（dependency tree）**，其边标注了语法关系（主语、宾语、修饰语等）。

!["the cat sat on the mat" 的依存树：箭头从 "sat" 指向 "cat"（nsubj）和 "on"（prep），从 "on" 指向 "mat"（pobj），从 "cat" 指向 "the"（det），从 "mat" 指向 "the"（det）](../images/dependency_tree.svg)

- 在依存视角中，"sat" 是根。"Cat" 依存于 "sat"，作其主语（nsubj）。"On" 依存于 "sat"，作介词修饰语。"Mat" 依存于 "on"，作介词宾语。每个词都挂在唯一一个中心词下，从而构成一棵树。

- 依存语法已成为现代 NLP 的主流框架，因为依存树更容易用统计分析器生成，而且这些关系更直接地映射到语义角色（谁对谁做了什么）。

- **配价（valency）**描述一个动词需要多少论元。"Sleep" 是**不及物（intransitive）**的（一个论元：睡觉者）。"Eat" 是**及物（transitive）**的（两个：吃者和被吃者）。"Give" 是**双及物（ditransitive）**的（三个：给予者、被给之物和接受者）。了解动词的配价可以约束哪些分析树是合法的。

- **语义学（semantics）**研究意义。句法告诉你句子如何构造；语义告诉你它意味着什么。

- **词汇语义学（lexical semantics）**关注单个词的意义。词之间以系统化的方式相互关联：

    - **同义关系（synonymy）**：（几乎）同义的词。"Big" 和 "large" 是同义词。真正完美的同义关系很罕见；几乎总存在内涵或用法上的细微差异。
    - **反义关系（antonymy）**：意义相反的词。"Hot" 与 "cold"、"buy" 与 "sell"。
    - **上下位关系（hypernymy/hyponymy）**："is-a" 关系。"Dog" 是 "animal" 的下位词（dog 是一种 animal）。"Animal" 是 "dog" 的上位词。它们构成分类层级。
    - **整体部分关系（meronymy）**："part-of" 关系。"Wheel" 是 "car" 的部分。
    - **多义性（polysemy）**：一个词具有多个相关意义。"Bank" 既可指金融机构，也可指河岸。语境可以消歧。

- **词义消歧（Word Sense Disambiguation, WSD）**是确定一个多义词在给定语境中所指意义的任务。在 "I deposited money at the bank" 中，金融义是正确的。在 "We sat by the river bank" 中，地理义才是对的。WSD 是早期 NLP 的核心问题；现代上下文 embedding（ELMo、BERT）通过为同一个词的不同用法产生不同的向量表示，基本解决了这一问题。

- **组合语义学（compositional semantics）**探讨单个词的意义如何组合成短语或句子的意义。**组合性原则（compositionality，归功于 Frege）**陈述：一个复杂表达的意义由其组成部分的意义以及组合它们的规则所决定。"The cat chased the dog" 与 "the dog chased the cat" 意义不同，因为句法结构（谁是主语、谁是宾语）与词义发生交互。

- 并非所有意义都是组合的。**习语（idiom）**如 "kick the bucket"（意为 "to die"）的意义无法从其组成部分推导。这对任何组合性方法都是挑战。

- **分布式语义学（distributional semantics）**是现代 NLP 背后的计算化意义方法。**分布式假设（distributional hypothesis，Firth, 1957）**陈述："You shall know a word by the company it keeps."（通过一个词的伙伴来认识它。）出现在相似语境中的词往往意义相似。这是词 embedding（Word2Vec、GloVe）的理论基础，我们将在文件 03 中探讨。

- **语用学（pragmatics）**研究语境如何影响意义。同一句话因说话人、时间、地点、目的不同，意义可能不同。

- "Can you pass the salt?" 在句法上是一个关于能力的是非问句。在语用上，它是一个请求。你不会回答 "Yes, I can" 然后坐着不动。理解这一点需要字面词语之外的知识，具体而言是**言语行为（speech acts）**的约定。

- **言语行为理论（speech act theory，Austin、Searle）**区分：
    - **言内行为（locutionary act）**：字面内容（"Can you pass the salt?"）
    - **言外行为（illocutionary act）**：意图功能（一个请求）
    - **言后行为（perlocutionary act）**：对听话人的影响（他们把盐递过来）

- **含义（implicature，Grice）**是指隐含但未明确陈述的意义。如果有人问 "Is John a good cook?" 而你回答 "He's British"，你并没有字面回答问题，但听者可以推断（通过文化刻板印象，不论公平与否）你的意思是 "no"。Grice 的**合作原则（cooperative principle）**指出，说话人通常尽量做到信息充分、真实、相关且清晰，而听者会假设这些准则成立来解释话语。

- **共指（coreference）**是一种语用现象，不同表达指代同一实体。在 "Alice went to the store. She bought milk." 中，"she" 指 Alice。解决共指对于理解多句文本至关重要，是 NLP 的关键任务。

- **语篇结构（discourse structure）**描述句子如何连接成连贯文本。叙事有开头、中间和结尾。论证有主张和证据。**修辞结构理论（Rhetorical Structure Theory, RST）**将文本分析为语段之间的语篇关系（elaboration、contrast、cause 等）树。

- 语用学是 NLP 最困难的地方。现代语言模型通过训练数据隐式地处理了大量句法和语义，但语用推理——理解讽刺、含义和依赖语境的意义——仍是前沿挑战。

- **音系学（phonology）**研究语言的语音系统。本章以文本为主，但简要概述可衔接至音频与语音章节（第 09 章）。

- **音位（phoneme）**是区分意义的最小语音单位。英语约有 44 个音位。"bat" 和 "pat" 仅相差一个音位（/b/ 与 /p/），意义却完全不同。这称为**最小对立对（minimal pair）**。

- **音位变体（allophones）**是同一音位的不同物理实现，不改变意义。"pin" 中的 "p"（送气，伴随一股气流）和 "spin" 中的 "p"（不送气）是英语中 /p/ 的音位变体；母语者把它们视为同一个音。

- **国际音标（International Phonetic Alphabet, IPA）**为所有语言的音位提供标准化记法。词 "cat" 转写为 /kæt/。IPA 是书面文本与语音系统之间的桥梁。

- **韵律（prosody）**涵盖言语的节奏、重音和语调。"I didn't say he stole the money" 因重读的词不同而有七种不同意义。韵律承载着文本本身丢失的信息，因此文本转语音系统必须仔细建模韵律。

- 在 NLP 中，音系学知识出现在文本转语音（字素到音位转换）、语音识别（将声学信号映射到音位），甚至拼写纠错和音译中。

1. 构建一个简单的形态分析器，使用常见前缀和后缀列表将英语词拆分为可能的语素。
```python
prefixes = ['un', 're', 'pre', 'dis', 'mis', 'over', 'under', 'out', 'non']
suffixes = ['ing', 'ed', 'ly', 'ness', 'ment', 'tion', 'able', 'ible', 'er', 'est', 'ful', 'less', 'ous']

def analyse_morphemes(word):
    """Simple morpheme analysis using known affixes."""
    parts = []
    remaining = word.lower()

    # Check prefixes
    for p in sorted(prefixes, key=len, reverse=True):
        if remaining.startswith(p) and len(remaining) > len(p) + 2:
            parts.append(f"[prefix: {p}]")
            remaining = remaining[len(p):]
            break

    # Check suffixes
    for s in sorted(suffixes, key=len, reverse=True):
        if remaining.endswith(s) and len(remaining) > len(s) + 2:
            root = remaining[:-len(s)]
            parts.append(f"[root: {root}]")
            parts.append(f"[suffix: {s}]")
            remaining = None
            break

    if remaining is not None:
        parts.append(f"[root: {remaining}]")

    return parts

for word in ['unhappiness', 'reusable', 'disconnected', 'overreacting', 'kindness']:
    print(f"{word:20s} → {' + '.join(analyse_morphemes(word))}")
```

2. 使用递归下降实现一个简单的上下文无关文法分析器。定义一个小型文法并将一个句子分析为成分树。
```python
class CFGParser:
    """Recursive descent parser for a tiny English grammar."""
    def __init__(self, tokens):
        self.tokens = tokens
        self.pos = 0

    def peek(self):
        return self.tokens[self.pos] if self.pos < len(self.tokens) else None

    def consume(self, expected=None):
        tok = self.peek()
        if expected and tok != expected:
            return None
        self.pos += 1
        return tok

    def parse_det(self):
        if self.peek() in ('the', 'a'):
            return ('Det', self.consume())
        return None

    def parse_noun(self):
        if self.peek() in ('cat', 'dog', 'mat', 'man'):
            return ('N', self.consume())
        return None

    def parse_verb(self):
        if self.peek() in ('sat', 'chased', 'saw'):
            return ('V', self.consume())
        return None

    def parse_prep(self):
        if self.peek() in ('on', 'under', 'with'):
            return ('P', self.consume())
        return None

    def parse_np(self):
        save = self.pos
        det = self.parse_det()
        noun = self.parse_noun()
        if det and noun:
            # Check for optional PP
            pp = self.parse_pp()
            if pp:
                return ('NP', det, noun, pp)
            return ('NP', det, noun)
        self.pos = save
        return None

    def parse_pp(self):
        save = self.pos
        prep = self.parse_prep()
        np = self.parse_np()
        if prep and np:
            return ('PP', prep, np)
        self.pos = save
        return None

    def parse_vp(self):
        save = self.pos
        verb = self.parse_verb()
        if verb:
            np = self.parse_np()
            if np:
                return ('VP', verb, np)
            pp = self.parse_pp()
            if pp:
                return ('VP', verb, pp)
        self.pos = save
        return None

    def parse_sentence(self):
        np = self.parse_np()
        vp = self.parse_vp()
        if np and vp and self.pos == len(self.tokens):
            return ('S', np, vp)
        return None

def print_tree(tree, indent=0):
    if isinstance(tree, str):
        print(' ' * indent + tree)
    elif isinstance(tree, tuple):
        print(' ' * indent + tree[0])
        for child in tree[1:]:
            print_tree(child, indent + 2)

sentences = [
    "the cat sat on the mat",
    "a dog chased the cat",
]

for sent in sentences:
    tokens = sent.split()
    parser = CFGParser(tokens)
    tree = parser.parse_sentence()
    print(f"\n'{sent}':")
    if tree:
        print_tree(tree)
    else:
        print("  (no parse found)")
```

3. 通过构建一个简单的词图来探索词汇关系。给定一个包含同义、反义和上下位关系的小型词汇表，找出词之间的路径。
```python
relations = {
    ('big', 'large'): 'synonym',
    ('big', 'small'): 'antonym',
    ('small', 'tiny'): 'synonym',
    ('dog', 'animal'): 'hypernym',
    ('cat', 'animal'): 'hypernym',
    ('puppy', 'dog'): 'hypernym',
    ('happy', 'glad'): 'synonym',
    ('happy', 'sad'): 'antonym',
    ('hot', 'cold'): 'antonym',
    ('hot', 'warm'): 'synonym',
}

# Build adjacency list
from collections import defaultdict, deque

graph = defaultdict(list)
for (w1, w2), rel in relations.items():
    graph[w1].append((w2, rel))
    graph[w2].append((w1, rel))

def find_path(start, end):
    """BFS to find a path between two words through the relation graph."""
    queue = deque([(start, [(start, None)])])
    visited = {start}
    while queue:
        node, path = queue.popleft()
        if node == end:
            return path
        for neighbor, rel in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append((neighbor, path + [(neighbor, rel)]))
    return None

pairs = [('big', 'tiny'), ('puppy', 'cat'), ('happy', 'sad')]
for w1, w2 in pairs:
    path = find_path(w1, w2)
    if path:
        steps = " → ".join(f"{w}({r})" if r else w for w, r in path)
        print(f"{w1} → {w2}: {steps}")
    else:
        print(f"{w1} → {w2}: no path found")
```
