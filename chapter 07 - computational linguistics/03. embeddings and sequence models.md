# Эмбеддинги и модели последовательностей

*Эмбеддинги слов сжимают разреженный, символьный текст в плотные векторные пространства, где семантическая близость становится геометрической близостью. В этом файле рассматриваются Word2Vec (CBOW, Skip-gram), GloVe, FastText, RNN, LSTM, GRU, seq2seq с механизмом внимания (attention), парадигма энкодера-декодера, а также переход от мешка слов (bag-of-words) к контекстуальным представлениям.*

- В файле 01 мы представили дистрибутивную гипотезу: слова, которые встречаются в похожих контекстах, как правило, имеют схожие значения. В файле 02 мы представляли текст с помощью разреженных, созданных вручную признаков, таких как векторы TF-IDF. Эти векторы существуют в пространствах очень высокой размерности (по одной размерности на каждое слово словаря) и по большей части состоят из нулей. **Эмбеддинги слов** сжимают эту информацию в плотные векторы низкой размерности, которые улавливают семантические отношения, и обучаются непосредственно на данных.

- **Word2Vec** (Mikolov et al., 2013) изучает эмбеддинги слов путем обучения неглубокой нейронной сети на простой задаче предсказания. Существует две архитектуры.

- Модель **Continuous Bag of Words (CBOW)** предсказывает целевое слово по окружающим его контекстным словам. Получив окно контекстных слов (например, "the cat ___ on the"), модель усредняет их векторы эмбеддингов и пропускает результат через линейный слой для предсказания пропущенного слова ("sat"). Целевая функция обучения максимизирует:

$$P(w_t \mid w_{t-k}, \ldots, w_{t-1}, w_{t+1}, \ldots, w_{t+k})$$

- Модель **Skip-gram** делает обратное: по целевому слову предсказывает окружающие контекстные слова. Для целевого слова "sat" модель пытается предсказать "the", "cat", "on", "the" в отдельных предсказаниях. Целевая функция максимизирует:

$$P(w_{t+j} \mid w_t) \quad \text{for each } j \in [-k, k], \; j \neq 0$$

![Архитектуры Skip-gram и CBOW рядом: CBOW усредняет эмбеддинги контекста для предсказания центрального слова, skip-gram использует эмбеддинг центрального слова для предсказания каждого контекстного слова](../images/word2vec_architectures.svg)

- Skip-gram, как правило, работает лучше для редких слов, поскольку каждое слово генерирует несколько обучающих примеров (по одному на каждую позицию в контексте). CBOW работает быстрее и немного лучше для частотных слов, так как усредняет сигналы по нескольким контекстам.

- Обучение на полном словаре затратно, так как знаменатель softmax суммирует все $V$ слов. **Отрицательное семплирование (negative sampling)** аппроксимирует это, превращая задачу в бинарную классификацию: отличить истинное контекстное слово (положительный пример) от случайно выбранных шумовых слов (отрицательные примеры). Вместо вычисления полного softmax модель обновляет эмбеддинги только для цели, истинного контекстного слова и нескольких отрицательных примеров:

$$\mathcal{L} = \log \sigma(v_{w_O}^T v_{w_I}) + \sum_{i=1}^{k} \mathbb{E}_{w_i \sim P_n} [\log \sigma(-v_{w_i}^T v_{w_I})]$$

- Здесь $v_{w_I}$ — эмбеддинг входного слова, $v_{w_O}$ — эмбеддинг выходного (контекстного) слова, а $P_n$ — распределение шума, обычно частота униграмм, возведенная в степень 3/4 (что снижает вес очень частотных слов, таких как "the").

- Почему эта простая целевая функция создает осмысленные эмбеддинги? Леви и Голдберг (2014) показали, что skip-gram с отрицательным семплированием неявно факторизует матрицу **сдвинутой поточечной взаимной информации (PMI)**. При сходимости скалярное произведение двух векторов слов аппроксимирует:

$$v_w^T v_c \approx \text{PMI}(w, c) - \log k$$

- где $\text{PMI}(w, c) = \log \frac{P(w, c)}{P(w) P(c)}$ измеряет, насколько чаще слова $w$ и $c$ встречаются вместе, чем ожидалось бы случайно (глава 05, теория информации), а $k$ — количество отрицательных примеров. Слова, которые встречаются вместе гораздо чаще, чем случайно, имеют высокое значение PMI и, следовательно, высокое скалярное произведение (похожие эмбеддинги). Слова, которые встречаются вместе реже, чем ожидалось, имеют отрицательное PMI и несхожие эмбеддинги. Это показывает, что Word2Vec делает то же самое, что и классические методы дистрибутивной семантики, такие как латентно-семантический анализ (SVD матриц совместной встречаемости), но более масштабируемым, потоковым способом.

- Самое удивительное свойство эмбеддингов Word2Vec заключается в том, что они улавливают **аналогии** с помощью векторной арифметики. Вектор $v_{\text{king}} - v_{\text{man}} + v_{\text{woman}}$ оказывается ближе всего к $v_{\text{queen}}$. Это работает, потому что пространство эмбеддингов кодирует семантические отношения как приблизительно линейные направления: направление "королевской власти" — это примерно $v_{\text{king}} - v_{\text{man}}$, и добавление его к $v_{\text{woman}}$ приводит к вектору, близкому к $v_{\text{queen}}$. Это связывает нас с линейной алгеброй из главы 01: семантические отношения — это векторные переносы.

- **GloVe** (Global Vectors for Word Representation, Pennington et al., 2014) использует другой подход. Вместо обучения на локальных контекстных окнах по одному, он строит глобальную матрицу совместной встречаемости слов $X$, где $X_{ij}$ подсчитывает, как часто слово $j$ появляется в контексте слова $i$ во всем корпусе. Затем модель изучает эмбеддинги, скалярное произведение которых аппроксимирует логарифм совместной встречаемости:

$$w_i^T \tilde{w}_j + b_i + \tilde{b}_j = \log X_{ij}$$

- Функция потерь взвешивает каждую пару с помощью функции ограничения $f(X_{ij})$, которая предотвращает доминирование очень частых совместных встречаемостей:

$$\mathcal{L} = \sum_{i,j=1}^{V} f(X_{ij}) \left(w_i^T \tilde{w}_j + b_i + \tilde{b}_j - \log X_{ij}\right)^2$$

- GloVe объединяет преимущества глобальной матричной факторизации (как в латентно-семантическом анализе) с обучением на локальном контексте, как в Word2Vec. На практике GloVe и Word2Vec создают эмбеддинги схожего качества.

- **FastText** (Bojanowski et al., 2017) расширяет skip-gram, представляя каждое слово как мешок символьных n-грамм. Слово "where" при $n = 3$ превращается в: "<wh", "whe", "her", "ere", "re>", плюс токен целого слова "<where>". Эмбеддинг слова — это сумма эмбеддингов всех его n-грамм.

- Это дает решающее преимущество: FastText может создавать эмбеддинги для слов, которые он никогда не видел во время обучения. Слово "whereabouts" разделяет n-граммы со словом "where", поэтому его эмбеддинг будет разумным, даже если "whereabouts" никогда не встречалось в обучающих данных. Это особенно полезно для морфологически богатых языков (файл 01), где слова имеют множество словоформ.

- **Оценка эмбеддингов** обычно использует два типа бенчмарков. **Задачи на аналогии** проверяют, выполняется ли $v_a - v_b + v_c \approx v_d$ (например, «Париж» $-$ «Франция» $+$ «Италия» $\approx$ «Рим»). **Бенчмарки на сходство** сравнивают косинусное сходство (глава 01) между парами слов с оценками людей. Распространенные датасеты включают WordSim-353, SimLex-999 и набор тестов на аналогии от Google. Практическое предостережение: эмбеддинги, которые отлично справляются с аналогиями, могут оказаться не лучшими для прикладных задач, таких как классификация тональности. Лучшая оценка — это зачастую сама задача.

- В главе 06 мы представили RNN, LSTM и GRU как архитектуры для последовательных данных. Здесь мы сосредоточимся на том, как они применяются именно в языковых задачах.

- **Языковая модель на базе RNN** считывает токены по одному и на каждом шаге предсказывает следующий токен. Скрытое состояние $h_t$ сжимает всю историю $w_1, \ldots, w_t$ в вектор фиксированного размера, а линейный слой и softmax отображают $h_t$ в распределение по словарю. Обучение использует функцию потерь кросс-энтропии относительно истинного следующего токена, что идентично минимизации перплексии (файл 02). Ключевое ограничение: скрытое состояние фиксированного размера должно кодировать всё об истории, и информация из ранних токенов постепенно перезаписывается.

- **Двунаправленные RNN (Bidirectional RNNs)** обрабатывают последовательность в обоих направлениях: одна RNN читает слева направо, другая — справа налево. В каждой позиции $t$ прямое скрытое состояние $\overrightarrow{h}_t$ и обратное скрытое состояние $\overleftarrow{h}_t$ конкатенируются, образуя контекстно-зависимое представление $h_t = [\overrightarrow{h}_t ; \overleftarrow{h}_t]$. Это дает модели доступ как к прошлому, так и к будущему контексту, что эффективно для таких задач, как POS-теггинг и NER (файл 02), где метка слова зависит от слов как до, так и после него. Двунаправленные RNN нельзя использовать для языкового моделирования, поскольку нельзя заглядывать в будущие токены при их предсказании.

![Двунаправленная RNN: прямая RNN читает слева направо, создавая скрытые состояния, обратная RNN читает справа налево, выходы конкатенируются в каждой позиции](../images/bidirectional_rnn.svg)

- **Глубокие стековые RNN** размещают несколько слоев RNN друг над другом. Скрытые состояния слоя $l$ на всех временных шагах становятся входной последовательностью для слоя $l + 1$. Стекинг из 2–4 слоев обычно улучшает производительность за счет построения иерархических представлений, подобно тому, как более глубокие CNN строят иерархии признаков (глава 06). При использовании более 4 слоев возникают проблемы затухающих градиентов и переобучения, если не добавить остаточные связи между слоями.

- Архитектура **sequence-to-sequence (seq2seq)** (Sutskever et al., 2014) отображает входную последовательность переменной длины в выходную последовательность переменной длины. Она состоит из **энкодера** (RNN), который считывает входные данные и сжимает их в контекстный вектор (финальное скрытое состояние), и **декодера** (RNN), который генерирует выходные данные по одному токену за раз, обуславливаясь этим контекстным вектором.

![Seq2seq энкодер-декодер: энкодер RNN считывает входные токены слева направо, финальное скрытое состояние передается как начальное состояние в декодер RNN, который генерирует выходные токены авторегрессионно](../images/seq2seq_architecture.svg)

- Seq2seq стала прорывной архитектурой для машинного перевода. Энкодер считывает французское предложение, декодер создает английский перевод. Декодер начинает со специального токена начала последовательности и генерирует токены авторегрессионно, пока не создаст токен конца последовательности. Практический трюк: разворот входной последовательности (подача «chat le» вместо «le chat») улучшил результаты, так как это расположило первое входное слово ближе к первому выходному слову в вычислительном графе, сокращая градиентный путь.

- Проблема «узкого места»: весь вход должен быть сжат в один вектор фиксированного размера. Для длинных предложений этот вектор не может захватить всю информацию, и производительность падает. Это послужило мотивацией для создания **механизмов внимания**.

- В главе 06 была представлена современная формулировка внимания через Q, K, V. Исходные механизмы внимания для NLP формулировались иначе — как модели выравнивания между состояниями энкодера и декодера.

- **Внимание Бахданау** (аддитивное внимание, Bahdanau et al., 2015) вычисляет оценку выравнивания между скрытым состоянием декодера $s_t$ и каждым скрытым состоянием энкодера $h_i$ с помощью обученной полносвязной сети:

$$e_{ti} = v^T \tanh(W_s s_{t-1} + W_h h_i)$$

- Оценки нормализуются в веса внимания через softmax, а контекстный вектор представляет собой взвешенную сумму состояний энкодера:

$$\alpha_{ti} = \frac{\exp(e_{ti})}{\sum_j \exp(e_{tj})}, \quad c_t = \sum_i \alpha_{ti} h_i$$

- Затем декодер использует и $s_{t-1}$, и $c_t$ для создания следующего выхода. Ключевая идея: вместо одного фиксированного контекстного вектора для всего предложения, каждый шаг декодера получает свою взвешенную комбинацию состояний энкодера, позволяя модели «оглядываться» на релевантные части входа.

- **Внимание Луонга** (мультипликативное внимание, Luong et al., 2015) упрощает вычисление оценки. Вариант **dot** использует $e_{ti} = s_t^T h_i$. Вариант **general** использует $e_{ti} = s_t^T W h_i$. Они быстрее, чем аддитивная оценка Бахданау, так как используют матричное умножение вместо полносвязной сети. Внимание Луонга также вычисляет контекстный вектор на основе текущего состояния декодера $s_t$ (а не $s_{t-1}$), что дает ему доступ к большему объему информации, но делает вычисления немного другими.

![Тепловая карта выравнивания внимания между исходным предложением и его переводом, показывающая, на какие исходные слова обращает внимание каждое целевое слово; более яркие ячейки указывают на более высокие веса внимания](../images/attention_alignment.svg)

- Веса внимания часто визуализируются в виде тепловых карт, показывающих, на какие входные токены декодер фокусируется при создании каждого выходного токена. В переводе эти тепловые карты примерно прослеживают выравнивание слов между исходным и целевым языками, при этом диагональный паттерн нарушается из-за переупорядочивания (например, порядок прилагательного и существительного различается во французском и английском языках).

- На этапе инференса декодер должен выбирать токен на каждом шаге. **Жадное декодирование (greedy decoding)** выбирает токен с наибольшей вероятностью на каждой позиции, но это может привести к субоптимальным последовательностям: локально хороший выбор может заставить модель прийти к глобально плохому предложению. **Поиск по лучу (beam search)** сохраняет топ-$k$ (ширина луча) частичных последовательностей на каждом шаге, расширяя каждую из них всеми возможными следующими токенами и оставляя лучшие $k$ в целом.

- При ширине луча $k = 1$ поиск по лучу (beam search) сводится к жадному декодированию. Типичные значения составляют от $k = 4$ до $k = 10$. Большие значения ширины луча позволяют находить лучшие последовательности, но пропорционально замедляют работу. Поиск по лучу также требует **нормализации по длине** (length normalisation), чтобы избежать предпочтения более коротким последовательностям, которые естественным образом имеют более высокую общую вероятность, так как в них перемножается меньше множителей. Нормализованная оценка вычисляется так:

$$\text{score}(y) = \frac{1}{|y|^\alpha} \sum_{t=1}^{|y|} \log P(y_t \mid y_{<t})$$

- где $|y|$ — длина последовательности, а $\alpha$ (обычно 0.6–0.7) управляет силой штрафа за длину. При $\alpha = 0$ нормализация по длине отсутствует. При $\alpha = 1$ оценка представляет собой логарифмическую вероятность на токен (геометрическое среднее). Промежуточное значение позволяет найти баланс между предпочтением лаконичных выводов и отсутствием слишком раннего усечения.

- В то время как RNN обрабатывают текст последовательно, **1D CNN** обрабатывают его параллельно, перемещая фильтры вдоль последовательности токенов. Каждый фильтр обнаруживает локальный паттерн (n-граммный признак).

- **TextCNN** (Kim, 2014) применяет несколько одномерных сверточных фильтров разной ширины (например, 3, 4, 5 токенов) к матрице входных эмбеддингов. Каждый фильтр создает карту признаков, а **max-over-time pooling** выбирает единственное максимальное значение из каждой карты, фиксируя факт обнаружения паттерна в любом месте текста, независимо от его позиции. Объединенные признаки со всех фильтров конкатенируются и передаются в классификатор.

![Архитектура TextCNN: входные эмбеддинги проходят через параллельные сверточные фильтры шириной 3, 4 и 5, каждый из которых сопровождается max-over-time pooling, после чего конкатенируются и подаются на полносвязный классификатор](../images/textcnn_architecture.svg)

- TextCNN работает быстро и на удивление эффективно для задач классификации текста, таких как анализ тональности. Модель улавливает локальные n-граммные паттерны, но не может моделировать долгосрочные зависимости: фильтр шириной 5 видит только 5 последовательных токенов. **Дилатированные причинные свертки** (dilated causal convolutions) решают эту проблему путем вставки пропусков (дилатации) между элементами фильтра. Стекирование слоев с экспоненциально возрастающими коэффициентами дилатации (1, 2, 4, 8, ...) позволяет экспоненциально увеличивать рецептивное поле без увеличения количества параметров, что дает модели возможность улавливать зависимости на расстоянии сотен токенов.

- Все рассмотренные до сих пор эмбеддинги (Word2Vec, GloVe, FastText) создают один вектор для каждого типа слова независимо от контекста. Слово "Bank" получает одинаковый эмбеддинг, независимо от того, означает ли оно финансовое учреждение или берег реки. Это фундаментальное ограничение, которое устраняют **контекстуальные эмбеддинги**.

- **ELMo** (Embeddings from Language Models, Peters et al., 2018) создает контекстуальные представления слов путем запуска глубокой двунаправленной LSTM-языковой модели на входном тексте. Прямая LSTM предсказывает следующее слово в каждой позиции, а отдельная обратная LSTM — предыдущее. Обе модели обучаются как языковые модели на больших корпусах.

- В каждой позиции $k$ ELMo объединяет скрытые состояния со всех $L$ слоев, используя обученные веса, специфичные для конкретной задачи:

$$\text{ELMo}_k = \gamma \sum_{j=0}^{L} s_j \, h_{k,j}$$

- Здесь $h_{k,j}$ — скрытое состояние в позиции $k$ на слое $j$ (слой 0 — это исходный эмбеддинг токена), $s_j$ — скалярные веса, нормализованные через softmax, а $\gamma$ — масштабирующий коэффициент, специфичный для задачи. Разные слои улавливают разную информацию: нижние слои отвечают за синтаксис (POS-теги, морфологию слов), верхние — за семантику (значение слова, семантическую роль). Смешивая все слои с помощью обученных весов, эмбеддинги ELMo адаптируются к разнообразным прикладным задачам.

- ELMo ознаменовал начало парадигмы **предобучения с последующей донастройкой** (pre-train then fine-tune): обучение большой языковой модели на огромных неразмеченных текстах с последующим использованием ее представлений для прикладных задач. ELMo использует предобученные представления как фиксированные или слегка донастраиваемые признаки, которые конкатенируются с входными данными конкретной задачи. BERT и GPT (файл 04) развивают этот подход, донастраивая всю модель целиком (end-to-end), что оказывается значительно эффективнее.

- Эволюция от Word2Vec к ELMo иллюстрирует повторяющуюся тему в NLP: переход от статических представлений к динамическим, от локального контекста к глобальному и от простых моделей к глубоким. Каждый шаг требует больших вычислительных затрат в обмен на более богатые представления. Трансформеры (файл 04) завершают этот процесс, полностью заменяя рекурсию механизмом внимания, что обеспечивает как глубокую контекстуализацию, так и параллельные вычисления.

## Задачи по программированию (используйте CoLab или ноутбук)

1. Реализуйте Word2Vec skip-gram с негативным семплированием (negative sampling) с нуля. Обучите модель на небольшом корпусе и визуализируйте полученные эмбеддинги с помощью PCA.

```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

# Small corpus
corpus = """the king ruled the kingdom . the queen ruled the kingdom .
the prince is the son of the king . the princess is the daughter of the queen .
a man worked in the castle . a woman worked in the castle .
the king and queen lived in the castle . the prince and princess played outside .""".lower().split()

vocab = sorted(set(corpus))
word2idx = {w: i for i, w in enumerate(vocab)}
idx2word = {i: w for w, i in word2idx.items()}
V = len(vocab)

# Generate skip-gram pairs with window size 2
window = 2
pairs = []
for i, word in enumerate(corpus):
    for j in range(max(0, i - window), min(len(corpus), i + window + 1)):
        if i != j:
            pairs.append((word2idx[word], word2idx[corpus[j]]))

pairs = jnp.array(pairs)
print(f"Vocabulary: {V} words, Training pairs: {len(pairs)}")

# Model parameters
embed_dim = 16
key = jax.random.PRNGKey(42)
k1, k2 = jax.random.split(key)
W_in = jax.random.normal(k1, (V, embed_dim)) * 0.1    # input embeddings
W_out = jax.random.normal(k2, (V, embed_dim)) * 0.1   # output embeddings

# Negative sampling loss for one pair
def neg_sampling_loss(W_in, W_out, target, context, neg_ids):
    v_in = W_in[target]      # (embed_dim,)
    v_out = W_out[context]   # (embed_dim,)
    v_neg = W_out[neg_ids]   # (k, embed_dim)

    pos_loss = -jax.nn.log_sigmoid(jnp.dot(v_in, v_out))
    neg_loss = -jnp.sum(jax.nn.log_sigmoid(-v_neg @ v_in))
    return pos_loss + neg_loss

# Training loop
num_neg = 5
lr = 0.05

@jax.jit
def train_step(W_in, W_out, target, context, neg_ids):
    loss, (g_in, g_out) = jax.value_and_grad(neg_sampling_loss, argnums=(0, 1))(
        W_in, W_out, target, context, neg_ids)
    return loss, W_in - lr * g_in, W_out - lr * g_out

key = jax.random.PRNGKey(0)
for epoch in range(50):
    total_loss = 0.0
    for i in range(len(pairs)):
        key, subkey = jax.random.split(key)
        neg_ids = jax.random.randint(subkey, (num_neg,), 0, V)
        loss, W_in, W_out = train_step(W_in, W_out, pairs[i, 0], pairs[i, 1], neg_ids)
        total_loss += loss
    if (epoch + 1) % 10 == 0:
        print(f"Epoch {epoch+1}: avg loss = {total_loss / len(pairs):.4f}")

# Visualise with PCA (chapter 01)
embeddings = W_in
mean = embeddings.mean(axis=0)
centered = embeddings - mean
U, S, Vt = jnp.linalg.svd(centered, full_matrices=False)
coords = centered @ Vt[:2].T  # project onto top 2 PCs

plt.figure(figsize=(10, 8))
for i, word in idx2word.items():
    plt.scatter(coords[i, 0], coords[i, 1], c='#3498db', s=40)
    plt.annotate(word, (coords[i, 0] + 0.02, coords[i, 1] + 0.02), fontsize=9)
plt.title("Word2Vec Skip-gram Embeddings (PCA projection)")
plt.grid(alpha=0.3); plt.show()
```

2. Build a character-level RNN language model that learns to generate text from a small training string.
```python
import jax
import jax.numpy as jnp

# Tiny training text
text = "to be or not to be that is the question "
chars = sorted(set(text))
char2idx = {c: i for i, c in enumerate(chars)}
idx2char = {i: c for c, i in char2idx.items()}
V = len(chars)
data = jnp.array([char2idx[c] for c in text])

# RNN parameters
hidden_dim = 64
key = jax.random.PRNGKey(0)
k1, k2, k3, k4, k5 = jax.random.split(key, 5)

params = {
    'Wx': jax.random.normal(k1, (V, hidden_dim)) * 0.1,
    'Wh': jax.random.normal(k2, (hidden_dim, hidden_dim)) * 0.05,
    'bh': jnp.zeros(hidden_dim),
    'Wy': jax.random.normal(k3, (hidden_dim, V)) * 0.1,
    'by': jnp.zeros(V),
}

def rnn_step(params, h, x_idx):
    x = jnp.eye(V)[x_idx]  # one-hot
    h = jnp.tanh(x @ params['Wx'] + h @ params['Wh'] + params['bh'])
    logits = h @ params['Wy'] + params['by']
    return h, logits

def loss_fn(params, inputs, targets):
    h = jnp.zeros(hidden_dim)
    total_loss = 0.0
    for t in range(len(inputs)):
        h, logits = rnn_step(params, h, inputs[t])
        log_probs = jax.nn.log_softmax(logits)
        total_loss -= log_probs[targets[t]]
    return total_loss / len(inputs)

grad_fn = jax.jit(jax.grad(loss_fn))

# Training
inputs = data[:-1]
targets = data[1:]
lr = 0.01

for step in range(500):
    grads = grad_fn(params, inputs, targets)
    params = {k: params[k] - lr * grads[k] for k in params}
    if (step + 1) % 100 == 0:
        l = loss_fn(params, inputs, targets)
        print(f"Step {step+1}: loss = {l:.4f}")

# Generate text
def generate(params, seed_char, length=60):
    h = jnp.zeros(hidden_dim)
    idx = char2idx[seed_char]
    result = [seed_char]
    key = jax.random.PRNGKey(42)
    for _ in range(length):
        h, logits = rnn_step(params, h, idx)
        key, subkey = jax.random.split(key)
        idx = jax.random.categorical(subkey, logits)
        result.append(idx2char[int(idx)])
    return ''.join(result)

print(f"\nGenerated: {generate(params, 't')}")
```

3. Implement a toy seq2seq model with Bahdanau attention for sequence reversal. Visualise the attention alignment matrix.
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

# Task: reverse a sequence of digits (e.g., [3, 1, 4] -> [4, 1, 3])
vocab_size = 10  # digits 0-9
SOS, EOS = 10, 11  # special tokens
total_vocab = 12
embed_dim, hidden_dim = 16, 32
max_len = 5

key = jax.random.PRNGKey(42)
keys = jax.random.split(key, 8)

params = {
    'embed': jax.random.normal(keys[0], (total_vocab, embed_dim)) * 0.1,
    'enc_Wx': jax.random.normal(keys[1], (embed_dim, hidden_dim)) * 0.1,
    'enc_Wh': jax.random.normal(keys[2], (hidden_dim, hidden_dim)) * 0.05,
    'dec_Wx': jax.random.normal(keys[3], (embed_dim, hidden_dim)) * 0.1,
    'dec_Wh': jax.random.normal(keys[4], (hidden_dim, hidden_dim)) * 0.05,
    # Bahdanau attention
    'Ws': jax.random.normal(keys[5], (hidden_dim, hidden_dim)) * 0.1,
    'Wh_att': jax.random.normal(keys[6], (hidden_dim, hidden_dim)) * 0.1,
    'v_att': jax.random.normal(keys[7], (hidden_dim,)) * 0.1,
    # Output projection (from hidden + context to vocab)
    'Wo': jax.random.normal(keys[0], (hidden_dim * 2, total_vocab)) * 0.1,
}

def encode(params, seq):
    """Encode input sequence, return all hidden states."""
    h = jnp.zeros(hidden_dim)
    states = []
    for t in range(len(seq)):
        x = params['embed'][seq[t]]
        h = jnp.tanh(x @ params['enc_Wx'] + h @ params['enc_Wh'])
        states.append(h)
    return jnp.stack(states), h

def bahdanau_attention(params, dec_state, enc_states):
    """Compute Bahdanau attention weights and context vector."""
    scores = jnp.tanh(enc_states @ params['Wh_att'] + dec_state @ params['Ws'])
    e = scores @ params['v_att']  # (src_len,)
    alpha = jax.nn.softmax(e)
    context = alpha @ enc_states
    return context, alpha

def decode_step(params, dec_h, prev_token, enc_states):
    x = params['embed'][prev_token]
    dec_h = jnp.tanh(x @ params['dec_Wx'] + dec_h @ params['dec_Wh'])
    context, alpha = bahdanau_attention(params, dec_h, enc_states)
    combined = jnp.concatenate([dec_h, context])
    logits = combined @ params['Wo']
    return dec_h, logits, alpha

def seq2seq_loss(params, src, tgt):
    enc_states, enc_final = encode(params, src)
    dec_h = enc_final
    loss = 0.0
    prev_token = SOS
    for t in range(len(tgt)):
        dec_h, logits, _ = decode_step(params, dec_h, prev_token, enc_states)
        log_probs = jax.nn.log_softmax(logits)
        loss -= log_probs[tgt[t]]
        prev_token = tgt[t]
    return loss / len(tgt)

# Generate training data: reverse sequences
key = jax.random.PRNGKey(0)
train_srcs, train_tgts = [], []
for _ in range(200):
    key, subkey = jax.random.split(key)
    length = jax.random.randint(subkey, (), 3, max_len + 1)
    key, subkey = jax.random.split(key)
    seq = jax.random.randint(subkey, (int(length),), 0, vocab_size)
    train_srcs.append(seq)
    train_tgts.append(seq[::-1])  # reverse

# Training
grad_fn = jax.grad(seq2seq_loss)
lr = 0.01

for epoch in range(100):
    total_loss = 0.0
    for src, tgt in zip(train_srcs, train_tgts):
        grads = grad_fn(params, src, tgt)
        params = {k: params[k] - lr * grads[k] for k in params}
        total_loss += seq2seq_loss(params, src, tgt)
    if (epoch + 1) % 20 == 0:
        print(f"Epoch {epoch+1}: avg loss = {total_loss / len(train_srcs):.4f}")

# Visualise attention for one example
test_src = jnp.array([3, 1, 4, 1, 5])
test_tgt = test_src[::-1]

enc_states, enc_final = encode(params, test_src)
dec_h = enc_final
attentions = []
prev_token = SOS
for t in range(len(test_tgt)):
    dec_h, logits, alpha = decode_step(params, dec_h, prev_token, enc_states)
    attentions.append(alpha)
    prev_token = test_tgt[t]

att_matrix = jnp.stack(attentions)
fig, ax = plt.subplots(figsize=(6, 5))
im = ax.imshow(att_matrix, cmap='Blues')
ax.set_xlabel("Source position"); ax.set_ylabel("Target position")
src_labels = [str(int(x)) for x in test_src]
tgt_labels = [str(int(x)) for x in test_tgt]
ax.set_xticks(range(len(src_labels))); ax.set_xticklabels(src_labels)
ax.set_yticks(range(len(tgt_labels))); ax.set_yticklabels(tgt_labels)
for i in range(len(tgt_labels)):
    for j in range(len(src_labels)):
        ax.text(j, i, f"{att_matrix[i,j]:.2f}", ha='center', va='center', fontsize=9)
ax.set_title("Bahdanau Attention Alignment (sequence reversal)")
plt.colorbar(im); plt.tight_layout(); plt.show()
```
