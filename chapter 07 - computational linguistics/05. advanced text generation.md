# Продвинутая генерация текста

*Продвинутая генерация текста выходит за рамки стандартного авторегрессионного декодирования для улучшения качества, управляемости и скорости. В этом файле рассматриваются диффузионные модели для текста (D3PM, MDLM), OCR, RLHF и DPO для выравнивания (alignment), методы для работы с длинным контекстом (масштабирование RoPE, ring attention), генерация с дополнением из поиска (RAG) и спекулятивное декодирование для ускорения инференса.*

- Стандартная авторегрессионная генерация (файл 04) создает текст по одному токену за раз, слева направо. Это просто и эффективно, но такой подход по своей сути последователен, не допускает глобального планирования и дает ограниченный контроль над результатом. В этом файле рассматриваются методы, выходящие за рамки стандартного авторегрессионного декодирования: диффузионные модели для текста, оптическое распознавание символов, управляемая генерация с использованием обратной связи от человека, работа с длинным контекстом, генерация с дополнением из поиска и спекулятивное декодирование для ускорения инференса.

- **Диффузионные модели для текста** применяют диффузионный фреймворк (представленный для изображений в главе 08) к дискретному тексту. Основная сложность заключается в том, что текст дискретен: нельзя добавить непрерывный гауссовский шум к токенам так же, как мы добавляем шум к пикселям. Эту проблему решают несколько подходов.

- **D3PM** (Discrete Denoising Diffusion Probabilistic Models, Austin et al., 2021) определяет прямой процесс искажения непосредственно над дискретными токенами с использованием матриц перехода. На каждом прямом шаге токен с некоторой вероятностью заменяется другим токеном (равномерный шум), маскируется (поглощающее состояние) или остается прежним. Обратный процесс обучается удалять шум, предсказывая чистый токен по искаженному. Матрица перехода $Q_t$ на шаге $t$ управляет искажением:

$$q(x_t \mid x_{t-1}) = \text{Cat}(x_t ; \, x_{t-1} Q_t)$$

- где $\text{Cat}$ обозначает категориальное распределение, а $x$ — вектор в формате one-hot. Многошаговый прямой процесс $q(x_t \mid x_0)$ имеет аналитическое решение: $q(x_t \mid x_0) = \text{Cat}(x_t ; \, x_0 \bar{Q}_t)$, где $\bar{Q}_t = Q_1 Q_2 \cdots Q_t$ — произведение всех матриц перехода вплоть до шага $t$. Обучение минимизирует вариационную нижнюю границу (ELBO), которая декомпозируется по временным шагам, аналогично непрерывному случаю (глава 08):

$$\mathcal{L}_{\text{D3PM}} = D_{\text{KL}}(q(x_T \mid x_0) \| p(x_T)) + \sum_{t=2}^{T} D_{\text{KL}}(q(x_{t-1} \mid x_t, x_0) \| p_\theta(x_{t-1} \mid x_t)) - \log p_\theta(x_0 \mid x_1)$$

- Первый член гарантирует, что полностью искаженное распределение соответствует априорному (равномерному или полностью замаскированному). Сумма членов KL обучает модель обращать каждый шаг искажения: истинное обратное апостериорное распределение $q(x_{t-1} \mid x_t, x_0)$ может быть вычислено в аналитическом виде с использованием правила Байеса и известных матриц перехода, а модель $p_\theta(x_{t-1} \mid x_t)$ обучается соответствовать ему.

- Поскольку оба распределения являются категориальными, дивергенция Кульбака — Лейблера представляет собой простую сумму по элементам словаря. Последний член измеряет качество реконструкции из наименее искаженного состояния.

- **MDLM** (Masked Diffusion Language Models, Sahoo et al., 2024) упрощает D3PM, используя маскирование в качестве единственной операции искажения: прямой процесс постепенно заменяет токены токеном [MASK], а обратный процесс предсказывает исходные токены. Это связывает диффузию текста с маскированным языковым моделированием (BERT, файл 04), где временной шаг диффузии определяет, какая часть токенов замаскирована. При $t = 0$ текст полностью чист; при $t = T$ он полностью замаскирован.

- **Непрерывная диффузия текста** обходит проблему дискретности, работая в непрерывном пространстве эмбеддингов. Токены сначала отображаются в свои векторы эмбеддингов (глава 06), в этом непрерывном пространстве добавляется шум, а модель шумоподавления (обычно трансформер) обучается обращать процесс. Во время генерации модель создает непрерывные векторы, которые отображаются обратно в дискретные токены путем поиска ближайшего эмбеддинга. Сложность заключается в том, что небольшие ошибки в непрерывном пространстве могут привести к выбору совершенно неверных токенов, поэтому требуются тщательное округление и ограничение значений (clamping).

- Привлекательность диффузии текста заключается в том, что она генерирует все токены одновременно посредством итеративного уточнения, а не слева направо. Это обеспечивает глобальную связность и легкое заполнение пропусков (генерация недостающего текста в середине фрагмента), но текущие диффузионные модели для текста все еще отстают от авторегрессионных моделей по качеству генерации длинных текстов.

- **OCR текста** (оптическое распознавание символов) — это задача извлечения текста из изображений. Хотя традиционно она не относится к генерации языка, современные системы OCR глубоко интегрированы с NLP и все чаще используют компоненты языковых моделей.

- **Детекция текста в сценах** находит текстовые области на естественных изображениях (дорожные знаки, этикетки товаров, номерные знаки). Это сложная задача, так как текст в реальных условиях может располагаться под произвольными углами, иметь разный масштаб, шрифты и находиться на пестром фоне. Методы детекции обычно используют CNN или трансформеры в качестве основы (backbone) для создания ограничивающих рамок (bounding boxes) или сегментационных масок вокруг текстовых областей.

- **CRNN** (Convolutional Recurrent Neural Network, Shi et al., 2017) — это классическая архитектура распознавания текста. CNN извлекает визуальные признаки из изображения текста, карта признаков нарезается на последовательность столбцов (по одному на каждую горизонтальную позицию), а двунаправленная LSTM считывает эту последовательность для моделирования контекста. Выход декодируется с помощью **CTC** (Connectionist Temporal Classification), которая обрабатывает выравнивание между входными столбцами и выходными символами без необходимости явной сегментации.

- Фундаментальная проблема, которую решает CTC: модель выдает $T$ выходных распределений (по одному на входной столбец), но целевой текст содержит $L \leq T$ символов.

- Мы не знаем, какие столбцы соответствуют каким символам. CTC вводит **пустой токен** $\epsilon$ и определяет отображение «многие-к-одному» $\mathcal{B}$, которое схлопывает повторяющиеся символы и удаляет пустые: $\mathcal{B}(\text{"HH-ee-ll-ll-oo"}) = \text{"Hello"}$ (где "-" — это пустой токен).

- Вероятность целевой последовательности $y$ — это сумма вероятностей всех входных выравниваний, которые схлопываются в $y$:

$$P(y \mid x) = \sum_{\pi \in \mathcal{B}^{-1}(y)} \prod_{t=1}^{T} P(\pi_t \mid x)$$

- где $\pi$ — путь выравнивания длины $T$ (одна метка на столбец, включая пробелы). Наивное суммирование по всем путям требует экспоненциального времени, однако **алгоритм прямого хода** (forward algorithm, глава 05, скрытые марковские модели) вычисляет эту сумму эффективно за время $O(T \cdot L)$ с помощью динамического программирования.

- Пробельный токен (blank token) необходим: без него повторяющиеся символы, такие как «лл» в слове «Hello», были бы неотличимы от одиночного «л». Обучение максимизирует $\log P(y \mid x)$, а на этапе инференса лучший путь находится с помощью лучевого поиска (beam search) или жадной декодировки (greedy decoding) по выходу CTC.

- **OCR документов** обрабатывает структурированные документы (счета, формы, научные статьи) и должен понимать макет (layout) помимо распознавания символов. Современные системы, такие как LayoutLM, объединяют распознавание текста с признаками пространственного положения: каждый токен получает как свой текстовый эмбеддинг, так и позиционный эмбеддинг, кодирующий его координаты $(x, y)$ на странице. Это позволяет модели понять, что число, расположенное под надписью «Total:», является итоговой суммой.

![Пайплайн CRNN OCR](../images/crnn_ocr_pipeline.svg)

- **Зрительно-языковые OCR-модели** (vision-language OCR), такие как TrOCR, рассматривают распознавание текста как генерацию текста по изображению: энкодер Vision Transformer обрабатывает изображение, а декодер языковой модели генерирует текст посимвольно. Это позволяет использовать возможности предобученных моделей компьютерного зрения и обработки естественного языка, а также работать с разнообразными шрифтами, начертаниями и макетами без ручного проектирования признаков.

- **Управляемая генерация** (controllable generation) — это задача направления языковой модели на создание вывода с заданными свойствами: определенным стилем, темой, тональностью, уровнем безопасности или фактической точностью. Модель должна следовать инструкциям, оставаясь при этом связной и последовательной.

- **Классификаторно-свободное руководство** (classifier-free guidance, CFG) для текста адаптирует метод из генерации изображений. Во время обучения сигнал обусловливания (например, промпт) случайным образом отбрасывается в некоторой доле случаев, что позволяет обучать условную и безусловную модели одновременно. На этапе инференса выходные логиты интерполируются:

$$\text{logits}_{\text{guided}} = (1 + w) \cdot \text{logits}_{\text{conditional}} - w \cdot \text{logits}_{\text{unconditional}}$$

- где $w > 0$ усиливает влияние условия. Более высокое значение $w$ заставляет вывод сильнее следовать промпту, но снижает разнообразие.

- **RLHF** (обучение с подкреплением на основе отзывов людей, Ouyang et al., 2022) — доминирующий метод согласования языковых моделей с предпочтениями человека. Процесс состоит из трех этапов:

- Во-первых, **обучение с учителем (SFT)**: дообучение базовой языковой модели на датасете высококачественных ответов на промпты, написанных людьми.

- Во-вторых, **обучение модели вознаграждения**: сбор сравнений от людей (дан промпт $x$ и два ответа $y_1, y_2$, какой из них лучше?) и обучение модели вознаграждения $r_\phi(x, y)$ предсказывать предпочтения людей. Модель вознаграждения обучается с использованием функции потерь парного ранжирования:

$$\mathcal{L}_{\text{RM}} = -\log \sigma(r_\phi(x, y_w) - r_\phi(x, y_l))$$

- где $y_w$ — предпочтительный ответ, а $y_l$ — менее предпочтительный.

- В-третьих, **RL-дообучение**: оптимизация языковой модели для максимизации вознаграждения при сохранении близости к SFT-модели (чтобы предотвратить коллапс моды). Для этого используется PPO (Proximal Policy Optimisation, из главы 06) с KL-штрафом:

$$\mathcal{L}_{\text{RL}} = -\mathbb{E}\left[r_\phi(x, y) - \beta \, D_{\text{KL}}(\pi_\theta \| \pi_{\text{SFT}})\right]$$

- KL-член предотвращает слишком сильное отклонение модели от базовой и эксплуатацию особенностей модели вознаграждения («взлом вознаграждения» или reward hacking).

![Пайплайн RLHF](../images/rlhf_pipeline.svg)

- **DPO** (прямая оптимизация предпочтений, Rafailov et al., 2023) упрощает RLHF, полностью исключая модель вознаграждения. Ключевая математическая идея заключается в том, что приведенная выше целевая функция RL с KL-ограничением имеет оптимальную стратегию в аналитическом виде:

$$\pi^\ast(y \mid x) = \frac{1}{Z(x)} \pi_{\text{ref}}(y \mid x) \exp\!\left(\frac{r(x, y)}{\beta}\right)$$

- где $Z(x)$ — нормирующая статистическая сумма. Выражая отсюда вознаграждение, получаем $r(x, y) = \beta \log \frac{\pi^\ast(y \mid x)}{\pi_{\text{ref}}(y \mid x)} + \beta \log Z(x)$. Подстановка этого неявного вознаграждения в модель предпочтений Брэдли-Терри $P(y_w \succ y_l) = \sigma(r(x, y_w) - r(x, y_l))$ приводит к сокращению трудновычислимых членов $Z(x)$, что дает функцию потерь DPO в явном виде:

$$\mathcal{L}_{\text{DPO}} = -\log \sigma\!\left(\beta \log \frac{\pi_\theta(y_w \mid x)}{\pi_{\text{ref}}(y_w \mid x)} - \beta \log \frac{\pi_\theta(y_l \mid x)}{\pi_{\text{ref}}(y_l \mid x)}\right)$$

- Математически это эквивалентно RLHF, но объединяет модель вознаграждения и RL-обучение в один шаг обучения с учителем.

- Выражение внутри сигмоиды можно интерпретировать так: «увеличить относительную вероятность предпочтительного ответа и уменьшить относительную вероятность менее предпочтительного ответа по сравнению с эталонной моделью».

- Параметр $\beta$ контролирует, насколько сильно стратегия может отклоняться от эталонной. На практике DPO проще реализовать (достаточно вычислить логарифмы вероятностей для обоих вариантов завершения текста в текущей и эталонной моделях), и он позволяет избежать нестабильности обучения PPO.

- **Конституционный ИИ** (Constitutional AI, Bai et al., 2022) автоматизирует части процесса согласования. Вместо сбора сравнений от людей он использует саму языковую модель для критики и исправления собственных ответов в соответствии с набором принципов («конституцией»), таких как «выбирай ответ, который является менее вредным». Сгенерированные ИИ сравнения затем используются для обучения на основе предпочтений (RLAIF: обучение с подкреплением на основе ИИ-отзывов).

- **Методы для длинных контекстов** решают проблему $O(n^2)$ памяти и вычислительных затрат стандартного механизма самовнимания, который ограничивает длину последовательности. Когда $n$ достигает десятков или сотен тысяч токенов, стандартное внимание становится невыполнимым.

- **Разреженное внимание** (sparse attention) заменяет плотную матрицу внимания $n \times n$ на разреженную структуру, где каждый токен учитывает только подмножество других токенов. Распространенные паттерны включают **локальное внимание** (каждый токен учитывает фиксированное окно соседей), **шаговое внимание** (учет каждого $k$-го токена) и **случайное внимание** (учет случайного подмножества). Комбинации этих паттернов (используемые в BigBird, Longformer) достигают сложности $O(n)$ или $O(n \sqrt{n})$, сохраняя при этом способность улавливать как локальные, так и глобальные зависимости.

![Схемы разреженного внимания](../images/sparse_attention_patterns.svg)

- **Внимание со скользящим окном (Sliding window attention)** ограничивает каждый токен так, чтобы он учитывал только предыдущие $w$ токенов (свое локальное окно). Это дает сложность $O(nw)$ вместо $O(n^2)$, но информация на больших дистанциях должна распространяться через перекрывающиеся окна по слоям. При $L$ слоях и размере окна $w$ эффективное рецептивное поле составляет $L \times w$ токенов.

- **Кольцевое внимание (Ring attention)** распределяет длинные последовательности между несколькими устройствами, организуя их в кольцевую топологию. Каждое устройство хранит фрагмент последовательности и вычисляет внимание для своего фрагмента, одновременно передавая блоки ключей-значений следующему устройству в кольце. Это позволяет совмещать вычисления с передачей данных и поддерживает последовательности произвольной длины, ограниченные только суммарным объемом памяти всех устройств, а не памятью одного из них.

- **Модели с расширенной памятью (Memory-augmented models)** увеличивают контекст, оснащая трансформер внешним банком памяти. На каждом слое модель может считывать данные из этой памяти и записывать их в нее с помощью механизма внимания. Модели Memorizing Transformers кэшируют пары ключ-значение из предыдущих фрагментов и учитывают их в последующих, эффективно расширяя контекст за пределы обучающего окна. Извлечение является приближенным (используется поиск $k$-ближайших соседей по кэшированным ключам) для поддержания эффективности.

- Приведенные выше методы являются **архитектурными** решениями для работы с длинным контекстом. Не менее важно то, как модели **обучаются** эффективно использовать длинные контексты.

- **Прогрессивное расширение контекста (Progressive context extension)** — стандартный подход. Обучение на очень длинных последовательностях с самого начала является непомерно дорогим ($O(n^2)$ затраты на внимание), поэтому модели сначала проходят предварительное обучение на короткой длине контекста (обычно 4–8 тыс. токенов), а затем **продолженное предварительное обучение** поэтапно расширяет его до целевой длины.

- Llama 3.1 расширяет контекст с 8 тыс. до 128 тыс. токенов на 800 млрд токенов с постепенно увеличивающейся длиной последовательности. DeepSeek-V3 обучается на 4 тыс., затем расширяется до 32 тыс., а после — до 128 тыс.

- Каждый этап использует умеренное количество токенов (по сравнению с полным бюджетом предварительного обучения), поскольку модели нужно лишь научиться использовать более длинные позиции, а не переучивать язык как таковой.

- При расширении необходимо корректировать позиционное кодирование. **Интерполяция RoPE (RoPE interpolation)** масштабирует индексы позиций вниз, чтобы модель видела те же углы поворота, на которых она обучалась, просто распределенные по более длинной последовательности. Если модель обучалась на длине $L$, а вы хотите расширить ее до $L' = 4L$, вы делите все индексы позиций на 4.

- Это означает, что модель никогда не видит углов поворота, с которыми она не сталкивалась, но эффективное разрешение между соседними позициями падает.

- **Экстраполяция RoPE (RoPE extrapolation)** оставляет исходные индексы позиций без изменений и просто применяет RoPE к позициям за пределами $L$, полагаясь на то, что модель обобщит данные на не встречавшиеся ранее углы.

- Интерполяция гораздо стабильнее; экстраполяция быстро деградирует без корректировки базовой частоты (ABF).

- **YaRN** (Yet another RoPE extensioN) улучшает наивную интерполяцию, признавая, что не ко всем размерностям RoPE следует относиться одинаково.

- Высокочастотные размерности (малые $i$ в $\theta_i = \theta_{\text{base}}^{-2i/d}$) совершают много оборотов в пределах обучающей длины и хорошо поддаются экстраполяции.

- Низкочастотные размерности (большие $i$) вращаются медленно и более чувствительны к расширению длины.

- YaRN интерполирует только низкочастотные размерности, экстраполирует высокочастотные и применяет температурное масштабирование $t$ к логитам внимания, чтобы компенсировать сдвиг распределения:

$$\text{score}'_{ij} = \frac{q_i^T k_j}{t \sqrt{d_k}}$$

- где $t > 1$ сглаживает распределение внимания, не позволяя модели слишком сильно фокусироваться на ближайших токенах, когда позиционные сигналы сжаты.

- **Курирование данных для длинного контекста** — критическая и часто недооцениваемая задача. Большинство корпусов для предварительного обучения состоят из коротких документов (новостные статьи, веб-страницы, посты в социальных сетях).

- Обучение на длинном контексте требует набора данных, который действительно задействует все окно контекста: книги, репозитории кода, длинные научные статьи, логи многоходовых диалогов и тематически связанные документы, объединенные вместе.

- Если модель обучается только на коротких документах, дополненных или упакованных для заполнения окна контекста, она учится игнорировать отдаленные токены, поскольку они никогда не бывают релевантными.

- **Упаковка последовательностей (Sequence packing)** — метод повышения эффективности обучения: несколько документов объединяются в одну обучающую последовательность, чтобы избежать потерь на заполнении (padding), при этом маски внимания предотвращают внимание между документами.

- Для обучения на длинном контексте стратегия упаковки имеет значение: упаковка множества несвязанных коротких документов учит модель тому, что отдаленные токены — это шум, в то время как упаковка меньшего количества по-настоящему длинных документов учит ее использовать весь контекст целиком.

- Известный тип сбоя — феномен **«потери в середине» (lost in the middle)** (Liu et al., 2023): языковые модели склонны эффективно использовать информацию в начале и конце окна контекста, но испытывают трудности с информацией, размещенной в середине.

- Это напоминает эффект края в человеческой памяти (первичность и недавность).

- Он возникает частично из-за распределений обучающих данных (важная информация часто находится в начале или конце документов), а частично из-за паттернов внимания, которые концентрируются на ближайших и начальных токенах.

- Обучение на длинном контексте с разнообразным размещением ключевой информации смягчает, но не решает эту проблему полностью.

- Тестирование **«иголка в стоге сена» (Needle-in-a-haystack)** проверяет, может ли модель извлечь конкретный факт («иголку»), помещенный в различные позиции внутри длинного отвлекающего контекста («стога сена»).

- Модель с подлинной способностью к работе с длинным контекстом должна достигать почти идеального извлечения независимо от того, где помещена иголка.

- Этот тест четко выявляет эффект потери в середине и используется для бенчмаркинга методов расширения контекста.

- **Fine-tuning на длинном контексте** после предварительного обучения использует целевые данные SFT: длинные многоходовые диалоги, ответы на вопросы по документам с доказательствами, разбросанными по тысячам токенов, суммаризация длинных текстов и понимание кода на уровне репозитория.

- Qwen3 использует **Dual Chunk Attention (DCA)** на этом этапе, который обрабатывает длинные последовательности как пары фрагментов, где внимание внутри фрагмента является полным, а внимание между фрагментами — эффективным, что позволяет увеличить эффективную емкость последовательности в 4 раза во время fine-tuning.

- **Модели в пространстве состояний (State Space Models, SSM)** предлагают принципиально иной подход к моделированию длинных последовательностей. Вместо модификации механизма внимания (attention) они полностью заменяют его линейной динамической системой, вдохновленной теорией управления в непрерывном времени.

- SSM отображает входную последовательность $u(t)$ в выходную $y(t)$ через скрытое состояние $x(t) \in \mathbb{R}^N$, которое описывается уравнениями:

$$x'(t) = Ax(t) + Bu(t), \quad y(t) = Cx(t) + Du(t)$$

- где $A \in \mathbb{R}^{N \times N}$ — матрица перехода состояний, $B \in \mathbb{R}^{N \times 1}$ — проекция входа, $C \in \mathbb{R}^{1 \times N}$ — проекция выхода, а $D$ — skip-connection (пропускающая связь).

- Чтобы применить это к дискретным последовательностям (токенам), непрерывная система **дискретизируется** с использованием шага $\Delta$. Дискретизация с фиксацией нулевого порядка дает:

$$\bar{A} = \exp(\Delta A), \quad \bar{B} = (\Delta A)^{-1}(\exp(\Delta A) - I) \cdot \Delta B$$

- Дискретное рекуррентное соотношение принимает вид $x_k = \bar{A} x_{k-1} + \bar{B} u_k$, $y_k = C x_k + D u_k$, что выглядит как RNN: обработка одного токена за раз при наличии скрытого состояния.

- В отличие от RNN, эту рекурсию можно также развернуть в виде **глобальной свёртки**: поскольку система линейна, выход равен $y = \bar{K} \ast u$, где ядро $\bar{K} = (C\bar{B}, \, C\bar{A}\bar{B}, \, C\bar{A}^2\bar{B}, \ldots)$ зависит только от фиксированных параметров.

- Этот **двойственный взгляд** — рекурсия для эффективного авторегрессионного инференса ($O(1)$ на шаг) и свёртка для эффективного параллельного обучения ($O(n \log n)$ через БПФ) — является ключевой идеей SSM.

![Двойственный взгляд SSM: рекурсия для инференса, свёртка для обучения и селективное расширение Mamba](../images/ssm_dual_view.svg)

- **S4** (Structured State Spaces for Sequence Modeling, Gu et al., 2022) сделала SSM практически применимыми, решив основную вычислительную проблему: матрица состояний $A$ должна улавливать долгосрочные зависимости, но ее наивная параметризация ведет к затуханию или взрыву динамики (та же проблема, что и у обычных RNN).

- В S4 матрица $A$ инициализируется с помощью матрицы **HiPPO** (High-order Polynomial Projection Operators), которая выводится из теории оптимальной полиномиальной аппроксимации непрерывных сигналов. Матрица HiPPO обладает специфической структурой, которая доказуемо позволяет состоянию поддерживать сжатое представление всей истории входных данных с плавным затуханием:

```math
A_{nk} = -\begin{cases} (2n+1)^{1/2}(2k+1)^{1/2} & \text{if } n > k \\ n+1 & \text{if } n = k \\ 0 & \text{if } n < k \end{cases}
```

- Эта нижнетреугольная структура гарантирует, что состояние выступает в роли онлайн-аппроксимации входного сигнала с использованием полиномов Лежандра. Вычисление $\bar{A}^k$ для длинных ядер обходится дорого, поэтому S4 использует тот факт, что матрица HiPPO может быть разложена на сумму низкоранговых и диагональных членов, что позволяет вычислять ядро за $O(n \log n)$.

- **Mamba** (Gu and Dao, 2023) вводит критически важное новшество — **селективные пространства состояний**: параметры SSM становятся зависимыми от входа. В S4 матрицы $A$, $B$, $C$ и шаг $\Delta$ фиксированы — одна и та же динамика применяется к каждому токену независимо от его содержания. Mamba делает $B$, $C$ и $\Delta$ функциями входа:

$$B_k = \text{Linear}(u_k), \quad C_k = \text{Linear}(u_k), \quad \Delta_k = \text{softplus}(\text{Linear}(u_k))$$

- Эта селективность позволяет модели решать на каждой позиции, какую информацию сохранить в состоянии, а какую проигнорировать — аналогично тому, как механизм внимания выбирает релевантные токены, но без квадратичных затрат. Шаг $\Delta_k$ управляет «гейтом»: большое значение $\Delta$ заставляет состояние сильно интегрировать текущий вход (непрерывная динамика делает большой шаг, фактически сбрасывая состояние), в то время как малое $\Delta$ сохраняет существующее состояние и игнорирует текущий вход.

- Компромисс заключается в том, что зависимые от входа параметры нарушают свёрточный взгляд (ядро больше не является фиксированным), поэтому Mamba не может использовать обучение на основе БПФ. Вместо этого она использует алгоритм **параллельного сканирования с учетом особенностей оборудования (hardware-aware parallel scan)**, который использует ассоциативность рекурсии: обновление состояния $(x_k, u_k) \mapsto x_{k+1}$ можно выразить как последовательность ассоциативных операций и распараллелить с помощью префиксной суммы (scan), аналогично параллельному префиксному сложению в проектировании оборудования. Это выполняется за время $O(n)$ с глубиной $O(\log n)$ на GPU, что почти соответствует эффективности свёртки.

- Mamba достигает инференса, который действительно имеет сложность $O(1)$ на токен (просто обновление состояния фиксированного размера, без KV-кеша, растущего вместе с контекстом), что делает ее фундаментально более эффективной по памяти, чем трансформеры, на длинных последовательностях. Размер состояния $N$ (обычно 16) намного меньше, чем KV-кеш трансформера, который хранит $O(n \cdot d)$ значений. На практике Mamba соответствует качеству трансформеров или превосходит его при том же количестве параметров на бенчмарках языкового моделирования, обеспечивая значительно более быстрый инференс на длинных последовательностях.

- **Гибридные архитектуры** сочетают слои SSM со слоями внимания, используя SSM для большинства слоев (эффективное распространение на большие расстояния) и добавляя несколько слоев внимания (точное извлечение на основе содержания). Модели, такие как Jamba и Zamba, чередуют блоки Mamba и трансформера, достигая лучшего качества, чем чистые SSM, при сохранении большей части преимуществ в эффективности инференса. Это говорит о том, что внимание и SSM обладают взаимодополняющими возможностями: SSM превосходно справляются с плавным распространением состояния на большие расстояния, в то время как внимание эффективно для точного поиска, зависящего от содержания.

- **Генерация с дополнением извлечением (Retrieval-Augmented Generation, RAG)** решает проблему ограниченности знаний языковых моделей, предоставляя им доступ к внешней базе знаний во время инференса. Вместо того чтобы полагаться исключительно на знания, закодированные в параметрах модели во время обучения, RAG извлекает релевантные документы и обусловливает генерацию на них.

- Классическая **архитектура «ретривер-ридер»** состоит из двух компонентов. **Ретривер (retriever)** принимает запрос и извлекает $k$ наиболее релевантных фрагментов из корпуса. **Ридер (reader)** (языковая модель) генерирует ответ, обусловленный как запросом, так и извлеченными фрагментами. Ретривер может использовать разреженные методы (BM25, который расширяет TF-IDF из файла 02) или плотные методы.

- **Dense passage retrieval (DPR)** использует архитектуру с двумя энкодерами: один энкодер отображает вопросы в векторы, другой — фрагменты текста (пассажи) в векторы. Оба обычно основаны на BERT. На этапе индексации все фрагменты кодируются и сохраняются. На этапе запроса кодируется вопрос, а ближайшие фрагменты находятся с помощью поиска приближенных ближайших соседей (например, FAISS). Метрикой сходства является скалярное произведение векторов вопроса и фрагмента.

- **Стратегии разбиения (chunking strategies)** существенно влияют на качество поиска. Документы должны быть разделены на фрагменты, достаточно малые для обработки поисковым механизмом, но достаточно большие, чтобы содержать законченные мысли. Разбиение на фрагменты фиксированного размера (например, 256 токенов с перекрытием в 50 токенов) просто в реализации, но может неудачно разрывать предложения. Семантическое разбиение выполняет разделение по границам абзацев или разделов. Иерархическое разбиение создает дерево суммаризаций с разной степенью детализации.

![Архитектура RAG](../images/rag_architecture.svg)

- RAG предоставляет несколько преимуществ: базу знаний можно обновлять без переобучения модели, модель может ссылаться на источники, а галлюцинации уменьшаются, поскольку модель может опираться на найденный текст при формировании ответов. Основными проблемами являются качество поиска (если найдены неверные фрагменты, модель может уверенно дать неверные ответы) и задержка (поиск добавляет дополнительный этап к инференсу).

- **Спекулятивное декодирование (speculative decoding)** ускоряет авторегрессионную генерацию за счет использования небольшой быстрой **черновой модели (draft model)** для предложения нескольких токенов параллельно, которые затем проверяются большой **целевой моделью (target model)** за один прямой проход.

- Алгоритм работает следующим образом: черновая модель авторегрессионно генерирует $k$ токенов-кандидатов (это происходит быстро, так как черновая модель невелика).

- Затем целевая модель оценивает все $k$ токенов одновременно за один прямой проход (это эффективно, так как вычисления выполняются батчами).

- Для каждого токена-кандидата $t$, выбранного из распределения черновой модели $p_d(t)$, он принимается с вероятностью $\min(1, \, p_{\text{target}}(t) / p_d(t))$. Если токен отклоняется, выполняется перевыборка исправленного токена из **скорректированного распределения** $p_{\text{adj}}(t) = \max(0, \, p_{\text{target}}(t) - p_d(t))$, которое затем нормализуется.

- Эта схема принятия-отклонения гарантирует, что выходное распределение идентично распределению, полученному только с помощью целевой модели.

- Чтобы понять почему, рассмотрим эффективную вероятность генерации токена $t$. Он может быть принят напрямую (вероятность $p_d(t) \cdot \min(1, p_{\text{target}}(t)/p_d(t))$) или получен в результате перевыборки.

- Для токенов, где $p_{\text{target}}(t) \leq p_d(t)$, прямое принятие дает вклад $p_{\text{target}}(t)$. Для токенов, где $p_{\text{target}}(t) > p_d(t)$, прямое принятие дает вклад $p_d(t)$, а перевыборка дает остаток $p_{\text{target}}(t) - p_d(t)$ (после учета вероятности отклонения).

- В обоих случаях общая вероятность генерации $t$ равна $p_{\text{target}}(t)$. Черновая модель влияет только на скорость, а не на качество.

![Спекулятивное декодирование](../images/speculative_decoding.svg)

- Ускорение зависит от коэффициента принятия: если черновая модель хорошо согласована с целевой, большинство токенов принимаются, и время выполнения примерно соответствует времени работы черновой модели. Типичное ускорение составляет 2–3 раза без потери качества.

- **Medusa** (Cai et al., 2024) использует другой подход: вместо отдельной черновой модели она добавляет несколько легковесных предсказательных «голов» к самой целевой модели. Каждая «голова» одновременно предсказывает токен на разной будущей позиции ($k = 1, 2, 3, \ldots$ шагов вперед). На каждом шаге Medusa предлагает несколько вариантов продолжения, используя древовидную структуру, а один прямой проход через слои внимания целевой модели проверяет, какие из них согласованы. Это полностью избавляет от необходимости в отдельной черновой модели.

- Методы **параллельной генерации** в более широком смысле направлены на преодоление узкого места последовательного авторегрессионного декодирования. Декодирование Якоби инициализирует все позиции догадками и итеративно уточняет их параллельно до сходимости, рассматривая генерацию как итерацию поиска неподвижной точки. Неавторегрессионные модели (NAT) генерируют все токены одновременно за один прямой проход, но обычно страдают от снижения качества и требуют таких методов, как итеративное уточнение, CTC-функция потерь или дистилляция знаний от авторегрессионных учителей для сокращения разрыва.

- Перечисленные выше методы — выравнивание (alignment), работа с длинным контекстом, поиск, эффективное декодирование, модели в пространстве состояний — объединяются в современных LLM, используемых в продакшне.

- В оставшейся части этого файла рассматриваются архитектурные инновации в передовых моделях, демонстрирующие, как теоретические идеи из файлов 01–04 и описанные выше методы сочетаются на практике.

- **Групповое внимание к запросам (Grouped Query Attention, GQA)** — наиболее широко используемая техника повышения эффективности внимания. Стандартное многоголовое внимание (MHA) поддерживает отдельные проекции ключей и значений для каждой головы, требуя хранения $n_{\text{heads}} \times d_{\text{head}}$ значений на токен. GQA группирует несколько голов запросов, чтобы они использовали общую голову ключей-значений.

- При 64 головах запросов и 8 головах KV (стандартная конфигурация в Llama 3, Qwen, Gemma) каждая голова KV разделяется между 8 головами запросов, что сокращает KV-кеш в 8 раз по сравнению с MHA.

- Качество вывода практически идентично MHA, поскольку запросы по-прежнему могут обращаться к различным паттернам, просто они используют одно и то же подпространство ключей-значений. Многозапросное внимание (MQA) является крайним случаем с одной головой KV для всех запросов, но GQA обеспечивает лучший баланс между качеством и эффективностью.

- **Многоголовое латентное внимание (Multi-head Latent Attention, MLA)**, представленное в DeepSeek-V2, достигает еще более агрессивного сжатия KV-кеша. Вместо кеширования полных проекций ключей и значений (даже при использовании GQA), MLA проецирует скрытое состояние вниз в низкоранговый **латентный вектор** $c_t \in \mathbb{R}^{d_c}$, где $d_c \ll n_{\text{heads}} \times d_{\text{head}}$:

$$c_t = W_{\text{down}} \, h_t$$

- Кешируется только этот сжатый вектор. Во время работы механизма внимания полные представления ключей и значений восстанавливаются через проекцию вверх: $k_t = W_{\text{up}}^K c_t$, $v_t = W_{\text{up}}^V c_t$. В DeepSeek-V3 (671 млрд параметров всего, 37 млрд активных) размерность сжатия составляет $d_c = 512$ против $128 \times 128 = 16{,}384$ для полного MHA, что дает сокращение KV-кеша на 93%.

- Тонкий момент: стандартный RoPE зависит от позиции и несовместим с общим сжатием, поэтому в MLA используется **развязанный RoPE (decoupled RoPE)**: небольшой отдельный поток запросов и ключей (по 64 размерности на голову) передает позиционную информацию через RoPE, в то время как основная часть представления проходит через путь сжатого латентного пространства.

![Стратегии кеширования KV-пар в механизме внимания: сравнение MHA, GQA и MLA](../images/mla_vs_gqa.svg)

- **Позиционное кодирование в больших масштабах** значительно отошло от исходной синусоидальной схемы. Все передовые модели используют **RoPE** (файл 04), но с ключевыми модификациями для длинного контекста. Базовая частота $\theta_{\text{base}}$ в исходной формуле RoPE $\theta_i = \theta_{\text{base}}^{-2i/d}$ обычно составляет 10 000, что ограничивает экстраполяцию за пределы длины обучающей выборки.

- **Корректировка базовой частоты (ABF)** просто увеличивает $\theta_{\text{base}}$ до 500 000 (Llama 3) или 1 000 000 (Qwen3, Gemma 3), растягивая периоды вращения, чтобы модель совершала меньше полных оборотов во время обучения и могла лучше экстраполировать данные.

- **YaRN** (Yet another RoPE extensioN) применяет частотно-зависимую интерполяцию: низкочастотные размерности интерполируются (масштабируются вниз), высокочастотные — экстраполируются, а температурный коэффициент корректирует распределение внимания. DeepSeek-V3, Qwen и Kimi K2 используют расширение на основе YaRN для достижения контекста 128K в моделях, предварительно обученных на 4K–8K.

- **iRoPE** (чередующийся RoPE), представленный в Llama 4, использует более радикальный подход: каждый 4-й слой внимания вообще не использует позиционное кодирование (NoPE), в то время как остальные слои используют стандартный RoPE с блочным вниманием.

- Слои NoPE могут учитывать все позиции без какого-либо позиционного смещения, тогда как слои RoPE обеспечивают локальный порядок. В сочетании с температурным масштабированием при инференсе это позволяет Llama 4 Scout работать с контекстным окном в 10 млн токенов — на порядки больше, чем любой подход на основе чистого RoPE.

- **Смесь экспертов (MoE) в больших масштабах** стала доминирующей архитектурой для передовых моделей (в файле 04 были представлены основы MoE). Ключевыми проектными решениями являются количество экспертов, разреженность маршрутизации и балансировка нагрузки.

- **Разреженность маршрутизации** значительно варьируется: DeepSeek-V3 использует 256 экспертов с маршрутизацией top-8 (разреженность 32x), Qwen3 использует 128 экспертов с top-8 (разреженность 16x), Mixtral использует 8 экспертов с top-2 (разреженность 4x), а Llama 4 Maverick использует 128 экспертов с top-1 плюс одного общего эксперта (разреженность 128x).

- Более высокая разреженность означает большее общее количество параметров при тех же активных вычислительных затратах, но требует более тщательной балансировки нагрузки и инфраструктуры связи.

- **Балансировка нагрузки без вспомогательной функции потерь** (DeepSeek-V3) заменяет традиционную функцию потерь для балансировки нагрузки (файл 04), которая, как выяснилось, снижает качество модели. Вместо этого каждый эксперт поддерживает динамический коэффициент смещения, корректируемый на каждом шаге обучения: у перегруженных экспертов смещение уменьшается (они получают меньше токенов), у недогруженных — увеличивается. Это позволяет добиться сбалансированной маршрутизации без вспомогательных функций потерь, искажающих основной сигнал обучения.

- **Общие эксперты** присутствуют в большинстве архитектур MoE: это один или несколько экспертных слоев FFN, которые обрабатывают каждый токен независимо от маршрутизации. Они обрабатывают общие паттерны, необходимые всем токенам (базовый синтаксис, служебные слова), освобождая маршрутизируемых экспертов для специализации. Llama 4 использует 1 общего эксперта плюс 1 маршрутизируемого эксперта на токен (очень разреженная модель); DeepSeek-V3 использует 1 общего и 8 маршрутизируемых.

- **Чередование плотных слоев и слоев MoE** — еще одно направление проектирования. Gemma 2 и 3 чередуют слои локального и глобального внимания (соотношение 5:1 в Gemma 3, где локальные слои используют скользящее окно в 1024 токена, а полный контекст 128K кешируется только в глобальных слоях).

- Llama 4 Maverick чередует плотные слои FFN со слоями MoE. Kimi K2 использует слои с гибридной разреженностью (один плотный слой, вкрапленный между экспертными слоями). Такая гетерогенная архитектура позволяет разным слоям выполнять разные функции.

- **Многотокеновое предсказание (MTP)**, используемое в DeepSeek-V3, обучает модель предсказывать не только следующий токен, но и токен, идущий за ним. В каждой позиции вторичный модуль предсказания (разделяющий эмбеддинги с основной моделью) предсказывает один дополнительный будущий токен. Функция потерь MTP имеет вес 0,1–0,3 относительно основной функции потерь предсказания следующего токена. Помимо улучшения качества представлений во время обучения, головы MTP могут служить «черновыми» головами для спекулятивного декодирования во время инференса, обеспечивая бесплатное ускорение.

- **Дистилляция знаний** — это стратегия обучения, при которой выходы большой «учительской» модели направляют обучение меньшей «ученической» модели. Gemma 2 и 3 активно используют дистилляцию: меньшие модели (2B, 4B) обучаются на объеме данных, в 50 раз превышающем вычислительно-оптимальный, с использованием вероятностных распределений учителя в качестве «мягких» целевых значений. Именно поэтому Gemma 3-4B по качеству соответствует Gemma 2-27B.

- Функция потерь дистилляции заменяет или дополняет стандартную кросс-энтропию: ученик минимизирует дивергенцию Кульбака — Лейблера между своим выходным распределением и распределением учителя:

$$\mathcal{L}_{\text{distill}} = D_{\text{KL}}(p_{\text{teacher}}(\cdot \mid x) \| p_{\text{student}}(\cdot \mid x))$$

- DeepSeek-R1 дистиллировала свою модель рассуждений на 671 млрд параметров в плотные модели размером до 1,5 млрд параметров, используя 800 тыс. отобранных примеров цепочек рассуждений (chain-of-thought), что позволило получить компактные модели с непропорционально сильными способностями к рассуждению.

- **Рассуждение через обучение с подкреплением** представляет собой наиболее значительный недавний прогресс в возможностях LLM. DeepSeek-R1 продемонстрировала, что чистое обучение с подкреплением базовой модели (без обучения с учителем) может вызвать появление цепочек рассуждений, самопроверки и исправления ошибок — поведения, которое возникает спонтанно, когда модель поощряется за правильные итоговые ответы.

- DeepSeek-R1 использует **GRPO** (Group Relative Policy Optimisation), которая исключает необходимость в сети оценки (value network), требуемой в PPO. Для каждого промпта GRPO выбирает группу из $G$ выходов, вычисляет их вознаграждения и нормализует преимущества (advantages) внутри группы:

$$A_i = \frac{r_i - \text{mean}(r_1, \ldots, r_G)}{\text{std}(r_1, \ldots, r_G)}$$

- Затем градиент стратегии использует эти групповые относительные преимущества с обрезанной целевой функцией (аналогично обрезке в PPO).

- Устранение сети критика сокращает требования к памяти и вычислительным ресурсам при обучении с подкреплением (RL) вдвое, что делает возможным обучение моделей с 671 млрд параметров с помощью RL.

- Важное проектное решение: DeepSeek-R1 использует **награды на основе правил** (проверка математических ответов по эталонной разметке, запуск тестовых случаев кода), а не нейронные модели вознаграждения, поскольку было обнаружено, что нейронные модели вознаграждения подвержены «взлому наград» (reward hacking) в таком масштабе.

- **Гибридный режим мышления Qwen3** объединяет рассуждение (с тегами `<think>` для пошаговой цепочки рассуждений) и быстрый прямой ответ в одной модели, позволяя пользователям контролировать «бюджет мышления», который обменивает задержку на глубину рассуждений.

- Это достигается путем обучения как на данных с рассуждениями, так и без них, а не через отдельные чекпоинты модели.

- **Стабилизация обучения в большом масштабе** требует новых методов, выходящих за рамки стандартных практик. **Logit soft-capping** (Gemma 2) пропускает оценки внимания через $s \cdot \tanh(\text{logits} / s)$ с мягким ограничением $s$ (обычно 30–50), чтобы предотвратить неограниченный рост.

- **QK-Norm** (Qwen3) применяет RMSNorm к векторам запросов и ключей перед вычислением оценок внимания, заменяя необходимость в смещении QKV. **QK-Clip** (оптимизатор MuonClip модели Kimi K2) отслеживает максимальный логит внимания во время обучения и масштабирует матрицы весов запросов-ключей, когда они превышают пороговое значение, что обеспечивает стабильное предварительное обучение моделей с 1 трлн параметров без единого случая нестабильности.

- **Обучение со смешанной точностью FP8** (DeepSeek-V3) использует 8-битные числа с плавающей запятой для вычислительно интенсивных матричных умножений при прямом и обратном проходах, сохраняя при этом мастер-веса в более высокой точности.

- Это примерно удваивает пропускную способность по сравнению с обучением в BF16/FP16 при пренебрежимо малой потере качества. DeepSeek-V3 обучила свою модель с 671 млрд параметров всего за 2,8 млн GPU-часов H800 — малую долю от времени обучения сопоставимых моделей — во многом благодаря этому и другим инженерным оптимизациям.

1. Реализуйте пайплайн генерации с дополнением поиском (RAG) с нуля. Проиндексируйте набор документов с помощью TF-IDF (файл 02), извлеките наиболее релевантный отрывок для запроса и добавьте его в начало промпта.
```python
import jax.numpy as jnp
import math
from collections import Counter

# Knowledge base: a set of short passages
knowledge_base = [
    "The Eiffel Tower is a wrought-iron lattice tower in Paris, France. It was constructed from 1887 to 1889 as the centerpiece of the 1889 World's Fair.",
    "The Great Wall of China is a series of fortifications built along the northern borders of China. Construction began in the 7th century BC.",
    "Photosynthesis is the process by which plants convert sunlight, water, and carbon dioxide into glucose and oxygen using chlorophyll.",
    "The theory of general relativity, published by Albert Einstein in 1915, describes gravity as the curvature of spacetime caused by mass and energy.",
    "Python is a high-level programming language known for its simple syntax and readability. It was created by Guido van Rossum and released in 1991.",
    "The mitochondria are organelles found in eukaryotic cells. They generate most of the cell's supply of ATP, used as a source of chemical energy.",
]

# Build TF-IDF index (reusing concepts from file 02)
def tokenise(text):
    return text.lower().split()

vocab = sorted(set(w for doc in knowledge_base for w in tokenise(doc)))
word2idx = {w: i for i, w in enumerate(vocab)}
V = len(vocab)
N = len(knowledge_base)

# Document frequencies
doc_freq = Counter()
for doc in knowledge_base:
    for w in set(tokenise(doc)):
        doc_freq[w] += 1

def tfidf_vector(text):
    words = tokenise(text)
    counts = Counter(words)
    vec = jnp.zeros(V)
    for w, c in counts.items():
        if w in word2idx:
            tf = 1 + math.log(c)
            idf = math.log(N / (doc_freq.get(w, 0) + 1))
            vec = vec.at[word2idx[w]].set(tf * idf)
    return vec

# Index all documents
doc_vectors = jnp.stack([tfidf_vector(doc) for doc in knowledge_base])

def cosine_sim(a, b):
    return jnp.dot(a, b) / (jnp.linalg.norm(a) * jnp.linalg.norm(b) + 1e-8)

def retrieve(query, top_k=2):
    """Retrieve top-k most relevant passages for a query."""
    q_vec = tfidf_vector(query)
    sims = jnp.array([cosine_sim(q_vec, doc_vectors[i]) for i in range(N)])
    top_indices = jnp.argsort(-sims)[:top_k]
    return [(int(i), float(sims[i]), knowledge_base[int(i)]) for i in top_indices]

# Test retrieval
queries = [
    "Who built the Eiffel Tower?",
    "How do plants make food?",
    "What did Einstein discover?",
]

for query in queries:
    results = retrieve(query, top_k=1)
    print(f"\nQuery: '{query}'")
    for idx, sim, passage in results:
        print(f"  Retrieved (sim={sim:.3f}): '{passage[:80]}...'")

    # RAG-style prompt construction
    context = results[0][2]
    rag_prompt = f"Context: {context}\n\nQuestion: {query}\nAnswer:"
    print(f"  RAG prompt:\n    {rag_prompt[:120]}...")
```

2. Реализуйте спекулятивное декодирование с игрушечной черновой и целевой моделью. Покажите, что принятый вывод соответствует распределению целевой модели.
```python
import jax
import jax.numpy as jnp

# Simulate a draft model (fast, less accurate) and target model (slow, accurate)
vocab_size = 8
seq_len = 5

key = jax.random.PRNGKey(42)

# Target model: returns logits given a sequence
def target_model(seq, key):
    """Simulated target model: produces token logits (expensive)."""
    # In practice this would be a large Transformer forward pass
    k1, k2 = jax.random.split(key)
    logits = jax.random.normal(k1, (len(seq), vocab_size)) * 2
    # Make it somewhat predictable: bias toward token (seq[-1] + 1) % vocab_size
    for i in range(len(seq)):
        logits = logits.at[i, (seq[i] + 1) % vocab_size].add(3.0)
    return logits

def draft_model(seq, key):
    """Simulated draft model: similar but noisier (cheap)."""
    k1, k2 = jax.random.split(key)
    logits = jax.random.normal(k1, (len(seq), vocab_size))
    for i in range(len(seq)):
        logits = logits.at[i, (seq[i] + 1) % vocab_size].add(2.0)
    return logits

def sample_token(logits, key):
    return jax.random.categorical(key, logits)

def speculative_decode(prefix, draft_steps=3, key=jax.random.PRNGKey(0)):
    """Speculative decoding: draft proposes, target verifies."""
    seq = list(prefix)
    total_accepted = 0
    total_proposed = 0

    for _ in range(4):  # generate 4 rounds
        key, *subkeys = jax.random.split(key, draft_steps + 3)

        # Draft model proposes draft_steps tokens
        draft_tokens = []
        draft_probs = []
        draft_seq = list(seq)
        for i in range(draft_steps):
            d_logits = draft_model(jnp.array(draft_seq), subkeys[i])
            d_probs = jax.nn.softmax(d_logits[-1])
            tok = sample_token(d_logits[-1], subkeys[i])
            draft_tokens.append(int(tok))
            draft_probs.append(d_probs)
            draft_seq.append(int(tok))

        # Target model scores all draft tokens in one pass
        target_logits = target_model(jnp.array(draft_seq), subkeys[draft_steps])
        target_start = len(seq) - 1  # position of last prefix token

        # Accept/reject each draft token
        accepted = 0
        for i in range(draft_steps):
            t_probs = jax.nn.softmax(target_logits[target_start + i])
            d_prob = draft_probs[i][draft_tokens[i]]
            t_prob = t_probs[draft_tokens[i]]

            # Accept with probability min(1, target_prob / draft_prob)
            accept_prob = jnp.minimum(1.0, t_prob / (d_prob + 1e-10))
            key, accept_key = jax.random.split(key)
            if jax.random.uniform(accept_key) < accept_prob:
                seq.append(draft_tokens[i])
                accepted += 1
            else:
                # Reject: sample from adjusted distribution
                key, resample_key = jax.random.split(key)
                adjusted = jnp.maximum(0, t_probs - draft_probs[i])
                adjusted = adjusted / (adjusted.sum() + 1e-10)
                new_tok = jax.random.categorical(resample_key, jnp.log(adjusted + 1e-10))
                seq.append(int(new_tok))
                break

        total_accepted += accepted
        total_proposed += draft_steps

    return seq, total_accepted, total_proposed

# Run speculative decoding
prefix = [0, 1]
result_seq, accepted, proposed = speculative_decode(prefix)
acceptance_rate = accepted / proposed if proposed > 0 else 0

print(f"Prefix: {prefix}")
print(f"Generated sequence: {result_seq}")
print(f"Draft proposals: {proposed}")
print(f"Accepted: {accepted}")
print(f"Acceptance rate: {acceptance_rate:.1%}")
print(f"Speedup potential: {(accepted + proposed) / proposed:.2f}x")
```

3. Создайте простой цикл обучения DPO. Имея пары предпочтительных и менее предпочтительных вариантов завершения, обновите небольшую модель, используя функцию потерь DPO.

```python
import jax
import jax.numpy as jnp

# Tiny language model: linear projection from one-hot to logits
vocab_size = 10
seq_len = 4

key = jax.random.PRNGKey(42)
k1, k2 = jax.random.split(key)

# Current policy parameters (trainable)
theta = jax.random.normal(k1, (vocab_size, vocab_size)) * 0.1
# Reference policy parameters (frozen copy of initial theta)
theta_ref = theta.copy()

def log_prob_sequence(params, sequence):
    """Compute log P(sequence) under a simple autoregressive model."""
    total = 0.0
    for t in range(1, len(sequence)):
        # Simple: logits at position t depend on token at t-1
        logits = params[sequence[t-1]]
        log_probs = jax.nn.log_softmax(logits)
        total += log_probs[sequence[t]]
    return total

def dpo_loss(theta, theta_ref, preferred, dispreferred, beta=0.1):
    """Direct Preference Optimisation loss for one pair."""
    log_pi_w = log_prob_sequence(theta, preferred)
    log_pi_l = log_prob_sequence(theta, dispreferred)
    log_ref_w = log_prob_sequence(theta_ref, preferred)
    log_ref_l = log_prob_sequence(theta_ref, dispreferred)

    # DPO objective
    return -jax.nn.log_sigmoid(
        beta * ((log_pi_w - log_ref_w) - (log_pi_l - log_ref_l))
    )

# Preference dataset: (prompt_prefix, preferred_completion, dispreferred_completion)
preferences = [
    (jnp.array([1, 3, 5, 7]), jnp.array([1, 3, 5, 2])),  # prefer 7 over 2 at end
    (jnp.array([0, 2, 4, 6]), jnp.array([0, 2, 4, 9])),  # prefer 6 over 9
    (jnp.array([3, 3, 3, 3]), jnp.array([3, 3, 3, 0])),  # prefer repeating over 0
    (jnp.array([5, 6, 7, 8]), jnp.array([5, 6, 7, 1])),  # prefer 8 over 1
]

grad_fn = jax.jit(jax.grad(dpo_loss))
lr = 0.05

print("Training DPO...")
for epoch in range(100):
    total_loss = 0.0
    for preferred, dispreferred in preferences:
        loss = dpo_loss(theta, theta_ref, preferred, dispreferred)
        grads = grad_fn(theta, theta_ref, preferred, dispreferred)
        theta = theta - lr * grads
        total_loss += loss
    if (epoch + 1) % 20 == 0:
        avg_loss = total_loss / len(preferences)
        print(f"  Epoch {epoch+1}: avg DPO loss = {avg_loss:.4f}")

# Check: the model should now prefer the preferred completions
print("\nPreference check after DPO training:")
for preferred, dispreferred in preferences:
    lp_w = log_prob_sequence(theta, preferred)
    lp_l = log_prob_sequence(theta, dispreferred)
    print(f"  Preferred {list(preferred.astype(int))}: logP={lp_w:.3f}  "
          f"Dispreferred {list(dispreferred.astype(int))}: logP={lp_l:.3f}  "
          f"{'correct' if lp_w > lp_l else 'WRONG'}")
```
