# Transformer 与语言模型

*Transformer 用 self-attention 取代循环，成为语言理解与生成的主导架构。本文件涵盖 BERT、GPT、T5、位置编码（sinusoidal、RoPE）、pre-training 目标（MLM、CLM）、fine-tuning、prompt 工程和 scaling laws——现代 LLM 背后的蓝图。*

- 在第 06 章中，我们介绍了 Transformer 架构：self-attention、multi-head attention、位置编码和 encoder-decoder 结构。这里我们关注 transformer 如何被适配于具体 NLP 范式、定义现代 NLP 的模型（BERT、GPT、T5）以及让它们在大规模下实用的技术。

- 回顾核心操作：**缩放点积 attention（scaled dot-product attention）**计算 $\text{softmax}(QK^T / \sqrt{d_k}) V$，其中查询、键、值是输入的线性投影。**Multi-head attention** 运行 $h$ 个并行 attention head，每个有不同的学习投影，并拼接结果。Transformer block 用残差连接、layer normalisation 和逐位置前馈网络（第 06 章）将其包裹起来。

- 一个微妙但重要的架构选择是 **layer normalisation** 的位置。原始 Transformer 用 **post-norm**：残差和归一化在子层之后，即 $\text{LayerNorm}(x + \text{Sublayer}(x))$。

- 大多数现代模型用 **pre-norm**：在子层之前归一化，即 $x + \text{Sublayer}(\text{LayerNorm}(x))$。pre-norm 在训练时更稳定，因为残差连接通过恒等路径直接传递梯度，不受归一化影响。这使训练非常深的模型更容易，而无需仔细的学习率预热。

- 每个 Transformer block 中的**前馈子层**是一个两层 MLP，独立应用于每个 token 位置：

$$\text{FFN}(x) = W_2 \cdot \text{GELU}(W_1 x + b_1) + b_2$$

- 内部维度通常是模型维度的 4 倍（如 $d_{\text{model}} = 768$，$d_{\text{ff}} = 3072$）。这个 FFN 约占每个 block 三分之二的参数，被认为充当一个键值存储，存放训练中习得的事实知识。

- **位置编码（positional encoding）**给模型关于 token 顺序的信息，因为 attention 本身是置换等变的。原始的**正弦编码（sinusoidal encoding）**（第 06 章）用不同频率的固定正弦和余弦函数。**学习式位置 embedding** 为每个位置添加一个可训练向量（BERT 和 GPT-2 使用）。二者都是绝对编码：位置 5 无论上下文如何都得到同一个向量。

- **旋转位置编码（Rotary Position Embedding, RoPE）**通过在 2D 子空间中旋转查询和键向量来编码位置。对于一对维度 $(q_{2i}, q_{2i+1})$，旋转角度 $m\theta_i$（其中 $m$ 是位置，$\theta_i = 10000^{-2i/d}$）应用：

```math
\begin{bmatrix} q'_{2i} \\ q'_{2i+1} \end{bmatrix} = \begin{bmatrix} \cos m\theta_i & -\sin m\theta_i \\ \sin m\theta_i & \cos m\theta_i \end{bmatrix} \begin{bmatrix} q_{2i} \\ q_{2i+1} \end{bmatrix}
```

![RoPE：每个位置在 2D 子空间中以不同角度旋转查询和键向量，使 attention 分数只依赖相对位置](../images/rope_rotation.svg)

- RoPE 之美在于旋转后的查询和键之间的点积 $q'^T k'$ 只依赖相对位置 $m - n$，而不依赖绝对位置。

- 为看清这一点，把旋转写成 $q' = R_m q$ 和 $k' = R_n k$，其中 $R_m$ 是块对角旋转矩阵。attention 分数变为：

$$q'^T k' = (R_m q)^T (R_n k) = q^T R_m^T R_n \, k = q^T R_{n-m} \, k$$

- 最后一步来自旋转群的性质：$R_m^T R_n = R_{n-m}$（先反向旋转 $m$ 再正向旋转 $n$ 等于旋转 $n - m$）。

- 这意味着 attention 分数只依赖相对距离 $n - m$，而不依赖绝对位置 $m$ 和 $n$ 各自的值。

- 模型获得了一种自然的距离概念，无需任何学习到的位置参数，并能泛化到训练中未见过的序列长度。

- **ALiBi**（Attention with Linear Biases）采取更简单的方法：它根据距离给 attention 分数加上一个固定的线性惩罚，即 $\text{score}_{ij} = q_i^T k_j - m \cdot |i - j|$，其中 $m$ 是 head 相关的斜率。不同的 head 用不同斜率，使一些 head 关注局部，另一些关注全局。ALiBi 无需学习位置参数，且能很好地泛化到比训练时更长的序列。

- 基于 Transformer 的语言模型有三种主导范式：**encoder-only**、**decoder-only** 和 **encoder-decoder**。它们在模型能看见什么（attention mask）以及如何训练上有所不同。

![三种 Transformer 范式：encoder-only（BERT）用双向 attention 做分类，decoder-only（GPT）用因果 attention 做生成，encoder-decoder（T5）结合二者做 seq2seq 任务](../images/transformer_paradigms.svg)

- **BERT**（Bidirectional Encoder Representations from Transformers，Devlin 等, 2019）是典型的 encoder-only 模型。它用完全双向 attention 处理文本：每个 token 既能关注它左侧、也能关注右侧的所有其他 token。这给 BERT 丰富的上下文表示，但意味着它无法自回归地生成文本。

- BERT 用两个目标进行 pre-training。**掩码语言建模（Masked Language Modelling, MLM）**随机掩蔽 15% 的输入 token 并训练模型预测它们。在被选中的 token 中，80% 被替换为 `[MASK]` token，10% 被替换为随机词，10% 保持不变（以防模型学到只在看到 `[MASK]` 时才预测）。训练目标是：

$$\mathcal{L}_{\text{MLM}} = -\sum_{i \in \mathcal{M}} \log P(w_i \mid w_{\backslash \mathcal{M}})$$

- 其中 $\mathcal{M}$ 是被掩蔽位置的集合，$w_{\backslash \mathcal{M}}$ 是这些位置被掩蔽后的句子。这是一个**去噪（denoising）**目标：模型学习重建被破坏的输入。

![BERT 掩码语言建模：输入中 15% 的 token 被掩蔽，双向 Transformer 在被掩蔽位置预测原始 token](../images/bert_mlm.svg)

- **下一句预测（Next Sentence Prediction, NSP）**训练 BERT 预测两句话在原文中是否连续。输入开头的一个特殊 [CLS] token 用于这个二分类。NSP 被包含进来以帮助需要理解句子关系的任务（如问答），尽管后续工作（RoBERTa）表明它贡献甚微，可以去掉。

- BERT 的预训练表示通过在顶部添加任务相关 head（一个简单线性层）并对整个模型 fine-tuning 来适配下游任务。对于分类任务，使用 [CLS] token 的表示。对于 token 级任务（NER、词性标注），使用每个 token 的表示。这种 **fine-tuning** 方法把 pre-training 中学到的语言知识迁移到新任务，所需标注数据相对较少。

- **GPT**（Generative Pre-trained Transformer，Radford 等, 2018）是典型的 decoder-only 模型。它使用**因果（自回归）attention**：每个 token 只能关注更早位置（及自身）的 token。这通过在 attention 矩阵中掩蔽未来位置（在 softmax 之前把它们的分数设为 $-\infty$）来强制。训练目标是简单的**因果语言建模（causal language modelling）**：给定所有之前的 token 预测下一个。

$$\mathcal{L}_{\text{CLM}} = -\sum_{i=1}^{n} \log P(w_i \mid w_1, \ldots, w_{i-1})$$

- 这与文件 02 的 n-gram 语言模型目标相同，但用 Transformer 参数化，能以整个前文为条件，而不仅仅是最后 $k-1$ 个 token。

- **GPT-2** 将规模扩展到 15 亿参数，展示了强大的零样本性能：无需任何 fine-tuning，它就能通过以自然语言 prompt 为条件来执行任务（"Translate English to French: ..."）。

- **GPT-3**（1750 亿参数）表明仅靠规模就能实现**上下文学习（in-context learning）**：在 prompt 中提供少量输入-输出示例，模型就能在没有任何梯度更新的情况下执行新任务。

- **encoder-decoder 模型**如 **T5**（Text-to-Text Transfer Transformer，Raffel 等, 2020）把每个 NLP 任务都表述为文本到文本：输入是一个文本串（可带任务前缀如 "translate English to German:"），输出是一个文本串。encoder 用双向 attention 处理输入，decoder 自回归地生成输出，并通过 cross-attention 关注 encoder。

- T5 用 **span corruption** 进行 pre-training：随机连续的 token 跨度被替换为哨兵 token，模型必须生成原始 token。例如，"The cat sat on the mat" 可能作为输入变成 "The [X] on [Y]"，目标是 "[X] cat sat [Y] the mat"。这是 BERT 的 MLM 向跨度而非单个 token 的推广。

- **BART**（Lewis 等, 2020）是另一个用去噪目标 pre-training 的 encoder-decoder 模型，但它应用更广的破坏策略：token 掩蔽、token 删除、跨度掩蔽、句子排列和文档旋转。破坏的多样性迫使模型学习更鲁棒的表示。

- 随着语言模型变大，**全量 fine-tuning**（更新所有参数）变得不切实际：一个 1750 亿参数模型仅存储优化器状态就需要数百 GB。**参数高效微调（Parameter-Efficient Fine-Tuning, PEFT）**方法只适配一小部分参数。

- **Adapter** 在现有 Transformer 层之间插入小型瓶颈层（通常是两个带非线性的线性层：先降维到小维度，再升维回去）。只训练 adapter 权重，原始模型权重冻结。这增加不到 5% 的新参数，却在大多数任务上匹敌全量 fine-tuning 的性能。

- **LoRA**（Low-Rank Adaptation）不添加新层，而是修改权重矩阵本身。它不更新完整权重矩阵 $W$，而是学习更新的低秩分解：$W' = W + BA$，其中 $B$ 为 $d \times r$，$A$ 为 $r \times d$，$r \ll d$（通常 $r = 4$ 到 $r = 64$）。原始 $W$ 冻结，只训练 $A$ 和 $B$。在推理时，更新可以合并到原始权重中，无额外延迟：

$$W' = W + BA$$

![LoRA：冻结的权重矩阵 W 由通过小矩阵 A 和 B 的低秩路径绕过，可训练参数减少 32 倍，同时匹敌全量 fine-tuning](../images/lora_decomposition.svg)

- **Prefix tuning** 在每个 attention 层的键和值矩阵前添加一串可学习的"虚拟 token"。模型像对待真实 token 一样关注这些前导向量，只训练前缀参数。这类似于 prompt tuning，但在激活空间而非 embedding 空间操作。

- **Prompt 工程**是一种艺术：设计输入文本以在无需任何参数更新的情况下从预训练模型中引发期望行为。

    - **零样本 prompting** 用自然语言描述任务（"Classify the sentiment of the following review:"）。

    - **少样本 prompting** 在实际查询之前提供输入-输出示例。

    - **思维链（Chain-of-thought, CoT）prompting** 添加 "Let's think step by step" 或在示例中包含推理轨迹，通过引导模型分解问题，显著提升算术和逻辑推理任务的性能。

- **上下文学习（in-context learning, ICL）**是这样一种现象：大型语言模型能从 prompt 中提供的示例学习执行任务，而无需任何梯度更新。模型权重不改变；它把示例当作一种隐式规范。

- ICL 在机制上如何工作仍是一个活跃的研究问题；一种假设是 attention 层在前向传播中实现了一种梯度下降，有效地在上下文示例上"训练"。

- **Scaling laws** 描述模型大小、数据量、计算预算和性能（以 loss 衡量）之间可预测的关系。Kaplan 等（2020）发现 loss 在每个变量上服从幂律：

$$L(N) \propto N^{-\alpha_N}, \quad L(D) \propto D^{-\alpha_D}, \quad L(C) \propto C^{-\alpha_C}$$

- 其中 $N$ 是参数数量，$D$ 是数据集大小，$C$ 是计算预算。这些幂律跨越多个数量级成立，并表明单纯扩大规模就能带来可预测的改进。

![Scaling laws：loss 在双对数坐标下按幂律下降，Kaplan 和 Chinchilla 的发现表明随规模有可预测的改进](../images/scaling_laws.svg)

- **Chinchilla scaling laws**（Hoffmann 等, 2022）对此作了修正，表明大多数大模型训练不足。对于固定计算预算 $C$，最优分配同等缩放模型大小和训练数据：

$$N_{\text{opt}} \propto C^{0.5}, \quad D_{\text{opt}} \propto C^{0.5}$$

- 这意味着如果你的计算预算翻倍，应把模型大小和数据集大小都增加 $\sqrt{2}$ 倍，而不是只把模型变大。

- Kaplan 等曾建议 $N$ 比 $D$ 增长更快，这导致了非常大但训练不足的模型。Chinchilla（700 亿参数，1.4T token）以相同计算预算匹敌 Gopher（2800 亿参数，3000 亿 token）的性能，证明早期模型严重数据匮乏。

- 实用的经验法则：每个参数大约训练 20 个 token。

- **混合专家（Mixture of Experts, MoE）**是一种在不按比例增加计算的情况下扩大模型容量的架构。MoE 不用单个大的前馈层，而是使用多个**专家** FFN 层和一个**门控网络（router）**，后者为每个 token 选择激活哪些专家。

- 门控函数为每个专家计算路由分数，并选择 top-$k$（通常 $k = 1$ 或 $k = 2$）：

$$G(x) = \text{TopK}(\text{softmax}(W_g x))$$

- 只有被选中的专家处理该 token，因此计算成本随 $k$（激活专家数）而非专家总数 $E$ 缩放。一个有 8 个专家、top-2 路由的模型有稠密模型 4 倍的参数但只有 2 倍的计算。

![MoE 层：输入 token 通过 router 为每个专家计算分数，选出 top-2 专家，其输出按门控分数加权求和](../images/moe_layer.svg)

- MoE 的一个关键挑战是**负载均衡（load balancing）**：如果 router 把大多数 token 送给少数受欢迎的专家，其他专家就被浪费。训练中添加一个辅助的**负载均衡损失**以鼓励均匀的专家利用：

$$\mathcal{L}_{\text{balance}} = E \cdot \sum_{i=1}^{E} f_i \cdot p_i$$

- 其中 $f_i$ 是分配给专家 $i$ 的 token 比例，$p_i$ 是专家 $i$ 的平均路由概率。当 token 比例和概率都均匀（每项都等于 $1/E$）时，该乘积最小化。

- **专家并行（expert parallelism）**把不同专家分布到不同加速器上。在前向传播中，一次 all-to-all 通信把 token 路由到承载其被分配专家的设备，再把结果路由回来。这种通信成本是大规模 MoE 的主要工程挑战。Switch Transformer、Mixtral 和 GShard 等模型使用 MoE 以实用推理成本实现强性能。

- 构建模型只完成了一半工作；衡量它们是否有效是另一半。NLP 评估格外困难，因为语言是歧义的、主观的、开放式的。

- 一个翻译可以有许多不同方式都正确。一个摘要即使与参考不共任何一个词也可以是好的。

- 一个聊天机器人的回答可能既有帮助、又无害、还诚实，但讲道理的人会意见不一。

- **精确匹配（Exact Match, EM）**是最简单的指标：模型输出是否与黄金答案完全匹配？它用于答案短且无歧义的任务，如抽取式问答（SQuAD）或闭卷数学题。

- EM 很苛刻；"New York City" 和 "new york city" 除非应用归一化否则无法匹配——但其简洁性使其毫无歧义。

- **token 级指标**把 NLP 视为 token 级分类问题，使用第 06 章的精确率、召回率和 F1。

- **精确率（precision）**衡量模型预测的 token 中有多少正确：$P = \text{TP} / (\text{TP} + \text{FP})$。一个预测很少实体但都正确的模型精确率高。

- **召回率（recall）**衡量模型找到的黄金 token 比例：$R = \text{TP} / (\text{TP} + \text{FN})$。一个把每个 token 都预测为实体的模型召回率完美但精确率糟糕。

- **F1** 是精确率和召回率的调和平均：

$$F_1 = \frac{2PR}{P + R}$$

- 调和平均（而非算术平均）惩罚不平衡：若 $P$ 或 $R$ 之一很低，$F_1$ 就低。对 NER（文件 02），F1 按实体类型计算，再跨类型做宏平均。对词性标注，token 级准确率更常见，因为每个 token 都有标签。

- **跨度级 F1（span-level F1）**（用于 SQuAD）把预测跨度中的 token 集合与黄金跨度中的 token 集合比较。这比精确匹配更宽容：如果黄金答案是 "the Eiffel Tower" 而模型预测 "Eiffel Tower"，span F1 很高（5 个中有 4 个重叠 token），尽管 EM 为零。

- **BLEU**（Bilingual Evaluation Understudy，Papineni 等, 2002）是机器翻译的经典指标。它衡量候选翻译与一个或多个参考翻译之间的 n-gram 重叠。分数结合多级 n-gram 精确率（unigram 到 4-gram）与简短惩罚：

$$\text{BLEU} = \text{BP} \cdot \exp\!\left(\sum_{n=1}^{N} w_n \log p_n\right)$$

- 其中 $p_n$ 是**修改后的 n-gram 精确率**：候选中每个 n-gram 的计数被截断为它在任一参考中的最大计数，以防止 "the the the the" 这样的退化候选得高分。权重 $w_n$ 通常均匀（$w_n = 1/N$，$N = 4$）。

- **简短惩罚（brevity penalty）** $\text{BP} = \min(1, \exp(1 - r/c))$ 惩罚比参考短的候选（$c$ 是候选长度，$r$ 是参考长度）。否则，模型可以通过输出极少、极安全的词获得高精确率。

- BLEU 在语料库级（多句平均）与人类判断的相关性合理，但在句子级相关性差。

- 它奖励精确的 n-gram 匹配，却遗漏合法的复述："the cat is on the mat" 和 "a feline sits atop the rug" 尽管意义相同，bigram 重叠为零。

- BLEU 还完全忽略召回率——一个只产生最常见词的候选在精确率上得分高。

- **ROUGE**（Recall-Oriented Understudy for Gisting Evaluation，Lin, 2004）是摘要的标准指标。与强调精确率的 BLEU 不同，ROUGE 强调召回率：参考的 n-gram 中有多少出现在候选中？

- **ROUGE-N** 计算 n-gram 的召回率：$\text{ROUGE-N} = \frac{|\text{n-grams}_{\text{ref}} \cap \text{n-grams}_{\text{cand}}|}{|\text{n-grams}_{\text{ref}}|}$。ROUGE-1（unigram）和 ROUGE-2（bigram）最常见。

- ROUGE-L 使用候选与参考之间的**最长公共子序列（longest common subsequence, LCS）**，它捕捉句子级的词序而不要求连续匹配。

- LCS 长度除以参考长度得到召回率，除以候选长度得到精确率，F-measure 把二者结合。

- LCS 用动态规划在 $O(mn)$ 时间内计算（类似文件 02 的编辑距离）：

$$R_{\text{LCS}} = \frac{\text{LCS}(X, Y)}{m}, \quad P_{\text{LCS}} = \frac{\text{LCS}(X, Y)}{n}, \quad F_{\text{LCS}} = \frac{(1 + \beta^2) R_{\text{LCS}} P_{\text{LCS}}}{R_{\text{LCS}} + \beta^2 P_{\text{LCS}}}$$

- 其中 $m$ 和 $n$ 是参考和候选的长度，$\beta$ 通常设置为偏向召回率（$\beta \to \infty$ 给出纯召回率）。

- **METEOR**（Metric for Evaluation of Translation with Explicit ORdering，Banerjee 和 Lavie, 2005）通过纳入同义词、词干还原和词序来解决 BLEU 的弱点。

- 它首先用精确匹配、词干匹配（通过文件 02 的 Porter 词干还原）和同义词匹配（通过文件 01 的 WordNet）在候选和参考之间对齐词。

- 然后计算偏向召回率的 unigram 精确率和召回率的调和平均，并应用一个碎片化惩罚，惩罚匹配词出现顺序与参考不同的候选。

- **ChrF**（Character n-gram F-score）在字符 n-gram 而非词 n-gram 上计算 F 分数。这使其对形态变化鲁棒（对文件 01 的黏着语至关重要），并部分处理分词差异。ChrF++ 在字符 n-gram 基础上加入词 bigram。

- 它已成为机器翻译中与 BLEU 并列的推荐指标，尤其对形态丰富的语言。

- **困惑度（perplexity）**（文件 02）衡量语言模型对留出测试集的预测好坏。它是语言模型的标准内在指标：$\text{PPL} = \exp(-\frac{1}{N} \sum_{i} \log P(w_i \mid w_{<i}))$。越低越好。

- 困惑度只在用相同分词的模型之间可比，因为不同分词器对相同文本产生不同的序列长度 $N$。

- 词表更大的模型往往每个 token 的困惑度更低，但每句处理的 token 更少。

- **每字节比特数（bits-per-byte, BPB）**按文本的 UTF-8 字节数而非 token 数归一化，使其与分词无关：

```math
\text{BPB} = \frac{-\sum_{i} \log_2 P(w_i \mid w_{<i})}{\text{number of UTF-8 bytes}}
```

- **BERTScore**（Zhang 等, 2020）通过在 embedding 空间计算相似度超越了表层 n-gram 匹配。候选中的每个 token 用上下文 embedding（通常来自预训练 BERT 模型）的余弦相似度匹配到参考中最相似的 token。分数聚合为精确率、召回率和 F1：

$$R_{\text{BERT}} = \frac{1}{|r|} \sum_{r_i \in r} \max_{c_j \in c} \cos(r_i, c_j), \quad P_{\text{BERT}} = \frac{1}{|c|} \sum_{c_j \in c} \max_{r_i \in r} \cos(c_j, r_i)$$

- 其中 $r_i$ 和 $c_j$ 是参考和候选 token 的上下文 embedding。这捕捉 n-gram 指标遗漏的语义相似性："automobile" 和 "car" 得分高，因为它们的 BERT embedding 相似，尽管它们不共享任何字符。

- **BLEURT**（Sellam 等, 2020）更进一步，直接在人类质量判断上 fine-tune 一个 BERT 模型。给定一个参考-候选对，它输出一个标量质量分数。BLEURT 在合成数据（参考翻译的随机扰动，由 BLEU 和 METEOR 等指标打分）上训练，再在人类评分上 fine-tune。它与人类判断的相关性优于任何表层指标。

- **COMET**（Crosslingual Optimized Metric for Evaluation of Translation，Rei 等, 2020）是一个机器翻译的学习指标，它以源句子、参考和候选为条件——而不仅仅是参考和候选。它用一个多语言 encoder（XLM-R）嵌入三者并预测质量分数。通过看到源句子，COMET 能发现仅参考的指标遗漏的意义错误（如流畅但事实上错误的翻译）。

- **LLM-as-judge** 是大规模评估的现代方法。不针对参考计算指标，而是用一个强大的语言模型（GPT-4、Claude）作为评判者评估模型输出的质量。评判者接收输入、模型响应，可选地接收参考答案，并产生一个评分（如 1-5）或一个两两偏好（响应 A 比响应 B 好）。

- **两两比较（pairwise comparison）**（用于 Chatbot Arena）是最可靠的 LLM-as-judge 形式。评判者看到两个响应并选出更好的一个，而非赋予绝对分数。这避免了校准问题（不同评判者对"5 分之 3"的基线可能不同）。结果聚合为 **Elo 评分**（源自国际象棋），每个模型从一个基础评分开始，根据与其他模型对战的胜负增减分数。模型 $A$ 对模型 $B$ 的期望胜率为：

$$P(A \succ B) = \frac{1}{1 + 10^{(R_B - R_A) / 400}}$$

- 其中 $R_A, R_B$ 是 Elo 评分。每次比较后，评分更新：$R_A' = R_A + K(S - P(A \succ B))$，其中 $S \in \{0, 1\}$ 是实际结果，$K$ 控制更新幅度。持续击败强对手的模型评分快速上升；输给弱对手的模型评分下降。

- **位置偏差（position bias）**是 LLM 评判者的已知问题：它们倾向于偏好先呈现的响应（或在某些模型中，后呈现的响应）。**交换（swapping）**（以两种顺序各评估一次并平均结果）可缓解此问题。

- **冗长偏差（verbosity bias）**是另一个：评判者倾向于偏好更长、更详细的响应，即使简洁的回答更好。

- **自一致性（self-consistency）**检查评判者对同一输入的多次评估是否给出相同评分。方差高表示评估信号噪声大。

- **标注者间一致性（inter-annotator agreement）**（Cohen's kappa 或 Krippendorff's alpha）衡量多个评判者是否一致，为评估可靠性提供上界。

- **污染（contamination）**是一个关键问题：如果评估数据出现在模型的训练集中，基准分数被夸大且毫无意义。

- 这对在网页爬取数据上训练的 LLM 尤其棘手，因为流行的基准很可能已包含其中。缓解策略包括：使用不公开发布的留出测试集、创建定期重新生成问题的动态基准、**金丝雀字符串（canary strings）**（嵌入基准数据中的唯一标识符以检测泄漏），以及在污染与干净子集上比较性能。

- **标准 NLU 基准**跨多样任务评估语言理解。

- **GLUE**（General Language Understanding Evaluation）和 **SuperGLUE** 是多任务基准，涵盖情感（SST-2）、文本相似度（STS-B）、自然语言推理（MNLI、RTE）、共指（WSC）和问答（BoolQ）。

- 模型在每个任务上单独评估，并由一个聚合指标打分。GLUE 现被视为饱和（模型在多数任务上超越人类）；SuperGLUE 仍更具挑战。

- **MMLU**（Massive Multitask Language Understanding）用多选题跨 57 个学术科目（数学、历史、法律、医学、计算机科学等）评估知识与推理。

- 它测试模型是否在 pre-training 中吸收了广博知识。分数按科目报告，并给出宏平均。

- **MMLU-Pro** 添加了更难的多步推理题，有 10 个选项而非 4 个。

- **HellaSwag** 通过要求模型选择一个情境的最合理延续来测试常识推理。错误答案是对抗性生成的（用模型），表面上合理但语义上错误。

- **WinoGrande** 用仅相差一个词的最小对测试常识共指消解。

- **ARC**（AI2 Reasoning Challenge）用小学科学题分简单集和挑战集，测试事实和推理能力。

- **推理与数学基准**评估区分强弱 LLM 的问题求解能力。

- **GSM8K**（Grade School Math 8K）包含 8,500 道小学数学应用题，需要多步算术推理。它是基本数学推理的标准基准，也用于评估思维链 prompting（文件 04）。

- **MATH** 是更难的数据集，包含跨代数、数论、几何、计数和概率的竞赛级数学题。问题需要多步符号推理，MATH-500 是常报告的 500 题子集。

- **AIME**（American Invitational Mathematics Examination）问题是竞赛级的：正确解答需要跨越许多步骤的深度数学推理。DeepSeek-R1 在 AIME 2024 上得 79.8%，表明经过 RL 训练的推理模型（文件 05）能接近强大的人类竞赛者。

- **HumanEval** 和 **MBPP**（Mostly Basic Programming Problems）通过检查模型代码是否通过单元测试来评估代码生成。HumanEval 含 164 个带函数签名和 docstring 的 Python 题；模型必须生成函数体。

- 指标是 **pass@k**：$k$ 个生成解中至少一个通过所有测试的概率。对单样本：

$$\text{pass@}k = 1 - \frac{\binom{n-c}{k}}{\binom{n}{k}}$$

- 其中 $n$ 是生成的样本总数，$c$ 是通过的样本数。该公式修正了简单取 $k$ 个样本中最佳的偏差。

- **SWE-bench** 更进一步，评估模型能否通过修改现有代码库来解决真实 GitHub issue——这是对实际软件工程能力的更难测试。

- **GPQA**（Graduate-Level Google-Proof QA）包含生物、物理、化学的专家级问题，即使是领域专家也觉得困难。它测试模型是真正理解还是模式匹配。"Diamond" 子集最难。

- **安全与对齐基准**评估模型是否有帮助、无害且诚实。

- **TruthfulQA** 测试模型是否复述常见误解。问题被设计成使最常见的互联网答案是错的（如 "What happens if you swallow gum?"，常见流言是它停留 7 年，但真相是正常通过）。记忆了流行但错误说法的模型得分差。

- **BBQ**（Bias Benchmark for QA）测试跨年龄、性别、种族、宗教等类别的社会偏见。问题被构造成有偏见的模型会系统性地选择刻板答案。**Toxigen** 评估模型对特定人口群体生成有毒内容的倾向。

- **MT-Bench** 用 80 个精心设计的问题跨写作、角色扮演、推理、数学、编码、抽取、STEM 和人文评估多轮对话能力。一个 LLM 评判者（GPT-4）按 1-10 分打分。多轮格式测试模型能否跟进、维持上下文并处理澄清请求。

- **Chatbot Arena**（LMSYS）用真实用户对匿名模型进行盲测两两比较。用户提交 prompt 并在不知道模型身份的情况下为更好的响应投票。所得到的 Elo 排行榜被认为是对通用 LLM 质量最具生态效度的评估，因为它反映了真实用户在多样、未策划的 prompt 上的偏好。

- **AlpacaEval** 通过将模型输出与参考模型（GPT-4）在一组固定指令上比较来自动化两两评估。一个评判模型决定胜率。

- **AlpacaEval 2.0** 使用长度控制的胜率来纠正冗长偏差。

- **任务特定评估**需要为专门领域量身定制的指标。

- **词错误率（Word Error Rate, WER）**用于语音识别：$\text{WER} = (S + D + I) / N$，其中 $S$、$D$、$I$ 是替换、删除和插入错误，$N$ 是参考词数。这是编辑距离（文件 02）按参考长度归一化，应用在词级。

- **槽位 F1（Slot F1）**用于任务型对话系统，衡量模型是否从用户话语中正确抽取结构化信息（如从 "Book me a flight to Paris tomorrow" 中抽取 "destination: Paris" 和 "date: tomorrow"）。

- **引用准确率**用于 RAG 系统（文件 05），检查模型生成的引用是否真正支持所做主张。主张根据检索到的段落验证，指标统计完全支持、部分支持或不支持的主张比例。

- **评估陷阱**常见，可能使整个基准比较失效。

- **应试教学（teaching to the test）**：为基准性能而非真实能力优化。在 MMLU 风格多选题上 fine-tune 的模型会在 MMLU 上得分高，但若以开放式格式提出相同问题可能失败。

- **指标博弈（metric gaming）**：模型可被优化为产生在自动指标上得分高的输出（高 BLEU、低困惑度）而并非真正好。BLEU 最优的翻译常常是安全、通用的复述，而非自然、流畅的。

- **基准饱和（benchmark saturation）**：当模型接近或超越基准上的人类性能时，该基准不再提供信息。GLUE、SQuAD 1.1 等现已饱和。

- 该领域持续创造更难的基准，但创建、饱和、替换的循环使纵向比较困难。

- **人工评估**仍是黄金标准，但昂贵、缓慢且难以复现。不同的标注者群体（众包工人 vs 领域专家、不同文化、不同语言）产生不同判断。报告标注者间一致性和标注者人口统计对可复现性至关重要。

1. 从头实现一个完整的 Transformer encoder block（multi-head attention、前馈、残差连接、layer norm）。将其应用于一个简单序列分类任务。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

def layer_norm(x, gamma, beta, eps=1e-5):
    mean = x.mean(axis=-1, keepdims=True)
    var = x.var(axis=-1, keepdims=True)
    return gamma * (x - mean) / jnp.sqrt(var + eps) + beta

def multi_head_attention(Q, K, V, W_q, W_k, W_v, W_o, n_heads):
    B, T, D = Q.shape
    head_dim = D // n_heads

    q = Q @ W_q  # (B, T, D)
    k = K @ W_k
    v = V @ W_v

    # Reshape to (B, n_heads, T, head_dim)
    q = q.reshape(B, T, n_heads, head_dim).transpose(0, 2, 1, 3)
    k = k.reshape(B, T, n_heads, head_dim).transpose(0, 2, 1, 3)
    v = v.reshape(B, T, n_heads, head_dim).transpose(0, 2, 1, 3)

    scores = q @ k.transpose(0, 1, 3, 2) / jnp.sqrt(head_dim)
    weights = jax.nn.softmax(scores, axis=-1)
    out = (weights @ v).transpose(0, 2, 1, 3).reshape(B, T, D)
    return out @ W_o, weights

def transformer_block(x, params):
    # Pre-norm multi-head self-attention
    normed = layer_norm(x, params['ln1_g'], params['ln1_b'])
    attn_out, weights = multi_head_attention(
        normed, normed, normed,
        params['W_q'], params['W_k'], params['W_v'], params['W_o'],
        n_heads=4
    )
    x = x + attn_out

    # Pre-norm feed-forward
    normed = layer_norm(x, params['ln2_g'], params['ln2_b'])
    ff = jax.nn.gelu(normed @ params['W1'] + params['b1'])
    ff = ff @ params['W2'] + params['b2']
    x = x + ff
    return x, weights

# Initialise parameters
d_model, d_ff, n_heads = 32, 128, 4
key = jax.random.PRNGKey(42)
keys = jax.random.split(key, 10)

params = {
    'W_q': jax.random.normal(keys[0], (d_model, d_model)) * 0.05,
    'W_k': jax.random.normal(keys[1], (d_model, d_model)) * 0.05,
    'W_v': jax.random.normal(keys[2], (d_model, d_model)) * 0.05,
    'W_o': jax.random.normal(keys[3], (d_model, d_model)) * 0.05,
    'ln1_g': jnp.ones(d_model), 'ln1_b': jnp.zeros(d_model),
    'ln2_g': jnp.ones(d_model), 'ln2_b': jnp.zeros(d_model),
    'W1': jax.random.normal(keys[4], (d_model, d_ff)) * 0.05,
    'b1': jnp.zeros(d_ff),
    'W2': jax.random.normal(keys[5], (d_ff, d_model)) * 0.05,
    'b2': jnp.zeros(d_model),
}

# Test with random input
x = jax.random.normal(keys[6], (2, 8, d_model))  # batch=2, seq_len=8
out, attn_weights = transformer_block(x, params)
print(f"Input shape:  {x.shape}")
print(f"Output shape: {out.shape}")
print(f"Attention weights shape: {attn_weights.shape}")  # (B, n_heads, T, T)

# Visualise attention patterns for each head
fig, axes = plt.subplots(1, 4, figsize=(16, 3.5))
for h in range(4):
    im = axes[h].imshow(attn_weights[0, h], cmap='Blues', vmin=0)
    axes[h].set_title(f"Head {h}")
    axes[h].set_xlabel("Key pos"); axes[h].set_ylabel("Query pos")
plt.suptitle("Multi-Head Attention Patterns")
plt.tight_layout(); plt.show()
```

2. 实现因果（自回归）attention 掩蔽并与双向 attention 比较。展示掩码如何阻止信息从未来 token 流向过去 token。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

def attention(Q, K, V, mask=None):
    d_k = Q.shape[-1]
    scores = Q @ K.T / jnp.sqrt(d_k)
    if mask is not None:
        scores = jnp.where(mask, scores, -1e9)
    weights = jax.nn.softmax(scores, axis=-1)
    return weights @ V, weights

seq_len, d_model = 6, 8
key = jax.random.PRNGKey(0)
k1, k2, k3 = jax.random.split(key, 3)
Q = jax.random.normal(k1, (seq_len, d_model))
K = jax.random.normal(k2, (seq_len, d_model))
V = jax.random.normal(k3, (seq_len, d_model))

# Bidirectional (encoder-style): all positions visible
bidir_mask = jnp.ones((seq_len, seq_len), dtype=bool)
bidir_out, bidir_weights = attention(Q, K, V, bidir_mask)

# Causal (decoder-style): only past and current positions visible
causal_mask = jnp.tril(jnp.ones((seq_len, seq_len), dtype=bool))
causal_out, causal_weights = attention(Q, K, V, causal_mask)

fig, axes = plt.subplots(1, 3, figsize=(14, 4))
tokens = [f"t{i}" for i in range(seq_len)]

axes[0].imshow(bidir_weights, cmap='Blues', vmin=0, vmax=0.5)
axes[0].set_title("Bidirectional Attention\n(BERT-style)")
axes[0].set_xticks(range(seq_len)); axes[0].set_xticklabels(tokens)
axes[0].set_yticks(range(seq_len)); axes[0].set_yticklabels(tokens)

axes[1].imshow(causal_mask.astype(float), cmap='Greys', vmin=0, vmax=1)
axes[1].set_title("Causal Mask\n(1 = allowed, 0 = blocked)")
axes[1].set_xticks(range(seq_len)); axes[1].set_xticklabels(tokens)
axes[1].set_yticks(range(seq_len)); axes[1].set_yticklabels(tokens)

axes[2].imshow(causal_weights, cmap='Blues', vmin=0, vmax=0.5)
axes[2].set_title("Causal Attention\n(GPT-style)")
axes[2].set_xticks(range(seq_len)); axes[2].set_xticklabels(tokens)
axes[2].set_yticks(range(seq_len)); axes[2].set_yticklabels(tokens)

for ax in axes:
    ax.set_xlabel("Key"); ax.set_ylabel("Query")
plt.tight_layout(); plt.show()

# Verify: in causal attention, output at position i depends only on positions <= i
print("Causal attention weight at position 2 (should only attend to 0, 1, 2):")
print(f"  Weights: {causal_weights[2]}")
print(f"  Sum of future weights (should be ~0): {causal_weights[2, 3:].sum():.6f}")
```

3. 实现 LoRA（Low-Rank Adaptation），展示它如何用远少于全量 fine-tuning 的可训练参数来修改权重矩阵。
```python
import jax
import jax.numpy as jnp

d_model = 256
rank = 4  # LoRA rank (much smaller than d_model)

key = jax.random.PRNGKey(42)
k1, k2, k3 = jax.random.split(key, 3)

# Original frozen weight matrix
W_frozen = jax.random.normal(k1, (d_model, d_model)) * 0.02

# LoRA matrices (only these are trainable)
B = jnp.zeros((d_model, rank))       # initialised to zero
A = jax.random.normal(k2, (rank, d_model)) * 0.01  # random init

# Forward pass: W_effective = W_frozen + B @ A
x = jax.random.normal(k3, (8, d_model))

# Without LoRA
y_original = x @ W_frozen.T

# With LoRA
W_effective = W_frozen + B @ A
y_lora = x @ W_effective.T

# Parameter counts
full_params = d_model * d_model
lora_params = d_model * rank + rank * d_model  # B + A

print(f"Model dimension: {d_model}")
print(f"LoRA rank: {rank}")
print(f"Full fine-tuning parameters: {full_params:,}")
print(f"LoRA parameters: {lora_params:,}")
print(f"Parameter reduction: {full_params / lora_params:.1f}x")
print(f"\nSince B is initialised to zeros, initial LoRA output matches original:")
print(f"  Max difference: {jnp.abs(y_original - y_lora).max():.2e}")

# Simulate training: only update A and B
def lora_forward(A, B, W_frozen, x):
    return x @ (W_frozen + B @ A).T

def dummy_loss(A, B, W_frozen, x, target):
    pred = lora_forward(A, B, W_frozen, x)
    return jnp.mean((pred - target) ** 2)

# Target: some transformation of x
target = x @ jax.random.normal(jax.random.PRNGKey(99), (d_model, d_model)).T * 0.02

grad_fn = jax.jit(jax.grad(dummy_loss, argnums=(0, 1)))
lr = 0.01

for step in range(200):
    gA, gB = grad_fn(A, B, W_frozen, x, target)
    A = A - lr * gA
    B = B - lr * gB

loss_before = dummy_loss(jnp.zeros_like(A), jnp.zeros_like(B), W_frozen, x, target)
loss_after = dummy_loss(A, B, W_frozen, x, target)
print(f"\nLoss before LoRA: {loss_before:.6f}")
print(f"Loss after LoRA:  {loss_after:.6f}")
print(f"Effective weight change rank: {jnp.linalg.matrix_rank(B @ A)}")
```
