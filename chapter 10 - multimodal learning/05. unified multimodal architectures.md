# Унифицированные мультимодальные архитектуры

*Унифицированные мультимодальные архитектуры заменяют отдельные специализированные модели единой системой, которая читает, рассуждает и генерирует контент, используя текст, изображения, аудио и видео. В этом файле рассматриваются модели «любое-в-любое» (CoDi, NExT-GPT), нативно мультимодальные LLM (Gemini, GPT-4o), стратегии мультимодальной токенизации и архитектурные компромиссы унификации.*

## Аргументы в пользу унификации

- Представьте переводчика, который говорит на пяти языках и может переключаться между ними прямо во время предложения, не делая пауз. Ранние мультимодальные системы были больше похожи на пять отдельных переводчиков, сидящих в разных комнатах: каждый из них владел одним языком и передавал записки через прорезь в стене. **Унифицированная мультимодальная архитектура** — это тот самый полиглот: одна модель с общими весами, которая читает, пишет и рассуждает, используя текст, изображения, аудио, видео и даже действия, и всё это в рамках одного прямого прохода.

- Мотивация здесь как практическая, так и теоретическая. С практической точки зрения, поддержка отдельных специализированных моделей для каждой пары модальностей (текст-в-изображение, изображение-в-текст, аудио-в-текст и т. д.) ведет к комбинаторному взрыву: $k$ модальностей требуют до $k(k-1)$ направленных пайплайнов. Унифицированная модель сводит всё это в единую систему. С теоретической точки зрения, человеческое познание не обрабатывает зрение и язык в изолированных модулях; кросс-модальная связка происходит рано и глубоко, и унификация пытается отразить это.

- Общие веса способствуют **переносу знаний между модальностями**. Трансформер, который выучил временные закономерности в тексте (подлежащее перед сказуемым, причина перед следствием), может переиспользовать те же механизмы внимания для временных закономерностей в видео (объект появляется до того, как начинает двигаться) или аудио (начало звука до его затухания). Это мультимодальный аналог обучения с переносом, который вы видели в главе 7 при дообучении языковых моделей и в главе 8 при предобучении на ImageNet.

- Формально, пусть $\mathcal{M} = \{m_1, m_2, \ldots, m_k\}$ — множество модальностей. Унифицированная модель определяет единственную параметризованную функцию $f_\theta$, которая отображает любое подмножество входных модальностей в любое подмножество выходных модальностей:

$$f_\theta : \mathcal{P}(\mathcal{M}) \rightarrow \mathcal{P}(\mathcal{M})$$

- где $\mathcal{P}(\mathcal{M})$ — булеан (множество всех подмножеств) модальностей. Ключевое ограничение заключается в том, что $\theta$ по большей части общие; различаются только тонкие, специфичные для каждой модальности адаптерные слои.

![Высокоуровневая схема, показывающая, как различные модальности (текст, изображение, аудио, видео) поступают в единый общий трансформерный бэкенд и производят выходные данные в любой модальности](../images/unified_multimodal_overview.svg)

- Перспективы унификации сопровождаются фундаментальным противоречием: модальности структурно различны. Текст — это одномерная последовательность дискретных токенов. Изображения — это двумерные сетки непрерывных значений пикселей. Аудио — это одномерная непрерывная волна с совершенно иным временным масштабом, чем у текста. Видео добавляет ось времени к изображениям. Согласование этих разрозненных структур в единую последовательность, которую может «переварить» трансформер, является главной инженерной задачей в этой области.

## Модели «любое-в-любое»

- Подумайте об универсальном пульте дистанционного управления, который может управлять телевизором, кондиционером и музыкальной системой через один и тот же интерфейс. **Модели «любое-в-любое»** — это ИИ-эквивалент: они принимают любую комбинацию модальностей на вход и производят любую комбинацию на выходе.

- **CoDi** (Composable Diffusion) достигает генерации «любое-в-любое» путем обучения специфичных для модальностей диффузионных моделей с последующим выравниванием их латентных пространств через общий механизм кондиционирования. У каждой модальности свой процесс диффузии (вспомните диффузионные модели из файла 04 этой главы), но сети предсказания шума обусловлены общим слоем кросс-внимания, который одновременно видит эмбеддинги всех входных модальностей. Это позволяет CoDi генерировать, скажем, изображение и соответствующее аудио по текстовому промпту за один проход.

- **NExT-GPT** использует другой архитектурный подход. Она соединяет бэкенд LLM («мозг») со специфичными для модальностей энкодерами на стороне входа и декодерами на стороне выхода через легковесные **проекционные слои**. Входные энкодеры (например, энкодер изображений из CLIP, аудиоэнкодер из CLAP) переводят каждую модальность в пространство эмбеддингов LLM. LLM рассуждает над объединенной последовательностью токенов и выдает специальные «сигнальные токены модальностей», которые направляют информацию к соответствующему декодеру (например, Stable Diffusion для изображений, AudioLDM для аудио). Обучаются только проекционные слои; LLM и специализированные энкодеры/декодеры остаются замороженными.

- **Gemini** (Google DeepMind) является нативно мультимодальной с этапа предобучения. В отличие от подхода NExT-GPT, где компоненты подключаются по принципу plug-and-play, трансформер Gemini обучается с нуля на перемежающихся последовательностях токенов текста, изображения, аудио и видео. Это означает, что паттерны кросс-модального внимания развиваются органично во время предобучения, а не добавляются поверх него. Модель использует токенизатор SentencePiece для текста и обучается визуальному токенизатору, похожему на VQ-подходы, обсуждавшиеся в файле 03 этой главы.

- **GPT-4o** («o» от «omni») представляет собой еще один паттерн: end-to-end модель, где все модальности разделяют один и тот же трансформер и одну и ту же задачу предсказания следующего токена. Аудиовход обрабатывается как спектральные токены, изображения — как патч-токены, а текст — как подслововые токены; все они подаются в единую последовательность. Модель генерирует выходные токены, которые декодируются специфичными для модальностей «головами». Ключевая инновация — низкая задержка, достигнутая за счет устранения каскада отдельных моделей ASR, LLM и TTS, на которые полагались более ранние системы, такие как GPT-4V.

![Сравнение архитектурных паттернов для CoDi (выровненная диффузия), NExT-GPT (LLM-хаб с замороженными специалистами) и стиля Gemini (нативно перемежающееся предобучение)](../images/any_to_any_architectures.svg)

- Эти модели располагаются в спектре глубины интеграции:

    - **Поверхностная интеграция** (NExT-GPT): замороженные специалисты, соединенные обученными адаптерами. Быстро строится, ограниченное кросс-модальное рассуждение.
    - **Средняя интеграция** (CoDi): общее кондиционирование между генераторами, специфичными для модальностей. Лучшее выравнивание, но всё еще модульность.
    - **Глубокая интеграция** (Gemini, GPT-4o): единая модель, обученная end-to-end на всех модальностях. Богатейшее кросс-модальное рассуждение, наиболее дорогостоящее обучение.

## Модально-специфичные энкодеры и декодеры с общим бэкендом

- Представьте себе фабрику с одной сборочной линией (общий бэкенд), но разными погрузочными платформами для сырья (энкодеры) и разными отделами отгрузки готовой продукции (декодеры). Каждая платформа специализируется на своем грузе, но как только он попадает на фабрику, всё движется по одному и тому же конвейеру.

- Доминирующий архитектурный паттерн для унифицированных моделей использует эту трехчастную структуру:

    - **Модальные энкодеры** $E_m$, которые преобразуют исходные входные данные модальности $m$ в последовательность векторов эмбеддингов $\mathbf{h}_1^m, \mathbf{h}_2^m, \ldots, \mathbf{h}_{n_m}^m$, каждый из которых имеет размерность $d$.
    - **Общий трансформерный бэкенд** $T_\theta$, который обрабатывает конкатенированные или чередующиеся эмбеддинги всех входных модальностей с использованием механизма самовнимания.
    - **Модальные декодеры** $D_m$, которые преобразуют выходные эмбеддинги бэкенда обратно в исходный формат модальности $m$ (текстовые токены, пиксели изображения, аудиосигналы).

- Для текста энкодер обычно представляет собой таблицу поиска эмбеддингов $E_\text{text}(w) = \mathbf{W}_e[w]$, где $w$ — индекс токена, что идентично тому, что вы видели в главе 7 при изучении трансформеров. Для изображений энкодером часто выступает **Vision Transformer** (ViT), который разбивает изображение на патчи и линейно проецирует каждый патч, как рассматривалось в главе 8. Для аудио энкодер вычисляет мел-спектрограмму и обрабатывает её либо с помощью сверточного фронтенда, либо с помощью Audio Spectrogram Transformer (AST), как обсуждалось в главе 9.

- Общий бэкенд — это стандартный трансформер с механизмом самовнимания по всем токенам модальностей. Учитывая конкатенированную входную последовательность $\mathbf{H} = [\mathbf{h}_1^{m_1}, \ldots, \mathbf{h}_{n_1}^{m_1}, \mathbf{h}_1^{m_2}, \ldots, \mathbf{h}_{n_2}^{m_2}]$, самовнимание позволяет каждому токену «обращать внимание» на любой другой токен независимо от модальности:

$$\text{Attention}(\mathbf{Q}, \mathbf{K}, \mathbf{V}) = \text{softmax}\left(\frac{\mathbf{Q}\mathbf{K}^\top}{\sqrt{d_k}}\right)\mathbf{V}$$

- Это та же формула внимания из главы 7, но теперь $\mathbf{Q}$, $\mathbf{K}$ и $\mathbf{V}$ содержат токены из нескольких модальностей. Токен патча изображения может «обращать внимание» на текстовый токен, что позволяет выполнять кросс-модальные рассуждения без какого-либо отдельного модуля кросс-внимания.

- **Модальные эмбеддинги** добавляются к каждому токену, чтобы бэкенд знал, из какой модальности происходит токен. Это аналогично позиционным эмбеддингам, но кодирует принадлежность к модальности, а не позицию в последовательности. Обучаемый вектор $\mathbf{e}_m \in \mathbb{R}^d$ добавляется к каждому токену из модальности $m$:

$$\tilde{\mathbf{h}}_i^m = \mathbf{h}_i^m + \mathbf{e}_m + \mathbf{p}_i$$

- где $\mathbf{p}_i$ — позиционный эмбеддинг для позиции $i$.

## Мультимодальная токенизация

- Представьте, что вы пишете письмо, которое включает в себя как английский текст, так и нарисованные от руки эскизы. Вы можете написать предложение, нарисовать диаграмму, написать еще одно предложение, ссылающееся на диаграмму, а затем вставить музыкальную партитуру. Письмо представляет собой единый линейный поток, в котором чередуются различные «модальности». Мультимодальная токенизация делает именно это: она преобразует текст, изображения, аудио и видео в единую плоскую последовательность токенов, которую трансформер обрабатывает слева направо.

- Для текста токенизация хорошо изучена: **byte-pair encoding** (BPE) или SentencePiece создают словарь подсловесных токенов, как рассматривалось в главе 7. Задача состоит в том, чтобы распространить эту идею на непрерывные модальности.

- Для изображений существует два основных подхода. **Дискретный** подход использует VQ-VAE или VQ-GAN (подробно описано в файле 03 этой главы) для отображения каждого изображения в последовательность индексов кодовой книги. Если кодовая книга имеет $|\mathcal{C}|$ записей, а изображение закодировано как $n$ кодов, изображение превращается в $n$ дискретных токенов, взятых из словаря размера $|\mathcal{C}|$, что напрямую совместимо со словарем текста. **Непрерывный** подход использует энкодер ViT или CNN для создания $n$ непрерывных векторов эмбеддингов, которые линейно проецируются в размерность эмбеддингов трансформера. Gemini и GPT-4o используют варианты непрерывного подхода; авторегрессионные генераторы изображений, такие как Parti и LlamaGen, предпочитают дискретный путь.

- Для аудио сигнал обычно преобразуется в мел-спектрограмму, а затем либо дискретизируется с помощью нейронного аудиокодека (например, EnCodec, SoundStream, которые создают иерархические дискретные токены), либо проецируется непрерывно через обученный энкодер. AudioLM, например, представляет аудио как последовательность дискретных токенов из нескольких уровней кодовой книги, а затем моделирует их авторегрессионно.

- Для видео токенизация опирается на токенизацию изображений, но также должна сжимать временное измерение. Распространенная стратегия использует **3D VQ-VAE** (как в VideoGPT или Cosmos Tokeniser из файла 03), который квантует пространственно-временные патчи в дискретные токены. Коэффициент временного сжатия имеет решающее значение: необработанное видео с частотой 24 кадра в секунду создает слишком много токенов в секунду без агрессивной временной дискретизации.

- Как только все модальности токенизированы, они **чередуются** в единую последовательность со специальными токенами-разделителями, отмечающими границы модальностей. Типичный формат выглядит так:

```
[TEXT] The cat sits on a mat [/TEXT] [IMAGE] <img_tok_1> <img_tok_2> ... <img_tok_n> [/IMAGE] [AUDIO] <aud_tok_1> ... <aud_tok_m> [/AUDIO]
```

- Затем трансформер обрабатывает всю эту смешанную последовательность, используя свой стандартный причинный (или двунаправленный) механизм внимания. Токены-разделители модальностей выполняют двойную функцию: они информируют модель о границах модальностей и действуют как «точки пулинга», представления которых суммируют каждый сегмент модальности.

![Чередующаяся последовательность токенов, показывающая текстовые токены, дискретные токены изображений и токены аудиокодека, проходящие через единый трансформер с маркерами границ модальностей](../images/multimodal_tokenisation_sequence.svg)

- Важным проектным решением является **бюджет токенов** (token budget). Одно изображение, токенизированное в 256 токенов, и текстовая подпись из 50 токенов означают, что изображение потребляет в 5 раз больше контекстного окна. Модели должны находить баланс между разрешением (больше токенов — больше деталей) и длиной контекста (больше токенов — выше затраты памяти и вычислительных ресурсов). Такие методы, как **объединение токенов** (token merging) (постепенное объединение похожих токенов) и **адаптивная токенизация** (использование меньшего количества токенов для простых областей и большего — для сложных), помогают управлять этим компромиссом.

## Рецепты обучения: поэтапное предварительное обучение и совместная донастройка

- Вы не стали бы учить ребенка математическому анализу до того, как он освоит арифметику. Аналогично, нельзя обучить унифицированную мультимодальную модель на всех модальностях одновременно со случайной инициализацией и ожидать хорошей сходимости. Доминирующим подходом является **поэтапное обучение** (staged training), при котором модель постепенно осваивает все более сложные кросс-модальные возможности на тщательно упорядоченных этапах.

- **Этап 1: Унимодальное предварительное обучение.** Каждый энкодер модальности обучается независимо на больших унимодальных датасетах. Текстовый бэкенд предварительно обучается с использованием стандартной целевой функции языкового моделирования (предсказание следующего токена) на триллионах текстовых токенов, точно так же, как в главе 7. Визуальный энкодер предварительно обучается на задачах классификации изображений или с помощью методов самообучения (MAE, DINO), как в главе 8. Аудиоэнкодер предварительно обучается на данных распознавания речи или классификации аудио, как в главе 9. Этот этап позволяет получить мощные экстракторы унимодальных признаков.

- **Этап 2: Кросс-модальное выравнивание.** Предварительно обученные энкодеры подключаются к общему бэкенду, и модель обучается на парных мультимодальных данных (пары «изображение-подпись», «аудио-транскрипт») с использованием контрастивной или генеративной целевой функции. На этом этапе веса энкодеров могут быть заморожены (для сохранения унимодальных знаний), в то время как обновляются только проекционные слои и бэкенд. Именно на этом этапе выравнивание в стиле CLIP (из файла 01 в этой главе) интегрируется в унифицированную модель.

- **Этап 3: Совместное мультимодальное предварительное обучение.** Все параметры (или большинство из них) размораживаются, и модель обучается на смеси унимодальных и мультимодальных данных с единой целевой функцией предсказания следующего токена для всех токенов модальностей. Функция потерь имеет вид:

$$\mathcal{L} = -\sum_{t=1}^{T} \log p_\theta(x_t \mid x_{<t})$$

- где $x_t$ может быть текстовым токеном, токеном изображения или аудиотокеном. Модель должна научиться предсказывать следующий токен независимо от модальности, что заставляет её развивать подлинное кросс-модальное понимание.

- **Этап 4: Инструктивная донастройка и выравнивание.** Предварительно обученная модель донастраивается на отобранных датасетах для следования инструкциям, которые включают мультимодальные инструкции (например, «Опиши это изображение подробно», «Какой звук издает это видео?», «Сгенерируй изображение X»). На этом этапе часто используется **обучение с подкреплением на основе отзывов людей** (RLHF) или прямая оптимизация предпочтений (DPO) для приведения выходов модели в соответствие с предпочтениями человека.

- **Модально-специфичный прогрев** (warm-up) — это метод, используемый внутри этапов для предотвращения коллапса модальностей. Если одна модальность (обычно текст, для которого больше всего данных обучения) доминирует в градиентном сигнале, модель может «забыть» более слабые модальности. Стратегии прогрева включают:

    - **Балансировка градиентов**: масштабирование градиентов каждой модальности так, чтобы они вносили равный вклад в обновление параметров.
    - **Планирование соотношения данных**: постепенное увеличение доли мультимодальных данных относительно унимодальных.
    - **Взвешивание потерь**: назначение модально-специфичных весов $\lambda_m$ так, чтобы общая функция потерь имела вид $\mathcal{L} = \sum_m \lambda_m \mathcal{L}_m$, где $\lambda_m$ настраиваются для балансировки скоростей обучения между модальностями.

![Диаграмма четырехэтапного пайплайна обучения, показывающая унимодальное предварительное обучение, кросс-модальное выравнивание, совместное мультимодальное предварительное обучение и инструктивную донастройку, со стрелками, указывающими, какие параметры заморожены или обучаемы на каждом этапе](../images/staged_multimodal_training.svg)

- **Почему нельзя пропустить этапы?** Обучение всего совместно с нуля выглядит заманчиво, но на практике не работает по нескольким причинам. Во-первых, модель должна одновременно изучать низкоуровневые признаки (детектирование границ, распознавание фонем) и высокоуровневые кросс-модальные рассуждения, которые имеют совершенно разную динамику обучения. Во-вторых, распределения данных по модальностям сильно несбалансированы (триллионы текстовых токенов против миллиардов токенов изображений и сотен миллионов аудиоклипов). В-третьих, ландшафт оптимизации является сильно невыпуклым, и поэтапное обучение обеспечивает учебную программу (curriculum), которая направляет модель к лучшему бассейну притяжения, подобно идее обучения по программе (curriculum learning) из главы 6.

## Мультимодальные рассуждения по цепочке мыслей

- Когда вы решаете задачу по геометрии, вы можете набросать чертеж, отметить углы, записать уравнение, а затем решить его шаг за шагом. Вы не переходите сразу от условия задачи к ответу. **Мультимодальные рассуждения по цепочке мыслей** (Chain-of-Thought, CoT) позволяют моделям делать то же самое: генерировать промежуточные шаги рассуждений, которые могут включать текст, визуальные аннотации или даже сгенерированные диаграммы, прежде чем прийти к окончательному ответу.

- В текстовом CoT (как рассматривалось в главе 7 при обсуждении стратегий промптинга) модель генерирует последовательность шагов рассуждения на естественном языке. Мультимодальный CoT расширяет это, позволяя промежуточным шагам ссылаться на визуальный контент или генерировать его. Например, при наличии изображения диаграммы и вопроса «В каком году были самые высокие продажи?», мультимодальная CoT-модель может сначала описать диаграмму («На диаграмме показаны продажи с 2018 по 2023 год...»), затем определить соответствующие визуальные признаки («Самый высокий столбец находится в 2021 году...») и, наконец, выдать ответ («2021»).

- Формально, пусть $\mathbf{x}$ — мультимодальный вход, а $y$ — целевой ответ. Стандартные модели предсказывают $p(y \mid \mathbf{x})$ напрямую. Цепочка мыслей вводит промежуточные рассуждения $\mathbf{r} = (r_1, r_2, \ldots, r_L)$ и факторизует предсказание как:

$$p(y \mid \mathbf{x}) = \sum_{\mathbf{r}} p(y \mid \mathbf{r}, \mathbf{x}) \cdot p(\mathbf{r} \mid \mathbf{x})$$

- На практике эта сумма аппроксимируется с помощью жадного поиска или лучевого поиска (beam-search) по цепочкам рассуждений. Шаги рассуждения $r_i$ могут представлять собой текстовые токены, ссылки на области изображения или даже сгенерированные визуальные токены (например, аннотация в виде ограничивающей рамки, наложенная на входное изображение).

- **Обучение мультимодальных цепочек рассуждений (CoT)** обычно включает в себя подготовку датасетов, в которых люди-аннотаторы предоставляют пошаговые мультимодальные траектории рассуждений, с последующей донастройкой (fine-tuning) модели на этих траекториях. Некоторые подходы используют дистилляцию способностей CoT от более крупных моделей-учителей: учитель генерирует траектории рассуждений для большого датасета, а меньшая модель-ученик обучается как на входных данных, так и на траекториях учителя.

- Мультимодальные CoT особенно эффективны для задач, требующих **пространственного мышления** (например, «Находится ли красный шар слева от синего куба?»), **математических рассуждений по диаграммам** (например, задачи по геометрии) и **многошаговых ответов на визуальные вопросы**, где ответ зависит от объединения информации из нескольких областей изображения.

## Мультимодальные агенты

- Представьте робота-повара на кухне. Он смотрит на ингредиенты на столе (зрение), читает рецепт на планшете (текст), слушает звуковой сигнал таймера (аудио), а затем физически берет нож и режет лук (действие). **Мультимодальный агент** — это цифровая версия такого робота: модель, которая воспринимает мир через несколько модальностей, рассуждает о том, что нужно сделать, и совершает действия, опираясь на свое восприятие.

- Цикл работы агента следует классической схеме **наблюдение-рассуждение-действие** (observe-reason-act):

    1. **Наблюдение**: Агент получает мультимодальные входные данные из своей среды (скриншот, устную инструкцию пользователя, видеопоток).
    2. **Рассуждение**: Единая модель обрабатывает мультимодальные входные данные, возможно, используя цепочку рассуждений для планирования последовательности шагов.
    3. **Действие**: Модель выдает действие (текстовый ответ, вызов инструмента, клик мышью по координатам $(x, y)$, команду для приводов робота).

- **Использование инструментов** — ключевая способность мультимодальных агентов. Модель обучается распознавать, когда она не может ответить на вопрос напрямую и должна вместо этого вызвать внешний инструмент: калькулятор, интерпретатор кода, веб-браузер или поисковую систему. Модель генерирует структурированный вызов инструмента (например, `search("current weather in London")`) как часть своей последовательности выходных токенов, система выполняет вызов, а результат возвращается в качестве дополнительных входных токенов для обработки моделью.

- **Визуальное заземление (visual grounding)** связывает язык с конкретными областями на изображении или видео. Когда агент говорит «нажми на синюю кнопку в правом верхнем углу», он должен соотнести фразу «синяя кнопка в правом верхнем углу» с пиксельными координатами. Архитектурно это достигается путем обучения модели выводить координаты ограничивающей рамки в виде специальных токенов или путем создания моделью тепловой карты изображения, указывающей на упоминаемую область. Это расширяет работу по заземлению и ссылочным выражениям, обсуждавшуюся в файле 02 этой главы (Визуально-языковые модели), на область действий.

- **Веб-агенты**, такие как WebVoyager и SeeAct, демонстрируют работу мультимодальных агентов при навигации по веб-сайтам. Агент получает скриншот веб-страницы, идентифицирует интерактивные элементы (кнопки, текстовые поля, ссылки) и выполняет действия (клик, ввод текста, прокрутка) для достижения цели, заданной пользователем. Ключевой проблемой является огромное пространство действий: на типичной веб-странице могут быть сотни возможных целей для клика.

![Цикл «наблюдение-рассуждение-действие» мультимодального агента, демонстрирующий визуальный ввод с экрана, процесс рассуждения внутри единой модели и выходные действия, такие как клик, ввод текста или вызов инструментов](../images/multimodal_agent_loop.svg)

- **Воплощенные агенты (embodied agents)** распространяют этот подход на физические среды. Робот с камерой и микрофоном получает визуальные и аудиоданные, обрабатывает их через единую модель и выдает команды для приводов. Проекты, такие как PaLM-E (Google), встраивают данные с датчиков робота непосредственно в последовательность токенов языковой модели, позволяя роботу следовать инструкциям типа «подними зеленый кубик рядом с миской», соотнося инструкцию со своим визуальным наблюдением и генерируя последовательность двигательных действий.

- Рецепт обучения агентов добавляет этап **обучения с подкреплением** (RL) поверх стандартного поэтапного предварительного обучения. Агент взаимодействует со средой (симулированный рабочий стол, веб-браузер, робототехнический симулятор), получает вознаграждение за выполнение задачи и обновляет свою стратегию (policy) с помощью алгоритмов, таких как PPO или REINFORCE. Сигнал вознаграждения обычно является разреженным (1 за успех задачи, 0 в противном случае), что делает эту оптимизацию сложной и сильно зависящей от сильных априорных знаний, полученных при мультимодальном предварительном обучении.

## Бенчмарки и оценка

- Оценка модели, которая может видеть, слышать, читать и действовать, требует разнообразного набора бенчмарков. Ни одна метрика не охватывает мультимодальную компетентность целиком, поэтому в этой области полагаются на совокупность специализированных оценок.

- **MMLU** (Massive Multitask Language Understanding) проверяет знания по 57 академическим дисциплинам. Хотя изначально этот бенчмарк был только текстовым, он служит базовым решением: единая мультимодальная модель не должна терять производительность в текстовых задачах при приобретении визуальных способностей. Падение показателей MMLU после мультимодального обучения сигнализирует о катастрофическом забывании.

- **MMBench** оценивает визуально-языковое понимание по 20 детализированным измерениям способностей, включая распознавание атрибутов, понимание пространственных отношений и OCR. Каждый вопрос представляет собой изображение и вопрос с несколькими вариантами ответа. Бенчмарк систематически проверяет, действительно ли модель понимает изображение или полагается на текстовые «шорткаты».

- **SEED-Bench** предоставляет 19 000 вопросов с несколькими вариантами ответов, охватывающих 12 измерений оценки как для понимания изображений, так и видео. Он специально проверяет временное понимание (что произошло до/после заданного кадра) и композиционное мышление (объединение нескольких визуальных атрибутов).

- **MM-Vet** оценивает интегрированные мультимодальные способности, требуя от моделей одновременного использования нескольких навыков: распознавания, OCR, пространственного восприятия, генерации языка и поиска знаний — всё в рамках одного вопроса.

- **MathVista** тестирует математическое мышление на основе визуальных входных данных: геометрических диаграмм, статистических графиков, графиков функций и научных рисунков. Этот бенчмарк специально нацелен на оценку мультимодальных способностей к рассуждению по цепочке (chain-of-thought).

- **Аудиовизуальные бенчмарки**, такие как AVQA (Audio-Visual Question Answering), проверяют, способны ли модели рассуждать о взаимосвязи между тем, что они видят, и тем, что они слышат. Например: «Говорящий человек находится слева или справа?»

- **Агентные бенчмарки**, такие как WebArena, OSWorld и SWE-bench, оценивают выполнение задач в интерактивных средах. Метрикой обычно является коэффициент успеха: какую долю задач агент выполняет правильно? Эти бенчмарки особенно сложны, так как требуют долгосрочного планирования и восстановления после ошибок.

- **Комплексные (holistic)** системы оценки, такие как LMSYS Chatbot Arena, используют экспертные оценки людей в формате «один на один». Двум моделям показывают одни и те же мультимодальные входные данные, и судья-человек выбирает, какой ответ лучше. Рейтинги Эло вычисляются на основе тысяч таких сравнений, предоставляя единый скаляр, который хорошо коррелирует с общим качеством модели.

- Постоянной проблемой в мультимодальной оценке является **загрязнение данных (data contamination)**: поскольку эти модели обучаются на данных масштаба интернета, изображения и вопросы из бенчмарков могут присутствовать в обучающей выборке. Тщательная дедупликация и создание отложенных тестовых выборок являются необходимыми, но несовершенными мерами защиты.

## Мировые модели (World Models)

- Представьте, что вы закрыли глаза и визуализируете, что произойдет, если столкнуть стакан с края стола. Вы «видите», как он падает, «слышите» звон разбитого стекла и «чувствуете», что это плохая идея. Ваш мозг использует **мировую модель**: внутреннюю симуляцию физической и причинно-следственной структуры окружающей среды, которая может предсказывать будущие состояния в различных модальностях.

- В контексте ИИ мировая модель — это обученная функция, которая предсказывает следующее состояние мира, исходя из текущего состояния и действия:

$$\hat{s}_{t+1} = g_\phi(s_t, a_t)$$

- где $s_t$ — представление текущего состояния (которое может включать визуальную, слуховую и проприоцептивную информацию), $a_t$ — действие, а $\hat{s}_{t+1}$ — предсказанное следующее состояние. Состояние $s_t$ существует в обученном латентном пространстве, а не в пространстве «сырых» пикселей, что делает задачу предсказания разрешимой.

- **Модели предсказания видео**, такие как Sora (OpenAI) и Genie (Google DeepMind), представляют собой важный шаг в сторону мировых моделей. Они учатся генерировать временно согласованные кадры видео, обусловленные текстовыми промптами и/или последовательностями действий. Хотя их часто обсуждают как генераторы видео, лежащая в их основе способность ближе к симуляции мира: модель усвоила достаточно физических законов (гравитацию, столкновения, окклюзию, динамику жидкостей), чтобы визуализировать правдоподобное будущее.

- Связь с мультимодальными архитектурами глубока. Мировая модель, предсказывающая только пиксели, ограничена; по-настоящему полезная мировая модель делает предсказания по всем модальностям. Если вы толкаете стакан, мировая модель должна предсказать визуальную траекторию (стакан падает), звуковое событие (стакан разбивается) и семантическое следствие (теперь у вас на полу разбитое стекло). Унифицированные мультимодальные архитектуры являются естественными кандидатами на роль мировых моделей, поскольку они уже представляют все модальности в общем пространстве.

- Формально мультимодальная мировая модель оптимизирует:

$$\mathcal{L}_\text{world} = \mathbb{E}\left[\sum_{m \in \mathcal{M}} \lambda_m \| s_{t+1}^m - g_\phi^m(s_t, a_t) \|^2 \right]$$

- где $s_{t+1}^m$ — эталонное представление следующего состояния в модальности $m$, а $g_\phi^m$ — специфичная для модальности голова предсказания мировой модели. Общая латентная динамика $g_\phi$ работает в объединенном мультимодальном пространстве, в то время как специфичные для модальностей головы декодируют предсказания в исходный формат каждой модальности.

![Диаграмма мировой модели, показывающая обновление латентного состояния под воздействием действия, с головами-декодерами, предсказывающими будущие визуальные кадры, аудиосигналы и семантические описания](../images/multimodal_world_model.svg)

- **JEPA** (Joint Embedding Predictive Architecture), предложенная Яном Лекуном, предлагает структуру для мировых моделей, которая позволяет избежать ловушек предсказания на уровне пикселей. Вместо предсказания «сырых» пикселей (что тратит ресурсы на нерелевантные детали, такие как точные текстуры), JEPA делает предсказания в пространстве эмбеддингов. Модель обучается с помощью энкодера, который отображает наблюдения в эмбеддинги, и предиктора, который прогнозирует будущие эмбеддинги:

$$\hat{\mathbf{z}}_{t+1} = h_\psi(\mathbf{z}_t, a_t), \quad \mathbf{z}_t = \text{Enc}(s_t)$$

- Функция потерь сравнивает эмбеддинги, а не исходные наблюдения, что более устойчиво к перцептивному алиасингу (различные конфигурации пикселей могут представлять одно и то же семантическое состояние). Этот подход особенно перспективен для мультимодальных мировых моделей, поскольку он естественным образом работает в общем пространстве эмбеддингов, которое уже предоставляют унифицированные архитектуры.

- Мировые модели имеют практическое применение, выходящее за рамки академического интереса. В **обучении с подкреплением на основе моделей (model-based reinforcement learning)** агент использует свою мировую модель, чтобы «представить» последствия действий перед их совершением, что значительно сокращает количество необходимых взаимодействий с реальным миром (вспомните обсуждение обучения с подкреплением на основе моделей из главы 11). В **автономном вождении** мировая модель предсказывает, как сцена будет развиваться в течение следующих нескольких секунд при различных решениях по управлению. В **робототехнике** мировая модель позволяет роботу мысленно отрепетировать последовательность манипуляций перед её выполнением.

- Передний край исследований мировых моделей движется в сторону **интерактивных мировых моделей**, которые работают в реальном времени и реагируют на произвольные действия пользователя, по сути становясь симуляторами общего назначения, обученными исключительно на данных. Genie 2 (Google DeepMind) демонстрирует это для 3D-сред: получив одно изображение, она генерирует интерактивный, управляемый 3D-мир, который пользователь может исследовать. Конвергенция мировых моделей и унифицированных мультимодальных архитектур предполагает будущее, в котором одна модель сможет воспринимать, предсказывать, симулировать и действовать во всех модальностях.

**Задача 1: Создание минимального мультимодального интерливера токенов**

- Напишите функцию, которая принимает текстовую строку и фиктивное «изображение» (небольшой двумерный массив) и объединяет их токенизированные представления в единую плоскую последовательность с эмбеддингами модальностей.

```python
import jax
import jax.numpy as jnp

# Simulate multimodal tokenisation: text tokens + "image patch" tokens
def interleave_modalities(text_tokens, image_patches, embed_dim=32, key=jax.random.PRNGKey(0)):
    """Interleave text and image tokens with learned modality embeddings."""
    k1, k2, k3 = jax.random.split(key, 3)
    n_text = text_tokens.shape[0]
    n_img = image_patches.shape[0]
    # Random projection matrices (stand-ins for real encoders)
    W_text = jax.random.normal(k1, (text_tokens.shape[-1], embed_dim)) * 0.02
    W_img = jax.random.normal(k2, (image_patches.shape[-1], embed_dim)) * 0.02
    # Modality embeddings: one for text, one for image
    mod_emb = jax.random.normal(k3, (2, embed_dim)) * 0.02
    text_embs = text_tokens @ W_text + mod_emb[0]  # (n_text, embed_dim)
    img_embs = image_patches @ W_img + mod_emb[1]   # (n_img, embed_dim)
    # Interleave: [IMG] tokens first, then [TEXT] tokens (like LLaVA)
    combined = jnp.concatenate([img_embs, text_embs], axis=0)
    print(f"Combined sequence: {n_img} image + {n_text} text = {combined.shape[0]} tokens")
    return combined

# Try it: 5 text tokens (dim 16) and 4 image patches (dim 64)
text = jax.random.normal(jax.random.PRNGKey(1), (5, 16))
image = jax.random.normal(jax.random.PRNGKey(2), (4, 64))
seq = interleave_modalities(text, image)
```

**Задача 2: Визуализация паттернов кросс-модального внимания**

- Создайте синтетическую мультимодальную последовательность и вычислите оценки самовнимания, чтобы увидеть, как токены изображения учитывают текстовые токены и наоборот.

```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

def cross_modal_attention(n_text=6, n_img=4, d=32, key=jax.random.PRNGKey(42)):
    """Compute and visualise attention between text and image tokens."""
    k1, k2, k3 = jax.random.split(key, 3)
    # Simulate token embeddings for two modalities
    text_embs = jax.random.normal(k1, (n_text, d))
    img_embs = jax.random.normal(k2, (n_img, d))
    seq = jnp.concatenate([img_embs, text_embs], axis=0)  # (n_img+n_text, d)
    # Learned Q, K projections
    Wq = jax.random.normal(k3, (d, d)) * 0.1
    Wk = jax.random.normal(jax.random.PRNGKey(99), (d, d)) * 0.1
    Q, K = seq @ Wq, seq @ Wk
    scores = Q @ K.T / jnp.sqrt(d)
    attn = jax.nn.softmax(scores, axis=-1)
    # Plot
    labels = [f"img_{i}" for i in range(n_img)] + [f"txt_{i}" for i in range(n_text)]
    fig, ax = plt.subplots(figsize=(7, 6))
    ax.imshow(attn, cmap="viridis")
    ax.set_xticks(range(len(labels))); ax.set_xticklabels(labels, rotation=45, fontsize=8)
    ax.set_yticks(range(len(labels))); ax.set_yticklabels(labels, fontsize=8)
    ax.set_xlabel("Key (attended to)"); ax.set_ylabel("Query (attending from)")
    ax.set_title("Cross-modal self-attention map")
    plt.colorbar(ax.images[0], ax=ax, shrink=0.8)
    plt.tight_layout(); plt.show()

cross_modal_attention()
```

**Задача 3: Имитация поэтапного обучения с взвешиванием потерь по модальностям**

- Продемонстрируйте, как веса потерь, специфичные для модальностей, влияют на игрушечный цикл мультимодального обучения. Пронаблюдайте, как балансировка потерь предотвращает доминирование одной модальности над другой.

```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

def staged_training_sim(steps=200, key=jax.random.PRNGKey(7)):
    """Simulate multimodal training with adjustable modality loss weights."""
    # Two 'modalities' with different loss scales (text loss ~10x larger than image loss)
    losses_text, losses_img = [], []
    param = jnp.array([0.0, 0.0])  # Shared param updated by both modality losses
    lr = 0.05
    # Try changing these weights to see the effect on convergence balance
    lambda_text, lambda_img = 1.0, 5.0  # upweight the weaker modality

    for step in range(steps):
        k1, k2, key = jax.random.split(key, 3)
        noise_t = jax.random.normal(k1, ()) * 0.3
        noise_i = jax.random.normal(k2, ()) * 0.1
        loss_t = (param[0] - 3.0) ** 2 + noise_t  # text target = 3.0
        loss_i = 0.1 * (param[1] - 1.0) ** 2 + noise_i  # image target = 1.0 (smaller scale)
        # Weighted combined gradient
        grad_t = lambda_text * 2 * (param[0] - 3.0)
        grad_i = lambda_img * 0.2 * (param[1] - 1.0)
        param = param - lr * jnp.array([grad_t, grad_i])
        losses_text.append(float(loss_t)); losses_img.append(float(loss_i))

    fig, ax = plt.subplots(figsize=(8, 4))
    ax.plot(losses_text, label=f"Text loss (weight={lambda_text})", alpha=0.7)
    ax.plot(losses_img, label=f"Image loss (weight={lambda_img})", alpha=0.7)
    ax.set_xlabel("Training step"); ax.set_ylabel("Loss"); ax.legend()
    ax.set_title("Modality loss balancing during staged training")
    plt.tight_layout(); plt.show()

staged_training_sim()
# Experiment: set lambda_img=1.0 and watch image loss converge much slower
```
