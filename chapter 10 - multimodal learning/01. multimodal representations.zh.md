# 多模态表示

*多模态表示将视觉、语言与音频桥接到共享 embedding 空间。本文件涵盖融合策略、CLIP、ALIGN、SigLIP、contrastive loss 函数（InfoNCE、NT-Xent）、zero-shot 分类与检索评估。*

- 想象你坐在咖啡馆。你看到桌上冒热气的杯子、听到陶瓷碰撞声、闻到烘焙咖啡豆的香气、感到杯具散发的热度。没有单一感官告诉你一切：大脑把这些信号融合为"热咖啡"的统一感知。**Multimodal learning**（多模态学习）为机器做同样的事：它结合来自多个模态（视觉、语言、音频及其他）的信息，构建比任何单一模态更丰富、更鲁棒的表示。

- 一个 **modality**（模态）是一个独立的信息通道。在机器学习中，最常见的模态有图像（像素网格）、文本（token 序列）、音频（waveform 或 spectrogram，见第 9 章）、视频（帧序列）与结构化数据（表格、图）。每个模态有自己的统计结构：图像空间连贯，文本序列且离散，音频时序且连续。多模态学习的挑战是桥接这些根本不同的数据类型。

- 为什么要组合模态？因为它们提供互补信息。一张狗的照片告诉你品种和毛色却不知道名字。一句 "my golden retriever Max" 的说明告诉你名字和品种却不知道具体姿态。合在一起，图像与文字给出比任一单独更完整的图景。这种互补性是核心动机：多模态模型能回答问题、生成内容、做出任何单模态模型无法做的决策。

## 融合策略

- 想象一个小组项目。有两种合并想法的方式：所有人从一开始就在同一间屋子里一起工作（共享原始笔记与草稿），或每人独立写自己那部分再合并最终文档。这对应多模态学习中的 **early fusion**（早期融合）与 **late fusion**（晚期融合）。

- **Early fusion**（也称特征级融合）在任何严肃处理发生前，拼接或混合不同模态的原始或低层特征。例如，可以把图像像素特征与文本 token embedding 拼接，把组合序列送入单个 transformer。模型从一开始就能学习细粒度跨模态交互，但输入空间大，模型须学会同时处理差异很大的数据类型。

- 形式上，给定两个模态的特征向量 $x_{\text{img}} \in \mathbb{R}^{d_1}$ 与 $x_{\text{txt}} \in \mathbb{R}^{d_2}$，early fusion 简单拼接：

$$x_{\text{fused}} = [x_{\text{img}}; x_{\text{txt}}] \in \mathbb{R}^{d_1 + d_2}$$

- 该拼接向量再由共享网络处理。优点是模型可在每一层发现跨模态相关。缺点是计算开销大，且对齐差异很大的特征类型（密集像素值 vs. 稀疏 token 索引）困难。

- **Late fusion**（也称决策级融合）让每个模态独立通过自己的 encoder 处理，产生高层表示甚至最终预测。这些输出再被组合，通常通过取平均得分、投票或学习的组合层。late fusion 简单且可复用现成预训练单模态模型，但无法捕捉低层跨模态交互，因为各模态从不"看到"彼此的原始特征。

- 给定模态特定预测 $\hat{y}_1$ 与 $\hat{y}_2$，一个简单的 late fusion 规则是：

$$\hat{y} = \alpha \hat{y}_1 + (1 - \alpha) \hat{y}_2$$

- 其中 $\alpha \in [0, 1]$ 是学习或手工调的混合权重。

- **Middle fusion**（也称中间融合）是大多数现代系统所用的务实折中。每个模态先由自己的 encoder 处理（提取模态特定特征），再在网络中部（常通过 cross-attention 层）组合编码后的表示。这让每个 encoder 专精于自己的模态，同时仍能实现丰富的跨模态交互。Flamingo、LLaVA 与大多数 vision-language model（第 02 篇文件）使用 middle fusion。

![early、middle 与 late fusion 策略：early fusion 拼接原始输入，middle fusion 通过 cross-attention 合并中间表示，late fusion 组合最终预测](../images/fusion_strategies.svg)

- 融合策略的选择取决于数据可用性、算力预算与任务。early fusion 强大但需数据多。late fusion 廉价但受限。带 cross-attention 的 middle fusion 已成为大规模多模态模型的主导方法，因为它在表达力与模块性之间取得平衡。

## 联合 embedding 空间

- 想象一台万能翻译机，能把任何语言的任何句子映射到共享"意义空间"中的同一点。英语、法语或日语的 "a dog on a beach" 都落在同一坐标。**Joint embedding space**（联合 embedding 空间）正是如此，但跨模态：海滩上的狗的图像与文本 "a dog on a beach" 应映射到同一向量空间中相近的点。

- 形式上，我们学习两个 encoder 函数：模态 1（如图像）的 $f_\theta : \mathcal{X}_1 \to \mathbb{R}^d$ 与模态 2（如文本）的 $g_\phi : \mathcal{X}_2 \to \mathbb{R}^d$。两者都把输入映射到同一 $d$ 维空间。训练目标确保语义匹配的对 $(x_1, x_2)$ 的 embedding $f_\theta(x_1)$ 与 $g_\phi(x_2)$ 接近（余弦相似度高），而不匹配的对相距远。

- 这是对第 7 章 word embedding 空间的直接推广。回忆 Word2Vec 与 GloVe 把语义相似的词放在向量空间中彼此附近。联合 embedding 空间跨模态推广该思想：我们度量图像-文本相似度、音频-文本相似度乃至图像-音频相似度，而非词-词相似度。

- 相似度度量几乎总是 **cosine similarity**（余弦相似度，第 1 章）：

$$\text{sim}(u, v) = \frac{u \cdot v}{\|u\| \|v\|}$$

- 通过对所有 embedding 做 $L_2$ 归一化到单位超球面，余弦相似度退化为简单点积 $u \cdot v$，计算极高效，且可用近似最近邻库加速。

![联合 embedding 空间：图像 encoder 与文本 encoder 将各自输入映射到共享向量空间，匹配的对聚集在一起](../images/joint_embedding_space.svg)

- 联合 embedding 空间的能力在于它使 **zero-shot transfer**（零样本迁移）成为可能。一旦对齐了图像与文本 embedding，就能把图像分类到从未训练过的类别：只需把类别名作为文本 embedding，找与图像 embedding 最近的文本 embedding 即可。无需任务特定 fine-tuning。这是 CLIP 及其后继的核心洞见。

## 用于多模态对齐的 Contrastive Learning

- 想象一个课堂练习：学生拿到打乱的照片与说明对，要把每张照片匹配到正确说明。要做好这件事，需同时理解视觉内容与语言以及它们的关系。**Contrastive learning**（对比学习）正是这样训练模型：给定一批 (image, text) 对，模型要弄清哪张图配哪段文。

- 如第 8 章（第 04 篇文件）所见，单模态场景的 contrastive learning（SimCLR、MoCo）拉近同一图像的增强视图、推开不同图像的视图。多模态 contrastive learning 用"匹配模态"代替"增强视图"：图像与它的说明是正对；图像与批内任何其他说明配对则是负对。

### CLIP

- **CLIP**（Contrastive Language-Image Pre-training，Radford 等，2021）是多模态对比学习的基础模型。它在 4 亿对从互联网抓取的 (image, text) 对上联合训练图像 encoder（ViT 或 ResNet，第 8 章）与文本 encoder（transformer，第 7 章）。

- 给定一批 $N$ 个图文对，CLIP 计算所有图像 embedding 与所有文本 embedding 之间余弦相似度的 $N \times N$ 矩阵。对角元素是匹配对（正例）；所有非对角元素是不匹配的（负例）。训练 loss 推高对角元素、压低非对角元素。

- 该 loss 是对称 cross-entropy。对图像 $i$ 配文本 $j = i$，image-to-text loss 为：

$$\mathcal{L}_{i \to t} = -\frac{1}{N} \sum_{i=1}^{N} \log \frac{\exp(\text{sim}(z_i^{\text{img}}, z_i^{\text{txt}}) / \tau)}{\sum_{k=1}^{N} \exp(\text{sim}(z_i^{\text{img}}, z_k^{\text{txt}}) / \tau)}$$

- text-to-image loss 角色互换同理：

$$\mathcal{L}_{t \to i} = -\frac{1}{N} \sum_{i=1}^{N} \log \frac{\exp(\text{sim}(z_i^{\text{txt}}, z_i^{\text{img}}) / \tau)}{\sum_{k=1}^{N} \exp(\text{sim}(z_i^{\text{txt}}, z_k^{\text{img}}) / \tau)}$$

- CLIP 总 loss 是两者平均：

$$\mathcal{L}_{\text{CLIP}} = \frac{1}{2}(\mathcal{L}_{i \to t} + \mathcal{L}_{t \to i})$$

- 这里 $\tau$ 是可学习的 **temperature**（温度）参数（初始化为 $\tau = 0.07$）。温度控制 softmax 分布的锐度：低 $\tau$ 使模型更聚焦于最近匹配，高 $\tau$ 让概率更均匀。CLIP 把 $\tau$ 与模型权重联合学习，而非当作固定超参。

![CLIP 训练：一批 N 个图文对产生 NxN 相似度矩阵，训练最大化对角元素、最小化非对角元素](../images/clip_contrastive_matrix.svg)

- CLIP 的图像 encoder 通常是 ViT-L/14（带 14x14 patch 的大型 Vision Transformer，第 8 章第 04 篇文件）。文本 encoder 是带 causal masking 的 12 层 transformer（如 GPT，第 7 章第 04 篇文件）。两个 encoder 通过学习的线性投影将输出投射到共享的 512 或 768 维空间，再做 $L_2$ 归一化。

- CLIP 最引人注目的特性是 **zero-shot 图像分类**。要把图像分类到 $K$ 类之一，构造 $K$ 个文本 prompt 如 "a photo of a {class name}"，用文本 encoder 嵌入每个 prompt，用图像 encoder 嵌入图像，选与图像 embedding 余弦相似度最高的文本 embedding 对应的类。在 ImageNet 上，CLIP 从未见过任何 ImageNet 训练样本就取得具竞争力的准确率。

### ALIGN

- **ALIGN**（Jia 等，2021）把 CLIP 的方法扩展到更嘈杂、更大的数据集：18 亿个图文对，仅最低限度过滤。CLIP 精心整理数据，而 ALIGN 表明规模可补偿噪声。ALIGN 用 EfficientNet 图像 encoder 与 BERT 文本 encoder，用相同 contrastive loss 训练。关键发现是：数据足够多时，无需昂贵的数据清洗——对比目标会自然降低嘈杂对的权重，因为它们产生不一致梯度。

### SigLIP

- **SigLIP**（Sigmoid Loss for Language-Image Pre-training，Zhai 等，2023）用更简单的 sigmoid loss 取代 CLIP 的基于 softmax 的 contrastive loss。SigLIP 不把 $N \times N$ 相似度矩阵当作分类问题（每行是列上的 softmax），而是把每个元素独立当作二元分类：这个 (image, text) 对是否匹配？

- 单对 $(i, j)$ 的 SigLIP loss 为：

$$\mathcal{L}_{ij} = -y_{ij} \log \sigma(z_i^{\text{img}} \cdot z_j^{\text{txt}} / \tau) - (1 - y_{ij}) \log(1 - \sigma(z_i^{\text{img}} \cdot z_j^{\text{txt}} / \tau))$$

- 其中 $i = j$（匹配）时 $y_{ij} = 1$，否则为 0，$\sigma$ 是 sigmoid 函数。

- SigLIP 的关键优势是消除了对整批全局 softmax 归一化的需求。CLIP 中，softmax 分母需收集所有设备上的所有 embedding，这是分布式训练的通信瓶颈。SigLIP 的逐对 sigmoid loss 可本地计算，使其更高效地扩展到超大批次。SigLIP 以更低训练成本达到与 CLIP 相当的质量。

## Contrastive Loss 函数详解

- contrastive learning 所用的 loss 函数共享一个结构：都试图让正对相似度高于负对相似度，并用某种"margin"或"temperature"控制模型推得多用力。让我们形式化关键变体。

### InfoNCE

- **InfoNCE**（Noise-Contrastive Estimation，van den Oord 等，2018）是 CLIP loss 背后的理论基础。给定 query $q$、一个正 key $k^+$ 与 $K$ 个负 key $\{k_1^-, \ldots, k_K^-\}$，loss 为：

$$\mathcal{L}_{\text{InfoNCE}} = -\log \frac{\exp(q \cdot k^+ / \tau)}{\exp(q \cdot k^+ / \tau) + \sum_{j=1}^{K} \exp(q \cdot k_j^- / \tau)}$$

- 这是一个 $(K+1)$ 路分类问题：在 $K+1$ 个候选中识别正例。InfoNCE 是 query 与正 key 之间互信息的下界，这就是为何最大化它能对齐语义匹配输入的表示。下界随负例数 $K$ 增加而变紧，解释了为何对比方法受益于大批次。

### NT-Xent

- **NT-Xent**（Normalised Temperature-scaled Cross-Entropy，Chen 等，2020）是 SimCLR（第 8 章第 04 篇文件）所用 loss，本质上是在批内对称应用的 InfoNCE。对 $N$ 对的批，$2N$ 个增强视图为每个锚产生 $2N - 2$ 个负例（除自身与其正例外的所有视图）。正对 $(i, j)$ 的 loss 为：

$$\ell_{i,j} = -\log \frac{\exp(\text{sim}(z_i, z_j) / \tau)}{\sum_{k=1}^{2N} \mathbf{1}_{[k \neq i]} \exp(\text{sim}(z_i, z_k) / \tau)}$$

- NT-Xent 与 InfoNCE 是同一数学公式；名字不同只因引入语境不同（自监督视觉 vs. 表示学习理论）。

### 温度的作用

- **temperature** $\tau$ 是 contrastive learning 中最重要的超参之一。为建立直觉，可联想物理学中的温度：高温下分子随机运动（softmax 平坦，所有负例看起来同样糟糕）；低温下分子排列成刚性结构（softmax 尖锐，只有最难的负例重要）。

- 形式上，$\tau \to 0$ 时 softmax 趋近只选最硬负例的 hard argmax。$\tau \to \infty$ 时所有负例等贡献。实践中，$\tau \in [0.01, 0.1]$ 对归一化 embedding 效果良好。温度过低导致训练不稳（对硬负例梯度变得很大）；温度过高使 loss 对违例不敏感。

- CLIP 初始化 $\tau = 0.07$，并作为 log 参数化的标量 $\tau = \exp(t)$ 学习，其中 $t$ 与模型权重一起由梯度下降更新。这使模型能在训练中自动调整对比任务的难度。

![温度对对比 softmax 的影响：低温产生聚焦于硬负例的尖锐分布，高温产生平坦分布](../images/contrastive_temperature.svg)

### Triplet Loss 与基于 Margin 的替代

- 在 InfoNCE 主导前，**triplet loss** 是 metric learning 的标准。给定 anchor $a$、正例 $p$、负例 $n$：

$$\mathcal{L}_{\text{triplet}} = \max(0, \|a - p\|^2 - \|a - n\|^2 + m)$$

- 其中 $m$ 是确保正例至少比负例近 $m$ 的 margin。triplet loss 作用于单个三元组而非批，使其比 InfoNCE 样本效率低。它也对挖掘策略敏感：随机负例常太容易（loss 为零），因此 **hard negative mining**（选最近的错误匹配）或 **semi-hard mining**（选 margin 内的负例）至关重要。

- InfoNCE 在整批上隐式执行硬负例挖掘，这是它在规模上优于 triplet loss 的原因之一。InfoNCE 的 softmax 归一化自动给硬负例（与 anchor 相似度高者）更高权重，提供无需显式挖掘的自然课程。

## 图文检索与 Zero-Shot 分类

- 一旦有训练好的联合 embedding 空间，就可做 **image-text retrieval**（图文检索）：给定图像 query，从数据库找最相关文本（image-to-text retrieval），或给定文本 query 找最相关图像（text-to-image retrieval）。这就是共享 embedding 空间中的最近邻搜索。

- 想象一位图书管理员能瞬间把任何照片与百万条目目录中的任何说明比较。他无需事先理解每个类别，只度量每张照片与每条说明有多"近"。这就是 CLIP 式模型做检索与 zero-shot 分类的方式。

- **Zero-shot 分类**是 text-to-image retrieval 的特例。给定 $K$ 个类名，构造文本 prompt $\{t_1, \ldots, t_K\}$（如 "a photo of a cat"、"a photo of a dog"）并嵌入。对新图像 $x$，预测类为：

$$\hat{y} = \arg\max_{k} \; \text{sim}(f_\theta(x), g_\phi(t_k))$$

- 关键洞见是文本 encoder 充当灵活的分类头。无需为每个下游任务训练新线性层，只需用自然语言描述任务。这是 CLIP 泛化良好的原因：文本 encoder 在预训练中见过数百万多样描述。

- **Prompt engineering** 很重要。CLIP 在 ImageNet 上的 zero-shot 准确率从 63.2% 提升到 68.4%，仅靠把 prompt 模板从 "{class name}" 改为 "a photo of a {class name}"。更好的是，**prompt ensembling** 对多个模板（如 "a photo of a {class name}"、"a good photo of a {class name}"、"a drawing of a {class name}"）的文本 embedding 取平均，产生更鲁棒的文本表示。

## 音频-视觉对应

- 闭上眼听某人拍篮球。你能从有节奏的砰砰声中判断何时落地。睁开眼：视觉弹跳与每次砰声完美对齐。音频与视觉事件之间这种紧密对应是机器可学习的免费监督信号。**Audio-visual correspondence learning**（音频-视觉对应学习）训练模型把声音与视觉来源关联，无需任何人类标注。

- 该思想与 CLIP 惊人地相似，但用音频代替文本。给定配对的视频帧与音频段，模型学习一个 embedding 空间，其中时间对齐的音视频对接近，未对齐的相距远。

- **Audio-Visual Embedding (AVE)** 方法（Arandjelovic 与 Zisserman，2017）在视频数据上用 contrastive loss 训练视觉 encoder $f$ 与音频 encoder $g$。正对是（同一时刻的视频帧、音频片段），负例是来自不同视频或不同时刻的音频片段。模型学到狗叫声配狗的图像、吉他声配吉他的图像，全程无标签。

- 音频 encoder 通常用 CNN 或音频 transformer 处理 **log-mel spectrogram**（第 9 章第 01 篇文件），产生定长 embedding。视觉 encoder 用标准图像骨干（ResNet、ViT）处理视频帧。两者都投射到共享 $d$ 维空间，训练用与 CLIP 相同的 InfoNCE loss：

$$\mathcal{L}_{\text{AV}} = -\log \frac{\exp(\text{sim}(z^{\text{vis}}, z^{\text{aud}}) / \tau)}{\sum_{k=1}^{N} \exp(\text{sim}(z^{\text{vis}}, z_k^{\text{aud}}) / \tau)}$$

![音频-视觉对应：视觉 encoder 处理视频帧、音频 encoder 处理 spectrogram，contrastive learning 对齐时间匹配的对](../images/audio_visual_correspondence.svg)

- 音频-视觉学习的**应用**包括：声源定位（声音来自图像何处？）、音频-视觉语音识别（结合嘴唇动作与音频，如第 9 章第 02 篇文件）、音频-视觉 source separation（通过观看人脸分离某说话人的声音，第 9 章第 05 篇文件的"cocktail party"问题）以及条件于音频的视频生成。

- **ImageBind**（Girdhar 等，2023）将其扩展到六种模态：图像、文本、音频、深度、热成像与 IMU 数据。关键洞见是不需要每种组合都有配对数据。通过把每个模态对齐到图像（文本用图文对、音频用音图对等），所有模态通过共享的图像 embedding 空间隐式对齐。这种通过共同锚点模态的"绑定"产生涌现对齐：音频与文本变得相似，尽管它们从未被直接配对训练。

## 评估

- 评估多模态模型需要能捕捉跨模态理解的指标。两种主导评估范式是 **zero-shot benchmark** 与 **retrieval metric**。

### Zero-Shot 基准

- zero-shot 评估度量模型能否执行从未被显式训练的任务。最常见基准是 **ImageNet zero-shot accuracy**：把全部 1,000 个 ImageNet 类名作为文本嵌入，嵌入每个测试图像，按余弦相似度度量 top-1 与 top-5 分类准确率。CLIP ViT-L/14 取得 75.5% 的 top-1 zero-shot 准确率，可与在 ImageNet 上监督训练的 ResNet-50 相当。

- 其他 zero-shot 基准包括：CIFAR-10/100、STL-10、Food-101、Oxford Pets 与 Flowers-102。跨多个数据集评估可测试模型是否真正具备通用视觉理解，还是仅记忆了预训练数据的模式。

- **Linear probe**（线性探测）评估是互补测试。冻结预训练图像 encoder，为标注数据集提取特征，在其上训练简单线性分类器。这独立于 zero-shot 检索机制度量所学表示的质量。CLIP 的特征是优秀的 linear probe 特征，常匹配或超过监督预训练。

### 检索指标

- 对检索任务（image-to-text 与 text-to-image），标准指标是 **Recall@K**（R@K）：正确匹配出现在前 $K$ 个检索结果中的 query 占比。常用值为 R@1、R@5、R@10。

- 形式上，对 $Q$ 个 query 的集合：

$$\text{R@}K = \frac{1}{Q} \sum_{q=1}^{Q} \mathbf{1}[\text{rank}(q) \leq K]$$

- 其中 $\text{rank}(q)$ 是正确匹配在 query $q$ 的排序列表中的位置。

- 标准检索基准包括 **Flickr30K**（31,000 张图、每张 5 条说明）与 **MS-COCO**（123,000 张图、每张 5 条说明）。在测试划分上评估：给定图像，从完整测试集检索正确说明（反之亦然）。

- **Median rank**（MedR）是互补指标：所有 query 中正确匹配的中位位置。完美模型 MedR = 1。越低越好。

- 除检索外，多模态模型还在组合理解基准上评估，如 **Winoground**（测试模型能否区分 "a mug in a dog" 与 "a dog in a mug"）与 **ARO**（Attribute, Relation, Order），测试模型是真正理解语言结构还是仅匹配词袋。CLIP 式模型常在这些上挣扎，揭示一个根本局限：contrastive 预训练对齐全局语义但可能不捕捉细粒度组合结构。

## 综合起来

- 本文件涵盖的多模态表示构成本章后续内容的基础。CLIP 及其后继训练的联合 embedding 空间是连接视觉与语言的"胶水"。第 02 篇文件在此基础上构建超越检索、生成关于图像文本的 vision-language model。第 03 篇文件探讨图像与视频如何被 token 化用于序列模型。第 04 篇文件涵盖跨模态生成（text-to-image、text-to-video）。第 05 篇文件考察在单个模型内处理多模态的统一架构。

- 核心要点：在配对数据上的 contrastive learning 产生不同模态可互换的 embedding 空间。图像 embedding 与文本 embedding 变成"同一种东西"，实现 zero-shot 分类、检索与无缝集成到更大系统。该思想的简单性——只是把匹配对拉近、不匹配对推开——掩盖了其非凡的有效性。

1. 从零实现 CLIP contrastive loss。创建随机图像与文本 embedding，计算相似度矩阵，并计算对称 cross-entropy loss。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

def clip_loss(image_embeds, text_embeds, temperature=0.07):
    """Compute symmetric CLIP contrastive loss."""
    # L2 normalise embeddings
    image_embeds = image_embeds / jnp.linalg.norm(image_embeds, axis=1, keepdims=True)
    text_embeds = text_embeds / jnp.linalg.norm(text_embeds, axis=1, keepdims=True)

    # Compute cosine similarity matrix (N x N)
    logits = image_embeds @ text_embeds.T / temperature  # (N, N)

    # Labels: the diagonal (i-th image matches i-th text)
    N = logits.shape[0]
    labels = jnp.arange(N)

    # Symmetric cross-entropy: image-to-text + text-to-image
    loss_i2t = -jnp.mean(jax.nn.log_softmax(logits, axis=1)[jnp.arange(N), labels])
    loss_t2i = -jnp.mean(jax.nn.log_softmax(logits, axis=0)[labels, jnp.arange(N)])
    return (loss_i2t + loss_t2i) / 2, logits * temperature

# Simulate a batch of 8 image-text pairs in 64-dim space
key = jax.random.PRNGKey(42)
k1, k2 = jax.random.split(key)
N, D = 8, 64
image_embeds = jax.random.normal(k1, (N, D))
text_embeds = jax.random.normal(k2, (N, D))

loss, sim_matrix = clip_loss(image_embeds, text_embeds)
print(f"CLIP loss (random embeddings): {loss:.4f}")

# Visualise the similarity matrix
fig, ax = plt.subplots(figsize=(6, 5))
im = ax.imshow(sim_matrix, cmap='coolwarm', vmin=-1, vmax=1)
ax.set_xlabel("Text index"); ax.set_ylabel("Image index")
ax.set_title(f"Cosine Similarity Matrix (loss={loss:.3f})")
plt.colorbar(im); plt.tight_layout(); plt.show()
# Try changing temperature (0.01, 0.1, 1.0) and observe how loss changes
# Try making matched pairs similar: set text_embeds = image_embeds + small noise
```

2. 构建一个玩具联合 embedding 模型，用 InfoNCE loss 与梯度下降学习把二维"图像"（随机向量）与"说明"（不同随机向量）对齐。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

def info_nce_loss(img_enc, txt_enc, img_data, txt_data, tau=0.1):
    """InfoNCE over a batch of paired (image, text) data."""
    z_img = img_data @ img_enc  # (N, D)
    z_txt = txt_data @ txt_enc  # (N, D)
    # L2 normalise
    z_img = z_img / jnp.linalg.norm(z_img, axis=1, keepdims=True)
    z_txt = z_txt / jnp.linalg.norm(z_txt, axis=1, keepdims=True)
    logits = z_img @ z_txt.T / tau
    labels = jnp.arange(logits.shape[0])
    return -jnp.mean(jax.nn.log_softmax(logits, axis=1)[jnp.arange(len(labels)), labels])

# Create 32 paired samples: img in R^8, txt in R^6, embed into R^4
key = jax.random.PRNGKey(0)
k1, k2, k3, k4 = jax.random.split(key, 4)
N, d_img, d_txt, d_embed = 32, 8, 6, 4

img_data = jax.random.normal(k1, (N, d_img))
txt_data = jax.random.normal(k2, (N, d_txt))

# Learnable projection matrices
img_enc = jax.random.normal(k3, (d_img, d_embed)) * 0.1
txt_enc = jax.random.normal(k4, (d_txt, d_embed)) * 0.1

grad_fn = jax.jit(jax.grad(info_nce_loss, argnums=(0, 1)))
lr = 0.05
losses = []

for step in range(300):
    loss = info_nce_loss(img_enc, txt_enc, img_data, txt_data)
    losses.append(float(loss))
    g_img, g_txt = grad_fn(img_enc, txt_enc, img_data, txt_data)
    img_enc = img_enc - lr * g_img
    txt_enc = txt_enc - lr * g_txt

print(f"Initial loss: {losses[0]:.3f}, Final loss: {losses[-1]:.3f}")
print(f"Random baseline (log N): {jnp.log(N):.3f}")

plt.figure(figsize=(8, 4))
plt.plot(losses, color='#2c3e50')
plt.axhline(y=0, color='green', linestyle='--', alpha=0.5, label='Perfect alignment')
plt.axhline(y=float(jnp.log(N)), color='red', linestyle='--', alpha=0.5, label='Random (log N)')
plt.xlabel("Step"); plt.ylabel("InfoNCE Loss")
plt.title("Learning a Joint Embedding Space")
plt.legend(); plt.grid(alpha=0.3); plt.tight_layout(); plt.show()
# Modify d_embed (try 2, 4, 16) to see how embedding dimension affects alignment
```

3. 用预计算 embedding 实现 zero-shot 分类。把类"原型"模拟为文本 embedding，通过最近邻查找分类新图像。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

# Simulate 5 classes, each with a prototype text embedding in R^32
key = jax.random.PRNGKey(42)
n_classes, d = 5, 32
class_names = ["cat", "dog", "car", "plane", "ship"]

# Class prototypes (imagine these came from a text encoder)
k1, k2 = jax.random.split(key)
class_prototypes = jax.random.normal(k1, (n_classes, d))
class_prototypes = class_prototypes / jnp.linalg.norm(class_prototypes, axis=1, keepdims=True)

# Generate 200 test "images" (embeddings near their class prototype + noise)
n_per_class = 40
true_labels = jnp.repeat(jnp.arange(n_classes), n_per_class)
keys = jax.random.split(k2, n_classes * n_per_class)

image_embeds = []
for i in range(n_classes):
    noise = jax.random.normal(keys[i], (n_per_class, d)) * 0.5
    cluster = class_prototypes[i] + noise
    image_embeds.append(cluster)
image_embeds = jnp.concatenate(image_embeds, axis=0)
image_embeds = image_embeds / jnp.linalg.norm(image_embeds, axis=1, keepdims=True)

# Zero-shot classification: cosine similarity with each prototype
similarities = image_embeds @ class_prototypes.T  # (200, 5)
predicted_labels = jnp.argmax(similarities, axis=1)
accuracy = jnp.mean(predicted_labels == true_labels)
print(f"Zero-shot accuracy: {accuracy:.1%}")

# Confusion matrix
conf = jnp.zeros((n_classes, n_classes), dtype=jnp.int32)
for true, pred in zip(true_labels, predicted_labels):
    conf = conf.at[true, pred].add(1)

fig, ax = plt.subplots(figsize=(6, 5))
im = ax.imshow(conf, cmap='Blues')
ax.set_xticks(range(n_classes)); ax.set_xticklabels(class_names, rotation=45)
ax.set_yticks(range(n_classes)); ax.set_yticklabels(class_names)
ax.set_xlabel("Predicted"); ax.set_ylabel("True")
for i in range(n_classes):
    for j in range(n_classes):
        ax.text(j, i, int(conf[i, j]), ha='center', va='center', fontsize=11)
ax.set_title(f"Zero-Shot Confusion Matrix (acc={accuracy:.1%})")
plt.colorbar(im); plt.tight_layout(); plt.show()
# Try increasing noise (0.5 -> 1.0 -> 2.0) to see accuracy degrade
# Try adding prompt ensembling: average 3 noisy copies of each prototype
```
