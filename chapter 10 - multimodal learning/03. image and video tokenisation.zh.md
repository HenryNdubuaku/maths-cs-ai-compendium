# 图像与视频 tokenisation

*图像与视频 tokenisation 将连续视觉数据转换为 transformer 可像文本一样处理的离散 token 序列。本文件涵盖 VQ-VAE、VQ-GAN、codebook 学习、DALL-E 的 dVAE、视频 tokenisation 与 lookup-free quantisation。*

## 为何要 tokenise 图像

- 把语言想成有限字母表：英语约 26 个字母，现代语言模型把文本切成 30,000-100,000 个子词 token。每个句子成为一串 transformer 可逐个预测的离散符号。而图像生活在连续高维空间：一张 256x256 RGB 图像是 $\mathbb{R}^{256 \times 256 \times 3} \approx \mathbb{R}^{196{,}608}$ 中的一点。若要语言模型用说英语的同一套机制"说"图像，需把这些连续像素阵列转换为取自有限词表的可管理离散 token 序列。这一转换就是 **image tokenisation**（图像 tokenisation）。

- 想象你是马赛克艺术家。你没有无限种瓷砖色，只有固定调色板，比如 8192 种。要把照片再现为马赛克，须 (1) 决定每块瓷砖代表照片哪片区域，(2) 为每片区域选最接近的瓷砖色，(3) 接受一些细节丢失但整体可辨。image tokenisation 正是如此：encoder 把空间 patch 压缩为 latent 向量，codebook 把每个向量映射到最近项，结果是一个每 patch 一个整数索引的网格，可供离散模型处理。

- tokenisation 的好处有三。其一，大幅压缩图像：256x256 图像可变为 16x16 token 网格，序列长从 65,536 像素降到 256 token，对代价随序列长二次增长的 attention 模型可处理。其二，统一表示：文本 token 与图像 token 处于同一离散词表，使单个自回归 transformer 可生成交错文本与图像。其三，强加有用瓶颈，迫使模型学习语义有意义的码而非记忆像素噪声。

![image tokenisation 流水线概览：连续图像进入 encoder，latent 向量对 codebook 量化，产生离散 token 索引网格](../images/image_tokenisation_overview.svg)

- 回忆第 8 章卷积网络如何从图像提取层次特征图，第 7 章文本 tokenizer 如何把字符串转为整数序列。image tokenisation 处于交叉点：用 CNN 或 vision transformer encoder（第 8 章）产生空间特征，再借用离散词表思想（第 7 章）把这些特征转为 token 索引。

## VQ-VAE：向量量化

- 如第 6 章所见，标准 **variational autoencoder**（VAE）把输入编码为连续 latent 分布，再从该分布采样解码重建。latent 空间连续，难以送入离散序列模型。van den Oord 等（2017）提出的 **Vector Quantised Variational Autoencoder**（VQ-VAE）用可学习的 codebook 引入离散 latent：把每个 encoder 输出"吸附"到最近的 codebook 项。

- 想象一个恰好有 $K$ 个标签书架的图书馆。新书（encoder 输出）到来时，馆员把它放上与已有书（codebook 向量）最相似的书架，并记下书架号。之后取书只需书架号：那架上的 codebook 项足够替代。这就是向量量化。

- 形式上，VQ-VAE 有三部分：

- **encoder** $E$ 把输入图像 $\mathbf{x} \in \mathbb{R}^{H \times W \times 3}$ 映射为连续 latent 向量的空间网格 $\mathbf{z}_e = E(\mathbf{x}) \in \mathbb{R}^{h \times w \times d}$，其中 $h \times w$ 是下采样空间分辨率，$d$ 是 embedding 维度。

- **codebook** $\mathcal{C} = \{\mathbf{e}_1, \mathbf{e}_2, \ldots, \mathbf{e}_K\} \subset \mathbb{R}^d$ 含 $K$ 个可学习 embedding 向量。典型 codebook 大小 512 到 16,384。

- **decoder** $D$ 从量化 latent 重建图像。

- **quantisation step** 把空间位置 $(i, j)$ 的 encoder 输出 $\mathbf{z}_e(\mathbf{x})$ 替换为最近 codebook 项：

$$\mathbf{z}_q(i,j) = \mathbf{e}_{k^\ast} \quad \text{where} \quad k^\ast = \arg\min_k \|\mathbf{z}_e(i,j) - \mathbf{e}_k\|_2$$

- 这是 embedding 空间中的最近邻查找，与 k-means 分配（第 6 章）操作相同。索引 $k^\ast$ 是空间位置 $(i,j)$ 的离散 token，整张图像表示为取自 $\{1, \ldots, K\}$ 的 $h \times w$ 整数网格。

![VQ-VAE 架构：encoder 产生连续 latent，每个 latent 向量匹配最近 codebook 项，decoder 从量化码重建](../images/vqvae_architecture.svg)

- 挑战在于 $\arg\min$ 不可微：无法通过离散选择反向传播。VQ-VAE 用 **straight-through estimator** 解决：前向时 decoder 接收 $\mathbf{z}_q$（量化向量）；反向时重建 loss 对 $\mathbf{z}_q$ 的梯度直接复制给 $\mathbf{z}_e$，仿佛量化步骤是恒等函数。紧凑写作：

$$\mathbf{z}_q = \mathbf{z}_e + \text{sg}(\mathbf{z}_q - \mathbf{z}_e)$$

- 其中 $\text{sg}(\cdot)$ 是 stop-gradient 算子。前向时等于 $\mathbf{z}_q$；反向时梯度只流过 $\mathbf{z}_e$ 项。

- VQ-VAE 完整 loss 有三项：

$$\mathcal{L} = \underbrace{\|\mathbf{x} - D(\mathbf{z}_q)\|_2^2}_{\text{reconstruction}} + \underbrace{\|\text{sg}(\mathbf{z}_e) - \mathbf{e}\|_2^2}_{\text{codebook (VQ)}} + \underbrace{\beta \|\mathbf{z}_e - \text{sg}(\mathbf{e})\|_2^2}_{\text{commitment}}$$

- **reconstruction loss** 训练 encoder 与 decoder 忠实重建输入。**codebook loss**（也称 VQ loss）把 codebook 向量拉向 encoder 输出；注意 $\text{sg}(\mathbf{z}_e)$ 意味 encoder 不从此项收梯度，故只更新 codebook。**commitment loss** 反向：鼓励 encoder 输出靠近 codebook 向量，防止 encoder "逃离"codebook。超参 $\beta$（通常 0.25）控制 codebook 与 commitment 项的平衡。

- 实践中，codebook 常用 **exponential moving average**（EMA）而非梯度下降更新，更稳定。设 $\mathbf{n}_k$ 为分配到 codebook 项 $k$ 的 encoder 输出计数，$\mathbf{s}_k$ 为其和。EMA 更新为：

$$\mathbf{n}_k \leftarrow \gamma \mathbf{n}_k + (1 - \gamma) |\{(i,j) : k^\ast_{ij} = k\}|$$

$$\mathbf{s}_k \leftarrow \gamma \mathbf{s}_k + (1 - \gamma) \sum_{(i,j) : k^\ast_{ij} = k} \mathbf{z}_e(i,j)$$

$$\mathbf{e}_k \leftarrow \frac{\mathbf{s}_k}{\mathbf{n}_k}$$

- 其中 $\gamma$ 是衰减率（通常 0.99）。这等价于在 encoder 输出上运行在线 k-means。

### Codebook Collapse

- VQ-VAE 一个臭名昭著的失败模式是 **codebook collapse**（也称 index collapse）：模型学会只用 $K$ 个 codebook 项中的一小部分，多数项"死掉"。想象一个图书馆 90% 书架空置，因为馆员总把书放到同样几个热门书架。这浪费表示容量。

- codebook collapse 发生因为 encoder、codebook 与 decoder 在训练中协同适应。若某项连续若干批未被选中，它就漂离 encoder 流形，更不可能被选中，形成正反馈循环。

- 缓解 codebook collapse 的几种技术：
    - **Codebook reset**：周期性地通过复制随机采样的 encoder 输出重初始化死项。给死项在 latent 空间活跃区附近的新起点。
    - **带 Laplace 平滑的 EMA 更新**：给 $\mathbf{n}_k$ 加小常数防止任何项计数为零，确保所有项都收梯度信号。
    - **Commitment loss 调参**：增大 $\beta$ 迫使 encoder 输出更紧地聚集在 codebook 项周围，更均匀地分配。
    - **Factorised codes**：把 codebook 查找分解为更小查找的乘积（如两个各 $\sqrt{K}$ 大小的 codebook），通过降低每次查找的有效 codebook 大小提升利用率。
    - **Entropy 正则化**：加入鼓励 codebook 使用均匀分布的惩罚，最大化 entropy $H = -\sum_k p_k \log p_k$，其中 $p_k$ 是经验分配概率。

## VQ-GAN：用于更高保真度的对抗训练

- VQ-VAE 产生尚可的重建，但像素级 $\ell_2$ loss 倾向于生成模糊输出，因为它等权惩罚每个像素偏差，对可能细节取平均而非选清晰者。想象让某人画一张最小化与所有可能面孔平均差异的脸——他会画一张模糊的平均脸，而非一张清晰的具体脸。

- **VQ-GAN**（Esser 等，2021）通过把 VQ-VAE 框架与 generative adversarial network（第 6 章）的 **discriminator** 结合解决此问题。discriminator 是基于 patch 的卷积网络，判断局部图像 patch 是真（来自训练数据）还是假（来自 decoder）。该对抗 loss 鼓励 decoder 产生感知上锐利、真实的纹理，而非像素级平均。

- VQ-GAN 目标在 VQ-VAE loss 上加两项：

$$\mathcal{L}_\text{VQ-GAN} = \mathcal{L}_\text{VQ-VAE} + \lambda_\text{adv} \mathcal{L}_\text{adv} + \lambda_\text{perc} \mathcal{L}_\text{perc}$$

- **adversarial loss** $\mathcal{L}_\text{adv}$ 是作用于 decoder 输出的标准 GAN 目标。discriminator $\mathcal{D}$ 试图区分真 patch 与解码 patch，decoder（generator）试图骗过它。非饱和形式为：

$$\mathcal{L}_\text{adv} = -\mathbb{E}[\log \mathcal{D}(D(\mathbf{z}_q))]$$

- **perceptual loss** $\mathcal{L}_\text{perc}$ 比较预训练网络（通常 VGG 或 LPIPS）在原图与重建图上的特征激活：

$$\mathcal{L}_\text{perc} = \sum_l \|\phi_l(\mathbf{x}) - \phi_l(D(\mathbf{z}_q))\|_2^2$$

- 其中 $\phi_l$ 是预训练网络第 $l$ 层的特征图。该 loss 捕捉高层结构相似性而非像素级准确度。

- 权重 $\lambda_\text{adv}$ 自适应设定，使对抗梯度与重建梯度平衡，防止训练早期重建差时对抗 loss 主导。

![VQ-GAN 训练：encoder 与 decoder 经量化步骤连接，patch discriminator 对解码输出提供对抗反馈](../images/vqgan_training.svg)

- 结果是一个在相同 codebook 大小下产生远比 VQ-VAE 锐利重建的 tokenizer。VQ-GAN 是许多主要图像生成系统背后的骨干 tokenizer，包括原版 DALL-E、Parti 及众多 text-to-image 模型。它把 256x256 图像转为来自 1024-16384 大小 codebook 的 16x16 或 32x32 离散 token 网格，每空间维 16x 到 64x 压缩比。

## 残差量化与多尺度 codebook

- 单一 codebook 对重建质量设下硬上限：每个空间位置恰好由一个 codebook 向量表示，比 codebook 能表达的更细的细节都会丢失。想象用固定调色板的一个词描述颜色："teal"接近但不精确。若能加一个修正——"teal，但略偏蓝且稍亮"——就接近得多。

- **Residual quantisation**（RQ，残差量化）迭代应用此思想。第一步量化产生 $\mathbf{z}_q^{(1)}$ 后，计算残差 $\mathbf{r}^{(1)} = \mathbf{z}_e - \mathbf{z}_q^{(1)}$，再对残差用第二个 codebook 量化得 $\mathbf{z}_q^{(2)}$，如此 $T$ 级：

$$\mathbf{r}^{(0)} = \mathbf{z}_e$$

$$\mathbf{z}_q^{(t)} = \text{Quantise}(\mathbf{r}^{(t-1)}, \mathcal{C}^{(t)})$$

$$\mathbf{r}^{(t)} = \mathbf{r}^{(t-1)} - \mathbf{z}_q^{(t)}$$

- 最终量化表示为 $\hat{\mathbf{z}} = \sum_{t=1}^{T} \mathbf{z}_q^{(t)}$。$T$ 级各用 $K$ 大小 codebook，有效词表大小为 $K^T$，但只需存储 $T \times K$ 个向量而非 $K^T$。例如 8 级配 $K = 1024$ 给出有效 $1024^8 \approx 10^{24}$ 项而只存 8192 个向量。

- 每级捕捉更细细节：第一 codebook 捕捉粗结构，第二捕捉中频修正，依此类推。这类似 JPEG 的逐次逼近或网页图像的渐进渲染，先出粗略版再逐步填细节。

![残差量化：原始向量在连续阶段被近似，每级量化上一级的残差](../images/residual_quantisation.svg)

- **多尺度 codebook** 通过在不同空间分辨率操作扩展此思想。不重复量化同一空间网格，而在多尺度量化：粗网格捕捉全局结构，细网格捕捉局部细节。这与第 8 章目标检测的特征金字塔思想相关，不同尺度特征捕捉不同细节层次。

- **Product quantisation**（乘积量化）是相关技术，把 $d$ 维 latent 向量拆为 $M$ 个 $d/M$ 维子向量，各用独立 codebook 量化。这给出有效 $K^M$ 词表而只存 $M \times K$ 个向量。乘积量化广泛用于近似最近邻搜索（第 13 章）并已被适配于 image tokenisation。

- Mentzer 等（2023）提出的 **Finite scalar quantisation**（FSQ）采取截然不同方法：不学 codebook，而简单地把 latent 向量每维舍入到固定整数级之一（如 $\{-2, -1, 0, 1, 2\}$）。每维 $L$ 级、$d$ 维给出隐式 codebook 大小 $L^d$。FSQ 完全避免 codebook collapse，因为没有可学习 codebook 向量，只有可学习 encoder 输出被确定性舍入。straight-through estimator 处理舍入的不可微性。

## 实践中的图像 tokenizer

- 从 VQ-VAE 到 VQ-GAN 再到残差量化的进展催生了一族用于 SOTA 生成模型的实用图像 tokenizer。

### DALL-E Tokenizer (dVAE)

- 原版 **DALL-E**（Ramesh 等，2021）用离散 VAE（dVAE）把 256x256 图像 tokenize 为来自 8192 大小 codebook 的 32x32 token 网格。dVAE 用 Gumbel-Softmax 松弛取代硬 $\arg\min$ 量化，使训练时前向可微。推理时用 $\arg\max$ 产生硬 token 分配。dVAE 用重建 loss、对均匀先验的 KL 散度与 Gumbel-Softmax 的学习温度调度组合训练。DALL-E 再训练 120 亿参数自回归 transformer 建模 256 个文本 token 与 1024 个图像 token（32x32）的联合分布。

### LlamaGen

- **LlamaGen**（Sun 等，2024）表明，只要有好的图像 tokenizer，可把标准 Llama 式语言模型架构（第 7 章）复用于自回归图像生成。LlamaGen 用改进的 VQ-GAN tokenizer 配大 codebook（16,384 项），训练一个朴素自回归 transformer（除 tokenizer 外无图像特定改动）按光栅扫描顺序从左到右预测图像 token。关键洞见是：一旦图像被 tokenize 为离散序列，对语言有效的同一 next-token-prediction 范式对图像也有效，验证了 tokenisation 真正桥接模态鸿沟。

### Cosmos Tokenizer

- **Cosmos tokenizer**（NVIDIA，2024）为图像与视频在统一框架中设计。它用因果 3D 架构把图像当作单帧视频处理，使同一 tokenizer 能处理两种模态。Cosmos 支持连续与离散 tokenisation 模式：连续模式输出实值 latent 向量（用于 diffusion 模型后端），离散模式应用 FSQ 产生整数 token（用于自回归模型后端）。encoder 用因果 3D 卷积，使每帧 token 只依赖当前与之前帧，支持流式视频 tokenisation。

## 视频 tokenisation

- 视频在图像的空间维度上加第三轴——时间。视频是帧序列，通常 24-30 fps，相邻帧高度冗余，因为视觉世界在 33 毫秒内不会剧变。视频 tokenisation 利用此时间冗余实现比独立 tokenize 每帧高得多的压缩。

- 把视频压缩想象成翻页书。若每页从零画，需数千张精细图。但多数页与邻居几乎相同，故可每 10 页画一张完整"关键帧"，其间只记小变化。视频 tokenizer 自动学会这一招。

### 3D VQ-VAE

- VQ-VAE 到视频最直接的扩展是 **3D VQ-VAE**，把 encoder 与 decoder 的 2D 卷积换成同时操作空间与时间维的 3D 卷积。若 encoder 空间下采样 $f_s$、时间下采样 $f_t$，则 $T \times H \times W$ 视频片段变为 $(T/f_t) \times (H/f_s) \times (W/f_s)$ token 网格。

- 例如 $f_s = 16$、$f_t = 4$ 时，16 帧 256x256 视频片段变为 $4 \times 16 \times 16 = 1024$ token 序列。这对 transformer 自回归建模足够紧凑，而原始像素数会是 $16 \times 256 \times 256 \times 3 \approx 3.1$ 百万值。

- 3D 卷积联合学习空间与时间特征。早期层捕捉局部运动（帧间移动的边缘），深层捕捉更高级动态（对象出现、消失或变形）。这与第 8 章卷积网络的层次特征提取原则相同，沿时间轴扩展。

![用于视频的 3D VQ-VAE：短视频片段由 3D 卷积编码为时空 latent 向量网格、量化、再解码为帧](../images/video_3d_vqvae.svg)

### 因果视频 tokenizer

- 标准 3D 卷积看过去、当前与未来帧，意味着需完整视频片段才能 tokenize。**因果视频 tokenizer** 约束时间卷积使每个输出只依赖当前与之前帧，从不依赖未来帧。这类似自回归 transformer（第 7 章）的 causal masking：信息只向前流，从不向后。

- 因果 tokenisation 对两个用例必不可少。其一，**streaming**：可随帧到达实时 tokenize 视频，无需缓冲未来帧。其二，**autoregressive generation**：transformer 逐帧生成视频时，帧 $t$ 的 token 必须在不知帧 $t+1$ 时可计算，因为帧 $t+1$ 尚未生成。

- 因果约束通过对时间卷积非对称 padding 实现：时间大小 $k$ 的 kernel 在过去侧 pad $k-1$ 个零、未来侧 pad 零，确保时刻 $t$ 的输出只依赖时刻 $t-k+1, \ldots, t$ 的输入。

- 因果视频 tokenizer 的一个优雅特性是它可无特殊处理地 tokenize 单张图像（一帧"视频"）。首帧无过去上下文，故其 token 仅从该帧计算。这种**图文统一**意味着单一 tokenizer 服务两种模态，简化架构并使模型能用同一 decoder 生成图像与视频。

### 时间压缩策略

- 不同应用需不同时间压缩比。动作识别（细微运动重要）需温和压缩（$f_t = 2$）保留时间细节。长视频生成（存数千帧不可行）需激进压缩（$f_t = 8$ 或更高）。

- 一些 tokenizer 用 **factorised compression**（分解压缩）：空间与时间压缩分阶段进行。先由 2D encoder 独立压缩每帧，产生每帧 latent 网格；再由 1D 时间 encoder 跨时间维压缩。该分解比完整 3D 卷积计算更便宜，并允许空间与时间用不同压缩比。权衡是无法像联合 3D 编码那样高效捕捉时空模式（如斜向移动的球）。

- **Temporal interpolation token** 是近期创新：tokenizer 仅完整编码关键帧，中间帧表示为描述如何在关键帧间形变的轻量插值码。这镜像经典视频压缩（H.264/HEVC 的 I 帧与 P 帧），但在学习的 latent 空间中。

## 连续 vs 离散 token

- 并非每个下游模型都需要离散 token。**Diffusion model**（第 10 章第 04 篇文件）原生处理连续值——它们迭代去噪 Gaussian 样本，loss 函数（denoising score matching）定义在连续空间上。对 diffusion 后端，tokenizer encoder 产生从不量化的连续 latent 向量。**Latent diffusion model**（Stable Diffusion、DALL-E 3、Flux）用类似 VQ-GAN 的 encoder-decoder 但完全跳过 codebook，在连续 latent 空间操作。

- **Autoregressive model**（GPT 式）则用对 $K$ 类的 softmax 从有限词表预测下一 token，根本性地需要离散 token。每个用自回归 transformer 的图像生成系统（DALL-E、Parti、LlamaGen、Chameleon）都依赖离散 tokenizer。

- 因此连续与离散 token 的选择由生成后端驱动：

- 用**离散 token** 当：模型自回归（next-token prediction 配 cross-entropy loss）、想与文本 token 共享词表以构建统一多模态模型、或需精确 token 级控制（如检索或 token 替换编辑）。

- 用**连续 token** 当：模型是 diffusion 或 flow-matching 模型、任务需极高保真重建（连续 latent 完全避免量化误差）、或想用对实值向量操作的回归 loss。

- 一些近期架构支持两种模式。例如 Cosmos tokenizer 可从同一 encoder 输出连续 latent（用于其 diffusion 模式）或 FSQ 离散 token（用于其自回归模式），带可开关的轻量量化 head。

- **Soft quantisation** 是中间方案：不做硬 $\arg\min$ 分配，而计算最近 $k$ 个 codebook 项的加权平均，权重由对负距离的 softmax 给出。这比硬量化保留更多信息同时仍近似离散。一些系统训练时用软量化、推理时用硬量化。

## 应用

### 自回归图像生成

- 一旦图像是离散 token 序列，就能训练标准自回归 transformer 建模它们。图像 token 被展平为一维序列（通常光栅扫描：从左到右、从上到下），transformer 用标准 cross-entropy loss 学习 $p(\text{token}_i | \text{token}_1, \ldots, \text{token}_{i-1})$。生成时逐个采样 token，完成的网格经 tokenizer 的 decoder 产生像素。

- 条件于文本很直接：把文本 token 前置于图像 token 序列，使模型学习 $p(\text{image tokens} | \text{text tokens})$。这正是 DALL-E、Parti、LlamaGen 做 text-to-image 生成的方式。文本与图像 token 共享同一 transformer、同一 attention 机制，常共享同一 embedding 表（文本与图像 token 占不同索引范围）。

- 光栅扫描顺序引入人为不对称：图像左上先生成，无右下上下文。多项工作解决此问题。**Masked image modelling**（MaskGIT）训练双向 transformer 同时生成所有 token 但置信度不同，迭代 unmask 最置信 token。**多尺度生成**先生成粗 token（捕捉全局构图）再用残差 token 精化。这些方法以牺牲纯从左到右生成的简洁换取更好的全局连贯。

### 统一 vision-language token

- image tokenisation 最深层的动机是**统一**：把视觉与语言放入同一表示格式，使单一模型架构处理两者。如第 7 章所讨论，语言模型是极强的序列到序列机器。把图像表示为 token 序列，我们免费继承语言建模的全部基础设施——预训练配方、缩放定律、RLHF、上下文长度扩展。

- **Chameleon**（Meta，2024）是突出示例：它用 8192 项的 VQ-GAN tokenizer 把图像转为 token，与文本 token 在约 65,000 项的单一词表（文本 + 图像）中交错。标准 transformer 在混合文本-图像序列上训练，使其能用同一前向生成图像给文本、文本给图像或交错的文本-图像内容。

- **Gemini**（Google，2024）在巨大规模上采取类似方法，在单一 transformer 内原生理解与生成图像、音频与文本，模态特定 tokenizer 喂入共享序列。

- 统一模型的关键工程挑战是 **vocabulary balance**（词表平衡）：若 65,000 词表项中 8192 个是图像 token，模型可能对视觉欠分配容量。解决方案包括每模态独立 embedding 层（仅在 attention 级共享）、模态特定 loss 加权与预训练时谨慎的数据混合比。

1. 在 JAX 中实现最小 VQ 层：给定一批 encoder 输出向量，执行最近邻 codebook 查找并计算 VQ-VAE loss（reconstruction + codebook + commitment）。将 codebook 利用率可视化为直方图。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

# --- Minimal VQ layer ---
key = jax.random.PRNGKey(42)
d = 8          # embedding dimension
K = 64         # codebook size
n_vectors = 256  # batch of encoder outputs

# Random encoder outputs and codebook
k1, k2 = jax.random.split(key)
z_e = jax.random.normal(k1, (n_vectors, d))       # encoder outputs
codebook = jax.random.normal(k2, (K, d)) * 0.1     # codebook (small init)

# Nearest-neighbour lookup: find closest codebook entry for each z_e
# distances[i, k] = ||z_e[i] - codebook[k]||^2
distances = (
    jnp.sum(z_e ** 2, axis=1, keepdims=True)
    - 2 * z_e @ codebook.T
    + jnp.sum(codebook ** 2, axis=1, keepdims=True).T
)
indices = jnp.argmin(distances, axis=1)       # token indices
z_q = codebook[indices]                        # quantised vectors

# VQ-VAE loss terms
beta = 0.25
loss_codebook = jnp.mean((jax.lax.stop_gradient(z_e) - z_q) ** 2)
loss_commit   = jnp.mean((z_e - jax.lax.stop_gradient(z_q)) ** 2)
loss_total    = loss_codebook + beta * loss_commit
print(f"Codebook loss: {loss_codebook:.4f}, Commitment loss: {loss_commit:.4f}")

# Codebook utilisation
unique, counts = jnp.unique(indices, return_counts=True, size=K, fill_value=-1)
plt.figure(figsize=(10, 4))
plt.bar(range(K), counts, color='#3498db', alpha=0.8)
plt.xlabel('Codebook Index'); plt.ylabel('Assignment Count')
plt.title(f'Codebook Utilisation ({jnp.sum(counts > 0)}/{K} entries used)')
plt.grid(True, alpha=0.3); plt.tight_layout(); plt.show()
# Try: increase K to 512 and observe collapse. Then add codebook reset logic.
```

2. 构建一个玩具二维向量量化器，学习铺贴二维分布。生成随机二维点，通过 EMA 更新学习 codebook，并可视化 Voronoi 区域。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

# Generate 2D data from a mixture of Gaussians
key = jax.random.PRNGKey(0)
n_points = 2000
K = 16  # codebook entries
gamma = 0.99  # EMA decay

# Four clusters
keys = jax.random.split(key, 5)
centres = jnp.array([[2, 2], [-2, 2], [-2, -2], [2, -2]], dtype=jnp.float32)
data = jnp.concatenate([
    jax.random.normal(keys[i], (n_points // 4, 2)) * 0.5 + centres[i]
    for i in range(4)
])

# Initialise codebook from random data points
idx = jax.random.choice(keys[4], n_points, (K,), replace=False)
codebook = data[idx]
ema_count = jnp.ones(K)
ema_sum = codebook.copy()

# Run EMA-based codebook learning for several epochs
for epoch in range(30):
    # Assign each point to nearest codebook entry
    dists = jnp.sum((data[:, None, :] - codebook[None, :, :]) ** 2, axis=2)
    assignments = jnp.argmin(dists, axis=1)
    # EMA update
    for k in range(K):
        mask = (assignments == k)
        count_k = jnp.sum(mask)
        ema_count = ema_count.at[k].set(gamma * ema_count[k] + (1 - gamma) * count_k)
        if count_k > 0:
            sum_k = jnp.sum(data[mask], axis=0)
            ema_sum = ema_sum.at[k].set(gamma * ema_sum[k] + (1 - gamma) * sum_k)
    codebook = ema_sum / ema_count[:, None]

# Visualise assignments and codebook
fig, ax = plt.subplots(1, 1, figsize=(8, 8))
colors = plt.cm.tab20(jnp.linspace(0, 1, K))
for k in range(K):
    mask = assignments == k
    ax.scatter(data[mask, 0], data[mask, 1], c=[colors[k]], s=5, alpha=0.3)
ax.scatter(codebook[:, 0], codebook[:, 1], c='black', s=120, marker='X',
           edgecolors='white', linewidths=1.5, zorder=10, label='Codebook')
ax.set_title(f'Learned VQ Codebook ({K} entries) on 2D Data')
ax.legend(); ax.set_aspect('equal'); ax.grid(True, alpha=0.3)
plt.tight_layout(); plt.show()
# Try: increase K to 64 and observe finer tiling. Reduce gamma and see instability.
```

3. 演示残差量化：用 $T$ 个连续量化阶段编码一批向量，测量每级如何降低重建误差。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

key = jax.random.PRNGKey(7)
d = 16         # embedding dimension
K = 32         # codebook size per level
T = 8          # number of residual levels
n_vectors = 512

# Random data to quantise
k1, *cb_keys = jax.random.split(key, T + 1)
z = jax.random.normal(k1, (n_vectors, d))

# Independent random codebooks for each level
codebooks = [jax.random.normal(cb_keys[t], (K, d)) * (0.5 ** t)
             for t in range(T)]

# Residual quantisation loop
residual = z.copy()
z_hat = jnp.zeros_like(z)
errors = []

for t in range(T):
    cb = codebooks[t]
    dists = (jnp.sum(residual ** 2, axis=1, keepdims=True)
             - 2 * residual @ cb.T
             + jnp.sum(cb ** 2, axis=1, keepdims=True).T)
    indices = jnp.argmin(dists, axis=1)
    z_q_t = cb[indices]
    z_hat = z_hat + z_q_t
    residual = residual - z_q_t
    mse = jnp.mean(jnp.sum((z - z_hat) ** 2, axis=1))
    errors.append(float(mse))
    print(f"Level {t+1}: MSE = {mse:.4f}")

plt.figure(figsize=(8, 5))
plt.plot(range(1, T + 1), errors, 'o-', color='#e74c3c', linewidth=2, markersize=8)
plt.xlabel('Residual Quantisation Level')
plt.ylabel('Reconstruction MSE')
plt.title('Error Reduction with Residual Quantisation')
plt.xticks(range(1, T + 1)); plt.grid(True, alpha=0.3)
plt.tight_layout(); plt.show()
# Try: use a single codebook of size K*T and compare with RQ. Which wins?
```

4. 模拟一个简单的一维"视频 tokenizer"：生成一维信号序列（模拟视频帧），应用因果时间压缩，并在重建质量上与非因果压缩比较。
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

key = jax.random.PRNGKey(99)
n_frames = 16
frame_len = 64

# Generate a "video": a slowly moving Gaussian bump across frames
x_axis = jnp.linspace(-3, 3, frame_len)
frames = jnp.stack([
    jnp.exp(-0.5 * (x_axis - (-2 + 4 * t / n_frames)) ** 2)
    for t in range(n_frames)
])  # shape: (n_frames, frame_len)

# Causal temporal compression: each frame's code depends only on past frames
# Simple approach: average current frame with exponential decay of past
alpha_causal = 0.6
causal_codes = jnp.zeros_like(frames)
causal_codes = causal_codes.at[0].set(frames[0])
for t in range(1, n_frames):
    causal_codes = causal_codes.at[t].set(
        alpha_causal * frames[t] + (1 - alpha_causal) * causal_codes[t - 1]
    )

# Non-causal: average with both past and future (bilateral smoothing)
kernel = jnp.array([0.2, 0.6, 0.2])  # past, current, future
padded = jnp.concatenate([frames[:1], frames, frames[-1:]], axis=0)
noncausal_codes = jnp.stack([
    kernel[0] * padded[t] + kernel[1] * padded[t+1] + kernel[2] * padded[t+2]
    for t in range(n_frames)
])

# Reconstruction error
mse_causal = jnp.mean((frames - causal_codes) ** 2)
mse_noncausal = jnp.mean((frames - noncausal_codes) ** 2)
print(f"Causal MSE: {mse_causal:.6f}, Non-causal MSE: {mse_noncausal:.6f}")

fig, axes = plt.subplots(1, 3, figsize=(15, 5))
for ax, data, title in zip(axes,
    [frames, causal_codes, noncausal_codes],
    ['Original Frames', f'Causal (MSE={mse_causal:.5f})',
     f'Non-causal (MSE={mse_noncausal:.5f})']):
    ax.imshow(data, aspect='auto', cmap='viridis', origin='lower')
    ax.set_xlabel('Spatial Position'); ax.set_ylabel('Frame Index')
    ax.set_title(title)
plt.tight_layout(); plt.show()
# Try: vary alpha_causal and the kernel weights. What happens with alpha=1.0?
```
