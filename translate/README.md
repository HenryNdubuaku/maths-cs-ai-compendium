# Переводчик сборника на русский язык

`translate.py` переводит все Markdown-файлы сборника на русский через LLM, аккуратно
сохраняя математику (LaTeX), код, картинки и структуру Markdown.

## Почему смысл не «утекает»

- **Математику и код не трогаем.** Формулы `$...$`, `$$...$$`, блоки кода и инлайн-код
  модель видит в контексте (чтобы правильно перевести окружающий текст), но обязана
  сохранить их посимвольно. После перевода скрипт **сам сверяет** множество формул, кода,
  путей и ссылок в оригинале и переводе; при расхождении — повтор с корректирующей
  подсказкой, а неисправленное попадает в `translation_report.txt` для ручной проверки.
- **Глоссарий.** `glossary.md` (≈150 терминов) передаётся модели, поэтому «matrix» везде
  «матрица», «gradient descent» везде «градиентный спуск» и т.д. Правьте под себя.
- **Аккуратная нарезка.** Файлы режутся на чанки по границам абзацев и **никогда** внутри
  блока кода или display-формулы.
- **Низкая температура (0.2)** — меньше «отсебятины».

## Установка

```bash
pip install requests            # единственная зависимость
```

## Быстрый старт (OpenRouter, по умолчанию)

```bash
export OPENROUTER_API_KEY=sk-or-...

# 1. Посмотреть план без изменений:
python3 translate/translate.py --dry-run

# 2. Прогнать на одном файле для проверки качества:
python3 translate/translate.py --limit 1

# 3. Перевести весь сборник (на месте):
python3 translate/translate.py
```

> ⚠️ Перевод идёт **на месте** и перезаписывает английские файлы. Скрипт откажется
> работать при незакоммиченных изменениях в git — **сначала закоммитьте** (оригинал
> останется в истории git). Либо переводите в отдельный каталог: `--output-dir ru/`.

## Локальный перевод через LM Studio

1. В LM Studio загрузите модель и запустите сервер (вкладка **Developer → Start Server**,
   по умолчанию `http://localhost:1234`).
2. Запустите:

```bash
python3 translate/translate.py --provider lmstudio --model "имя-загруженной-модели"
```

Для качественного перевода на русский локально берите крупную многоязычную модель
(напр. Qwen2.5-72B-Instruct, Gemma-2-27B, Llama-3.3-70B) — маленькие модели заметно
портят термины и формулы.

## Выбор модели (OpenRouter)

По умолчанию — `google/gemini-2.5-pro`: сильный технический перевод на русский, длинный
контекст, стабильно держит LaTeX. Альтернативы через `--model`:

| Модель | Когда брать |
| --- | --- |
| `google/gemini-2.5-pro` | **по умолчанию** — качество/контекст |
| `google/gemini-2.5-flash` | дёшево и быстро, качество чуть ниже (хорошо для черновика) |
| `anthropic/claude-sonnet-4.5` | максимально аккуратная терминология и тон |
| `openai/gpt-5` | сильный на технической документации |
| `deepseek/deepseek-chat` | самый бюджетный для массового объёма |

Актуальные цены/модели — на openrouter.ai/models.

## Полезные опции

| Опция | Значение |
| --- | --- |
| `--dry-run` | показать план, ничего не менять |
| `--limit N` | перевести не более N файлов (для теста) |
| `--force` | перевести заново даже уже готовые файлы |
| `--output-dir ru/` | писать перевод в отдельный каталог, не трогая оригинал |
| `--files "chapter 01*/*.md"` | ограничить набор файлов glob-шаблоном(ами) |
| `--concurrency 8` | больше параллельных запросов (быстрее, но упирается в rate limit) |
| `--max-chars 6000` | размер чанка |
| `--allow-dirty` | не проверять чистоту git |
| `--glossary path` | другой файл глоссария |

## Резюмируемость

Прогресс пишется в `.translate_state.json` (по хэшу результата). Повторный запуск
пропускает уже переведённые и неизменившиеся файлы — можно прерывать (`Ctrl-C`) и
продолжать. Чтобы перевести файл заново, отредактируйте его (хэш изменится) или используйте
`--force`.

## Что переводится

По умолчанию: `chapter */*.md`, `index.md`, `README.md`, `llms.txt`. Навигация в
`mkdocs.yml` (заголовки разделов) **не** переводится этим скриптом — при желании
переведите её вручную или отдельным прогоном.
