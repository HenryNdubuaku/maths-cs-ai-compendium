# 离散数学

*离散数学研究可数的、分离的结构，是计算所依赖的基础。本文件涵盖命题逻辑与谓词逻辑、证明技术、集合、关系、函数、图论基础以及递推关系*

- 在前面的章节中，我们处理的是连续数学：微积分（第 3 章）、概率分布（第 5 章）以及针对实值参数的优化（第 6 章）。但计算机本质上是**离散的**机器。它们存储比特（0 或 1），处理整数，遵循分支逻辑，并在有限的数据结构上运算。**离散数学**提供了对这类结构进行推理的形式化语言。

- 本章的一切都建立在离散数学之上：处理器的逻辑门就是布尔代数，调度算法需要正确性证明，内存管理使用集合运算，而算法分析需要递推关系。

## 命题逻辑

- **命题逻辑**是关于真假陈述的代数。一个**命题**是一条要么为真（T）要么为假（F）的陈述，二者不能兼有。“正在下雨”是一个命题。“现在几点了？”则不是（它是一个问题，而不是有真值的陈述）。

- 命题可以用**逻辑联结词**组合：

    - **AND**（合取，$p \wedge q$）：仅当 $p$ 和 $q$ 都为真时为真。
    - **OR**（析取，$p \vee q$）：当 $p$ 或 $q$ 中至少一个为真时为真。
    - **NOT**（否定，$\neg p$）：翻转真值。
    - **IMPLIES**（蕴涵，$p \to q$）：仅当 $p$ 为真而 $q$ 为假时为假。“如果下雨，地面就会湿”只有在下雨且地面是干的时才被违反。
    - **IFF**（双条件，$p \leftrightarrow q$）：当两者真值相同时为真。

- **真值表**穷举所有可能的输入组合及其输出结果。对于 $n$ 个命题，真值表有 $2^n$ 行。我们就是这样验证逻辑等价性的：

| $p$ | $q$ | $p \wedge q$ | $p \vee q$ | $p \to q$ |
|-----|-----|--------------|------------|-----------|
| T | T | T | T | T |
| T | F | F | T | F |
| F | T | F | T | T |
| F | F | F | F | T |

- 蕴涵行中 $p$ 为假的情况值得关注：$F \to q$ 永远为真，无论 $q$ 如何。这就是**空真**。“如果猪会飞，那么我就是英格兰国王”在逻辑上为真，因为前提为假。这看似违反直觉，但对数学推理至关重要。

- **逻辑等价**是对所有真值都成立的恒等式：

    - **德摩根律**：$\neg(p \wedge q) \equiv \neg p \vee \neg q$ 和 $\neg(p \vee q) \equiv \neg p \wedge \neg q$。要否定一个 AND，就对各部分分别取否定再改用 OR（反之亦然）。这些规律直接出现在编程中：`!(a && b)` 等价于 `(!a || !b)`。

    - **逆否命题**：$p \to q \equiv \neg q \to \neg p$。“如果下雨，地面就会湿”等价于“如果地面没湿，就没下雨”。这是一种强有力的证明技巧。

    - **双重否定**：$\neg(\neg p) \equiv p$。

    - **分配律**：$p \wedge (q \vee r) \equiv (p \wedge q) \vee (p \wedge r)$。

- 永远为真（对所有真值指派都成立）的公式是**重言式**。永远为假的是**矛盾式**。有时真有时假的是**偶然式**。例如，$p \vee \neg p$ 是重言式，$p \wedge \neg p$ 是矛盾式。

## 谓词逻辑与量词

- 命题逻辑无法表达关于集合中*所有*或*某些*元素的陈述。“每一个大于 2 的素数都是奇数”需要**谓词逻辑**，它通过变量、谓词和量词扩展了命题逻辑。

- **谓词**是依赖于变量的陈述：$P(x)$ = “$x$ 是偶数”。当 $x$ 取定某个值时它就成为一个命题：$P(4)$ 为真，$P(7)$ 为假。

- **量词**表示范围：

    - **全称量词**（$\forall$）：“对所有”。$\forall x \, P(x)$ 表示“对论域中的每个 $x$，$P(x)$ 都为真”。
    - **存在量词**（$\exists$）：“存在”。$\exists x \, P(x)$ 表示“存在至少一个 $x$ 使 $P(x)$ 为真”。

- 否定量词会翻转它们：$\neg(\forall x \, P(x)) \equiv \exists x \, \neg P(x)$。“并非所有人都通过了”意味着“有人没通过”。而 $\neg(\exists x \, P(x)) \equiv \forall x \, \neg P(x)$。“不存在完美的算法”意味着“每个算法都有缺陷”。

- 嵌套量词表达复杂的关系。$\forall x \, \exists y \, (y > x)$ 表示“对每个数，都存在一个更大的数”（对整数成立）。顺序很重要：$\exists y \, \forall x \, (y > x)$ 表示“存在一个数比其他所有数都大”（对整数不成立）。

- 谓词逻辑是形式化规约的语言。当我们说一个算法是“正确的”，意思是 $\forall \text{inputs} \, x, \, \text{output}(x) = \text{desired}(x)$。当我们说它“会终止”，意思是 $\forall x \, \exists t \, \text{halts}(x, t)$。

## 证明技术

- **证明**是一段逻辑论证，使某陈述的真实性无可置疑。不同于经验证据（只表明对测试过的案例成立），证明保证对所有案例都成立。这是计算机科学中正确性的标准。

- **直接证明**：假设前提，通过逻辑步骤推导出结论。要证明“如果 $n$ 是偶数，则 $n^2$ 也是偶数”：假设 $n = 2k$（$k$ 为某整数），则 $n^2 = 4k^2 = 2(2k^2)$，是偶数。

- **反证法**：假设陈述为假，导出矛盾。要证明 $\sqrt{2}$ 是无理数：假设 $\sqrt{2} = a/b$（已化简）。则 $2 = a^2/b^2$，所以 $a^2 = 2b^2$，意味着 $a^2$ 是偶数，所以 $a$ 是偶数，设 $a = 2c$。则 $4c^2 = 2b^2$，所以 $b^2 = 2c^2$，意味着 $b$ 也是偶数。但我们说 $a/b$ 已化简——矛盾。

- **数学归纳法**：通过以下两步证明一个对所有自然数成立的陈述：(1) **基础情形**成立（通常是 $n = 0$ 或 $n = 1$），(2) **归纳步骤**：如果陈述对 $n = k$ 成立（归纳假设），那么对 $n = k + 1$ 也成立。

- 例如，证明 $\sum_{i=1}^{n} i = \frac{n(n+1)}{2}$：
    - 基础情形：$n = 1$：$1 = \frac{1 \cdot 2}{2} = 1$。成立。
    - 归纳步骤：假设 $\sum_{i=1}^{k} i = \frac{k(k+1)}{2}$。则 $\sum_{i=1}^{k+1} i = \frac{k(k+1)}{2} + (k+1) = \frac{k(k+1) + 2(k+1)}{2} = \frac{(k+1)(k+2)}{2}$。这正是 $n = k+1$ 时的公式。证毕。

- 归纳法是证明递归算法和数据结构性质的主力工具。每个递归算法都隐含一个归纳正确性证明：基础情形是终止条件，归纳步骤是递归调用。

- **强归纳法**假设陈述对到 $k$ 为止的所有值（不只是 $k$）成立，然后证明对 $k + 1$ 成立。当递归依赖于不止前一个值时，强归纳法很有用。

- **鸽巢原理**：如果把 $n+1$ 个物体放入 $n$ 个盒子，至少有一个盒子包含两个物体。简单却出人意料地强大。它证明了任意 13 人的群体中，至少有两人共享同一个出生月份。在网络中，它证明了当项目数多于桶数时哈希冲突不可避免。

## 集合

- **集合**是不同元素的无序集合。集合是数学中最原始的数据结构，从类型系统到数据库查询，一切都以它为基础。

- **集合运算**（联系第 5 章，我们在概率中用过这些）：

    - **并** $A \cup B$：属于 $A$ 或 $B$ 或两者的元素。
    - **交** $A \cap B$：同时属于 $A$ 和 $B$ 的元素。
    - **补** $\bar{A}$：不属于 $A$ 的元素（相对于全集而言）。
    - **差** $A \setminus B$：属于 $A$ 但不属于 $B$ 的元素。
    - **笛卡尔积** $A \times B$：所有有序对 $(a, b)$，其中 $a \in A, b \in B$。

- **幂集** $\mathcal{P}(A)$ 是 $A$ 的所有子集构成的集合。若 $|A| = n$，则 $|\mathcal{P}(A)| = 2^n$。对于 $A = \{1, 2\}$：$\mathcal{P}(A) = \{\emptyset, \{1\}, \{2\}, \{1, 2\}\}$。

- **基数**衡量集合的大小。有限集有整数基数。无限集大小不同：自然数 $\mathbb{N}$ 和有理数 $\mathbb{Q}$ 是**可数无限**（可列出），而实数 $\mathbb{R}$ 是**不可数无限**（无法列出，由康托尔对角线论证证明）。这种区别在可计算性理论中很重要：函数有不可数多个，但程序只有可数多个，所以大多数函数不可计算。

## 关系

- 集合 $A$ 上的**关系** $R$ 是 $A \times A$ 的一个子集：一组有序对，指明哪些元素相关。例如，整数上的 $\leq$ 就是集合 $\{(a, b) : a \leq b\}$。

- 关系的重要性质：

    - **自反**：每个元素与自身相关。对所有 $a$，$a R a$。例如 $\leq$（每个数都 $\leq$ 自身）。
    - **对称**：如果 $a R b$ 则 $b R a$。例如“是…的兄弟姐妹”。
    - **反对称**：如果 $a R b$ 且 $b R a$ 则 $a = b$。例如 $\leq$。
    - **传递**：如果 $a R b$ 且 $b R c$ 则 $a R c$。例如 $<$、$\leq$、“是…的后裔”。

- **等价关系**是自反、对称且传递的。它把集合划分为**等价类**，同一类中的元素相互关联，但与其它类中的元素不相关。模运算是等价关系：$a \equiv b \pmod{n}$ 把整数划分为 $n$ 个类。编程语言中的类型等价也是等价关系。

- **偏序**是自反、反对称且传递的。它定义了一种“小于等于”结构，但可能留下某些不可比较的元素。file system 目录构成偏序（父-子关系），但兄弟目录不可比较。**全序**是每对元素都可比较的偏序（如整数上的 $\leq$）。

- 偏序在并发中至关重要：事件上的“先行发生”（happens-before）关系是偏序。不被 happens-before 排序的事件是并发的，可以以任意相对顺序执行。

## 函数

- **函数** $f: A \to B$ 将 $A$（定义域）的每个元素映射到 $B$（陪域）中唯一一个元素。函数是确定性计算的数学模型：给定输入，恰好有一个输出。

- **单射**（一一）：不同输入总给出不同输出。$f(a) = f(b) \implies a = b$。无损压缩是单射：不同输入必须压缩到不同输出（否则无法唯一解压）。

- **满射**：$B$ 的每个元素都被 $A$ 中某个元素命中。值域等于陪域。把字符串映射到 256 位哈希的哈希函数，如果字符串数少于可能的哈希数，则不是满射。

- **双射**：既单射又满射。$A$ 和 $B$ 之间的完美一一对应。双射有逆。加密必须是双射：每个明文映射到唯一密文，解密函数就是其逆。

- **复合** $(g \circ f)(x) = g(f(x))$：先施加 $f$，再施加 $g$。函数复合满足结合律（第 2 章：与矩阵乘法满足结合律同理）。软件中的流水线就是函数复合：数据流经一连串变换。

## 图论基础

- 我们在第 12 章（图神经网络）中详细介绍过图，包括邻接矩阵、图类型、拉普拉斯算子和谱理论。这里我们关注与计算机科学相关的**算法**和**结构**性质。

- **tree（树）**是无环的连通图。等价地，它有 $n$ 个节点和 $n-1$ 条边。tree 是 file system、XML/HTML 文档、决策过程和递归分解的结构。**有根树**有一个指定的根节点；其他每个节点都恰好有一个父节点。

- 图 $G$ 的**生成树**是一棵用其边的子集包含 $G$ 所有节点的 tree。**最小生成树（MST）**使总边权最小。Kruskal 算法（将边排序，贪心地添加不形成环的最轻边）和 Prim 算法（从一个起始节点生长 tree，总是添加到新节点的最轻边）都能在 $O(|E| \log |V|)$ 内找到 MST。

- **平面性**：如果一个图可以在平面上画出而无边交叉，它就是可平面的。根据**欧拉公式**，对于连通平面图：$|V| - |E| + |F| = 2$，其中 $|F|$ 是面（区域，包括外面）的数量。这蕴含平面图 $|E| \leq 3|V| - 6$，所以平面图是稀疏的。电路板布线和地图着色利用了平面性。

- **图着色**为节点分配颜色，使相邻的两个节点不共享同一种颜色。所需的最少颜色数是**色数** $\chi(G)$。**四色定理**指出对任何平面图都有 $\chi(G) \leq 4$。在计算机科学中，图着色建模寄存器分配（把变量分配到 CPU 寄存器，使同时活跃的变量得到不同寄存器）和 scheduling（把任务分配到时间段，使冲突任务不重叠）。

- **欧拉路径**访问每条边恰好一次。当且仅当图中恰好有 0 或 2 个奇数度节点时它存在。**哈密顿路径**访问每个节点恰好一次。判断哈密顿路径是否存在是 NP 完全的——计算机科学中经典的难题之一。这种对比（欧拉：多项式，哈密顿：NP 完全）说明听起来相似的问题可能有截然不同的计算 complexity。

## 递推关系

- **递推关系**定义一个序列，其中每一项依赖于前面的项。它们自然地产生于递归算法。

- 最简单的例子：$T(n) = T(n-1) + 1$，其中 $T(0) = 0$。展开：$T(n) = T(n-1) + 1 = T(n-2) + 2 = \cdots = n$。这是 $O(n)$，即简单循环的时间复杂度。

- **归并排序**给出 $T(n) = 2T(n/2) + O(n)$：把 array 对半分（两个大小为 $n/2$ 的子问题），递归地排序每一半，然后合并（$O(n)$ 的工作量）。解为 $T(n) = O(n \log n)$。

- **主定理**求解形如 $T(n) = aT(n/b) + O(n^d)$ 的递推：

    - 若 $d > \log_b a$：$T(n) = O(n^d)$（每层的工作量占主导）
    - 若 $d = \log_b a$：$T(n) = O(n^d \log n)$（各层工作量平衡）
    - 若 $d < \log_b a$：$T(n) = O(n^{\log_b a})$（子问题数量占主导）

- 对于归并排序：$a = 2, b = 2, d = 1$。由于 $d = \log_2 2 = 1$，我们处于平衡情形：$T(n) = O(n \log n)$。

- **斐波那契递推** $F(n) = F(n-1) + F(n-2)$，其中 $F(0) = 0, F(1) = 1$，其闭式解为 $F(n) = \frac{\phi^n - \psi^n}{\sqrt{5}}$，其中 $\phi = \frac{1+\sqrt{5}}{2}$（黄金比例）而 $\psi = \frac{1-\sqrt{5}}{2}$。这说明斐波那契数列以 $O(\phi^n)$ 指数增长，这也是朴素的递归斐波那契呈指数级缓慢的原因。

- **组合学**（排列、组合、二项式定理、容斥原理）在第 5 章（概率）中介绍。这些计数技巧对算法分析至关重要（有多少可能的输入？多少次比较？），但这里不再重复。

## 可计算性

- 并非所有东西都可以计算。这是整个数学中最深刻的结论之一，它设定了计算机能做什么的根本极限。

- **图灵机**是计算的抽象模型：一条无限的格子带（每个格子持有一个符号）、一个读写头，以及一组带转移规则的有限状态。尽管简单，图灵机能计算任何真实计算机能计算的东西。这就是**丘奇-图灵论题**：任何可有效计算的函数都能被图灵机计算。

- 每种编程语言（Python、C、Haskell）都是**图灵完备**的：它能模拟图灵机，因此能计算一切可计算的东西。语言之间的差异在于便利性、速度和安全性，而不在于它们根本上能计算什么。

- **停机问题**问的是：给定一个程序和一个输入，这个程序最终会停下，还是会永远运行下去？图灵证明（1936 年）没有任何算法能一般性地解决这个问题。证明用反证法：假设存在停机检测器 $H(P, x)$。构造程序 $D$，它运行 $H(D, D)$ 并做出与 $H$ 所说相反的事。如果 $H$ 说 $D$ 会停下，$D$ 就永远循环。如果 $H$ 说 $D$ 会循环，$D$ 就停下。矛盾。

- 这不是当前技术的局限；而是数学上的不可能。无论多少算力、多少聪明才智或 AI，都永远无法一般性地解决停机问题。它是计算机科学中哥德尔不完备性定理的对应物。

- 现实后果：你无法写出完美的 deadlock 检测器、完美的病毒扫描器或完美的优化编译器。这些中的每一个都要求一般性地解决停机问题（或等价的不可判定问题）。真实工具使用启发式方法和近似方法，在常见情形下有效，但无法保证对所有输入都正确。

- 一个问题是**可判定的**，如果存在一个算法总能以正确的“是/否”答案终止。如果不存在这样的算法，它就是**不可判定的**。停机问题不可判定。素性测试可判定。大多数编程语言中的类型检查是可判定的（这是设计使然）。

## 复杂度理论

- 即使在可计算的问题中，有些也比其他问题难得多。**复杂度理论**按求解问题所需的资源（时间、空间）随输入增长的规律对问题进行分类。

![P、NP 与 NP 完全：P 包含在 NP 中，NP 完全位于边界处，而 P=NP 是否成立是核心的开放问题](../images/p_np_complexity.svg)

- **P**（多项式时间）：对某个常数 $k$，可在 $O(n^k)$ 时间内求解的问题。sorting（$O(n \log n)$）、最短路径（$O(|V|^2)$）、矩阵乘法（$O(n^3)$）。这些被认为是“高效的”或“易处理的”。

- **NP**（非确定性多项式时间）：提议的解可在多项式时间内**被验证**的问题，即使**找到**解可能需要指数时间。例如，给定一个声称的哈密顿路径，你可以在 $O(n)$ 时间内通过检查每条边来验证它。但找到一条可能需要尝试指数级多的可能性。

- P 中的每个问题也属于 NP（如果你能快速求解，当然也能快速验证一个解）。核心问题在于 $P = NP$ 是否成立：每个其解可被快速验证的问题是否也能被快速求解？这是计算机科学中最重要的开放问题，设有 100 万美元的克雷千禧奖。

- 多数专家相信 $P \neq NP$，即某些问题在根本上比求解更难验证。如果 $P = NP$，密码学将崩溃（破解加密属于 NP），而优化、scheduling、药物设计将变得轻而易举。

- **NP 完全**问题是 NP 中最难的问题。一个问题如果是 NP 完全的，需满足：(1) 它属于 NP，(2) NP 中的其他所有问题都能在多项式时间内**归约**到它。如果你能高效求解任何 NP 完全问题，你就能求解所有这些问题（且 $P = NP$）。

- **归约**把一个问题变换成另一个。如果问题 A 归约到问题 B，则 B 至少和 A 一样难。Cook（1971）证明了 **SAT**（布尔可满足性：给定一个逻辑公式，是否存在一组变量指派使它为真？）是 NP 完全的。Karp（1972）通过把 SAT 归约到每个问题，又证明了 21 个经典问题是 NP 完全的。

- 著名的 NP 完全问题：
    - **旅行商问题（TSP）**：找到恰好访问所有城市一次的最短路线。
    - **图着色**：用 $k$ 种颜色给节点着色，使相邻节点不共享同一种颜色（$k \geq 3$）。
    - **子集和**：给定一组整数，是否存在一个子集之和等于目标值？
    - **布尔可满足性（SAT）**：是否存在一组真值指派使逻辑公式成立？
    - **哈密顿路径**（上文图论中提到过）。

- 在实践中遇到 NP 完全问题时，你不会对大规模输入精确求解它。相反，你会使用：**近似算法**（在保证的最优因子内找到解）、**启发式方法**（贪心、局部搜索、模拟退火）或**特殊情形求解器**（许多 NP 完全问题对受限输入是容易的）。例如，现代 SAT 求解器通过利用实际实例中的结构，尽管最坏情况是指数复杂度，但常规地求解含数百万变量的实例。

- **NP 困难**问题至少和 NP 完全问题一样难，但可能不属于 NP（它们的解甚至可能无法在多项式时间内验证）。NP 完全问题的优化版本通常是 NP 困难的：“找到最短的 TSP 路线”是 NP 困难的，而“是否存在短于 $k$ 的 TSP 路线？”是 NP 完全的。

```python
import itertools

def truth_table(n_vars, expr_fn):
    """Generate truth table for a boolean function of n_vars variables."""
    headers = [f"p{i}" for i in range(n_vars)]
    print(" | ".join(headers + ["result"]))
    print("-" * (len(headers) * 4 + 10))
    for vals in itertools.product([False, True], repeat=n_vars):
        result = expr_fn(*vals)
        row = [str(v)[0] for v in vals] + [str(result)[0]]
        print(" | ".join(f"{r:>2}" for r in row))

# De Morgan's law: NOT(p AND q) == (NOT p) OR (NOT q)
print("De Morgan's Law verification:")
truth_table(2, lambda p, q: (not (p and q)) == ((not p) or (not q)))
```

3. 用主定理求解归并排序的递推，并通过统计操作数进行实证验证。
```python
import jax.numpy as jnp

def merge_sort_ops(n):
    """Count comparisons in merge sort (recurrence: T(n) = 2T(n/2) + n)."""
    if n <= 1:
        return 0
    half = n // 2
    return merge_sort_ops(half) + merge_sort_ops(n - half) + n

for n in [8, 64, 512, 4096, 32768]:
    ops = merge_sort_ops(n)
    predicted = n * jnp.log2(n)
    ratio = ops / predicted
    print(f"n={n:5d}  ops={ops:>10d}  n log n={int(predicted):>10d}  ratio={ratio:.3f}")
```
