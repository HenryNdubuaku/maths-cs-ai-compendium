# 假设检验

*假设检验提供了一个严谨的框架，用于判断观测到的效应是真实的还是出于偶然。本文件涵盖 null 与 alternative 假设、p-value、显著性水平、t 检验、卡方检验、ANOVA 以及第一类/第二类错误——同样的逻辑也用于 A/B 测试、模型比较和研究中。*

- 统计学不只是描述数据。你常常需要做决定：新药有效吗？某个算法比另一个快吗？均值变了吗？假设检验用数据给你一个结构化的框架来回答这些问题。

- 思路很简单：先假设没有变化（"null hypothesis"），再检查数据是否极端到让这个假设难以置信。

- **null hypothesis（零假设）**（$H_0$）是默认论断，通常是"无效应"或"无差异"的陈述。例如："平均配送时间仍是 30 分钟"或"新模型不比旧模型好"。

- **alternative hypothesis（备择假设）**（$H_1$ 或 $H_a$）是你怀疑可能成立的论断："平均配送时间变了"或"新模型更好"。

- 你从不直接证明 $H_1$。相反，你问：如果 $H_0$ 为真，我看到这么极端的数据的可能性有多大？如果可能性很低，就拒绝 $H_0$ 转而接受 $H_1$。

- **检验统计量** 是一个数字，概括你的 sample 结果与 $H_0$ 预测相差多远。不同检验用不同公式，但逻辑始终一样：测量观测值与期望值之间的距离。

- **p-value** 是在假设 $H_0$ 为真时，观测到至少像你这样极端的检验统计量的概率。p-value 小意味着数据在 $H_0$ 下令人惊讶。

- **显著性水平**（$\alpha$）是你在看数据之前设定的阈值。若 $p \le \alpha$，就拒绝 $H_0$。常见选择是 $\alpha = 0.05$（5%）和 $\alpha = 0.01$（1%）。

![正态曲线，拒绝域阴影显示，检验统计量标记，p-value 面积高亮](../images/hypothesis_test.svg)

- 阴影尾部是拒绝域。如果你的检验统计量落在那里，数据在 $H_0$ 下就足够令人惊讶，可以拒绝它。绿色区域显示某个检验统计量对应的 p-value。

- 步骤如下：
    - **第 1 步**：陈述 $H_0$ 和 $H_1$
    - **第 2 步**：选择显著性水平 $\alpha$
    - **第 3 步**：收集数据并计算检验统计量
    - **第 4 步**：求 p-value（或把检验统计量与临界值比较）
    - **第 5 步**：若 $p \le \alpha$，拒绝 $H_0$；否则，未能拒绝 $H_0$

- **示例**：某工厂声称其螺栓平均长度为 10 cm。你测量 36 根螺栓，得到 sample 均值 10.3 cm。已知 population standard deviation 为 0.9 cm。是否有证据表明均值变了？

- $H_0$：$\mu = 10$，$H_1$：$\mu \neq 10$，$\alpha = 0.05$

- 检验统计量（z 检验，因为 $\sigma$ 已知且 $n$ 较大）：

$$z = \frac{\bar{x} - \mu_0}{\sigma / \sqrt{n}} = \frac{10.3 - 10}{0.9 / \sqrt{36}} = \frac{0.3}{0.15} = 2.0$$

- 对 $\alpha = 0.05$ 的双侧检验，临界值为 $\pm 1.96$。我们的 $z = 2.0 > 1.96$，所以拒绝 $H_0$。p-value 约为 0.046，小于 0.05。

- 结论：有统计学上显著的证据表明螺栓平均长度不同于 10 cm。

- **单侧检验** 检查某一特定方向的效应（$H_1$：$\mu > 10$ 或 $\mu < 10$）。整个 $\alpha$ 都放入一个尾部，使该方向上更容易拒绝 $H_0$，但无法检测相反方向的效应。

- **双侧检验** 检查任何差异（$H_1$：$\mu \neq 10$）。$\alpha$ 在两个尾部间均分（各 $\alpha/2$）。这更保守，但能捕捉任一方向的效应。

- 即使有好程序，错误仍会发生。恰好有两类错误：

![2x2 网格显示第一类和第二类错误：现实 vs 决定](../images/type_errors.svg)

- **第一类错误**（假阳性）：当 $H_0$ 实际为真时你却拒绝了它。其概率为 $\alpha$，你可以通过选择显著性水平来控制。就像没有火灾时火警响起。

- **第二类错误**（假阴性）：当 $H_0$ 实际为假时你却未能拒绝它。其概率为 $\beta$。就像真发生火灾时火警却沉默。

- **功效** 是 $1 - \beta$，即正确拒绝一个为假的 $H_0$ 的概率。功效越高意味着你更擅长检测真实效应。在以下情况功效会增加：
    - 真实效应量更大（更大的差异更易检测）
    - 样本量更大（更多数据 = 更精确）
    - 显著性水平 $\alpha$ 更大（但这会提高第一类错误风险）
    - 变异性更低（噪声更少）

- 第一类和第二类错误之间存在张力。降低 $\alpha$（对假阳性更谨慎）会增大 $\beta$（更多假阴性）。在固定样本量下，你无法同时最小化两者。

- **参数检验** 假设数据服从特定 distribution（通常为正态）。当假设成立时它们更有功效。

- **z 检验**：当 $\sigma$ 已知且 $n$ 较大（$n \ge 30$）时，把 sample 均值与已知值比较。检验统计量：

$$z = \frac{\bar{x} - \mu_0}{\sigma / \sqrt{n}}$$

- **t 检验**：类似 z 检验，但用于 $\sigma$ 未知（从 sample 估计）或 $n$ 较小的情况。使用 t 分布，它比正态分布尾部更重。更重的尾部考虑了估计 $\sigma$ 带来的额外不确定性。

$$t = \frac{\bar{x} - \mu_0}{s / \sqrt{n}}$$

- t 分布有一个参数叫 **自由度**（$df = n - 1$）。随着 $df$ 增大，t 分布趋近正态分布。

- t 检验有几种变体：
    - **单样本 t 检验**：sample 均值是否与某个特定值不同？
    - **独立双样本 t 检验**：两个独立组的均值是否不同？
    - **配对 t 检验**：两次相关测量的均值是否不同（例如同一组受试者处理前后的比较）？

- **ANOVA（方差分析）**：检验三个或更多组的均值是否相等。ANOVA 不做多次 t 检验（那样会膨胀第一类错误率），而是通过比较组间 variance 与组内 variance 做一次检验。

$$F = \frac{\text{variance between groups}}{\text{variance within groups}}$$

- 较大的 $F$ 比值意味着组间差异比仅由随机变异所预期的更大。

- **非参数检验** 对数据 distribution 作更少假设。它们基于排名而非原始值，因此对离群点和非正态性稳健。

- **卡方检验**（$\chi^2$）：检验观测频数是否匹配期望频数。用于分类数据。例如：红、蓝、绿汽车的比例是否与厂商声称的比例一致？

$$\chi^2 = \sum \frac{(O_i - E_i)^2}{E_i}$$

- **Mann-Whitney U 检验**：独立双样本 t 检验的非参数替代。它通过比较排名来检验一组是否倾向比另一组取更大值。

- **Wilcoxon 符号秩检验**：配对 t 检验的非参数替代。通过考察差异的大小和方向来比较配对观测。

- **Kruskal-Wallis 检验**：单因素 ANOVA 的非参数替代。通过比较所有组的排名来检验多个组是否来自同一 distribution。

- **拟合优度检验** 检查你的数据是否服从某个特定理论 distribution。卡方拟合优度检验把观测频数与假设 distribution 下的期望频数比较。

- **正态性检验** 专门检查数据是否服从正态分布。常见的有 Shapiro-Wilk 检验（对小样本功效强）和 Kolmogorov-Smirnov 检验（把 sample CDF 与理论 CDF 比较）。

- 在 ML 中，比较模型性能时常出现假设检验。如果模型 A 达到 92% 准确率、模型 B 达到 91%，差异是真实的还是只是噪声？对交叉验证得分做配对 t 检验可以回答这个问题。

## 编程任务（使用 CoLab 或 notebook）

1. 对正文中的螺栓工厂示例做 z 检验。计算检验统计量、p-value 并做出决定。
```python
import jax.numpy as jnp

x_bar = 10.3    # sample mean
mu_0 = 10.0     # null hypothesis value
sigma = 0.9     # known population std
n = 36           # sample size
alpha = 0.05

# Test statistic
z = (x_bar - mu_0) / (sigma / jnp.sqrt(n))
print(f"z = {z:.4f}")

# p-value (two-tailed) using the normal CDF approximation
# For |z| = 2.0, p ≈ 0.0456
from jax.scipy.stats import norm
p_value = 2 * (1 - norm.cdf(jnp.abs(z)))
print(f"p-value = {p_value:.4f}")
print(f"Reject H₀? {p_value <= alpha}")
```

2. 模拟第一类错误：当 $H_0$ 为真时，我们错误地拒绝它多频繁？运行 10,000 次实验，检查拒绝率是否匹配 $\alpha$。
```python
import jax
import jax.numpy as jnp

key = jax.random.PRNGKey(0)
mu_0 = 50.0
sigma = 10.0
n = 30
alpha = 0.05
n_experiments = 10_000

rejections = 0
for i in range(n_experiments):
    key, subkey = jax.random.split(key)
    sample = mu_0 + sigma * jax.random.normal(subkey, shape=(n,))
    z = (sample.mean() - mu_0) / (sigma / jnp.sqrt(n))
    p_value = 2 * (1 - __import__("jax").scipy.stats.norm.cdf(jnp.abs(z)))
    if p_value <= alpha:
        rejections += 1

print(f"Rejection rate: {rejections/n_experiments:.4f}")
print(f"Expected (α):   {alpha}")
```

3. 比较两组的 t 检验与 Mann-Whitney U 检验。生成一组均值略高的数据，看哪个检验能检测到差异。
```python
import jax
import jax.numpy as jnp

key = jax.random.PRNGKey(99)
k1, k2 = jax.random.split(key)

group_a = jax.random.normal(k1, shape=(25,)) * 5 + 100
group_b = jax.random.normal(k2, shape=(25,)) * 5 + 103  # slightly higher mean

# Two-sample t-test (equal variance assumed)
n_a, n_b = len(group_a), len(group_b)
mean_a, mean_b = group_a.mean(), group_b.mean()
pooled_var = ((n_a - 1) * group_a.var() + (n_b - 1) * group_b.var()) / (n_a + n_b - 2)
se = jnp.sqrt(pooled_var * (1/n_a + 1/n_b))
t_stat = (mean_a - mean_b) / se
print(f"T-test statistic: {t_stat:.4f}")

# Mann-Whitney: count how often group_a values beat group_b values
u_stat = jnp.sum(group_a[:, None] < group_b[None, :])
print(f"Mann-Whitney U:   {u_stat}")
print(f"\nGroup A mean: {mean_a:.2f}, Group B mean: {mean_b:.2f}")
```
