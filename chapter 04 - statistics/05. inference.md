# Статистический вывод

*Статистический вывод выходит за рамки принятия решений «да/нет», позволяя оценивать параметры генеральной совокупности с количественной оценкой неопределенности. В этом файле рассматриваются доверительные интервалы, точечное и интервальное оценивание, метод максимального правдоподобия, метод моментов и регрессионный анализ — мост между «сырыми» данными и предиктивными моделями в машинном обучении.*

- Проверка гипотез дает решение «да/нет»: отвергнуть или не отвергнуть нулевую гипотезу. Однако часто требуется нечто более информативное, например, диапазон правдоподобных значений для оцениваемого параметра. Именно это и обеспечивают **доверительные интервалы**.

- **Точечная оценка** — это единственное число, вычисленное по выборке, например, выборочное среднее $\bar{x}$. Это ваше наилучшее предположение о параметре генеральной совокупности, но само по себе оно не дает представления о точности оценки.

- **Доверительный интервал** дополняет эту точечную оценку диапазоном, который отражает неопределенность. Он имеет вид:

$$\text{CI} = \bar{x} \pm \text{ME}$$

- **Погрешность (margin of error, ME)** зависит от трех факторов: желаемого уровня доверия, степени изменчивости данных и размера выборки:

$$\text{ME} = z^\ast \cdot \frac{\sigma}{\sqrt{n}}$$

- Здесь $z^\ast$ — критическое значение нормального распределения, соответствующее желаемому уровню доверия. Для доверительной вероятности 95% $z^\ast = 1.96$. Для 99% $z^\ast = 2.576$.

![Доверительный интервал: точечная оценка с погрешностью по обе стороны](../images/confidence_interval.svg)

- **95%-ный доверительный интервал** означает следующее: если вы повторите эксперимент много раз и каждый раз будете строить интервал, то примерно в 95% случаев эти интервалы будут содержать истинный параметр генеральной совокупности. Это не означает, что существует 95%-ная вероятность того, что параметр находится в данном конкретном интервале. Параметр фиксирован; варьируются именно интервалы.

- **Пример**: Вы измерили рост 50 человек и получили $\bar{x} = 170$ см при $\sigma = 8$ см. Постройте 95%-ный доверительный интервал.

$$\text{ME} = 1.96 \cdot \frac{8}{\sqrt{50}} = 1.96 \cdot 1.131 = 2.22 \text{ cm}$$

$$\text{CI} = [170 - 2.22, \; 170 + 2.22] = [167.78, \; 172.22]$$

- Вы можете с 95%-ной уверенностью утверждать, что истинное среднее значение роста лежит в диапазоне от 167.78 до 172.22 см.

- Когда $\sigma$ неизвестно (обычный случай), используйте выборочное стандартное отклонение $s$ и t-распределение:

$$\text{CI} = \bar{x} \pm t^\ast_{n-1} \cdot \frac{s}{\sqrt{n}}$$

- Более широкие интервалы обеспечивают большую уверенность, но меньшую точность. Более узкие интервалы более точны, но менее надежны. Вы можете сузить интервал без потери уверенности, увеличив размер выборки.

- **Анализ мощности (power analysis)** помогает спланировать эксперимент до его проведения. Вопрос заключается в следующем: какой размер выборки необходим для обнаружения эффекта заданного размера при указанной мощности?

- Вспомните из предыдущего файла, что мощность = $1 - \beta$, вероятность правильного отклонения ложной нулевой гипотезы $H_0$. Стандартным целевым значением является мощность 80%.

- Необходимый размер выборки для z-теста при обнаружении разности $\delta$ с уровнем значимости $\alpha$ и мощностью $1-\beta$ равен:

$$n = \left(\frac{(z_{\alpha/2} + z_{\beta}) \cdot \sigma}{\delta}\right)^2$$

- Например, чтобы обнаружить разницу в 2 см в среднем росте ($\sigma = 8$) при $\alpha = 0.05$ и мощности 80% ($z_{0.025} = 1.96$, $z_{0.20} = 0.84$):

$$n = \left(\frac{(1.96 + 0.84) \cdot 8}{2}\right)^2 = \left(\frac{22.4}{2}\right)^2 = 11.2^2 \approx 126$$

- Вам потребуется около 126 человек на группу.

- Анализ мощности предотвращает две распространенные ошибки: проведение эксперимента с недостаточной выборкой для обнаружения реального эффекта (низкая мощность) или трату ресурсов на эксперимент, размер которого значительно превышает необходимый (избыточная мощность).

- **Методы Монте-Карло** используют случайную выборку для решения задач, которые трудно или невозможно решить аналитически. Основная идея: если вы не можете вычислить что-то точно, смоделируйте это много раз и используйте результаты в качестве приближения.

- Название происходит от казино Монте-Карло, что является отсылкой к роли случайности. Эти методы являются «рабочими лошадками» в машинном обучении для таких задач, как оценка интегралов, оценка неопределенности модели и аппроксимация сложных распределений.

- Общий рецепт метода Монте-Карло:
    - Определить область возможных входных данных
    - Сгенерировать случайные входные данные из этой области
    - Вычислить значение функции для каждого входного параметра
    - Агрегировать результаты (среднее, количество и т. д.)

- Классический пример — оценка числа $\pi$. Представьте квадрат со стороной 2, центрированный в начале координат, с вписанным в него кругом радиуса 1. Площадь квадрата равна 4, а площадь круга равна $\pi$.

![Квадрат с вписанным кругом, случайные точки окрашены в зависимости от того, находятся ли они внутри или снаружи](../images/monte_carlo_pi.svg)

- Равномерно распределите случайные точки внутри квадрата. Доля точек, попавших внутрь круга, аппроксимирует $\pi/4$:

$$\pi \approx 4 \times \frac{\text{points inside circle}}{\text{total points}}$$

- Точка $(x, y)$ находится внутри круга, если $x^2 + y^2 \le 1$. Чем больше точек вы используете, тем ближе ваша оценка к истинному значению $\pi$.

- В машинном обучении методы Монте-Карло применяются в:
    - **Monte Carlo dropout**: выполнение инференса несколько раз с включенным dropout для оценки неопределенности предсказания
    - **MCMC (цепи Маркова — Монте-Карло)**: выборка из сложных апостериорных распределений в байесовских моделях
    - **Методы градиента стратегии (policy gradient)**: оценка градиентов в обучении с подкреплением путем выборки траекторий

- **Факторный анализ** — это метод обнаружения скрытых (латентных) переменных, которые объясняют корреляции между наблюдаемыми переменными. Если 10 вопросов психологического опроса можно объяснить 3 базовыми чертами (экстраверсия, доброжелательность, добросовестность), факторный анализ выявляет эти черты.

- Модель предполагает, что каждая наблюдаемая переменная $x_i$ является линейной комбинацией нескольких латентных факторов $f_j$ плюс шум:

$$x_i = \lambda_{i1} f_1 + \lambda_{i2} f_2 + \ldots + \lambda_{ik} f_k + \epsilon_i$$

- Значения $\lambda$ называются **факторными нагрузками** и показывают, насколько сильно каждая наблюдаемая переменная связана с каждым фактором. Это напрямую связано с матричными разложениями из главы 2; факторный анализ тесно связан с разложением по собственным значениям и SVD.

- **Планирование эксперимента** — это искусство структурирования эксперимента таким образом, чтобы можно было сделать обоснованные выводы. Плохое планирование может сделать даже большой датасет бесполезным.

- Ключевые компоненты хорошо спланированного эксперимента:
    - **Независимая переменная (IV)**: то, чем вы манипулируете (например, дозировка препарата, архитектура модели)
    - **Зависимая переменная (DV)**: то, что вы измеряете (например, время восстановления, точность)
    - **Контрольная группа**: не получает воздействия (или получает плацебо), обеспечивая базовый уровень для сравнения
    - **Рандомизированное распределение**: участники распределяются по группам случайным образом, что уравновешивает мешающие переменные, которые вы не измеряли

- **Распространенные типы экспериментального дизайна**:
    - **Полностью рандомизированный дизайн**: испытуемые случайным образом распределяются по группам воздействия. Простой и эффективный метод, когда группы сопоставимы.
    - **Рандомизированный блочный дизайн**: испытуемые сначала группируются в блоки (например, по возрасту), а затем случайным образом распределяются по воздействиям внутри каждого блока. Это снижает вариативность, обусловленную фактором блокировки, и по своей сути близко к стратифицированной выборке.
    - **Факторный дизайн**: тестирует несколько независимых переменных одновременно. Факторный дизайн $2 \times 3$ имеет 2 уровня одной переменной и 3 уровня другой, что дает 6 комбинаций воздействий. Это позволяет выявлять **взаимодействия**, при которых эффект одной переменной зависит от уровня другой.
    - **Перекрестный дизайн (crossover design)**: каждый испытуемый последовательно получает все виды воздействия (с периодами вымывания между ними). Каждый испытуемый выступает в качестве собственного контроля, что снижает влияние индивидуальных различий.

- В экспериментах по машинному обучению эти принципы имеют решающее значение. При сравнении моделей следует контролировать случайное зерно (random seed), разбиение датасета и аппаратное обеспечение. Кросс-валидация является формой перекрестного дизайна. Абляционные исследования (ablation studies), в которых вы удаляете по одному компоненту за раз, следуют логике факторных дизайнов.

## Задачи по программированию (используйте CoLab или ноутбук)

1. Постройте 95% доверительный интервал для примера с ростом, а затем поэкспериментируйте с различными уровнями доверия и размерами выборки.
```python
import jax.numpy as jnp

x_bar = 170.0    # sample mean
sigma = 8.0      # population std (known)
n = 50           # sample size

# Critical values for common confidence levels
z_stars = {0.90: 1.645, 0.95: 1.960, 0.99: 2.576}

for conf, z_star in z_stars.items():
    me = z_star * (sigma / jnp.sqrt(n))
    lower, upper = x_bar - me, x_bar + me
    print(f"{conf*100:.0f}% CI: [{lower:.2f}, {upper:.2f}]  (ME = {me:.2f})")
```

2. Оцените $\pi$ с помощью симуляции методом Монте-Карло. Постройте график того, как оценка сходится по мере увеличения количества точек.
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

key = jax.random.PRNGKey(42)

# Generate random points in [-1, 1] x [-1, 1]
n_points = 100_000
k1, k2 = jax.random.split(key)
x = jax.random.uniform(k1, shape=(n_points,), minval=-1, maxval=1)
y = jax.random.uniform(k2, shape=(n_points,), minval=-1, maxval=1)

# Check which points are inside the unit circle
inside = (x**2 + y**2) <= 1.0
cumulative_inside = jnp.cumsum(inside)
counts = jnp.arange(1, n_points + 1)
pi_estimates = 4.0 * cumulative_inside / counts

plt.figure(figsize=(10, 4))
plt.plot(pi_estimates, color="#3498db", alpha=0.7, linewidth=0.5)
plt.axhline(y=jnp.pi, color="#e74c3c", linestyle="--", label=f"π = {jnp.pi:.6f}")
plt.xlabel("Number of points")
plt.ylabel("Estimate of π")
plt.title("Monte Carlo estimation of π")
plt.legend()
plt.ylim(2.8, 3.5)
plt.show()

print(f"Final estimate: {pi_estimates[-1]:.6f}")
print(f"True value:     {jnp.pi:.6f}")
print(f"Error:          {abs(pi_estimates[-1] - jnp.pi):.6f}")
```

3. Выполните простой анализ мощности (power analysis): для заданного размера эффекта и стандартного отклонения вычислите необходимый размер выборки и проверьте его с помощью симуляции.
```python
import jax
import jax.numpy as jnp

# Parameters
delta = 2.0      # effect size (difference in means)
sigma = 8.0      # population std
alpha = 0.05
power_target = 0.80

# Analytical sample size
z_alpha = 1.96   # two-tailed, alpha=0.05
z_beta = 0.84    # power=0.80
n_required = ((z_alpha + z_beta) * sigma / delta) ** 2
print(f"Required n per group: {n_required:.0f}")

# Verify by simulation
key = jax.random.PRNGKey(7)
n = int(jnp.ceil(n_required))
n_sims = 5000
rejections = 0

for _ in range(n_sims):
    key, k1, k2 = jax.random.split(key, 3)
    group_a = jax.random.normal(k1, shape=(n,)) * sigma + 50
    group_b = jax.random.normal(k2, shape=(n,)) * sigma + 50 + delta
    pooled_se = jnp.sqrt(2 * sigma**2 / n)
    z = (group_b.mean() - group_a.mean()) / pooled_se
    p = 2 * (1 - __import__("jax").scipy.stats.norm.cdf(jnp.abs(z)))
    if p <= alpha:
        rejections += 1

print(f"Simulated power: {rejections/n_sims:.3f}")
print(f"Target power:    {power_target:.3f}")
```

4. Визуализируйте, как ширина доверительного интервала меняется в зависимости от размера выборки. Это показывает, почему сбор большего количества данных дает более точные оценки.
```python
import jax.numpy as jnp
import matplotlib.pyplot as plt

sigma = 8.0
z_star = 1.96  # 95% confidence

sample_sizes = jnp.array([10, 20, 30, 50, 100, 200, 500, 1000], dtype=jnp.float32)
margins = z_star * sigma / jnp.sqrt(sample_sizes)

plt.figure(figsize=(8, 4))
plt.bar([str(int(n)) for n in sample_sizes], margins, color="#3498db", alpha=0.7)
plt.xlabel("Sample size")
plt.ylabel("Margin of error (cm)")
plt.title("95% CI margin of error shrinks with larger samples")
plt.show()
```
