# Детекция объектов и сегментация

*Детекция объектов локализует и классифицирует каждый объект на изображении; сегментация присваивает метку каждому пикселю. В этом файле рассматриваются IoU, mAP, якорные рамки (anchor boxes), семейство R-CNN, YOLO, SSD, пирамидальные сети признаков (Feature Pyramid Networks), семантическая/инстанс/паноптическая сегментация (U-Net, Mask R-CNN, SAM), а также метрики для их оценки.*

- Классификация изображений (файл 02) отвечает на вопрос: «что изображено на этом снимке?». Детекция объектов ставит более сложную задачу: «какие объекты присутствуют на изображении и где они находятся?».

- Сегментация идет еще дальше: «какие пиксели принадлежат какому объекту или категории?». Эти задачи образуют иерархию всё более точного пространственного понимания.

- Модель **детекции объектов** выдает набор **ограничивающих рамок** (bounding boxes), каждая из которых определяется четырьмя координатами (верхний левый угол $x, y$, ширина, высота) и меткой класса с показателем уверенности. Одно изображение может содержать ноль, один или сотни объектов различных классов.

![Входное изображение с несколькими объектами, каждый из которых заключен в цветную ограничивающую рамку с меткой класса и показателем уверенности](../images/detection_boxes.svg)

- **Пересечение над объединением (IoU)** измеряет, насколько хорошо предсказанная ограничивающая рамка совпадает с эталонной разметкой (ground truth). Это площадь пересечения, деленная на площадь объединения:

$$\text{IoU} = \frac{\text{Area of Intersection}}{\text{Area of Union}}$$

- Значение IoU, равное 1, означает идеальное совпадение; IoU, равное 0, означает полное отсутствие пересечения. Стандартный порог для «правильной» детекции составляет IoU $\geq 0.5$, хотя также используются более строгие пороги (0.75, 0.9).

- Детекция считается **истинно положительной (TP)**, если её IoU с эталонной рамкой превышает порог, а класс определен верно.

- **Ложноположительный результат (FP)** — это предсказанная рамка, которая не совпадает ни с одной эталонной рамкой.

- **Ложноотрицательный результат (FN)** — это эталонный объект, для которого не нашлось соответствующего предсказания. Это те же концепции точности (precision) и полноты (recall) из главы 06.

- **Средняя точность (AP)** суммирует качество детекции для одного класса. Для каждого класса отранжируйте все детекции по показателю уверенности, вычислите точность и полноту для каждого ранга и рассчитайте площадь под кривой «точность-полнота»:

$$\text{AP} = \int_0^1 p(r) \, dr$$

- На практике кривая интерполируется: на каждом уровне полноты точность устанавливается как максимальная точность при любой полноте $\geq r$. Это сглаживает кривую и делает её монотонно убывающей.

- **Средняя средняя точность (mAP)** усредняет AP по всем классам. «mAP@0.5» использует порог IoU 0.5. «mAP@[.5:.95]» (стандарт COCO) усредняет mAP по десяти порогам IoU от 0.5 до 0.95 с шагом 0.05, что поощряет как саму детекцию, так и точную локализацию.

- **Немаксимальное подавление (NMS)** удаляет дублирующиеся детекции. Когда модель предсказывает несколько перекрывающихся рамок для одного и того же объекта, NMS оставляет рамку с наибольшей уверенностью и удаляет все остальные, которые перекрываются с ней выше порога IoU. Это применяется для каждого класса после того, как модель выдает свои «сырые» предсказания.

- **Двухэтапные детекторы** сначала предлагают области-кандидаты, а затем классифицируют и уточняют каждое предложение.

- **R-CNN** (Girshick et al., 2014) был первым успешным детектором на основе глубокого обучения. Он использует селективный поиск (классический алгоритм) для предложения ~2000 областей-кандидатов, приводит каждую область к фиксированному размеру, независимо пропускает каждую через CNN и классифицирует с помощью SVM (глава 06). R-CNN был точным, но крайне медленным: он запускал CNN 2000 раз на изображение.

- **Fast R-CNN** (Girshick, 2015) решил проблему избыточности, запуская CNN один раз на всё изображение для создания общей карты признаков, а затем извлекая признаки для каждого предложения из этой общей карты с помощью **RoI-пулинга** (Region of Interest pooling).

- RoI-пулинг берет область карты признаков переменного размера и создает выход фиксированного размера, разделяя область на сетку и выполняя max-pooling внутри каждой ячейки. Это намного быстрее, так как дорогостоящие вычисления CNN происходят только один раз.

- **Faster R-CNN** (Ren et al., 2015) исключил внешний алгоритм предложения областей, внедрив **сеть предложения областей (RPN)** — небольшую CNN, которая работает поверх общей карты признаков и предсказывает предложения напрямую. RPN перемещает небольшое окно по карте признаков и в каждой позиции предсказывает $k$ предложений (по одному для каждой **якорной рамки**).

![Пайплайн Faster R-CNN: входное изображение → базовая CNN → общая карта признаков → RPN генерирует предложения → RoI-пулинг → головы классификации и регрессии рамок](../images/faster_rcnn.svg)

- **Якорные рамки** — это предопределенные ограничивающие рамки в каждой пространственной позиции карты признаков, охватывающие различные масштабы и соотношения сторон (например, три масштаба $\times$ три соотношения = 9 якорей на позицию). RPN предсказывает две вещи для каждого якоря: показатель объектности (объект или фон) и смещения координат, которые уточняют якорь до более точного предложения. Такая параметризация упрощает задачу регрессии: вместо предсказания абсолютных координат сеть предсказывает небольшие корректировки для подходящей начальной рамки.

- Смещения якорей параметризуются как:

$$t_x = \frac{x - x_a}{w_a}, \quad t_y = \frac{y - y_a}{h_a}, \quad t_w = \log\frac{w}{w_a}, \quad t_h = \log\frac{h}{h_a}$$

- где $(x, y, w, h)$ — центр и размер предсказанной рамки, а $(x_a, y_a, w_a, h_a)$ — якорь. Логарифмическое преобразование для ширины и высоты гарантирует, что предсказанная рамка всегда будет положительной, и делает регрессию инвариантной к масштабу.

- Faster R-CNN обучается с многозадачной функцией потерь: функция потерь классификации (кросс-энтропия из главы 05) для метки класса плюс **сглаженная L1-функция потерь** (smooth L1 loss) для регрессии рамок. Сглаженная L1 менее чувствительна к выбросам, чем L2:

```math
\text{smooth}_{L1}(x) = \begin{cases} 0.5x^2 & \text{if } |x| < 1 \\ |x| - 0.5 & \text{otherwise} \end{cases}
```

- **Пирамидальные сети признаков (FPN)** (Lin et al., 2017) решают проблему мультимасштабности, выстраивая нисходящий путь с латеральными связями, который объединяет высокоуровневую семантику с низкоуровневыми пространственными деталями. Базовая сеть создает карты признаков на нескольких масштабах (каждый слой пулинга уменьшает разрешение вдвое). FPN добавляет нисходящий путь, где каждый уровень получает апсемплинговые признаки с уровня выше и объединяет их с соответствующим восходящим уровнем через латеральные свёртки 1x1. Результатом является пирамида карт признаков, каждая из которых обладает как сильной семантикой, так и хорошим пространственным разрешением.

- Малые объекты обнаруживаются на уровнях пирамиды с более высоким разрешением; крупные объекты — на уровнях с более низким разрешением. FPN в настоящее время является стандартным компонентом большинства современных архитектур для детекции.

- **Одностадийные детекторы** полностью пропускают этап формирования предложений (proposal), предсказывая метки классов и ограничивающие рамки (bounding boxes) за один проход. Это быстрее, но исторически было менее точно, чем у двухстадийных детекторов, пока фокусная функция потерь (focal loss) не сократила этот разрыв.

- **YOLO** (You Only Look Once, Redmon et al., 2016) делит изображение на сетку $S \times S$. Каждая ячейка сетки предсказывает $B$ ограничивающих рамок и $C$ вероятностей классов. Если центр объекта попадает в ячейку сетки, эта ячейка отвечает за его обнаружение. YOLO работает чрезвычайно быстро, так как вся детекция представляет собой один прямой проход без этапа формирования предложений.

- **YOLOv2** добавила якорные рамки (anchor boxes), пакетную нормализацию и обучение на нескольких масштабах. **YOLOv3** использовала Feature Pyramid Network и выполняла предсказания на трех масштабах. **YOLOv4-v8** продолжили совершенствоваться за счет более эффективных архитектур (backbones), сетей агрегации путей (path aggregation networks) и мозаичной аугментации данных (склеивание четырех изображений вместе во время обучения для увеличения разнообразия контекста).

- **SSD** (Single Shot MultiBox Detector, Liu et al., 2016) выполняет предсказания на нескольких масштабах карт признаков внутри основной архитектуры, используя якорные рамки на каждом масштабе. Ранние (высокого разрешения) карты признаков обнаруживают малые объекты; поздние (низкого разрешения) карты обнаруживают крупные объекты. SSD работает быстрее, чем Faster R-CNN, при сопоставимой точности.

- **RetinaNet** (Lin et al., 2017) выявила основную проблему одностадийных детекторов: дисбаланс классов. Подавляющее большинство якорных рамок соответствуют фону, что порождает «легкие» негативные примеры, которые доминируют в функции потерь и подавляют градиенты от редких позитивных примеров.

- **Фокусная функция потерь (focal loss)** решает эту проблему путем уменьшения веса легких примеров:

$$\text{FL}(p_t) = -\alpha_t (1 - p_t)^\gamma \log(p_t)$$

- где $p_t$ — предсказанная вероятность для правильного класса. Когда модель уверена и права ($p_t$ велико), $(1 - p_t)^\gamma$ мало, что снижает вклад легких негативных примеров в функцию потерь. Гиперпараметр $\gamma$ (обычно равный 2) управляет силой уменьшения веса. При $\gamma = 0$ фокусная функция потерь сводится к стандартной кросс-энтропии. Благодаря фокусной функции потерь RetinaNet достигла точности, сопоставимой с двухстадийными детекторами, при скорости одностадийных.

- **Безякорная детекция (anchor-free detection)** полностью исключает якорные рамки, уменьшая необходимость в настройке гиперпараметров и упрощая пайплайн.

- **FCOS** (Fully Convolutional One-Stage, Tian et al., 2019) предсказывает в каждой пространственной позиции карты признаков расстояния от этой позиции до четырех сторон ближайшей ограничивающей рамки (слева, сверху, справа, снизу), а также метку класса. Оценка **центрированности (centerness)** уменьшает вес предсказаний, далеких от центра объекта, что повышает качество. FCOS использует FPN для работы с несколькими масштабами.

- **CenterNet** (Zhou et al., 2019) обнаруживает объекты как точки: она предсказывает тепловую карту, где пики соответствуют центрам объектов, а затем регрессирует ширину и высоту в каждом пике. Детекция превращается в задачу оценки ключевых точек. Это элегантный и безякорный подход, но он требует тщательной постобработки тепловой карты.

- **CornerNet** обнаруживает объекты как пары углов (верхний левый и нижний правый). Она предсказывает две тепловые карты (по одной для каждого типа угла) и использует **ассоциативный эмбеддинг** для сопоставления соответствующих углов в ограничивающие рамки. Это позволяет избежать использования якорей и работать с объектами произвольной формы.

- **Семантическая сегментация** присваивает метку класса каждому пикселю изображения. В отличие от детекции (которая выдает рамки), сегментация создает плотную карту на уровне пикселей. В сцене улицы каждый пиксель может быть помечен как дорога, тротуар, автомобиль, пешеход, здание, небо и т. д.

![Семантическая сегментация: входная сцена улицы и соответствующая ей карта меток на уровне пикселей, где каждый цвет представляет свой класс](../images/semantic_segmentation.svg)

- **Полносверточные сети (FCN)** (Long et al., 2015) адаптировали классификационные CNN для сегментации путем замены полносвязных слоев на сверточные, что позволило сети выводить пространственную карту вместо одного класса. Увеличение разрешения (через транспонированные свертки или билинейную интерполяцию) восстанавливает выход до разрешения входа. Пропускающие связи (skip connections) из более ранних слоев добавляют пространственные детали, потерянные при уменьшении разрешения.

- **Транспонированная свертка** (иногда называемая «деконволюцией») — это аналог свертки для увеличения разрешения. Там, где свертка со страйдом уменьшает пространственные размерности, транспонированная свертка их увеличивает. Она вставляет нули между элементами входа, а затем применяет стандартную свертку, фактически обучаясь тому, как выполнять апсемплинг.

- **U-Net** (Ronneberger et al., 2015) представила симметричную архитектуру энкодера-декодера с пропускающими связями на каждом уровне. Энкодер (сжимающий путь) уменьшает пространственное разрешение, увеличивая количество каналов, точно так же, как в классификационной CNN. Декодер (расширяющий путь) выполняет апсемплинг обратно до полного разрешения. Пропускающие связи конкатенируют карты признаков энкодера с картами признаков декодера на каждом уровне, предоставляя декодеру мелкие пространственные детали. Такое сочетание высокоуровневой семантики и низкоуровневых деталей позволяет получать четкие и точные границы сегментации.

![Архитектура U-Net: путь энкодера слева с уменьшением разрешения, путь декодера справа с увеличением разрешения и пропускающие связи, соединяющие соответствующие уровни](../images/unet_architecture.svg)

- U-Net была изначально разработана для сегментации биомедицинских изображений (где обучающих данных мало), и ее архитектура стала фундаментом для многих последующих моделей, включая U-Net в латентных диффузионных моделях (файл 04).

- **DeepLab** (Chen et al., 2014-2018) представила два ключевых нововведения для сегментации:

    - **Атрозная (разреженная) свертка (atrous/dilated convolution)**: стандартная свертка с промежутками, вставленными между элементами фильтра, управляемыми коэффициентом расширения $r$. Фильтр 3x3 с расширением $r$ имеет поле зрения $(2r + 1) \times (2r + 1)$, используя при этом всего 9 параметров. Это позволяет захватывать контекст на нескольких масштабах без уменьшения разрешения, сохраняя пространственную детализацию.

- **Atrous Spatial Pyramid Pooling (ASPP)**: применяет несколько атроус-свёрток (atrous convolutions) с различными коэффициентами расширения параллельно (например, 1, 6, 12, 18), объединяет результаты и выполняет слияние с помощью свёртки $1 \times 1$. ASPP захватывает контекст на нескольких масштабах одновременно, что по духу близко к модулю Inception (файл 02), но использует расширение вместо различных размеров ядер.

- DeepLab также использовала **условное случайное поле (CRF)** (глава 05) в качестве этапа постобработки для уточнения границ сегментации, побуждая пространственно близкие пиксели с похожими цветами иметь одну и ту же метку.

- **Сегментация экземпляров (Instance segmentation)** объединяет детекцию и сегментацию: она идентифицирует каждый отдельный экземпляр объекта и создает маску на уровне пикселей для каждого из них. Два автомобиля на сцене получают две отдельные маски, а не просто «автомобиль» для обоих.

- **Mask R-CNN** (He et al., 2017) расширяет Faster R-CNN, добавляя небольшую «голову» сегментации, которая предсказывает бинарную маску для каждого обнаруженного объекта. Архитектура представляет собой Faster R-CNN + ветвь маски: ветвь маски принимает признаки, полученные с помощью RoI-пулинга, и выводит бинарную маску $m \times m$ для каждого класса. В ней используется **RoIAlign** вместо RoI-пулинга: билинейная интерполяция в точно выбранных точках вместо квантованных ячеек сетки, что позволяет избежать пространственного рассогласования, вызываемого квантованием. Это небольшое изменение значительно улучшает качество маски.

- Mask R-CNN обучается с использованием многозадачной функции потерь: функция потерь классификации + функция потерь регрессии ограничивающих рамок + функция потерь маски (попиксельная бинарная кросс-энтропия). Ветвь маски предсказывает маску для каждого класса независимо; используется только маска, соответствующая предсказанному классу, что отделяет предсказание маски от классификации и улучшает оба процесса.

- **Паноптическая сегментация (Panoptic segmentation)** объединяет семантическую сегментацию и сегментацию экземпляров в одну задачу. Каждый пиксель получает как метку класса (семантическую), так и ID экземпляра (для классов «объектов», таких как автомобили и люди). Классы «фона» (небо, дорога, трава) получают только семантические метки, поскольку они представляют собой аморфные области без счетных экземпляров.

- Метрика паноптического качества (PQ) оценивает это путем разложения на качество сегментации (среднее IoU для сопоставленных сегментов) и качество распознавания (F1-мера для сопоставленных сегментов):

$$\text{PQ} = \underbrace{\frac{\sum_{(p,g) \in \text{TP}} \text{IoU}(p,g)}{|\text{TP}|}}_{\text{SQ}} \times \underbrace{\frac{|\text{TP}|}{|\text{TP}| + \frac{1}{2}|\text{FP}| + \frac{1}{2}|\text{FN}|}}_{\text{RQ}}$$

- **Сегментация в реальном времени** критически важна для таких приложений, как автономное вождение и дополненная реальность, где бюджет задержки очень ограничен (часто менее 30 миллисекунд на кадр).

- **BiSeNet** (Bilateral Segmentation Network, Yu et al., 2018) использует два параллельных пути: **пространственный путь** с широкими, неглубокими слоями, который сохраняет пространственные детали, и **контекстный путь** с глубокими, узкими слоями, который захватывает семантику. Выходные данные объединяются, обеспечивая как скорость, так и точность.

- **DDRNet** (Deep Dual-Resolution Network, Hong et al., 2021) поддерживает две ветви с разным разрешением на протяжении всей сети с постоянным обменом информацией между ними. Ветвь высокого разрешения сохраняет пространственные детали, в то время как ветвь низкого разрешения захватывает глобальный контекст. Множественные модули двустороннего слияния объединяют информацию в обоих направлениях.

- Общая тенденция в сегментации в реальном времени заключается в отказе от тяжелой архитектуры энкодер-декодер в пользу поддержания достаточного пространственного разрешения на протяжении всей сети, что позволяет пожертвовать некоторой точностью ради значительно меньшей задержки.

1. Реализуйте вычисление IoU и подавление немаксимумов (NMS) с нуля. Примените NMS к набору перекрывающихся ограничивающих рамок и визуализируйте результат.
```python
import jax.numpy as jnp
import matplotlib.pyplot as plt
import matplotlib.patches as patches

def compute_iou(box1, box2):
    """Compute IoU between two boxes [x1, y1, x2, y2]."""
    x1 = jnp.maximum(box1[0], box2[0])
    y1 = jnp.maximum(box1[1], box2[1])
    x2 = jnp.minimum(box1[2], box2[2])
    y2 = jnp.minimum(box1[3], box2[3])

    intersection = jnp.maximum(0, x2 - x1) * jnp.maximum(0, y2 - y1)
    area1 = (box1[2] - box1[0]) * (box1[3] - box1[1])
    area2 = (box2[2] - box2[0]) * (box2[3] - box2[1])
    union = area1 + area2 - intersection

    return intersection / (union + 1e-6)

def nms(boxes, scores, iou_threshold=0.5):
    """Non-Maximum Suppression."""
    order = jnp.argsort(-scores)  # sort by descending confidence
    keep = []

    remaining = list(range(len(scores)))
    order_list = order.tolist()

    while order_list:
        idx = order_list[0]
        keep.append(idx)
        order_list = order_list[1:]

        new_order = []
        for j in order_list:
            iou = compute_iou(boxes[idx], boxes[j])
            if iou < iou_threshold:
                new_order.append(j)
        order_list = new_order

    return keep

# Example: overlapping detections of the same object
boxes = jnp.array([
    [50, 60, 150, 160],   # high confidence
    [55, 65, 155, 165],   # overlapping duplicate
    [52, 58, 148, 158],   # overlapping duplicate
    [200, 100, 300, 200], # different object
    [205, 105, 305, 205], # overlapping duplicate
])
scores = jnp.array([0.95, 0.80, 0.70, 0.90, 0.60])

keep = nms(boxes, scores, iou_threshold=0.5)

fig, axes = plt.subplots(1, 2, figsize=(14, 5))
colors = ['#3498db', '#e74c3c', '#27ae60', '#9b59b6', '#f39c12']

for ax, title, indices in zip(axes, ['Before NMS', 'After NMS'],
                               [range(len(boxes)), keep]):
    ax.set_xlim(0, 400); ax.set_ylim(0, 300)
    ax.set_aspect('equal'); ax.invert_yaxis()
    ax.set_title(title)
    for i in indices:
        b = boxes[i]
        rect = patches.Rectangle((b[0], b[1]), b[2]-b[0], b[3]-b[1],
                                  linewidth=2, edgecolor=colors[i],
                                  facecolor='none')
        ax.add_patch(rect)
        ax.text(b[0], b[1]-5, f'{scores[i]:.2f}', color=colors[i], fontsize=10)

plt.tight_layout(); plt.show()
print(f"Kept {len(keep)} of {len(boxes)} boxes after NMS")
```

2. Реализуйте упрощенную сеть формирования предложений регионов (Region Proposal Network, RPN). Получив карту признаков, сгенерируйте якорные рамки (anchor boxes) с различными масштабами и соотношениями сторон, а также предскажите оценки объектности (objectness scores) и смещения рамок.

import jax

def generate_anchors(feature_h, feature_w, stride, scales, ratios):
    """Generate anchor boxes for each position on the feature map."""
    anchors = []
    for y in range(feature_h):
        for x in range(feature_w):
            cx = (x + 0.5) * stride
            cy = (y + 0.5) * stride
            for s in scales:
                for r in ratios:
                    w = s * jnp.sqrt(r)
                    h = s / jnp.sqrt(r)
                    anchors.append([cx - w/2, cy - h/2, cx + w/2, cy + h/2])
    return jnp.array(anchors)

def rpn_forward(feature_map, params):
    """Simplified RPN: predicts objectness and box offsets per anchor."""
    H, W, C = feature_map.shape
    n_anchors = params['cls_w'].shape[1]

    # Slide a 1x1 conv over the feature map (simplified)
    cls_scores = feature_map.reshape(-1, C) @ params['cls_w']  # (H*W, n_anchors)
    box_offsets = feature_map.reshape(-1, C) @ params['reg_w']  # (H*W, n_anchors*4)

    cls_scores = jax.nn.sigmoid(cls_scores)
    return cls_scores.ravel(), box_offsets.reshape(-1, 4)

3. Реализуйте упрощенный энкодер-декодер U-Net с пробросами (skip connections) для 1D-сегментации (бинарная разметка 1D-сигнала).

```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

def conv1d_same(x, kernel):
    """1D convolution with same padding."""
    k = len(kernel)
    pad = k // 2
    x_pad = jnp.pad(x, pad, mode='edge')
    n = len(x)
    out = jnp.zeros(n)
    for i in range(n):
        out = out.at[i].set(jnp.sum(x_pad[i:i+k] * kernel))
    return out

def downsample(x):
    return x[::2]

def upsample(x, target_len):
    return jnp.interp(jnp.linspace(0, 1, target_len), jnp.linspace(0, 1, len(x)), x)

def unet_1d(x, params):
    """Simplified 1D U-Net with 2 encoder/decoder levels."""
    # Encoder
    e1 = jnp.maximum(0, conv1d_same(x, params['enc1']))
    e1_down = downsample(e1)

    e2 = jnp.maximum(0, conv1d_same(e1_down, params['enc2']))
    e2_down = downsample(e2)

    # Bottleneck
    bottleneck = jnp.maximum(0, conv1d_same(e2_down, params['bottleneck']))

    # Decoder with skip connections
    d2_up = upsample(bottleneck, len(e2))
    d2 = jnp.maximum(0, conv1d_same(d2_up + e2, params['dec2']))  # skip connection

    d1_up = upsample(d2, len(e1))
    d1 = conv1d_same(d1_up + e1, params['dec1'])  # skip connection

    return jax.nn.sigmoid(d1)

# Create signal with labelled regions
n = 128
t = jnp.linspace(0, 4 * jnp.pi, n)
signal = jnp.sin(t) + 0.5 * jnp.sin(3 * t)
labels = (signal > 0.5).astype(jnp.float32)  # binary segmentation target

key = jax.random.PRNGKey(42)
keys = jax.random.split(key, 5)
params = {
    'enc1': jax.random.normal(keys[0], (5,)) * 0.3,
    'enc2': jax.random.normal(keys[1], (5,)) * 0.3,
    'bottleneck': jax.random.normal(keys[2], (3,)) * 0.3,
    'dec2': jax.random.normal(keys[3], (5,)) * 0.3,
    'dec1': jax.random.normal(keys[4], (5,)) * 0.3,
}

def loss_fn(params, signal, labels):
    pred = unet_1d(signal, params)
    return -jnp.mean(labels * jnp.log(pred + 1e-7) + (1 - labels) * jnp.log(1 - pred + 1e-7))

grad_fn = jax.jit(jax.grad(loss_fn))
lr = 0.05

for step in range(500):
    grads = grad_fn(params, signal, labels)
    params = {k: params[k] - lr * grads[k] for k in params}

pred = unet_1d(signal, params)

fig, axes = plt.subplots(3, 1, figsize=(12, 7), sharex=True)
axes[0].plot(t, signal, color='#3498db', linewidth=1.5)
axes[0].set_title('Input Signal'); axes[0].set_ylabel('Value')

axes[1].fill_between(t, 0, labels, alpha=0.3, color='#27ae60')
axes[1].set_title('Ground Truth Labels'); axes[1].set_ylabel('Label')

axes[2].plot(t, pred, color='#e74c3c', linewidth=1.5)
axes[2].fill_between(t, 0, (pred > 0.5).astype(float), alpha=0.2, color='#e74c3c')
axes[2].set_title('U-Net Prediction'); axes[2].set_ylabel('Probability')
axes[2].set_xlabel('t')

plt.tight_layout(); plt.show()
print(f"Final loss: {loss_fn(params, signal, labels):.4f}")
print(f"Pixel accuracy: {jnp.mean((pred > 0.5) == labels):.2%}")
```
