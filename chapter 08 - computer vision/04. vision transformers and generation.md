# Vision Transformer и генерация

*Vision Transformer применяют механизм самовнимания к патчам изображения, бросая вызов доминированию CNN за счет пространственного обучения на основе данных. В этом файле рассматриваются ViT, DeiT, Swin Transformer, генерация изображений с помощью GAN (StyleGAN), VAE и диффузионных моделей (DDPM, Stable Diffusion), а также суперразрешение и нейросетевой перенос стиля.*

- CNN (файл 02) обладают сильными пространственными индуктивными смещениями: локальной связностью, разделением весов и эквивариантностью к сдвигу. Vision Transformer (ViT) задают провокационный вопрос: что, если полностью отказаться от этих смещений и позволить модели изучать пространственную структуру на основе данных, используя только механизм внимания из главы 06?

- **Vision Transformer (ViT)** (Dosovitskiy et al., 2021) применяет стандартный энкодер Transformer непосредственно к изображениям. Ключевая идея заключается в том, чтобы рассматривать изображение как последовательность патчей, подобно тому как в NLP текст рассматривается как последовательность токенов.

- Процесс выглядит следующим образом:
    1. Изображение (высота $H$, ширина $W$, каналы $C$) разбивается на сетку неперекрывающихся патчей размером $P \times P$. Это дает $N = HW / P^2$ патчей.
    2. Каждый патч вытягивается в вектор длиной $P^2 \cdot C$ и проецируется в размерность модели $D$ с помощью обучаемого линейного эмбеддинга (одно матричное умножение, глава 02).
    3. Добавляется обучаемый эмбеддинг **[CLS]-токена** (аналогично [CLS] в BERT, глава 07). Этот токен взаимодействует со всеми патчами, а его итоговое представление используется для классификации.
    4. Добавляются **позиционные эмбеддинги** (по одному обучаемому вектору на каждую позицию) для обеспечения пространственной информации, так как механизм внимания перестановочно-эквивариантен.
    5. Последовательность из $(N + 1)$ эмбеддингов токенов проходит через стандартный энкодер Transformer (многоголовое самовнимание + FFN, глава 06).
    6. Итоговое представление [CLS]-токена проходит через классификационную «голову» (небольшой MLP).

- **Эмбеддинг патчей** эквивалентен свёртке с размером ядра $P$ и шагом $P$ (без перекрытий). ViT буквально преобразует 2D-изображение в 1D-последовательность, а затем обрабатывает её с помощью той же архитектуры, что используется для языка.

- ViT обладает меньшим индуктивным смещением, чем CNN: модель не навязывает локальную связность или эквивариантность к сдвигу. Это означает, что для изучения пространственной структуры с нуля требуется больше обучающих данных. На небольших датасетах CNN превосходят ViT. Однако при обучении на очень больших датасетах (JFT-300M, 300 миллионов изображений) ViT сравнивается с лучшими CNN или превосходит их, что говорит о том, что индуктивные смещения CNN полезны для эффективности использования данных, но не являются обязательными для достижения максимальной производительности.

- Самовнимание в ViT имеет сложность $O(N^2)$ относительно количества патчей. Для изображения 224x224 с патчами 16x16 $N = 196$, что вполне приемлемо. Но для изображений более высокого разрешения или меньших патчей квадратичная стоимость становится непомерно высокой.

- **DeiT** (Data-efficient Image Transformer, Touvron et al., 2021) показал, что ViT можно эффективно обучать только на ImageNet (без огромного датасета JFT), используя сильную аугментацию данных, регуляризацию (стохастическая глубина, сглаживание меток, dropout) и **дистилляцию знаний**: предобученная модель CNN («учитель») предоставляет «мягкие» метки, которые модель ViT («ученик») учится воспроизводить. В DeiT был добавлен **дистилляционный токен** наряду с [CLS]-токеном, обученный предсказывать выходные данные учителя.

- **Swin Transformer** (Liu et al., 2021) решает два основных ограничения ViT: квадратичную стоимость относительно размера изображения и отсутствие иерархических карт признаков (необходимых для детекции и сегментации).

- Swin вводит **сдвигающиеся окна**: вместо глобального самовнимания по всем патчам внимание вычисляется внутри локальных окон (например, 7x7 патчей). Это делает стоимость линейной относительно размера изображения: $O(N)$ вместо $O(N^2)$. Однако одни лишь локальные окна препятствовали бы передаче информации между регионами.

- **Сдвиг окон** решает эту проблему: в чередующихся слоях разбиение на окна сдвигается на половину размера окна. Это создает связи между окнами, позволяя информации передаваться между всеми частями изображения через слои без затрат на глобальное внимание.

![Swin Transformer: слой l вычисляет внимание внутри обычных окон, слой l+1 сдвигает разбиение окон наполовину, создавая связи между окнами](../images/swin_shifted_windows.svg)

- Swin также строит **иерархическое представление** путем объединения патчей на разных этапах. После каждого этапа соседние патчи 2x2 конкатенируются и проецируются, чтобы удвоить размерность каналов и уменьшить пространственное разрешение вдвое. Это создает многомасштабные карты признаков, аналогичные тем, что используются в CNN и FPN (файл 03), делая Swin напрямую совместимым с «головами» для детекции, такими как Faster R-CNN, и «головами» для сегментации, такими как U-Net.

- **PVT** (Pyramid Vision Transformer) использует аналогичный иерархический подход с пространственным уменьшением внимания: на каждом этапе ключи и значения подвергаются пространственной дискретизации перед вычислением внимания, что снижает квадратичную стоимость при сохранении глобального рецептивного поля.

- **Самообучающееся визуальное обучение** тренирует представления на неразмеченных изображениях. Сбор меток стоит дорого, а изображений много. Цель состоит в том, чтобы изучить признаки, которые хорошо переносятся на последующие задачи без какой-либо ручной разметки.

- **Контрастивное обучение** тренирует модель распознавать, что два аугментированных представления одного и того же изображения («положительная пара») должны иметь схожие представления, в то время как представления разных изображений («отрицательные пары») должны быть несхожими.

- **SimCLR** (Chen et al., 2020) создает два аугментированных представления каждого изображения в батче, кодирует оба с помощью общего бэкбона и проекционной «головы», а также применяет **функцию потерь NT-Xent** (нормализованная кросс-энтропия с температурным масштабированием):

$$\ell_{i,j} = -\log \frac{\exp(\text{sim}(z_i, z_j) / \tau)}{\sum_{k \neq i} \exp(\text{sim}(z_i, z_k) / \tau)}$$

- где $\text{sim}$ — косинусное сходство (глава 01), а $\tau$ — температурный параметр. Числитель сближает положительные пары; знаменатель отдаляет отрицательные пары. SimCLR требует больших размеров батча (4096+), чтобы обеспечить достаточное количество отрицательных примеров.

- **MoCo** (Momentum Contrast, He et al., 2020) решает проблему необходимости использования больших батчей за счет поддержания **очереди с импульсным обновлением** (momentum-updated queue) негативных эмбеддингов. Энкодер запросов (query encoder) обновляется с помощью градиентного спуска; энкодер ключей (key encoder) обновляется как экспоненциальное скользящее среднее (EMA, глава 04) энкодера запросов: $\theta_k \leftarrow m \theta_k + (1 - m) \theta_q$, где $m = 0.999$. Очередь хранит недавние эмбеддинги ключей, обеспечивая большой и согласованный набор негативных примеров без необходимости использования огромных батчей.

- **BYOL** (Bootstrap Your Own Latent, Grill et al., 2020) полностью исключает использование негативных пар. В нем используются две сети: «онлайн-сеть» и «целевая сеть» (EMA онлайн-сети). Онлайн-сеть предсказывает представление целевой сети для другого аугментированного вида изображения. Без негативных примеров BYOL избегает проблемы коллапса (когда модель выдает один и тот же вектор для всего) за счет асимметрии головы предиктора и целевой сети EMA.

- **DINO** (Self-Distillation with No Labels, Caron et al., 2021) применяет самодистилляцию к ViT. Сеть-студент предсказывает выход сети-учителя (EMA студента) для различных аугментированных видов. Учитель использует более крупные фрагменты (crops), студент — более мелкие. DINO создает признаки, содержащие явную информацию о структуре сцены: карты самовнимания (self-attention maps) обученных с помощью DINO ViT естественным образом сегментируют объекты без какого-либо обучения с учителем для сегментации.

- **Маскированное моделирование изображений** (Masked image modelling) — это визуальный аналог маскированного языкового моделирования BERT (глава 07). Большая часть входных патчей маскируется, и модель учится их восстанавливать.

- **MAE** (Masked Autoencoders, He et al., 2022) маскирует 75% патчей и обучает энкодер-декодер ViT восстанавливать пропущенные значения пикселей. Энкодер обрабатывает только немаскированные патчи (что экономит в 4 раза больше вычислений во время предварительного обучения), а легкий декодер восстанавливает полное изображение из закодированных видимых патчей и обучаемых маскирующих токенов.

- **BEiT** (BERT Pre-training of Image Transformers, Bao et al., 2022) маскирует патчи и предсказывает дискретные визуальные токены (полученные с помощью предварительно обученного токенизатора dVAE), а не исходные пиксели. Это параллельно предсказанию дискретных токенов слов в BERT и позволяет избежать работы с низкоуровневыми деталями восстановления пикселей.

- **Генерация изображений** направлена на создание новых реалистичных изображений, которых нет в обучающей выборке. Основная задача заключается в моделировании многомерного распределения вероятностей естественных изображений.

- **Генеративно-состязательные сети (GAN)** (Goodfellow et al., 2014) используют две конкурирующие сети: **генератор** $G$, который создает поддельные изображения из случайного шума, и **дискриминатор** $D$, который пытается отличить реальные изображения от поддельных. Они обучаются состязательно: $G$ пытается обмануть $D$, а $D$ пытается поймать $G$.

$$\min_G \max_D \; \mathbb{E}_{x \sim p_{\text{data}}}[\log D(x)] + \mathbb{E}_{z \sim p(z)}[\log(1 - D(G(z)))]$$

- Генератор принимает случайный латентный вектор $z$ (выбранный из простого распределения, например, гауссовского) и отображает его через серию транспонированных свёрток для создания изображения. Дискриминатор представляет собой стандартный классификатор CNN. В состоянии равновесия $G$ создает изображения, неотличимые от реальных данных, а $D$ выдает 0.5 для всех входных данных.

- **Коллапс моды** (Mode collapse) — основной вид сбоя GAN: генератор учится создавать лишь несколько типов изображений, которые обманывают дискриминатор, игнорируя разнообразие обучающих данных. Генератор находит небольшой набор «безопасных» выходов вместо того, чтобы покрывать все распределение.

- Трюки обучения, стабилизирующие GAN, включают: спектральную нормализацию (ограничение константы Липшица дискриминатора), прогрессивное увеличение (обучение сначала при низком разрешении, затем постепенное увеличение), сопоставление признаков (сопоставление статистики промежуточных признаков дискриминатора, а не финального выхода) и использование расстояния Вассерштейна вместо исходной целевой функции дивергенции JS.

- **StyleGAN** (Karras et al., 2019) — самая влиятельная архитектура GAN для высококачественного синтеза изображений. Ее ключевое новшество — **генератор на основе стилей**: вместо подачи латентного вектора $z$ напрямую в генератор, он сначала проходит через **сеть отображения** (mapping network, 8-слойный MLP) для создания вектора стиля $w$. Этот вектор стиля внедряется в каждый слой генератора с помощью **адаптивной нормализации экземпляров (AdaIN)**, которая модулирует статистику карты признаков:

$$\text{AdaIN}(x, y) = y_{s} \cdot \frac{x - \mu(x)}{\sigma(x)} + y_{b}$$

- где $y_s$ и $y_b$ — масштаб и смещение, полученные из $w$. Разные слои управляют разными аспектами: ранние слои управляют грубыми признаками (поза, форма лица), средние слои — средними признаками (прическа, глаза), а поздние слои — мелкими деталями (веснушки, текстура волос). StyleGAN может генерировать фотореалистичные лица с разрешением 1024x1024.

- **Вариационные автокодировщики (VAE)** (глава 06) предлагают альтернативный генеративный подход. В отличие от GAN, VAE имеют принципиальную вероятностную основу с четкой целевой функцией обучения (ELBO). Они склонны создавать более размытые изображения, чем GAN, но предлагают более плавное и структурированное латентное пространство. VAE — это пара энкодер-декодер, используемая в латентных диффузионных моделях для сжатия изображений в латентное пространство и обратно.

- **Диффузионные модели** стали доминирующей парадигмой генерации изображений, превзойдя GAN как по качеству, так и по разнообразию. Идея концептуально проста: постепенно добавлять шум к данным, пока они не превратятся в чистый гауссовский шум (**прямой процесс**), а затем научиться обращать этот процесс шаг за шагом (**обратный процесс**).

- **Прямой процесс** добавляет гауссовский шум в течение $T$ временных шагов:

$$q(x_t | x_{t-1}) = \mathcal{N}(x_t; \sqrt{1 - \beta_t} \, x_{t-1}, \beta_t I)$$

- где $\beta_t$ — график шума, который увеличивается со временем. После достаточного количества шагов $x_T$ становится приблизительно чистым гауссовским шумом независимо от исходного изображения $x_0$. Используя трюк репараметризации (глава 06) и полагая $\alpha_t = 1 - \beta_t$, $\bar{\alpha}_t = \prod_{s=1}^{t} \alpha_s$, мы можем выбирать $x_t$ непосредственно из $x_0$:

$$x_t = \sqrt{\bar{\alpha}_t} \, x_0 + \sqrt{1 - \bar{\alpha}_t} \, \epsilon, \quad \epsilon \sim \mathcal{N}(0, I)$$

- **Обратный процесс** (reverse process) учится удалять шум: начиная с чистого шума $x_T$, модель предсказывает шум $\epsilon$, добавленный на каждом шаге, и вычитает его, чтобы восстановить $x_{t-1}$. Этот процесс параметризуется нейронной сетью $\epsilon_\theta$ (обычно U-Net, из файла 03), которая обучается с помощью простой функции потерь MSE:

$$\mathcal{L} = \mathbb{E}_{t, x_0, \epsilon}\left[\|\epsilon - \epsilon_\theta(x_t, t)\|^2\right]$$

![Прямой и обратный диффузионные процессы: чистое изображение постепенно искажается шумом в течение T шагов (прямой процесс), нейронная сеть учится обращать каждый шаг (обратный процесс), начиная с чистого шума для генерации чистого изображения](../images/diffusion_process.svg)

- **DDPM** (Denoising Diffusion Probabilistic Models, Ho et al., 2020) заложили основу этого фреймворка. Сэмплирование требует итерации по всем $T$ шагам (обычно 1000), что происходит медленно. **DDIM** (Denoising Diffusion Implicit Models, Song et al., 2021) переформулируют процесс сэмплирования как детерминированное отображение, позволяя пропускать большое количество шагов (например, 50 шагов вместо 1000) с минимальной потерей качества.

- **Модели на основе оценки градиента** (Score-based models, Song and Ermon, 2019) предлагают альтернативный взгляд. Вместо предсказания шума $\epsilon$ модель оценивает **score-функцию** $\nabla_{x_t} \log p(x_t)$, то есть градиент логарифма вероятности по отношению к зашумленному изображению. Этот градиент указывает в сторону областей с более высокой вероятностью (более чистых изображений) распределения данных. Сэмплирование следует за этим градиентом с использованием динамики Ланжевена. Модели на основе оценки градиента и DDPM были объединены в рамках **стохастических дифференциальных уравнений (SDE)**: прямой процесс представляет собой SDE, добавляющее шум, а обратный процесс — это SDE, обращенное во времени.

- **Classifier-free guidance** (Ho and Salimans, 2022) управляет компромиссом между качеством сэмпла и разнообразием. Модель обучается как условно (с текстовым промптом или меткой класса), так и безусловно (с пропуском условия случайным образом). Во время сэмплирования предсказание представляет собой взвешенную комбинацию:

$$\hat{\epsilon} = \epsilon_\theta(x_t, \varnothing) + s \cdot (\epsilon_\theta(x_t, c) - \epsilon_\theta(x_t, \varnothing))$$

- где $c$ — условие, $\varnothing$ — нулевое условие, а $s > 1$ — коэффициент направляющего воздействия (guidance scale). Более высокое значение $s$ позволяет получать изображения, которые сильнее соответствуют условию, но с меньшим разнообразием. $s = 1$ соответствует модели без направляющего воздействия; $s = 7.5$ является распространенным значением по умолчанию.

- **Латентная диффузия** (Latent diffusion, Rombach et al., 2022; Stable Diffusion) переносит диффузионный процесс из пиксельного пространства в обученное латентное пространство. Предобученный VAE-энкодер сжимает изображения в низкоразмерное латентное представление (обычно с пространственным понижением размерности в 4 или 8 раз), диффузия работает в этом сжатом пространстве, а VAE-декодер восстанавливает пиксели из очищенного от шума латентного представления. Это значительно эффективнее: диффузия изображения 512x512 в пиксельном пространстве означает обработку тензора $512 \times 512 \times 3$, тогда как в латентном пространстве — только тензора $64 \times 64 \times 4$.

- Denoising U-Net в латентной диффузии получает на вход зашумленный латент, временной шаг (закодированный как синусоидальный эмбеддинг, аналогично позиционному кодированию в трансформерах) и сигнал условия (текстовый эмбеддинг из замороженного текстового энкодера CLIP или T5). Текстовое условие вводится через слои кросс-внимания (cross-attention) внутри U-Net: текстовые эмбеддинги служат ключами и значениями, а признаки изображения — запросами. Это позволяет модели фокусироваться на релевантных частях текстового промпта в каждой пространственной локации.

- **Flow matching** — это новая альтернатива диффузии, которая изучает прямой путь переноса между шумом и данными, а не итеративное удаление шума, как в DDPM.

- **Непрерывный нормализующий поток** (Continuous normalising flow, CNF) определяет зависящее от времени поле скоростей $v_\theta(x, t)$, которое перемещает сэмплы из простого распределения $p_0$ (шум) в распределение данных $p_1$ вдоль плавных траекторий. Преобразование следует обыкновенному дифференциальному уравнению (ODE):

$$\frac{dx}{dt} = v_\theta(x, t), \quad t \in [0, 1]$$

- Начиная с $x_0 \sim \mathcal{N}(0, I)$, интегрирование ODE вперед до $t = 1$ дает сэмпл из распределения данных. Поле скоростей параметризуется нейронной сетью и обучается соответствовать целевому условному потоку.

- **Оптимальный транспорт** (Optimal transport, OT) flow matching (Lipman et al., 2023) использует прямолинейные пути между шумом и данными в качестве целевого потока: условный путь от сэмпла шума $x_0$ к сэмплу данных $x_1$ — это просто $x_t = (1 - t) x_0 + t x_1$, а целевая скорость равна $v = x_1 - x_0$. Функция потерь при обучении принимает вид:

$$\mathcal{L} = \mathbb{E}_{t, x_0, x_1} \left[\|v_\theta(x_t, t) - (x_1 - x_0)\|^2\right]$$

- **Rectified flows** (Liu et al., 2022) итеративно выпрямляют изученные пути потока. После начального этапа обучения модель используется для генерации пар (шум, данные) путем симуляции ODE. Эти пары, которые более тесно согласованы, чем случайные пары, используются для дообучения модели. Повторение этого процесса приводит к получению все более прямых путей, которые можно пройти за меньшее количество шагов ODE (даже за один шаг), что обеспечивает чрезвычайно быструю генерацию.

- Flow matching имеет несколько преимуществ перед диффузией: целевая функция обучения проще (прямая регрессия скорости, отсутствие расписания шума), ODE для сэмплирования более гладкое (требует меньше шагов интегрирования), а связь с оптимальным транспортом обеспечивает теоретическое обоснование. Stable Diffusion 3 и Flux используют flow matching вместо традиционного DDPM.

1. Реализуйте ViT patch embedding с нуля. Разделите изображение на патчи, вытяните их в вектор, спроецируйте в размерность модели, добавьте позиционные эмбеддинги и добавьте [CLS] токен в начало.
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

def create_patch_embedding(image, patch_size, d_model, params):
    """Convert an image into a sequence of patch embeddings."""
    H, W, C = image.shape
    n_patches_h = H // patch_size
    n_patches_w = W // patch_size
    n_patches = n_patches_h * n_patches_w

    # Extract patches
    patches = []
    for i in range(n_patches_h):
        for j in range(n_patches_w):
            patch = image[i*patch_size:(i+1)*patch_size,
                          j*patch_size:(j+1)*patch_size, :]
            patches.append(patch.ravel())
    patches = jnp.stack(patches)  # (N, P*P*C)

    # Linear projection to d_model
    embeddings = patches @ params['proj_w'] + params['proj_b']  # (N, d_model)

    # Prepend CLS token
    cls_token = params['cls_token']  # (1, d_model)
    embeddings = jnp.concatenate([cls_token, embeddings], axis=0)  # (N+1, d_model)

    # Add position embeddings
    embeddings = embeddings + params['pos_embed']  # (N+1, d_model)

    return embeddings, patches

# Setup
H, W, C = 32, 32, 3
patch_size = 8
d_model = 64
n_patches = (H // patch_size) * (W // patch_size)  # 16

key = jax.random.PRNGKey(42)
keys = jax.random.split(key, 5)

# Create a synthetic image with distinct quadrants
image = jnp.zeros((H, W, C))
image = image.at[:16, :16, 0].set(1.0)   # red top-left
image = image.at[:16, 16:, 1].set(1.0)   # green top-right
image = image.at[16:, :16, 2].set(1.0)   # blue bottom-left
image = image.at[16:, 16:, :2].set(1.0)  # yellow bottom-right

params = {
    'proj_w': jax.random.normal(keys[0], (patch_size**2 * C, d_model)) * 0.02,
    'proj_b': jnp.zeros(d_model),
    'cls_token': jax.random.normal(keys[1], (1, d_model)) * 0.02,
    'pos_embed': jax.random.normal(keys[2], (n_patches + 1, d_model)) * 0.02,
}

embeddings, patches = create_patch_embedding(image, patch_size, d_model, params)

print(f"Image shape: {image.shape}")
print(f"Patch size: {patch_size}x{patch_size}")
print(f"Number of patches: {n_patches}")
print(f"Patch vector length: {patch_size**2 * C}")
print(f"Embedding shape: {embeddings.shape}  (CLS + {n_patches} patches)")

# Visualise patches
fig, axes = plt.subplots(2, 5, figsize=(14, 6))
axes[0, 0].imshow(image); axes[0, 0].set_title('Full Image'); axes[0, 0].axis('off')
for idx in range(min(9, n_patches)):
    ax = axes[(idx+1) // 5, (idx+1) % 5]
    patch_img = patches[idx].reshape(patch_size, patch_size, C)
    ax.imshow(patch_img); ax.set_title(f'Patch {idx}'); ax.axis('off')
plt.suptitle('ViT Patch Decomposition')
plt.tight_layout(); plt.show()
```

2. Реализуйте простой цикл обучения GAN. Обучите генератор и дискриминатор на 2D-данных и визуализируйте, как сгенерированное распределение сходится к реальному.

```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

def generator(z, params):
    h = jnp.tanh(z @ params['g_w1'] + params['g_b1'])
    h = jnp.tanh(h @ params['g_w2'] + params['g_b2'])
    return h @ params['g_w3'] + params['g_b3']

def discriminator(x, params):
    h = jax.nn.leaky_relu(x @ params['d_w1'] + params['d_b1'], 0.2)
    h = jax.nn.leaky_relu(h @ params['d_w2'] + params['d_b2'], 0.2)
    return jax.nn.sigmoid(h @ params['d_w3'] + params['d_b3'])

def init_params(key):
    keys = jax.random.split(key, 6)
    z_dim, h_dim, data_dim = 2, 32, 2
    scale = 0.1
    return {
        'g_w1': jax.random.normal(keys[0], (z_dim, h_dim)) * scale,
        'g_b1': jnp.zeros(h_dim),
        'g_w2': jax.random.normal(keys[1], (h_dim, h_dim)) * scale,
        'g_b2': jnp.zeros(h_dim),
        'g_w3': jax.random.normal(keys[2], (h_dim, data_dim)) * scale,
        'g_b3': jnp.zeros(data_dim),
        'd_w1': jax.random.normal(keys[3], (data_dim, h_dim)) * scale,
        'd_b1': jnp.zeros(h_dim),
        'd_w2': jax.random.normal(keys[4], (h_dim, h_dim)) * scale,
        'd_b2': jnp.zeros(h_dim),
        'd_w3': jax.random.normal(keys[5], (h_dim, 1)) * scale,
        'd_b3': jnp.zeros(1),
    }

def d_loss(params, real_data, fake_data):
    real_score = discriminator(real_data, params)
    fake_score = discriminator(fake_data, params)
    return -jnp.mean(jnp.log(real_score + 1e-7) + jnp.log(1 - fake_score + 1e-7))

def g_loss(params, fake_data):
    fake_score = discriminator(fake_data, params)
    return -jnp.mean(jnp.log(fake_score + 1e-7))

# Real data: ring distribution
key = jax.random.PRNGKey(42)
theta = jax.random.uniform(key, (512,)) * 2 * jnp.pi
real_data = jnp.stack([jnp.cos(theta), jnp.sin(theta)], axis=1)
real_data = real_data + jax.random.normal(key, real_data.shape) * 0.05

params = init_params(jax.random.PRNGKey(0))
d_grad = jax.grad(d_loss)
g_grad = jax.grad(g_loss)
lr = 0.001

snapshots = []
for step in range(3000):
    key, k1 = jax.random.split(key)
    z = jax.random.normal(k1, (512, 2))
    fake_data = generator(z, params)

    # Update discriminator
    grads = d_grad(params, real_data, fake_data)
    for k in ['d_w1', 'd_b1', 'd_w2', 'd_b2', 'd_w3', 'd_b3']:
        params[k] = params[k] - lr * grads[k]

    # Update generator
    fake_data = generator(z, params)
    grads = g_grad(params, fake_data)
    for k in ['g_w1', 'g_b1', 'g_w2', 'g_b2', 'g_w3', 'g_b3']:
        params[k] = params[k] - lr * grads[k]

    if step in [0, 500, 1500, 2999]:
        snapshots.append((step, fake_data.copy()))

fig, axes = plt.subplots(1, 4, figsize=(16, 4))
for ax, (step, fake) in zip(axes, snapshots):
    ax.scatter(real_data[:, 0], real_data[:, 1], s=5, alpha=0.3, c='#3498db', label='Real')
    ax.scatter(fake[:, 0], fake[:, 1], s=5, alpha=0.3, c='#e74c3c', label='Generated')
    ax.set_title(f'Step {step}'); ax.set_xlim(-2, 2); ax.set_ylim(-2, 2)
    ax.set_aspect('equal'); ax.legend(markerscale=3)
plt.suptitle('GAN Training: Generator Learns the Ring Distribution')
plt.tight_layout(); plt.show()
```

3. Реализуйте прямой процесс диффузии: добавляйте шум к изображению на возрастающих временных шагах и визуализируйте прогрессивное искажение. Затем реализуйте один шаг удаления шума.

```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

def noise_schedule(T, beta_start=0.0001, beta_end=0.02):
    """Linear noise schedule."""
    betas = jnp.linspace(beta_start, beta_end, T)
    alphas = 1.0 - betas
    alpha_bars = jnp.cumprod(alphas)
    return betas, alphas, alpha_bars

def forward_diffusion(x0, t, alpha_bars, key):
    """Add noise to x0 at timestep t."""
    alpha_bar_t = alpha_bars[t]
    noise = jax.random.normal(key, x0.shape)
    xt = jnp.sqrt(alpha_bar_t) * x0 + jnp.sqrt(1 - alpha_bar_t) * noise
    return xt, noise

# Create a simple 2D "image" (checkerboard)
img = jnp.zeros((32, 32))
for i in range(4):
    for j in range(4):
        if (i + j) % 2 == 0:
            img = img.at[i*8:(i+1)*8, j*8:(j+1)*8].set(1.0)

T = 1000
betas, alphas, alpha_bars = noise_schedule(T)

# Visualise forward process
timesteps = [0, 50, 200, 500, 999]
key = jax.random.PRNGKey(42)

fig, axes = plt.subplots(1, len(timesteps), figsize=(16, 3.5))
for ax, t in zip(axes, timesteps):
    key, subkey = jax.random.split(key)
    xt, noise = forward_diffusion(img, t, alpha_bars, subkey)
    ax.imshow(xt, cmap='gray', vmin=-2, vmax=2)
    ax.set_title(f't={t}\n$\\bar{{\\alpha}}$={alpha_bars[t]:.3f}')
    ax.axis('off')
plt.suptitle('Diffusion Forward Process: Progressive Noise Addition')
plt.tight_layout(); plt.show()

# Simple denoising: train a tiny network to predict noise at t=200
t_denoise = 200
key, k1 = jax.random.split(key)
xt, true_noise = forward_diffusion(img, t_denoise, alpha_bars, k1)

# Tiny "denoiser": just learn a constant noise estimate (for illustration)
noise_estimate = jnp.zeros_like(img)
lr = 0.01
for step in range(100):
    residual = noise_estimate - true_noise
    noise_estimate = noise_estimate - lr * residual

# Reverse one step
alpha_bar_t = alpha_bars[t_denoise]
x_denoised = (xt - jnp.sqrt(1 - alpha_bar_t) * noise_estimate) / jnp.sqrt(alpha_bar_t)

fig, axes = plt.subplots(1, 3, figsize=(12, 4))
axes[0].imshow(img, cmap='gray'); axes[0].set_title('Original $x_0$'); axes[0].axis('off')
axes[1].imshow(xt, cmap='gray', vmin=-2, vmax=2)
axes[1].set_title(f'Noisy $x_{{200}}$'); axes[1].axis('off')
axes[2].imshow(x_denoised, cmap='gray')
axes[2].set_title('Denoised (one step)'); axes[2].axis('off')
plt.tight_layout(); plt.show()

mse = jnp.mean((x_denoised - img)**2)
print(f"Denoising MSE: {mse:.4f}")
```
