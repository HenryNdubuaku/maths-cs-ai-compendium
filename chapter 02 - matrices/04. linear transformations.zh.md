# 线性变换

*每一次 matrix 乘法都是一个 linear transformation，一个重塑、旋转或投影向量同时保持线性性的函数。本文件涵盖旋转、反射、缩放、剪切、projection、映射的 kernel 与 image，以及神经网络层如何把这些变换串联起来。*

- **linear transformation**（或线性映射）是一个接收一个向量、产生另一个向量，同时保持加法和缩放的函数。若 $T$ 是线性的，则：

    - $T(\mathbf{u} + \mathbf{v}) = T(\mathbf{u}) + T(\mathbf{v})$
    - $T(c\mathbf{u}) = cT(\mathbf{u})$

- 每个 linear transformation 都可以表示为乘以一个 matrix。这个 matrix *就是* 该变换。当你用 matrix 乘一个向量时，就是在对该向量施加一个 linear transformation。

- 把 $2 \times 2$ matrix 想成一台接收 2D 向量并输出新 2D 向量的机器。matrix 的各列告诉你标准 basis 向量 $\hat{\mathbf{i}}$ 和 $\hat{\mathbf{j}}$ 在变换后落在哪里。其余一切都由线性性推出。

![matrix 的各列显示 basis 向量落在何处](../images/basis_transform.svg)

- 例如，若

```math
A = \begin{bmatrix} 2 & 1 \\ 1 & 2 \end{bmatrix}
```

  则 $\hat{\mathbf{i}} = [1, 0]^T$ 落在 $[2, 1]^T$（第 1 列），$\hat{\mathbf{j}} = [0, 1]^T$ 落在 $[1, 2]^T$（第 2 列）。其他每个向量都是这两个的组合，所以其输出自动确定。

- 两个 matrix 相乘可以理解为依次施加两个变换。若 $B$ 把向量从一个空间变换，$A$ 再把结果变换，那么 $AB$ 依次完成两步。在游戏引擎中，先旋转角色再向前移动，与先移动再旋转结果不同，这就是为什么 matrix 乘法不可交换。

- **旋转** 在不改变向量长度的前提下把它转动一个角度 $\theta$。向量大小不变，只是指向新方向。

![旋转保持长度但改变方向](../images/rotation.svg)

- 在 2D 中，旋转矩阵为：

```math
R(\theta) = \begin{bmatrix} \cos\theta & -\sin\theta \\ \sin\theta & \cos\theta \end{bmatrix}
```

- 对 $\theta = 90°$：

```math
R = \begin{bmatrix} 0 & -1 \\ 1 & 0 \end{bmatrix}
```

  所以 $[1, 0]^T$ 变为 $[0, 1]^T$。指向右的向量现在指向上。旋转矩阵是 orthogonal 矩阵且 determinant 总为 1。当你在手机上旋转照片时，对每个像素坐标施加的就是这个 matrix。

- 在 3D 中，每条轴都有各自的旋转矩阵。机械臂的每个关节绕特定轴旋转，每个关节就是一个旋转矩阵。绕 z 轴的旋转看起来像 2D 情形嵌入到 3D 中：

```math
R_z(\theta) = \begin{bmatrix} \cos\theta & -\sin\theta & 0 \\ \sin\theta & \cos\theta & 0 \\ 0 & 0 & 1 \end{bmatrix}
```

- **缩放** 沿每条轴独立地拉伸或压缩向量：

```math
S(s_x, s_y) = \begin{bmatrix} s_x & 0 \\ 0 & s_y \end{bmatrix}
```

![缩放按不同因子拉伸每条轴](../images/scaling.svg)

- $S(2, 1.5)$ 把 x 分量加倍、把 y 分量乘以 1.5。沿一条轴用 $-1$ 缩放会翻转该分量。对角矩阵总是一种缩放变换。当把图像缩放到 50% 时，就是对每个像素坐标施加 $S(0.5, 0.5)$。

- **反射** 像镜子一样把向量沿某条轴或直线翻转。沿 x 轴反射保持 x 分量、把 y 分量取反：

```math
\text{Ref}_x = \begin{bmatrix} 1 & 0 \\ 0 & -1 \end{bmatrix}
```

![沿 x 轴反射翻转 y 分量](../images/reflection.svg)

- 例如，$[3, 2]^T$ 变为 $[3, -2]^T$。当你的手机水平翻转自拍使文字读起来正确时，施加的就是反射矩阵。沿直线 $y = x$ 反射会交换两个分量：

```math
\text{Ref}_{y=x} = \begin{bmatrix} 0 & 1 \\ 1 & 0 \end{bmatrix}
```

- 反射矩阵的 determinant 为 $-1$，确认它们翻转朝向。

- 旋转和反射都是 **刚性变换**：它们保持距离和角度不变。表示它们的 matrix 是 orthogonal 矩阵，这就是为什么 orthogonal 矩阵的 determinant 总是 $+1$（旋转）或 $-1$（反射）。

- **剪切** 让向量沿一条轴按另一条轴的比例倾斜。按因子 $k$ 的水平剪切：

```math
\text{Sh}_x(k) = \begin{bmatrix} 1 & k \\ 0 & 1 \end{bmatrix}
```

![剪切使顶部侧滑而底部固定](../images/shearing.svg)

- 每个点水平移动它高度的 $k$ 倍。当 $k = 0.5$ 时，高度为 2 的点向右移 1。底部不动，顶部侧滑。这就是斜体文字的原理：直立字母被剪切从而向右倾斜。

- 上述所有变换（旋转、缩放、反射、剪切）都是 **线性** 变换。它们保持原点不动并保持直线。但 **平移**（按固定量整体移动）呢？

- 平移 *不是* linear transformation，因为它移动了原点。如果每个点向右移 3，零向量就移到 $[3, 0]^T$，破坏了线性性。为处理它，我们用 **仿射变换**，它把一个 linear transformation 与一个平移组合起来：

$$\mathbf{y} = A\mathbf{x} + \mathbf{t}$$

- 要把它表示为单次 matrix 乘法，我们用 **齐次坐标**：给每个向量加一个额外的 1，并使用 $(n+1) \times (n+1)$ matrix：

```math
\begin{bmatrix} A & \mathbf{t} \\ \mathbf{0}^T & 1 \end{bmatrix} \begin{bmatrix} \mathbf{x} \\ 1 \end{bmatrix} = \begin{bmatrix} A\mathbf{x} + \mathbf{t} \\ 1 \end{bmatrix}
```

- 仿射变换保持直线和平行性，但不一定保持角度或长度。视频游戏中的每个物体都用仿射变换定位：旋转它、缩放它，再放到正确位置，全部编码在一个 matrix 中。

- **退化变换**（奇异 matrix）把空间塌缩到更低 dimension。

- 例如，下面的 matrix

```math
\begin{bmatrix} 1 & 2 \\ 2 & 4 \end{bmatrix}
```

  把每个 2D 向量映射到一条直线上，因为两列指向同一方向。determinant 为零，信息丢失，变换无法撤销。

- 把彩色图像（每个像素 3 个值：红、绿、蓝）转成灰度（每个像素 1 个值）是一种退化变换：颜色信息永久消失。

- 在 ML 中，linear transformation 是神经网络的核心，数据表示为一个 matrix（一摺表示人、飞机、文本、图像等对象特征的向量）。

- 每一层施加一次 matrix 乘法（linear transformation），细节在其他章节给出，我们需要解释如何组织这些数据，并恰当地引出神经网络。

- 不过，当今最常用的技术通常几乎只把数据依次穿过一堆 linear transformation，我们称之为 **Transformer**。

- Gemini、ChatGPT、Claude、Qwen、DeepSeek，以及当今世界上表现最好的 AI，都是 transformer！

## 编程任务（使用 CoLab 或 notebook）

1. 对一个向量施加旋转矩阵，并画出原始向量和旋转向量。尝试不同角度。
```python
import jax.numpy as jnp
import matplotlib.pyplot as plt

theta = jnp.pi / 3
R = jnp.array([[jnp.cos(theta), -jnp.sin(theta)],
               [jnp.sin(theta),  jnp.cos(theta)]])

v = jnp.array([1.0, 0.0])
v_rot = R @ v

plt.figure(figsize=(5, 5))
plt.quiver(0, 0, v[0], v[1], angles='xy', scale_units='xy', scale=1, color='red', label='original')
plt.quiver(0, 0, v_rot[0], v_rot[1], angles='xy', scale_units='xy', scale=1, color='blue', label='rotated')
plt.xlim(-1.5, 1.5); plt.ylim(-1.5, 1.5)
plt.grid(True); plt.legend(); plt.gca().set_aspect('equal')
plt.show()
```

2. 对构成正方形的一组点施加剪切变换，并可视化变形后的形状。
```python
import jax.numpy as jnp
import matplotlib.pyplot as plt

square = jnp.array([[0,0],[1,0],[1,1],[0,1],[0,0]]).T

k = 0.5
shear = jnp.array([[1, k],
                    [0, 1]])
sheared = shear @ square

plt.figure(figsize=(6, 4))
plt.plot(square[0], square[1], 'r-o', label='original')
plt.plot(sheared[0], sheared[1], 'b-o', label='sheared')
plt.grid(True); plt.legend(); plt.gca().set_aspect('equal')
plt.show()
```
