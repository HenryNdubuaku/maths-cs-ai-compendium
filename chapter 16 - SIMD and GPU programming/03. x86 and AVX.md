# x86 и AVX

*Процессоры x86 от Intel и AMD доминируют на серверах дата-центров, где происходит большая часть обучения моделей машинного обучения. В этом файле рассматривается эволюция x86 SIMD, программирование с использованием интринсиков AVX/AVX2, AVX-512, Intel AMX для матричных операций, выравнивание памяти, типичные ошибки производительности и профилирование — инструменты для выжимания максимальной производительности из самых распространенных серверных процессоров в мире.*

- Если ваше обучение запускается на облачных виртуальных машинах (AWS, GCP, Azure), оно почти наверняка выполняется на x86. Даже при обучении с интенсивным использованием GPU существуют «узкие места» со стороны CPU: загрузка данных, предобработка, агрегация градиентов и создание чекпоинтов — всё это выполняется на CPU. Оптимизация этих задач с помощью x86 SIMD может существенно сократить общее время обучения.

## Эволюция x86 SIMD

- x86 SIMD развивался за счет постепенного увеличения ширины векторных регистров:

| Поколение | Год | Ширина регистра | Регистры | Ключевые особенности |
|------------|------|---------------|-----------|--------------|
| MMX | 1997 | 64-бит | 8 (mm0-7) | Только целые числа, общие с FPU |
| SSE | 1999 | 128-бит | 8 (xmm0-7) | 4 числа float, выделенные регистры |
| SSE2 | 2001 | 128-бит | 8/16 | 2 числа double, операции с целыми числами |
| AVX | 2011 | 256-бит | 16 (ymm0-15) | 8 чисел float, 3-операндная система команд |
| AVX2 | 2013 | 256-бит | 16 | 256-битные целые числа, FMA, gather |
| AVX-512 | 2017 | 512-бит | 32 (zmm0-31) | 16 чисел float, маскирующие регистры, scatter |
| AMX | 2023 | Тайловые регистры | 8 тайлов | Матричное умножение (BF16, INT8) |

- Каждое поколение удваивало пропускную способность для векторизованного кода. Код, написанный с использованием интринсиков SSE, работает на любом процессоре x86, выпущенном после 2001 года. Для AVX2 требуется процессор 2013 года выпуска или новее. AVX-512 поддерживается процессорами Intel Xeon и некоторыми потребительскими чипами. AMX — новейшая технология (Sapphire Rapids и более поздние).

- **Обратная совместимость**: регистры x86 SSE (xmm) представляют собой младшие 128 бит регистров AVX (ymm), которые, в свою очередь, являются младшими 256 битами регистров AVX-512 (zmm). Старый код SSE работает на новых процессорах без изменений.

## Программирование на AVX2

- AVX2 работает с 256-битными регистрами (YMM), обрабатывая 8 чисел float или 4 числа double одновременно. Это «золотая середина» для переносимого высокопроизводительного кода: доступно практически на всех современных процессорах x86 (2013+).

### Соглашение об именовании интринсиков

- Все интринсики x86 следуют шаблону: `_mm[width]_[operation]_[type]`

    - `_mm` = MMX/SSE (128-бит), `_mm256` = AVX (256-бит), `_mm512` = AVX-512 (512-бит)
    - Операция: `add`, `mul`, `fmadd`, `load`, `store`, `set` и т. д.
    - Тип: `ps` = packed single (float32), `pd` = packed double (float64), `epi32` = packed int32, `si256` = 256-битное целое число

```cpp
#include <immintrin.h>  // all x86 SIMD intrinsics

// Data types
__m256  a;   // 256-bit register holding 8 float32s
__m256d b;   // 256-bit register holding 4 float64s
__m256i c;   // 256-bit register holding integers (8x32, 16x16, or 32x8)
```

### Загрузка и сохранение данных

```cpp
// Load 8 floats from memory
__m256 v = _mm256_loadu_ps(ptr);      // unaligned load (works with any address)
__m256 v = _mm256_load_ps(ptr);       // aligned load (ptr must be 32-byte aligned, faster)

// Store 8 floats to memory
_mm256_storeu_ps(out_ptr, v);          // unaligned store
_mm256_store_ps(out_ptr, v);           // aligned store

// Broadcast a single value to all 8 lanes
__m256 ones = _mm256_set1_ps(1.0f);    // [1, 1, 1, 1, 1, 1, 1, 1]

// Set individual values (rarely needed)
__m256 v = _mm256_set_ps(7,6,5,4,3,2,1,0);  // note: reverse order!

// Zero register
__m256 z = _mm256_setzero_ps();
```

### Арифметика

```cpp
__m256 c = _mm256_add_ps(a, b);        // c[i] = a[i] + b[i]
__m256 d = _mm256_mul_ps(a, b);        // d[i] = a[i] * b[i]
__m256 e = _mm256_sub_ps(a, b);        // e[i] = a[i] - b[i]
__m256 f = _mm256_div_ps(a, b);        // f[i] = a[i] / b[i] (slower than mul)

// Fused Multiply-Add: r = a * b + c (one instruction, one rounding)
__m256 r = _mm256_fmadd_ps(a, b, c);   // THE most important instruction for ML

// Min and max
__m256 mn = _mm256_min_ps(a, b);       // min(a[i], b[i]) — useful for clipping
__m256 mx = _mm256_max_ps(a, b);       // max(a[i], b[i]) — useful for ReLU
```

### Практический пример: скалярное произведение на AVX2

```cpp
#include <immintrin.h>

float dot_avx2(const float* a, const float* b, int n) {
    __m256 sum = _mm256_setzero_ps();  // 8 accumulators initialised to 0

    int i = 0;
    for (; i + 8 <= n; i += 8) {
        __m256 va = _mm256_loadu_ps(a + i);
        __m256 vb = _mm256_loadu_ps(b + i);
        sum = _mm256_fmadd_ps(va, vb, sum);  // sum += va * vb
    }

    // Horizontal reduction: sum the 8 elements of sum
    // Step 1: add upper 128 bits to lower 128 bits
    __m128 hi = _mm256_extractf128_ps(sum, 1);
    __m128 lo = _mm256_castps256_ps128(sum);
    __m128 sum128 = _mm_add_ps(hi, lo);        // 4 partial sums

    // Step 2: horizontal add within 128-bit register
    sum128 = _mm_hadd_ps(sum128, sum128);       // [a+b, c+d, a+b, c+d]
    sum128 = _mm_hadd_ps(sum128, sum128);       // [a+b+c+d, ...]

    float result = _mm_cvtss_f32(sum128);       // extract scalar

    // Scalar cleanup
    for (; i < n; i++) {
        result += a[i] * b[i];
    }

    return result;
}
```

- **Почему горизонтальная редукция выглядит некрасиво**: SIMD спроектирован для вертикальных операций (канал 0 с каналом 0, канал 1 с каналом 1). Горизонтальные операции (суммирование по каналам) противоречат архитектуре оборудования. Именно поэтому скалярные произведения имеют такой громоздкий код редукции в конце. Векторизованный цикл выглядит чисто, а редукция — это шаблонный код.

- **Производительность**: по сравнению с версией для NEON (файл 02), AVX2 обрабатывает 8 чисел float за итерацию против 4 у NEON. Для длинных векторов это дает двукратное ускорение по сравнению с NEON (без учета ограничений пропускной способности памяти).

### Практический пример: Softmax на AVX2 (упрощенно)

- Softmax требует: найти максимум, вычесть его, возвести в степень, просуммировать, разделить. Вот этап поиска максимума:

```cpp
float vector_max_avx2(const float* data, int n) {
    __m256 max_vec = _mm256_set1_ps(-INFINITY);

    int i = 0;
    for (; i + 8 <= n; i += 8) {
        __m256 v = _mm256_loadu_ps(data + i);
        max_vec = _mm256_max_ps(max_vec, v);
    }

    // Reduce 8 maxes to 1
    __m128 hi = _mm256_extractf128_ps(max_vec, 1);
    __m128 lo = _mm256_castps256_ps128(max_vec);
    __m128 max128 = _mm_max_ps(hi, lo);

    // Shuffle and max to find the single maximum
    max128 = _mm_max_ps(max128, _mm_shuffle_ps(max128, max128, 0b01001110));
    max128 = _mm_max_ps(max128, _mm_shuffle_ps(max128, max128, 0b10110001));

    float result = _mm_cvtss_f32(max128);

    for (; i < n; i++) {
        result = result > data[i] ? result : data[i];
    }

    return result;
}
```

- Инструкция `_mm_shuffle_ps` переупорядочивает элементы внутри регистра. Бинарная константа `0b01001110` управляет тем, какие элементы куда перемещаются. Это называется **перестановкой** (permutation), и она напрямую связана с матрицами перестановок (глава 2): перетасовка SIMD-линий — это аппаратный эквивалент умножения на матрицу перестановок.

## AVX-512

- AVX-512 снова удваивает ширину: 512-битные регистры (ZMM), обрабатывающие 16 чисел с плавающей запятой одновременно.

```cpp
__m512 a = _mm512_loadu_ps(ptr);                // load 16 floats
__m512 c = _mm512_fmadd_ps(a, b, c);            // 16 FMAs at once
float sum = _mm512_reduce_add_ps(a);             // built-in horizontal sum (no manual reduction!)

// Mask operations: operate on a subset of lanes
__mmask16 mask = _mm512_cmpgt_ps_mask(a, zero);  // which lanes are > 0?
__m512 relu = _mm512_maskz_mov_ps(mask, a);       // zero out negative lanes = ReLU
```

- **Масочные регистры** (`__mmask16`) — самая мощная функция AVX-512. Каждый бит управляет тем, участвует ли линия в операции. Это заменяет скалярный цикл очистки: последняя итерация использует маску, в которой активны только валидные линии, что позволяет обрабатывать любую длину вектора без отдельного скалярного цикла.

- **Троттлинг частоты AVX-512**: на многих процессорах Intel использование инструкций AVX-512 заставляет процессор временно снижать тактовую частоту (чтобы оставаться в пределах тепловых ограничений). Это означает, что AVX-512 не всегда быстрее, чем AVX2 при коротких нагрузках — штраф по частоте может перевесить преимущество более широких векторов. Для длительных нагрузок (таких как умножение матриц) AVX-512 выигрывает. Для смешанного кода (частично SIMD, частично скалярный) переключения частоты могут снижать производительность.

## Intel AMX: аппаратное умножение матриц

- **AMX** (Advanced Matrix eXtensions) добавляет специализированные блоки умножения матриц. Вместо SIMD-векторов AMX работает с **тайлами** (tiles): 2D-блоками данных (до 16 строк × 64 байта каждый).

```cpp
#include <immintrin.h>

// AMX tile multiply: C += A * B (in BF16)
// A is 16x32 BF16, B is 32x16 BF16, C is 16x16 FP32
_tile_loadd(0, a_ptr, stride_a);   // load tile 0 from A
_tile_loadd(1, b_ptr, stride_b);   // load tile 1 from B
_tile_dpbf16ps(2, 0, 1);           // tile 2 += tile 0 * tile 1 (BF16 matmul, FP32 accumulation)
_tile_stored(2, c_ptr, stride_c);  // store tile 2 to C
```

- AMX выполняет полное умножение матриц 16×32 × 32×16 за одну инструкцию. Это сотни операций FMA одновременно, специально разработанные для умножения небольших матриц, которые преобладают в инференсе трансформеров (вычисление оценок внимания, слои MLP).

- AMX поддерживает форматы BF16 (bfloat16) и INT8, соответствующие точности, используемой в инференсе моделей машинного обучения. В сочетании с AVX-512 для других операций процессоры с поддержкой AMX (Intel Sapphire Rapids, Emerald Rapids) могут выполнять инференс трансформеров на уровне, сопоставимом с GPU начального уровня.

## Выравнивание памяти

- **Выровненный доступ к памяти** — это когда адрес данных кратен ширине векторного регистра (16 байт для SSE, 32 байта для AVX, 64 байта для AVX-512). Выровненный доступ быстрее на некоторых процессорах и требуется для `_mm256_load_ps` (в отличие от `_mm256_loadu_ps`).

```cpp
// Allocate aligned memory
float* data = (float*)aligned_alloc(32, n * sizeof(float));  // 32-byte aligned for AVX

// C++ aligned allocation
#include <new>
float* data = new (std::align_val_t(32)) float[n];

// Alternatively, use the compiler attribute
alignas(32) float data[1024];
```

- **На практике**: на современных процессорах (Haswell и новее) невыровненные загрузки (`loadu`) почти так же быстры, как выровненные, если данные не пересекают границу строки кеша. Штраф за производительность при невыровненном доступе по большей части исчез, но разделение строк кеша (когда данные охватывают две 64-байтовые строки кеша) все еще может вызвать замедление примерно в 2 раза для этой конкретной операции загрузки. Выровненное выделение памяти позволяет полностью избежать этого.

## Проблемы производительности

- **Штраф при переходе AVX-SSE**: на старых процессорах Intel (до архитектуры Skylake) переключение между инструкциями AVX (256-бит) и SSE (128-бит) вызывало штраф (~70 тактов). Именно поэтому следует использовать `_mm256_zeroupper()` (или инструкцию `vzeroupper`) перед выходом из функции, использующей AVX, чтобы очистить верхние 128 бит регистров YMM. Современные процессоры (Skylake+) не имеют этого штрафа.

- **Регистровое давление**: AVX2 имеет 16 регистров YMM. Если ваше ядро использует слишком много переменных, компилятор вытесняет регистры в стек (память), уничтожая производительность. Старайтесь делать внутренние циклы простыми, с небольшим количеством активных переменных.

- **Зависимости данных**: `sum = _mm256_fmadd_ps(a, b, sum)` имеет зависимость от `sum`: каждая итерация должна ждать завершения предыдущей операции FMA (задержка ~4-5 тактов). Решение: используйте несколько независимых аккумуляторов и выполняйте редукцию в конце:

```cpp
// Single accumulator: limited by FMA latency (4-5 cycles)
__m256 sum = _mm256_setzero_ps();
for (...) {
    sum = _mm256_fmadd_ps(a, b, sum);  // each depends on previous
}

// Four accumulators: 4x throughput (hide latency)
__m256 sum0 = _mm256_setzero_ps();
__m256 sum1 = _mm256_setzero_ps();
__m256 sum2 = _mm256_setzero_ps();
__m256 sum3 = _mm256_setzero_ps();
for (...) {
    sum0 = _mm256_fmadd_ps(a0, b0, sum0);  // independent
    sum1 = _mm256_fmadd_ps(a1, b1, sum1);  // independent
    sum2 = _mm256_fmadd_ps(a2, b2, sum2);  // independent
    sum3 = _mm256_fmadd_ps(a3, b3, sum3);  // independent
}
sum0 = _mm256_add_ps(sum0, sum1);
sum2 = _mm256_add_ps(sum2, sum3);
sum0 = _mm256_add_ps(sum0, sum2);
```

- Это **развертывание цикла** (loop unrolling) для скрытия задержки. Процессор может выполнять операции FMA одну за другой, так как они записывают данные в разные регистры. Это одна из наиболее эффективных микрооптимизаций в численном коде.

## Профилирование

- **Счетчики производительности** предоставляют измерения на аппаратном уровне:

```bash
# Linux perf (requires kernel support)
perf stat ./my_program                    # basic counters: cycles, instructions, IPC
perf stat -e cache-misses,cache-references ./my_program  # cache behaviour
perf record -g ./my_program && perf report              # call graph profiling

# Intel VTune (detailed x86 profiling)
vtune -collect hotspots -- ./my_program
vtune -collect memory-access -- ./my_program   # memory bandwidth analysis
```

- **На что обратить внимание**:
    - **IPC** (Instructions Per Cycle — количество инструкций за цикл): насколько эффективно используется CPU. IPC > 2 — это хороший показатель. IPC < 1 указывает на простои из-за памяти или неверные предсказания переходов.
    - **Частота промахов кеша (Cache miss rate)**: высокая частота промахов L1/L2 указывает на плохую локальность данных. Перестройте шаблоны доступа к данным.
    - **Частота неверных предсказаний переходов (Branch misprediction rate)**: > 5% указывает на непредсказуемые переходы. По возможности замените их на бесусловный код (сравнения SIMD + смешивание).
    - **Достигнутые FLOPS в сравнении с моделью roofline**: сравните измеренные FLOPS с моделью roofline (файл 01). Если вы находитесь ниже линии roofline, есть потенциал для оптимизации.

## Задачи по программированию (компиляция с помощью g++ или clang++ на x86 — Intel/AMD)

1. Напишите скалярное произведение для скаляров и для AVX2. Проведите бенчмарк обоих вариантов и измерьте ускорение от использования 8-канального SIMD.
```cpp
// task1_avx_dot.cpp
// Compile: g++ -O3 -mavx2 -mfma -o task1 task1_avx_dot.cpp

#include <iostream>
#include <chrono>
#include <vector>
#include <immintrin.h>

float dot_scalar(const float* a, const float* b, int n) {
    float sum = 0.0f;
    for (int i = 0; i < n; i++) sum += a[i] * b[i];
    return sum;
}

float dot_avx2(const float* a, const float* b, int n) {
    __m256 sum = _mm256_setzero_ps();
    int i = 0;
    for (; i + 8 <= n; i += 8) {
        __m256 va = _mm256_loadu_ps(a + i);
        __m256 vb = _mm256_loadu_ps(b + i);
        sum = _mm256_fmadd_ps(va, vb, sum);
    }
    // Reduce: add upper 128 to lower 128, then horizontal add
    __m128 hi = _mm256_extractf128_ps(sum, 1);
    __m128 lo = _mm256_castps256_ps128(sum);
    __m128 r = _mm_add_ps(hi, lo);
    r = _mm_hadd_ps(r, r);
    r = _mm_hadd_ps(r, r);
    float result = _mm_cvtss_f32(r);
    for (; i < n; i++) result += a[i] * b[i];
    return result;
}

int main() {
    const int N = 10'000'000;
    std::vector<float> a(N, 1.0f), b(N, 2.0f);

    volatile float s1 = dot_scalar(a.data(), b.data(), N);
    volatile float s2 = dot_avx2(a.data(), b.data(), N);

    auto bench = [&](auto fn, const char* name) {
        auto start = std::chrono::high_resolution_clock::now();
        volatile float s;
        for (int t = 0; t < 100; t++) s = fn(a.data(), b.data(), N);
        auto end = std::chrono::high_resolution_clock::now();
        double ms = std::chrono::duration<double, std::milli>(end - start).count() / 100;
        std::cout << name << ": " << ms << " ms (result: " << s << ")\n";
        return ms;
    };

    double t1 = bench(dot_scalar, "Scalar");
    double t2 = bench(dot_avx2,   "AVX2  ");
    std::cout << "Speedup: " << t1 / t2 << "x\n";
    return 0;
}
```

2. Реализуйте ReLU на AVX2, используя `_mm256_max_ps`, и сравните со скалярным циклом. Затем попробуйте использовать несколько аккумуляторов (развертывание цикла), чтобы скрыть задержку FMA.
```cpp
// task2_avx_relu.cpp
// Compile: g++ -O3 -mavx2 -o task2 task2_avx_relu.cpp

#include <iostream>
#include <chrono>
#include <vector>
#include <immintrin.h>

void relu_scalar(const float* in, float* out, int n) {
    for (int i = 0; i < n; i++) {
        out[i] = in[i] > 0.0f ? in[i] : 0.0f;
    }
}

void relu_avx2(const float* in, float* out, int n) {
    __m256 zero = _mm256_setzero_ps();
    int i = 0;
    for (; i + 8 <= n; i += 8) {
        __m256 x = _mm256_loadu_ps(in + i);
        _mm256_storeu_ps(out + i, _mm256_max_ps(x, zero));
    }
    for (; i < n; i++) out[i] = in[i] > 0.0f ? in[i] : 0.0f;
}

// Unrolled: process 32 floats per iteration (4 x 8)
void relu_avx2_unrolled(const float* in, float* out, int n) {
    __m256 zero = _mm256_setzero_ps();
    int i = 0;
    for (; i + 32 <= n; i += 32) {
        __m256 x0 = _mm256_loadu_ps(in + i);
        __m256 x1 = _mm256_loadu_ps(in + i + 8);
        __m256 x2 = _mm256_loadu_ps(in + i + 16);
        __m256 x3 = _mm256_loadu_ps(in + i + 24);
        _mm256_storeu_ps(out + i,      _mm256_max_ps(x0, zero));
        _mm256_storeu_ps(out + i + 8,  _mm256_max_ps(x1, zero));
        _mm256_storeu_ps(out + i + 16, _mm256_max_ps(x2, zero));
        _mm256_storeu_ps(out + i + 24, _mm256_max_ps(x3, zero));
    }
    for (; i + 8 <= n; i += 8) {
        _mm256_storeu_ps(out + i, _mm256_max_ps(_mm256_loadu_ps(in + i), zero));
    }
    for (; i < n; i++) out[i] = in[i] > 0.0f ? in[i] : 0.0f;
}

int main() {
    const int N = 16'000'000;
    std::vector<float> in(N), out(N);
    for (int i = 0; i < N; i++) in[i] = (float)(i % 200) - 100.0f;

    auto bench = [&](auto fn, const char* name) {
        fn(in.data(), out.data(), N);  // warm up
        auto start = std::chrono::high_resolution_clock::now();
        for (int t = 0; t < 100; t++) fn(in.data(), out.data(), N);
        auto end = std::chrono::high_resolution_clock::now();
        double ms = std::chrono::duration<double, std::milli>(end - start).count() / 100;
        double bw = 2.0 * N * sizeof(float) / ms / 1e6;  // read + write
        std::cout << name << ": " << ms << " ms (" << bw << " GB/s)\n";
    };

    bench(relu_scalar,        "Scalar        ");
    bench(relu_avx2,          "AVX2          ");
    bench(relu_avx2_unrolled, "AVX2 unrolled ");
    return 0;
}
```

3. Измерьте влияние выравнивания памяти. Сравните выровненные и невыровненные загрузки на больших массивах.
```cpp
// task3_alignment.cpp
// Compile: g++ -O3 -mavx2 -o task3 task3_alignment.cpp

#include <iostream>
#include <chrono>
#include <cstdlib>
#include <immintrin.h>

int main() {
    const int N = 16'000'000;

    // Aligned allocation (32-byte for AVX2)
    float* aligned = (float*)aligned_alloc(32, N * sizeof(float));

    // Unaligned: offset by 4 bytes (1 float) from aligned boundary
    float* raw = (float*)malloc((N + 1) * sizeof(float));
    float* unaligned = raw + 1;  // guaranteed misaligned

    for (int i = 0; i < N; i++) {
        aligned[i] = 1.0f;
        unaligned[i] = 1.0f;
    }

    auto bench = [&](float* ptr, bool use_aligned, const char* name) {
        __m256 sum = _mm256_setzero_ps();
        // Warm up
        for (int i = 0; i + 8 <= N; i += 8) {
            __m256 v = use_aligned ? _mm256_load_ps(ptr + i) : _mm256_loadu_ps(ptr + i);
            sum = _mm256_add_ps(sum, v);
        }

        auto start = std::chrono::high_resolution_clock::now();
        for (int t = 0; t < 100; t++) {
            sum = _mm256_setzero_ps();
            for (int i = 0; i + 8 <= N; i += 8) {
                __m256 v = use_aligned ? _mm256_load_ps(ptr + i) : _mm256_loadu_ps(ptr + i);
                sum = _mm256_add_ps(sum, v);
            }
        }
        auto end = std::chrono::high_resolution_clock::now();
        double ms = std::chrono::duration<double, std::milli>(end - start).count() / 100;
        double bw = (double)N * sizeof(float) / ms / 1e6;
        std::cout << name << ": " << ms << " ms (" << bw << " GB/s)\n";
    };

    bench(aligned,   true,  "Aligned load  ");
    bench(unaligned, false, "Unaligned load");

    free(aligned);
    free(raw);
    return 0;
}
```
