# ARM 与 NEON

*ARM 处理器驱动着每一部智能手机、大多数平板电脑、Apple 的笔记本电脑，以及日益增长的数据中心服务器份额。本文件涵盖 ARM 架构、使用 C++ intrinsics 的 NEON SIMD 编程、面向可缩放向量处理的 SVE/SVE2、Apple Silicon 的特性，以及实际向量化 kernel 示例*

- 如果你拥有 iPhone、MacBook，或使用 AWS Graviton 实例，你就在运行 ARM。ARM 的能效优势使其在移动和嵌入式领域占据主导地位，并在服务器与 ML 推理中日益具备竞争力。理解 ARM SIMD 能让你在大多数人实际使用的硬件上写出更快的代码。

- 关于生产环境中 ARM SIMD kernel 的真实案例，参见 **Cactus**——一个面向移动设备与可穿戴设备的低 latency AI 引擎：[github.com/cactus-compute/cactus](https://github.com/cactus-compute/cactus)。Cactus 实现了自定义的 ARM NEON 与 NPU 加速 kernel，用于 attention、KV-cache 量化与分块 prefill，在 ARM CPU 上以比其他引擎低 10 倍的 RAM 实现最快的推理。其三层架构（Engine → Graph → Kernels）是本文件中 SIMD 概念如何被用于构建生产级 ML 基础设施的具体示例。

## ARM 架构基础

- ARM 是一种 **RISC**（Reduced Instruction Set Computer，精简指令集计算机）架构（第 13 章）。关键特性：

    - **Load-store 架构**：算术指令只在 register 上操作，绝不直接访问内存。要把内存中的两个数相加，你必须：(1) 将它们 load 到 register，(2) 将 register 相加，(3) 将结果 store 回内存。这比 x86（一条指令就能完成 register 与内存位置相加）更简单，但能实现更干净的流水线。

    - **定宽指令**：每条 ARMv8（AArch64）指令正好 32 位。这让译码快速且可预测（不像 x86 的变长指令，长度为 1-15 字节）。

    - **32 个通用 register**（x0-x30，每个 64 位），外加 stack pointer（sp）和 zero register（xzr）。对比 x86 的 16 个通用 register。register 更多 = 内存访问更少 = 代码更快。

    - **32 个 SIMD/浮点 register**（v0-v31，每个 128 位），用于 NEON 与浮点运算。

```cpp
// ARM 汇编（仅用于感受风格——你将使用 intrinsics，而非汇编）
// 两个 register 相加
add x0, x1, x2    // x0 = x1 + x2

// 从内存加载
ldr x0, [x1]      // x0 = *x1（从 x1 中的地址加载 64 位）

// NEON：四个 float 相加
fadd v0.4s, v1.4s, v2.4s  // v0 = v1 + v2（四个 32 位 float）
```

- 你不会写汇编。你将使用 **intrinsics**：与特定指令一一对应的 C/C++ 函数。编译器负责 register 分配、调度以及其他底层细节。

## NEON：128 位 SIMD

- **NEON** 是 ARM 的 SIMD 扩展。每个 NEON register 宽 128 位，可容纳：

| 数据类型 | 每 register 元素数 | 记法 |
|-----------|-----------------------|----------|
| float32 | 4 | `float32x4_t` |
| float16 | 8 | `float16x8_t` |
| int32 | 4 | `int32x4_t` |
| int16 | 8 | `int16x8_t` |
| int8 | 16 | `int8x16_t` |

- 128 位比 x86 的 AVX（256 位）或 AVX-512（512 位）更窄。但 ARM 以出色的能效和广泛的可用性来弥补。

### NEON Intrinsics：基础

- NEON intrinsics 遵循命名约定：`v[operation][qualifier]_[type]`

```cpp
#include <arm_neon.h>

// 从内存加载 4 个 float 到 NEON register
float32x4_t a = vld1q_f32(ptr);        // vld1q = vector load 1，q = 128 位（quad）

// 将 4 个 float 从 NEON register 存到内存
vst1q_f32(out_ptr, a);                   // vst1q = vector store 1，q = 128 位

// 算术
float32x4_t c = vaddq_f32(a, b);        // c = a + b（4 个 float）
float32x4_t d = vmulq_f32(a, b);        // d = a * b（4 个 float）
float32x4_t e = vfmaq_f32(c, a, b);     // e = c + a * b（融合乘加，4 个 float）

// 比较（返回掩码：真为全 1，假为全 0）
uint32x4_t mask = vcgtq_f32(a, b);      // mask[i] = (a[i] > b[i]) ? 0xFFFFFFFF : 0

// 根据掩码选择元素（类似 numpy.where）
float32x4_t result = vbslq_f32(mask, a, b);  // result[i] = mask[i] ? a[i] : b[i]

// 归约：将 4 个元素求和为一个标量
float total = vaddvq_f32(a);             // total = a[0] + a[1] + a[2] + a[3]
```

- **`vfmaq_f32`**（融合乘加，fused multiply-add）是 ML 中最重要的 SIMD 指令。它在一条指令中用单次舍入步骤计算 $c = c + a \times b$（比先乘后加更精确）。点积、矩阵乘法和卷积都由 FMA 构成。

### 实战示例：向量化点积

- 点积是矩阵乘法的内层循环。我们先用标量 C++ 实现，再用 NEON 向量化。

```cpp
#include <arm_neon.h>

// 标量点积
float dot_scalar(const float* a, const float* b, int n) {
    float sum = 0.0f;
    for (int i = 0; i < n; i++) {
        sum += a[i] * b[i];
    }
    return sum;
}

// NEON 向量化点积
float dot_neon(const float* a, const float* b, int n) {
    float32x4_t sum_vec = vdupq_n_f32(0.0f);  // 将 4 个累加器初始化为 0

    int i = 0;
    for (; i + 4 <= n; i += 4) {
        float32x4_t va = vld1q_f32(a + i);     // 从 a 加载 4 个元素
        float32x4_t vb = vld1q_f32(b + i);     // 从 b 加载 4 个元素
        sum_vec = vfmaq_f32(sum_vec, va, vb);   // sum_vec += va * vb
    }

    // 将 4 个累加器归约为单个标量
    float sum = vaddvq_f32(sum_vec);

    // 处理剩余元素（若 n 不是 4 的倍数）
    for (; i < n; i++) {
        sum += a[i] * b[i];
    }

    return sum;
}
```

- **关键 C++ 概念**：
    - `const float*`：指向只读 float 数据的指针。`const` 承诺我们不会通过此指针修改数据。
    - `a + i`：指针算术。`a + i` 指向数组第 $i$ 个元素（等同于 `&a[i]`）。
    - 末尾的“清理循环”处理 $n$ 不是 4 的倍数的情况。这是 SIMD 代码中的通用模式：主体用向量化块处理，剩余部分用标量代码处理。

- **为什么 `sum_vec` 中有 4 个累加器**：我们不用单个标量累加器，而用 4 个独立的累加器（每个 SIMD lane 一个）。这避免了数据依赖：每次迭代的 FMA 都依赖 `sum_vec`，但有 4 个独立的 lane，CPU 可以对 FMA 做流水线。最后，我们将 4 个部分和归约为一个。

### 实战示例：向量化 ReLU

```cpp
#include <arm_neon.h>

void relu_neon(const float* input, float* output, int n) {
    float32x4_t zero = vdupq_n_f32(0.0f);

    int i = 0;
    for (; i + 4 <= n; i += 4) {
        float32x4_t x = vld1q_f32(input + i);
        float32x4_t result = vmaxq_f32(x, zero);  // max(x, 0) = ReLU
        vst1q_f32(output + i, result);
    }

    // 标量清理
    for (; i < n; i++) {
        output[i] = input[i] > 0 ? input[i] : 0;
    }
}
```

- `vmaxq_f32` 计算两个向量的逐元素最大值。由于其中一个向量全为 0，这恰好是 ReLU。没有分支，没有比较——只有一条指令。

## I8MM：整数矩阵乘法

- **I8MM**（Int8 Matrix Multiply，Int8 矩阵乘法）是 ARMv8.6 扩展，新增了用于 INT8 矩阵乘法并带 INT32 累加的专用指令——正是量化 ML 推理所需要的。

- 关键指令是 **`SMMLA`**（Signed Matrix Multiply-Accumulate，有符号矩阵乘累加）：它取两个 8×2 的 INT8 值块，将结果累加到一个 2×2 的 INT32 块：

```cpp
#include <arm_neon.h>

// I8MM：两个 8 元素 INT8 向量相乘，累加到 4 个 INT32 结果
// 这从 2x8 x 8x2 的输入 tile 计算输出矩阵的一个 2x2 tile
void matmul_i8mm_tile(const int8_t* A, const int8_t* B, int32_t* C) {
    // 从 A 加载 8 字节（2 行 × 4 元素，紧凑排列）
    int8x16_t va = vld1q_s8(A);   // 16 字节 = 2 行 × 8 元素
    int8x16_t vb = vld1q_s8(B);   // 16 字节 = 2 行 × 8 元素

    // 加载现有累加器（2x2 = 4 个 int32 值）
    int32x4_t acc = vld1q_s32(C);

    // I8MM 指令：acc += A_tile × B_tile^T
    // 从 2×8 × 8×2 的输入计算 2×2 输出
    acc = vmmlaq_s32(acc, va, vb);  // 即 I8MM 指令

    vst1q_s32(C, acc);
}
```

- **I8MM 为何重要**：没有 I8MM，NEON 上的 INT8 矩阵乘法需要扩展乘法（`vmull`）再加成对加法——每个输出元素多条指令。有了 I8MM，硬件在单条指令中完成 8 元素点积（2×8 × 8×2 = 2×2）。对 INT8 推理工作负载，这比纯 NEON 快 4-8 倍。

- **可用性**：Apple M1+（所有 Apple Silicon）、ARM Cortex-A510/A710/X2+（ARMv9）、AWS Graviton3+。通过 `#ifdef __ARM_FEATURE_MATMUL_INT8` 检测。

- 对 ML 推理：在 ARM 服务器（Graviton）或 Apple Silicon 上运行的 INT8 量化模型（第 18 章）从 I8MM 中受益巨大。ONNX Runtime 和 llama.cpp 等框架在运行时检测 I8MM 并自动使用优化 kernel。

## SME 与 SME2：可缩放矩阵扩展

- **SME**（Scalable Matrix Extension，可缩放矩阵扩展）是 ARM 对 Intel AMX 和 NVIDIA Tensor Cores 的回应：用于矩阵运算的专用硬件。SME2（ARMv9.2）进一步扩展了它。

- SME 引入了 **ZA tile register**：存储在硬件中的 2D 矩阵，最大 SVL×SVL 字节（其中 SVL 是 streaming vector length，流式向量长度，通常每维 128-512 位）。与 NEON（1D 向量）甚至 SVE（1D 可缩放向量）不同，SVE 原生地对 **2D tile** 操作。

- 编程模型有两种模式：
    - **Normal 模式**：标准 ARM 执行（NEON、SVE 照常工作）。
    - **Streaming SVE 模式**：通过 `smstart` 进入，启用 SME 指令。SVE 指令在此模式下也能工作，但可能使用不同的 register 宽度。

```cpp
#include <arm_sme.h>

// SME2：用于矩阵乘法的外积累加
// 将 A_col × B_row 累加到 ZA tile register
void sme2_matmul_outer(const float* A_col, const float* B_row, int K) {
    // 进入 streaming 模式
    // smstart;  //（通过编译器 intrinsic 或 inline asm 完成）

    // 将 ZA tile 累加器清零
    svzero_za();

    for (int k = 0; k < K; k++) {
        // 将 A 的一列和 B 的一行加载到 SVE register
        svfloat32_t a = svld1_f32(svptrue_b32(), &A_col[k * SVL]);
        svfloat32_t b = svld1_f32(svptrue_b32(), &B_row[k * SVL]);

        // 外积：ZA += a × b^T
        // 在一条指令中将一个 SVL×SVL tile 累加
        svmopa_za32_f32_m(0, svptrue_b32(), svptrue_b32(), a, b);
    }

    // 将 ZA tile 存到内存
    // svst1_za(...);

    // 退出 streaming 模式
    // smstop;
}
```

- **关键概念**：
    - **`svmopa`**（outer product accumulate，外积累加）：SME 的核心指令。它计算两个向量的完整外积并累加到 ZA tile。对 SVL=512 位（16 个 float），这是一个 16×16 的外积——一条指令中完成 256 次 FMA 运算。
    - **ZA tile**：在 streaming 模式内跨指令持久存在。你将多个外积（每个 K 迭代一次）累加到同一 tile，逐步构建出完整的矩阵乘法 tile。
    - **Streaming 模式**：SME 指令只在 streaming 模式下工作。进入/退出 streaming 模式的开销意味着 SME 最适合持续的矩阵计算，而非短促的突发。

- **SME2 新增**：多向量运算（同时处理 2 个或 4 个 SVE 向量）、额外的 tile 运算，以及与 normal 模式更好的集成。

- **可用性**：ARM Neoverse V2（AWS Graviton4）、部分即将发布的移动芯片。截至 2026 年尚未出现在 Apple Silicon 上。SME 仍处于早期阶段——大多数 ML 框架尚无 SME 优化 kernel。

- **演进路线**：NEON（128 位向量，逐元素）→ I8MM（INT8 矩阵 tile）→ SVE（可缩放向量）→ SME（可缩放 2D 矩阵 tile）。每一代都向硬件中原生矩阵运算更近一步。

## SVE 与 SVE2：可缩放向量扩展

- NEON 具有固定的 128 位宽度。**SVE**（Scalable Vector Extension，可缩放向量扩展）引入了 **vector-length agnostic (VLA)** 编程：代码写一次，即可在任何向量宽度（128 到 2048 位）的硬件上运行。宽度由硬件在运行时决定。

```cpp
#include <arm_sve.h>

void add_sve(const float* a, const float* b, float* c, int n) {
    int i = 0;
    svbool_t pred = svwhilelt_b32(i, n);  // 谓词：哪些 lane 处于活跃

    while (svptest_any(svptrue_b32(), pred)) {
        svfloat32_t va = svld1(pred, a + i);
        svfloat32_t vb = svld1(pred, b + i);
        svst1(pred, c + i, svadd_x(pred, va, vb));

        i += svcntw();  // 按硬件向量宽度（以 32 位元素计）前进
        pred = svwhilelt_b32(i, n);
    }
}
```

- **谓词 register**（`svbool_t`）取代了标量清理循环。每个 lane 有一个谓词位：活跃 lane 参与，非活跃 lane 被屏蔽。`svwhilelt_b32(i, n)` 指令创建一个谓词，使对应 `i, i+1, ..., n-1` 的 lane 活跃。这自动处理尾部。

- **`svcntw()`** 在运行时返回每个向量 register 中 32 位元素的数量。在 256 位 SVE 的 CPU 上返回 8；在 512 位 SVE 上返回 16。你的代码会自动适配。

- SVE 可用于 ARM Neoverse V1/V2（AWS Graviton3/4、部分服务器芯片）。尚未在 Apple Silicon 上提供。

## Apple Silicon 特性

- Apple 的 M 系列芯片（M1、M2、M3、M4）基于 ARM，带有自定义微架构：

- **性能核与能效核**：P-core（Firestorm/Avalanche 等）用于重负载计算，E-core（Icestorm/Blizzard 等）用于后台任务。调度器将 thread 分配到合适的核类型。

- **AMX**（Apple Matrix eXtensions，Apple 矩阵扩展）：专用矩阵乘法单元，独立于 NEON。AMX 未公开文档化（Apple 不发布其 ISA），但 Accelerate 框架在内部使用它进行 BLAS 运算。在 Mac 上调用 `np.dot` 时，它经由 Accelerate，后者使用 AMX。你无法直接对 AMX 编程（除非逆向工程）。

- **统一内存**：CPU 与 GPU 共享同一物理 RAM。在其他系统上，数据必须从 CPU 内存复制到 GPU 内存（经 PCIe，约 32 GB/s）。在 Apple Silicon 上，无需复制——GPU 直接读取 CPU 写入的同一内存。这消除了 ML 工作负载中的一个主要瓶颈。

- **Neural Engine**：16 核专用 ML 加速器。对 INT8 推理可达约 30 TOPS（每秒万亿次运算）。Core ML 用其进行设备端推理。

- **在 Apple Silicon 上做 ML**：使用 MLX（Apple 的 ML 框架），它为统一内存架构而设计。PyTorch 也有 MPS（Metal Performance Shaders）后端支持，但不如 CUDA 成熟。

## 自动向量化

- 写 SIMD intrinsics 很繁琐。**编译器**能否自动向量化你的代码？

- 可以，但有条件。现代编译器（GCC、Clang）能自动向量化简单循环：

```cpp
// 编译器可以自动向量化此循环（加 -O3 -march=native）
void add_auto(const float* a, const float* b, float* c, int n) {
    for (int i = 0; i < n; i++) {
        c[i] = a[i] + b[i];
    }
}
```

- **有助于自动向量化的模式**：
    - 已知迭代次数的简单循环。
    - 迭代间无数据依赖（`c[i]` 不依赖 `c[i-1]`）。
    - 连续内存访问（无 scatter/gather）。
    - `const` 与 `restrict` 指针（告知编译器数组不重叠）。

```cpp
// restrict 告知编译器：a、b、c 指向不重叠的内存
void add_restrict(const float* __restrict__ a,
                  const float* __restrict__ b,
                  float* __restrict__ c, int n) {
    for (int i = 0; i < n; i++) {
        c[i] = a[i] + b[i];
    }
}
```

- 没有 `restrict`，编译器必须假定 `c` 可能与 `a` 或 `b` 重叠（写 `c[i]` 可能改变 `a[i+1]`），从而阻止向量化。

- **阻止自动向量化的模式**：
    - 数据依赖：`a[i] = a[i-1] + b[i]`（每次迭代依赖上一次）。
    - 复杂控制流：循环内有 `if` 语句（除非编译器能转为 predication）。
    - 循环内有函数调用（除非函数被内联）。
    - 指针别名（数组可能重叠，且无 `restrict`）。

- **检查自动向量化**：使用编译器标志查看向量化情况：

```bash
# GCC：显示向量化决策
g++ -O3 -march=native -fopt-info-vec-optimized code.cpp

# Clang：显示向量化报告
clang++ -O3 -march=native -Rpass=loop-vectorize code.cpp
```

- **何时使用 intrinsics vs 自动向量化**：从干净的 C++ 与编译器优化开始。若编译器已向量化你的循环，很好。若性能仍不足，检查编译器的向量化报告以理解原因，然后再为关键内层循环编写 intrinsics。过早使用 intrinsics 会使代码不可读且无确定的收益。

## 编程任务（在 ARM 上用 g++ 或 clang++ 编译——Mac M 系列或 Linux aarch64）

1. 编写标量点积与 NEON 向量化点积。对两者 benchmark 并测量加速比。
```cpp
// task1_neon_dot.cpp
// 编译（Mac/ARM Linux）：clang++ -O3 -o task1 task1_neon_dot.cpp
// 注：AArch64 上默认启用 NEON，无需特殊标志

#include <iostream>
#include <chrono>
#include <vector>
#include <arm_neon.h>

float dot_scalar(const float* a, const float* b, int n) {
    float sum = 0.0f;
    for (int i = 0; i < n; i++) {
        sum += a[i] * b[i];
    }
    return sum;
}

float dot_neon(const float* a, const float* b, int n) {
    float32x4_t sum_vec = vdupq_n_f32(0.0f);
    int i = 0;
    for (; i + 4 <= n; i += 4) {
        float32x4_t va = vld1q_f32(a + i);
        float32x4_t vb = vld1q_f32(b + i);
        sum_vec = vfmaq_f32(sum_vec, va, vb);
    }
    float sum = vaddvq_f32(sum_vec);
    for (; i < n; i++) sum += a[i] * b[i];
    return sum;
}

int main() {
    const int N = 10'000'000;
    std::vector<float> a(N, 1.0f), b(N, 2.0f);

    // 预热
    volatile float s1 = dot_scalar(a.data(), b.data(), N);
    volatile float s2 = dot_neon(a.data(), b.data(), N);

    // 标量 benchmark
    auto start = std::chrono::high_resolution_clock::now();
    for (int t = 0; t < 100; t++) {
        s1 = dot_scalar(a.data(), b.data(), N);
    }
    auto end = std::chrono::high_resolution_clock::now();
    double scalar_ms = std::chrono::duration<double, std::milli>(end - start).count() / 100;

    // NEON benchmark
    start = std::chrono::high_resolution_clock::now();
    for (int t = 0; t < 100; t++) {
        s2 = dot_neon(a.data(), b.data(), N);
    }
    end = std::chrono::high_resolution_clock::now();
    double neon_ms = std::chrono::duration<double, std::milli>(end - start).count() / 100;

    std::cout << "Scalar: " << scalar_ms << " ms (result: " << s1 << ")\n";
    std::cout << "NEON:   " << neon_ms << " ms (result: " << s2 << ")\n";
    std::cout << "Speedup: " << scalar_ms / neon_ms << "x\n";
    return 0;
}
```

2. 实现 NEON ReLU 与 softmax 求最大值。用不同操作练习 load→compute→store 模式。
```cpp
// task2_neon_ops.cpp
// 编译：clang++ -O3 -o task2 task2_neon_ops.cpp

#include <iostream>
#include <vector>
#include <cmath>
#include <arm_neon.h>

void relu_neon(const float* in, float* out, int n) {
    float32x4_t zero = vdupq_n_f32(0.0f);
    int i = 0;
    for (; i + 4 <= n; i += 4) {
        float32x4_t x = vld1q_f32(in + i);
        vst1q_f32(out + i, vmaxq_f32(x, zero));
    }
    for (; i < n; i++) out[i] = in[i] > 0 ? in[i] : 0;
}

float max_neon(const float* data, int n) {
    float32x4_t max_vec = vdupq_n_f32(-INFINITY);
    int i = 0;
    for (; i + 4 <= n; i += 4) {
        max_vec = vmaxq_f32(max_vec, vld1q_f32(data + i));
    }
    float result = vmaxvq_f32(max_vec);
    for (; i < n; i++) result = result > data[i] ? result : data[i];
    return result;
}

int main() {
    std::vector<float> data = {-3, 1, -1, 4, 2, -5, 0, 7, -2, 3};
    std::vector<float> out(data.size());

    relu_neon(data.data(), out.data(), data.size());
    std::cout << "ReLU: ";
    for (float x : out) std::cout << x << " ";
    std::cout << "\n";

    float mx = max_neon(data.data(), data.size());
    std::cout << "Max: " << mx << " (expected: 7)\n";
    return 0;
}
```

3. 将自动向量化代码与手写 NEON intrinsics 对比。用 `-fopt-info-vec`（GCC）或 `-Rpass=loop-vectorize`（Clang）编译，查看编译器做了什么。
```cpp
// task3_auto_vs_manual.cpp
// 编译：clang++ -O3 -Rpass=loop-vectorize -o task3 task3_auto_vs_manual.cpp
//    （或）：g++ -O3 -fopt-info-vec-optimized -o task3 task3_auto_vs_manual.cpp

#include <iostream>
#include <chrono>
#include <vector>
#include <arm_neon.h>

// 让编译器自动向量化
void add_auto(const float* __restrict__ a, const float* __restrict__ b,
              float* __restrict__ c, int n) {
    for (int i = 0; i < n; i++) {
        c[i] = a[i] + b[i];
    }
}

// 手写 NEON
void add_neon(const float* a, const float* b, float* c, int n) {
    int i = 0;
    for (; i + 4 <= n; i += 4) {
        vst1q_f32(c + i, vaddq_f32(vld1q_f32(a + i), vld1q_f32(b + i)));
    }
    for (; i < n; i++) c[i] = a[i] + b[i];
}

int main() {
    const int N = 10'000'000;
    std::vector<float> a(N, 1.0f), b(N, 2.0f), c(N);

    auto bench = [&](auto fn, const char* name) {
        fn(a.data(), b.data(), c.data(), N);  // 预热
        auto start = std::chrono::high_resolution_clock::now();
        for (int t = 0; t < 100; t++) fn(a.data(), b.data(), c.data(), N);
        auto end = std::chrono::high_resolution_clock::now();
        double ms = std::chrono::duration<double, std::milli>(end - start).count() / 100;
        std::cout << name << ": " << ms << " ms\n";
    };

    bench(add_auto, "Auto-vectorised");
    bench(add_neon, "Hand-written NEON");
    // 它们应该非常接近——编译器能很好地自动向量化这种简单循环
    return 0;
}
```
