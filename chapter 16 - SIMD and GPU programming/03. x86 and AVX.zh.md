# x86 与 AVX

*来自 Intel 和 AMD 的 x86 处理器主导着数据中心服务器，大多数 ML 训练都在其上发生。本文件涵盖 x86 SIMD 的演进、AVX/AVX2 intrinsics 编程、AVX-512、用于矩阵运算的 Intel AMX、内存对齐、性能陷阱与 profiling——即从世界最常见的服务器 CPU 中榨取最大性能所需的工具。*

- 如果你的训练跑在云 VM（AWS、GCP、Azure）上，那几乎肯定是 x86。即使是 GPU 重负载的训练也有 CPU 瓶颈：数据加载、预处理、梯度聚合与 checkpointing 都在 CPU 上运行。用 x86 SIMD 优化这些环节可以显著降低端到端训练时间。

## x86 SIMD 的演进

- x86 SIMD 通过越来越宽的向量 register 演进：

| 代际 | 年份 | Register 宽度 | Register 数 | 关键特性 |
|------------|------|---------------|-----------|--------------|
| MMX | 1997 | 64 位 | 8 (mm0-7) | 仅整数，与 FPU 共享 |
| SSE | 1999 | 128 位 | 8 (xmm0-7) | 4 个 float，专用 register |
| SSE2 | 2001 | 128 位 | 8/16 | 2 个 double，整数运算 |
| AVX | 2011 | 256 位 | 16 (ymm0-15) | 8 个 float，3 操作数指令 |
| AVX2 | 2013 | 256 位 | 16 | 256 位整数，FMA，gather |
| AVX-512 | 2017 | 512 位 | 32 (zmm0-31) | 16 个 float，掩码 register，scatter |
| AMX | 2023 | Tile register | 8 个 tile | 矩阵乘法（BF16、INT8） |

- 每一代都让向量化代码的吞吐量翻倍。用 SSE intrinsics 写的代码可在 2001 年以来的任何 x86 CPU 上运行。AVX2 需要 2013+ 的 CPU。AVX-512 用于 Intel Xeon 与部分消费级芯片。AMX 是最新的（Sapphire Rapids 及之后）。

- **向后兼容**：x86 SSE register（xmm）是 AVX register（ymm）的低 128 位，而后者又是 AVX-512 register（zmm）的低 256 位。旧的 SSE 代码无需修改即可在新 CPU 上运行。

## AVX2 编程

- AVX2 操作 256 位 register（YMM），同时处理 8 个 float 或 4 个 double。它是可移植高性能代码的最佳选择：几乎所有现代 x86 CPU（2013+）都支持。

### Intrinsics 命名约定

- 所有 x86 intrinsics 遵循模式：`_mm[width]_[operation]_[type]`

    - `_mm` = MMX/SSE（128 位），`_mm256` = AVX（256 位），`_mm512` = AVX-512（512 位）
    - 操作：`add`、`mul`、`fmadd`、`load`、`store`、`set` 等
    - 类型：`ps` = packed single（float32），`pd` = packed double（float64），`epi32` = packed int32，`si256` = 256 位整数

```cpp
#include <immintrin.h>  // 所有 x86 SIMD intrinsics

// 数据类型
__m256  a;   // 持有 8 个 float32 的 256 位 register
__m256d b;   // 持有 4 个 float64 的 256 位 register
__m256i c;   // 持有整数（8x32、16x16 或 32x8）的 256 位 register
```

### 加载与存储数据

```cpp
// 从内存加载 8 个 float
__m256 v = _mm256_loadu_ps(ptr);      // 非对齐加载（任意地址可用）
__m256 v = _mm256_load_ps(ptr);       // 对齐加载（ptr 必须 32 字节对齐，更快）

// 将 8 个 float 存到内存
_mm256_storeu_ps(out_ptr, v);          // 非对齐存储
_mm256_store_ps(out_ptr, v);           // 对齐存储

// 将单个值广播到全部 8 个 lane
__m256 ones = _mm256_set1_ps(1.0f);    // [1, 1, 1, 1, 1, 1, 1, 1]

// 设置单个值（很少需要）
__m256 v = _mm256_set_ps(7,6,5,4,3,2,1,0);  // 注：顺序是反的！

// 清零 register
__m256 z = _mm256_setzero_ps();
```

### 算术

```cpp
__m256 c = _mm256_add_ps(a, b);        // c[i] = a[i] + b[i]
__m256 d = _mm256_mul_ps(a, b);        // d[i] = a[i] * b[i]
__m256 e = _mm256_sub_ps(a, b);        // e[i] = a[i] - b[i]
__m256 f = _mm256_div_ps(a, b);        // f[i] = a[i] / b[i]（比 mul 慢）

// 融合乘加：r = a * b + c（一条指令，一次舍入）
__m256 r = _mm256_fmadd_ps(a, b, c);   // ML 中最重要的指令

// 最小与最大
__m256 mn = _mm256_min_ps(a, b);       // min(a[i], b[i]) — 用于截断
__m256 mx = _mm256_max_ps(a, b);       // max(a[i], b[i]) — 用于 ReLU
```

### 实战示例：AVX2 点积

```cpp
#include <immintrin.h>

float dot_avx2(const float* a, const float* b, int n) {
    __m256 sum = _mm256_setzero_ps();  // 8 个累加器初始化为 0

    int i = 0;
    for (; i + 8 <= n; i += 8) {
        __m256 va = _mm256_loadu_ps(a + i);
        __m256 vb = _mm256_loadu_ps(b + i);
        sum = _mm256_fmadd_ps(va, vb, sum);  // sum += va * vb
    }

    // 水平归约：将 sum 的 8 个元素求和
    // 第 1 步：高 128 位加到低 128 位
    __m128 hi = _mm256_extractf128_ps(sum, 1);
    __m128 lo = _mm256_castps256_ps128(sum);
    __m128 sum128 = _mm_add_ps(hi, lo);        // 4 个部分和

    // 第 2 步：128 位 register 内水平相加
    sum128 = _mm_hadd_ps(sum128, sum128);       // [a+b, c+d, a+b, c+d]
    sum128 = _mm_hadd_ps(sum128, sum128);       // [a+b+c+d, ...]

    float result = _mm_cvtss_f32(sum128);       // 提取标量

    // 标量清理
    for (; i < n; i++) {
        result += a[i] * b[i];
    }

    return result;
}
```

- **为什么水平归约如此丑陋**：SIMD 为垂直运算而设计（lane 0 与 lane 0、lane 1 与 lane 1）。水平运算（跨 lane 求和）与硬件相悖。这就是点积末尾需要那段别扭归约代码的原因。向量化循环很干净；归约是样板。

- **性能**：与 NEON 版本（文件 02）相比，AVX2 每次迭代处理 8 个 float，而 NEON 处理 4 个。对长向量而言，这相对 NEON 是 2 倍加速（忽略内存带宽限制）。

### 实战示例：AVX2 Softmax（简化版）

- Softmax 需要：求最大值、减去它、取指数、求和、相除。下面是求最大值的步骤：

```cpp
float vector_max_avx2(const float* data, int n) {
    __m256 max_vec = _mm256_set1_ps(-INFINITY);

    int i = 0;
    for (; i + 8 <= n; i += 8) {
        __m256 v = _mm256_loadu_ps(data + i);
        max_vec = _mm256_max_ps(max_vec, v);
    }

    // 将 8 个最大值归约为 1 个
    __m128 hi = _mm256_extractf128_ps(max_vec, 1);
    __m128 lo = _mm256_castps256_ps128(max_vec);
    __m128 max128 = _mm_max_ps(hi, lo);

    // shuffle 与 max 以找出唯一最大值
    max128 = _mm_max_ps(max128, _mm_shuffle_ps(max128, max128, 0b01001110));
    max128 = _mm_max_ps(max128, _mm_shuffle_ps(max128, max128, 0b10110001));

    float result = _mm_cvtss_f32(max128);

    for (; i < n; i++) {
        result = result > data[i] ? result : data[i];
    }

    return result;
}
```

- `_mm_shuffle_ps` 指令重排 register 内的元素。二进制常量 `0b01001110` 控制哪些元素去到哪里。这被称为 **permutation**（置换），它直接对应置换矩阵（第 2 章）：重排 SIMD lane 就是乘以置换矩阵的硬件等价物。

## AVX-512

- AVX-512 再次将宽度翻倍：512 位 register（ZMM），同时处理 16 个 float。

```cpp
__m512 a = _mm512_loadu_ps(ptr);                // 加载 16 个 float
__m512 c = _mm512_fmadd_ps(a, b, c);            // 一次 16 次 FMA
float sum = _mm512_reduce_add_ps(a);             // 内置水平求和（无需手动归约！）

// 掩码运算：只对部分 lane 操作
__mmask16 mask = _mm512_cmpgt_ps_mask(a, zero);  // 哪些 lane > 0？
__m512 relu = _mm512_maskz_mov_ps(mask, a);       // 将负 lane 置零 = ReLU
```

- **掩码 register**（`__mmask16`）是 AVX-512 最强大的特性。每一位控制一个 lane 是否参与运算。这取代了标量清理循环：最后一次迭代使用只让有效 lane 活跃的掩码，无需单独的标量循环即可处理任意向量长度。

- **AVX-512 频率降速**：在许多 Intel CPU 上，使用 AVX-512 指令会导致 CPU 临时降低时钟频率（以保持在热限制内）。这意味着对短促突发而言，AVX-512 并不总比 AVX2 快——频率惩罚可能超过更宽向量的收益。对持续工作负载（如矩阵乘法），AVX-512 胜出。对混合代码（部分 SIMD、部分标量），频率切换可能反而有害。

## Intel AMX：矩阵乘法硬件

- **AMX**（Advanced Matrix eXtensions，高级矩阵扩展）新增了专用矩阵乘法单元。AMX 不使用 SIMD 向量，而是对 **tile** 操作：2D 数据块（最大 16 行 × 64 字节）。

```cpp
#include <immintrin.h>

// AMX tile 乘法：C += A * B（BF16）
// A 为 16x32 BF16，B 为 32x16 BF16，C 为 16x16 FP32
_tile_loadd(0, a_ptr, stride_a);   // 从 A 加载 tile 0
_tile_loadd(1, b_ptr, stride_b);   // 从 B 加载 tile 1
_tile_dpbf16ps(2, 0, 1);           // tile 2 += tile 0 * tile 1（BF16 矩阵乘，FP32 累加）
_tile_stored(2, c_ptr, stride_c);  // 将 tile 2 存到 C
```

- AMX 在一条指令中完成完整的 16×32 × 32×16 矩阵乘法。这相当于一次数百次 FMA 运算，专为 transformer 推理中占主导的小矩阵乘法（attention score 计算、MLP 层）而设计。

- AMX 支持 BF16（bfloat16）和 INT8，与 ML 推理使用的精度匹配。结合 AVX-512 用于其他运算，配备 AMX 的 CPU（Intel Sapphire Rapids、Emerald Rapids）能以接近入门级 GPU 的竞争力运行 transformer 推理。

## 内存对齐

- **对齐内存访问**是指数据地址是向量 register 宽度的整数倍（SSE 为 16 字节，AVX 为 32 字节，AVX-512 为 64 字节）。对齐访问在某些 CPU 上更快，且 `_mm256_load_ps`（相对 `_mm256_loadu_ps`）要求对齐。

```cpp
// 分配对齐内存
float* data = (float*)aligned_alloc(32, n * sizeof(float));  // 为 AVX 32 字节对齐

// C++ 对齐分配
#include <new>
float* data = new (std::align_val_t(32)) float[n];

// 或者使用编译器属性
alignas(32) float data[1024];
```

- **实践中**：在现代 CPU（Haswell 及之后）上，当数据不跨越 cache line 边界时，非对齐加载（`loadu`）几乎与对齐加载一样快。非对齐访问的性能惩罚已基本消失，但 cache line 分裂（数据横跨两个 64 字节 cache line）仍可能让该次加载慢约 2 倍。对齐分配可完全避免此问题。

## 性能陷阱

- **AVX-SSE 切换惩罚**：在较老的 Intel CPU（pre-Skylake）上，在 AVX（256 位）与 SSE（128 位）指令之间切换会产生惩罚（约 70 个周期）。这就是为何在使用 AVX 的函数返回前应调用 `_mm256_zeroupper()`（或 `vzeroupper` 指令）以清除 YMM register 的高 128 位。现代 CPU（Skylake+）无此惩罚。

- **register 压力**：AVX2 有 16 个 YMM register。若你的 kernel 使用过多变量，编译器会把 register 溢出到栈（内存），毁掉性能。保持内层循环简单、活跃变量少。

- **数据依赖**：`sum = _mm256_fmadd_ps(a, b, sum)` 对 `sum` 有依赖：每次迭代必须等上一次 FMA 完成（约 4-5 周期 latency）。修复方法：使用多个独立累加器，最后归约：

```cpp
// 单累加器：受 FMA latency（4-5 周期）限制
__m256 sum = _mm256_setzero_ps();
for (...) {
    sum = _mm256_fmadd_ps(a, b, sum);  // 每次依赖上一次
}

// 四个累加器：4 倍吞吐（隐藏 latency）
__m256 sum0 = _mm256_setzero_ps();
__m256 sum1 = _mm256_setzero_ps();
__m256 sum2 = _mm256_setzero_ps();
__m256 sum3 = _mm256_setzero_ps();
for (...) {
    sum0 = _mm256_fmadd_ps(a0, b0, sum0);  // 独立
    sum1 = _mm256_fmadd_ps(a1, b1, sum1);  // 独立
    sum2 = _mm256_fmadd_ps(a2, b2, sum2);  // 独立
    sum3 = _mm256_fmadd_ps(a3, b3, sum3);  // 独立
}
sum0 = _mm256_add_ps(sum0, sum1);
sum2 = _mm256_add_ps(sum2, sum3);
sum0 = _mm256_add_ps(sum0, sum2);
```

- 这就是 **loop unrolling**（循环展开）以隐藏 latency。CPU 可以背靠背发射 FMA，因为它们写入不同的 register。这是数值代码中最有效的微优化之一。

## Profiling

- **性能计数器**提供硬件级测量：

```bash
# Linux perf（需要内核支持）
perf stat ./my_program                    # 基本计数器：cycles、instructions、IPC
perf stat -e cache-misses,cache-references ./my_program  # cache 行为
perf record -g ./my_program && perf report              # 调用图 profiling

# Intel VTune（详细的 x86 profiling）
vtune -collect hotspots -- ./my_program
vtune -collect memory-access -- ./my_program   # 内存带宽分析
```

- **关注什么**：
    - **IPC**（Instructions Per Cycle，每周期指令数）：CPU 利用效率如何。IPC > 2 为良好。IPC < 1 提示内存停顿或分支误预测。
    - **Cache miss 率**：高 L1/L2 miss 率表明数据局部性差。重组数据访问模式。
    - **分支误预测率**：> 5% 提示分支不可预测。如可能，转为无分支代码（SIMD 比较 + blending）。
    - **实测 FLOPS vs roofline**：将你测得的 FLOPS 与 roofline 模型（文件 01）对比。若低于 roofline，仍有提升空间。

## 编程任务（在 x86——Intel/AMD 上用 g++ 或 clang++ 编译）

1. 编写标量点积与 AVX2 点积。对两者 benchmark 并测量 8 宽 SIMD 的加速。
```cpp
// task1_avx_dot.cpp
// 编译：g++ -O3 -mavx2 -mfma -o task1 task1_avx_dot.cpp

#include <iostream>
#include <chrono>
#include <vector>
#include <immintrin.h>

float dot_scalar(const float* a, const float* b, int n) {
    float sum = 0.0f;
    for (int i = 0; i < n; i++) sum += a[i] * b[i];
    return sum;
}

float dot_avx2(const float* a, const float* b, int n) {
    __m256 sum = _mm256_setzero_ps();
    int i = 0;
    for (; i + 8 <= n; i += 8) {
        __m256 va = _mm256_loadu_ps(a + i);
        __m256 vb = _mm256_loadu_ps(b + i);
        sum = _mm256_fmadd_ps(va, vb, sum);
    }
    // 归约：高 128 加到低 128，再水平相加
    __m128 hi = _mm256_extractf128_ps(sum, 1);
    __m128 lo = _mm256_castps256_ps128(sum);
    __m128 r = _mm_add_ps(hi, lo);
    r = _mm_hadd_ps(r, r);
    r = _mm_hadd_ps(r, r);
    float result = _mm_cvtss_f32(r);
    for (; i < n; i++) result += a[i] * b[i];
    return result;
}

int main() {
    const int N = 10'000'000;
    std::vector<float> a(N, 1.0f), b(N, 2.0f);

    volatile float s1 = dot_scalar(a.data(), b.data(), N);
    volatile float s2 = dot_avx2(a.data(), b.data(), N);

    auto bench = [&](auto fn, const char* name) {
        auto start = std::chrono::high_resolution_clock::now();
        volatile float s;
        for (int t = 0; t < 100; t++) s = fn(a.data(), b.data(), N);
        auto end = std::chrono::high_resolution_clock::now();
        double ms = std::chrono::duration<double, std::milli>(end - start).count() / 100;
        std::cout << name << ": " << ms << " ms (result: " << s << ")\n";
        return ms;
    };

    double t1 = bench(dot_scalar, "Scalar");
    double t2 = bench(dot_avx2,   "AVX2  ");
    std::cout << "Speedup: " << t1 / t2 << "x\n";
    return 0;
}
```

2. 用 `_mm256_max_ps` 实现 AVX2 ReLU，并与标量循环对比。然后尝试多个累加器（循环展开）以隐藏 FMA latency。
```cpp
// task2_avx_relu.cpp
// 编译：g++ -O3 -mavx2 -o task2 task2_avx_relu.cpp

#include <iostream>
#include <chrono>
#include <vector>
#include <immintrin.h>

void relu_scalar(const float* in, float* out, int n) {
    for (int i = 0; i < n; i++) {
        out[i] = in[i] > 0.0f ? in[i] : 0.0f;
    }
}

void relu_avx2(const float* in, float* out, int n) {
    __m256 zero = _mm256_setzero_ps();
    int i = 0;
    for (; i + 8 <= n; i += 8) {
        __m256 x = _mm256_loadu_ps(in + i);
        _mm256_storeu_ps(out + i, _mm256_max_ps(x, zero));
    }
    for (; i < n; i++) out[i] = in[i] > 0.0f ? in[i] : 0.0f;
}

// 展开：每次迭代处理 32 个 float（4 x 8）
void relu_avx2_unrolled(const float* in, float* out, int n) {
    __m256 zero = _mm256_setzero_ps();
    int i = 0;
    for (; i + 32 <= n; i += 32) {
        __m256 x0 = _mm256_loadu_ps(in + i);
        __m256 x1 = _mm256_loadu_ps(in + i + 8);
        __m256 x2 = _mm256_loadu_ps(in + i + 16);
        __m256 x3 = _mm256_loadu_ps(in + i + 24);
        _mm256_storeu_ps(out + i,      _mm256_max_ps(x0, zero));
        _mm256_storeu_ps(out + i + 8,  _mm256_max_ps(x1, zero));
        _mm256_storeu_ps(out + i + 16, _mm256_max_ps(x2, zero));
        _mm256_storeu_ps(out + i + 24, _mm256_max_ps(x3, zero));
    }
    for (; i + 8 <= n; i += 8) {
        _mm256_storeu_ps(out + i, _mm256_max_ps(_mm256_loadu_ps(in + i), zero));
    }
    for (; i < n; i++) out[i] = in[i] > 0.0f ? in[i] : 0.0f;
}

int main() {
    const int N = 16'000'000;
    std::vector<float> in(N), out(N);
    for (int i = 0; i < N; i++) in[i] = (float)(i % 200) - 100.0f;

    auto bench = [&](auto fn, const char* name) {
        fn(in.data(), out.data(), N);  // 预热
        auto start = std::chrono::high_resolution_clock::now();
        for (int t = 0; t < 100; t++) fn(in.data(), out.data(), N);
        auto end = std::chrono::high_resolution_clock::now();
        double ms = std::chrono::duration<double, std::milli>(end - start).count() / 100;
        double bw = 2.0 * N * sizeof(float) / ms / 1e6;  // 读 + 写
        std::cout << name << ": " << ms << " ms (" << bw << " GB/s)\n";
    };

    bench(relu_scalar,        "Scalar        ");
    bench(relu_avx2,          "AVX2          ");
    bench(relu_avx2_unrolled, "AVX2 unrolled ");
    return 0;
}
```

3. 测量内存对齐的影响。在大数组上对比对齐与非对齐加载。
```cpp
// task3_alignment.cpp
// 编译：g++ -O3 -mavx2 -o task3 task3_alignment.cpp

#include <iostream>
#include <chrono>
#include <cstdlib>
#include <immintrin.h>

int main() {
    const int N = 16'000'000;

    // 对齐分配（AVX2 需 32 字节）
    float* aligned = (float*)aligned_alloc(32, N * sizeof(float));

    // 非对齐：从对齐边界偏移 4 字节（1 个 float）
    float* raw = (float*)malloc((N + 1) * sizeof(float));
    float* unaligned = raw + 1;  // 保证未对齐

    for (int i = 0; i < N; i++) {
        aligned[i] = 1.0f;
        unaligned[i] = 1.0f;
    }

    auto bench = [&](float* ptr, bool use_aligned, const char* name) {
        __m256 sum = _mm256_setzero_ps();
        // 预热
        for (int i = 0; i + 8 <= N; i += 8) {
            __m256 v = use_aligned ? _mm256_load_ps(ptr + i) : _mm256_loadu_ps(ptr + i);
            sum = _mm256_add_ps(sum, v);
        }

        auto start = std::chrono::high_resolution_clock::now();
        for (int t = 0; t < 100; t++) {
            sum = _mm256_setzero_ps();
            for (int i = 0; i + 8 <= N; i += 8) {
                __m256 v = use_aligned ? _mm256_load_ps(ptr + i) : _mm256_loadu_ps(ptr + i);
                sum = _mm256_add_ps(sum, v);
            }
        }
        auto end = std::chrono::high_resolution_clock::now();
        double ms = std::chrono::duration<double, std::milli>(end - start).count() / 100;
        double bw = (double)N * sizeof(float) / ms / 1e6;
        std::cout << name << ": " << ms << " ms (" << bw << " GB/s)\n";
    };

    bench(aligned,   true,  "Aligned load  ");
    bench(unaligned, false, "Unaligned load");

    free(aligned);
    free(raw);
    return 0;
}
```
