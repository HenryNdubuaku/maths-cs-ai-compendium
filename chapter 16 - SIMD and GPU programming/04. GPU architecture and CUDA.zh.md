# GPU 架构与 CUDA

*GPU 通过提供数千个核心实现大规模并行，从而改变了 AI。本文件涵盖 GPU 与 CPU 的设计理念差异、GPU 内存层次结构、C++ 中的 CUDA 编程、SIMT 执行模型、内存访问模式、同步、流、性能分析以及 NVIDIA GPU 各代产品——这些知识是编写和理解 GPU kernel 所必需的。*

- 如需包含完整可运行示例的 CUDA 实战教程，请参见配套仓库：[github.com/HenryNdubuaku/cuda-tutorials](https://github.com/HenryNdubuaku/cuda-tutorials)。

- 一块现代 NVIDIA GPU 拥有超过 10,000 个 CUDA 核心。而 CPU 只有 4-128 个核心。这种 100-1000 倍的核心优势正是 GPU 主导机器学习的原因：训练一个 transformer 需要数万亿次乘加运算，GPU 能以 CPU 无法企及的规模并行处理它们。

- 即便你从不亲自编写 CUDA kernel，理解 GPU 架构也能解释：为什么 batch size 很重要（需要足够多的工作量来饱和 GPU）、为什么内存通常是瓶颈（而非计算）、以及为什么某些操作（scatter、条件分支）在 GPU 上很慢。

## GPU 与 CPU：根本不同的设计

- CPU 为 **latency**（延迟）而设计：最小化完成单个任务的时间。它把大部分晶体管预算投入到 cache、分支预测和乱序执行——全是让单个 thread 跑得更快的技巧。

- GPU 为 **throughput**（吞吐量）而设计：最大化每秒完成的任务数。它把大部分晶体管投入到执行单元（ALU）。单个 thread 很慢，但有数千个之多。

| | CPU | GPU |
|--|-----|-----|
| 核心 | 4-128（复杂、快速） | 1,000-20,000（简单、慢速） |
| 时钟频率 | 3-5 GHz | 1-2.5 GHz |
| Cache | 大（32 MB+ L3） | 小（每个 SM 的 shared memory） |
| 分支预测 | 复杂精良 | 无（所有 thread 走同一条路径） |
| 最适合 | 低 latency、复杂控制流 | 高 throughput、数据并行工作 |
| 典型 FLOPS（FP32） | 1-5 TFLOPS | 30-80 TFLOPS |
| 内存带宽 | 50-100 GB/s | 1-3 TB/s |

- GPU 的内存带宽优势（10-30 倍）往往比其计算优势更重要。许多 ML 操作是内存受限的（element-wise 操作、normalization、attention），而 GPU 的带宽让它能足够快地把数据喂给自己的核心。

## GPU 内存层次结构

- 理解 GPU 内存至关重要，因为 **内存访问是主要瓶颈**，而非计算。

| 内存 | 容量 | Latency | 带宽 | 作用域 |
|--------|------|---------|-----------|-------|
| Registers | 每个 SM 约 256 KB | 0 周期 | 最高 | 每个 thread |
| Shared memory | 每个 SM 48-228 KB | 约 5 周期 | 约 20 TB/s | 每个 thread block |
| L1 cache | 每个 SM 128-256 KB | 约 30 周期 | | 每个 SM |
| L2 cache | 4-96 MB | 约 200 周期 | 约 6 TB/s | 全局 |
| Global memory (HBM) | 24-192 GB | 约 400 周期 | 1-3.3 TB/s | 全局 |

- **Registers** 最快但也最有限。每个 thread 拥有一组私有 register（通常最多 255 个）。每个 thread 使用过多 register 会降低 **occupancy**（同时能运行的 thread 更少）。

- **Shared memory** 是由程序员管理的 cache，为一个 block 内所有 thread 共享。它是编写快速 CUDA kernel 的关键：把一块数据从慢速 global memory 加载到快速的 shared memory，然后在上面计算。这就是主导 GPU 编程的 **tiling**（分块）模式。

- **Global memory (HBM)**：GPU 主内存（VRAM）。容量大但慢（400 周期 latency）。所有数据都从这里开始和结束。kernel 优化的目标就是最小化 global memory 访问。

## CUDA 编程模型

- CUDA（Compute Unified Device Architecture）是 NVIDIA 面向 GPU 的编程模型。你编写 **kernel**：运行在 GPU 上、由数千个 thread 同时执行的函数。

### 层次结构：Grid、Block、Thread

```
Grid (整个 launch)
├── Block (0,0)
│   ├── Thread (0,0)
│   ├── Thread (1,0)
│   ├── Thread (2,0)
│   └── ... (每个 block 最多 1024 个 thread)
├── Block (1,0)
│   ├── Thread (0,0)
│   └── ...
└── ... (可能有数百万个 block)
```

- **Thread**：最小单位。每个 thread 在其 block 内有唯一 ID（`threadIdx.x`）。
- **Block**：一组可以共享内存并同步的 thread。Block ID：`blockIdx.x`。Block 大小：`blockDim.x`（最多 1024 个 thread）。
- **Grid**：单个 kernel launch 出来的所有 block。可以是 1D、2D 或 3D。

- 每个 thread 计算自己的全局索引：`int idx = blockIdx.x * blockDim.x + threadIdx.x;`

### 你的第一个 CUDA Kernel

```cpp
// vector_add.cu — CUDA 源文件（.cu 扩展名）

#include <stdio.h>

// __global__ 将此标记为 GPU kernel（从 CPU 调用，在 GPU 上运行）
__global__ void vector_add(const float* a, const float* b, float* c, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    if (idx < n) {           // 边界检查（grid 可能比数据大）
        c[idx] = a[idx] + b[idx];
    }
}

int main() {
    int n = 1 << 20;  // 约 100 万个元素
    size_t bytes = n * sizeof(float);

    // 分配 host（CPU）内存
    float *h_a = new float[n];
    float *h_b = new float[n];
    float *h_c = new float[n];

    // 初始化
    for (int i = 0; i < n; i++) {
        h_a[i] = 1.0f;
        h_b[i] = 2.0f;
    }

    // 分配 device（GPU）内存
    float *d_a, *d_b, *d_c;
    cudaMalloc(&d_a, bytes);
    cudaMalloc(&d_b, bytes);
    cudaMalloc(&d_c, bytes);

    // 把数据从 CPU 拷贝到 GPU
    cudaMemcpy(d_a, h_a, bytes, cudaMemcpyHostToDevice);
    cudaMemcpy(d_b, h_b, bytes, cudaMemcpyHostToDevice);

    // 启动 kernel：每个 block 256 个 thread，使用足够多的 block 覆盖 n 个元素
    int block_size = 256;
    int grid_size = (n + block_size - 1) / block_size;  // 向上取整除法
    vector_add<<<grid_size, block_size>>>(d_a, d_b, d_c, n);

    // 把结果从 GPU 拷贝回 CPU
    cudaMemcpy(h_c, d_a, bytes, cudaMemcpyDeviceToHost);

    // 验证
    printf("c[0] = %f (expected 3.0)\n", h_c[0]);

    // 释放内存
    cudaFree(d_a); cudaFree(d_b); cudaFree(d_c);
    delete[] h_a; delete[] h_b; delete[] h_c;

    return 0;
}
```

```bash
# 使用 NVIDIA 的编译器编译
nvcc -O3 -o vector_add vector_add.cu
./vector_add
```

- **CUDA 中的关键 C++ 概念**：
    - `__global__`：标记 kernel 函数的 CUDA 关键字。从 CPU（`host`）调用，在 GPU（`device`）上运行。
    - `<<<grid_size, block_size>>>`：kernel launch 语法。指定使用多少个 block 和多少个 thread。
    - `cudaMalloc` / `cudaFree`：分配/释放 GPU 内存（类似于 `new`/`delete`，但用于 GPU）。
    - `cudaMemcpy`：在 CPU 与 GPU 之间拷贝数据。这通常是最大的瓶颈（PCIe 带宽约 32 GB/s，而 GPU 内存带宽约 3 TB/s）。

### Warp 与 SIMT

- GPU 以 32 个 thread 为一组执行，称为 **warp**。一个 warp 中的 32 个 thread 同时执行 **同一条指令**（Single Instruction, Multiple Threads，即 SIMT）。这是 GPU 版的 SIMD，但是是在 thread 层面。

- **Warp divergence**（warp 分支发散）发生在同一 warp 中的 thread 走了 `if` 语句的不同分支时。GPU 无法在一个 warp 中同时执行两条不同的指令，因此它会顺序执行两个分支，并把不该参与的 thread 屏蔽掉。这会使性能减半（甚至更糟）。

```cpp
// 糟糕：warp divergence（同一 warp 中的 thread 走不同路径）
if (threadIdx.x % 2 == 0) {
    c[idx] = a[idx] + b[idx];    // 偶数 thread 做这个
} else {
    c[idx] = a[idx] - b[idx];    // 奇数 thread 做这个（同一 warp，被串行化）
}

// 更好：无分支（无 divergence）
float sign = (threadIdx.x % 2 == 0) ? 1.0f : -1.0f;
c[idx] = a[idx] + sign * b[idx];  // 所有 thread 执行同一条指令
```

### Memory Coalescing（内存合并访问）

- **Coalesced access**（合并访问）：当连续的 thread 访问连续的内存地址时，GPU 会把它们合并成一次内存事务。这对性能至关重要。

```cpp
// 好：coalesced — thread 0 读 a[0]，thread 1 读 a[1]，...
c[idx] = a[idx] + b[idx];

// 糟糕：跨步访问 — thread 0 读 a[0]，thread 1 读 a[stride]，...
c[idx] = a[idx * stride] + b[idx * stride];  // stride > 1 浪费带宽
```

- 对于 32 个 thread 的一个 warp，coalesced access 一次事务加载 128 字节（float32 下 32 × 4 字节）。跨步访问需要多次事务，每次加载 128 字节却只用其中一小部分。stride 为 32 是最坏情况：每次事务加载 128 字节却只有一个 thread 用到 4 字节（利用率 3%）。

### Shared Memory 与 Tiling

- **Tiling 模式**是最重要的 GPU 优化技术。其思路：把一块数据从慢速 global memory 加载到快速 shared memory，在上面计算，再把结果写回。

```cpp
// 使用 shared memory tiling 的矩阵乘法（简化版）
__global__ void matmul_tiled(const float* A, const float* B, float* C,
                              int M, int N, int K) {
    // shared memory 中存放 A 的一个 tile 和 B 的一个 tile
    __shared__ float tile_A[TILE_SIZE][TILE_SIZE];
    __shared__ float tile_B[TILE_SIZE][TILE_SIZE];

    int row = blockIdx.y * TILE_SIZE + threadIdx.y;
    int col = blockIdx.x * TILE_SIZE + threadIdx.x;
    float sum = 0.0f;

    // 遍历各 tile
    for (int t = 0; t < (K + TILE_SIZE - 1) / TILE_SIZE; t++) {
        // 把 A 和 B 的一个 tile 加载到 shared memory
        if (row < M && t * TILE_SIZE + threadIdx.x < K)
            tile_A[threadIdx.y][threadIdx.x] = A[row * K + t * TILE_SIZE + threadIdx.x];
        else
            tile_A[threadIdx.y][threadIdx.x] = 0.0f;

        if (col < N && t * TILE_SIZE + threadIdx.y < K)
            tile_B[threadIdx.y][threadIdx.x] = B[(t * TILE_SIZE + threadIdx.y) * N + col];
        else
            tile_B[threadIdx.y][threadIdx.x] = 0.0f;

        __syncthreads();  // 等待所有 thread 完成加载

        // 从该 tile 计算部分点积
        for (int k = 0; k < TILE_SIZE; k++) {
            sum += tile_A[threadIdx.y][k] * tile_B[k][threadIdx.x];
        }

        __syncthreads();  // 在加载下一个 tile 之前等待
    }

    if (row < M && col < N)
        C[row * N + col] = sum;
}
```

- **`__shared__`**：声明由 block 内所有 thread 共享的内存（快速、片上）。
- **`__syncthreads()`**：一个 barrier，会等待直到 block 内所有 thread 都到达此处。在写入 shared memory 与从中读取之间必需（否则某些 thread 会读到过期数据）。
- **为什么 tiling 有效**：不用它，每个 thread 每次乘法都要从 global memory 加载。用了 tiling，一块 TILE_SIZE × TILE_SIZE 的数据被一次性加载到 shared memory，并被 block 内所有 thread 复用。复用系数是 TILE_SIZE，从而把 global memory 流量降低这个倍数。

## Stream 与并发

- 默认情况下，CUDA 操作是串行的：CPU 启动一个 kernel，等它结束，再启动下一个。**Stream** 能让操作重叠：

```cpp
cudaStream_t stream1, stream2;
cudaStreamCreate(&stream1);
cudaStreamCreate(&stream2);

// 这些操作可以重叠：不同的 stream 并发执行
cudaMemcpyAsync(d_a, h_a, bytes, cudaMemcpyHostToDevice, stream1);
cudaMemcpyAsync(d_b, h_b, bytes, cudaMemcpyHostToDevice, stream2);

kernel1<<<grid, block, 0, stream1>>>(d_a, d_c);
kernel2<<<grid, block, 0, stream2>>>(d_b, d_d);
```

- Stream 把数据传输与计算重叠：当一个 stream 的 kernel 运行时，另一个 stream 在拷贝数据。这隐藏了 PCIe 传输 latency 并让 GPU 保持忙碌。

## 分析 CUDA 代码

```bash
# NVIDIA Nsight Compute：kernel 级性能分析
ncu --set full ./my_program

# NVIDIA Nsight Systems：系统级时间线
nsys profile ./my_program

# 快速指标
ncu --metrics sm__throughput,dram__throughput ./my_program
```

- **要看什么**：
    - **Occupancy**：SM 容量被使用的比例。低 occupancy（< 50%）意味着没有足够的 thread 来掩盖内存 latency。原因：每个 thread 使用的 register 过多、每个 block 使用的 shared memory 过多。
    - **内存吞吐**：与峰值带宽对比。如果只达到峰值的 < 50%，说明内存访问模式低效（未合并、bank conflict）。
    - **计算吞吐**：与峰值 FLOPS 对比。如果内存和计算吞吐都低，说明 kernel 是 latency-bound（并行度不足）。

## 高级优化技术

- 在 coalescing 和 shared memory tiling 这些基础之外，高性能的 GPU（和 CPU）代码还会用到若干高级技术：

### 数据布局：AoS 与 SoA

- **Array of Structures（AoS，结构体数组）**：每个元素把所有字段存放在一起。`[{x,y,z}, {x,y,z}, {x,y,z}]`。
- **Structure of Arrays（SoA，数组结构体）**：每个字段存放在自己连续的数组里。`{[x,x,x], [y,y,y], [z,z,z]}`。

```cpp
// AoS：对 SIMD/GPU 不利（访问所有 x 值时会触碰非连续内存）
struct Particle { float x, y, z, mass; };
Particle particles[N];
// particles[0].x, particles[1].x 相隔 16 字节

// SoA：对 SIMD/GPU 有利（所有 x 值连续）
struct Particles {
    float x[N], y[N], z[N], mass[N];
};
// x[0], x[1] 相隔 4 字节 — 非常适合 coalesced access 和 SIMD
```

- 对于数据并行工作负载（SIMD、GPU），SoA 几乎总是更快。当你总是同时访问一个元素的所有字段时（在数值代码中很少见），AoS 更好。PyTorch 的 tensor 天生就是 SoA：每个特征都是一个连续维度。

### 软件预取

- 可以让 CPU 在需要数据之前就开始加载，从而隐藏内存 latency：

```cpp
#include <xmmintrin.h>  // 用于 _mm_prefetch

for (int i = 0; i < n; i += 4) {
    _mm_prefetch((char*)(a + i + 64), _MM_HINT_T0);  // 提前 64 个元素预取
    // 用 SIMD 处理 a[i:i+4]
    __m128 va = _mm_load_ps(a + i);
    // ...
}
```

- 预取指令只是一个提示：如果数据已经在 cache 中，它是空操作；如果不在，CPU 会在后台开始取数据，同时执行其他指令。预取距离（本例中为提前 64 个元素）应根据内存 latency 和循环迭代时间进行调整。

### Kernel Fusion

- **Kernel fusion** 把多个操作合并为一个 kernel，从而避免把中间结果写入内存。这是 ML 中影响最大的单项 GPU 优化：

```
// 未融合：3 次 kernel launch，3 次 global memory 往返
y = matmul(x, W)     // 把 y 写入 global memory
z = y + bias          // 读 y，写 z
out = relu(z)         // 读 z，写 out

// 融合后：1 次 kernel launch，1 次 global memory 写入
out = fused_matmul_bias_relu(x, W, bias)  // y 和 z 从不离开 SRAM
```

- 对于内存受限的操作（bias add、ReLU、layer norm），内存流量主导执行时间。融合能彻底消除这部分流量。PyTorch 的 `torch.compile` 和 Triton 能自动或以极小代价实现融合。

### 混合精度 Kernel

- 用低精度（FP16、BF16、INT8）做计算、用高精度（FP32）做累加，能两全其美：

```cpp
// Tensor Core：FP16 矩阵相乘，FP32 累加
// 每条 Tensor Core 指令：D (FP32) = A (FP16) × B (FP16) + C (FP32)
nvcuda::wmma::mma_sync(c_frag, a_frag, b_frag, c_frag);
```

- FP16 比 FP32 小 2 倍，因此能把内存带宽（常见瓶颈）翻倍，并在 cache 中装下 2 倍的数据。Tensor Core 处理 FP16 的速度是 FP32 CUDA core 的 8-16 倍。这正是混合精度训练（第 6 章）能带来 2-3 倍加速且精度损失极小的原因。

### 内存池分配器

- `cudaMalloc` 很慢（每次约 1 ms），因为它要与 GPU 同步。在每次迭代都分配临时 buffer 的训练循环中，这会累积起来。

- **内存池**（PyTorch 的 caching allocator、CUDA memory pool）会预分配一大块 GPU 内存，并从中子分配，无需系统调用：

```python
# PyTorch 会自动做这件事 — 但理解其原因很重要
# 每次 torch.empty() 都从池中复用内存，不会调用 cudaMalloc
temp = torch.empty(1024, 1024, device='cuda')  # 微秒级，而非毫秒级
```

- 这就是为什么 PyTorch 的 `torch.cuda.memory_allocated()` 与 `torch.cuda.max_memory_allocated()` 不同的原因：allocated 是当前使用的量，max 是峰值（池可能持有比当前使用量更多的内存）。

### Profile-Guided 优化

- 不要盲目优化。**先 profile**，定位瓶颈，对其优化，再重新 profile。Roofline 模型（文件 01）告诉你瓶颈是内存还是计算：

    - **Memory-bound**（算术强度低）：优化数据布局（SoA）、融合 kernel、使用更低精度、预取。
    - **Compute-bound**（算术强度高）：使用 Tensor Core、提高并行度、使用更快的指令（FMA）。
    - **Latency-bound**（并行度不足）：提高 occupancy、减少 register 使用、启动更多 thread。

- 大多数 ML 工作负载是 **memory-bound** 的。一个出人意料的推论：更快的 GPU（更多 FLOPS）往往无济于事。更快的内存（HBM3 vs HBM2e）更有帮助。这就是为什么 A100→H100 升级不仅仅关乎 FLOPS——H100 的内存带宽也是 2 倍。

## NVIDIA GPU 各代产品

| 代次 | 年份 | 关键创新 | AI 相关性 |
|------------|------|----------------|--------------|
| Pascal (P100) | 2016 | HBM2、NVLink | 首款面向严肃深度学习的 GPU |
| Volta (V100) | 2017 | **Tensor Cores**（混合精度 matmul） | 使 FP16 训练成为可能，125 TFLOPS TF32 |
| Ampere (A100) | 2020 | TF32、Sparsity、第三代 Tensor Cores | 312 TFLOPS TF32，结构化稀疏 2:4 |
| Hopper (H100) | 2022 | **Transformer Engine**（FP8）、HBM3 | 989 TFLOPS FP8，动态精度切换 |
| Blackwell (B200) | 2024 | 第二代 Transformer Engine、NVLink 5 | 2.5 PFLOPS FP4，多 die 设计 |

- **Tensor Cores** 是专用的矩阵乘单元。一条 Tensor Core 指令在一个周期内完成一次 4×4 矩阵乘（D = A×B + C）。普通 CUDA core 需要做 64 次 FMA 运算。Tensor Core 正是混合精度训练（float16 计算、float32 累加）速度快的根本原因。

- **Transformer Engine**（Hopper 及以后）在单层内动态地在 FP8 与 FP16 精度之间切换，只在需要的地方选择更高精度。这在不牺牲模型质量的前提下最大化吞吐。它专门为 transformer 架构（attention + MLP）设计，而后者主导着现代 AI。

## 编程任务（用 nvcc 编译）

1. 编写一个 CUDA kernel 对数组应用 ReLU。测量包含内存传输在内的耗时。这一任务教你编写 kernel、使用 cudaMalloc/cudaMemcpy，以及 host↔device 传输瓶颈。
```cpp
// task1_relu.cu
// 编译：nvcc -O3 -o task1_relu task1_relu.cu

#include <stdio.h>
#include <stdlib.h>
#include <cuda_runtime.h>

__global__ void relu_kernel(const float* input, float* output, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    if (idx < n) {
        output[idx] = input[idx] > 0.0f ? input[idx] : 0.0f;
    }
}

int main() {
    const int N = 1 << 24;  // 约 1600 万个元素
    size_t bytes = N * sizeof(float);

    // 分配 host 内存
    float* h_input  = (float*)malloc(bytes);
    float* h_output = (float*)malloc(bytes);
    for (int i = 0; i < N; i++) {
        h_input[i] = (float)(i % 100) - 50.0f;  // 正负值混合
    }

    // 分配 device 内存
    float *d_input, *d_output;
    cudaMalloc(&d_input, bytes);
    cudaMalloc(&d_output, bytes);

    // 对整个流水线计时：拷贝到 GPU、计算、拷贝回
    cudaEvent_t start, stop;
    cudaEventCreate(&start);
    cudaEventCreate(&stop);

    cudaEventRecord(start);
    cudaMemcpy(d_input, h_input, bytes, cudaMemcpyHostToDevice);

    int block_size = 256;
    int grid_size = (N + block_size - 1) / block_size;
    relu_kernel<<<grid_size, block_size>>>(d_input, d_output, N);

    cudaMemcpy(h_output, d_output, bytes, cudaMemcpyDeviceToHost);
    cudaEventRecord(stop);
    cudaEventSynchronize(stop);

    float ms = 0;
    cudaEventElapsedTime(&ms, start, stop);

    // 验证
    int errors = 0;
    for (int i = 0; i < N; i++) {
        float expected = h_input[i] > 0.0f ? h_input[i] : 0.0f;
        if (h_output[i] != expected) errors++;
    }

    printf("Time (including transfers): %.2f ms\n", ms);
    printf("Bandwidth: %.1f GB/s\n", 2.0 * bytes / ms / 1e6);  // 读 + 写
    printf("Errors: %d / %d\n", errors, N);

    cudaFree(d_input); cudaFree(d_output);
    free(h_input); free(h_output);
    return 0;
}
```

2. 用 CUDA 的 shared memory 编写一个 tiled 矩阵乘法，并与朴素（非 tiled）版本比较性能。这一任务教你 shared memory、`__syncthreads` 以及为什么 tiling 重要。
```cpp
// task2_matmul.cu
// 编译：nvcc -O3 -o task2_matmul task2_matmul.cu

#include <stdio.h>
#include <cuda_runtime.h>

#define TILE 16

// 朴素 matmul：每个 thread 计算 C 的一个元素
__global__ void matmul_naive(const float* A, const float* B, float* C, int N) {
    int row = blockIdx.y * blockDim.y + threadIdx.y;
    int col = blockIdx.x * blockDim.x + threadIdx.x;
    if (row < N && col < N) {
        float sum = 0.0f;
        for (int k = 0; k < N; k++) {
            sum += A[row * N + k] * B[k * N + col];
        }
        C[row * N + col] = sum;
    }
}

// Tiled matmul：使用 shared memory 减少 global memory 访问
__global__ void matmul_tiled(const float* A, const float* B, float* C, int N) {
    __shared__ float sA[TILE][TILE];
    __shared__ float sB[TILE][TILE];

    int row = blockIdx.y * TILE + threadIdx.y;
    int col = blockIdx.x * TILE + threadIdx.x;
    float sum = 0.0f;

    for (int t = 0; t < (N + TILE - 1) / TILE; t++) {
        sA[threadIdx.y][threadIdx.x] = (row < N && t*TILE+threadIdx.x < N)
            ? A[row * N + t*TILE + threadIdx.x] : 0.0f;
        sB[threadIdx.y][threadIdx.x] = (t*TILE+threadIdx.y < N && col < N)
            ? B[(t*TILE + threadIdx.y) * N + col] : 0.0f;

        __syncthreads();
        for (int k = 0; k < TILE; k++)
            sum += sA[threadIdx.y][k] * sB[k][threadIdx.x];
        __syncthreads();
    }

    if (row < N && col < N)
        C[row * N + col] = sum;
}

int main() {
    const int N = 1024;
    size_t bytes = N * N * sizeof(float);

    float *d_A, *d_B, *d_C;
    cudaMalloc(&d_A, bytes); cudaMalloc(&d_B, bytes); cudaMalloc(&d_C, bytes);

    // 用 1 初始化（便于验证：C 应全为 N）
    float* h_A = new float[N*N];
    for (int i = 0; i < N*N; i++) h_A[i] = 1.0f;
    cudaMemcpy(d_A, h_A, bytes, cudaMemcpyHostToDevice);
    cudaMemcpy(d_B, h_A, bytes, cudaMemcpyHostToDevice);

    dim3 block(TILE, TILE);
    dim3 grid((N+TILE-1)/TILE, (N+TILE-1)/TILE);

    // 对朴素版本基准测试
    cudaEvent_t start, stop;
    cudaEventCreate(&start); cudaEventCreate(&stop);

    cudaEventRecord(start);
    for (int i = 0; i < 10; i++)
        matmul_naive<<<grid, block>>>(d_A, d_B, d_C, N);
    cudaEventRecord(stop);
    cudaEventSynchronize(stop);
    float naive_ms; cudaEventElapsedTime(&naive_ms, start, stop);

    // 对 tiled 版本基准测试
    cudaEventRecord(start);
    for (int i = 0; i < 10; i++)
        matmul_tiled<<<grid, block>>>(d_A, d_B, d_C, N);
    cudaEventRecord(stop);
    cudaEventSynchronize(stop);
    float tiled_ms; cudaEventElapsedTime(&tiled_ms, start, stop);

    double gflops_naive = 2.0 * N * N * N * 10 / naive_ms / 1e6;
    double gflops_tiled = 2.0 * N * N * N * 10 / tiled_ms / 1e6;

    printf("Naive:  %.2f ms, %.1f GFLOPS\n", naive_ms/10, gflops_naive);
    printf("Tiled:  %.2f ms, %.1f GFLOPS\n", tiled_ms/10, gflops_tiled);
    printf("Speedup: %.1fx\n", naive_ms / tiled_ms);

    cudaFree(d_A); cudaFree(d_B); cudaFree(d_C);
    delete[] h_A;
    return 0;
}
```

3. 演示 warp divergence。编写一个 kernel，使同一 warp 中的 thread 走不同分支，并与无分支版本比较。
```cpp
// task3_divergence.cu
// 编译：nvcc -O3 -o task3_diverge task3_divergence.cu

#include <stdio.h>
#include <cuda_runtime.h>

// 糟糕：warp divergence — 偶数/奇数 thread 走不同路径
__global__ void divergent_kernel(float* data, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    if (idx < n) {
        if (idx % 2 == 0) {
            data[idx] = data[idx] * 2.0f + 1.0f;
        } else {
            data[idx] = data[idx] * 0.5f - 1.0f;
        }
    }
}

// 良好：无分支 — 所有 thread 执行同一条指令
__global__ void branchless_kernel(float* data, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    if (idx < n) {
        float scale = (idx % 2 == 0) ? 2.0f : 0.5f;
        float offset = (idx % 2 == 0) ? 1.0f : -1.0f;
        data[idx] = data[idx] * scale + offset;
    }
}

int main() {
    const int N = 1 << 24;
    float* d_data;
    cudaMalloc(&d_data, N * sizeof(float));
    cudaMemset(d_data, 0, N * sizeof(float));

    int block = 256, grid = (N + block - 1) / block;

    cudaEvent_t start, stop;
    cudaEventCreate(&start); cudaEventCreate(&stop);

    // Divergent
    cudaEventRecord(start);
    for (int i = 0; i < 100; i++)
        divergent_kernel<<<grid, block>>>(d_data, N);
    cudaEventRecord(stop);
    cudaEventSynchronize(stop);
    float div_ms; cudaEventElapsedTime(&div_ms, start, stop);

    // Branchless
    cudaEventRecord(start);
    for (int i = 0; i < 100; i++)
        branchless_kernel<<<grid, block>>>(d_data, N);
    cudaEventRecord(stop);
    cudaEventSynchronize(stop);
    float nodiv_ms; cudaEventElapsedTime(&nodiv_ms, start, stop);

    printf("Divergent:  %.2f ms\n", div_ms / 100);
    printf("Branchless: %.2f ms\n", nodiv_ms / 100);
    printf("Speedup:    %.2fx\n", div_ms / nodiv_ms);

    cudaFree(d_data);
    return 0;
}
```
