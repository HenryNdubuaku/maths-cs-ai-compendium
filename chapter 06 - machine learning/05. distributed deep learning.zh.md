# 分布式深度学习

*分布式训练将计算分散到多个 GPU 和机器上，用于训练那些在单个设备上过大或过慢的模型。本文件涵盖混合精度、数据并行、模型并行、流水线并行、ZeRO、FSDP、张量并行，以及 all-reduce 等通信原语，这些都是大规模训练 LLM 所必需的技术。*

- 在单个 GPU 上训练大型 neural network（神经网络）最终会触及瓶颈。模型可能放不进内存，训练也可能耗时数月。分布式训练将工作分散到多个设备（GPU、TPU 或整台机器）上，以加快训练速度并训练更大的模型。本文件介绍实现这一目标的各种技术。

- 要理解分布式为何重要，先从训练的**计算开销**入手。对于输入维度为 $d_{\text{in}}$、输出维度为 $d_{\text{out}}$ 的稠密层（dense layer），在 $B$ 个样本的 batch 上做一次前向传播大约需要 $2 \cdot B \cdot d_{\text{in}} \cdot d_{\text{out}}$ FLOPs（浮点运算）：输出矩阵每个元素对应一次乘法和一次加法。反向传播的开销大约是前向传播的两倍（需要同时计算对输入和对权重的 gradient），因此一个稠密层上的一次训练步骤大约是 $6 \cdot B \cdot d_{\text{in}} \cdot d_{\text{out}}$ FLOPs。

- 对于 hidden dimension 为 $d$ 的 transformer 层，self-attention 块包含四个投影（Q、K、V 和输出），每个开销为 $O(B \cdot n \cdot d^2)$ FLOPs（其中 $n$ 为序列长度），加上注意力矩阵计算的开销 $O(B \cdot n^2 \cdot d)$。feed-forward 块有两个稠密层，通常扩展到 $4d$ 再缩回：$O(B \cdot n \cdot 8d^2)$。每层总计大约 $O(B \cdot n \cdot 12d^2 + B \cdot n^2 \cdot d)$。再乘以层数，就能明白训练 GPT 规模的模型为何需要数千 GPU 小时。

- **内存墙**往往是更紧的约束。训练期间，GPU 内存必须同时容纳四样东西：

![展示训练内存构成的堆叠柱状图：参数、梯度、优化器状态、激活值](../images/training_memory_breakdown.svg)

- **Parameters（参数）**：模型权重。一个 70 亿参数的模型用 FP32（每参数 4 字节）存储权重就需要 28 GB。
- **Gradients（梯度）**：与参数同等大小，再需 28 GB。
- **Optimizer states（优化器状态）**：Adam 维护两个额外的缓冲区（一阶矩和二阶矩估计），每个都与参数同等大小。即便模型使用低精度，这些状态也会以 FP32 保存以保证数值稳定性。对于 7B 模型，即 $2 \times 28 = 56$ GB。
- **Activations（激活值）**：前向传播中保存、供反向传播使用的中间值。其大小取决于 batch size、序列长度和模型宽度。这往往是最大的一项，且随 batch size 线性增长。

- 对于 7B 模型搭配 FP32 Adam：28（参数）+ 28（梯度）+ 56（优化器）= 112 GB，这还没算激活值。单块 80 GB 的 A100 GPU 容不下这些。这就是分布式策略不可或缺的原因。

- **混合精度训练（mixed precision training）**是第一道防线。不再把所有数据都以 FP32（32 位浮点）存储，而是在前向和反向传播中使用 FP16 或 BF16（16 位），同时为优化器更新保留一份 FP32 的权重主副本。

- **FP16** 精度较高（10 位尾数）但范围有限，可能导致溢出/下溢。Loss scaling（损失缩放，即反向传播前将 loss 乘以一个大因子，再将梯度除以同一因子）可缓解此问题。

- **BF16**（brain float）与 FP32 拥有相同的指数范围（8 位指数），但精度较低（7 位尾数）。它几乎不会溢出，也很少需要 loss scaling，使用更简单。BF16 是现代 transformer 训练的默认选择。

- 混合精度大致将激活值和梯度的内存减半（它们是前向/反向传播中的主要开销），同时将优化器状态保留在 FP32 以保证数值稳定性。

- **数据并行（data parallelism）**是最简单的分布式策略。将整个模型复制到 $N$ 个 GPU 上，把每个 mini-batch 切成 $N$ 等份，每份发给一个 GPU。每个 GPU 在自己的数据块上独立执行前向和反向传播。然后将所有 GPU 上的梯度求平均（通过 all-reduce 操作），每个 GPU 再各自更新本地的模型副本。

- 从模型的视角看，这等价于用大 $N$ 倍的 mini-batch 训练。若每个 GPU 处理 batch size 为 $B$，则有效 batch size 为 $N \cdot B$。

![并列对比：数据并行复制模型并切分数据，模型并行切分模型并共享数据](../images/data_model_parallelism.svg)

- 梯度求平均可以同步或异步进行。**同步 SGD（synchronous SGD）** 等待所有 GPU 完成后再求平均，保证在数学上等价于用更大 batch 的单 GPU 训练。其缺点是最慢的 GPU（"拖后腿者"）会拖累所有人。

- **异步 SGD（asynchronous SGD）** 让每个 GPU 独立更新一个共享的参数服务器，无需等待。这消除了拖后腿者的问题，但引入了"陈旧梯度（stale gradients）"：某个 GPU 可能基于略为过时的参数计算梯度。陈旧梯度会引入噪声并可能拖慢收敛。实践中，带有高效通信的同步 SGD 更受青睐。

- **梯度累积（gradient accumulation）** 是一种在有限硬件上模拟更大 batch size 的软件技巧。不再每个 mini-batch 做一次更新，而是运行若干次前向/反向传播并累积梯度，再做一次更新。这能得到与更大 batch 相同的结果，却无需为激活值占用更多 GPU 内存（同一时刻内存中只有一个 mini-batch 的激活值）。

- 当模型本身大到放不进单个 GPU 时，就需要**模型并行（model parallelism）**。它有两种主要形式。

- **张量并行（tensor parallelism）** 将单个层切分到多个 GPU 上。一个大矩阵乘法 $Y = XW$ 可以按列切分：把 $W$ 拆成 $[W_1, W_2]$ 分到两个 GPU 上，并行计算 $Y_1 = XW_1$ 和 $Y_2 = XW_2$，再拼接。这适用于注意力投影和 feed-forward 层。它要求 GPU 之间通信极快（通常是节点内的 NVLink），因为每一层都必须合并部分结果。

- **流水线并行（pipeline parallelism）** 将不同的层分配给不同的 GPU。GPU 0 跑第 1-4 层，GPU 1 跑第 5-8 层，依此类推。数据像流水线一样依次流过。朴素做法会产生"流水线气泡（pipeline bubble）"：当 GPU 0 在处理 micro-batch 1 的前向传播时，GPU 1-3 闲着。**微批处理（micro-batching）** 通过将 mini-batch 拆成更小的 micro-batch，让它们依次流过流水线，使所有 GPU 大部分时间都忙碌，从而缓解该问题。

- **混合并行（hybrid parallelism）** 组合数据、张量和流水线并行。典型的大模型设置会在节点内（8 个由高速 NVLink 相连的 GPU）使用张量并行，跨节点使用流水线并行，跨节点组使用数据并行。GPT-4 和 Llama 等模型就是这样训练的。

- 分布式训练的效率在很大程度上取决于**通信**。关键操作是 **all-reduce**：给定 $N$ 个 GPU 上各有一个值，计算它们的和（或平均），并把结果分发给所有 GPU。

- 朴素的 all-reduce 把所有数据发到一个 GPU 上，求和后再广播回去。其通信开销为 $O(N)$，并在根节点形成瓶颈。

- **Ring all-reduce（环形 all-reduce）** 高效得多。将 $N$ 个 GPU 排成一个环，每个 GPU 把自己的数据切成 $N$ 份。在 $N - 1$ 步内，每个 GPU 把一份发给邻居，并从另一个邻居接收一份，累加部分和。再经过 $N - 1$ 步，完整的和就传播到所有 GPU。每个 GPU 传输的数据量为数据大小的 $2(N-1)/N$ 倍，随着 $N$ 增大趋近 $2\times$。关键在于，这不会随 $N$ 增长，因而是带宽最优的。

![四个 GPU 排成环形，各自向邻居传递梯度块，直到所有 GPU 都得到完整的和](../images/ring_allreduce.svg)

- **参数服务器（parameter servers）** 是另一种架构，由专门的服务器节点持有模型参数。Worker 计算梯度并发给服务器，服务器更新参数后再发回。这种方式更简单，但可能在服务器处产生通信瓶颈。

- **NCCL**（NVIDIA Collective Communications Library）是 GPU 间通信的标准库。它提供了 all-reduce、all-gather、broadcast 等集合操作的高度优化实现，并自动为网络拓扑选择最佳算法。

- **缩放定律（scaling laws）** 描述模型性能如何随计算、数据和模型规模而提升。Kaplan 等人（2020）最初的缩放定律发现，loss 随上述各量按幂律下降：

$$L(N) \propto N^{-\alpha_N}, \quad L(D) \propto D^{-\alpha_D}, \quad L(C) \propto C^{-\alpha_C}$$

- 其中 $N$ 是参数数量，$D$ 是数据集大小，$C$ 是计算预算。

- **Chinchilla 缩放定律**（Hoffmann 等人，2022）表明，大多数模型训练不足：对于给定的计算预算，应当训练一个更小的模型、使用比先前认为的更多的数据。最优比例大约是每参数 20 个 token。一个 7B 模型应看到约 140B token，而不是 Llama 1 在 65B 模型上所用的 300B token。这一发现使该领域转向"计算最优（compute-optimal）"训练。

- **混合专家（Mixture of Experts，MoE）** 是一种在不按比例增加计算的前提下扩大模型容量的架构。每个 transformer 层不再只有一个 feed-forward 网络，而是有 $N$ 个"专家"网络（每个都是标准 FFN）。一个**门控网络（gating network，router）** 检视每个 token，并将其送到 top-$K$ 个专家（通常 $K = 1$ 或 $K = 2$）。

![Token 通过门控网络路由到选定的专家，采用 top-K 稀疏路由并对输出加权组合](../images/moe_routing.svg)

- 总参数量大得多（因为有 $N$ 个专家），但每个 token 的 FLOPs 基本不变（因为每个 token 只激活 $K$ 个专家）。例如，Mixtral 8x7B 拥有 47B 总参数，但每次前向传播仅使用约 13B，以小模型的代价换来大模型的性能。

- MoE 也带来挑战。**负载均衡（load balancing）**：若 router 把大多数 token 都送到同一个专家，其他专家就闲置了。一个辅助 loss 会鼓励均匀路由。**通信**：不同专家可能分布在不同的 GPU 上，因此路由 token 需要 all-to-all 通信，开销很大。

- 当训练在数千 GPU 上持续数周或数月时，**容错（fault tolerance）** 至关重要。若某块 GPU 故障，你不会希望丢失全部进度。**检查点（checkpointing）** 会周期性地将模型权重、优化器状态和训练状态（学习率、步数、数据位置）保存到磁盘。一旦发生故障，可从最近的检查点重启。

- **梯度检查点（gradient checkpointing，又称 activation recomputation）** 是一种内存优化，并非容错机制。在前向传播中，不再为反向传播保存全部激活值，而只在某些检查点处保存。反向传播时，从检查点重新计算缺失的激活值。这是用计算换内存：前向传播开销约增加 33%，但激活值内存可减少约 $\sqrt{L}$ 倍（$L$ 为层数）。

- 综合起来，训练前沿模型要把所有这些技术组合起来：BF16 混合精度、跨数千 GPU 用 ring all-reduce 的数据并行、节点内的张量并行、跨节点的流水线并行、用以压缩内存的梯度检查点、用以提升参数效率的 MoE，以及用以容错的周期性检查点。系统工程之艰难不亚于算法设计。

- 下面总结分布式训练工具箱：

| 技术 | 作用 | 代价 |
|---|---|---|
| 混合精度（BF16） | 将激活值/梯度内存减半 | 轻微的数值差异 |
| 数据并行 | 跨 GPU 扩展 batch size | 梯度同步的通信开销 |
| 张量并行 | 跨 GPU 切分层 | 需要高速互联 |
| 流水线并行 | 跨 GPU 切分模型阶段 | 流水线气泡（浪费的算力） |
| 梯度累积 | 模拟大 batch | 更慢（多次前向/反向传播） |
| 梯度检查点 | 减少激活值内存 | 约多 33% 计算 |
| Ring all-reduce | 高效地给梯度求平均 | 对大模型受带宽限制 |
| MoE | 容量更大、FLOPs 不变 | 负载均衡与路由复杂度 |
| 缩放定律 | 指导算力分配 | 经验性，未必在所有尺度成立 |

## 编程任务（使用 CoLab 或 notebook）

1. 计算一个 transformer 层的 FLOPs 和内存需求。给定 hidden dimension $d$、序列长度 $n$、batch size $B$ 和层数，估计总训练开销。
```python
import jax.numpy as jnp

def transformer_layer_flops(d, n, B):
    """Approximate FLOPs for one transformer layer forward pass."""
    # QKV projections: 3 * (B * n * d * d) * 2 (multiply-add)
    qkv_flops = 3 * 2 * B * n * d * d
    # Attention: (B * n * n * d) * 2 for QK^T, (B * n * n * d) * 2 for attn*V
    attn_flops = 2 * 2 * B * n * n * d
    # Output projection: (B * n * d * d) * 2
    out_flops = 2 * B * n * d * d
    # FFN: two layers, d->4d and 4d->d: 2 * (B * n * d * 4d) * 2
    ffn_flops = 2 * 2 * B * n * d * 4 * d
    return qkv_flops + attn_flops + out_flops + ffn_flops

def transformer_layer_memory(d, n, B, dtype_bytes=2):
    """Approximate activation memory (bytes) for one layer."""
    # QKV: 3 * B * n * d
    qkv_mem = 3 * B * n * d * dtype_bytes
    # Attention weights: B * heads * n * n (approx B * n * n * sizeof)
    attn_mem = B * n * n * dtype_bytes
    # FFN intermediate: B * n * 4d
    ffn_mem = B * n * 4 * d * dtype_bytes
    return qkv_mem + attn_mem + ffn_mem

# Example: GPT-2 scale
d, n, B, L = 1024, 1024, 8, 24
fwd_flops = transformer_layer_flops(d, n, B)
total_flops = 3 * L * fwd_flops  # 3x for forward + backward
act_mem = L * transformer_layer_memory(d, n, B)
param_count = L * (12 * d * d + 13 * d)  # approximate

print(f"Model: d={d}, n={n}, B={B}, L={L}")
print(f"Parameters: {param_count / 1e6:.0f}M")
print(f"FLOPs per step: {total_flops / 1e12:.2f} TFLOPs")
print(f"Activation memory: {act_mem / 1e9:.2f} GB (BF16)")
print(f"Parameter memory (FP32): {param_count * 4 / 1e9:.2f} GB")
print(f"Adam optimizer memory: {param_count * 8 / 1e9:.2f} GB")
print(f"Total training memory: {(param_count * 16 + act_mem) / 1e9:.2f} GB")
```

2. 模拟数据并行训练。将数据集切分到多个"虚拟 GPU"上，独立计算梯度并求平均，验证结果与单 GPU 训练一致。
```python
import jax
import jax.numpy as jnp

# Simple linear model: y = wx + b
key = jax.random.PRNGKey(0)
X = jax.random.normal(key, (64, 4))
w_true = jnp.array([1.0, -2.0, 3.0, 0.5])
y = X @ w_true + 0.1 * jax.random.normal(key, (64,))

def loss_fn(w, X, y):
    return jnp.mean((X @ w - y) ** 2)

grad_fn = jax.grad(loss_fn)

# Single GPU: full batch gradient
w = jnp.zeros(4)
grad_single = grad_fn(w, X, y)

# Data parallel: split across 4 "GPUs"
n_gpus = 4
chunk_size = len(X) // n_gpus
grads = []
for i in range(n_gpus):
    X_chunk = X[i*chunk_size:(i+1)*chunk_size]
    y_chunk = y[i*chunk_size:(i+1)*chunk_size]
    grads.append(grad_fn(w, X_chunk, y_chunk))

# All-reduce: average gradients
grad_parallel = jnp.mean(jnp.stack(grads), axis=0)

print("Single-GPU gradient:", grad_single)
print("Data-parallel gradient (avg):", grad_parallel)
print(f"Match: {jnp.allclose(grad_single, grad_parallel, atol=1e-5)}")

# Train both and compare
w_single, w_parallel = jnp.zeros(4), jnp.zeros(4)
lr = 0.1
for step in range(100):
    w_single = w_single - lr * grad_fn(w_single, X, y)

    grads = [grad_fn(w_parallel, X[i*chunk_size:(i+1)*chunk_size],
                     y[i*chunk_size:(i+1)*chunk_size]) for i in range(n_gpus)]
    avg_grad = jnp.mean(jnp.stack(grads), axis=0)
    w_parallel = w_parallel - lr * avg_grad

print(f"\nAfter 100 steps:")
print(f"Single-GPU weights: {w_single}")
print(f"Data-parallel weights: {w_parallel}")
print(f"Max difference: {jnp.max(jnp.abs(w_single - w_parallel)):.2e}")
```

3. 实现一个简单的混合专家（Mixture of Experts）层。创建一个门控网络，将 token 路由到 top-K 个专家，并组合它们的输出。
```python
import jax
import jax.numpy as jnp

def expert_fn(x, W1, b1, W2, b2):
    """Simple 2-layer FFN expert."""
    h = jnp.maximum(0, x @ W1 + b1)  # ReLU
    return h @ W2 + b2

def moe_layer(x, gate_W, experts_params, top_k=2):
    """
    MoE forward pass.
    x: (batch, d_model)
    gate_W: (d_model, n_experts)
    experts_params: list of (W1, b1, W2, b2) per expert
    """
    n_experts = len(experts_params)

    # Gating: compute routing scores
    gate_logits = x @ gate_W  # (batch, n_experts)
    gate_probs = jax.nn.softmax(gate_logits, axis=-1)

    # Top-K selection
    top_k_indices = jnp.argsort(-gate_probs, axis=-1)[:, :top_k]
    top_k_probs = jnp.take_along_axis(gate_probs, top_k_indices, axis=-1)
    # Renormalise
    top_k_probs = top_k_probs / jnp.sum(top_k_probs, axis=-1, keepdims=True)

    # Compute expert outputs (simplified: run all experts, mask later)
    expert_outputs = jnp.stack([
        expert_fn(x, *experts_params[i]) for i in range(n_experts)
    ], axis=1)  # (batch, n_experts, d_model)

    # Gather top-K expert outputs and weight them
    batch_idx = jnp.arange(x.shape[0])[:, None]
    selected_outputs = expert_outputs[batch_idx, top_k_indices]  # (batch, top_k, d_model)
    output = jnp.sum(selected_outputs * top_k_probs[:, :, None], axis=1)

    return output, gate_probs

# Setup
key = jax.random.PRNGKey(42)
batch, d_model, d_ff, n_experts = 8, 16, 32, 4

# Initialise experts
experts_params = []
for i in range(n_experts):
    k1, k2, key = jax.random.split(key, 3)[0], jax.random.split(key, 3)[1], jax.random.split(key, 3)[2]
    experts_params.append((
        jax.random.normal(k1, (d_model, d_ff)) * 0.1,
        jnp.zeros(d_ff),
        jax.random.normal(k2, (d_ff, d_model)) * 0.1,
        jnp.zeros(d_model),
    ))

key, subkey = jax.random.split(key)
gate_W = jax.random.normal(subkey, (d_model, n_experts)) * 0.1
x = jax.random.normal(key, (batch, d_model))

output, gate_probs = moe_layer(x, gate_W, experts_params, top_k=2)

print(f"Input shape: {x.shape}")
print(f"Output shape: {output.shape}")
print(f"Gate probabilities (first sample): {gate_probs[0]}")
print(f"Expert usage (avg across batch):")
for i in range(n_experts):
    usage = jnp.mean(gate_probs[:, i])
    print(f"  Expert {i}: {usage:.3f}")
```
