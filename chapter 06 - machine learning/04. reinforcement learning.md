# Обучение с подкреплением

*Обучение с подкреплением (reinforcement learning) обучает агентов принимать последовательные решения путем максимизации накопленной награды методом проб и ошибок. В этом файле рассматриваются марковские процессы принятия решений (MDP), функции ценности, уравнения Беллмана, Q-обучение, градиенты стратегии, методы actor-critic, PPO и RLHF — фреймворк, лежащий в основе агентов, играющих в игры, и настройки языковых моделей.*

- Обучение с учителем требует размеченных данных. Обучение без учителя находит закономерности в неразмеченных данных. **Обучение с подкреплением (RL)** отличается от обоих: агент учится, взаимодействуя со средой, совершая действия и получая награды. Здесь нет правильных меток; агент должен обнаружить правильное поведение методом проб и ошибок.

- Представьте, что вы учите собаку новому трюку. Вы не показываете ей датасет с правильными действиями. Вместо этого она пробует разные варианты, вы даете лакомство за правильные действия, и со временем она понимает, чего вы от нее хотите. RL формализует этот процесс.

- Структура RL состоит из пяти основных компонентов. **Агент** — это обучающийся субъект, принимающий решения. **Среда** — это все, что находится вне агента и с чем он взаимодействует. На каждом временном шаге агент наблюдает **состояние** $s_t$, выбирает **действие** $a_t$, получает **награду** $r_t$ и переходит в новое состояние $s_{t+1}$. Цель агента — максимизировать общую награду, которую он собирает с течением времени.

![Цикл «агент-среда»: агент наблюдает состояние, совершает действие, получает награду, среда переходит в новое состояние](../images/mdp_agent_loop.svg)

- **Стратегия** $\pi$ — это алгоритм действий агента: отображение из состояний в действия. Детерминированная стратегия выдает одно действие для каждого состояния: $a = \pi(s)$. Стохастическая стратегия выдает распределение вероятностей по действиям: $\pi(a \mid s)$. Цель RL — найти оптимальную стратегию, которая максимизирует ожидаемую накопленную награду.

- Математическим фреймворком для RL является **марковский процесс принятия решений (MDP)**, определяемый кортежем $(S, A, P, R, \gamma)$: множество состояний $S$, множество действий $A$, вероятности переходов $P(s' \mid s, a)$, функция награды $R(s, a)$ и коэффициент дисконтирования $\gamma$.

- **Марковское свойство** (из главы 05) гласит, что будущее зависит только от текущего состояния, а не от истории того, как вы в него попали: $P(s_{t+1} \mid s_t, a_t, s_{t-1}, \ldots) = P(s_{t+1} \mid s_t, a_t)$. Это означает, что состояние содержит всю информацию, необходимую для принятия решения.

- **Коэффициент дисконтирования** $\gamma \in [0, 1)$ определяет, насколько агент заботится о будущих наградах по сравнению с немедленными. Дисконтированный доход с момента времени $t$ равен:

$$G_t = r_t + \gamma r_{t+1} + \gamma^2 r_{t+2} + \cdots = \sum_{k=0}^{\infty} \gamma^k r_{t+k}$$

- При $\gamma = 0$ агент является полностью близоруким, заботясь только о следующей награде. При $\gamma$, близком к 1, агент является дальновидным. Коэффициент дисконтирования также гарантирует сходимость суммы (если награды ограничены), что важно для математической корректности.

- **Функции ценности** оценивают, насколько хорошо находиться в состоянии (или совершать действие в состоянии). **Функция ценности состояния** $V^\pi(s)$ — это ожидаемый доход, начиная с состояния $s$ и следуя стратегии $\pi$:

$$V^\pi(s) = \mathbb{E}_\pi \left[ G_t \mid s_t = s \right]$$

- **Функция ценности действия** $Q^\pi(s, a)$ — это ожидаемый доход, начиная с состояния $s$, совершая действие $a$, а затем следуя стратегии $\pi$:

$$Q^\pi(s, a) = \mathbb{E}_\pi \left[ G_t \mid s_t = s, a_t = a \right]$$

- Связь между ними: $V^\pi(s) = \sum_a \pi(a \mid s) \, Q^\pi(s, a)$. Ценность состояния — это среднее значение ценностей действий, взвешенное согласно стратегии.

- **Уравнение Беллмана** выражает рекурсивную зависимость: ценность состояния равна немедленной награде плюс дисконтированная ценность следующего состояния. Для функции ценности состояния:

$$V^\pi(s) = \sum_a \pi(a \mid s) \sum_{s'} P(s' \mid s, a) \left[ R(s, a) + \gamma \, V^\pi(s') \right]$$

- Для оптимальной функции ценности $V^{*}(s)$ агент всегда выбирает лучшее действие:

$$V^{*}(s) = \max_a \sum_{s'} P(s' \mid s, a) \left[ R(s, a) + \gamma \, V^{*}(s') \right]$$

- Аналогично, **уравнение оптимальности Беллмана** для $Q^{*}$:

$$Q^{*}(s, a) = \sum_{s'} P(s' \mid s, a) \left[ R(s, a) + \gamma \max_{a'} Q^{*}(s', a') \right]$$

- Как только у вас есть $Q^{*}$, оптимальная стратегия становится тривиальной: всегда выбирайте действие с наибольшим Q-значением: $\pi^{*}(s) = \arg\max_a Q^{*}(s, a)$.

- Методы **динамического программирования** решают MDP, когда известны вероятности переходов и награды (полная модель). **Оценка стратегии** вычисляет $V^\pi$ для заданной стратегии путем итеративного применения уравнения Беллмана до сходимости. **Улучшение стратегии** берет функцию ценности и строит лучшую стратегию, действуя жадно: $\pi'(s) = \arg\max_a \sum_{s'} P(s' \mid s, a)[R(s,a) + \gamma V^\pi(s')]$.

- **Итерация стратегии** чередует оценку и улучшение до тех пор, пока стратегия не перестанет меняться. Гарантируется, что она сойдется к оптимальной стратегии.

- **Итерация по ценности** объединяет оба шага в один: она многократно применяет уравнение оптимальности Беллмана до тех пор, пока $V^{*}$ не сойдется, а затем извлекает стратегию.

$$V(s) \leftarrow \max_a \sum_{s'} P(s' \mid s, a) \left[ R(s, a) + \gamma \, V(s') \right]$$

- Динамическое программирование требует знания $P(s' \mid s, a)$, что часто непрактично. В большинстве реальных задач агент не знает динамику среды; он может только взаимодействовать с ней. Здесь на помощь приходят **модельно-свободные (model-free)** методы.

- **Обучение на основе временных различий (TD-обучение)** учится на опыте, не зная модели. Ключевая идея — **бутстрэппинг (bootstrapping)**: вместо того чтобы ждать окончания эпизода для вычисления фактического дохода $G_t$, вы оцениваете его, используя текущую функцию ценности:

$$V(s_t) \leftarrow V(s_t) + \alpha \left[ r_t + \gamma \, V(s_{t+1}) - V(s_t) \right]$$

- Выражение в скобках — это **TD-ошибка**: разница между **TD-целью** ($r_t + \gamma V(s_{t+1})$) и текущей оценкой $V(s_t)$. Если TD-ошибка положительна, состояние оказалось лучше, чем ожидалось, поэтому мы увеличиваем его ценность. Если отрицательна — уменьшаем.

![Переход состояния, показывающий TD-цель: текущую ценность, награду и бутстрэппинг следующей ценности с формулой обновления](../images/td_update.svg)

- Обучение с временными различиями (TD-обучение) обновляет значения после каждого шага (а не после завершения эпизодов), что делает его гораздо более эффективным, чем методы Монте-Карло. Оно также работает в непрерывных (неэпизодических) средах.

- **SARSA** (State-Action-Reward-State-Action) — это TD-обучение, примененное к Q-значениям. Агент выполняет действие $a$ в состоянии $s$, наблюдает награду $r$ и следующее состояние $s'$, а затем выбирает следующее действие $a'$ в соответствии со своей стратегией:

$$Q(s, a) \leftarrow Q(s, a) + \alpha \left[ r + \gamma \, Q(s', a') - Q(s, a) \right]$$

- SARSA является **on-policy** алгоритмом: он обновляется с использованием действия, которое агент совершает на самом деле, включая исследование среды. Это делает SARSA более консервативным; он изучает стратегию, которая учитывает собственный шум исследования.

- **Q-learning** — самый известный алгоритм обучения с подкреплением (RL). Он похож на SARSA, но вместо использования действия, которое агент совершает на самом деле, он использует наилучшее возможное действие:

$$Q(s, a) \leftarrow Q(s, a) + \alpha \left[ r + \gamma \max_{a'} Q(s', a') - Q(s, a) \right]$$

- Q-learning является **off-policy** алгоритмом: он изучает оптимальные Q-значения независимо от стратегии, которой следует агент. Агент может исследовать среду случайным образом, продолжая при этом изучать оптимальные значения действий. Это делает Q-learning более агрессивным и часто более быстрым в сходимости, но он может переоценивать значения.

- **Исследование против использования (Exploration vs exploitation)** — это фундаментальная дилемма: должен ли агент использовать то, что он уже знает (выбирать действие с наивысшим оценочным значением), или исследовать неизвестные действия (которые могут оказаться лучше)?

- Простейшая стратегия — **epsilon-greedy**: с вероятностью $\epsilon$ совершить случайное действие (исследование); с вероятностью $1 - \epsilon$ совершить жадное действие (использование). Распространенный график обучения начинается с высокого значения $\epsilon$ (много исследований) и постепенно снижает его с течением времени.

- Табличные методы (хранение значения для каждой пары состояние-действие в таблице) работают для небольших дискретных пространств состояний. Для больших или непрерывных пространств состояний требуется аппроксимация функций. **Deep Q-Networks (DQN)** используют нейронную сеть для аппроксимации $Q(s, a; \theta)$, где $\theta$ — веса сети.

- В DQN были представлены два критически важных метода стабилизации. **Experience replay (буфер воспроизведения опыта)**: вместо обучения на последовательных переходах (которые сильно коррелируют), переходы сохраняются в буфере воспроизведения, из которого для обучения выбираются случайные мини-батчи. Это устраняет корреляции и эффективно использует данные.

- **Целевая сеть (Target network)**: использование отдельной, медленно обновляемой копии сети для вычисления TD-целей. Без этого цель меняется каждый раз при обновлении сети, создавая нестабильность типа «погоня за собственным хвостом». Целевая сеть обновляется периодически (жесткое обновление каждые $N$ шагов) или непрерывно (мягкое обновление: $\theta^{-} \leftarrow \tau\theta + (1-\tau)\theta^{-}$).

- Функция потерь DQN — это просто среднеквадратичная ошибка (MSE) между предсказанными Q-значениями и TD-целями:

$$\mathcal{L}(\theta) = \mathbb{E} \left[ \left( r + \gamma \max_{a'} Q(s', a'; \theta^{-}) - Q(s, a; \theta) \right)^2 \right]$$

- Все рассмотренные до сих пор методы изучают функции ценности и выводят из них стратегии. Методы **градиента стратегии (policy gradient)** используют другой подход: они напрямую параметризуют стратегию $\pi(a \mid s; \theta)$ и оптимизируют ее с помощью градиентного подъема по ожидаемой доходности.

- **Теорема о градиенте стратегии** дает градиент ожидаемой доходности по отношению к параметрам стратегии:

$$\nabla_\theta J(\theta) = \mathbb{E}_\pi \left[ \nabla_\theta \log \pi(a \mid s; \theta) \cdot G_t \right]$$

- Это означает: увеличивать вероятность действий, которые привели к высокой доходности, и уменьшать вероятность действий, которые привели к низкой доходности. Градиент логарифмической вероятности задает направление изменения стратегии, а $G_t$ масштабирует величину этого изменения.

- **REINFORCE** — простейший алгоритм градиента стратегии. Выполните эпизод, вычислите доходность $G_t$ для каждого шага и обновите параметры:

$$\theta \leftarrow \theta + \alpha \, \nabla_\theta \log \pi(a_t \mid s_t; \theta) \cdot G_t$$

- REINFORCE обладает высокой дисперсией, поскольку $G_t$ — это зашумленная оценка ожидаемой доходности по одной выборке. Распространенное решение — вычесть **baseline (базовое значение)** (обычно среднюю доходность или обученную функцию ценности), чтобы уменьшить дисперсию без внесения смещения:

$$\theta \leftarrow \theta + \alpha \, \nabla_\theta \log \pi(a_t \mid s_t; \theta) \cdot (G_t - b)$$

- Методы **Actor-Critic (Актор-Критик)** используют две сети. **Актор** — это стратегия $\pi(a \mid s; \theta)$. **Критик** — это функция ценности $V(s; \phi)$, которая служит базовым значением. Преимущество $A_t = r_t + \gamma V(s_{t+1}) - V(s_t)$ заменяет $G_t - b$:

$$\theta \leftarrow \theta + \alpha \, \nabla_\theta \log \pi(a_t \mid s_t; \theta) \cdot A_t$$

- Критик обновляется путем минимизации TD-ошибки, точно так же, как в методах на основе функций ценности. Актор обновляется с использованием градиента стратегии, при этом оценка преимущества от критика снижает дисперсию. Это сочетает лучшие стороны обоих подходов.

![Двухголовая архитектура: актор выводит вероятности действий, критик выводит оценку ценности, сигнал преимущества направляет обновления актора](../images/actor_critic.svg)

- **PPO** (Proximal Policy Optimization) — наиболее широко используемый на практике алгоритм градиента стратегии. Он решает ключевую проблему: если обновление стратегии слишком велико, производительность может катастрофически упасть.

- PPO использует **суррогатную функцию потерь с ограничением (clipped surrogate objective)**. Пусть $r_t(\theta) = \frac{\pi(a_t | s_t; \theta)}{\pi(a_t | s_t; \theta_{\text{old}})}$ — отношение вероятностей между новой и старой стратегиями. Функция потерь выглядит так:

$$\mathcal{L}^{\text{CLIP}}(\theta) = \mathbb{E} \left[ \min\!\left( r_t(\theta) A_t, \; \text{clip}(r_t(\theta), 1-\epsilon, 1+\epsilon) A_t \right) \right]$$

- Ограничение (обычно $\epsilon = 0.2$) не дает отношению отклониться слишком далеко от 1, что делает обновления небольшими и стабильными. Если преимущество положительно (действие было хорошим), отношение ограничивается значением $1 + \epsilon$. Если отрицательно (действие было плохим), отношение ограничивается значением $1 - \epsilon$. Это проще и стабильнее, чем более ранние методы доверительной области (TRPO).

- PPO использовался для обучения моделей в стиле ChatGPT с помощью **RLHF** (обучение с подкреплением на основе отзывов людей). В RLHF модель вознаграждения обучается на данных о предпочтениях людей (какой из двух вариантов ответа люди предпочитают?), а затем PPO оптимизирует стратегию языковой модели для максимизации этой изученной награды.

- **DPO** (Direct Preference Optimization) упрощает RLHF, полностью исключая модель вознаграждения. Вместо обучения модели вознаграждения с последующим запуском RL, DPO выводит функцию потерь в замкнутой форме, которая напрямую оптимизирует стратегию на основе данных о предпочтениях:

$$\mathcal{L}_{\text{DPO}}(\theta) = -\mathbb{E} \left[ \log \sigma\!\left( \beta \log \frac{\pi_\theta(y_w \mid x)}{\pi_{\text{ref}}(y_w \mid x)} - \beta \log \frac{\pi_\theta(y_l \mid x)}{\pi_{\text{ref}}(y_l \mid x)} \right) \right]$$

- Здесь $y_w$ — предпочтительный (выигрышный) ответ, а $y_l$ — менее предпочтительный (проигрышный) ответ. DPO увеличивает относительную вероятность предпочтительных выходов и гораздо проще в реализации, чем RLHF на основе PPO.

- Два важных различия в алгоритмах обучения с подкреплением. **On-policy против off-policy**: методы on-policy (SARSA, PPO) обучаются на данных, сгенерированных текущей стратегией; методы off-policy (Q-learning, DQN) могут обучаться на данных, сгенерированных любой стратегией. Методы off-policy более эффективны с точки зрения использования выборки (они повторно используют старые данные), но могут быть менее стабильными.

- **На основе модели (model-based) против свободных от модели (model-free)**: методы model-free (все, что обсуждалось до сих пор) обучают значения или стратегии непосредственно на основе опыта. Методы model-based изучают модель среды ($P(s' \mid s, a)$ и $R(s, a)$) и используют её для планирования (моделирования будущих траекторий без фактического выполнения действий). Методы model-based более эффективны с точки зрения использования выборки, но добавляют сложность, связанную с обучением точной модели.

- Краткий обзор ландшафта обучения с подкреплением:

| Метод | Тип | Основная идея | Преимущество |
|---|---|---|---|
| Итерация по значениям | DP, model-based | Оптимальность Беллмана | Точное решение (малые MDP) |
| SARSA | TD, on-policy | Обучение Q on-policy | Консервативность, безопасность |
| Q-Learning | TD, off-policy | Обучение Q*, жадная цель | Простота, эффективность |
| DQN | Deep, off-policy | Нейронная Q + replay + целевая сеть | Масштабируемость до состояний высокой размерности |
| REINFORCE | Градиент стратегии | Градиент лог-вероятности * доход | Простая оптимизация стратегии |
| Actor-Critic | PG + значение | Актор + критик для низкой дисперсии | Практичность и гибкость |
| PPO | PG, clipped | Стабильность типа trust-region | Отраслевой стандарт |
| DPO | Прямое предпочтение | Пропуск модели вознаграждения | Более простой RLHF |

```python
# 4x4 gridworld: goal at (3,3), reward -1 per step, 0 at goal
grid_size = 4
gamma = 0.99
goal = (3, 3)

# Actions: up, down, left, right
actions = [(-1, 0), (1, 0), (0, -1), (0, 1)]
action_names = ['up', 'down', 'left', 'right']
action_arrows = ['\u2191', '\u2193', '\u2190', '\u2192']

def step(s, a):
    """Deterministic transition."""
    ns = (max(0, min(grid_size-1, s[0]+a[0])),
          max(0, min(grid_size-1, s[1]+a[1])))
    return ns

# Value iteration
V = jnp.zeros((grid_size, grid_size))
for iteration in range(100):
    V_new = jnp.array(V)
    for i in range(grid_size):
        for j in range(grid_size):
            if (i, j) == goal:
                continue
            values = []
            for a in actions:
                ns = step((i, j), a)
                values.append(-1 + gamma * float(V[ns[0], ns[1]]))
            V_new = V_new.at[i, j].set(max(values))
    if jnp.max(jnp.abs(V_new - V)) < 1e-6:
        print(f"Converged in {iteration+1} iterations")
        break
    V = V_new

# Extract policy
policy = [['' for _ in range(grid_size)] for _ in range(grid_size)]
for i in range(grid_size):
    for j in range(grid_size):
        if (i, j) == goal:
            policy[i][j] = 'G'
            continue
        best_a = max(range(4), key=lambda a: -1 + gamma * float(V[step((i,j), actions[a])[0], step((i,j), actions[a])[1]]))
        policy[i][j] = action_arrows[best_a]

fig, axes = plt.subplots(1, 2, figsize=(10, 4))
im = axes[0].imshow(V, cmap='YlOrRd_r')
axes[0].set_title("Optimal Value Function")
for i in range(grid_size):
    for j in range(grid_size):
        axes[0].text(j, i, f"{V[i,j]:.1f}", ha='center', va='center', fontsize=10)
plt.colorbar(im, ax=axes[0])

axes[1].imshow(jnp.ones((grid_size, grid_size)), cmap='Greys', vmin=0, vmax=2)
axes[1].set_title("Optimal Policy")
for i in range(grid_size):
    for j in range(grid_size):
        axes[1].text(j, i, policy[i][j], ha='center', va='center', fontsize=18)
plt.tight_layout(); plt.show()
```

2. Реализуйте табличный Q-learning для простого сеточного мира. Обучите агента, постройте кривую обучения и покажите выученные Q-значения.
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

grid_size = 5
goal = (4, 4)
actions = [(-1,0), (1,0), (0,-1), (0,1)]

# Q-table
Q = {}
for i in range(grid_size):
    for j in range(grid_size):
        Q[(i,j)] = [0.0] * 4

alpha = 0.1
gamma = 0.95
epsilon = 1.0
epsilon_decay = 0.995
min_epsilon = 0.01

def step(s, a_idx):
    a = actions[a_idx]
    ns = (max(0, min(grid_size-1, s[0]+a[0])),
          max(0, min(grid_size-1, s[1]+a[1])))
    r = 0.0 if ns == goal else -1.0
    done = ns == goal
    return ns, r, done

key = jax.random.PRNGKey(42)
rewards_per_episode = []

for ep in range(500):
    s = (0, 0)
    total_reward = 0
    for _ in range(100):
        key, subkey = jax.random.split(key)
        if float(jax.random.uniform(subkey)) < epsilon:
            key, subkey = jax.random.split(key)
            a = int(jax.random.randint(subkey, (), 0, 4))
        else:
            a = max(range(4), key=lambda i: Q[s][i])

        ns, r, done = step(s, a)
        total_reward += r
        # Q-learning update
        Q[s][a] += alpha * (r + gamma * max(Q[ns]) - Q[s][a])
        s = ns
        if done:
            break
    rewards_per_episode.append(total_reward)
    epsilon = max(min_epsilon, epsilon * epsilon_decay)

plt.figure(figsize=(8, 4))
# Smooth the curve
window = 20
smoothed = [sum(rewards_per_episode[max(0,i-window):i+1])/min(i+1, window)
            for i in range(len(rewards_per_episode))]
plt.plot(smoothed, color='#3498db', linewidth=1.5)
plt.xlabel("Episode"); plt.ylabel("Total Reward (smoothed)")
plt.title("Q-Learning on Gridworld")
plt.grid(alpha=0.3); plt.show()

# Show learned policy
arrow = ['\u2191', '\u2193', '\u2190', '\u2192']
print("Learned policy:")
for i in range(grid_size):
    row = ""
    for j in range(grid_size):
        if (i,j) == goal:
            row += " G "
        else:
            row += f" {arrow[max(range(4), key=lambda a: Q[(i,j)][a])]} "
    print(row)
```

3. Реализуйте алгоритм REINFORCE для задачи о многоруком бандите. Покажите, как стратегия (policy) эволюционирует в процессе обучения, отдавая предпочтение лучшему рычагу.

```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

# 5-armed bandit with different expected rewards
true_rewards = jnp.array([0.2, 0.5, 0.8, 0.3, 0.1])
n_arms = len(true_rewards)

# Policy: softmax over logits
logits = jnp.zeros(n_arms)
lr = 0.1
key = jax.random.PRNGKey(42)

policy_history = []
reward_history = []

for step in range(2000):
    probs = jax.nn.softmax(logits)
    policy_history.append(probs)

    # Sample action
    key, subkey = jax.random.split(key)
    action = jax.random.choice(subkey, n_arms, p=probs)

    # Get reward (Bernoulli)
    key, subkey = jax.random.split(key)
    reward = float(jax.random.uniform(subkey) < true_rewards[action])
    reward_history.append(reward)

    # REINFORCE update
    # grad log pi(a) = e_a - probs (for softmax parameterisation)
    grad_log_pi = -probs.at[action].add(1.0)  # one-hot(a) - probs
    logits = logits + lr * reward * grad_log_pi

policy_history = jnp.stack(policy_history)

fig, axes = plt.subplots(1, 2, figsize=(12, 4))
colors = ['#3498db', '#e74c3c', '#27ae60', '#9b59b6', '#f39c12']
for i in range(n_arms):
    axes[0].plot(policy_history[:, i], color=colors[i],
                 label=f'Arm {i} (true={true_rewards[i]:.1f})', linewidth=1.5)
axes[0].set_xlabel("Step"); axes[0].set_ylabel("P(arm)")
axes[0].set_title("Policy Evolution (REINFORCE)")
axes[0].legend(fontsize=8); axes[0].grid(alpha=0.3)

# Smoothed reward
window = 50
smoothed = [sum(reward_history[max(0,i-window):i+1])/min(i+1,window)
            for i in range(len(reward_history))]
axes[1].plot(smoothed, color='#27ae60', linewidth=1.5)
axes[1].axhline(y=0.8, color='#e74c3c', linestyle='--', alpha=0.5, label='Best arm')
axes[1].set_xlabel("Step"); axes[1].set_ylabel("Avg Reward")
axes[1].set_title("Reward Over Time"); axes[1].legend()
axes[1].grid(alpha=0.3)
plt.tight_layout(); plt.show()
```
