# 多元微积分

*多元微积分把 derivative 和 integral 推广到多变量函数，这至关重要，因为 ML 模型有数百万个参数。本文件涵盖 partial derivative（偏导数）、gradient（梯度）、Jacobian、Hessian，以及让反向传播成为可能的多元 chain rule。*

- 到目前为止，我们的函数都接受单一输入 $x$ 并产生单一输出 $f(x)$。但在 ML 中，我们几乎从不只处理一个变量。

- 考虑一个二元函数，如 $f(x, y) = x^2 + y^2$。它在三维空间中定义了一个曲面，一个碗状形状。我们想知道：如果在保持 $y$ 不变的同时微调 $x$，$f$ 会如何变化？这就是 **partial derivative（偏导数）**。

- $f$ 对 $x$ 的 partial derivative，写作 $\frac{\partial f}{\partial x}$，把其他所有变量视为常数，然后对 $x$ 正常求导。

- 对于 $f(x, y) = x^2y + 3x - 2y$：

$$\frac{\partial f}{\partial x} = 2xy + 3 \qquad \frac{\partial f}{\partial y} = x^2 - 2$$

- 计算 $\frac{\partial f}{\partial x}$ 时，我们把 $y$ 当作常数，所以 $x^2y$ 求导得 $2xy$，$3x$ 得 $3$，$-2y$ 得 $0$。

- 计算 $\frac{\partial f}{\partial y}$ 时，我们把 $x$ 当作常数，所以 $x^2y$ 求导得 $x^2$，$3x$ 得 $0$，$-2y$ 得 $-2$。

- 几何上，对 $x$ 求 partial derivative 就像用一个平行于 $xz$ 平面的平面（在固定 $y$ 值处）切开三维曲面，并求所得曲线的斜率。

![偏导数：固定一个变量来切出曲面](../images/partial_derivative.svg)

- **gradient（梯度）**把所有 partial derivative 收集到一个向量中：

$$\nabla f = \left(\frac{\partial f}{\partial x_1}, \frac{\partial f}{\partial x_2}, \ldots, \frac{\partial f}{\partial x_n}\right)$$

- 对于 $f(x, y) = x^2 + y^2$：$\nabla f(x, y) = (2x, 2y)$。在点 $(1, 2)$ 处：$\nabla f(1, 2) = (2, 4)$。

- gradient 有两个关键性质：

    - **方向**：它指向上升最陡的方向。想象一位登山者在山上，其所在位置的 gradient 指向正上方，沿最陡的路径。

    - **大小**：$\|\nabla f\|$ 给出在该最陡方向上的上升速率。gradient 大意味着地势陡峭；gradient 小意味着几乎平坦。

![梯度向量指向上坡，垂直于等高线](../images/gradient_contour.svg)

- 既然 gradient 指向上坡，沿相反方向（$-\nabla f$）移动就是下坡，朝更小的值走。这个简单的想法是 **梯度下降** 的基础，一种我们将在后续章节详细探讨的 optimization 技术。就目前而言，关键要点是：gradient 告诉你哪个方向是"上"以及攀爬有多陡。

- **方向导数** 推广了 partial derivative。它不是问"$f$ 沿 $x$ 轴怎么变化？"，而是问"$f$ 沿任意方向 $\mathbf{u}$ 怎么变化？" 它的计算方式是 gradient 与一个单位向量的点积：

$$D_{\mathbf{u}} f = \nabla f \cdot \mathbf{u}$$

- 对于 $f(x, y) = x^2 + y^2$ 在 $(1, 2)$ 处沿 $\mathbf{v} = (3, 4)$ 方向：先归一化得 $\mathbf{u} = (3/5, 4/5)$，则 $D_{\mathbf{u}} f = (2, 4) \cdot (3/5, 4/5) = 6/5 + 16/5 = 22/5$。

- partial derivative 是方向导数在方向沿坐标轴时的特例。如果某方向上方向导数为零，则函数在该点沿该方向是平坦的。

- **等高线**（或等值线）连接函数值相同的点。对于 $f(x, y) = x^2 + y^2$，等高线是以原点为圆心的同心圆：对不同 $c$ 值有 $x^2 + y^2 = c$。

- 等高线绝不会相交（一个点不可能有两个不同的函数值）。

- gradient 总是垂直于等高线，从较低值指向较高值。

- 等高线密集表示地势陡峭；稀疏表示坡度平缓。

- 到目前为止，我们的函数都产生单一输出。但许多函数产生多个输出。函数 $\mathbf{F}: \mathbb{R}^n \to \mathbb{R}^m$ 接受 $n$ 个输入并产生 $m$ 个输出。**Jacobian 矩阵** 把这种向量值函数的所有 partial derivative 组织起来：

```math
J = \begin{bmatrix} \frac{\partial f_1}{\partial x_1} & \cdots & \frac{\partial f_1}{\partial x_n} \\ \vdots & \ddots & \vdots \\ \frac{\partial f_m}{\partial x_1} & \cdots & \frac{\partial f_m}{\partial x_n} \end{bmatrix}
```

- Jacobian 的每一行是某一个输出分量的 gradient。对于一个 3 输入 2 输出的函数，Jacobian 是 $2 \times 3$ 矩阵。

- Jacobian 把 derivative 推广到向量值函数。

- 正如标量函数的 derivative 告诉你每单位输入变化引起多少输出变化，Jacobian 告诉你每个输出相对每个输入如何变化。

- **Jacobian 的行列式** 衡量一个变换在局部对空间的拉伸或压缩程度。

- 如果行列式为 2，小区域的面积加倍。如果为 0，该变换把空间压缩到更低维度（回忆我们在矩阵那一章中提到的，零行列式意味着一个奇异、不可逆的变换）。

- 当多个变换复合时（一个接一个地输入），整体映射的 Jacobian 是各 Jacobian 的乘积。我们将在后续章节看到这一思想成为核心。

- gradient 捕捉一阶信息（斜率），而 **Hessian 矩阵** 捕捉二阶信息（曲率）。

- 对于标量函数 $f(x_1, \ldots, x_n)$，Hessian 是由所有二阶 partial derivative 组成的 $n \times n$ 矩阵：

```math
H = \begin{bmatrix} \frac{\partial^2 f}{\partial x_1^2} & \frac{\partial^2 f}{\partial x_1 \partial x_2} & \cdots \\ \frac{\partial^2 f}{\partial x_2 \partial x_1} & \frac{\partial^2 f}{\partial x_2^2} & \cdots \\ \vdots & \vdots & \ddots \end{bmatrix}
```

- 对于 $f(x, y) = x^3 + 2xy^2 - y^3$，gradient 为 $(3x^2 + 2y^2,\; 4xy - 3y^2)$，Hessian 为：

```math
H = \begin{bmatrix} 6x & 4y \\ 4y & 4x - 6y \end{bmatrix}
```

- 对角元素（$6x$ 和 $4x - 6y$）告诉你沿 $x$ 方向移动时 $x$ 方向的斜率如何变化，$y$ 方向类似。

- 非对角元素（$4y$）告诉你沿一个方向移动时另一方向的斜率如何变化。

- **Clairaut 定理** 保证：对于具有连续二阶导数的函数，混合偏导数相等：$\frac{\partial^2 f}{\partial x \partial y} = \frac{\partial^2 f}{\partial y \partial x}$。

- 这意味着 Hessian 是对称的，这（如我们在矩阵那一章所见）保证了实特征值和正交特征向量。

- Hessian 告诉我们函数在 critical point（gradient 为零处）附近的形状：

    - 若 $H$ 正定（所有特征值为正），该点是 **局部极小值**，曲面在每个方向都向上弯曲，像一个碗。
    - 若 $H$ 负定（所有特征值为负），该点是 **局部极大值**，曲面在每个方向都向下弯曲，像一个倒扣的碗。
    - 若 $H$ 既有正又有负特征值，该点是 **鞍点**，曲面在某些方向向上弯曲、在其他方向向下弯曲，像一个山口。

- **多元 chain rule** 把 chain rule 推广到多变量函数。若 $z = f(x, y)$，其中 $x = g(t)$、$y = h(t)$，则：

$$\frac{dz}{dt} = \frac{\partial f}{\partial x}\frac{dx}{dt} + \frac{\partial f}{\partial y}\frac{dy}{dt}$$

- 从 $t$ 到 $z$ 的每条路径贡献一项：沿该路径的 partial derivative 乘以中间变量对 $t$ 的 derivative。

- 例如，若 $z = x^2 y + 3x - y^2$，$x = \cos(t)$，$y = \sin(t)$：

$$\frac{dz}{dt} = (2xy + 3)(-\sin t) + (x^2 - 2y)(\cos t)$$

- 除了手算 derivative，还有三种方法：

    - **数值微分**：用 $f'(x) \approx \frac{f(x+h) - f(x-h)}{2h}$ 对小 $h$ 作近似。简单但有噪声且不精确。
    - **符号微分**：代数地应用求导法则，得到精确公式。可能产生按指数增长过大的表达式。
    - **自动微分（autodiff）**：跟踪运算链并高效地计算精确 derivative。这就是 JAX、PyTorch 和 TensorFlow 所使用的方法。它给出精确的数值（不是近似），又不会产生臃肿的符号表达式。

## 编程任务（使用 CoLab 或 notebook）

1. 用 `jax.grad` 计算 $f(x, y) = x^2 y + 3x - 2y$ 在点 $(1, 2)$ 处的 gradient。由于 $f$ 接受向量输入，需使用带 `argnums` 的 `jax.grad`。
```python
import jax
import jax.numpy as jnp

def f(x, y):
    return x**2 * y + 3*x - 2*y

df_dx = jax.grad(f, argnums=0)
df_dy = jax.grad(f, argnums=1)

x, y = 1.0, 2.0
print(f"∂f/∂x = {df_dx(x, y):.4f}  (expected: {2*x*y + 3:.4f})")
print(f"∂f/∂y = {df_dy(x, y):.4f}  (expected: {x**2 - 2:.4f})")
```

2. 用 `jax.jacobian` 计算向量值函数的 Jacobian，并与手算结果比较。
```python
import jax
import jax.numpy as jnp

def F(x):
    return jnp.array([x[0]**2 + x[1], x[0] * x[1]**2])

J = jax.jacobian(F)
x = jnp.array([1.0, 2.0])
print(f"Jacobian at (1,2):\n{J(x)}")
# Expected: [[2*x[0], 1], [x[1]**2, 2*x[0]*x[1]]] = [[2, 1], [4, 4]]
```

3. 用 `jax.hessian` 计算 $f(x, y) = x^3 + 2xy^2 - y^3$ 的 Hessian，并验证其对称性。
```python
import jax
import jax.numpy as jnp

def f(xy):
    x, y = xy[0], xy[1]
    return x**3 + 2*x*y**2 - y**3

H = jax.hessian(f)
point = jnp.array([1.0, 2.0])
hess = H(point)
print(f"Hessian:\n{hess}")
print(f"Symmetric: {jnp.allclose(hess, hess.T)}")
# Expected: [[6x, 4y], [4y, 4x-6y]] = [[6, 8], [8, -8]]
```

4. 从零开始搭建一个最小的自动微分引擎。
    - 每个 `Var` 跟踪自己的值以及如何沿 chain rule 反向传播 gradient。
    - 尝试用更多运算（除法、幂运算等）扩展它。
    - 这正是 JAX、PyTorch 和 Numpy 设计的基础。
```python
class Var:
    def __init__(self, val, children=(), backward_fn=None):
        self.val = val
        self.grad = 0.0
        self.children = children
        self.backward_fn = backward_fn

    def __add__(self, other):
        out = Var(self.val + other.val, children=(self, other))
        def _backward():
            self.grad += out.grad    # d(a+b)/da = 1
            other.grad += out.grad   # d(a+b)/db = 1
        out.backward_fn = _backward
        return out

    def __mul__(self, other):
        out = Var(self.val * other.val, children=(self, other))
        def _backward():
            self.grad += other.val * out.grad  # d(a*b)/da = b
            other.grad += self.val * out.grad  # d(a*b)/db = a
        out.backward_fn = _backward
        return out

    def backward(self):
        # topological sort then propagate gradients
        # we will go through this in data structures and algorithms
        order, visited = [], set()
        def topo(v):
            if v not in visited:
                visited.add(v)
                for c in v.children:
                    topo(c)
                order.append(v)
        topo(self)
        self.grad = 1.0
        for v in reversed(order):
            if v.backward_fn:
                v.backward_fn()

# f(x, y) = x*x*y + x  at (3, 2)
x = Var(3.0)
y = Var(2.0)
f = x * x * y + x       # = 3*3*2 + 3 = 21

f.backward()
print(f"f = {f.val}")           # 21.0
print(f"df/dx = {x.grad}")     # 2*x*y + 1 = 13.0
print(f"df/dy = {y.grad}")     # x*x = 9.0
```
