# 优化

*优化是模型训练的数学核心，用于寻找使 loss 函数最小化的参数。本文件涵盖 critical point、凸性、梯度下降、牛顿法、用 Lagrange 乘子的约束 optimization，以及驱动现代深度学习的优化器（SGD、Adam）。*

- 训练神经网络、拟合回归直线、调超参数：几乎每个 ML 算法的核心都是一个 **optimization** 问题。

- 我们有某个函数（loss、cost、objective），想找到使其尽可能小（或大）的输入。

- 在优化之前，我们需要理解函数的 **零点**（或根）。$f(x)$ 的零点是使 $f(x) = 0$ 的 $x$ 值。图像上这些是 x 截距。

- 例如，$f(x) = x^2 - 3x + 2 = (x-1)(x-2)$ 在 $x = 1$ 和 $x = 2$ 处有零点。两个零点之间函数为负（$f(1.5) = -0.25$）；零点之外函数为正。零点把数轴分成函数符号恒定的区域。

- 零点的 **重数** 是相应因式出现的次数。

- 在单重零点（重数 1）处，图像穿过 x 轴。在二重零点（重数 2）处，图像触碰 x 轴但反弹回不穿过，在该点显得"平坦"。

- 寻找零点之所以重要，是因为 derivative $f'(x)$ 的零点就是 $f(x)$ 的 **critical point**，即极大值和极小值的候选。

- 在极大值或极小值处，切线是水平的（斜率 = 0），所以 $f'(x) = 0$。

![critical point：导数为零处，函数有峰、谷或鞍点](../images/critical_points.svg)

- 但并非每个 critical point 都是极大值或极小值。$f'(x) = 0$ 的点也可能是 **拐点**（如 $f(x) = x^3$ 在 $x = 0$ 处），函数在那里瞬时变平但不改变方向。

- **二阶导数判别法** 解决这一问题。在满足 $f'(c) = 0$ 的 critical point $x = c$ 处：

    - 若 $f''(c) > 0$：曲线上凹（像碗），所以 $c$ 是 **局部极小值**。
    - 若 $f''(c) < 0$：曲线下凹（像山丘），所以 $c$ 是 **局部极大值**。
    - 若 $f''(c) = 0$：判别法无法定论；需要更高阶导数或其他方法。

- 例如，$f(x) = x^3 - 3x$。derivative 为 $f'(x) = 3x^2 - 3 = 3(x-1)(x+1)$，所以 critical point 在 $x = -1$ 和 $x = 1$ 处。二阶 derivative 为 $f''(x) = 6x$。在 $x = -1$ 处：$f''(-1) = -6 < 0$（局部极大）。在 $x = 1$ 处：$f''(1) = 6 > 0$（局部极小）。

- 如果连接函数图像上任意两点的线段位于图像上方（或其上），则函数是 **凸的**。可以想象成一个碗形，处处向上弯曲。数学上，若对所有 $x$ 都有 $f''(x) \geq 0$，则 $f$ 是凸的。

![凸函数有唯一全局最小值；非凸函数可能有许多局部极小值](../images/convex_nonconvex.svg)

- 凸性之所以强大，是因为凸函数有一个显著性质：每个局部极小值同时也是 **全局最小值**。没有会困住人的欺骗性局部谷底。如果把一个球滚进凸碗，它总会到达底部。

- 若 $-f$ 是凸的，则函数是 **凹的**（向下弯曲）。函数在凹与凸之间过渡的点为 **拐点**，出现在 $f''(x) = 0$ 处。

- **牛顿法** 利用切线寻找函数的零点（进而寻找其 derivative 的 critical point）。从一个初始猜测 $x_0$ 出发，迭代地精化：

$$x_{n+1} = x_n - \frac{f(x_n)}{f'(x_n)}$$

![牛顿法：沿切线找到对根的更好逼近](../images/newtons_method.svg)

- 思路是：在 $x_n$ 处作切线，求它与 x 轴的交点。该交点就是 $x_{n+1}$。对于良态函数和良好的起点，牛顿法收敛极快（二次收敛，即正确位数每步大约翻倍）。

- 例如，求 $\sqrt{5}$（$f(x) = x^2 - 5$ 的零点）：$f'(x) = 2x$，所以 $x_{n+1} = x_n - \frac{x_n^2 - 5}{2x_n}$。从 $x_0 = 2$ 出发：$x_1 = 2.25$，$x_2 = 2.2361\ldots$，已经精确到四位小数。

- 若初始猜测离根太远、根附近 $f'(x) = 0$，或函数附近有拐点，牛顿法可能失败。它还需要计算 derivative，可能代价较高。

- 对于 optimization（求极小而非零点），我们对 $f'(x) = 0$ 应用牛顿法，得到更新：

$$x_{n+1} = x_n - \frac{f'(x_n)}{f''(x_n)}$$

- 在多维情况下，这变为 $\mathbf{x}_{n+1} = \mathbf{x}_n - H^{-1} \nabla f(\mathbf{x}_n)$，其中 $H$ 是 Hessian 矩阵。这正是上一文件中二阶 Taylor 逼近的实际运用：把函数近似为二次函数，跳到该二次函数的极小值，重复。

- **Lagrange 乘子** 求解 **约束 optimization**：在约束 $g(x, y) = c$ 下求 $f(x, y)$ 的最优。我们不是在整个 $\mathbb{R}^n$ 上搜索，而是被限制在约束成立的集合上（一条曲线或曲面）。

- 关键洞察是几何上的：在约束最优处，$f$ 的 gradient 必须与 $g$ 的 gradient 平行。如果它们不平行，我们就能沿约束朝某个仍能改进 $f$ 的方向移动，那就还没到达最优。

- 我们引入新变量 $\lambda$（Lagrange 乘子），定义 **拉格朗日函数**：

$$\mathcal{L}(x, y, \lambda) = f(x, y) - \lambda(g(x, y) - c)$$

- 令所有偏导数为零，得到一个方程组，其解即约束最优：

$$\frac{\partial \mathcal{L}}{\partial x} = 0, \quad \frac{\partial \mathcal{L}}{\partial y} = 0, \quad \frac{\partial \mathcal{L}}{\partial \lambda} = 0$$

![Lagrange 乘子：在最优处，f 与 g 的梯度平行](../images/lagrange_multiplier.svg)

- 例如，在 $x^2 + y^2 = 1$ 约束下最大化 $f(x,y) = x^2 y$。Lagrangian 为 $\mathcal{L} = x^2 y - \lambda(x^2 + y^2 - 1)$。求偏导：

$$2xy - 2\lambda x = 0, \quad x^2 - 2\lambda y = 0, \quad x^2 + y^2 = 1$$

- 由第一个方程（假设 $x \neq 0$）：$\lambda = y$。代入第二个：$x^2 = 2y^2$。结合约束：$2y^2 + y^2 = 1$，所以 $y = \frac{1}{\sqrt{3}}$。最大值为 $f = \frac{2}{3\sqrt{3}}$。

- 对于不等式约束（$g(x,y) \leq c$ 而非 $= c$），**Karush-Kuhn-Tucker（KKT）条件** 推广了 Lagrange 乘子。约束要么是积极的（起作用，按等式处理），要么是消极的（解位于内部，约束无关）。

- 在实践中，我们很少手算优化。以下是主要的算法族：

    - **一阶方法**（只用 gradient）：梯度下降、随机梯度下降（SGD）、Adam。每步代价低但收敛可能很慢，尤其在病态条件问题上。

    - **二阶方法**（用 gradient 和 Hessian）：牛顿法收敛快，但计算和求逆 Hessian 代价高（对 $n$ 个参数为 $O(n^3)$）。**拟牛顿法**（如 BFGS 和 L-BFGS）只用 gradient 信息近似 Hessian，收敛快于一阶方法，又没有二阶方法的全部代价。

    - **共轭梯度法**：适用于大型稀疏系统，只用矩阵-向量乘积而不存储完整 Hessian。

    - **Gauss-Newton** 和 **Levenberg-Marquardt**：专用于最小二乘问题（回归中常见），通过 Jacobian 近似 Hessian。

    - **自然梯度下降**：用 Fisher 信息矩阵考虑参数空间的几何，对概率模型可能更有效。

- 优化器的选择取决于问题。对于深度学习，一阶方法（尤其是 Adam）占主导，因为参数数量极大（数百万到数十亿），计算 Hessian 不切实际。对于较小的光滑目标问题，二阶方法可能快得多。

## 编程任务（使用 CoLab 或 notebook）

1. 实现牛顿法求 $\sqrt{7}$（$f(x) = x^2 - 7$ 的零点），观察快速收敛。
```python
import jax.numpy as jnp

f = lambda x: x**2 - 7
df = lambda x: 2*x

x = 3.0  # initial guess
for i in range(6):
    x = x - f(x) / df(x)
    print(f"step {i+1}: x = {x:.10f}  (error: {abs(x - jnp.sqrt(7.0)):.2e})")
```

2. 用梯度下降最小化 $f(x, y) = (x - 3)^2 + (y + 1)^2$。极小值在 $(3, -1)$ 处。尝试不同的学习率。
```python
import jax
import jax.numpy as jnp

def f(params):
    x, y = params
    return (x - 3)**2 + (y + 1)**2

grad_f = jax.grad(f)
params = jnp.array([0.0, 0.0])
lr = 0.1

for i in range(20):
    g = grad_f(params)
    params = params - lr * g
    if i % 5 == 0 or i == 19:
        print(f"step {i:2d}: ({params[0]:.4f}, {params[1]:.4f})  loss={f(params):.6f}")
```

3. 数值求解一个约束 optimization 问题。在 $x + y = 10$ 约束下最大化 $f(x,y) = xy$：把 $y$ 参数化为 $y = 10 - x$，然后求该一元函数的最优。
```python
import jax
import jax.numpy as jnp

# Substitute constraint: y = 10 - x, so f = x(10 - x) = 10x - x²
f = lambda x: x * (10 - x)
df = jax.grad(f)

# Gradient ascent (we want maximum, so add gradient)
x = 1.0
lr = 0.1
for i in range(20):
    x = x + lr * df(x)
print(f"x={x:.4f}, y={10-x:.4f}, f={f(x):.4f}")  # should be x=5, y=5, f=25
```
