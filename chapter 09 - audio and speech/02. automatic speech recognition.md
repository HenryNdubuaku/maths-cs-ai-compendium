# Автоматическое распознавание речи

*Автоматическое распознавание речи преобразует аудиозапись человеческой речи в письменный текст, преодолевая разрыв между устной речью и машиночитаемым языком. В этом файле рассматриваются GMM-HMM, функция потерь CTC, RNN-преобразователь (RNN-Transducer), модели энкодер-декодер на основе внимания (LAS), Whisper и end-to-end ASR — от классических пайплайнов до современных нейросетевых архитектур.*

- **Автоматическое распознавание речи** (ASR) — это задача преобразования аудиозаписи речи в письменный текст. Это одна из старейших проблем в области ИИ (первые системы в 1950-х годах распознавали отдельные цифры) и одна из наиболее коммерчески востребованных (голосовые помощники, сервисы транскрибации, создание субтитров).

- Сложность задачи обусловлена огромной вариативностью речи: разные дикторы, акценты, темп речи, фоновый шум, характеристики микрофона, а также фундаментальной неоднозначностью отображения непрерывного акустического сигнала в дискретные слова.

- Представьте ASR как судебного стенографиста. Стенографист слышит непрерывный поток звуков, мысленно сегментирует его на слова, разрешает неоднозначности с помощью контекста (например, различая «they're», «their» и «there») и записывает результат. Система ASR делает то же самое, но по этапам, которые можно сделать явными и оптимизировать независимо или совместно.

- **Классический пайплайн ASR** обрабатывает аудио в цепочке отдельных этапов: необработанное аудио преобразуется в признаки (MFCC или лог-мел спектрограммы, см. файл 01), **акустическая модель** оценивает, насколько хорошо каждый кадр признаков соответствует каждой фонетической единице, **модель произношения** (лексикон) сопоставляет фонетические единицы со словами, **языковая модель** оценивает вероятность последовательностей слов, а **декодер** ищет последовательность слов, максимизирующую совокупную оценку. Каждый компонент обучается и настраивается отдельно.

- **Фонемы** — это мельчайшие единицы звука, различающие слова в языке. В английском языке насчитывается примерно 39–44 фонемы (точное число зависит от диалекта и используемого набора фонем). Например, слова «bat» и «pat» различаются одной фонемой (/b/ против /p/). Большинство систем ASR моделируют **контекстно-зависимые фонемы**, называемые **трифонами**: фонема, определяемая своими левым и правым соседями (например, «a» в контексте «b_t» — это единица, отличная от «a» в «c_t»), поскольку акустическая реализация фонемы сильно зависит от соседей (это называется **коартикуляцией**).

- Количество возможных трифонов огромно (40 фонем в кубе = 64 000), поэтому **кластеризация с помощью деревьев решений** группирует акустически похожие трифоны в **сеноны** (обычно 2000–10 000 классов). Для каждого сенона создается своя акустическая модель. Эта кластеризация является разновидностью алгоритмов деревьев решений из главы 06.

- **GMM-HMM** (смесь гауссовых распределений — скрытая марковская модель) была доминирующим подходом к акустическому моделированию с 1980-х по начало 2010-х годов. HMM (из главы 05) моделирует временную структуру речи: каждая фонема представляет собой HMM слева направо с 3–5 состояниями, где каждое состояние представляет собой субфонетический сегмент (начало, середина, конец). Переходы между состояниями неявно моделируют длительность.

- В каждом состоянии HMM вероятность эмиссии (насколько вероятен конкретный вектор признаков при заданном состоянии) моделируется с помощью **смеси гауссовых распределений** (GMM): взвешенной суммы многомерных гауссовых распределений (из главы 05):

```math
p(\mathbf{x} | s) = \sum_{m=1}^{M} w_m \cdot \mathcal{N}(\mathbf{x} ; \boldsymbol{\mu}_m, \boldsymbol{\Sigma}_m)
```

- где $\mathbf{x}$ — вектор признаков (например, 39-мерные MFCC), $s$ — состояние HMM, $M$ — количество компонентов смеси (обычно 8–64), $w_m$ — веса смеси, а $\boldsymbol{\mu}_m$ и $\boldsymbol{\Sigma}_m$ — среднее значение и ковариация каждого гауссова компонента. Ковариационные матрицы обычно являются диагональными для вычислительной эффективности (в предположении, что размерности признаков независимы, что приблизительно верно для MFCC благодаря декорреляции DCT).

- Обучение использует **алгоритм Баума-Велша** (частный случай EM-алгоритма из главы 05) для итеративной оценки параметров GMM и вероятностей переходов HMM на основе данных транскрибированной речи. Декодирование (поиск наиболее вероятной последовательности состояний) использует **алгоритм Витерби** (динамическое программирование, из главы 05):

```math
\delta_t(j) = \max_{i} \left[ \delta_{t-1}(i) \cdot a_{ij} \right] \cdot b_j(\mathbf{x}_t)
```

- где $\delta_t(j)$ — вероятность наилучшего пути, заканчивающегося в состоянии $j$ в момент времени $t$, $a_{ij}$ — вероятность перехода из состояния $i$ в состояние $j$, а $b_j(\mathbf{x}_t)$ — вероятность эмиссии признака $\mathbf{x}_t$ в состоянии $j$.

- **DNN-HMM** (Hinton et al., 2012) заменила модель эмиссии GMM на глубокую нейронную сеть (DNN, из главы 06), которая предсказывает апостериорные вероятности сенонов $p(s | \mathbf{x})$ на основе окна кадров признаков. HMM по-прежнему отвечает за временную структуру и последовательность, но нейронная сеть обеспечивает гораздо более дискриминативные оценки эмиссии. Этот гибридный подход снизил частоту ошибок в словах (WER) на 20–30% по сравнению с GMM и был доминирующей парадигмой в 2012–2016 годах.

- **Декодирование с помощью WFST** (взвешенный конечный преобразователь) является стандартным фреймворком декодирования для традиционных систем ASR. Каждый компонент (топология HMM H, контекстная зависимость C, лексикон L, грамматика/языковая модель G) представляется как взвешенный конечный преобразователь, и они объединяются в единый поисковый граф $H \circ C \circ L \circ G$. Поиск по Витерби затем находит путь с наименьшей стоимостью в этом составном графе. WFST позволяют модульно комбинировать источники знаний и эффективно выполнять поиск методом динамического программирования. Математический аппарат основан на теории конечных автоматов (связанной с конечными автоматами из главы 05).

- **End-to-end ASR** исключает отдельные компоненты (модель произношения, набор фонем, декодер WFST) и обучает единую нейронную сеть, которая отображает аудиопризнаки непосредственно в символы или части слов. Ключевой проблемой является **проблема выравнивания**: входные данные (сотни кадров признаков в секунду) и выходные данные (несколько символов в секунду) имеют очень разную длину, а выравнивание между ними неизвестно во время обучения.

- **Connectionist Temporal Classification** (CTC) (Graves et al., 2006) решает проблему выравнивания путем введения специального токена **blank** (пустой символ) и позволяя сети выводить любую последовательность символов и пустых символов, при условии, что схлопывание последовательных дубликатов и удаление пустых символов дает правильную транскрипцию. Например, транскрипция "cat" может быть получена выходной последовательностью "--cc-aa-t--" (где "-" — это пустой символ).

- Формально CTC определяет отображение $\mathcal{B}$ «многие-к-одному» из множества всех выходных последовательностей длины $T$ (над алфавитом плюс пустой символ) в последовательности меток. Вероятность последовательности меток $\mathbf{y}$ — это сумма по всем выравниваниям, которые схлопываются в нее:

$$P(\mathbf{y} | \mathbf{x}) = \sum_{\boldsymbol{\pi} \in \mathcal{B}^{-1}(\mathbf{y})} \prod_{t=1}^{T} p(\pi_t | \mathbf{x})$$

![Выравнивание CTC, показывающее множество возможных путей через пустые символы и токены символов, которые схлопываются в один и тот же выходной текст](../images/ctc_alignment.svg)

- Наивное вычисление этой суммы потребовало бы перечисления экспоненциально большого количества выравниваний, но **алгоритм прямого-обратного прохода CTC** вычисляет ее эффективно за время $O(T \cdot |\mathbf{y}|)$ с использованием динамического программирования, аналогично алгоритму прямого-обратного прохода для скрытых марковских моделей (HMM) из главы 05.

- CTC делает **предположение о условной независимости**: выходные данные на каждом временном шаге независимы от всех остальных выходных данных при заданном входе. Это означает, что CTC не может моделировать зависимости выходных данных (например, он не может выучить, что за "q" почти всегда следует "u"). Для обработки таких зависимостей необходимо использовать внешнюю языковую модель.

- Варианты **декодирования CTC**:
    - **Жадное декодирование (Greedy decoding)**: выбор наиболее вероятного токена на каждом временном шаге с последующим схлопыванием. Быстро, но субоптимально.
    - **Поиск по лучу (Beam search)**: поддержание топ-$k$ частичных гипотез на каждом шаге с объединением гипотез, которые схлопываются в один и тот же префикс. Позволяет учитывать оценку языковой модели.
    - **Префиксный поиск по лучу (Prefix beam search)**: модифицированный поиск по лучу, который корректно обрабатывает схлопывание пустых символов CTC, гарантируя, что гипотезы сравниваются после схлопывания.

- **RNN-Transducer** (RNN-T) (Graves, 2012) расширяет CTC путем добавления явной **сети предсказания** (RNN, подобной языковой модели), которая обуславливает каждый выход предыдущими выходами, устраняя предположение об условной независимости. RNN-T состоит из трех компонентов:
    - **Энкодер**: обрабатывает аудиопризнаки для получения скрытых представлений $\mathbf{h}_t^\text{enc}$ (обычно стек слоев LSTM или Conformer).
    - **Сеть предсказания**: авторегрессионная RNN, которая создает скрытые представления $\mathbf{h}_u^\text{pred}$ на основе ранее выданных меток.
    - **Совместная сеть (Joint network)**: объединяет выходы энкодера и сети предсказания в каждой позиции (время, метка) и создает распределение по следующему токену (включая пустой символ):

$$p(y | t, u) = \text{softmax}(W \cdot \text{tanh}(W_\text{enc} \mathbf{h}_t^\text{enc} + W_\text{pred} \mathbf{h}_u^\text{pred} + b))$$

- RNN-T может выдавать ноль или более меток на временной шаг (путем выдачи непустых токенов перед переходом к следующему временному шагу или выдачи пустого символа для перехода без вывода). Обучение использует алгоритм прямого-обратного прохода по двумерной решетке (время, метка) со сложностью $O(T \cdot U)$, где $U$ — длина выхода. RNN-T является доминирующей архитектурой для потокового распознавания речи (ASR) на устройствах (используется в телефонах Google Pixel и аналогичных продуктах), поскольку она естественным образом поддерживает потоковую передачу: энкодер обрабатывает аудио слева направо, а сеть предсказания генерирует выход инкрементально.

- **Listen, Attend and Spell** (LAS) (Chan et al., 2016) — это модель энкодера-декодера на основе внимания (архитектура «последовательность-в-последовательность» из главы 06). Она состоит из трех компонентов:
    - **Слушатель (Listener, энкодер)**: пирамидальная двунаправленная LSTM, которая обрабатывает полную входную последовательность и выполняет понижающую дискретизацию с коэффициентом 8 (путем конкатенации пар последовательных скрытых состояний на каждом слое), создавая более короткую последовательность скрытых состояний энкодера.
    - **Внимание (Attention)**: на каждом шаге декодера вычисляет веса внимания по всем состояниям энкодера для формирования контекстного вектора (тот же механизм внимания из главы 07).
    - **Писатель (Speller, декодер)**: авторегрессионная LSTM, которая генерирует выходную транскрипцию по одному символу за раз, обусловленную контекстным вектором и ранее сгенерированными символами.

- LAS достигает высоких результатов, но требует наличия полной фразы перед началом декодирования (поскольку механизм внимания обращается ко всем состояниям энкодера), что делает ее непригодной для потоковых приложений. Она также испытывает трудности с очень длинными фразами, так как внимание на длинных последовательностях становится размытым.

- **Conformer** (Gulati et al., 2020) сочетает способность сверток к выявлению локальных паттернов с моделированием глобальных зависимостей с помощью самовнимания. Каждый блок Conformer имеет четыре модуля в структуре «сэндвича»:
    1. **Модуль прямой связи (Feed-forward module)** (полушаг): сеть прямой связи с остаточной связью, использующая половину веса остатка.
    2. **Модуль многоголового самовнимания**: стандартное самовнимание трансформера (из главы 07) с относительным позиционным кодированием.
    3. **Сверточный модуль**: поточечная свертка, GLU (gated linear unit), одномерная глубинная свертка, пакетная нормализация, активация Swish и еще одна поточечная свертка. Глубинная свертка улавливает локальный контекст (подобно n-грамме по последовательности признаков).
    4. **Модуль прямой связи** (полушаг): идентичен модулю 1.

- Выход имеет вид: $\mathbf{y} = \text{LayerNorm}(\mathbf{x} + \frac{1}{2}\text{FFN}_1 + \text{MHSA} + \text{Conv} + \frac{1}{2}\text{FFN}_2)$. Эмпирически было установлено, что структура типа «макарон» (FFN-Attention-Conv-FFN) с полушаговыми остатками превосходит другие варианты порядка слоев. Conformer стали стандартным энкодером как для систем CTC, так и для RNN-T, превосходя чистые трансформеры и чистые LSTM-энкодеры.

- **Whisper** (Radford et al., 2023) — это крупномасштабная модель ASR на основе внимания от OpenAI. Она использует стандартную архитектуру трансформера энкодер-декодер (из главы 07), обученную на 680 000 часах слабо размеченных данных, собранных из интернета (аудио в паре с приблизительными транскрипциями). Ключевые проектные решения:
    - Вход: 80-канальная лог-мел спектрограмма (из файла 01) с окнами 25 мс и шагом 10 мс, нормализованная к нулевому среднему и единичной дисперсии.
    - Энкодер: стандартный трансформерный энкодер с синусоидальными позиционными эмбеддингами и нормализацией слоев с предварительной активацией.
    - Декодер: трансформерный декодер, который авторегрессионно генерирует токены с использованием побайтового BPE-токенизатора (из главы 07).
    - Многозадачность: одна модель обрабатывает транскрипцию, перевод, идентификацию языка и предсказание временных меток, обусловленные специальными токенами задачи в промпте декодера.
    - Масштаб обучающих данных (а не новизна архитектуры) является основным фактором высокой способности Whisper к обобщению на различные домены, акценты и языки.

- **wav2vec 2.0** (Baevski et al., 2020) — это фреймворк **самообучения** (self-supervised) для предварительного обучения представлений речи. Основная идея заключается в изучении представлений речи на больших объемах неразмеченных аудиоданных с последующей донастройкой (fine-tuning) на небольшом количестве размеченных данных. Этот подход следует той же парадигме самообучения, что и BERT (из главы 07), но адаптирован для непрерывных аудиосигналов.

- Архитектура wav2vec 2.0 состоит из трех частей:
    - **Признаковый энкодер (Feature encoder)**: многослойная одномерная свёрточная нейронная сеть (CNN), которая обрабатывает сырые отсчеты аудиосигнала и создает латентные представления $\mathbf{z}_t$ с частотой кадров 20 мс (один вектор на каждые 320 отсчетов при частоте дискретизации 16 кГц).
    - **Модуль квантования (Quantisation module)**: дискретизирует латентные представления в конечное кодовое пространство (codebook) с использованием **векторного квантования произведений (product quantisation)** (разделение вектора на группы и независимое квантование каждой группы с выбором из $G$ кодовых книг по $V$ записей в каждой). Это создает целевые значения $\mathbf{q}_t$ для задачи контрастивного обучения.
    - **Контекстная сеть (Context network)**: трансформерный энкодер, который принимает (частично замаскированные) латентные представления и создает контекстуализированные представления $\mathbf{c}_t$.

- Во время предварительного обучения случайные фрагменты латентных представлений **маскируются** (заменяются на обученный эмбеддинг маски), и модель должна идентифицировать истинное квантованное представление замаскированной позиции среди набора дистракторов (отрицательных примеров, выбранных из других позиций в том же высказывании). Контрастивная функция потерь имеет вид:

$$\mathcal{L} = -\log \frac{\exp(\text{sim}(\mathbf{c}_t, \mathbf{q}_t) / \kappa)}{\sum_{\tilde{\mathbf{q}} \in Q_t} \exp(\text{sim}(\mathbf{c}_t, \tilde{\mathbf{q}}) / \kappa)}$$

- где $\text{sim}$ — косинусное сходство, $\kappa$ — параметр температуры, а $Q_t$ включает истинную квантованную цель и дистракторы. Дополнительная **функция потерь на разнообразие (diversity loss)** поощряет равномерное использование всех записей кодовой книги. Эта функция потерь по сути является контрастивной функцией потерь InfoNCE — того же семейства контрастивных целевых функций, которые используются в визуальном самообучении.

- После предварительного обучения поверх модели добавляются линейная проекция и CTC-голова, после чего модель донастраивается на размеченных данных. wav2vec 2.0 достиг результатов, близких к передовому уровню (state of the art), всего на 10 минутах размеченных данных (используя 53 000 часов неразмеченного аудио для предварительного обучения), что демонстрирует эффективность самообучения для задач распознавания речи с малым количеством ресурсов.

- **HuBERT** (Hsu et al., 2021) — еще один подход к самообучению, который заменяет контрастивную целевую функцию на задачу **маскированного предсказания** (предсказание дискретных кластерных назначений для замаскированных кадров). Целевые значения создаются с помощью автономного этапа кластеризации (k-means на MFCC в первой итерации, затем k-means на признаках HuBERT в последующих итерациях). HuBERT упрощает пайплайн обучения по сравнению с wav2vec 2.0 (не требуется модуль квантования или контрастивная выборка) и достигает сопоставимых или лучших результатов.

- **Fast Conformer** (Rekesh et al., 2023, NVIDIA NeMo) заменяет квадратичное самовнимание в стандартном Conformer на механизм **понижающего дискретизацию внимания (down-sampled attention)**: входная последовательность сжимается (обычно в 8 раз с помощью свёртки со страйдом) перед вычислением внимания, а затем разворачивается обратно. Это снижает вычислительную сложность внимания с $O(T^2)$ до $O(T^2/64)$ при сохранении глобального контекста, что позволяет проводить обучение на очень длинных высказываниях (до нескольких минут) без проблем с памятью. Fast Conformer является энкодером по умолчанию в инструментарии NVIDIA NeMo и составляет основу их моделей промышленного уровня.

- **Parakeet** (NVIDIA, 2024) — это семейство высокоточных моделей распознавания английской речи (ASR), построенных на энкодере Fast Conformer с декодерами CTC и RNN-T, обученных на 64 000 часах английской речи. Модели Parakeet (0,6 млрд и 1,1 млрд параметров) на момент выпуска достигли самых низких показателей частоты ошибок в словах (WER) на стандартных бенчмарках, превзойдя Whisper large-v3 на большинстве тестовых наборов для английского языка. Ключевыми компонентами являются эффективная архитектура Fast Conformer, агрессивная аугментация данных (SpecAugment, изменение скорости, смешивание шума) и крупномасштабные данные для обучения с учителем, что доказывает, что тщательная инженерная проработка известных компонентов все еще может продвигать передовой уровень (state of the art).

- **Canary** (NVIDIA, 2024) расширяет фреймворк NeMo для многоязычного и многозадачного распознавания речи. Он использует энкодер Fast Conformer с декодером на основе внимания (вместо CTC или RNN-T) и выполняет транскрибирование и перевод для нескольких языков в рамках одной модели (подобно многозадачному дизайну Whisper, но с более эффективным бэкендом Fast Conformer). Модели Canary поддерживают английский, немецкий, испанский и французский языки с конкурентоспособной точностью.

- **Moonshine** (Useful Sensors, 2024) — это семейство моделей ASR, специально оптимизированных для **развертывания на устройствах и периферийных вычислений (edge deployment)**. Энкодер использует гибридную архитектуру, которая заменяет начальные слои трансформера/conformer на небольшую CNN, за которой следуют несколько слоев трансформера, что значительно уменьшает размер модели (базовая модель содержит менее 30 млн параметров). Moonshine нацелен на потоковую передачу в реальном времени на CPU и маломощных устройствах, где Whisper был бы слишком большим и медленным, жертвуя частью точности ради 5–10-кратного снижения задержки и объема используемой памяти.

- **Distil-Whisper** (Gandhi et al., 2023) применяет **дистилляцию знаний** (глава 06) для сжатия Whisper в меньшую и более быструю модель. Модель-ученик использует только 2 слоя декодера (по сравнению с 32 слоями у Whisper), сохраняя при этом полный энкодер, и обучается соответствовать распределениям выходов Whisper. Distil-Whisper достигает показателя WER в пределах 1% от модели-учителя, будучи при этом в 6 раз быстрее, что делает ее практичной для приложений реального времени, где полная модель Whisper работает слишком медленно.

- **Universal Speech Model (USM)** (Zhang et al., 2023, Google) масштабирует самообучение до 12 миллионов часов неразмеченного аудио на более чем 300 языках с последующей донастройкой с учителем. USM демонстрирует, что парадигма wav2vec 2.0 / самообучения масштабируется до действительно огромных объемов данных, достигая высокой производительности на языках с низкими ресурсами при очень ограниченном количестве размеченных данных.

- **Massively Multilingual Speech (MMS)** (Pratap et al., 2023, Meta) расширяет предварительное обучение wav2vec 2.0 на более чем 1100 языков, используя религиозные записи и другие источники многоязычных аудиоданных. MMS охватывает гораздо больше языков, чем любая предыдущая система ASR, впервые обеспечивая распознавание речи для многих языков с ограниченными ресурсами.

- Ландшафт современного ASR сходится к нескольким доминирующим паттернам: (1) энкодеры семейства Conformer с CTC или RNN-T для потоковой передачи, (2) трансформеры с архитектурой энкодер-декодер для офлайн-задач или многозадачного обучения, (3) самообучение (self-supervised pre-training) для условий с малым объемом данных и (4) масштабирование — больше данных и более крупные модели стабильно повышают точность. Выбор между ними зависит от ограничений развертывания: бюджета задержки, доступных вычислительных мощностей, количества языков и того, является ли приложение потоковым или пакетным.

- **Интеграция языковой модели** улучшает ASR за счет включения лингвистических знаний, выходящих за рамки того, что улавливает акустическая модель. Основная идея заключается в объединении оценки акустической модели $p(\mathbf{x} | \mathbf{y})$ (насколько хорошо аудио соответствует транскрипции) с оценкой языковой модели $p(\mathbf{y})$ (насколько вероятна транскрипция как предложение) во время декодирования.

- **Shallow fusion** (поверхностная интеграция) объединяет оценки во время поиска по лучу (beam search):

$$\hat{\mathbf{y}} = \arg\max_\mathbf{y} \left[ \log p_\text{AM}(\mathbf{y} | \mathbf{x}) + \lambda \log p_\text{LM}(\mathbf{y}) \right]$$

- где $\lambda$ — настраиваемый вес, а $p_\text{LM}$ — внешняя языковая модель (обычно n-грамма или нейронная языковая модель из главы 07). Это простой и эффективный метод, но он требует, чтобы языковая модель работала с тем же словарем токенов, что и модель ASR.

- **Deep fusion** (глубокая интеграция) (Gulcehre et al., 2015) встраивает языковую модель внутрь сети декодера: скрытое состояние языковой модели конкатенируется со скрытым состоянием декодера и проходит через механизм гейтинга перед проекцией на выход. Вся система (включая предобученную языковую модель) дообучается совместно. Это позволяет добиться более глубокой интеграции, но требует более сложных методов обучения.

- **Cold fusion** (холодная интеграция) (Sriram et al., 2018) похожа на глубокую интеграцию, но обучает декодер ASR с нуля с уже интегрированной языковой моделью, а не дообучает предобученный декодер. Это заставляет акустическую модель изучать дополнительную информацию, а не дублировать то, что уже известно языковой модели.

- **Переранжирование** (N-best rescoring) — это двухпроходный подход: сначала генерируется $N$ кандидатных транскрипций с помощью поиска по лучу, а затем они переранжируются с использованием более мощной языковой модели (например, большой трансформерной языковой модели). Это просто реализовать, и такой подход позволяет использовать очень большие языковые модели, которые были бы слишком медленными для декодирования на первом проходе.

- **Оценка внутренней языковой модели** (ILME) решает тонкую проблему: модели end-to-end неявно изучают внутреннюю языковую модель из обучающих транскрипций, что может конфликтовать с внешней языковой моделью во время shallow fusion (по сути, происходит двойной учет лингвистического априорного распределения). ILME оценивает внутреннюю языковую модель и вычитает ее оценку во время интеграции:

$$\hat{\mathbf{y}} = \arg\max_\mathbf{y} \left[ \log p_\text{E2E}(\mathbf{y} | \mathbf{x}) - \beta \log p_\text{ILM}(\mathbf{y}) + \lambda \log p_\text{LM}(\mathbf{y}) \right]$$

- **Потоковое vs. офлайн ASR** — это фундаментальный архитектурный выбор. Офлайн (или пакетное) ASR обрабатывает всю речевую последовательность целиком перед выдачей результата. Потоковое ASR выдает результат инкрементально по мере поступления аудио с ограниченной задержкой.

- Потоковая передача необходима для приложений реального времени: субтитры в прямом эфире, голосовые помощники (пользователь ожидает ответа до того, как закончит говорить), транскрипция телефонных звонков. Сложность заключается в том, что некоторый будущий контекст полезен для распознавания (знание того, что следующее слово — «Йорк», помогает однозначно интерпретировать «Нью»), но потоковые системы не могут ждать произвольно долгого будущего контекста.

- **Однонаправленные энкодеры** (LSTMs слева направо, причинные свёртки, причинные трансформеры) естественным образом поддерживают потоковую передачу, поскольку каждый выход зависит только от прошлых и текущих входных данных. Двунаправленные энкодеры (которые учитывают будущий контекст) не поддерживают потоковую передачу напрямую.

- **Chunked attention** (также называемое блочным или сегментарным вниманием) делит входные данные на чанки фиксированной длины и применяет самовнимание только внутри каждого чанка (и, опционально, к нескольким предыдущим чанкам). Это ограничивает задержку размером чанка плюс временем обработки, при этом сохраняя некоторый локальный двунаправленный контекст внутри каждого чанка. Компромисс заключается в том, что точность снижается по мере уменьшения размера чанка.

- **Lookahead** (упреждающий просмотр) позволяет потоковому энкодеру заглядывать на небольшое количество будущих кадров (например, 300–900 мс) перед выдачей результата для текущего кадра. Это реализуется путем добавления небольшого правого контекста к однонаправленным вычислениям. Окно упреждающего просмотра увеличивает задержку, но значительно повышает точность.

- **Задержка** в потоковом ASR состоит из нескольких компонентов:
    - **Алгоритмическая задержка**: время от момента поступления аудио до момента, когда модель может его обработать (определяется размером чанка, упреждающим просмотром и извлечением признаков).
    - **Вычислительная задержка**: время выполнения прямого прохода модели.
    - **Задержка определения конца фразы**: задержка при обнаружении того, что пользователь закончил говорить.
    - **Задержка первого токена**: как быстро появляется первое слово. **Задержка финализации**: как быстро подтверждается окончательный результат (потоковые системы часто выдают предварительный результат, который корректируется по мере поступления новых аудиоданных).

- **Метрики оценки** для ASR:

- **Word Error Rate** (WER) — основная метрика. Она вычисляется путем сопоставления гипотезы (вывода системы) с эталоном (эталонной транскрипцией) с использованием редакционного расстояния (минимальное количество замен, вставок и удалений, необходимых для преобразования одного в другое), затем:

$$\text{WER} = \frac{S + D + I}{N}$$

- где $S$ — замены, $D$ — удаления, $I$ — вставки, а $N$ — общее количество слов в эталоне. WER может превышать 100%, если много вставок. WER на уровне 5% считается примерно человеческим уровнем для чистой дикторской речи; разговорная или зашумленная речь распознается гораздо сложнее (10–20%+).

- **Character Error Rate** (CER) — это та же формула, применяемая на уровне символов, а не слов. CER более информативен для языков без четких границ слов (китайский, японский) и для оценки близости ошибочных вариантов («cat» против «bat» — это 100% WER, но 33% CER).

- **Word Information Lost** (WIL) и **Word Information Preserved** (WIP) — это теоретико-информационные альтернативы, которые учитывают корреляцию между эталоном и гипотезой точнее, чем WER, однако они используются реже.

- **Real-Time Factor** (RTF) измеряет вычислительную эффективность: отношение времени обработки к длительности аудио. RTF < 1 означает, что система работает быстрее реального времени; RTF > 1 означает, что она не успевает за потоковым аудио. Потоковые системы должны поддерживать RTF < 1.

- **Аугментация данных** (data augmentation) критически важна для устойчивости ASR. Распространенные методы:
    - **Изменение скорости** (speed perturbation): передискретизация аудио с коэффициентами 0.9x и 1.1x (изменяет высоту тона и длительность).
    - **SpecAugment** (Park et al., 2019): маскирование случайных частотных полос и временных шагов в спектрограмме. Это аудиоаналог dropout и один из самых эффективных методов регуляризации для ASR. Он не требует дополнительных данных.
    - **Шумовая аугментация**: смешивание чистой речи с записанным шумом при различных отношениях сигнал/шум.
    - **Симуляция импульсной характеристики помещения**: свертка чистой речи с симулированной акустикой помещения для имитации реверберации.

- **Токенизация** для ASR определяет выходной словарь модели. Варианты включают:
    - **Символы**: просто, малый словарь (~30 для английского), но длинные выходные последовательности и отсутствие неявного языкового моделирования.
    - **Word pieces / BPE** (из главы 07): подсловесные единицы, балансирующие размер словаря и длину последовательности. Стандарт для современных систем (Whisper использует побайтовый BPE с ~50 000 токенов).
    - **Слова**: большой словарь (50 000+), короткие выходные последовательности, но невозможность обработки слов вне словаря.
    - **Фонемы**: лингвистически обоснованы, компактны, но требуют словаря произношений.

- Эволюцию ASR можно представить как переход от сложных модульных систем (GMM-HMM + декодирование WFST, 1990-е–2010-е годы) к гибридным системам (DNN-HMM, 2012–2016 годы), затем к end-to-end системам, которые поглощают все большую часть пайплайна в единую нейронную сеть (CTC, RNN-T, LAS, 2016–2020 годы), и, наконец, к крупномасштабным предобученным моделям, использующим огромные объемы неразмеченных или слабо размеченных данных (wav2vec 2.0, Whisper, 2020–настоящее время). Каждый переход упрощал инженерию, одновременно повышая точность, следуя общей тенденции в машинном обучении — переходу от ручного проектирования признаков к их изучению из данных (та же история, что описана в главе 06 для признаков изображений, замененных CNN, и в главе 07 для признаков NLP, замененных трансформерами).

1. Реализуйте функцию потерь CTC с нуля на JAX. Создайте игрушечный пример с короткой последовательностью логитов и целевой меткой, вычислите прямой алгоритм CTC для получения общей вероятности и вычислите функцию потерь как отрицательное логарифмическое правдоподобие.
```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

def ctc_forward(log_probs, targets):
    """
    CTC forward algorithm (log-domain for numerical stability).
    log_probs: (T, V) log probabilities over vocabulary (index 0 = blank)
    targets: (U,) target label indices (no blanks)
    Returns: log probability of the target sequence under CTC.
    """
    T, V = log_probs.shape
    U = len(targets)

    # Build the extended label sequence with blanks: [blank, y1, blank, y2, ..., yU, blank]
    S = 2 * U + 1
    labels = jnp.zeros(S, dtype=jnp.int32)  # all blanks
    for i in range(U):
        labels = labels.at[2 * i + 1].set(targets[i])

    # Initialise alpha (log domain)
    NEG_INF = -1e30
    alpha = jnp.full((T, S), NEG_INF)
    alpha = alpha.at[0, 0].set(log_probs[0, labels[0]])        # start with blank
    alpha = alpha.at[0, 1].set(log_probs[0, labels[1]])        # or first label

    # Fill forward
    for t in range(1, T):
        for s in range(S):
            # Same state
            a = alpha[t - 1, s]
            # From previous state
            if s > 0:
                a = jnp.logaddexp(a, alpha[t - 1, s - 1])
            # Skip blank (if current and two-back labels are different)
            if s > 1 and labels[s] != 0 and labels[s] != labels[s - 2]:
                a = jnp.logaddexp(a, alpha[t - 1, s - 2])
            alpha = alpha.at[t, s].set(a + log_probs[t, labels[s]])

    # Total log probability: sum of last two states at final time step
    log_prob = jnp.logaddexp(alpha[T - 1, S - 1], alpha[T - 1, S - 2])
    return log_prob, alpha

# --- Toy example ---
T = 12   # input length (time steps)
V = 5    # vocab size (0=blank, 1='c', 2='a', 3='t', 4='x')
targets = jnp.array([1, 2, 3])  # "c", "a", "t"

# Create random logits and convert to log-probabilities
key = jax.random.PRNGKey(42)
logits = jax.random.normal(key, (T, V))
log_probs = jax.nn.log_softmax(logits, axis=-1)

log_prob, alpha = ctc_forward(log_probs, targets)
ctc_loss = -log_prob

print(f"Target sequence: {targets.tolist()} ('c', 'a', 't')")
print(f"Input length T={T}, Vocab size V={V}")
print(f"CTC log-probability: {log_prob:.4f}")
print(f"CTC loss (neg log-prob): {ctc_loss:.4f}")

# Visualise the forward variable (alpha) lattice
fig, ax = plt.subplots(figsize=(12, 5))
# Convert from log to linear for visualisation
alpha_linear = jnp.exp(alpha - jnp.max(alpha))  # normalise for visibility
im = ax.imshow(alpha_linear.T, aspect='auto', origin='lower', cmap='viridis')
ax.set_xlabel('Time step (t)')
ax.set_ylabel('Extended label index (s)')

label_names = ['_', 'c', '_', 'a', '_', 't', '_']  # _ = blank
ax.set_yticks(range(len(label_names)))
ax.set_yticklabels(label_names)
ax.set_title(f'CTC Forward Variable (alpha lattice) | Loss = {ctc_loss:.2f}')
plt.colorbar(im, ax=ax, label='Normalised probability')
plt.tight_layout(); plt.show()
```

2. Постройте простую модель ASR на основе энкодера-декодера с механизмом внимания (минимальная архитектура типа LAS) в JAX. Используйте одномерный сверточный энкодер и однослойный декодер с вниманием на основе скалярного произведения. Запустите её на синтетических данных и визуализируйте веса внимания.

```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

# --- Minimal attention-based encoder-decoder for ASR ---

def init_params(key, input_dim, hidden_dim, vocab_size):
    """Initialise parameters for a tiny LAS-like model."""
    keys = jax.random.split(key, 8)
    scale = 0.1
    params = {
        # Encoder: simple linear projection (simulating conv output)
        'enc_w': jax.random.normal(keys[0], (input_dim, hidden_dim)) * scale,
        'enc_b': jnp.zeros(hidden_dim),
        # Attention: query, key, value projections
        'attn_q': jax.random.normal(keys[1], (hidden_dim, hidden_dim)) * scale,
        'attn_k': jax.random.normal(keys[2], (hidden_dim, hidden_dim)) * scale,
        'attn_v': jax.random.normal(keys[3], (hidden_dim, hidden_dim)) * scale,
        # Decoder RNN (simple Elman RNN for illustration)
        'dec_wh': jax.random.normal(keys[4], (hidden_dim, hidden_dim)) * scale,
        'dec_wx': jax.random.normal(keys[5], (vocab_size, hidden_dim)) * scale,
        'dec_wc': jax.random.normal(keys[6], (hidden_dim, hidden_dim)) * scale,
        'dec_b': jnp.zeros(hidden_dim),
        # Output projection
        'out_w': jax.random.normal(keys[7], (hidden_dim, vocab_size)) * scale,
        'out_b': jnp.zeros(vocab_size),
    }
    return params

def encode(params, x):
    """Encoder: linear projection (placeholder for conv/LSTM stack)."""
    return jnp.tanh(x @ params['enc_w'] + params['enc_b'])

def attend(params, query, enc_out):
    """Dot-product attention over encoder outputs."""
    q = query @ params['attn_q']                   # (hidden,)
    k = enc_out @ params['attn_k']                 # (T_enc, hidden)
    v = enc_out @ params['attn_v']                 # (T_enc, hidden)
    d_k = q.shape[-1]
    scores = (k @ q) / jnp.sqrt(d_k)              # (T_enc,)
    weights = jax.nn.softmax(scores)               # (T_enc,)
    context = weights @ v                          # (hidden,)
    return context, weights

def decode_step(params, h_prev, y_prev_onehot, enc_out):
    """Single decoder step: RNN + attention."""
    # Embed previous token
    y_emb = y_prev_onehot @ params['dec_wx']       # (hidden,)
    # Attend to encoder
    context, attn_w = attend(params, h_prev, enc_out)
    # RNN update
    h = jnp.tanh(h_prev @ params['dec_wh'] + y_emb + context @ params['dec_wc']
                  + params['dec_b'])
    # Output logits
    logits = h @ params['out_w'] + params['out_b']
    return h, logits, attn_w

# --- Setup ---
key = jax.random.PRNGKey(0)
input_dim = 40       # e.g., 40 mel bands
hidden_dim = 64
vocab_size = 10      # small vocab for demo
T_enc = 30           # encoder time steps
T_dec = 8            # decoder steps

params = init_params(key, input_dim, hidden_dim, vocab_size)

# Synthetic input: random mel-like features
key, subkey = jax.random.split(key)
x = jax.random.normal(subkey, (T_enc, input_dim))

# Encode
enc_out = encode(params, x)

# Decode (teacher forcing with random targets)
key, subkey = jax.random.split(key)
targets = jax.random.randint(subkey, (T_dec,), 0, vocab_size)

h = jnp.zeros(hidden_dim)
all_logits = []
all_attn = []

for t in range(T_dec):
    y_prev = jax.nn.one_hot(targets[t] if t > 0 else 0, vocab_size)
    h, logits, attn_w = decode_step(params, h, y_prev, enc_out)
    all_logits.append(logits)
    all_attn.append(attn_w)

all_attn = jnp.stack(all_attn)  # (T_dec, T_enc)
all_logits = jnp.stack(all_logits)  # (T_dec, vocab_size)

# --- Visualise attention weights ---
fig, axes = plt.subplots(1, 2, figsize=(14, 5))

im = axes[0].imshow(all_attn, aspect='auto', cmap='Blues', origin='lower')
axes[0].set_xlabel('Encoder time step')
axes[0].set_ylabel('Decoder step')
axes[0].set_title('Attention Weights (decoder -> encoder)')
plt.colorbar(im, ax=axes[0])

# Show predicted token distribution for each decoder step
im2 = axes[1].imshow(jax.nn.softmax(all_logits, axis=-1), aspect='auto',
                      cmap='Oranges', origin='lower')
axes[1].set_xlabel('Vocabulary index')
axes[1].set_ylabel('Decoder step')
axes[1].set_title('Output Token Probabilities')
plt.colorbar(im2, ax=axes[1])

plt.suptitle('Minimal Attention-based ASR Model (untrained)')
plt.tight_layout(); plt.show()
```

3. Вычислите частоту ошибок в словах (WER) с нуля, используя динамическое программирование (расстояние редактирования), и оцените несколько гипотез относительно эталона. Визуализируйте матрицу расстояния редактирования.

```python
import jax.numpy as jnp
import matplotlib.pyplot as plt
import numpy as np

def compute_wer(reference, hypothesis):
    """
    Compute WER using dynamic programming (Levenshtein distance on words).
    Returns WER, number of substitutions, deletions, insertions, and the DP matrix.
    """
    ref_words = reference.split()
    hyp_words = hypothesis.split()
    N = len(ref_words)
    M = len(hyp_words)

    # DP matrix: d[i][j] = edit distance between ref[:i] and hyp[:j]
    d = np.zeros((N + 1, M + 1), dtype=np.int32)
    # Backtrack matrix to count S, D, I
    ops = np.zeros((N + 1, M + 1, 3), dtype=np.int32)  # [sub, del, ins]

    for i in range(N + 1):
        d[i][0] = i  # all deletions
    for j in range(M + 1):
        d[0][j] = j  # all insertions

    for i in range(1, N + 1):
        for j in range(1, M + 1):
            if ref_words[i - 1] == hyp_words[j - 1]:
                sub_cost = d[i - 1][j - 1]  # match, no edit
            else:
                sub_cost = d[i - 1][j - 1] + 1  # substitution
            del_cost = d[i - 1][j] + 1      # deletion
            ins_cost = d[i][j - 1] + 1      # insertion

            d[i][j] = min(sub_cost, del_cost, ins_cost)

    # Backtrack to count operations
    i, j = N, M
    S, D, I = 0, 0, 0
    while i > 0 or j > 0:
        if i > 0 and j > 0 and d[i][j] == d[i-1][j-1] and ref_words[i-1] == hyp_words[j-1]:
            i -= 1; j -= 1  # correct
        elif i > 0 and j > 0 and d[i][j] == d[i-1][j-1] + 1:
            S += 1; i -= 1; j -= 1  # substitution
        elif i > 0 and d[i][j] == d[i-1][j] + 1:
            D += 1; i -= 1  # deletion
        elif j > 0 and d[i][j] == d[i][j-1] + 1:
            I += 1; j -= 1  # insertion
        else:
            break

    wer = (S + D + I) / N if N > 0 else 0.0
    return wer, S, D, I, d

# --- Test cases ---
reference = "the cat sat on the mat"
hypotheses = [

]
```

```python
def spec_augment(spec, key, n_freq_masks=2, freq_mask_width=15,

    """Apply SpecAugment: frequency and time masking."""
    augmented = spec.copy()
```
