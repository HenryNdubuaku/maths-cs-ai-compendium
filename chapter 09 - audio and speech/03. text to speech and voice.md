# Синтез речи и голос

*Синтез речи (Text-to-speech, TTS) — это процесс, обратный пайплайну автоматического распознавания речи (ASR), при котором из письменного текста генерируется аудио, звучащее естественно. В этом файле рассматриваются пайплайн TTS (нормализация текста, G2P, акустические модели, вокодеры), Tacotron, WaveNet, HiFi-GAN, клонирование голоса, преобразование голоса и детектирование активности голоса (VAD).*

- В файле 01 мы создали инструментарий для обработки сигналов: формы волны (waveforms), спектрограммы, мел-фильтры и MFCC. В файле 02 мы превратили речь в текст. Теперь мы меняем направление: по заданному тексту синтезируем естественно звучащую речь. Это и есть **синтез речи (TTS)** — задача, которая также открывает путь к преобразованию голоса, клонированию голоса и детектированию активности голоса.

- Представьте TTS как театральную постановку. Сценарий — это входной текст. Режиссер (акустическая модель) решает, как должна звучать каждая реплика: её высоту, темп, акценты. Оркестр (вокодер) затем исполняет партитуру, создавая реальные звуковые волны, которые слышит аудитория. Современный нейросетевой TTS заменяет жесткую, роботизированную подачу систем на основе правил выступлениями, которые соперничают с живыми дикторами.

- **Пайплайн синтеза речи** стандартный TTS-пайплайн состоит из четырех этапов: (1) нормализация текста, (2) преобразование в фонемы, (3) акустическая модель и (4) вокодер. Некоторые современные системы объединяют этапы 3 и 4 в единую сквозную (end-to-end) модель, но концептуальное разделение остается полезным.

- **Нормализация текста** преобразует «сырой» текст в форму, пригодную для произношения. Сокращения расшифровываются («д-р» в «доктор»), числа превращаются в слова («1984» в «тысяча девятьсот восемьдесят четвертый»), символы валют вербализуются («$5» в «пять долларов»), а также обрабатываются URL-адреса и специальные символы. Этот этап часто основан на правилах с использованием грамматик для конкретных языков, хотя существуют и нейросетевые модели нормализации. Ошибки на этом этапе распространяются на все последующие: если «ул.» будет прочитано как «улица» вместо «улей» (в зависимости от контекста), вся фраза будет неверной.

- **Графемно-фонемное (G2P) преобразование** отображает нормализованный текст в последовательность фонем. Английский язык известен своей нерегулярностью («though», «through», «tough» используют «ough» по-разному), поэтому поиск по словарю (CMU Pronouncing Dictionary) обрабатывает распространенные слова, в то время как нейросетевая модель «последовательность-в-последовательность» (энкодер-декодер из главы 06 или трансформер из главы 07) обрабатывает слова, отсутствующие в словаре. Языки с простой орфографией (испанский, финский) требуют более простого G2P. На выходе обычно получается последовательность IPA (Международного фонетического алфавита) или эквивалентный внутренний набор фонем.

- **Акустические модели** принимают на вход последовательность фонем и создают промежуточное акустическое представление, почти всегда это **мел-спектрограмма** (файл 01). Мел-спектрограмма фиксирует спектральную огибающую для каждого временного кадра, которая кодирует перцептивно значимую информацию, необходимую вокодеру для восстановления формы волны. Акустическая модель должна определить тайминг (длительность каждой фонемы), высоту тона (основная частота $F_0$) и энергию (громкость).

- **Вокодеры** принимают мел-спектрограмму и создают «сырую» аудиоформу волны. Это некорректно поставленная обратная задача: многие формы волны могут давать одну и ту же спектрограмму, поскольку фазовая информация была отброшена. Классические вокодеры (Griffin-Lim, WORLD) используют итеративные подходы или модели сигналов, но сейчас по качеству доминируют нейросетевые вокодеры.

- **Вокодеры: WaveNet** (van den Oord et al., 2016) стал первым нейросетевым вокодером, который генерировал речь, почти неотличимую от человеческих записей. Он моделирует форму волны авторегрессионно, предсказывая каждый отсчет $x_t$ в зависимости от всех предыдущих отсчетов:

$$P(x) = \prod_{t=1}^{T} P(x_t \mid x_1, \ldots, x_{t-1}, c)$$

- где $c$ — обусловливающий сигнал (мел-спектрограмма). Каждый отсчет является 16-битным, поэтому наивный softmax по 65536 значениям непрактичен. WaveNet использует **mu-law компандирование** для сокращения до 256 уровней квантования, либо более поздние варианты используют смесь логистических распределений.

- Основным строительным блоком WaveNet является **дилатационная причинная свёртка** (dilated causal convolution). «Причинная» означает, что веса фильтра смотрят только на прошлые отсчеты (никакой утечки из будущего). «Дилатационная» означает, что фильтр пропускает отсчеты с экспоненциально увеличивающимися промежутками: коэффициенты расширения $1, 2, 4, 8, \ldots, 512$. Это дает экспоненциально большое поле зрения при сохранении линейного количества параметров.

- Гейтовая функция активации для каждого слоя имеет вид:

$$z = \tanh(W_{f} \ast x) \odot \sigma(W_{g} \ast x)$$

- где $W_f$ и $W_g$ — веса свёртки фильтра и гейта, $\ast$ обозначает дилатационную причинную свёртку, а $\odot$ — поэлементное умножение. Этот механизм гейтирования (из LSTM в главе 06) позволяет сети контролировать поток информации.

- WaveNet обеспечивает исключительное качество, но работает мучительно медленно при инференсе: генерация одной секунды аудио с частотой 24 кГц требует 24000 последовательных проходов. Это послужило стимулом для всех последующих исследований вокодеров.

- **WaveRNN** (Kalchbrenner et al., 2018) заменяет глубокий свёрточный стек WaveNet на однослойную рекуррентную сеть. Она разбивает каждый 16-битный отсчет на грубые (старшие 8 бит) и точные (младшие 8 бит) компоненты, предсказывая каждый из них с помощью GRU (глава 06). Этот подход с двойным softmax значительно снижает вычислительные затраты при сохранении высокого качества. WaveRNN достаточно быстр для работы в реальном времени на мобильных CPU при тщательной оптимизации ядер.

- **WaveGlow** (Prenger et al., 2019) — это **потоковый (flow-based)** вокодер, который полностью избегает авторегрессионной генерации. Он использует последовательность обратимых преобразований (аффинные связующие слои, нормализующие потоки из главы 06) для отображения простого гауссовского распределения в распределение формы волны. Обучение максимизирует точное логарифмическое правдоподобие с использованием формулы замены переменных:

$$\log P(x) = \log P(z) + \sum_{i} \log \left| \det \frac{\partial f_i}{\partial f_{i-1}} \right|$$

- где $z = f(x)$ — латентная переменная, полученная путем пропускания $x$ через поток. При инференсе извлекается выборка $z \sim \mathcal{N}(0, I)$, которая пропускается через обратный поток за один параллельный проход. WaveGlow жертвует размером модели (большие сети для связующих слоев) ради скорости генерации.

- **HiFi-GAN** (Kong et al., 2020) использует **генеративно-состязательную сеть** для синтеза сигналов из мел-спектрограмм. Генератор выполняет апсемплинг мел-спектрограммы с помощью серии транспонированных свёрток, за каждой из которых следует модуль **слияния с несколькими рецептивными полями (MRF)**. Модуль MRF параллельно применяет несколько остаточных блоков с разными размерами ядер и коэффициентами расширения (dilation rates), а затем суммирует их выходы. Это позволяет генератору одновременно улавливать паттерны на различных временных масштабах.

![Архитектура генератора HiFi-GAN: входная мел-спектрограмма проходит через слои апсемплинга на основе транспонированных свёрток, за каждым из которых следуют блоки слияния с несколькими рецептивными полями, объединяющие параллельные остаточные стеки с различными паттернами расширения](../images/hifi_gan_generator.svg)

- HiFi-GAN использует два типа дискриминаторов. **Мультипериодный дискриминатор (MPD)** преобразует одномерный сигнал в двумерный, «складывая» его с разными периодами (2, 3, 5, 7, 11), а затем применяет 2D-свёртки. Это позволяет улавливать периодические структуры на разных фундаментальных частотах. **Мультимасштабный дискриминатор (MSD)** работает с исходным сигналом, а также с его версиями, уменьшенными в 2 и 4 раза, что позволяет фиксировать паттерны с разным временным разрешением.

- Целевая функция обучения объединяет состязательную функцию потерь, **функцию потерь реконструкции мел-спектрограммы** (L1-расстояние между мел-спектрограммой синтезированного и эталонного аудио) и **функцию потерь на основе сопоставления признаков** (L1-расстояние между промежуточными признаками дискриминатора):

$$\mathcal{L}_G = \mathcal{L}_{\text{adv}}(G) + \lambda_{\text{mel}} \mathcal{L}_{\text{mel}}(G) + \lambda_{\text{fm}} \mathcal{L}_{\text{fm}}(G)$$

- HiFi-GAN обеспечивает качество синтеза, сопоставимое с WaveNet, при этом работая более чем в 1000 раз быстрее, что позволяет осуществлять генерацию в реальном времени на одном GPU.

- **Нейронные модели «источник-фильтр» (NSF)** объединяют традиционную обработку сигналов с нейронными сетями. В классической модели «источник-фильтр» озвученная речь создается источником возбуждения (периодическая последовательность импульсов на фундаментальной частоте $F_0$), который проходит через фильтр голосового тракта (спектральная огибающая). Модели NSF заменяют спроектированный вручную фильтр нейронной сетью, сохраняя при этом явный сигнал источника. Входной контур $F_0$ обеспечивает точное управление высотой тона, с чем иногда с трудом справляются чисто дата-ориентированные вокодеры.

- **Акустические модели: Tacotron** (Wang et al., 2017) стала первой сквозной (end-to-end) нейронной системой TTS, которая напрямую преобразовывала последовательности символов в мел-спектрограммы. Она использует архитектуру энкодер-декодер с механизмом внимания (глава 07). Энкодер обрабатывает последовательность символов/фонем с помощью банка свёрток, полносвязной сети (highway network) и двунаправленного GRU. Декодер представляет собой авторегрессионный GRU, который предсказывает кадры мел-спектрограммы по одному, используя в качестве входа предыдущий кадр и контекст внимания.

- **Tacotron 2** (Shen et al., 2018) значительно дорабатывает эту архитектуру. Энкодер представляет собой стек из 3 слоев 1D-свёрток, за которыми следует двунаправленная LSTM (глава 06). Декодер состоит из 2-слойной LSTM с **вниманием, чувствительным к местоположению (location-sensitive attention)**, которое обуславливает механизм внимания не только выходами энкодера и состоянием декодера, но и накопленными весами внимания с предыдущих шагов. Это предотвращает типичную ошибку, при которой механизм внимания пропускает или повторяет слова.

![Архитектура Tacotron 2: энкодер символов/фонем со свёрточными слоями и BiLSTM, внимание, чувствительное к местоположению, для выравнивания с кадрами мел-спектрограммы, авторегрессионный декодер с предсказанием стоп-токена](../images/tacotron2_architecture.svg)

- Энергия внимания, чувствительного к местоположению, для позиции энкодера $j$ на шаге декодера $i$ вычисляется как:

$$e_{i,j} = w^T \tanh(W_s s_{i-1} + W_h h_j + W_f f_{i,j} + b)$$

- где $s_{i-1}$ — предыдущее состояние декодера, $h_j$ — выход энкодера в позиции $j$, а $f_{i,j}$ — признак местоположения, полученный путем свёртки накопленных весов внимания $\sum_{k<i} \alpha_{k,j}$ с 1D-свёрточным фильтром. Веса внимания равны $\alpha_{i,j} = \text{softmax}(e_{i,j})$.

- Декодер Tacotron 2 также предсказывает вероятность **стоп-токена** на каждом шаге, указывая, когда мел-спектрограмма завершена. Полученная мел-спектрограмма затем передается в вокодер (изначально WaveNet, позже замененный на HiFi-GAN или аналоги).

- Авторегрессионная природа Tacotron 2 означает, что скорость синтеза ограничена количеством кадров мел-спектрограммы. Для типичной мел-спектрограммы с частотой 80 кадров в секунду 5-секундное высказывание требует 400 последовательных шагов декодера.

- **FastSpeech** (Ren et al., 2019) решает проблему скорости с помощью **неавторегрессионной** акустической модели. Вместо последовательной генерации кадров мел-спектрограммы FastSpeech генерирует все кадры параллельно. Ключевая задача заключается в определении того, сколько кадров мел-спектрограммы должна порождать каждая фонема, с чем FastSpeech справляется с помощью **предиктора длительности**.

- Предиктор длительности — это небольшая свёрточная сеть, которая предсказывает целочисленную длительность (количество кадров мел-спектрограммы) для каждой фонемы. Во время обучения эталонные длительности извлекаются из предварительно обученной авторегрессионной модели-учителя (Tacotron 2) с использованием её выравниваний внимания. Во время инференса предсказанные длительности используются для расширения скрытой последовательности на уровне фонем до уровня кадров с помощью **регулятора длины**, который просто повторяет скрытое представление каждой фонемы заданное количество раз.

- **FastSpeech 2** (Ren et al., 2021) улучшает FastSpeech, отказываясь от дистилляции «учитель-ученик». Модель извлекает эталонные длительности напрямую с помощью принудительного выравнивания (forced alignment, из акустических моделей в файле 02) и добавляет явные **адаптеры вариативности** для высоты тона ($F_0$) и энергии в дополнение к длительности. Каждый адаптер представляет собой небольшой свёрточный предиктор, выход которого обуславливает декодер:

```math
\begin{aligned}
\hat{d}_i &= \text{DurationPredictor}(h_i) \\
\hat{p}_i &= \text{PitchPredictor}(h_i) \\
\hat{e}_i &= \text{EnergyPredictor}(h_i)
\end{aligned}
```

- где $h_i$ — скрытое состояние энкодера для фонемы $i$. Во время обучения используются эталонные значения; при инференсе предсказанные значения обеспечивают явное управление просодией. Эта управляемость является главным преимуществом FastSpeech 2: настройка высоты тона, скорости или энергии осуществляется так же просто, как масштабирование выходов предикторов.

- FastSpeech 2 обычно работает в 10–20 раз быстрее, чем Tacotron 2 при инференсе, и позволяет избежать типичных проблем авторегрессионных моделей, таких как пропуск слов, повторы и коллапс внимания (attention collapse).

- **VITS** (Kim et al., 2021) — это **end-to-end** модель синтеза речи (TTS), которая генерирует форму волны непосредственно из текста, исключая отдельный этап вокодера. VITS объединяет условный вариационный автокодировщик (глава 06) с нормализующими потоками и состязательным обучением. Апостериорный энкодер отображает эталонные мел-спектрограммы в латентное пространство, априорный энкодер отображает фонемы (через текстовый энкодер на базе трансформера и предиктора длительности) в то же латентное пространство, а декодер (на базе HiFi-GAN) генерирует форму волны из латентных выборок.

- Целевая функция обучения VITS объединяет:
    - **Функцию потерь реконструкции**: VAE заставляет латентное распределение кодировать акустическую информацию.
    - **KL-дивергенцию**: выравнивает априорное распределение, обусловленное текстом, с апостериорным распределением, обусловленным аудио.
    - **Состязательную функцию потерь**: дискриминаторы обеспечивают качество формы волны.
    - **Функцию потерь длительности**: обучает стохастический предиктор длительности.

- VITS обеспечивает более высокое качество, чем двухэтапные системы (FastSpeech 2 + HiFi-GAN), поскольку акустическая модель и вокодер оптимизируются совместно, что позволяет избежать рассогласования между предсказанными и эталонными мел-спектрограммами, которое снижает качество двухэтапных систем.

- **VALL-E** (Wang et al., 2023) радикально переосмысливает TTS как **задачу языкового моделирования** на дискретных аудиотокенах. Она использует нейронный аудиокодек (EnCodec) для представления речи в виде последовательности дискретных кодов из нескольких уровней кодовой книги. Получая на вход текстовый промпт и 3-секундный образец голоса (также закодированный как дискретные токены), VALL-E использует языковую модель на базе трансформера для авторегрессионного предсказания аудиотокенов.

- VALL-E использует две модели: **авторегрессионную (AR) модель**, которая генерирует первый уровень кодовой книги токен за токеном, и **неавторегрессионную (NAR) модель**, которая предсказывает остальные уровни кодовой книги параллельно, обуславливаясь первым уровнем и друг другом. Такой подход с использованием языковой модели кодека обеспечивает впечатляющее zero-shot клонирование голоса: 3-секундного образца достаточно, чтобы воспроизвести голос, тембр и даже эмоциональную окраску говорящего.

- **StyleTTS** (Li et al., 2022) и **StyleTTS 2** разделяют речь на компоненты содержания и стиля. Энкодер стиля извлекает вектор стиля из эталонного аудио, улавливая личность говорящего, просодию и условия записи. Во время инференса стиль можно выбирать из обученного априорного распределения или переносить из эталонного высказывания. StyleTTS 2 использует диффузионные модели (глава 08) для априорного распределения стиля, генерируя разнообразную и естественную просодию.

- **Kokoro** (2024) — это легковесная высококачественная TTS-модель с открытым исходным кодом, примечательная своим малым размером (~82 млн параметров) и впечатляющей естественностью. Она использует архитектуру, вдохновленную StyleTTS 2, с диффузионным априорным распределением стиля и дообученным вокодером ISTFTNet, который предсказывает коэффициенты STFT (из файла 01) напрямую, а не сырые отсчеты формы волны. Несмотря на то, что Kokoro во много раз меньше таких моделей, как VALL-E, она достигает почти человеческой естественности для английского, японского, французского, корейского и китайского языков, доказывая, что тщательно отобранные обучающие данные и эффективный дизайн архитектуры могут конкурировать с «грубой силой» масштабирования. Малый размер Kokoro делает её практичной для локального развертывания и развертывания на периферийных устройствах (edge deployment).

- **Orpheus** (Canopy Labs, 2025) — это семейство TTS-моделей с открытым исходным кодом (1 млрд и 3 млрд параметров), построенных на парадигме **языковой модели кодека**, впервые предложенной VALL-E. Orpheus развивает эту идею дальше с помощью LLM-основы (дообученная Llama 3), которая генерирует токены аудиокодека SNAC напрямую. Её отличительная черта — человекоподобная эмоциональная выразительность: она обрабатывает смех, вздохи, колебания и аффективную просодию с поразительной естественностью. Orpheus можно задавать теги, такие как `[laugh]` или `[sigh]`, во входном тексте, что дает возможность тонкой настройки паралингвистической экспрессии.

- **Dia** (Nari Labs, 2025) — это TTS-модель для диалогов с открытым исходным кодом, которая генерирует реалистичные многоголосые беседы на основе одной текстовой расшифровки. Построенная на энкодер-декодер трансформере с 1,6 млрд параметров, Dia управляет очередностью реплик, специфичными для говорящих голосами и невербальными сигналами (смех, паузы) внутри разговора. Она также поддерживает клонирование голоса по короткому аудиопромпту, обеспечивая zero-shot генерацию говорящего в контексте диалога.

- **Sesame CSM** (Conversational Speech Model, 2025) фокусируется на естественной многоходовой разговорной речи. Вместо оптимизации под TTS в стиле чтения, Sesame моделирует динамику реального разговора: поддакивания ("угу"), перебивания, изменения ритма между говорящими и эмоциональную отзывчивость. Модель использует трансформерную основу, обусловленную контекстом разговора (как текстом, так и историей аудио), создавая речь, которая адаптирует свой стиль к ходу диалога.

- **Fish Speech** (Fish Audio, 2024) — это TTS-система с открытым исходным кодом, использующая двойную авторегрессионную архитектуру: большая языковая модель генерирует семантические токены из текста, а меньшая модель преобразует их в акустические токены VQGAN, которые декодируются в форму волны вокодером. Fish Speech поддерживает zero-shot клонирование голоса по 10–15-секундному образцу и обеспечивает низкую задержку, подходящую для приложений реального времени. Модульный дизайн позволяет независимо заменять компоненты (например, разные вокодеры).

- **ChatTTS** (2024) — это разговорная TTS-модель с открытым исходным кодом, предназначенная для диалоговых приложений, таких как чат-боты и виртуальные ассистенты. Она генерирует естественную, разговорную речь с тонкой настройкой просодических признаков (смех, паузы, слова-паразиты) с помощью специальных токенов, встроенных во входной текст. ChatTTS поддерживает смешанный китайско-английский синтез и многоголосую генерацию.

- **Bark** (Suno, 2023) — это трансформерная модель с открытым исходным кодом, которая генерирует речь, музыку и звуковые эффекты из текстовых промптов. Она использует трехэтапный пайплайн трансформерных моделей (текст → семантические токены → грубые акустические токены → точные акустические токены) и поддерживает клонирование голоса, мультиязычный синтез и неречевые аудио, такие как музыка и фоновые звуки. Универсальность Bark достигается ценой управляемости — она менее точна, чем специализированные TTS-системы, но более гибкая.

- **Parler-TTS** (Hugging Face, 2024) использует подход **описания на естественном языке** для управления голосом: вместо того чтобы требовать эталонный аудиоклип для задания стиля, пользователь предоставляет текстовое описание, например: «женский голос, теплый и выразительный, в тихой комнате». Parler-TTS обучена на аннотированных речевых данных, где каждое высказывание сопоставлено с описанием стиля речи на естественном языке, что обеспечивает интуитивное управление без использования эталонного аудио.

- **Neuphonic** — это API-платформа для синтеза речи (TTS), оптимизированная для сверхнизкой задержки, ориентированная на голосовых агентов в реальном времени и приложения разговорного ИИ. Она достигает времени до начала воспроизведения аудио (time-to-first-audio) менее 100 мс благодаря потоковой архитектуре, которая начинает генерацию аудио до того, как будет доступен весь входной текст. Neuphonic фокусируется на уровне развёртывания и оптимизации задержки, а не на новой архитектуре моделей, предоставляя инфраструктуру промышленного уровня для современных нейронных TTS.

- **KittenTTS** — это компактная и быстрая TTS-модель, разработанная для эффективности и развёртывания в условиях ограниченных ресурсов. Она отдает приоритет минимальной задержке и небольшому размеру модели для граничных (edge) и встраиваемых приложений, жертвуя частью естественности ради производительности в реальном времени на CPU и мобильных устройствах.

- Современный ландшафт TTS разделяется на две парадигмы: (1) **кодековые языковые модели** (VALL-E, Orpheus, Fish Speech), которые рассматривают генерацию речи как предсказание следующего токена на основе дискретных аудиокодов, используя законы масштабирования LLM; и (2) **поточные/диффузионные модели** (VITS, StyleTTS 2, Kokoro), которые генерируют непрерывные мел-спектрограммы или формы волны посредством итеративного уточнения. Кодековые языковые модели превосходны в клонировании с нулевым обучением (zero-shot) и выразительности; поточные/диффузионные модели, как правило, меньше и быстрее. Обе парадигмы стремительно приближаются к естественности человеческого уровня.

- **Моделирование просодии** управляет «музыкой» речи: высотой тона, длительностью, энергией, ритмом и интонацией. Без качественной просодии синтезированная речь звучит монотонно и роботизированно, даже если отдельные фонемы произносятся четко. Представьте просодию как разницу между монотонным голосом GPS-навигатора и выразительным чтецом аудиокниг.

- **Высота тона** (основная частота $F_0$) — это воспринимаемая высота звука речи. Она повышается в конце вопросов, понижается в конце утверждений и непрерывно варьируется во время эмоциональной речи. $F_0$ извлекается из аудио с помощью таких алгоритмов, как CREPE (нейронный трекер высоты тона) или YIN (на основе автокорреляции, из файла 01). В TTS высота тона либо предсказывается акустической моделью (предиктор высоты тона в FastSpeech 2), либо изучается неявно (Tacotron 2).

- **Длительность** определяет темп и ритм речи. Ударные слоги длиннее, служебные слова сокращаются, а паузы обозначают границы фраз. Моделирование длительности является явным в неавторегрессионных моделях (FastSpeech) и неявным в авторегрессионных моделях (выравнивание внимания в Tacotron определяет длительность).

- **Энергия** (громкость) передает акценты. Фразы «Я не говорил, что ОН это украл» и «Я не говорил, что он УКРАЛ это» имеют разные значения, передаваемые исключительно через паттерны энергии.

- **Эмбеддинги стиля** улавливают просодические паттерны более высокого уровня. Фреймворк **Global Style Token (GST)** (Wang et al., 2018) обучается на банке токенов стиля (мягкое внимание над набором обученных эмбеддингов), которые фиксируют стили речи, такие как «взволнованный», «грустный» или «шепот». Эмбеддинг стиля извлекается из эталонного высказывания и добавляется к выходу энкодера, что позволяет выполнять перенос стиля при инференсе.

- **Преобразование голоса (VC)** изменяет личность говорящего в высказывании, сохраняя при этом лингвистическое содержание. Представьте, что вы записываете себя, а на выходе голос звучит как у конкретного целевого диктора. VC требует отделения личности говорящего от содержания.

- **Эмбеддинги говорящего** (подробно описаны в файле 04) кодируют личность говорящего в виде вектора фиксированной размерности. Они могут быть получены из предобученной модели верификации диктора (x-векторы, ECAPA-TDNN). В VC исходная речь кодируется в представление содержания, которое не зависит от говорящего, а затем декодируется с использованием эмбеддинга целевого говорящего.

- **Раздельные представления** (disentangled representations) разделяют речь на независимые факторы: содержание (фонемы), личность говорящего, высоту тона и ритм. Подходы включают:
    - **Информационное узкое горлышко (information bottleneck)**: сжатие представления содержания настолько сильно, что информация о говорящем теряется (AutoVC)
    - **Состязательное обучение**: обучение классификатора говорящих на представлении содержания и использование градиентной инверсии для удаления информации о говорящем
    - **Векторное квантование**: VQ-VAE пропускает содержание через дискретное узкое горлышко, что естественным образом отсекает личность говорящего (поскольку элементы кодовой книги представляют фонетические категории, а не черты диктора)

- **Клонирование голоса** синтезирует речь голосом целевого диктора. **Многодикторный TTS** обучается на данных многих дикторов, обусловливая модель на эмбеддинге говорящего. При инференсе эмбеддинг нового диктора извлекается из аудиозаписи (enrollment audio) и используется для управления генерацией.

- **Клонирование голоса с малым количеством данных (few-shot voice cloning)** адаптируется к новому диктору с использованием небольшого объема данных (несколько минут). Энкодер диктора извлекает эмбеддинг из аудиозаписи, а TTS-модель генерирует речь, обусловленную этим эмбеддингом. Это подход, используемый в SV2TTS (Jia et al., 2018): отдельно обученный энкодер диктора, синтезатор Tacotron 2, обусловленный эмбеддингом диктора, и вокодер WaveRNN.

- **Клонирование голоса с нулевым обучением (zero-shot voice cloning)** не требует никакой адаптации: достаточно одного короткого высказывания (3–30 секунд). VALL-E достигает этого, рассматривая аудиозапись как промпт для языковой модели. Модель учится продолжать генерацию тем же голосом, поскольку она была обучена на крупномасштабных многодикторных данных, где согласованность голоса внутри высказывания является статистической нормой.

- **Детекция речевой активности (VAD)** отвечает на простой бинарный вопрос для каждого временного кадра: говорит кто-то или нет? Несмотря на свою простоту, VAD является критически важным этапом предварительной обработки для ASR (файл 02), диаризации спикеров (файл 04) и шумоподавления (файл 05). Хороший VAD снижает вычислительные затраты за счет пропуска тишины и повышает точность, предотвращая обработку шума как речи.

- Классический VAD использует пороговую обработку энергии (речь громче тишины), частоту пересечения нуля (речь имеет характерные паттерны пересечений) и спектральные признаки. Эти методы неэффективны в шумной среде, где отношение сигнал/шум низкое.

- **Нейросетевые VAD**-модели рассматривают эту задачу как бинарную классификацию на уровне кадров. Небольшая RNN или CNN принимает акустические признаки (логарифмические мел-энергии из файла 01) и предсказывает вероятности речи/не-речи.

- **WebRTC VAD** (Google) — это классический легковесный VAD, использующий классификатор на основе GMM для простых спектральных признаков. Он работает на четырех уровнях агрессивности (0-3) и работает крайне быстро, но плохо справляется с музыкой, неречевыми вокализациями и условиями с низким отношением сигнал/шум. Он остается широко используемым базовым решением (baseline) благодаря своей простоте и отсутствию зависимостей.

- **Silero VAD** (команда Silero, 2021) — это де-факто стандартный нейросетевой VAD для использования в продакшене. Его архитектура представляет собой небольшой стек глубинно-разделимых 1D-свёрток (идея MobileNet из главы 08, примененная к аудио), за которыми следует один слой LSTM для учета временного контекста, а завершает архитектуру линейная «голова», выдающая вероятность речи для каждого кадра. Вся модель весит менее 2 МБ (~1 млн параметров) и обрабатывает аудио чанками по 30-100 мс.
    - **Вход**: сырое аудио с частотой дискретизации 16 кГц (ручное извлечение признаков не требуется — свёрточный фронтенд обучается извлекать собственные признаки непосредственно из формы волны).
    - **Оконный инференс с сохранением состояния**: скрытое состояние LSTM переносится между чанками, поэтому модель обрабатывает потоковое аудио без повторной обработки всей истории. Каждый вызов обрабатывает чанк длительностью 30, 60 или 100 мс и возвращает вероятность речи в диапазоне $[0, 1]$.
    - **Адаптивная пороговая обработка**: вместо одного фиксированного порога Silero VAD использует отдельные пороги для начала и конца речи с учетом минимальной длительности речи/тишины, что предотвращает быстрое переключение на шумных границах. Речевой сегмент должен превышать порог начала в течение минимального времени, прежде чем будет подтвержден, а тишина должна сохраняться ниже порога конца, прежде чем сегмент будет закрыт.
    - **Производительность**: Silero VAD работает с коэффициентом реального времени 1-2% на CPU (обработка 1 секунды аудио занимает ~10-20 мс), что делает его подходящим для граничных устройств (edge devices), мобильных телефонов и пайплайнов реального времени. Он значительно превосходит WebRTC VAD на шумном и насыщенном музыкой аудио, оставаясь при этом достаточно компактным для развёртывания на устройствах.
    - Silero VAD обычно используется в качестве фронтенда для Whisper (файл 02) для сегментации длинного аудио на чанки уровня высказываний перед транскрибацией, а также для пайплайнов диаризации спикеров (файл 04) для идентификации речевых областей перед извлечением эмбеддингов спикеров.

- **Детекция акустической активности (AAD)** обобщает VAD для обнаружения любой акустической активности, а не только речи. Это полезно в устройствах «умного дома», системах безопасности и мониторинге дикой природы. Модели AAD обнаруживают такие события, как разбитие стекла, лай собак или сигналы тревоги, часто используя фреймворки классификации аудио, описанные в файле 04.

- **Метрики оценки для TTS** измеряют как объективное качество, так и субъективную естественность:
    - **Mean Opinion Score (MOS)**: слушатели-люди оценивают естественность по шкале от 1 до 5. Золотой стандарт, но дорогой и медленный метод.
    - **Mel cepstral distortion (MCD)**: измеряет расстояние между синтезированными и эталонными мел-кепстрами. Чем ниже значение, тем лучше, но оно не всегда коррелирует с восприятием.
    - **PESQ / POLQA**: стандартизированные метрики перцептивной оценки, изначально разработанные для телефонии.
    - **Сходство спикеров**: косинусное сходство между эмбеддингами спикеров синтезированного и эталонного аудио (актуально для клонирования голоса).
    - **Разборчивость**: измеряется путем подачи синтезированного аудио в систему ASR (файл 02) и вычисления WER.

- **Задача 1: Вокодер Griffin-Lim из мел-спектрограммы.** Реализуйте итеративный алгоритм реконструкции фазы Griffin-Lim для преобразования мел-спектрограммы обратно в форму волны. Это демонстрирует проблему вокодера и необходимость нейронных вокодеров.

```python
import jax
import jax.numpy as jnp
import matplotlib.pyplot as plt

# Generate a synthetic waveform (sum of harmonics simulating a vowel)
sr = 16000
duration = 1.0
t = jnp.linspace(0, duration, int(sr * duration))
f0 = 220.0  # fundamental frequency
waveform = (
    0.6 * jnp.sin(2 * jnp.pi * f0 * t) +
    0.3 * jnp.sin(2 * jnp.pi * 2 * f0 * t) +
    0.1 * jnp.sin(2 * jnp.pi * 3 * f0 * t)
)

# Compute STFT
n_fft = 1024
hop_length = 256
window = jnp.hanning(n_fft)

def stft(signal, n_fft, hop_length, window):
    """Compute Short-Time Fourier Transform."""
    n_frames = 1 + (len(signal) - n_fft) // hop_length
    frames = jnp.stack([
        signal[i * hop_length : i * hop_length + n_fft] * window
        for i in range(n_frames)
    ])
    return jnp.fft.rfft(frames, n=n_fft)

def istft(stft_matrix, hop_length, window, length):
    """Compute inverse STFT with overlap-add."""
    n_fft = (stft_matrix.shape[1] - 1) * 2
    n_frames = stft_matrix.shape[0]
    frames = jnp.fft.irfft(stft_matrix, n=n_fft)
    frames = frames * window[None, :]
    output = jnp.zeros(length)
    for i in range(n_frames):
        start = i * hop_length
        end = start + n_fft
        if end <= length:
            output = output.at[start:end].add(frames[i])
    return output

# Forward STFT
S = stft(waveform, n_fft, hop_length, window)
magnitude = jnp.abs(S)

# Mel filterbank
n_mels = 80
mel_low = 0.0
mel_high = 2595 * jnp.log10(1 + (sr / 2) / 700)
mel_points = jnp.linspace(mel_low, mel_high, n_mels + 2)
hz_points = 700 * (10 ** (mel_points / 2595) - 1)
freq_bins = jnp.floor((n_fft + 1) * hz_points / sr).astype(int)

mel_filterbank = jnp.zeros((n_mels, n_fft // 2 + 1))
for m in range(n_mels):
    f_left = freq_bins[m]
    f_center = freq_bins[m + 1]
    f_right = freq_bins[m + 2]
    for k in range(f_left, f_center):
        mel_filterbank = mel_filterbank.at[m, k].set(
            (k - f_left) / max(f_center - f_left, 1)
        )
    for k in range(f_center, f_right):
        mel_filterbank = mel_filterbank.at[m, k].set(
            (f_right - k) / max(f_right - f_center, 1)
        )

# To mel and back (pseudo-inverse)
mel_spec = magnitude @ mel_filterbank.T
magnitude_reconstructed = mel_spec @ jnp.linalg.pinv(mel_filterbank.T)
magnitude_reconstructed = jnp.maximum(magnitude_reconstructed, 1e-7)

# Griffin-Lim algorithm
def griffin_lim(magnitude, n_iter, hop_length, window, signal_length):
    """Iterative phase reconstruction."""
    n_fft = (magnitude.shape[1] - 1) * 2
    key = jax.random.PRNGKey(42)
    phase = jax.random.uniform(key, magnitude.shape, minval=-jnp.pi, maxval=jnp.pi)

    for _ in range(n_iter):
        complex_spec = magnitude * jnp.exp(1j * phase)
        signal = istft(complex_spec, hop_length, window, signal_length)
        reanalysis = stft(signal, n_fft, hop_length, window)
        phase = jnp.angle(reanalysis)

    complex_spec = magnitude * jnp.exp(1j * phase)
    return istft(complex_spec, hop_length, window, signal_length)

reconstructed = griffin_lim(magnitude_reconstructed, n_iter=60, hop_length=hop_length,
                            window=window, signal_length=len(waveform))

# Plot comparison
fig, axes = plt.subplots(3, 1, figsize=(12, 8))

axes[0].plot(t[:1000], waveform[:1000], color='#3498db', linewidth=0.8)
axes[0].set_title('Original Waveform')
axes[0].set_ylabel('Amplitude')

axes[1].imshow(jnp.log1p(mel_spec.T), aspect='auto', origin='lower', cmap='magma')
axes[1].set_title('Mel Spectrogram (intermediate representation)')
axes[1].set_ylabel('Mel bin')

axes[2].plot(t[:1000], reconstructed[:1000], color='#e74c3c', linewidth=0.8)
axes[2].set_title('Griffin-Lim Reconstructed Waveform (60 iterations)')
axes[2].set_xlabel('Time (s)')
axes[2].set_ylabel('Amplitude')

plt.tight_layout()
plt.show()

# Measure reconstruction error
mse = jnp.mean((waveform[:len(reconstructed)] - reconstructed[:len(waveform)]) ** 2)
print(f"MSE between original and reconstructed: {mse:.6f}")
print("Note: phase information loss through mel inversion causes artifacts.")
```

- **Task 2: Duration predictor (FastSpeech-style).** Train a small convolutional duration predictor that maps phoneme embeddings to durations. This is the core component enabling non-autoregressive TTS.

```python
import jax
import jax.numpy as jnp
import jax.random as jr
import matplotlib.pyplot as plt

# Simulate phoneme sequences with ground-truth durations
# In real TTS, durations come from forced alignment or teacher attention
def generate_synthetic_data(key, n_samples=200, max_phonemes=30, embed_dim=64):
    """Generate synthetic phoneme embeddings and durations."""
    keys = jr.split(key, 4)
    lengths = jr.randint(keys[0], (n_samples,), 5, max_phonemes)

    all_embeddings = []
    all_durations = []
    all_masks = []

    for i in range(n_samples):
        L = int(lengths[i])
        emb = jr.normal(keys[1], (max_phonemes, embed_dim))
        # Durations: vowels (even indices) are longer, consonants shorter
        base_dur = jnp.where(jnp.arange(max_phonemes) % 2 == 0, 8.0, 4.0)
        noise = jr.normal(jr.fold_in(keys[2], i), (max_phonemes,)) * 1.5
        dur = jnp.clip(base_dur + noise, 1.0, 20.0).astype(jnp.float32)
        mask = (jnp.arange(max_phonemes) < L).astype(jnp.float32)

        all_embeddings.append(emb)
        all_durations.append(dur * mask)
        all_masks.append(mask)

    return (jnp.stack(all_embeddings), jnp.stack(all_durations),
            jnp.stack(all_masks))

key = jr.PRNGKey(42)
embeddings, durations, masks = generate_synthetic_data(key)

# Duration predictor: 2-layer 1D convolution + linear projection
def init_duration_predictor(key, embed_dim=64, hidden_dim=128, kernel_size=3):
    """Initialise duration predictor weights."""
    keys = jr.split(key, 4)
    scale1 = jnp.sqrt(2.0 / (embed_dim * kernel_size))
    scale2 = jnp.sqrt(2.0 / (hidden_dim * kernel_size))
    params = {
        'conv1_w': jr.normal(keys[0], (kernel_size, embed_dim, hidden_dim)) * scale1,
        'conv1_b': jnp.zeros(hidden_dim),
        'conv2_w': jr.normal(keys[1], (kernel_size, hidden_dim, hidden_dim)) * scale2,
        'conv2_b': jnp.zeros(hidden_dim),
        'linear_w': jr.normal(keys[2], (hidden_dim, 1)) * jnp.sqrt(2.0 / hidden_dim),
        'linear_b': jnp.zeros(1),
    }
    return params

def duration_predictor(params, x):
    """Predict log-durations from phoneme embeddings. x: (batch, seq, embed)."""
    # Conv layer 1 with ReLU
    h = jax.lax.conv_general_dilated(
        x.transpose(0, 2, 1),  # (batch, embed, seq)
        params['conv1_w'].transpose(2, 1, 0),  # (out, in, kernel)
        window_strides=(1,), padding='SAME'
    ).transpose(0, 2, 1) + params['conv1_b']  # back to (batch, seq, hidden)
    h = jax.nn.relu(h)

    # Conv layer 2 with ReLU
    h = jax.lax.conv_general_dilated(
        h.transpose(0, 2, 1),
        params['conv2_w'].transpose(2, 1, 0),
        window_strides=(1,), padding='SAME'
    ).transpose(0, 2, 1) + params['conv2_b']
    h = jax.nn.relu(h)

    # Linear projection to scalar
    log_dur = (h @ params['linear_w'] + params['linear_b']).squeeze(-1)
    return log_dur

# Loss: MSE on log-durations (standard in FastSpeech)
def loss_fn(params, embeddings, durations, masks):
    log_dur_pred = duration_predictor(params, embeddings)
    log_dur_true = jnp.log(jnp.clip(durations, 1.0, None))
    sq_err = (log_dur_pred - log_dur_true) ** 2 * masks
    return jnp.sum(sq_err) / jnp.sum(masks)

grad_fn = jax.jit(jax.value_and_grad(loss_fn))

# Training loop
params = init_duration_predictor(jr.PRNGKey(0))
lr = 1e-3
losses = []

for epoch in range(300):
    loss_val, grads = grad_fn(params, embeddings, durations, masks)
    params = jax.tree.map(lambda p, g: p - lr * g, params, grads)
    losses.append(float(loss_val))

# Evaluate on a sample
log_dur_pred = duration_predictor(params, embeddings[:1])
dur_pred = jnp.exp(log_dur_pred[0])
dur_true = durations[0]
mask = masks[0]
valid_len = int(jnp.sum(mask))

fig, axes = plt.subplots(1, 2, figsize=(14, 5))

axes[0].plot(losses, color='#3498db', linewidth=1.5)
axes[0].set_xlabel('Epoch')
axes[0].set_ylabel('MSE Loss (log-duration)')
axes[0].set_title('Duration Predictor Training')
axes[0].set_yscale('log')

x_pos = jnp.arange(valid_len)
width = 0.35
axes[1].bar(x_pos - width/2, dur_true[:valid_len], width, color='#27ae60',
            label='Ground truth', alpha=0.8)
axes[1].bar(x_pos + width/2, dur_pred[:valid_len], width, color='#e74c3c',
            label='Predicted', alpha=0.8)
axes[1].set_xlabel('Phoneme index')
axes[1].set_ylabel('Duration (frames)')
axes[1].set_title('Duration Prediction vs Ground Truth')
axes[1].legend()

plt.tight_layout()
plt.show()
```

- **Task 3: Simple neural vocoder with upsampling convolutions.** Build a minimal HiFi-GAN-style generator that upsamples a mel spectrogram to a waveform using transposed convolutions and residual blocks.

```python
import jax
import jax.numpy as jnp
import jax.random as jr
import matplotlib.pyplot as plt

def init_residual_block(key, channels, kernel_size, dilation):
    """Initialise a dilated residual convolution block."""
    k1, k2 = jr.split(key)
    scale = jnp.sqrt(2.0 / (channels * kernel_size))
    return {
        'conv1_w': jr.normal(k1, (kernel_size, channels, channels)) * scale,
        'conv1_b': jnp.zeros(channels),
        'conv2_w': jr.normal(k2, (kernel_size, channels, channels)) * scale,
        'conv2_b': jnp.zeros(channels),
        'dilation': dilation
    }

def residual_block(params, x):
    """x: (batch, time, channels). Dilated conv residual block with LeakyReLU."""
    h = jax.nn.leaky_relu(x, negative_slope=0.1)
    # Simplified: use standard conv (dilation handled conceptually)
    h = jax.lax.conv_general_dilated(
        h.transpose(0, 2, 1),
        params['conv1_w'].transpose(2, 1, 0),
        window_strides=(1,),
        padding='SAME',
        rhs_dilation=(params['dilation'],)
    ).transpose(0, 2, 1) + params['conv1_b']
    h = jax.nn.leaky_relu(h, negative_slope=0.1)
    h = jax.lax.conv_general_dilated(
        h.transpose(0, 2, 1),
        params['conv2_w'].transpose(2, 1, 0),
        window_strides=(1,),
        padding='SAME'
    ).transpose(0, 2, 1) + params['conv2_b']
    return x + h

def init_generator(key, n_mels=80, upsample_rates=(8, 8, 4),
                   channels=128):
    """Initialise a minimal HiFi-GAN-style generator."""
    keys = jr.split(key, 10)
    params = {}

    # Input projection: mel bins -> channels
    params['input_w'] = jr.normal(keys[0], (7, n_mels, channels)) * 0.02
    params['input_b'] = jnp.zeros(channels)

    # Upsample blocks (transposed convolutions)
    in_ch = channels
    for i, rate in enumerate(upsample_rates):
        k_size = rate * 2
        scale = jnp.sqrt(2.0 / (in_ch * k_size))
        out_ch = in_ch // 2
        params[f'up{i}_w'] = jr.normal(keys[i+1], (k_size, in_ch, out_ch)) * scale
        params[f'up{i}_b'] = jnp.zeros(out_ch)
        # Residual blocks at each scale
        params[f'res{i}_0'] = init_residual_block(jr.fold_in(keys[i+4], 0),
                                                    out_ch, 3, 1)
        params[f'res{i}_1'] = init_residual_block(jr.fold_in(keys[i+4], 1),
                                                    out_ch, 3, 3)
        in_ch = out_ch

    # Output projection to mono waveform
    params['output_w'] = jr.normal(keys[8], (7, in_ch, 1)) * 0.02
    params['output_b'] = jnp.zeros(1)
    params['upsample_rates'] = upsample_rates

    return params

def generator_forward(params, mel):
    """mel: (batch, time, n_mels) -> waveform: (batch, time * prod(rates), 1)."""
    # Input projection
    h = jax.lax.conv_general_dilated(
        mel.transpose(0, 2, 1),
        params['input_w'].transpose(2, 1, 0),
        window_strides=(1,), padding='SAME'
    ).transpose(0, 2, 1) + params['input_b']

    for i, rate in enumerate(params['upsample_rates']):
        h = jax.nn.leaky_relu(h, negative_slope=0.1)
        # Upsample via transposed convolution
        k_size = rate * 2
        h = jax.lax.conv_transpose(
            h.transpose(0, 2, 1),
            params[f'up{i}_w'].transpose(2, 1, 0),
            strides=(rate,),
            padding='SAME'
        ).transpose(0, 2, 1) + params[f'up{i}_b']
        # Residual blocks
        h = residual_block(params[f'res{i}_0'], h)
        h = residual_block(params[f'res{i}_1'], h)

    h = jax.nn.leaky_relu(h, negative_slope=0.1)
    out = jax.lax.conv_general_dilated(
        h.transpose(0, 2, 1),
        params['output_w'].transpose(2, 1, 0),
        window_strides=(1,), padding='SAME'
    ).transpose(0, 2, 1) + params['output_b']

    return jnp.tanh(out)

# Create a synthetic mel spectrogram (simulating a vowel)
n_mels = 80
n_frames = 50
mel = jnp.zeros((1, n_frames, n_mels))
# Add energy in low-frequency mel bins (simulating formants)
mel = mel.at[:, :, 5:15].set(1.0)
mel = mel.at[:, :, 20:25].set(0.6)

# Initialise and run generator
key = jr.PRNGKey(42)
params = init_generator(key, n_mels=n_mels, upsample_rates=(8, 8, 4),
                         channels=128)
waveform = generator_forward(params, mel)

print(f"Input mel shape:  {mel.shape}")
print(f"Output waveform shape: {waveform.shape}")
print(f"Upsample factor: {8 * 8 * 4} = {8*8*4}x")

fig, axes = plt.subplots(2, 1, figsize=(12, 6))

axes[0].imshow(mel[0].T, aspect='auto', origin='lower', cmap='magma')
axes[0].set_title('Input Mel Spectrogram')
axes[0].set_ylabel('Mel bin')
axes[0].set_xlabel('Frame')

waveform_np = waveform[0, :, 0]
axes[1].plot(waveform_np[:2000], color='#9b59b6', linewidth=0.5)
axes[1].set_title('Generator Output Waveform (untrained - random noise)')
axes[1].set_ylabel('Amplitude')
axes[1].set_xlabel('Sample')

plt.tight_layout()
plt.show()
print("Note: The output is noise because the generator is untrained.")
print("In practice, adversarial + mel loss training shapes this into speech.")
```

- **Задача 4: Детекция голосовой активности (VAD) с помощью простой рекуррентной нейронной сети (RNN).** Обучите небольшую модель VAD на основе GRU на синтетических аудиопризнаках для классификации кадров как «речь» или «тишина».

```python
import jax
import jax.numpy as jnp
import jax.random as jr
import matplotlib.pyplot as plt

# Generate synthetic log-mel energy features with speech/silence labels
def generate_vad_data(key, n_sequences=100, n_frames=200, n_features=40):
    """Simulate log-mel features: speech regions are higher energy with structure."""
    keys = jr.split(key, 5)
    all_features = []
    all_labels = []

    for i in range(n_sequences):
        k = jr.fold_in(keys[0], i)
        k1, k2, k3 = jr.split(k, 3)

        # Random speech/silence pattern
        label = jnp.zeros(n_frames)
        n_segments = jr.randint(k1, (), 2, 6)
        for seg in range(int(n_segments)):
            start = jr.randint(jr.fold_in(k2, seg), (), 0, n_frames - 20)
            length = jr.randint(jr.fold_in(k3, seg), (), 10, 50)
            end = jnp.minimum(start + length, n_frames)
            label = label.at[int(start):int(end)].set(1.0)

        # Features: speech frames have higher energy + spectral structure
        noise = jr.normal(jr.fold_in(keys[1], i), (n_frames, n_features)) * 0.3
        speech_pattern = jnp.outer(label, jnp.exp(-jnp.arange(n_features) / 15.0))
        features = speech_pattern * 2.0 + noise + 0.1

        all_features.append(features)
        all_labels.append(label)

    return jnp.stack(all_features), jnp.stack(all_labels)

key = jr.PRNGKey(123)
features, labels = generate_vad_data(key)
train_features, train_labels = features[:80], labels[:80]
test_features, test_labels = features[80:], labels[80:]

# Simple GRU-based VAD model
def init_vad_model(key, input_dim=40, hidden_dim=64):
    keys = jr.split(key, 6)
    scale_ih = jnp.sqrt(2.0 / input_dim)
    scale_hh = jnp.sqrt(2.0 / hidden_dim)
    return {
        'W_z': jr.normal(keys[0], (input_dim, hidden_dim)) * scale_ih,
        'U_z': jr.normal(keys[1], (hidden_dim, hidden_dim)) * scale_hh,
        'b_z': jnp.zeros(hidden_dim),
        'W_r': jr.normal(keys[2], (input_dim, hidden_dim)) * scale_ih,
        'U_r': jr.normal(keys[3], (hidden_dim, hidden_dim)) * scale_hh,
        'b_r': jnp.zeros(hidden_dim),
        'W_h': jr.normal(keys[4], (input_dim, hidden_dim)) * scale_ih,
        'U_h': jr.normal(keys[5], (hidden_dim, hidden_dim)) * scale_hh,
        'b_h': jnp.zeros(hidden_dim),
        'W_out': jr.normal(jr.fold_in(keys[0], 99), (hidden_dim, 1)) * 0.1,
        'b_out': jnp.zeros(1),
    }

def gru_step(params, h, x):
    """Single GRU step."""
    z = jax.nn.sigmoid(x @ params['W_z'] + h @ params['U_z'] + params['b_z'])
    r = jax.nn.sigmoid(x @ params['W_r'] + h @ params['U_r'] + params['b_r'])
    h_tilde = jnp.tanh(x @ params['W_h'] + (r * h) @ params['U_h'] + params['b_h'])
    h_new = (1 - z) * h + z * h_tilde
    return h_new

def vad_forward(params, x):
    """x: (batch, time, features) -> logits: (batch, time)."""
    batch_size, n_frames, _ = x.shape
    hidden_dim = params['W_z'].shape[1]
    h = jnp.zeros((batch_size, hidden_dim))

    outputs = []
    for t in range(n_frames):
        h = gru_step(params, h, x[:, t, :])
        logit = (h @ params['W_out'] + params['b_out']).squeeze(-1)
        outputs.append(logit)

    return jnp.stack(outputs, axis=1)

def bce_loss(params, features, labels):
    """Binary cross-entropy loss for VAD."""
    logits = vad_forward(params, features)
    probs = jax.nn.sigmoid(logits)
    probs = jnp.clip(probs, 1e-7, 1 - 1e-7)
    loss = -(labels * jnp.log(probs) + (1 - labels) * jnp.log(1 - probs))
    return jnp.mean(loss)

grad_fn = jax.jit(jax.value_and_grad(bce_loss))

# Training
params = init_vad_model(jr.PRNGKey(0))
lr = 5e-3
losses = []

for epoch in range(200):
    loss_val, grads = grad_fn(params, train_features, train_labels)
    params = jax.tree.map(lambda p, g: p - lr * g, params, grads)
    losses.append(float(loss_val))
    if epoch % 50 == 0:
        print(f"Epoch {epoch}: loss = {loss_val:.4f}")

# Evaluate on test set
test_logits = vad_forward(params, test_features)
test_preds = (jax.nn.sigmoid(test_logits) > 0.5).astype(jnp.float32)
accuracy = jnp.mean(test_preds == test_labels)
print(f"\nTest accuracy: {accuracy:.4f}")

# Visualise a test example
idx = 0
fig, axes = plt.subplots(3, 1, figsize=(14, 7))

axes[0].imshow(test_features[idx].T, aspect='auto', origin='lower', cmap='magma')
axes[0].set_title('Log-Mel Energy Features')
axes[0].set_ylabel('Mel bin')

axes[1].fill_between(range(200), test_labels[idx], alpha=0.4, color='#27ae60',
                     label='Ground truth')
axes[1].plot(jax.nn.sigmoid(test_logits[idx]), color='#e74c3c',
             linewidth=1.5, label='Predicted probability')
axes[1].axhline(0.5, color='gray', linestyle='--', linewidth=0.8)
axes[1].set_ylabel('Speech probability')
axes[1].legend()
axes[1].set_title('VAD Predictions')

axes[2].fill_between(range(200), test_labels[idx], alpha=0.4, color='#27ae60',
                     label='Ground truth')
```
