# Графы

*Графы моделируют отношения и связи: от социальных сетей до дорожных карт и цепочек зависимостей. В этом файле рассматриваются способы представления, BFS, DFS, кратчайшие пути, топологическая сортировка и компоненты связности, а также шаблоны обхода и поиска путей, которые доминируют в задачах про графы на собеседованиях.*

- Мы рассмотрели теорию графов в главе 12 (матрицы смежности, лапласиан, спектральные свойства) и главе 13 (деревья, планарность, раскраска). Здесь мы сосредоточимся на **алгоритмических шаблонах**: как обходить, искать и оптимизировать графы в коде.

- Два фундаментальных алгоритма на графах — это **BFS** и **DFS**. Почти любая задача на графы сводится к одному из них, возможно, с модификациями. Освойте эти два алгоритма, и вы сможете решить подавляющее большинство задач на графы.

## Представления графов

- **Список смежности**: для каждого узла храним список его соседей. Память: $O(|V| + |E|)$. Лучший выбор для разреженных графов (большинство графов в реальных задачах).

```python
# Undirected graph
graph = {
    0: [1, 2],
    1: [0, 3],
    2: [0, 3],
    3: [1, 2]
}

# From edge list
def build_graph(n, edges):
    graph = {i: [] for i in range(n)}
    for u, v in edges:
        graph[u].append(v)
        graph[v].append(u)  # omit for directed
    return graph
```

- **Матрица смежности**: матрица $n \times n$, где $A[i][j] = 1$, если ребро $(i, j)$ существует. Память: $O(|V|^2)$. Лучший выбор для плотных графов или когда требуется поиск ребра за $O(1)$.

- **Что и когда использовать**: список смежности почти для всего. Матрицу — только если граф плотный ($|E| \approx |V|^2$) или если нужна проверка существования ребра за константное время.

## Шаблон: BFS (поиск в ширину)

- BFS исследует узлы **уровень за уровнем**, используя очередь. Это основной алгоритм для:
    - Кратчайшего пути в **невзвешенных** графах
    - Обхода в порядке уровней
    - Поиска компонент связности
    - Любой задачи, где требуется найти «минимальное количество шагов»

```python
from collections import deque

def bfs(graph, start):
    visited = {start}
    queue = deque([start])

    while queue:
        node = queue.popleft()
        for neighbour in graph[node]:
            if neighbour not in visited:
                visited.add(neighbour)
                queue.append(neighbour)
```

- **Важно**: добавляйте в `visited` при **постановке в очередь**, а не при извлечении. Если помечать посещенным при извлечении, один и тот же узел может быть добавлен в очередь несколько раз из разных предшественников, что приведет к потере времени и потенциально неверным результатам.

### Легкий уровень: Количество островов

- **Задача**: дана двумерная сетка из '1' (суша) и '0' (вода), подсчитайте количество островов.

- **Шаблон**: итерируйтесь по сетке. Когда находите '1', запускайте BFS/DFS, чтобы пометить все связанные клетки суши как посещенные. Каждый запуск BFS соответствует одному острову.

```python
from collections import deque

def num_islands(grid):
    if not grid:
        return 0

    rows, cols = len(grid), len(grid[0])
    count = 0

    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == '1':
                count += 1
                # BFS to mark entire island
                queue = deque([(r, c)])
                grid[r][c] = '0'  # mark visited
                while queue:
                    cr, cc = queue.popleft()
                    for dr, dc in [(0,1),(0,-1),(1,0),(-1,0)]:
                        nr, nc = cr + dr, cc + dc
                        if 0 <= nr < rows and 0 <= nc < cols and grid[nr][nc] == '1':
                            grid[nr][nc] = '0'
                            queue.append((nr, nc))

    return count
```

- **Подвох**: шаблон `directions = [(0,1),(0,-1),(1,0),(-1,0)]` для 4-связных соседей в сетке используется почти в каждой задаче на сетки. Запомните его. Для 8-связных добавьте диагонали.

- **Подвох**: изменение входной сетки (`grid[r][c] = '0'`) позволяет избежать использования отдельного множества `visited`. Это допустимо на собеседованиях, но четко проговаривайте этот компромисс (мутация входных данных).

### Средний уровень: Гниющие апельсины

- **Задача**: свежие апельсины гниют, если находятся рядом с гнилым. Верните минимальное время, пока все апельсины не сгниют (или -1, если это невозможно).

- **Шаблон**: многоисточниковый BFS. Начните с помещения всех изначально гнилых апельсинов в очередь одновременно. Каждый уровень BFS — это один шаг по времени.

```python
from collections import deque

def oranges_rotting(grid):
    rows, cols = len(grid), len(grid[0])
    queue = deque()
    fresh = 0

    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == 2:
                queue.append((r, c))
            elif grid[r][c] == 1:
                fresh += 1

    if fresh == 0:
        return 0

    time = 0
    while queue and fresh > 0:
        time += 1
        for _ in range(len(queue)):
            cr, cc = queue.popleft()
            for dr, dc in [(0,1),(0,-1),(1,0),(-1,0)]:
                nr, nc = cr + dr, cc + dc
                if 0 <= nr < rows and 0 <= nc < cols and grid[nr][nc] == 1:
                    grid[nr][nc] = 2
                    fresh -= 1
                    queue.append((nr, nc))

    return time if fresh == 0 else -1
```

- **Ключевая идея**: многоисточниковый BFS обрабатывает все источники одновременно. Это дает кратчайшее расстояние от *любого* источника, что в точности соответствует вопросу «сколько времени пройдет, пока сгниет последний свежий апельсин».

## Шаблон: DFS (поиск в глубину)

- DFS исследует путь как можно глубже перед возвратом (backtracking). Использует стек (явный или стек вызовов через рекурсию). DFS — основной алгоритм для:
    - Обнаружения циклов
    - Топологической сортировки
    - Компонент связности
    - Поиска с возвратом / исчерпывающего поиска
    - Поиска пути с ограничениями

```python
def dfs(graph, node, visited=None):
    if visited is None:
        visited = set()
    visited.add(node)
    for neighbour in graph[node]:
        if neighbour not in visited:
            dfs(graph, neighbour, visited)
```

### Средний уровень: Расписание курсов (обнаружение цикла)

- **Задача**: даны $n$ курсов и пререквизиты, определите, можно ли завершить все курсы (т.е. нет ли циклических зависимостей).

- **Шаблон**: обнаружение цикла в ориентированном графе. Используйте DFS с тремя состояниями: не посещен, в процессе (на текущем пути DFS) и завершен.

```python
def can_finish(num_courses, prerequisites):
    graph = {i: [] for i in range(num_courses)}
    for course, prereq in prerequisites:
        graph[course].append(prereq)

    # 0 = unvisited, 1 = in-progress, 2 = completed
    state = [0] * num_courses

    def has_cycle(node):
        if state[node] == 1:
            return True   # back edge → cycle
        if state[node] == 2:
            return False  # already fully explored

        state[node] = 1  # mark in-progress
        for neighbour in graph[node]:
            if has_cycle(neighbour):
                return True
        state[node] = 2  # mark completed
        return False

    for course in range(num_courses):
        if has_cycle(course):
            return False
    return True
```

- **Зачем нужны три состояния**: два состояния (посещен/не посещен) не позволяют отличить «я сейчас исследую этот узел» от «я закончил исследование этого узла». Обнаружение узла, который находится в процессе исследования (состояние = 1), означает, что мы нашли цикл. Обнаружение узла, который уже полностью исследован (состояние = 2), — это просто перекрестное ребро, а не цикл.

### Средний уровень: Курсы II (Топологическая сортировка)

- **Задача**: вернуть допустимый порядок прохождения курсов (топологическую сортировку).

- **Шаблон (алгоритм Кана — на основе BFS)**: начните с узлов, у которых нет входящих ребер (входящая степень 0). Обработайте их, уменьшите входящую степень их соседей. Повторяйте.

```python
from collections import deque

def find_order(num_courses, prerequisites):
    graph = {i: [] for i in range(num_courses)}
    indegree = [0] * num_courses

    for course, prereq in prerequisites:
        graph[prereq].append(course)
        indegree[course] += 1

    queue = deque([i for i in range(num_courses) if indegree[i] == 0])
    order = []

    while queue:
        node = queue.popleft()
        order.append(node)
        for neighbour in graph[node]:
            indegree[neighbour] -= 1
            if indegree[neighbour] == 0:
                queue.append(neighbour)

    return order if len(order) == num_courses else []  # empty = cycle exists
```

- **Подвох**: если в результате оказалось меньше узлов, чем в графе, значит, существует цикл (входящая степень некоторых узлов так и не достигла 0).

## Кратчайшие пути

### Алгоритм Дейкстры

- Находит кратчайшие пути от источника до всех остальных узлов в графе с **неотрицательными** весами. Использует очередь с приоритетом (min-heap).

```python
import heapq

def dijkstra(graph, start):
    # graph: {node: [(neighbour, weight), ...]}
    dist = {node: float('inf') for node in graph}
    dist[start] = 0
    heap = [(0, start)]

    while heap:
        d, node = heapq.heappop(heap)
        if d > dist[node]:
            continue  # stale entry

        for neighbour, weight in graph[node]:
            new_dist = d + weight
            if new_dist < dist[neighbour]:
                dist[neighbour] = new_dist
                heapq.heappush(heap, (new_dist, neighbour))

    return dist
```

- Временная сложность: $O((|V| + |E|) \log |V|)$ с использованием бинарной кучи.

- **Подвох**: строка `if d > dist[node]: continue` обязательна. Без нее вы будете обрабатывать устаревшие записи из кучи, что может привести к деградации производительности до $O(|V|^2)$.

- **Подвох**: алгоритм Дейкстры не работает с отрицательными весами. Если ребро имеет отрицательный вес, жадное предположение (как только узел зафиксирован, его расстояние оптимально) нарушается. Используйте вместо него алгоритм Беллмана-Форда.

### Сложный уровень: Сетевая задержка

- **Задача**: даны $n$ узлов и взвешенные направленные ребра, найдите время, за которое сигнал дойдет до всех узлов от источника. Верните -1, если не все узлы достижимы.

```python
def network_delay(times, n, k):
    graph = {i: [] for i in range(1, n + 1)}
    for u, v, w in times:
        graph[u].append((v, w))

    dist = dijkstra(graph, k)
    max_time = max(dist.values())
    return max_time if max_time < float('inf') else -1
```

## Сильно связные компоненты

- В направленном графе **сильно связная компонента (SCC)** — это максимальное подмножество узлов, в котором каждый узел достижим из любого другого.

- **Алгоритм Косарайю**: (1) DFS на исходном графе, записывая порядок завершения. (2) Транспонирование графа (разворот всех ребер). (3) DFS на транспонированном графе в обратном порядке завершения. Каждое дерево DFS на шаге 3 является одной SCC.

- **Когда использовать**: поиск циклических зависимостей, 2-SAT, конденсация направленного графа в DAG из SCC.

## Резюме типичных ошибок

| Подвох | Пример | Решение |
|---------|---------|-----|
| Пометка посещенным при извлечении из очереди | Один и тот же узел добавлен в очередь несколько раз | Помечать посещенным при добавлении в очередь |
| Двухсостояний посещения в направленных графах | Невозможно отличить обратное ребро от перекрестного | Использовать три состояния: не посещен/в процессе/завершен |
| Дейкстра с отрицательными весами | Неверные кратчайшие пути | Использовать алгоритм Беллмана-Форда |
| Забыли `if d > dist[node]: continue` | Обработка устаревших записей в куче | Всегда пропускать, если текущее расстояние хуже |
| Проверка границ сетки | Выход индекса за пределы диапазона | `0 <= nr < rows and 0 <= nc < cols` |
| Не учтен граничный случай time=0 | Гниющие апельсины: нет свежих апельсинов | Проверить `fresh == 0` перед BFS |
| Построение направленного графа как ненаправленного | Пререквизиты работают в одну сторону | Добавлять ребро только в одном направлении |
