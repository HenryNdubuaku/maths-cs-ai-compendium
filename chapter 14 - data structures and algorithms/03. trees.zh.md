# 树

*tree 是文件系统、数据库、编译器及无数面试题背后的层级数据结构。本文件覆盖二叉树、BST、平衡树、trie、segment tree、Fenwick tree 与 Union-Find，并讲解遍历 pattern、递归思维及难度递增的题目。*

- **tree（树）** 是一种连通无环图（第 13 章）。最重要的变体是 **二叉树（binary tree）**：每个节点最多两个子节点（左、右）。tree 无处不在：编译器里的解析树、浏览器里的 DOM 树、机器学习里的决策树、数据库里的 B 树。

- 解 tree 题的关键直觉：**大多数 tree 题都能递归求解**。结构本身是递归的（一棵 tree 是根加两棵子树），解法也应是。掌握 "求解左子树、求解右子树、合并" 这个 pattern，你就能解决大多数 tree 题。

## 二叉树遍历

- 有四种标准方式访问每个节点：

    - **中序（inorder）**（左、根、右）：对 BST 而言，这会按升序访问节点。
    - **前序（preorder）**（根、左、右）：适用于序列化和复制 tree。
    - **后序（postorder）**（左、右、根）：适用于删除和计算大小。
    - **层序（level-order）**（BFS）：用队列逐层访问。

```python
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

def inorder(root):
    if not root:
        return []
    return inorder(root.left) + [root.val] + inorder(root.right)

def preorder(root):
    if not root:
        return []
    return [root.val] + preorder(root.left) + preorder(root.right)

def postorder(root):
    if not root:
        return []
    return postorder(root.left) + postorder(root.right) + [root.val]

from collections import deque

def level_order(root):
    if not root:
        return []
    result, queue = [], deque([root])
    while queue:
        level = []
        for _ in range(len(queue)):
            node = queue.popleft()
            level.append(node.val)
            if node.left:
                queue.append(node.left)
            if node.right:
                queue.append(node.right)
        result.append(level)
    return result
```

- **陷阱**：上面的递归遍历每步都因 `+` 拼接创建新 list，是 $O(n^2)$。要高效，就传入一个结果 list 并就地 append：

```python
def inorder_efficient(root, result=None):
    if result is None:
        result = []
    if root:
        inorder_efficient(root.left, result)
        result.append(root.val)
        inorder_efficient(root.right, result)
    return result
```

### 简单：Maximum Depth of Binary Tree

```python
def max_depth(root):
    if not root:
        return 0
    return 1 + max(max_depth(root.left), max_depth(root.right))
```

- **递归 pattern**：基本情况（空 → 0）、在子节点上递归、合并（1 + max）。同一 pattern 适用于几十道 tree 题。

### 简单：Invert Binary Tree

```python
def invert_tree(root):
    if not root:
        return None
    root.left, root.right = invert_tree(root.right), invert_tree(root.left)
    return root
```

### 中等：Lowest Common Ancestor

- **问题**：找同时是 $p$ 和 $q$ 祖先的最低节点。

- **Pattern**：若 $p$ 和 $q$ 都在左子树，LCA 在左子树。若都在右子树，则在右子树。若它们分散（一左一右），当前节点就是 LCA。

```python
def lowest_common_ancestor(root, p, q):
    if not root or root == p or root == q:
        return root

    left = lowest_common_ancestor(root.left, p, q)
    right = lowest_common_ancestor(root.right, p, q)

    if left and right:
        return root  # p and q are in different subtrees
    return left if left else right
```

- **陷阱**：上面假定 $p$ 和 $q$ 都存在于树中。若可能不存在，需要额外检查。

### 困难：Binary Tree Maximum Path Sum

- **问题**：找任意两节点间路径的最大和（路径不必经过根）。

```python
def max_path_sum(root):
    best = [float('-inf')]

    def dfs(node):
        if not node:
            return 0
        left = max(dfs(node.left), 0)   # ignore negative paths
        right = max(dfs(node.right), 0)

        # path through this node (possibly as the "bend")
        best[0] = max(best[0], node.val + left + right)

        # return the max gain this node can contribute to its parent
        return node.val + max(left, right)

    dfs(root)
    return best[0]
```

- **关键直觉**：在每个节点上有两个问题：(1) 经过该节点的最佳路径是什么（左 + 节点 + 右）？(2) 该节点能贡献给 **父节点** 的最佳路径是什么（节点 + max(左, 右)，因为路径不能在两层都分叉）？混淆这两个是最常见错误。

## 二叉搜索树（BST）

- **BST** 满足：对每个节点，左子树所有值更小、右子树所有值更大。这让搜索、插入、删除都是 $O(\log n)$（在平衡时）。

```python
def search_bst(root, target):
    if not root:
        return None
    if target < root.val:
        return search_bst(root.left, target)
    elif target > root.val:
        return search_bst(root.right, target)
    else:
        return root

def insert_bst(root, val):
    if not root:
        return TreeNode(val)
    if val < root.val:
        root.left = insert_bst(root.left, val)
    else:
        root.right = insert_bst(root.right, val)
    return root
```

- **陷阱**：BST 操作只有在树平衡时才是 $O(\log n)$。由有序插入构建的 BST 会退化为链表：每次操作 $O(n)$。这正是平衡 BST（AVL、红黑树）存在的原因。

### 中等：Validate Binary Search Tree

```python
def is_valid_bst(root, lo=float('-inf'), hi=float('inf')):
    if not root:
        return True
    if root.val <= lo or root.val >= hi:
        return False
    return (is_valid_bst(root.left, lo, root.val) and
            is_valid_bst(root.right, root.val, hi))
```

- **陷阱**：只检查 `left.val < root.val < right.val` 是错的。约束是左子树 *所有* 节点都更小，而不只是直接子节点。`lo`/`hi` 边界把约束向下传播。

### 中等：Kth Smallest Element in a BST

- **Pattern**：BST 的中序遍历按升序访问节点。第 $k$ 个被访问的节点就是答案。

```python
def kth_smallest(root, k):
    count = [0]
    result = [None]

    def inorder(node):
        if not node or result[0] is not None:
            return
        inorder(node.left)
        count[0] += 1
        if count[0] == k:
            result[0] = node.val
            return
        inorder(node.right)

    inorder(root)
    return result[0]
```

## Trie（前缀树）

- **trie（前缀树）** 按字符逐字符把字符串存进一棵 tree。每条边代表一个字符，从根到被标记节点的路径代表存入的字符串。trie 支持 $O(L)$ 查找，$L$ 为字符串长度，与存了多少字符串无关。

```python
class TrieNode:
    def __init__(self):
        self.children = {}
        self.is_end = False

class Trie:
    def __init__(self):
        self.root = TrieNode()

    def insert(self, word):
        node = self.root
        for char in word:
            if char not in node.children:
                node.children[char] = TrieNode()
            node = node.children[char]
        node.is_end = True

    def search(self, word):
        node = self.root
        for char in word:
            if char not in node.children:
                return False
            node = node.children[char]
        return node.is_end

    def starts_with(self, prefix):
        node = self.root
        for char in prefix:
            if char not in node.children:
                return False
            node = node.children[char]
        return True
```

- **何时用**：自动补全、拼写检查、文字游戏、IP 路由表。凡是需要基于前缀的操作时。

### 困难：Word Search II

- **问题**：给定一个字符棋盘和一个单词列表，找出所有能通过遍历相邻单元格形成的单词。

- **Pattern**：由单词列表构建 trie，再从每个单元格 DFS，用 trie 提前剪枝（若没有单词以当前前缀开头，就停）。

- **陷阱**：没有 trie 时，会对每个单词分别 DFS：$O(w \cdot m \cdot n \cdot 4^L)$。trie 在单词间共享前缀计算，大幅减少工作量。

## Union-Find（并查集）

- **Union-Find（并查集）** 跟踪一组不相交集合。两个操作：`find(x)` 返回 $x$ 所在集合的代表元，`union(x, y)` 合并 $x$ 和 $y$ 所在集合。

```python
class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n
        self.count = n  # number of connected components

    def find(self, x):
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])  # path compression
        return self.parent[x]

    def union(self, x, y):
        rx, ry = self.find(x), self.find(y)
        if rx == ry:
            return False  # already connected
        # union by rank
        if self.rank[rx] < self.rank[ry]:
            rx, ry = ry, rx
        self.parent[ry] = rx
        if self.rank[rx] == self.rank[ry]:
            self.rank[rx] += 1
        self.count -= 1
        return True
```

- 加上路径压缩和按秩合并，两个操作都是 $O(\alpha(n)) \approx O(1)$ 摊还（反阿克曼函数，实质常数）。

- **何时用**：连通分量、无向图环检测、Kruskal 最小生成树、等价物品分组。

### 中等：Number of Connected Components

```python
def count_components(n, edges):
    uf = UnionFind(n)
    for u, v in edges:
        uf.union(u, v)
    return uf.count
```

### 中等：Redundant Connection

- **问题**：找出那条删去后使图变成 tree 的边（即造成环的那条边）。

- **Pattern**：逐条处理边。第一条两端点已在同一连通分量的边就造成环。

```python
def find_redundant(edges):
    uf = UnionFind(len(edges) + 1)
    for u, v in edges:
        if not uf.union(u, v):
            return [u, v]  # already connected → this edge creates a cycle
```

## Segment Tree 与 Fenwick Tree

- **segment tree（线段树）** 回答区间查询（子数组求和、最小、最大），并支持单点更新，二者均为 $O(\log n)$。

- **Fenwick tree（树状数组，Binary Indexed Tree）** 是更简单、更快的替代方案，适用于前缀和查询和单点更新。它用一个巧妙的位运算技巧：每个位置存一段部分和，覆盖范围由最低置位决定。

```python
class FenwickTree:
    def __init__(self, n):
        self.n = n
        self.tree = [0] * (n + 1)

    def update(self, i, delta):
        i += 1  # 1-indexed
        while i <= self.n:
            self.tree[i] += delta
            i += i & (-i)  # add lowest set bit

    def prefix_sum(self, i):
        i += 1
        total = 0
        while i > 0:
            total += self.tree[i]
            i -= i & (-i)  # remove lowest set bit
        return total

    def range_sum(self, l, r):
        return self.prefix_sum(r) - (self.prefix_sum(l - 1) if l > 0 else 0)
```

- **何时用**：需要反复做带更新的区间查询。只需要前缀和时优先用 Fenwick tree；需要任意区间操作（最小、最大、GCD）时用 segment tree。

## 常见陷阱汇总

| 陷阱 | 示例 | 修复 |
|---------|---------|-----|
| BST 只检查直接子节点 | `left.val < root.val` 漏掉深层违规 | 传 `lo`/`hi` 边界 |
| 递归中 $O(n^2)$ 的 list 拼接 | `inorder(left) + [val] + inorder(right)` | append 到共享 list |
| 忘基本情况 | 空 tree 上无限递归 | `if not root: return` |
| 混淆 "经过该节点" 与 "贡献给父节点" | 最大路径和：在两层分叉 | 返回单枝给父、单独跟踪双枝 |
| Fenwick 的 1 索引 vs 0 索引 | 树数组差一 | 进入时始终 `i += 1` |
| Union-Find 不加路径压缩 | 最坏每次 find $O(n)$ | `self.parent[x] = self.find(self.parent[x])` |
