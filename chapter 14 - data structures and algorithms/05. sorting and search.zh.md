# 排序、搜索与算法设计

*排序与搜索是最基础的算法操作。本文件覆盖排序算法、二分查找 pattern、分治、贪心、动态规划与回溯。*

- 每种数据结构支撑算法，每种算法依赖数据结构。本文件覆盖 **设计范式**：解题的高层策略。一旦识别出适用哪种范式，实现就水到渠成。

## 排序算法

- 排序是计算机科学中研究最多的问题。理解这些算法能培养对递归、分治、复杂度分析的直觉。

| 算法 | 最好 | 平均 | 最坏 | 空间 | 稳定？ |
|-----------|------|---------|-------|-------|---------|
| 冒泡排序 | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 是 |
| 插入排序 | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 是 |
| 归并排序 | $O(n \log n)$ | $O(n \log n)$ | $O(n \log n)$ | $O(n)$ | 是 |
| 快速排序 | $O(n \log n)$ | $O(n \log n)$ | $O(n^2)$ | $O(\log n)$ | 否 |
| 堆排序 | $O(n \log n)$ | $O(n \log n)$ | $O(n \log n)$ | $O(1)$ | 否 |
| 计数排序 | $O(n + k)$ | $O(n + k)$ | $O(n + k)$ | $O(k)$ | 是 |
| 基数排序 | $O(d(n + k))$ | $O(d(n + k))$ | $O(d(n + k))$ | $O(n + k)$ | 是 |

- **稳定** 指相等元素保持相对顺序。在按多键排序时这很重要。

- 基于比较的排序的 **下界** 是 $\Omega(n \log n)$。证明用到决策树（第 13 章）：任何比较排序必须区分所有 $n!$ 种排列，需要至少 $\log_2(n!) = \Omega(n \log n)$ 次比较。计数排序和基数排序靠不比较元素打破了这一下界。

### 归并排序

- 把数组对半分，递归排序两半，再合并两段有序数组。始终 $O(n \log n)$，额外 $O(n)$ 空间。

```python
def merge_sort(arr):
    if len(arr) <= 1:
        return arr

    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])

    return merge(left, right)

def merge(left, right):
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:  # <= for stability
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    result.extend(left[i:])
    result.extend(right[j:])
    return result
```

- **陷阱**：合并时用 `<` 而非 `<=` 会破坏稳定性（右半的相等元素会排到左半之前）。

### 快速排序

- 选一个 **pivot（基准）**，把元素划分为 "小于 pivot" 和 "大于 pivot"，递归排序每段。平均 $O(n \log n)$，最坏 $O(n^2)$（pivot 总是最小或最大元素时）。

```python
def quicksort(arr, lo=0, hi=None):
    if hi is None:
        hi = len(arr) - 1
    if lo >= hi:
        return

    pivot_idx = partition(arr, lo, hi)
    quicksort(arr, lo, pivot_idx - 1)
    quicksort(arr, pivot_idx + 1, hi)

def partition(arr, lo, hi):
    pivot = arr[hi]  # Lomuto: pivot is last element
    i = lo
    for j in range(lo, hi):
        if arr[j] < pivot:
            arr[i], arr[j] = arr[j], arr[i]
            i += 1
    arr[i], arr[hi] = arr[hi], arr[i]
    return i
```

- **pivot 策略**：末元素（简单，对已排序输入差）、随机（期望 $O(n \log n)$）、三数取中（实用选择）。面试中始终优先用随机 pivot，免得讨论最坏情形。

- **陷阱**：快速排序的 $O(n^2)$ 最坏发生在已排序数组配上首/末 pivot。实际中，随机 pivot 或三数取中能消除它。

### 计数排序

- 当值是已知范围 $[0, k)$ 内的整数时，统计出现次数再重构：$O(n + k)$。非比较，所以能打破 $O(n \log n)$。

```python
def counting_sort(arr, k):
    count = [0] * k
    for x in arr:
        count[x] += 1
    result = []
    for val in range(k):
        result.extend([val] * count[val])
    return result
```

- **何时用**：范围 $k$ 不比 $n$ 大太多。若 $k = O(n)$，这是 $O(n)$。若 $k \gg n$（例如对 10 个在 $[0, 10^9]$ 内的数排序），计数排序浪费内存。

## Pattern：二分查找

- 二分查找通过反复减半搜索空间，在有序数组中 $O(\log n)$ 找到目标。但二分查找远不止 "在有序数组里找一个数"。通用 pattern 是：**在单调条件上搜索**。

- **模板**（能避免差一错误的那一版）：

```python
def binary_search(arr, target):
    lo, hi = 0, len(arr) - 1

    while lo <= hi:
        mid = lo + (hi - lo) // 2  # avoids overflow in other languages
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            lo = mid + 1
        else:
            hi = mid - 1

    return -1  # not found
```

- **下界**（第一个 $\geq$ 目标的元素）：

```python
def lower_bound(arr, target):
    lo, hi = 0, len(arr)
    while lo < hi:
        mid = (lo + hi) // 2
        if arr[mid] < target:
            lo = mid + 1
        else:
            hi = mid
    return lo
```

- **陷阱**：`lo <= hi` 与 `lo < hi`、以及 `hi = mid` 与 `hi = mid - 1` 之间的差别，决定你找的是精确匹配还是边界。用 2 元素数组画一画验证。

### 中等：Search in Rotated Sorted Array

- **问题**：一个有序数组在某个 pivot 处被旋转。找一个目标值。

- **Pattern**：每步总有一半是有序的。判断哪一半有序，并检查目标是否落在那一半。

```python
def search_rotated(nums, target):
    lo, hi = 0, len(nums) - 1

    while lo <= hi:
        mid = (lo + hi) // 2
        if nums[mid] == target:
            return mid

        # left half is sorted
        if nums[lo] <= nums[mid]:
            if nums[lo] <= target < nums[mid]:
                hi = mid - 1
            else:
                lo = mid + 1
        # right half is sorted
        else:
            if nums[mid] < target <= nums[hi]:
                lo = mid + 1
            else:
                hi = mid - 1

    return -1
```

- **陷阱**：`nums[lo] <= nums[mid]` 里的 `<=`（而非 `<`）很关键。当 `lo == mid`（剩 2 个元素）时，我们必须正确识别出有序的一半。

### 困难：Median of Two Sorted Arrays

- **问题**：在 $O(\log(m + n))$ 内找出两个有序数组的中位数。

- **Pattern**：在较短数组的划分点上做二分查找。划分把两个数组分成两部分，使左边所有元素都小于右边所有元素。

```python
def find_median(nums1, nums2):
    if len(nums1) > len(nums2):
        nums1, nums2 = nums2, nums1  # ensure nums1 is shorter

    m, n = len(nums1), len(nums2)
    lo, hi = 0, m
    half = (m + n + 1) // 2

    while lo <= hi:
        i = (lo + hi) // 2          # partition point in nums1
        j = half - i                 # partition point in nums2

        left1 = nums1[i - 1] if i > 0 else float('-inf')
        right1 = nums1[i] if i < m else float('inf')
        left2 = nums2[j - 1] if j > 0 else float('-inf')
        right2 = nums2[j] if j < n else float('inf')

        if left1 <= right2 and left2 <= right1:
            # correct partition
            if (m + n) % 2 == 1:
                return max(left1, left2)
            return (max(left1, left2) + min(right1, right2)) / 2
        elif left1 > right2:
            hi = i - 1
        else:
            lo = i + 1
```

- 这是二分查找里最难的题之一。关键直觉是：你不是在找一个值，而是在找一个满足条件的 **划分点**。

### 元 pattern：对答案二分

- 很多不像二分查找的题，都能通过对答案做二分查找解决。若答案是个数，且你能写出一个单调函数 `is_feasible(x)`（对所有 $x \geq$ 最优解为 True，或对所有 $x \geq$ 最优解为 False），那么对 $x$ 做二分即可。

- **示例**："在 $d$ 天内运完所有包裹所需的最小船容量是多少？" 对容量做二分。对每个候选容量，贪心检查能否在 $d$ 天内运完。

```python
def ship_within_days(weights, days):
    lo, hi = max(weights), sum(weights)

    while lo < hi:
        mid = (lo + hi) // 2
        # can we ship with capacity mid in <= days?
        current_load, num_days = 0, 1
        for w in weights:
            if current_load + w > mid:
                num_days += 1
                current_load = 0
            current_load += w

        if num_days <= days:
            hi = mid
        else:
            lo = mid + 1

    return lo
```

## Pattern：贪心算法

- **贪心（greedy）** 算法在每一步做局部最优选择，希望导向全局最优。当问题具有 **贪心选择性质**（局部最优导致全局最优）和 **最优子结构**（最优解包含子问题的最优解）时贪心有效。

### 中等：Jump Game

- **问题**：给定一个数组，`nums[i]` 是位置 $i$ 处的最大跳跃长度，判断能否到达最后那个索引。

```python
def can_jump(nums):
    max_reach = 0
    for i, jump in enumerate(nums):
        if i > max_reach:
            return False  # cannot reach this position
        max_reach = max(max_reach, i + jump)
    return True
```

- **为什么贪心有效**：我们只需知道最远能到哪。若当前位置已超出最远可达，就卡住了。否则更新最远可达。

### 中等：Merge Intervals

- **问题**：合并重叠区间。

```python
def merge_intervals(intervals):
    intervals.sort(key=lambda x: x[0])
    merged = [intervals[0]]

    for start, end in intervals[1:]:
        if start <= merged[-1][1]:
            merged[-1][1] = max(merged[-1][1], end)
        else:
            merged.append([start, end])

    return merged
```

- **Pattern**：按开始时间排序，再贪心合并。若当前区间与最后一段合并区间重叠就延伸；否则开始新的合并区间。

- **陷阱**：用 `merged[-1][1] = end` 而非 `merged[-1][1] = max(merged[-1][1], end)`。一个区间可能完全被另一个包含（如 [1, 10] 和 [2, 5]）。

## Pattern：动态规划

- **动态规划（dynamic programming，DP）** 通过把问题拆成重叠子问题、每个子问题只解一次并保存结果来求解。当问题具有 **最优子结构** 和 **重叠子问题** 时 DP 有效。

- **两种做法**：
    - **自顶向下（记忆化）**：写出自然的递归解法，再用字典缓存结果。
    - **自底向上（制表）**：从最小子问题起构建一张表。

- **如何识别 DP**：问题求最优值（最小/最大）、计数、或存在性，且当前决策依赖之前的决策。若画出递归树能看到重复子问题，就是 DP。

### 简单：Climbing Stairs

- **问题**：$n$ 级台阶，每次可爬 1 或 2 级。有多少种不同方法？

- 这是 Fibonacci：$f(n) = f(n-1) + f(n-2)$。

```python
def climb_stairs(n):
    if n <= 2:
        return n
    a, b = 1, 2
    for _ in range(3, n + 1):
        a, b = b, a + b
    return b
```

- $O(n)$ 时间、$O(1)$ 空间。不需要整张记忆化表，因为每个状态只依赖前两个。

### 中等：Coin Change

- **问题**：给定硬币面额和目标金额，找出所需的最少硬币数。

- **状态**：`dp[amount]` = 凑出 `amount` 所需的最少硬币数。
- **转移**：`dp[amount] = min(dp[amount - coin] + 1)`，遍历每种硬币。
- **基本情况**：`dp[0] = 0`。

```python
def coin_change(coins, amount):
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0

    for a in range(1, amount + 1):
        for coin in coins:
            if coin <= a and dp[a - coin] + 1 < dp[a]:
                dp[a] = dp[a - coin] + 1

    return dp[amount] if dp[amount] != float('inf') else -1
```

- **陷阱**：初始化用 `float('inf')`（而非 0 或 -1）。最小值比较只有在不可达状态为无穷时才有效。

### 中等：Longest Common Subsequence

- **问题**：给定两个字符串，找它们最长公共子序列的长度。

- **状态**：`dp[i][j]` = `text1[:i]` 与 `text2[:j]` 的 LCS。
- **转移**：若 `text1[i-1] == text2[j-1]`，则 `dp[i][j] = dp[i-1][j-1] + 1`；否则 `dp[i][j] = max(dp[i-1][j], dp[i][j-1])`。

```python
def longest_common_subsequence(text1, text2):
    m, n = len(text1), len(text2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if text1[i - 1] == text2[j - 1]:
                dp[i][j] = dp[i - 1][j - 1] + 1
            else:
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])

    return dp[m][n]
```

### 困难：0/1 Knapsack

- **问题**：给定带重量和价值的物品及容量 $W$，在不超 $W$ 的前提下使总价值最大。

- **状态**：`dp[i][w]` = 用前 $i$ 个物品、容量为 $w$ 时的最大价值。
- **转移**：`dp[i][w] = max(dp[i-1][w], dp[i-1][w - weight[i]] + value[i])`（跳过或取第 $i$ 个物品）。

```python
def knapsack(weights, values, capacity):
    n = len(weights)
    dp = [[0] * (capacity + 1) for _ in range(n + 1)]

    for i in range(1, n + 1):
        for w in range(capacity + 1):
            dp[i][w] = dp[i - 1][w]  # skip item i
            if weights[i - 1] <= w:
                dp[i][w] = max(dp[i][w],
                               dp[i - 1][w - weights[i - 1]] + values[i - 1])

    return dp[n][capacity]
```

- **空间优化**：因为每行只依赖上一行，改用一维数组，且 $w$ 从右向左迭代：

```python
def knapsack_optimised(weights, values, capacity):
    dp = [0] * (capacity + 1)
    for i in range(len(weights)):
        for w in range(capacity, weights[i] - 1, -1):  # right to left!
            dp[w] = max(dp[w], dp[w - weights[i]] + values[i])
    return dp[capacity]
```

- **陷阱**：一维版本从左到右迭代会让第 $i$ 个物品被多次使用（完全背包）。从右向左保证每个物品最多用一次。

## Pattern：回溯

- **回溯（backtracking）** 是带剪枝的穷举搜索。逐步构建解，一旦部分解不可能导向合法完整解就放弃（回溯）。

- **模板**：

```python
def backtrack(candidates, path, result):
    if is_solution(path):
        result.append(path[:])  # copy!
        return

    for candidate in get_candidates(path):
        if is_valid(candidate, path):
            path.append(candidate)     # choose
            backtrack(candidates, path, result)  # explore
            path.pop()                 # unchoose (backtrack)
```

### 中等：Subsets

```python
def subsets(nums):
    result = []
    def backtrack(start, path):
        result.append(path[:])
        for i in range(start, len(nums)):
            path.append(nums[i])
            backtrack(i + 1, path)
            path.pop()
    backtrack(0, [])
    return result
```

### 中等：Combination Sum

- **问题**：找出和为目标值的所有唯一组合（元素可复用）。

```python
def combination_sum(candidates, target):
    result = []
    def backtrack(start, path, remaining):
        if remaining == 0:
            result.append(path[:])
            return
        for i in range(start, len(candidates)):
            if candidates[i] > remaining:
                break  # prune: sorted, so all further candidates are too large
            path.append(candidates[i])
            backtrack(i, path, remaining - candidates[i])  # i, not i+1: reuse allowed
            path.pop()

    candidates.sort()  # sort for pruning
    backtrack(0, [], target)
    return result
```

- **陷阱**：`backtrack(i, ...)` 允许复用同一元素；`backtrack(i + 1, ...)` 会跳到下一个元素（不复用）。弄反了是最常见的回溯 bug。

### 困难：N-Queens

- **问题**：在 $n \times n$ 棋盘上放 $n$ 个皇后，使任意两个互不攻击。

```python
def solve_n_queens(n):
    result = []
    cols = set()
    pos_diag = set()  # (row + col) is constant on / diagonals
    neg_diag = set()  # (row - col) is constant on \ diagonals

    board = [['.' ] * n for _ in range(n)]

    def backtrack(row):
        if row == n:
            result.append([''.join(r) for r in board])
            return

        for col in range(n):
            if col in cols or (row + col) in pos_diag or (row - col) in neg_diag:
                continue

            cols.add(col)
            pos_diag.add(row + col)
            neg_diag.add(row - col)
            board[row][col] = 'Q'

            backtrack(row + 1)

            cols.remove(col)
            pos_diag.remove(row + col)
            neg_diag.remove(row - col)
            board[row][col] = '.'

    backtrack(0)
    return result
```

- **关键直觉**：对角线编码。对 `/` 对角线，`row + col` 恒定；对 `\` 对角线，`row - col` 恒定。用集合跟踪列和对角线使合法性检查 $O(1)$。

## 常见陷阱汇总

| 陷阱 | 示例 | 修复 |
|---------|---------|-----|
| 二分里 `lo <= hi` vs `lo < hi` | 边界差一 | 按 `hi` 是闭区间还是开区间来选 |
| 一维背包从左到右 | 物品被多次使用 | 0/1 背包从右向左迭代 |
| 回溯不拷贝 path | `result.append(path)`——所有条目指向同一 list | `result.append(path[:])` 或 `path.copy()` |
| `backtrack(i)` vs `backtrack(i+1)` | 元素复用 vs 不复用 | 与题意匹配 |
| 排序回溯里漏 `break` | 探索过大的候选 | 排序 + 候选超余量时 break |
| DP 初始化 | `dp[0]` 错 → 后续全错 | 仔细定义基本情况 |
| 不加证明就贪心 | 贪心不一定有效 | 验证贪心选择性质 |
| 多键排序用不稳定排序 | 相等元素相对顺序丢失 | 用稳定排序（归并排序、Python 的 `sorted`） |
