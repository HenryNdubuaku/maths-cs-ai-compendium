# Связные списки, стеки и очереди

*Связные списки, стеки и очереди являются строительными блоками более сложных структур данных. В этом файле рассматриваются принципы их работы, а затем выстраиваются ключевые паттерны: быстрые/медленные указатели, монотонные стеки и приоритетные очереди на основе кучи — через постепенно усложняющиеся задачи с разбором типичных ошибок на каждом этапе.*

- Массивы обеспечивают быстрый произвольный доступ, но требуют затратных операций вставки. **Связные списки** обеспечивают быструю вставку, но не поддерживают произвольный доступ. **Стеки** и **очереди** ограничивают доступ к одному или двум концам, и именно это ограничение делает их мощными инструментами: ограничивая возможные действия, они упрощают то, о чем вам нужно думать.

## Связные списки

- **Односвязный список** — это цепочка узлов. Каждый узел хранит значение и указатель на следующий узел. Последний узел указывает на `null`.

```python
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next
```

- **Преимущества перед массивами**: вставка или удаление в известной позиции выполняются за $O(1)$ (достаточно перенаправить указатели). Нет необходимости сдвигать элементы.

- **Недостатки**: доступ к элементу $i$ требует обхода за $O(i)$ (нет произвольного доступа). Низкая локальность кеша (узлы разбросаны в памяти).

- **Двусвязные списки** добавляют указатель `prev`, позволяя выполнять обход в обратном направлении. Используются в LRU-кешах (удаление любого узла за константное время) и истории браузера (назад/вперед).

| Операция | Односвязный | Двусвязный |
|-----------|--------|--------|
| Доступ по индексу | $O(n)$ | $O(n)$ |
| Вставка в начало | $O(1)$ | $O(1)$ |
| Вставка в конец | $O(n)$ или $O(1)$* | $O(1)$ |
| Удаление узла | $O(n)$** | $O(1)$ |
| Поиск | $O(n)$ | $O(n)$ |

*с указателем на хвост. **требуется предшественник, что требует обхода.

- **Фиктивные узлы** (dummy head/tail) упрощают обработку граничных случаев. Без фиктивной головы вставка в начало или удаление головы требуют написания кода для особых случаев. С фиктивным узлом у каждого реального узла есть предшественник.

```python
# Without dummy: special case for head deletion
def delete_head(head):
    if not head:
        return None
    return head.next

# With dummy: uniform logic
dummy = ListNode(0)
dummy.next = head
# now every deletion is: prev.next = prev.next.next
```

- **Ловушка**: забыть обработать пустой список (`head is None`) или список из одного элемента. Всегда тестируйте эти граничные случаи.

## Паттерн: быстрые и медленные указатели (алгоритм Флойда)

- Используйте два указателя, движущихся с разной скоростью, для обнаружения свойств связных списков. **Медленный** указатель перемещается на один шаг за раз; **быстрый** указатель — на два шага.

### Легко: цикл в связном списке

- **Задача**: определить, содержит ли связный список цикл.

- **Паттерн**: если цикл существует, быстрый указатель в конечном итоге догонит медленный (они встретятся). Если цикла нет, быстрый указатель достигнет `null`.

```python
def has_cycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            return True
    return False
```

- **Почему это работает**: если длина цикла равна $c$, быстрый указатель сокращает разрыв на 1 узел за шаг. Они должны встретиться в пределах $c$ шагов после того, как медленный указатель войдет в цикл.

- **Ловушка**: проверка `fast and fast.next` (а не только `fast.next`). Если `fast` равен `None`, вызов `fast.next` приведет к ошибке.

### Средне: поиск середины связного списка

- **Задача**: вернуть средний узел.

- **Паттерн**: когда быстрый указатель достигает конца, медленный указатель находится в середине.

```python
def find_middle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
    return slow  # slow is at the middle (or second middle if even length)
```

### Средне: цикл в связном списке II (поиск начала цикла)

- **Задача**: вернуть узел, с которого начинается цикл.

- **Паттерн**: после того как быстрый и медленный указатели встретились, сбросьте один указатель в начало списка. Перемещайте оба со скоростью 1. Они встретятся в начале цикла.

```python
def detect_cycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            # reset one pointer to head
            slow = head
            while slow != fast:
                slow = slow.next
                fast = fast.next
            return slow
    return None
```

- **Почему это работает**: пусть расстояние от головы до начала цикла равно $a$, а расстояние от начала цикла до точки встречи — $b$. Медленный указатель прошел $a + b$. Быстрый указатель прошел $2(a + b)$. Разница составляет один полный цикл: $a + b = c$ (длина цикла). Таким образом, $a = c - b$: расстояние от головы до начала цикла равно расстоянию от точки встречи до начала цикла (при движении вперед по циклу).

### Сложно: разворот связного списка группами по K

- **Задача**: разворачивать каждые $k$ последовательных узлов в связном списке.

```python
def reverse_k_group(head, k):
    # check if we have k nodes left
    node = head
    for _ in range(k):
        if not node:
            return head
        node = node.next

    # reverse k nodes
    prev, curr = None, head
    for _ in range(k):
        nxt = curr.next
        curr.next = prev
        prev = curr
        curr = nxt

    # head is now the tail of the reversed group
    # recursively process the rest
    head.next = reverse_k_group(curr, k)
    return prev  # prev is the new head of this group
```

- **Ловушка**: паттерн разворота на месте (`prev, curr, nxt`) стоит запомнить. Нарисуйте его: на каждом шаге вы направляете `curr.next` назад к `prev`, а затем продвигаете все три указателя. Ошибка в порядке действий приведет к повреждению списка.

## Стеки

- **Стек** работает по принципу LIFO (Last In, First Out — «последним пришел, первым ушел»): последний добавленный элемент удаляется первым. Представьте стопку тарелок.

- Операции: `push(x)` добавляет элемент на вершину, `pop()` удаляет элемент с вершины, `peek()` просматривает вершину без удаления. Все операции выполняются за $O(1)$.

- Стеки являются неявной структурой, лежащей в основе **рекурсии** (стек вызовов), **вычисления выражений** (преобразование инфиксной записи в постфиксную) и **операций отмены** (каждое действие помещается в стек, отмена извлекает последнее из них).

### Easy: Valid Parentheses

- **Problem**: given a string of brackets `()[]{}`, determine if they are balanced.

- **Pattern**: push opening brackets onto the stack. When you see a closing bracket, check that the top of the stack is the matching opener.

```python
def is_valid(s):
    stack = []
    matching = {')': '(', ']': '[', '}': '{'}

    for char in s:
        if char in matching:
            if not stack or stack[-1] != matching[char]:
                return False
            stack.pop()
        else:
            stack.append(char)

    return len(stack) == 0
```

- **Pitfall**: forgetting `len(stack) == 0` at the end. The string "(((" has no mismatches but is not valid because unclosed brackets remain.

## Pattern: Monotonic Stack

- A **monotonic stack** maintains elements in sorted order (either increasing or decreasing). When a new element would violate the order, you pop elements until the order is restored.

- **When to use**: problems asking "for each element, find the next/previous greater/smaller element." The stack gives $O(n)$ total because each element is pushed and popped at most once.

### Medium: Daily Temperatures

- **Problem**: given daily temperatures, for each day find how many days until a warmer temperature.

- **Pattern**: use a stack of indices. When the current temperature is higher than the stack's top, pop and record the distance.

```python
def daily_temperatures(temperatures):
    n = len(temperatures)
    result = [0] * n
    stack = []  # stack of indices, temperatures in decreasing order

    for i in range(n):
        while stack and temperatures[i] > temperatures[stack[-1]]:
            prev = stack.pop()
            result[prev] = i - prev
        stack.append(i)

    return result
```

- Each element is pushed once and popped at most once: $O(n)$ total.

- **Pitfall**: storing indices on the stack (not values). You need the index to compute the distance.

### Hard: Largest Rectangle in Histogram

- **Problem**: given an array of bar heights, find the area of the largest rectangle.

- **Pattern**: for each bar, find how far left and right it can extend (i.e., the nearest shorter bar on each side). A monotonic increasing stack tracks this efficiently.

```python
def largest_rectangle(heights):
    stack = []  # stack of indices, heights in increasing order
    max_area = 0
    heights.append(0)  # sentinel to flush the stack at the end

    for i, h in enumerate(heights):
        start = i
        while stack and stack[-1][1] > h:
            idx, height = stack.pop()
            max_area = max(max_area, height * (i - idx))
            start = idx  # this bar can extend back to where the popped bar started
        stack.append((start, h))

    heights.pop()  # remove sentinel
    return max_area
```

- **Pitfall**: the `start = idx` line is subtle. When we pop a bar that is taller than the current bar, the current bar can extend backward to where the popped bar started (because all bars in between were at least as tall as the popped bar). Missing this line gives incorrect areas.

- **Pitfall**: the sentinel `heights.append(0)` ensures all remaining bars in the stack are processed. Without it, bars that never encounter a shorter bar to their right are missed.

## Queues

- A **queue** is FIFO (First In, First Out): elements are added at the back and removed from the front. Think of a line at a shop.

- A **deque** (double-ended queue) supports $O(1)$ insertion and removal at both ends. Python's `collections.deque` is the standard implementation.

- Queues are the structure behind **BFS** (breadth-first search, chapter 14 file 4), **task scheduling**, and **message passing**.

### Easy: Implement Queue Using Stacks

- **Problem**: implement a queue using only two stacks.

- **Pattern**: use one stack for pushing and one for popping. When the pop stack is empty, transfer all elements from the push stack (reversing the order).

```python
class MyQueue:
    def __init__(self):
        self.push_stack = []
        self.pop_stack = []

    def push(self, x):
        self.push_stack.append(x)

    def pop(self):
        if not self.pop_stack:
            while self.push_stack:
                self.pop_stack.append(self.push_stack.pop())
        return self.pop_stack.pop()

    def peek(self):
        if not self.pop_stack:
            while self.push_stack:
                self.pop_stack.append(self.push_stack.pop())
        return self.pop_stack[-1]

    def empty(self):
        return not self.push_stack and not self.pop_stack
```

- Amortised $O(1)$ per operation: each element is moved between stacks at most once.

## Priority Queues and Heaps

- A **priority queue** returns the smallest (or largest) element first, regardless of insertion order. The standard implementation is a **binary heap**.

- A **min-heap** is a complete binary tree where every parent is smaller than its children. The minimum is always at the root. Stored as an array: the children of node $i$ are at positions $2i + 1$ and $2i + 2$.

| Operation | Time |
|-----------|------|
| Insert | $O(\log n)$ |
| Get min | $O(1)$ |
| Extract min | $O(\log n)$ |
| Build heap from array | $O(n)$ |

- Python's `heapq` module provides a min-heap. For a max-heap, negate the values.

```python
import heapq

# Min-heap
h = []
heapq.heappush(h, 5)
heapq.heappush(h, 2)
heapq.heappush(h, 8)
print(heapq.heappop(h))  # 2 (smallest)

# Max-heap trick: negate values
heapq.heappush(h, -10)
print(-heapq.heappop(h))  # 10 (largest)
```

### Medium: Kth Largest Element in an Array

- **Problem**: find the kth largest element.

- **Pattern**: maintain a min-heap of size $k$. The heap's root is the kth largest. If the heap has $k$ elements and a new element is larger than the root, replace the root.

```python
import heapq

def find_kth_largest(nums, k):
    heap = nums[:k]
    heapq.heapify(heap)  # O(k)

    for num in nums[k:]:
        if num > heap[0]:
            heapq.heapreplace(heap, num)  # pop min, push num: O(log k)

    return heap[0]
```

- Временная сложность $O(n \log k)$, пространственная сложность $O(k)$. Это значительно лучше сортировки ($O(n \log n)$), когда $k \ll n$.

- **Ловушка**: использование max-heap размера $n$ с $k$ операциями извлечения также работает, но медленнее: $O(n + k \log n)$. Min-heap размера $k$ — оптимальный подход.

### Сложная задача: слияние K отсортированных списков

- **Задача**: объединить $k$ отсортированных связных списков в один отсортированный список.

- **Шаблон**: используйте min-heap, содержащий головы каждого из списков. Извлекайте наименьший элемент, добавляйте его в результат и помещайте его следующий узел в кучу.

```python
import heapq

def merge_k_lists(lists):
    heap = []
    for i, lst in enumerate(lists):
        if lst:
            heapq.heappush(heap, (lst.val, i, lst))

    dummy = ListNode(0)
    curr = dummy

    while heap:
        val, i, node = heapq.heappop(heap)
        curr.next = node
        curr = curr.next
        if node.next:
            heapq.heappush(heap, (node.next.val, i, node.next))

    return dummy.next
```

- $O(n \log k)$, где $n$ — общее количество узлов. В куче всегда находится не более $k$ элементов.

- **Ловушка**: `i` (индекс) в кортеже кучи служит для разрешения конфликтов при сравнении. Без него Python попытается сравнить объекты `ListNode` при равенстве значений, что приведет к ошибке, так как `ListNode` не поддерживает оператор `<`. Индекс гарантирует корректное сравнение.

## Сводка типичных ловушек

| Ловушка | Пример | Исправление |
|---------|---------|-----|
| Нулевой указатель при `fast.next` | Поиск цикла через `while fast.next` | Проверяйте `fast and fast.next` |
| Отсутствие обработки пустого списка | Разворот `None` | Добавьте проверку `if not head` |
| Опустошение стека | Извлечение из пустого стека | Проверяйте `len(stack) > 0` или `if stack` |
| Забытый фиктивный узел (sentinel) | Гистограмма пропускает последние столбцы | Добавьте 0 в конец для очистки стека |
| Отсутствие разрешения конфликтов в куче | Сравнение несравнимых объектов | Добавьте индекс в кортеж кучи |
| Изменение списка во время итерации | Удаление узлов при обходе | Используйте паттерн prev/curr или фиктивную голову |
