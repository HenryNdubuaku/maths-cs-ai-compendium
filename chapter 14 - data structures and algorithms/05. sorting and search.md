## Алгоритмы сортировки

- Сортировка — самая изученная задача в информатике. Понимание этих алгоритмов развивает интуицию в отношении рекурсии, метода «разделяй и властвуй» и анализа сложности.

| Алгоритм | Лучший случай | Средний случай | Худший случай | Память | Устойчивый? |
|-----------|------|---------|-------|-------|---------|
| Пузырьковая сортировка | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | Да |
| Сортировка вставками | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | Да |
| Сортировка слиянием | $O(n \log n)$ | $O(n \log n)$ | $O(n \log n)$ | $O(n)$ | Да |
| Быстрая сортировка | $O(n \log n)$ | $O(n \log n)$ | $O(n^2)$ | $O(\log n)$ | Нет |
| Пирамидальная сортировка | $O(n \log n)$ | $O(n \log n)$ | $O(n \log n)$ | $O(1)$ | Нет |
| Сортировка подсчетом | $O(n + k)$ | $O(n + k)$ | $O(n + k)$ | $O(k)$ | Да |
| Поразрядная сортировка | $O(d(n + k))$ | $O(d(n + k))$ | $O(d(n + k))$ | $O(n + k)$ | Да |

- **Устойчивость** означает, что равные элементы сохраняют свой относительный порядок. Это важно при сортировке по нескольким ключам.

- **Нижняя граница** для сортировки на основе сравнений составляет $\Omega(n \log n)$. Доказательство использует деревья решений (глава 13): любая сортировка сравнениями должна различать все $n!$ перестановок, что требует как минимум $\log_2(n!)  = \Omega(n \log n)$ сравнений. Сортировка подсчетом и поразрядная сортировка обходят это ограничение, так как не сравнивают элементы между собой.

### Сортировка слиянием

- Разделите массив пополам, рекурсивно отсортируйте каждую половину, затем слейте отсортированные половины. Всегда $O(n \log n)$, $O(n)$ дополнительной памяти.

```python
def merge_sort(arr):
    if len(arr) <= 1:
        return arr

    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])

    return merge(left, right)

def merge(left, right):
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:  # <= for stability
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    result.extend(left[i:])
    result.extend(right[j:])
    return result
```

- **Подвох**: использование `<` вместо `<=` при слиянии нарушает устойчивость (равные элементы из правой половины окажутся перед элементами из левой).

### Быстрая сортировка

- Выберите **опорный элемент** (pivot), разделите элементы на «меньше опорного» и «больше опорного», рекурсивно отсортируйте каждую часть. $O(n \log n)$ в среднем, $O(n^2)$ в худшем случае (когда опорный элемент всегда является наименьшим или наибольшим).

```python
def quicksort(arr, lo=0, hi=None):
    if hi is None:
        hi = len(arr) - 1
    if lo >= hi:
        return

    pivot_idx = partition(arr, lo, hi)
    quicksort(arr, lo, pivot_idx - 1)
    quicksort(arr, pivot_idx + 1, hi)

def partition(arr, lo, hi):
    pivot = arr[hi]  # Lomuto: pivot is last element
    i = lo
    for j in range(lo, hi):
        if arr[j] < pivot:
            arr[i], arr[j] = arr[j], arr[i]
            i += 1
    arr[i], arr[hi] = arr[hi], arr[i]
    return i
```

- **Стратегии выбора опорного элемента**: последний элемент (просто, плохо для отсортированных входных данных), случайный (ожидаемое $O(n \log n)$), медиана из трех (практичный выбор). На собеседованиях всегда лучше выбирать случайный опорный элемент, чтобы избежать обсуждения худшего случая.

- **Подвох**: худший случай быстрой сортировки $O(n^2)$ возникает на уже отсортированных массивах при выборе первого или последнего элемента в качестве опорного. На практике случайный выбор или медиана из трех устраняют эту проблему.

### Сортировка подсчетом

- Если значения являются целыми числами в известном диапазоне $[0, k)$, подсчитайте вхождения и восстановите массив: время $O(n + k)$. Это не сортировка сравнениями, поэтому она может быть быстрее $O(n \log n)$.

```python
def counting_sort(arr, k):
    count = [0] * k
    for x in arr:
        count[x] += 1
    result = []
    for val in range(k):
        result.extend([val] * count[val])
    return result
```

- **Когда использовать**: диапазон $k$ ненамного больше $n$. Если $k = O(n)$, сложность составляет $O(n)$. Если $k \gg n$ (например, сортировка 10 чисел в диапазоне $[0, 10^9]$), сортировка подсчетом расходует память впустую.

## Шаблон: Бинарный поиск

- Бинарный поиск находит цель в отсортированном массиве за $O(\log n)$ путем многократного деления области поиска пополам. Но бинарный поиск — это гораздо больше, чем просто «найти число в отсортированном массиве». Общий шаблон таков: **поиск по монотонному условию**.

- **Шаблон** (тот, который позволяет избежать ошибок «на единицу»):

```python
def binary_search(arr, target):
    lo, hi = 0, len(arr) - 1

    while lo <= hi:
        mid = lo + (hi - lo) // 2  # avoids overflow in other languages
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            lo = mid + 1
        else:
            hi = mid - 1

    return -1  # not found
```

- **Нижняя граница** (первый элемент $\geq$ цели):

```python
def lower_bound(arr, target):
    lo, hi = 0, len(arr)
    while lo < hi:
        mid = (lo + hi) // 2
        if arr[mid] < target:
            lo = mid + 1
        else:
            hi = mid
    return lo
```

- **Подвох**: разница между `lo <= hi` и `lo < hi`, а также между `hi = mid` и `hi = mid - 1` определяет, найдете ли вы точное совпадение или границу. Нарисуйте схему для массива из 2 элементов, чтобы проверить себя.

### Легкий уровень: Бинарный поиск

- Стандартная задача. Используйте шаблон выше.

### Средний уровень: Поиск в повернутом отсортированном массиве

- **Задача**: отсортированный массив был повернут относительно некоторой точки. Найдите цель.

- **Шаблон**: на каждом шаге одна из половин всегда отсортирована. Определите, какая половина отсортирована, и проверьте, находится ли цель в этой половине.

```python
def search_rotated(nums, target):
    lo, hi = 0, len(nums) - 1

    while lo <= hi:
        mid = (lo + hi) // 2
        if nums[mid] == target:
            return mid

        # left half is sorted
        if nums[lo] <= nums[mid]:
            if nums[lo] <= target < nums[mid]:
                hi = mid - 1
            else:
                lo = mid + 1
        # right half is sorted
        else:
            if nums[mid] < target <= nums[hi]:
                lo = mid + 1
            else:
                hi = mid - 1

    return -1
```

- **Подводный камень**: использование `<=` в `nums[lo] <= nums[mid]` (а не `<`) критически важно. Когда `lo == mid` (осталось 2 элемента), мы должны правильно определить отсортированную половину.

### Сложно: Медиана двух отсортированных массивов

- **Задача**: найти медиану двух отсортированных массивов за $O(\log(m + n))$.

- **Паттерн**: бинарный поиск по точке разбиения меньшего массива. Разбиение делит оба массива так, что все элементы слева меньше всех элементов справа.

```python
def find_median(nums1, nums2):
    if len(nums1) > len(nums2):
        nums1, nums2 = nums2, nums1  # ensure nums1 is shorter

    m, n = len(nums1), len(nums2)
    lo, hi = 0, m
    half = (m + n + 1) // 2

    while lo <= hi:
        i = (lo + hi) // 2          # partition point in nums1
        j = half - i                 # partition point in nums2

        left1 = nums1[i - 1] if i > 0 else float('-inf')
        right1 = nums1[i] if i < m else float('inf')
        left2 = nums2[j - 1] if j > 0 else float('-inf')
        right2 = nums2[j] if j < n else float('inf')

        if left1 <= right2 and left2 <= right1:
            # correct partition
            if (m + n) % 2 == 1:
                return max(left1, left2)
            return (max(left1, left2) + min(right1, right2)) / 2
        elif left1 > right2:
            hi = i - 1
        else:
            lo = i + 1
```

- Это одна из самых сложных задач на бинарный поиск. Ключевая идея заключается в том, что вы ищете не значение, а **точку разбиения**, удовлетворяющую условию.

### Мета-паттерн: Бинарный поиск по ответу

- Многие задачи, которые не выглядят как бинарный поиск, могут быть решены с помощью бинарного поиска по ответу. Если ответ — это число, и вы можете написать функцию `is_feasible(x)`, которая монотонна (True для всех $x \geq$ оптимального или False для всех $x \geq$ оптимального), тогда используйте бинарный поиск по $x$.

- **Пример**: «какова минимальная грузоподъемность судна, чтобы доставить все посылки в течение $d$ дней?» Бинарный поиск по грузоподъемности. Для каждой потенциальной грузоподъемности жадно проверяйте, можно ли доставить все посылки за $d$ дней.

```python
def ship_within_days(weights, days):
    lo, hi = max(weights), sum(weights)

    while lo < hi:
        mid = (lo + hi) // 2
        # can we ship with capacity mid in <= days?
        current_load, num_days = 0, 1
        for w in weights:
            if current_load + w > mid:
                num_days += 1
                current_load = 0
            current_load += w

        if num_days <= days:
            hi = mid
        else:
            lo = mid + 1

    return lo
```

## Паттерн: Жадные алгоритмы

- **Жадный** алгоритм делает локально оптимальный выбор на каждом шаге в надежде, что это приведет к глобально оптимальному решению. Жадные алгоритмы работают, когда задача обладает **свойством жадного выбора** (локальный оптимум ведет к глобальному) и **оптимальной подструктурой** (оптимальное решение содержит оптимальные решения подзадач).

### Средне: Игра с прыжками (Jump Game)

- **Задача**: дан массив, где `nums[i]` — максимальная длина прыжка из позиции $i$; определите, можно ли достичь последнего индекса.

```python
def can_jump(nums):
    max_reach = 0
    for i, jump in enumerate(nums):
        if i > max_reach:
            return False  # cannot reach this position
        max_reach = max(max_reach, i + jump)
    return True
```

- **Почему это работает**: нам нужно знать только самую дальнюю достижимую позицию. Если текущая позиция находится дальше, чем самая дальняя достижимая, мы застряли. В противном случае мы обновляем самую дальнюю достижимую позицию.

### Средне: Объединение интервалов

- **Задача**: объединить пересекающиеся интервалы.

```python
def merge_intervals(intervals):
    intervals.sort(key=lambda x: x[0])
    merged = [intervals[0]]

    for start, end in intervals[1:]:
        if start <= merged[-1][1]:
            merged[-1][1] = max(merged[-1][1], end)
        else:
            merged.append([start, end])

    return merged
```

- **Паттерн**: сортировка по времени начала, затем жадное объединение. Если текущий интервал пересекается с последним объединенным, расширьте его. В противном случае начните новый объединенный интервал.

- **Подводный камень**: использование `merged[-1][1] = end` вместо `merged[-1][1] = max(merged[-1][1], end)`. Интервал может быть полностью включен в другой (например, [1, 10] и [2, 5]).

## Паттерн: Динамическое программирование

- **Динамическое программирование (DP)** решает задачи путем разбиения их на перекрывающиеся подзадачи, решения каждой из которых один раз и сохранения результатов. Оно работает, когда задача обладает **оптимальной подструктурой** и **перекрывающимися подзадачами**.

- **Два подхода**:
    - **Сверху вниз (мемоизация)**: напишите естественное рекурсивное решение, затем кешируйте результаты в словаре.
    - **Снизу вверх (табуляция)**: постройте таблицу от наименьших подзадач к большим.

- **Как распознать DP**: задача требует найти оптимум (минимум/максимум), количество или факт существования, и текущее решение зависит от предыдущих. Если вы рисуете дерево рекурсии и видите повторяющиеся подзадачи, это DP.

### Легко: Восхождение по лестнице

- **Задача**: $n$ ступеней, вы можете подниматься на 1 или 2 ступени за раз. Сколько существует различных способов?

- Это числа Фибоначчи: $f(n) = f(n-1) + f(n-2)$.

```python
def climb_stairs(n):
    if n <= 2:
        return n
    a, b = 1, 2
    for _ in range(3, n + 1):
        a, b = b, a + b
    return b
```

- Временная сложность $O(n)$, пространственная $O(1)$. Полная таблица мемоизации не нужна, так как каждое состояние зависит только от двух предыдущих.

### Средне: Размен монет

- **Задача**: даны номиналы монет и целевая сумма, найдите минимальное количество монет, необходимое для получения этой суммы.

- **Состояние**: `dp[amount]` = минимальное количество монет для получения `amount`.
- **Переход**: `dp[amount] = min(dp[amount - coin] + 1)` для каждой монеты.
- **Базовый случай**: `dp[0] = 0`.

```python
def coin_change(coins, amount):
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0

    for a in range(1, amount + 1):
        for coin in coins:
            if coin <= a and dp[a - coin] + 1 < dp[a]:
                dp[a] = dp[a - coin] + 1

    return dp[amount] if dp[amount] != float('inf') else -1
```

- **Подводный камень**: инициализация `float('inf')` (а не 0 или -1). Сравнение на минимум работает только в том случае, если недостижимые состояния равны бесконечности.

### Medium: Наибольшая общая подпоследовательность (Longest Common Subsequence)

- **Задача**: даны две строки, найти длину их наибольшей общей подпоследовательности.

- **Состояние**: `dp[i][j]` = НОП для `text1[:i]` и `text2[:j]`.
- **Переход**: если `text1[i-1] == text2[j-1]`, то `dp[i][j] = dp[i-1][j-1] + 1`. В противном случае `dp[i][j] = max(dp[i-1][j], dp[i][j-1])`.

```python
def longest_common_subsequence(text1, text2):
    m, n = len(text1), len(text2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if text1[i - 1] == text2[j - 1]:
                dp[i][j] = dp[i - 1][j - 1] + 1
            else:
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])

    return dp[m][n]
```

### Hard: Задача о рюкзаке 0/1 (0/1 Knapsack)

- **Задача**: даны предметы с весами и ценностями, а также вместимость $W$; максимизировать общую ценность, не превышая $W$.

- **Состояние**: `dp[i][w]` = максимальная ценность при использовании первых $i$ предметов с вместимостью $w$.
- **Переход**: `dp[i][w] = max(dp[i-1][w], dp[i-1][w - weight[i]] + value[i])` (пропустить или взять предмет $i$).

```python
def knapsack(weights, values, capacity):
    n = len(weights)
    dp = [[0] * (capacity + 1) for _ in range(n + 1)]

    for i in range(1, n + 1):
        for w in range(capacity + 1):
            dp[i][w] = dp[i - 1][w]  # skip item i
            if weights[i - 1] <= w:
                dp[i][w] = max(dp[i][w],
                               dp[i - 1][w - weights[i - 1]] + values[i - 1])

    return dp[n][capacity]
```

- **Оптимизация памяти**: поскольку каждая строка зависит только от предыдущей, используйте одномерный массив и итерируйтесь по $w$ справа налево:

```python
def knapsack_optimised(weights, values, capacity):
    dp = [0] * (capacity + 1)
    for i in range(len(weights)):
        for w in range(capacity, weights[i] - 1, -1):  # right to left!
            dp[w] = max(dp[w], dp[w - weights[i]] + values[i])
    return dp[capacity]
```

- **Подвох**: итерация слева направо в одномерной версии позволяет использовать предмет $i$ несколько раз (рюкзак с неограниченным количеством предметов). Итерация справа налево гарантирует, что каждый предмет используется не более одного раза.

## Паттерн: Поиск с возвратом (Backtracking)

- **Поиск с возвратом** — это исчерпывающий поиск с отсечением. Построение решения происходит инкрементально, и ветка отбрасывается (возврат), как только становится ясно, что частичное решение не может привести к валидному полному решению.

- **Шаблон**:

```python
def backtrack(candidates, path, result):
    if is_solution(path):
        result.append(path[:])  # copy!
        return

    for candidate in get_candidates(path):
        if is_valid(candidate, path):
            path.append(candidate)     # choose
            backtrack(candidates, path, result)  # explore
            path.pop()                 # unchoose (backtrack)
```

### Medium: Подмножества (Subsets)

```python
def subsets(nums):
    result = []
    def backtrack(start, path):
        result.append(path[:])
        for i in range(start, len(nums)):
            path.append(nums[i])
            backtrack(i + 1, path)
            path.pop()
    backtrack(0, [])
    return result
```

### Medium: Сумма комбинаций (Combination Sum)

- **Задача**: найти все уникальные комбинации, сумма которых равна целевому значению (элементы можно использовать повторно).

```python
def combination_sum(candidates, target):
    result = []
    def backtrack(start, path, remaining):
        if remaining == 0:
            result.append(path[:])
            return
        for i in range(start, len(candidates)):
            if candidates[i] > remaining:
                break  # prune: sorted, so all further candidates are too large
            path.append(candidates[i])
            backtrack(i, path, remaining - candidates[i])  # i, not i+1: reuse allowed
            path.pop()

    candidates.sort()  # sort for pruning
    backtrack(0, [], target)
    return result
```

- **Подвох**: `backtrack(i, ...)` позволяет повторно использовать тот же элемент. `backtrack(i + 1, ...)` переходит к следующему элементу (без повторного использования). Ошибки в этом месте — самая частая проблема при реализации поиска с возвратом.

### Hard: N ферзей (N-Queens)

- **Задача**: расставить $n$ ферзей на доске $n \times n$ так, чтобы ни один из них не атаковал другого.

```python
def solve_n_queens(n):
    result = []
    cols = set()
    pos_diag = set()  # (row + col) is constant on / diagonals
    neg_diag = set()  # (row - col) is constant on \ diagonals

    board = [['.' ] * n for _ in range(n)]

    def backtrack(row):
        if row == n:
            result.append([''.join(r) for r in board])
            return

        for col in range(n):
            if col in cols or (row + col) in pos_diag or (row - col) in neg_diag:
                continue

            cols.add(col)
            pos_diag.add(row + col)
            neg_diag.add(row - col)
            board[row][col] = 'Q'

            backtrack(row + 1)

            cols.remove(col)
            pos_diag.remove(row + col)
            neg_diag.remove(row - col)
            board[row][col] = '.'

    backtrack(0)
    return result
```

- **Key idea**: diagonal encoding. For `/` diagonals, the sum `row + col` is constant. For `\` diagonals, the difference `row - col` is constant. Using sets to track columns and diagonals makes validity checking $O(1)$.

## Summary of common mistakes

| Mistake | Example | Fix |
|---------|---------|-----|
| `lo <= hi` vs `lo < hi` in binary search | Off-by-one error in bounds | Choose based on whether the `hi` bound is inclusive |
| 1D knapsack left-to-right | Items are used multiple times | Iterate right-to-left for 0/1 knapsack |
| Not copying the path in backtracking | `result.append(path)` — all entries point to the same list | `result.append(path[:])` or `path.copy()` |
| `backtrack(i)` vs `backtrack(i+1)` | Reusing vs not reusing | Match the problem requirements |
| Missing `break` in sorted backtracking | Exploring too many candidates | Sort + `break` when the candidate exceeds the remainder |
| DP initialization | Incorrect `dp[0]` → all subsequent values are wrong | Define the base case carefully |
| Greedy algorithm without proof | Greedy approach doesn't always work | Verify the greedy choice property |
| Unstable sorting for multiple keys | Relative order of equal elements is lost | Use a stable sort (merge sort, `sorted` in Python) |
