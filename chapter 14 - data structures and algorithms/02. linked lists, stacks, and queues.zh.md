# 链表、栈与队列

*linked list、stack、queue 是更复杂数据结构的构件。本文件讲解它们的机制，然后通过难度递增的题目构建关键 pattern：快慢指针、单调栈、基于 heap 的优先队列，并指出每一步的常见陷阱。*

- 数组给你快速的随机访问，但插入昂贵。**链表** 给你快速的插入，但没有随机访问。**栈** 与 **队列** 把访问限制在一端或两端，正是这种限制让它们强大：通过限制你能做的事，简化了你需要思考的东西。

## 链表

- **单链表（singly linked list）** 是一串节点。每个节点存一个值和一个指向下一节点的指针。最后一个节点指向 `null`。

```python
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next
```

- **相对数组的优势**：在已知位置插入或删除是 $O(1)$（只需改指针），无需移动元素。

- **劣势**：访问第 $i$ 个元素需要 $O(i)$ 遍历（无随机访问）。缓存局部性差（节点散落在内存各处）。

- **双链表（doubly linked list）** 加一个 `prev` 指针，支持反向遍历。用于 LRU 缓存（任意节点 $O(1)$ 删除）和浏览器历史（前进/后退）。

| 操作 | 单链表 | 双链表 |
|-----------|--------|--------|
| 按索引访问 | $O(n)$ | $O(n)$ |
| 头部插入 | $O(1)$ | $O(1)$ |
| 尾部插入 | $O(n)$ 或 $O(1)$* | $O(1)$ |
| 给定节点删除 | $O(n)$** | $O(1)$ |
| 查找 | $O(n)$ | $O(n)$ |

*带尾指针。**需要前驱，得遍历。

- **哨兵节点（sentinel node）**（哑头/尾）能简化边界情况。没有哑头时，在头部插入或删除头节点需要特判。有了哑头，每个真实节点都有前驱。

```python
# Without dummy: special case for head deletion
def delete_head(head):
    if not head:
        return None
    return head.next

# With dummy: uniform logic
dummy = ListNode(0)
dummy.next = head
# now every deletion is: prev.next = prev.next.next
```

- **陷阱**：忘了处理空链表（`head is None`）或单元素链表。始终测试这些边界情况。

## Pattern：快慢指针（Floyd 算法）

- 用两个速度不同的指针检测链表性质。**慢** 指针每次走一步；**快** 指针每次走两步。

### 简单：Linked List Cycle

- **问题**：判断链表是否有环。

- **Pattern**：若有环，快指针最终会追上慢指针（相遇）。若无环，快指针会到达 `null`。

```python
def has_cycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            return True
    return False
```

- **为什么有效**：若环长 $c$，快指针每步拉近 1 个节点。慢指针进环后 $c$ 步内必相遇。

- **陷阱**：检查 `fast and fast.next`（不只是 `fast.next`）。若 `fast` 为 `None`，调用 `fast.next` 会崩。

### 中等：Find the Middle of a Linked List

- **问题**：返回中间节点。

- **Pattern**：当快指针到达末尾时，慢指针正好在中间。

```python
def find_middle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
    return slow  # slow is at the middle (or second middle if even length)
```

### 中等：Linked List Cycle II（找环入口）

- **问题**：返回环开始的节点。

- **Pattern**：快慢相遇后，把一个指针重置回头部。两者都按速度 1 前进，会在环入口相遇。

```python
def detect_cycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            # reset one pointer to head
            slow = head
            while slow != fast:
                slow = slow.next
                fast = fast.next
            return slow
    return None
```

- **为什么有效**：设头到环入口距离为 $a$，环入口到相遇点距离为 $b$。慢指针走了 $a + b$。快指针走了 $2(a + b)$。差值正好是一整圈：$a + b = c$（环长）。所以 $a = c - b$：从头到环入口的距离等于从相遇点沿环走到环入口的距离。

### 困难：Reverse Linked List in Groups of K

- **问题**：每 $k$ 个连续节点一组翻转链表。

```python
def reverse_k_group(head, k):
    # check if we have k nodes left
    node = head
    for _ in range(k):
        if not node:
            return head
        node = node.next

    # reverse k nodes
    prev, curr = None, head
    for _ in range(k):
        nxt = curr.next
        curr.next = prev
        prev = curr
        curr = nxt

    # head is now the tail of the reversed group
    # recursively process the rest
    head.next = reverse_k_group(curr, k)
    return prev  # prev is the new head of this group
```

- **陷阱**：原地翻转模式（`prev, curr, nxt`）值得背熟。画出来：每步把 `curr.next` 反指回 `prev`，再把三个指针都前进一位。顺序弄错会破坏链表。

## 栈

- **栈（stack）** 是 LIFO（后进先出）：最近加入的元素最先被移除。想象一摞盘子。

- 操作：`push(x)` 加到顶部、`pop()` 从顶部移除、`peek()` 查看顶部但不移除。全是 $O(1)$。

- 栈是 **递归**（调用栈）、**表达式求值**（中缀转后缀）、**撤销操作**（每个动作入栈，撤销就弹最后一个）背后的隐式结构。

### 简单：Valid Parentheses

- **问题**：给定一个由括号 `()[]{}` 组成的字符串，判断是否平衡。

- **Pattern**：把左括号压栈。遇到右括号时，检查栈顶是否是对应的左括号。

```python
def is_valid(s):
    stack = []
    matching = {')': '(', ']': '[', '}': '{'}

    for char in s:
        if char in matching:
            if not stack or stack[-1] != matching[char]:
                return False
            stack.pop()
        else:
            stack.append(char)

    return len(stack) == 0
```

- **陷阱**：结尾忘加 `len(stack) == 0`。字符串 "(((" 没有失配，但因有未闭合括号而不合法。

## Pattern：单调栈

- **单调栈（monotonic stack）** 按有序方式（递增或递减）维护元素。当新元素会破坏顺序时，就弹出元素直到恢复顺序。

- **何时用**：题目问 "对每个元素，找下一个/上一个更大/更小的元素"。栈让总复杂度变成 $O(n)$，因为每个元素最多入栈、出栈各一次。

### 中等：Daily Temperatures

- **问题**：给定每日温度，对每一天找出还要等几天才会更暖。

- **Pattern**：用索引栈。当当前温度高于栈顶时，弹栈并记录距离。

```python
def daily_temperatures(temperatures):
    n = len(temperatures)
    result = [0] * n
    stack = []  # stack of indices, temperatures in decreasing order

    for i in range(n):
        while stack and temperatures[i] > temperatures[stack[-1]]:
            prev = stack.pop()
            result[prev] = i - prev
        stack.append(i)

    return result
```

- 每个元素入栈一次、最多出栈一次：总 $O(n)$。

- **陷阱**：栈里存的是索引（不是值）。算距离需要索引。

### 困难：Largest Rectangle in Histogram

- **问题**：给定一组柱高，找出最大矩形面积。

- **Pattern**：对每根柱子，找它能向左、向右延伸多远（即两侧最近更矮的柱子）。单调递增栈能高效跟踪这一点。

```python
def largest_rectangle(heights):
    stack = []  # stack of indices, heights in increasing order
    max_area = 0
    heights.append(0)  # sentinel to flush the stack at the end

    for i, h in enumerate(heights):
        start = i
        while stack and stack[-1][1] > h:
            idx, height = stack.pop()
            max_area = max(max_area, height * (i - idx))
            start = idx  # this bar can extend back to where the popped bar started
        stack.append((start, h))

    heights.pop()  # remove sentinel
    return max_area
```

- **陷阱**：`start = idx` 这一行很微妙。当我们弹出一根比当前柱高的柱时，当前柱可以回退到那根被弹柱的起点（因为中间所有柱至少与被弹柱一样高）。漏掉这行会得到错误面积。

- **陷阱**：哨兵 `heights.append(0)` 保证栈中所有剩余柱都被处理。没有它，那些右侧从未遇到更矮柱的柱会被漏掉。

## 队列

- **队列（queue）** 是 FIFO（先进先出）：元素从尾部加入、从头部移除。想象商店里排队。

- **双端队列（deque，double-ended queue）** 支持两端 $O(1)$ 插入和删除。Python 的 `collections.deque` 是标准实现。

- 队列是 **BFS**（广度优先搜索，第 14 章文件 4）、**任务调度**、**消息传递** 背后的结构。

### 简单：Implement Queue Using Stacks

- **问题**：只用两个栈实现一个队列。

- **Pattern**：一个栈用于压入、一个用于弹出。当弹出栈为空时，把压入栈里所有元素搬过来（顺序反转）。

```python
class MyQueue:
    def __init__(self):
        self.push_stack = []
        self.pop_stack = []

    def push(self, x):
        self.push_stack.append(x)

    def pop(self):
        if not self.pop_stack:
            while self.push_stack:
                self.pop_stack.append(self.push_stack.pop())
        return self.pop_stack.pop()

    def peek(self):
        if not self.pop_stack:
            while self.push_stack:
                self.pop_stack.append(self.push_stack.pop())
        return self.pop_stack[-1]

    def empty(self):
        return not self.push_stack and not self.pop_stack
```

- 每次操作摊还 $O(1)$：每个元素在两栈之间最多搬一次。

## 优先队列与堆

- **优先队列（priority queue）** 不管插入顺序如何，都先返回最小（或最大）元素。标准实现是 **二叉堆（binary heap）**。

- **最小堆（min-heap）** 是一棵完全二叉树，每个父节点都小于其子节点。最小值总在根。用数组存：节点 $i$ 的子节点位于 $2i + 1$ 和 $2i + 2$。

| 操作 | 时间 |
|-----------|------|
| 插入 | $O(\log n)$ |
| 取最小 | $O(1)$ |
| 提取最小 | $O(\log n)$ |
| 由数组建堆 | $O(n)$ |

- Python 的 `heapq` 模块提供最小堆。要做最大堆，把值取负。

```python
import heapq

# Min-heap
h = []
heapq.heappush(h, 5)
heapq.heappush(h, 2)
heapq.heappush(h, 8)
print(heapq.heappop(h))  # 2 (smallest)

# Max-heap trick: negate values
heapq.heappush(h, -10)
print(-heapq.heappop(h))  # 10 (largest)
```

### 中等：Kth Largest Element in an Array

- **问题**：找第 k 大的元素。

- **Pattern**：维护一个大小为 $k$ 的最小堆。堆根就是第 k 大。若堆已有 $k$ 个元素且新元素比堆根大，就替换堆根。

```python
import heapq

def find_kth_largest(nums, k):
    heap = nums[:k]
    heapq.heapify(heap)  # O(k)

    for num in nums[k:]:
        if num > heap[0]:
            heapq.heapreplace(heap, num)  # pop min, push num: O(log k)

    return heap[0]
```

- $O(n \log k)$ 时间、$O(k)$ 空间。当 $k \ll n$ 时比排序（$O(n \log n)$）好得多。

- **陷阱**：用大小为 $n$ 的最大堆弹 $k$ 次也可行但更慢：$O(n + k \log n)$。大小为 $k$ 的最小堆才是最优做法。

### 困难：Merge K Sorted Lists

- **问题**：把 $k$ 个有序链表合并成一个有序链表。

- **Pattern**：用一个含每条链表头节点的最小堆。弹出最小的，加进结果，再把它的下一个节点压入堆。

```python
import heapq

def merge_k_lists(lists):
    heap = []
    for i, lst in enumerate(lists):
        if lst:
            heapq.heappush(heap, (lst.val, i, lst))

    dummy = ListNode(0)
    curr = dummy

    while heap:
        val, i, node = heapq.heappop(heap)
        curr.next = node
        curr = curr.next
        if node.next:
            heapq.heappush(heap, (node.next.val, i, node.next))

    return dummy.next
```

- $O(n \log k)$，$n$ 为总节点数。堆里最多 $k$ 个元素。

- **陷阱**：堆元组里的 `i`（索引）是决胜项。没有它，当值相等时 Python 会尝试比较 `ListNode` 对象，而 `ListNode` 不支持 `<`，会崩。索引保证比较合法。

## 常见陷阱汇总

| 陷阱 | 示例 | 修复 |
|---------|---------|-----|
| `fast.next` 上空指针 | 环检测用 `while fast.next` | 改成 `fast and fast.next` |
| 不处理空链表 | 对 `None` 翻转 | 加 `if not head` 守卫 |
| 栈下溢 | 从空栈弹出 | 检查 `len(stack) > 0` 或 `if stack` |
| 忘加哨兵 | 直方图漏掉最后几根柱 | 末尾 append 0 冲刷栈 |
| 堆里缺决胜项 | 比较不可比较的对象 | 给堆元组加索引 |
| 遍历时改链表 | 边遍历边删节点 | 用 prev/curr 模式或哑头 |
