# Деревья

*Деревья — это иерархическая структура данных, лежащая в основе файловых систем, баз данных, компиляторов и бесчисленного множества задач на собеседованиях. В этом разделе рассматриваются бинарные деревья, BST, сбалансированные деревья, префиксные деревья (tries), деревья отрезков, деревья Фенвика и система непересекающихся множеств (Union-Find), а также шаблоны обхода, рекурсивное мышление и задачи возрастающей сложности.*

- **Дерево** — это связный ациклический граф (глава 13). Наиболее важным вариантом является **бинарное дерево**: каждый узел имеет не более двух потомков (левого и правого). Деревья встречаются повсюду: деревья разбора в компиляторах, DOM-деревья в браузерах, деревья решений в машинном обучении и B-деревья в базах данных.

- Ключевое понимание для задач на деревья: **большинство задач на деревья решаются рекурсивно**. Структура сама по себе рекурсивна (дерево — это корень с двумя поддеревьями), поэтому и решения должны быть такими же. Освойте шаблон «решить для левого поддерева, решить для правого поддерева, объединить», и вы сможете решить большинство задач на деревья.

## Обходы бинарного дерева

- Существует четыре стандартных способа посетить каждый узел:

    - **Inorder** (левый, корень, правый): для BST посещает узлы в отсортированном порядке.
    - **Preorder** (корень, левый, правый): полезен для сериализации и копирования деревьев.
    - **Postorder** (левый, правый, корень): полезен для удаления и вычисления размеров.
    - **Level-order** (BFS): посещение узлов уровень за уровнем с использованием очереди.

```python
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

def inorder(root):
    if not root:
        return []
    return inorder(root.left) + [root.val] + inorder(root.right)

def preorder(root):
    if not root:
        return []
    return [root.val] + preorder(root.left) + preorder(root.right)

def postorder(root):
    if not root:
        return []
    return postorder(root.left) + postorder(root.right) + [root.val]

from collections import deque

def level_order(root):
    if not root:
        return []
    result, queue = [], deque([root])
    while queue:
        level = []
        for _ in range(len(queue)):
            node = queue.popleft()
            level.append(node.val)
            if node.left:
                queue.append(node.left)
            if node.right:
                queue.append(node.right)
        result.append(level)
    return result
```

- **Подвох**: рекурсивные обходы выше создают новые списки на каждом шаге (из-за конкатенации `+`), что дает сложность $O(n^2)$. Для эффективности передавайте список результатов и добавляйте элементы на месте:

```python
def inorder_efficient(root, result=None):
    if result is None:
        result = []
    if root:
        inorder_efficient(root.left, result)
        result.append(root.val)
        inorder_efficient(root.right, result)
    return result
```

### Легко: Максимальная глубина бинарного дерева

```python
def max_depth(root):
    if not root:
        return 0
    return 1 + max(max_depth(root.left), max_depth(root.right))
```

- **Рекурсивный шаблон**: базовый случай (null → 0), рекурсия для потомков, объединение (1 + max). Этот же шаблон применяется к десяткам задач на деревья.

### Легко: Инвертирование бинарного дерева

```python
def invert_tree(root):
    if not root:
        return None
    root.left, root.right = invert_tree(root.right), invert_tree(root.left)
    return root
```

### Средне: Наименьший общий предок

- **Задача**: найти наименьший узел, который является предком как для $p$, так и для $q$.

- **Шаблон**: если и $p$, и $q$ находятся в левом поддереве, то LCA находится в левом поддереве. Если оба в правом — в правом. Если они разделяются (один слева, другой справа), то текущий узел и есть LCA.

```python
def lowest_common_ancestor(root, p, q):
    if not root or root == p or root == q:
        return root

    left = lowest_common_ancestor(root.left, p, q)
    right = lowest_common_ancestor(root.right, p, q)

    if left and right:
        return root  # p and q are in different subtrees
    return left if left else right
```

- **Подвох**: это решение предполагает, что $p$ и $q$ существуют в дереве. Если это не гарантировано, потребуются дополнительные проверки.

### Сложно: Максимальная сумма пути в бинарном дереве

- **Задача**: найти путь с максимальной суммой между любыми двумя узлами (путь не обязательно должен проходить через корень).

```python
def max_path_sum(root):
    best = [float('-inf')]

    def dfs(node):
        if not node:
            return 0
        left = max(dfs(node.left), 0)   # ignore negative paths
        right = max(dfs(node.right), 0)

        # path through this node (possibly as the "bend")
        best[0] = max(best[0], node.val + left + right)

        # return the max gain this node can contribute to its parent
        return node.val + max(left, right)

    dfs(root)
    return best[0]
```

- **Ключевое понимание**: для каждого узла возникают два вопроса: (1) какой путь является лучшим из тех, что проходят *через* этот узел (левый + узел + правый)? (2) какой лучший путь этот узел может передать своему *родителю* (узел + max(левый, правый), так как путь не может разветвляться на двух уровнях)? Путаница между этими двумя вопросами — самая частая ошибка.

## Бинарные деревья поиска (BST)

- **BST** удовлетворяет условию: для каждого узла все значения в левом поддереве меньше, а все значения в правом поддереве больше. Это обеспечивает поиск, вставку и удаление за $O(\log n)$ (при условии сбалансированности).

```python
def search_bst(root, target):
    if not root:
        return None
    if target < root.val:
        return search_bst(root.left, target)
    elif target > root.val:
        return search_bst(root.right, target)
    else:
        return root

def insert_bst(root, val):
    if not root:
        return TreeNode(val)
    if val < root.val:
        root.left = insert_bst(root.left, val)
    else:
        root.right = insert_bst(root.right, val)
    return root
```

- **Подвох**: операции в BST имеют сложность $O(\log n)$ только тогда, когда дерево сбалансировано. BST, построенное из отсортированных данных, вырождается в связный список: $O(n)$ на операцию. Именно поэтому существуют сбалансированные BST (AVL, красно-черные деревья).

### Средне: Проверка бинарного дерева поиска

```python
def is_valid_bst(root, lo=float('-inf'), hi=float('inf')):
    if not root:
        return True
    if root.val <= lo or root.val >= hi:
        return False
    return (is_valid_bst(root.left, lo, root.val) and
            is_valid_bst(root.right, root.val, hi))
```

- **Подвох**: проверка только `left.val < root.val < right.val` является неверной. Ограничение заключается в том, что *все* узлы в левом поддереве меньше, а не только непосредственный потомок. Границы `lo`/`hi` распространяют это ограничение вниз.

### Средний уровень: K-й наименьший элемент в BST

- **Шаблон**: центрированный обход (inorder traversal) BST посещает узлы в отсортированном порядке. $k$-й посещенный узел и есть ответ.

```python
def kth_smallest(root, k):
    count = [0]
    result = [None]

    def inorder(node):
        if not node or result[0] is not None:
            return
        inorder(node.left)
        count[0] += 1
        if count[0] == k:
            result[0] = node.val
            return
        inorder(node.right)

    inorder(root)
    return result[0]
```

## Префиксные деревья (Tries)

- **Префиксное дерево (trie)** хранит строки посимвольно в дереве. Каждое ребро представляет символ, а пути от корня к отмеченным узлам представляют сохраненные строки. Префиксные деревья обеспечивают поиск за $O(L)$, где $L$ — длина строки, независимо от того, сколько строк сохранено.

```python
class TrieNode:
    def __init__(self):
        self.children = {}
        self.is_end = False

class Trie:
    def __init__(self):
        self.root = TrieNode()

    def insert(self, word):
        node = self.root
        for char in word:
            if char not in node.children:
                node.children[char] = TrieNode()
            node = node.children[char]
        node.is_end = True

    def search(self, word):
        node = self.root
        for char in word:
            if char not in node.children:
                return False
            node = node.children[char]
        return node.is_end

    def starts_with(self, prefix):
        node = self.root
        for char in prefix:
            if char not in node.children:
                return False
            node = node.children[char]
        return True
```

- **Когда использовать**: автодополнение, проверка орфографии, игры со словами, таблицы IP-маршрутизации. Везде, где нужны операции, основанные на префиксах.

### Сложный уровень: Поиск слов II

- **Задача**: дана доска с символами и список слов, найти все слова, которые можно составить, перемещаясь по соседним ячейкам.

- **Шаблон**: построить префиксное дерево из списка слов, затем выполнить DFS из каждой ячейки, используя дерево для раннего отсечения ветвей (если ни одно слово не начинается с текущего префикса, остановиться).

- **Подвох**: без префиксного дерева пришлось бы выполнять DFS для каждого слова отдельно: $O(w \cdot m \cdot n \cdot 4^L)$. Префиксное дерево позволяет совместно использовать вычисления префиксов для разных слов, значительно сокращая объем работы.

## Система непересекающихся множеств (Union-Find)

- **Union-Find** отслеживает коллекцию непересекающихся множеств. Две операции: `find(x)` возвращает представителя множества, содержащего $x$, а `union(x, y)` объединяет множества, содержащие $x$ и $y$.

```python
class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n
        self.count = n  # number of connected components

    def find(self, x):
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])  # path compression
        return self.parent[x]

    def union(self, x, y):
        rx, ry = self.find(x), self.find(y)
        if rx == ry:
            return False  # already connected
        # union by rank
        if self.rank[rx] < self.rank[ry]:
            rx, ry = ry, rx
        self.parent[ry] = rx
        if self.rank[rx] == self.rank[ry]:
            self.rank[rx] += 1
        self.count -= 1
        return True
```

- При использовании сжатия путей и объединения по рангу обе операции выполняются за $O(\alpha(n)) \approx O(1)$ в амортизированном смысле (обратная функция Аккермана, фактически константа).

- **Когда использовать**: связные компоненты, поиск циклов в неориентированных графах, алгоритм Краскала для поиска MST, группировка эквивалентных элементов.

### Средний уровень: Количество связных компонент

```python
def count_components(n, edges):
    uf = UnionFind(n)
    for u, v in edges:
        uf.union(u, v)
    return uf.count
```

### Средний уровень: Избыточное соединение

- **Задача**: найти ребро, удаление которого превращает граф в дерево (т.е. ребро, создающее цикл).

- **Шаблон**: обрабатывать ребра по одному. Первое ребро, оба конца которого уже находятся в одной компоненте, создает цикл.

```python
def find_redundant(edges):
    uf = UnionFind(len(edges) + 1)
    for u, v in edges:
        if not uf.union(u, v):
            return [u, v]  # already connected → this edge creates a cycle
```

## Деревья отрезков и деревья Фенвика

- **Деревья отрезков** отвечают на запросы на диапазонах (сумма, минимум, максимум на подмассиве) и поддерживают точечные обновления, и то и другое за $O(\log n)$.

- **Деревья Фенвика** (двоично-индексированные деревья) — это более простая и быстрая альтернатива для запросов префиксных сумм и точечных обновлений. Они используют хитрый трюк с битовыми операциями: каждая позиция хранит частичную сумму, покрывающую диапазон, определяемый младшим установленным битом.

```python
class FenwickTree:
    def __init__(self, n):
        self.n = n
        self.tree = [0] * (n + 1)

    def update(self, i, delta):
        i += 1  # 1-indexed
        while i <= self.n:
            self.tree[i] += delta
            i += i & (-i)  # add lowest set bit

    def prefix_sum(self, i):
        i += 1
        total = 0
        while i > 0:
            total += self.tree[i]
            i -= i & (-i)  # remove lowest set bit
        return total

    def range_sum(self, l, r):
        return self.prefix_sum(r) - (self.prefix_sum(l - 1) if l > 0 else 0)
```

- **Когда использовать**: задачи, требующие повторяющихся запросов на диапазонах с обновлениями. Деревья Фенвика предпочтительнее, если нужны только префиксные суммы; деревья отрезков — если нужны произвольные операции на диапазонах (минимум, максимум, НОД).

## Сводка типичных ошибок

| Подвох | Пример | Исправление |
|---------|---------|-----|
| Проверка только прямых потомков для BST | `left.val < root.val` пропускает нарушения глубже | Передавать границы `lo`/`hi` |
| Конкатенация списков $O(n^2)$ в рекурсии | `inorder(left) + [val] + inorder(right)` | Добавлять в общий список |
| Забытый базовый случай | Бесконечная рекурсия на пустом дереве | `if not root: return` |
| Путаница между путем через узел и путем к родителю | Максимальная сумма пути: разветвление на двух уровнях | Возвращать одну ветвь родителю, отслеживать две ветви отдельно |
| 1-индексация против 0-индексации в Фенвике | Ошибка на единицу в массиве дерева | Всегда `i += 1` при входе |
| Union-Find без сжатия путей | $O(n)$ на операцию find в худшем случае | `self.parent[x] = self.find(self.parent[x])` |
