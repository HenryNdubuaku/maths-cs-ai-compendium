# 数组与哈希

*array 与 hash table 是编程中最基础的两类数据结构。本文件讲解它们的底层工作原理，然后通过难度递增的题目构建关键解题 pattern：双指针、滑动窗口、前缀和、以及基于 hash 的查找，并指出每一步的常见陷阱。*

- 如果你深刻理解 array 和 hash map，就能解决约 40% 的编程面试题。这两类结构无处不在，因为它们提供了算法最需要的两样东西：**快速按索引访问**（array）和 **按键快速查找**（hash map）。

- 本文件教的是 pattern，而非解法。目标是当你看到新题时，能识别出哪个 pattern 适用、为什么适用，而不是试图回忆某个背过的解法。

## 数组

- **array（数组）** 是一块连续内存，元素以固定偏移量存储。访问第 $i$ 个元素的代价是 $O(1)$，因为地址就是 `base + i * element_size`。这是可能最快的数据访问方式，也是 array 作为默认选择的原因。

- **动态数组（dynamic array）**（Python 的 `list`、Java 的 `ArrayList`、C++ 的 `vector`）在满了时自动扩容。策略是 **摊还倍增**：满了就分配双倍大小的新数组，把所有元素拷过去。拷贝开销是 $O(n)$，但很罕见（每 $n$ 次插入才发生一次），所以每次插入的摊还开销是 $O(1)$。

- **缓存局部性（cache locality）** 是 array 在实践中（而非仅在理论上）快的原因。由于元素连续存储，访问一个元素会把附近的元素一起加载进 CPU 缓存（第 13 章）。遍历 array 对缓存友好；跟随链表指针则不然。这一常数因子差异在实践中可达 10–100 倍。

| 操作 | 数组 | 动态数组 |
|-----------|-------|---------------|
| 按索引访问 | $O(1)$ | $O(1)$ |
| Append | n/a | $O(1)$ 摊还 |
| 在位置 $i$ 插入 | $O(n)$ | $O(n)$ |
| 在位置 $i$ 删除 | $O(n)$ | $O(n)$ |
| 查找（未排序） | $O(n)$ | $O(n)$ |

- **陷阱**：在数组中部插入或删除是 $O(n)$，因为后面所有元素都要移动。若频繁需要在中间插入，考虑用 linked list 或换一种思路。

## 字符串

- **字符串** 是字符数组。在 Python 中字符串不可变：每次拼接都创建一个新字符串。在循环里逐字符构建字符串是 $O(n^2)$，因为每次拼接都要拷贝到目前为止整段字符串。

```python
# BAD: O(n^2) string concatenation
s = ""
for c in characters:
    s += c  # copies entire string each time

# GOOD: O(n) using a list then join
parts = []
for c in characters:
    parts.append(c)
s = "".join(parts)
```

- **陷阱**：在 Python 中，循环里写 `s += c` 是最常见的性能 bug 之一。始终先收集进 list 再 `.join()`。

- **编码**：ASCII 用 7 位（128 个字符）。**UTF-8** 是变长的：ASCII 字符用 1 字节，带重音字符用 2 字节，中文/日文字符用 3 字节，emoji 用 4 字节。当题目说 "小写英文字母" 时，字母表大小是 26，意味着可以用定长数组代替 hash map。

## 哈希表

- **hash table（哈希表）** 通过 **哈希函数** $h(key)$ 把键映射为值，平均情况下查找、插入、删除均为 $O(1)$。哈希函数把键转换为一个数组索引。

- 哈希函数必须：**确定性**（同一键总产生同一哈希）、**均匀**（把键均匀分布到各桶）、**计算快**。

- **冲突** 在两个不同键哈希到同一索引时发生。两种主要策略：

    - **链地址（chaining）**：每个桶存一个键值对链表。冲突时追加到链表。最坏情况（所有键都哈希到同一桶）：$O(n)$。哈希函数良好时平均：$O(1)$。

    - **开放寻址（open addressing）**：冲突时探查下一个空槽。**线性探查（linear probing）** 查下一个槽、再下一个……，对缓存友好但会受 **聚集（clustering）**（一长串占用槽）困扰。**Robin Hood hashing** 通过把 "离家更近" 的条目换出来减小方差。

- **负载因子** $\alpha = n / m$（元素数 / 桶数）决定性能。当 $\alpha$ 超过阈值（通常 0.75），表会 **rehash**：分配更大的表并重新插入所有元素。开销 $O(n)$，但不常发生。

- **Hash map**（Python 的 `dict`、Java 的 `HashMap`）存键值对。**Hash set**（Python 的 `set`、Java 的 `HashSet`）只存键（用于快速成员判定）。

| 操作 | 平均 | 最坏 |
|-----------|---------|------------|
| 查找 | $O(1)$ | $O(n)$ |
| 插入 | $O(1)$ | $O(n)$ |
| 删除 | $O(1)$ | $O(n)$ |

- **Bloom filter** 是空间高效的概率集合。它能告诉你 "肯定不在集合里" 或 "可能在集合里"（有可调的误报率）。它使用 $k$ 个哈希函数和一个位数组。用于数据库（避免对不存在的键读盘）、web 缓存、拼写检查器。

- **何时用 hash map**：每当需要 $O(1)$ 回答 "我见过这个吗？" 或 "这个键对应的计数/索引/值是什么？"。若你在反复线性扫描找东西，hash map 几乎肯定能让它更快。

## Pattern：Hash Map 查找

- 最基础的 pattern：用 hash map 把 $O(n)$ 扫描替换成 $O(1)$ 查找。

### 简单：Two Sum

- **问题**：给定一个整数数组和目标值，返回相加等于目标值的两个数的索引。

- **暴力** $O(n^2)$：检查每一对。

- **Pattern 直觉**：对每个数 `num`，需要 `target - num` 出现在数组某处。与其扫描数组，不如把已见过的数存进 hash map。

```python
def two_sum(nums, target):
    seen = {}  # value -> index
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i
```

- **为什么有效**：一趟遍历数组。对每个元素，hash map 查找是 $O(1)$。总计：$O(n)$ 时间、$O(n)$ 空间。

- **陷阱**：不要在检查补数之前就把当前数加进 hash map，否则可能把一个元素与自己匹配。上面代码的顺序正确：先查再加。

### 中等：Group Anagrams

- **问题**：给定一组字符串，把 anagram 分组。（"eat"、"tea"、"ate" 是一组。）

- **Pattern 直觉**：anagram 是相同字符的不同排列。若把每个字符串排序，anagram 会产生相同的排序键。用这个排序键作为 hash map 的键。

```python
from collections import defaultdict

def group_anagrams(strs):
    groups = defaultdict(list)
    for s in strs:
        key = tuple(sorted(s))  # or use character count tuple
        groups[key].append(s)
    return list(groups.values())
```

- **优化**：对每个字符串排序开销是 $O(k \log k)$，$k$ 为字符串长度。想得到更快的键，可以统计字符频次，用频次元组作键：

```python
def group_anagrams_fast(strs):
    groups = defaultdict(list)
    for s in strs:
        count = [0] * 26
        for c in s:
            count[ord(c) - ord('a')] += 1
        groups[tuple(count)].append(s)
    return list(groups.values())
```

- 这对每个字符串是 $O(k)$，而非 $O(k \log k)$。字符计数元组是一种 **规范型（canonical form）**：对同组成员都相同的表示。

- **陷阱**：在 Python 中，list 不可哈希（不能作 dict 键），必须转成 tuple。这在 `groups[count].append(s)` 时会绊倒人。

### 困难：Longest Consecutive Sequence

- **问题**：给定一个未排序数组，找出最长连续序列的长度（例如 [100, 4, 200, 1, 3, 2] → 4，因为 [1, 2, 3, 4]）。

- **暴力** $O(n \log n)$：排序数组，再扫描连续段。

- **Pattern 直觉**：把所有数放进 hash set 以 $O(1)$ 查找。对每个数，检查它是否是某段序列的 **起点**（即 `num - 1` 不在集合中）。若是，数数序列能延伸多远。

```python
def longest_consecutive(nums):
    num_set = set(nums)
    best = 0

    for num in num_set:
        # only start counting from the beginning of a sequence
        if num - 1 not in num_set:
            length = 1
            while num + length in num_set:
                length += 1
            best = max(best, length)

    return best
```

- **为什么是 $O(n)$**：内层 `while` 在所有迭代中总共最多跑 $n$ 次（每个数最多被访问两次：外层一次、`while` 扩展时一次）。`if num - 1 not in num_set` 这个守卫保证只在序列起点开始计数。

- **陷阱**：没有 `if num - 1 not in num_set` 检查，你会从每个元素都开始数，最坏是 $O(n^2)$（例如 [1, 2, 3, ..., n] 会从每个起点扫整段序列）。

## Pattern：双指针

- **双指针（two pointers）** pattern 用两个索引在数组中移动，通常从两端相向而行，或从同一端按不同速度前进。它在数组已排序或需要比较数对时有效。

- **何时用**：问题涉及数对、子数组或划分，且数组已排序（或排序不会丢失所需信息）。

### 简单：Valid Palindrome

- **问题**：判断一个字符串是否是回文，只考虑字母数字字符并忽略大小写。

- **Pattern**：一个指针在头、一个在尾，向中间靠拢比较字符。

```python
def is_palindrome(s):
    left, right = 0, len(s) - 1

    while left < right:
        # skip non-alphanumeric characters
        while left < right and not s[left].isalnum():
            left += 1
        while left < right and not s[right].isalnum():
            right -= 1

        if s[left].lower() != s[right].lower():
            return False

        left += 1
        right -= 1

    return True
```

- **陷阱**：内层 while 循环里忘加 `left < right` 检查。没有它，在 "!!!"（全是非字母数字）这样的字符串上指针会越界。

### 中等：Three Sum

- **问题**：找出数组中所有和为零的唯一三元组。

- **Pattern**：排序数组。固定一个元素，再在剩余部分用双指针找和等于该固定元素相反数的数对。

```python
def three_sum(nums):
    nums.sort()
    result = []

    for i in range(len(nums) - 2):
        # skip duplicate fixed elements
        if i > 0 and nums[i] == nums[i - 1]:
            continue

        left, right = i + 1, len(nums) - 1
        target = -nums[i]

        while left < right:
            total = nums[left] + nums[right]
            if total < target:
                left += 1
            elif total > target:
                right -= 1
            else:
                result.append([nums[i], nums[left], nums[right]])
                # skip duplicates
                while left < right and nums[left] == nums[left + 1]:
                    left += 1
                while left < right and nums[right] == nums[right - 1]:
                    right -= 1
                left += 1
                right -= 1

    return result
```

- **为什么有效**：排序 $O(n \log n)$。对每个固定元素，双指针扫描 $O(n)$。总计 $O(n^2)$，对该问题已是最优（必须考虑所有数对）。

- **陷阱**：处理重复是最难的部分。若没有去重逻辑（固定元素和双指针结果都要去重），会返回重复三元组。`if i > 0 and nums[i] == nums[i-1]: continue` 这一行至关重要。

### 困难：Trapping Rain Water

- **问题**：给定一个海拔图（非负整数数组），计算下雨后能接多少水。

- **Pattern 直觉**：对每个位置，水位由左侧最大高度与右侧最大高度的较小值减去当前高度决定。两端双指针分别维护左右运行最大值。

```python
def trap(height):
    left, right = 0, len(height) - 1
    left_max, right_max = 0, 0
    water = 0

    while left < right:
        if height[left] < height[right]:
            if height[left] >= left_max:
                left_max = height[left]
            else:
                water += left_max - height[left]
            left += 1
        else:
            if height[right] >= right_max:
                right_max = height[right]
            else:
                water += right_max - height[right]
            right -= 1

    return water
```

- **为什么有效**：关键直觉是若 `height[left] < height[right]`，位置 `left` 处的水位由 `left_max` 决定（我们知道右侧一定有更高的条，所以右侧不会成为瓶颈）。处理较短的一侧，保证另一侧有更高的条。

- **陷阱**：很多人先预计算 `left_max[i]` 和 `right_max[i]` 数组（可行但用 $O(n)$ 空间）。双指针做法达到 $O(1)$ 空间。另外，在更新最大值时混淆 `>=` 与 `>` 会导致水量差一。

## Pattern：滑动窗口

- **滑动窗口（sliding window）** pattern 维护一个窗口（连续子数组），随迭代扩展和收缩。适用于关于满足某条件的子数组或子串的问题。

- **何时用**：问题求满足某约束的最长/最短子数组或子串，且窗口扩展/收缩具有单调性（加元素只会让约束更难/更易满足，而非两者皆有）。

- **模板**：

```python
def sliding_window(arr):
    left = 0
    state = ...  # window state (counts, sum, etc.)
    best = ...

    for right in range(len(arr)):
        # expand: add arr[right] to the window state
        update_state(state, arr[right])

        # contract: shrink from the left while constraint is violated
        while constraint_violated(state):
            remove_from_state(state, arr[left])
            left += 1

        # update answer
        best = max(best, right - left + 1)  # or min, depending on problem

    return best
```

### 简单：Best Time to Buy and Sell Stock

- **问题**：给定每日价格，找一次买入一次卖出（先买后卖）的最大利润。

- **Pattern**：跟踪到目前为止的最低价（窗口左边界），并在每天计算利润。

```python
def max_profit(prices):
    min_price = float('inf')
    max_profit = 0

    for price in prices:
        min_price = min(min_price, price)
        max_profit = max(max_profit, price - min_price)

    return max_profit
```

- 这是一种退化版的滑动窗口：左指针（最低价）只在出现更小值时才前移。$O(n)$ 时间、$O(1)$ 空间。

### 中等：Longest Substring Without Repeating Characters

- **问题**：找出不含任何重复字符的最长子串长度。

- **Pattern**：移动 `right` 扩展窗口。出现重复时，从左侧收缩直到重复消除。

```python
def length_of_longest_substring(s):
    char_index = {}  # character -> its most recent index
    left = 0
    best = 0

    for right, char in enumerate(s):
        if char in char_index and char_index[char] >= left:
            left = char_index[char] + 1  # jump past the duplicate

        char_index[char] = right
        best = max(best, right - left + 1)

    return best
```

- **为什么 `char_index[char] >= left`**：该字符可能是在当前窗口开始之前就在映射里。没有这个检查，会对实际上不在当前窗口里的字符错误收缩。

- **陷阱**：用 set 从左一个个移除字符可行但较慢。hash map 做法直接跳到正确位置。

### 困难：Minimum Window Substring

- **问题**：给定字符串 `s` 和 `t`，在 `s` 中找出包含 `t` 所有字符的最小窗口。

- **Pattern**：扩展窗口直至包含所有所需字符，再从左侧收缩以找最小合法窗口。

```python
from collections import Counter

def min_window(s, t):
    if not t or not s:
        return ""

    need = Counter(t)       # characters we need and their counts
    have = 0                # how many unique characters we have in sufficient quantity
    required = len(need)    # how many unique characters we need

    left = 0
    best = (float('inf'), 0, 0)  # (length, left, right)

    window_counts = {}

    for right in range(len(s)):
        char = s[right]
        window_counts[char] = window_counts.get(char, 0) + 1

        # check if this character's count now meets the requirement
        if char in need and window_counts[char] == need[char]:
            have += 1

        # contract from the left while the window is valid
        while have == required:
            # update best
            if (right - left + 1) < best[0]:
                best = (right - left + 1, left, right)

            # remove leftmost character
            left_char = s[left]
            window_counts[left_char] -= 1
            if left_char in need and window_counts[left_char] < need[left_char]:
                have -= 1
            left += 1

    length, start, end = best
    return s[start:end + 1] if length != float('inf') else ""
```

- **陷阱**：`have` 计数器是关键优化。没有它，每步都要把 `window_counts` 与 `need` 整体比较，开销为每步 $O(|\text{unique chars}|)$。`have` 让合法性检查变成 $O(1)$。

- **陷阱**：检查 `window_counts[char] == need[char]`（而非 `>=`）保证我们对每个字符只把 `have` 加一次。若用 `>=`，就会多算。

## Pattern：前缀和

- **前缀和（prefix sum）** 数组存累加和：`prefix[i] = sum(arr[0:i])`。一旦在 $O(n)$ 内建好，任意子数组和都可在 $O(1)$ 内算出：`sum(arr[l:r]) = prefix[r] - prefix[l]`。

```python
def build_prefix(arr):
    prefix = [0] * (len(arr) + 1)
    for i in range(len(arr)):
        prefix[i + 1] = prefix[i] + arr[i]
    return prefix

# sum of arr[l:r] (inclusive l, exclusive r)
def range_sum(prefix, l, r):
    return prefix[r] - prefix[l]
```

- **何时用**：问题涉及多次子数组求和查询，或要找具有特定和的子数组。

### 简单：Range Sum Query

- **问题**：给定数组，多次回答 "从索引 $l$ 到 $r$ 的和是多少？"

- 不用前缀和：每次查询 $O(n)$。用前缀和：$O(n)$ 预处理，每次查询 $O(1)$。

### 中等：Subarray Sum Equals K

- **问题**：数出和为 $k$ 的连续子数组个数。

- **Pattern 直觉**：从索引 $l$ 到 $r$ 的子数组和等于 `prefix[r+1] - prefix[l]`。我们希望它等于 $k$，所以 `prefix[l] = prefix[r+1] - k`。对每个位置，用 hash map 数有多少更早的前缀和等于 `current_prefix - k`。

```python
def subarray_sum(nums, k):
    count = 0
    prefix = 0
    prefix_counts = {0: 1}  # empty prefix sum

    for num in nums:
        prefix += num
        # how many earlier prefix sums equal prefix - k?
        count += prefix_counts.get(prefix - k, 0)
        prefix_counts[prefix] = prefix_counts.get(prefix, 0) + 1

    return count
```

- 这把前缀和与 hash map 查找结合：$O(n)$ 时间、$O(n)$ 空间。

- **陷阱**：忘了初始化 `prefix_counts = {0: 1}`。空前缀（在任何元素之前）和为 0。少了它，会漏掉从索引 0 开始的子数组。

### 困难：Product of Array Except Self

- **问题**：给定数组，返回一个新数组，其中每个元素是其余所有元素的乘积。不能用除法。

- **Pattern**：从左侧建前缀积、从右侧建后缀积。每个位置的答案是 `left_product * right_product`。

```python
def product_except_self(nums):
    n = len(nums)
    result = [1] * n

    # left pass: result[i] = product of nums[0..i-1]
    prefix = 1
    for i in range(n):
        result[i] = prefix
        prefix *= nums[i]

    # right pass: multiply by product of nums[i+1..n-1]
    suffix = 1
    for i in range(n - 1, -1, -1):
        result[i] *= suffix
        suffix *= nums[i]

    return result
```

- $O(n)$ 时间、$O(1)$ 额外空间（输出数组不计）。这里用输出数组本身存中间前缀积，再在第二趟里乘入后缀积。

- **陷阱**：若数组含零，基于除法的方案会失败。这种前缀/后缀做法因为从不做除法，能正确处理零。

## 常见陷阱汇总

| 陷阱 | 示例 | 修复 |
|---------|---------|-----|
| 窗口大小差一 | `right - left` vs `right - left + 1` | 画一个 2 元素示例 |
| Python 可变默认参数 | `def f(seen={})` 跨调用共享状态 | 用 `def f(seen=None)` |
| 循环内字符串拼接 | `s += c` 在 Python 中是 $O(n^2)$ | 用 `list.append` + `"".join` |
| 前缀和忘加 `{0: 1}` | 漏掉从索引 0 开始的子数组 | 始终用空前缀初始化 |
| 检查前就加进 hash map | Two Sum：先加 `num` 再查补数 | 先查再加 |
| 不处理重复 | Three Sum 返回重复三元组 | 跳过连续相等的值 |
| 整数溢出 | C++/Java 中大数组求和 | 用 `long` 或检查边界 |
