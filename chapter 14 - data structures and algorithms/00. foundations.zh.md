*在深入数据结构与算法之前，你需要先掌握四个基础概念：用于衡量效率的 Big O 表示法、用于将问题分解为子问题的递归、用于带剪枝的穷举搜索的回溯，以及用于避免重复计算的动态规划。本文件将从基本原理出发逐一讲解。*

- 本章其余文件都假定你已经熟悉这四个概念。如果你跳过本文件，后续文件中的 $O(n \log n)$ 标注、递归式的 tree 遍历、回溯模板以及 DP 状态转移会让你觉得像魔法，而非工程。

## 为什么学模式，而非死记硬背

- LeetCode、NeetCode、HackerRank 上有成千上万道编程题。没人能把它们全记住，试图这样做本身就是失败的策略。面试官不会从一份固定清单里挑题，他们会改造、组合、伪装题目。当面试官问你一个你从未见过的变种时，你背下的 "Two Sum" 解法帮不上忙。

- 好消息是：核心 **pattern（模式）** 大约只有 **15–20 个**（双指针、滑动窗口、BFS/DFS、DP、回溯等）。每道题，无论表面看上去多新颖，都能归约为某一个或几个 pattern 的组合。面试不是在考你是否见过这道原题，而是在考你能否 **剥开上下文**——故事、特定数据类型、边界情况——识别出底层 pattern。

- 看下面三道题：
    - "在一个 array 中找出两个数，使其和等于目标值。"
    - "找出两个分子，使它们的结合能之和达到阈值。"
    - "给定一组账户余额，找出两个账户，使二者合计恰好等于某笔债务。"

- 它们看似不同，其实是同一道题：**Two Sum**。上下文（数字、分子、账户）无关紧要。结构是：在一个集合中搜索补数 → hash map 查找。

- 这就是本章 **从直觉出发教 pattern**、而非靠重复刷题教解法的原因。对每个 pattern，我们会讲：
    - 问题的 **什么结构特征** 暗示该 pattern（已排序输入 → 双指针；子数组约束 → 滑动窗口；最优子结构 + 重叠子问题 → DP）。
    - **为什么这个 pattern 有效**——背后的数学或逻辑推理，而不只是 "它能给出正确答案"。
    - **如何改造它**——通过简单、中等、困难三种变体，展示同一核心思路在不同上下文中的应用。

- 当你深刻理解了 *为什么* 滑动窗口有效（约束的单调性意味着扩展/收缩就足够了），你就能把它套用到任何具有这种结构的题目上，哪怕是没见过的题。而如果你只是背下 "Longest Substring Without Repeating Characters" 的代码，题目一变你就卡住了。

- 实操策略：
    1. **学习 pattern**（本章）。
    2. **练习识别** 伪装过的题目（每节末尾的 NeetCode 课后题）。
    3. **在时间压力下练习实现**。
    4. 面试中：读题 → 剥开上下文 → 识别 pattern → 实现。

## Big O 表示法

- 当我们说一个算法 "快" 或 "慢" 时，需要一种精确的度量方式。**Big O 表示法** 描述算法的运行时间（或空间占用）如何随输入规模 $n$ 增长，忽略常数因子和低阶项。

- 形式化定义：$f(n) = O(g(n))$ 表示存在常数 $c > 0$ 和 $n_0$，使得对所有 $n \geq n_0$ 都有 $f(n) \leq c \cdot g(n)$。用大白话说：对足够大的输入，$f$ 增长得不比 $g$ 快。

- 为什么要忽略常数？因为 $2n$ 算法和 $5n$ 算法都是 $O(n)$：它们的增长方式相同。在更快的计算机上，常数会变，但增长规律不变。Big O 捕捉的是问题的 **内在** 难度，与硬件无关。

### 增长率层级

- 从快到慢：

| Big O | 名称 | 示例 | $n = 10^6$ 次操作 |
|-------|------|---------|----------------------|
| $O(1)$ | 常数 | 数组访问、hash 查找 | 1 |
| $O(\log n)$ | 对数 | 二分查找 | 20 |
| $O(n)$ | 线性 | 线性扫描、单层循环 | $10^6$ |
| $O(n \log n)$ | 线性对数 | 归并排序、高效排序 | $2 \times 10^7$ |
| $O(n^2)$ | 平方 | 嵌套循环、暴力枚举对 | $10^{12}$（太慢） |
| $O(n^3)$ | 立方 | 三层嵌套循环、矩阵乘法 | $10^{18}$（慢得离谱） |
| $O(2^n)$ | 指数 | 所有子集、暴力回溯 | $10^{301030}$（不可能） |
| $O(n!)$ | 阶乘 | 所有排列 | 荒谬 |

- **经验法则**：现代计算机每秒大约执行 $10^8$–$10^9$ 次简单操作。对于 1 秒时限：
    - $O(n)$ 适用于 $n \leq 10^8$
    - $O(n \log n)$ 适用于 $n \leq 10^7$
    - $O(n^2)$ 适用于 $n \leq 10^4$
    - $O(2^n)$ 适用于 $n \leq 25$

- 这张表能立刻告诉你方案是否够快。如果 $n = 10^5$ 而你的解法是 $O(n^2)$，那就是 $10^{10}$ 次操作——太慢。你需要更好的算法。

### 如何分析 Big O

- 对 $n$ 个元素 **单层循环**：$O(n)$。

```python
total = 0
for x in arr:   # n iterations
    total += x   # O(1) per iteration
# Total: O(n)
```

- **嵌套循环**：把迭代次数相乘。

```python
for i in range(n):       # n iterations
    for j in range(n):   # n iterations each
        process(i, j)    # O(1)
# Total: O(n^2)
```

- **每次减半的循环**：$O(\log n)$。每次迭代把问题规模减半，所以需要 $\log_2 n$ 次迭代。

```python
i = n
while i > 0:
    process(i)
    i //= 2
# Total: O(log n)
```

- **内层依赖外层的嵌套循环**：

```python
for i in range(n):
    for j in range(i):   # j goes from 0 to i-1
        process(i, j)
# Total: 0 + 1 + 2 + ... + (n-1) = n(n-1)/2 = O(n^2)
```

- **递归**：写出递推关系再求解（第 13 章讲过 Master Theorem）。例如归并排序：$T(n) = 2T(n/2) + O(n) = O(n \log n)$。

### 常见陷阱

- **隐藏循环**：在 Python 中 `x in list` 是 $O(n)$（线性扫描），但 `x in set` 是 $O(1)$。在循环内对 list 使用 `in` 会得到 $O(n^2)$，而不是 $O(n)$。

```python
# BAD: O(n^2) — "in" on a list is O(n)
for x in arr:
    if x in another_list:
        process(x)

# GOOD: O(n) — convert to set first
another_set = set(another_list)
for x in arr:
    if x in another_set:
        process(x)
```

- **字符串拼接**：Python 中 `s += c` 每次都会拷贝整段字符串。在 $n$ 次迭代的循环里：$O(1 + 2 + \cdots + n) = O(n^2)$。

- **排序占主导**：如果你的算法先排序（$O(n \log n)$）再做一次线性扫描（$O(n)$），总计是 $O(n \log n)$——排序占主导。

- **摊还复杂度**：有些操作偶尔昂贵，但平均便宜。动态数组的 append 是 $O(1)$ 摊还，因为罕见的 $O(n)$ 扩容被分摊到 $n$ 次廉价 append 上。不要把摊还 $O(1)$ 与最坏 $O(1)$ 混为一谈。

### 空间复杂度

- 空间复杂度遵循同样的 Big O 规则，只是把时间换成内存占用。

- **原地（in-place）** 算法使用 $O(1)$ 额外空间（不计输入）。快速排序是 $O(\log n)$ 空间（递归栈深度）。归并排序是 $O(n)$（用于合并的临时数组）。

- **递归栈**：每次递归调用都占用栈空间。深度为 $n$ 的递归用 $O(n)$ 空间，即便每次调用本身不额外分配内存。这就是为什么在 $n$ 个节点的图上做递归 DFS 用 $O(n)$ 空间。

- 在面试中，始终同时说出时间和空间复杂度。$O(n)$ 时间、$O(n)$ 空间的解法通常可接受，而 $O(n)$ 时间、$O(1)$ 空间的解法更优。面试官可能要求你优化其中之一。

## 递归

- **递归（recursion）** 是指函数调用自身来求解同一问题的更小实例。它对具有递归结构的问题最自然：tree、嵌套结构、分治、以及数学序列。

- 每个递归函数都有两部分：
    1. **基本情况（base case）**：可以直接求解（无需递归）的最小实例，它让递归停下来。
    2. **递归情况（recursive case）**：把问题拆成更小的子问题，递归求解，再合并结果。

### 示例：阶乘

```python
def factorial(n):
    if n <= 1:        # base case
        return 1
    return n * factorial(n - 1)  # recursive case
```

- `factorial(4)` 的执行过程：
    - `factorial(4)` 调用 `factorial(3)`
    - `factorial(3)` 调用 `factorial(2)`
    - `factorial(2)` 调用 `factorial(1)`
    - `factorial(1)` 返回 `1`（基本情况）
    - `factorial(2)` 返回 `2 * 1 = 2`
    - `factorial(3)` 返回 `3 * 2 = 6`
    - `factorial(4)` 返回 `4 * 6 = 24`

- 每次调用都被压入 **调用栈**。栈会一直增长，直到命中基本情况，然后随着每次调用返回而一层层出栈。如果递归过深（例如在 Python 中调用 `factorial(1000000)`），栈会溢出（`RecursionError`）。Python 默认递归上限是 1000。

### 如何用递归思维思考

- 关键的心智转换：**信任递归**。写递归函数时，假定递归调用对更小的子问题会返回正确答案。你只需要：
    1. 处理基本情况。
    2. 把问题拆成更小的部分。
    3. 合并结果。

- 你不必在脑中逐步追踪每次递归调用。那就像靠在脑中执行每一次迭代来理解循环。相反，去验证："如果递归调用对更小输入给出正确答案，我的合并步骤对完整输入是否也给出正确答案？"

### 示例：链表上的递归

- 递归翻转链表：

```python
def reverse(head):
    if not head or not head.next:   # base case: 0 or 1 nodes
        return head

    new_head = reverse(head.next)   # reverse the rest
    head.next.next = head           # point the next node back to me
    head.next = None                # I am now the tail
    return new_head
```

- **信任递归**：`reverse(head.next)` 会正确地翻转链表剩余部分并返回新头节点。我们只需把当前节点接到末尾。

### 示例：树上的递归

- 计算二叉树高度：

```python
def height(root):
    if not root:           # base case: empty tree has height 0
        return 0
    left_h = height(root.left)    # height of left subtree
    right_h = height(root.right)  # height of right subtree
    return 1 + max(left_h, right_h)  # this node adds 1 level
```

- 这个模式——"递归左、递归右、合并"——能解决绝大多数 tree 问题（见文件 03）。

### 递归 vs 迭代

- 任何递归算法都可以转成迭代（用显式栈或循环）。迭代避免了调用栈开销和栈溢出风险。

- **何时优先用递归**：问题具有天然递归结构（tree、嵌套数据、分治）。递归解法更简洁、更易推理。

- **何时优先用迭代**：递归深度可能非常大（例如处理 $10^6$ 个节点的链表）。迭代解法避免栈溢出。

- **尾递归**：若递归调用是函数中最后一步操作（递归调用返回后不再做任何事），则称其为 "尾递归"。某些语言（Scheme、Scala）会对尾调用做优化，使用常量栈空间。Python **不** 做尾调用优化，所以 Python 中的尾递归仍占用 $O(n)$ 栈空间。

### 常见陷阱

| 陷阱 | 示例 | 修复 |
|---------|---------|-----|
| 缺少基本情况 | 无限递归 → 栈溢出 | 始终定义停止条件 |
| 基本情况错误 | 递归分解中差一 | 用最小输入（0、1、2）测试 |
| 没有缩小问题 | `f(n)` 调用 `f(n)` 而非 `f(n-1)` | 确保子问题严格更小 |
| 重复计算 | Fibonacci：`f(n) = f(n-1) + f(n-2)` 指数级重复计算 | 用记忆化（→ DP） |
| Python 递归上限 | `factorial(10000)` 崩溃 | 用 `sys.setrecursionlimit` 或改迭代 |

## 回溯

- **回溯（backtracking）** 是一种系统地探索所有可能解的方法：逐步构建解，并抛弃不可能导向有效答案的部分解。

- 把它想象成在迷宫里走。每个岔路口你选一条路。如果走到死胡同，就退回上一个岔路口换条路。你不会从头重来——而是 **回退** 到最近的决策点。

### 三步法

每个回溯算法都遵循同一模式：

1. **选择（Choose）**：挑一个候选来扩展当前部分解。
2. **探索（Explore）**：递归地从这个候选出发尝试构建完整解。
3. **撤销（Unchoose）**：撤销刚才的选择（回溯），再试下一个候选。

```python
def backtrack(state, choices, result):
    if is_complete(state):
        result.append(state.copy())
        return

    for choice in choices:
        if is_valid(choice, state):
            state.add(choice)           # 1. choose
            backtrack(state, choices, result)  # 2. explore
            state.remove(choice)        # 3. unchoose (backtrack)
```

- **撤销** 这一步是回溯区别于普通递归的关键。没有它，state 会累积所有选择，无法探索其他路径。

### 何时使用回溯

- 问题要求 **枚举所有合法配置**：所有排列、所有子集、所有合法摆放（例如 N-Queens）。
- 问题要求 **找出任意一个合法配置**：数独求解、迷宫寻路。
- 搜索空间很大但可以 **剪枝**：大多数部分解可以被早早丢弃，无需完整探索。

### 剪枝如何让它变快

- 没有剪枝时，回溯会探索每一种组合——指数级时间。**剪枝** 提前砍掉分支：

```python
for choice in choices:
    if not is_valid(choice, state):
        continue  # PRUNE: skip this entire subtree

    state.add(choice)
    backtrack(state, choices, result)
    state.remove(choice)
```

- 在 N-Queens（文件 05）中，放置皇后前检查列冲突和对角冲突，把搜索树从 $n^n$ 剪到大约 $n!$ 个候选。对 $n = 8$，就是 1600 万 → 4 万。良好的剪枝能让指数算法在中等规模 $n$ 下可用。

### 生成所有子集（最简单的回溯）

```python
def subsets(nums):
    result = []

    def backtrack(start, path):
        result.append(path[:])  # every partial solution is a valid subset

        for i in range(start, len(nums)):
            path.append(nums[i])        # choose
            backtrack(i + 1, path)       # explore (i+1: no reuse)
            path.pop()                   # unchoose

    backtrack(0, [])
    return result
```

- 对 `[1, 2, 3]`，递归树：
    - `[]` → `[1]` → `[1,2]` → `[1,2,3]`（回溯）→ `[1,3]`（回溯）→ `[2]` → `[2,3]`（回溯）→ `[3]`

- 树中每个节点都是一次 `backtrack` 调用。每个叶子（以及中间节点）产出一个子集。子集总数：$2^n$。

### 生成所有排列

```python
def permutations(nums):
    result = []

    def backtrack(path, remaining):
        if not remaining:
            result.append(path[:])
            return

        for i in range(len(remaining)):
            path.append(remaining[i])                    # choose
            backtrack(path, remaining[:i] + remaining[i+1:])  # explore
            path.pop()                                   # unchoose

    backtrack([], nums)
    return result
```

- 排列总数：$n!$。每个排列需要 $O(n)$ 工作来构造 `remaining`，所以总计 $O(n \cdot n!)$。

### 常见陷阱

| 陷阱 | 示例 | 修复 |
|---------|---------|-----|
| 忘记拷贝 path | `result.append(path)`——所有条目共享同一 list | `result.append(path[:])` 或 `path.copy()` |
| 没有回溯（撤销） | state 一直增长，后续候选看到的是陈旧 state | 递归调用后始终 `path.pop()` 或 `state.remove()` |
| 循环起点错误 | 子集去重、或排列出现不该有的重复使用 | 用 `start` 参数避免重访更早的索引 |
| 跳过剪枝 | 探索明显无效的分支 | 在递归调用前加 `if not is_valid: continue` |

## 动态规划

- **动态规划（dynamic programming，DP）** 是一种用于反复求解同一子问题的优化技术。DP 不重复计算，而是每个子问题只算一次并保存结果。

- 当问题具有以下两个性质时 DP 适用：
    1. **最优子结构（optimal substructure）**：最优解可由子问题的最优解构造。
    2. **重叠子问题（overlapping subproblems）**：同一子问题在递归中多次出现。

### 来自 Fibonacci 的动机

- 朴素递归 Fibonacci：

```python
def fib(n):
    if n <= 1:
        return n
    return fib(n - 1) + fib(n - 2)
```

- 对 `fib(5)`，递归树：
    - `fib(5)` 调用 `fib(4)` 和 `fib(3)`
    - `fib(4)` 调用 `fib(3)` 和 `fib(2)`
    - `fib(3)` 被算 **两次**，`fib(2)` 被算 **三次**

- 这是 $O(2^n)$，因为树每层都在分支，且大多数分支在重算同样的值。对 `fib(50)`，需要超过 $10^{15}$ 次操作——不可行。

- 用 **记忆化（memoisation，自顶向下 DP）**：

```python
def fib_memo(n, memo={}):
    if n in memo:
        return memo[n]
    if n <= 1:
        return n
    memo[n] = fib_memo(n - 1, memo) + fib_memo(n - 2, memo)
    return memo[n]
```

- 现在 `fib(3)` 只算一次，存起来，后续调用直接查表。总计：$O(n)$ 时间、$O(n)$ 空间。

- 用 **制表（tabulation，自底向上 DP）**：

```python
def fib_tab(n):
    if n <= 1:
        return n
    dp = [0] * (n + 1)
    dp[1] = 1
    for i in range(2, n + 1):
        dp[i] = dp[i - 1] + dp[i - 2]
    return dp[n]
```

- 同样是 $O(n)$ 时间，但自底向上、不靠递归地构建解。还可进一步优化到 $O(1)$ 空间，因为每个值只依赖前两个。

### DP 套路

任何 DP 题都遵循以下步骤：

1. **定义状态**：`dp[i]`（或 `dp[i][j]`）代表什么？这是最难的一步。状态必须捕捉足够的信息以做出最优决策。

2. **写出递推**：`dp[i]` 与更小子问题有什么关系？这就是转移公式。

3. **确定基本情况**：哪些最小子问题可以直接求解？

4. **确定迭代顺序**：哪些子问题必须先于哪些算？自底向上：按依赖关系已被解决的顺序迭代。自顶向下：递归自动处理。

5. **优化空间**（可选）：如果 `dp[i]` 只依赖上一行或前几个条目，就不需要整张表。

### 示例：思考过程

**问题**：给定一个正整数数组，找出非相邻元素的最大和（House Robber）。

**第 1 步——定义状态**：`dp[i]` = 考虑 `nums[0..i]` 时能获得的最大和。

**第 2 步——写出递推**：对元素 $i$，我们要么：
- 跳过它：`dp[i] = dp[i-1]`（不含元素 $i$ 的最优和）。
- 取它：`dp[i] = dp[i-2] + nums[i]`（必须跳过元素 $i-1$，再加上元素 $i$）。

所以：`dp[i] = max(dp[i-1], dp[i-2] + nums[i])`。

**第 3 步——基本情况**：`dp[0] = nums[0]`，`dp[1] = max(nums[0], nums[1])`。

**第 4 步——迭代顺序**：从左到右（每个状态依赖前两个状态）。

**第 5 步——空间优化**：只需最后两个值。

```python
def rob(nums):
    if len(nums) == 1:
        return nums[0]

    prev2, prev1 = nums[0], max(nums[0], nums[1])

    for i in range(2, len(nums)):
        curr = max(prev1, prev2 + nums[i])
        prev2, prev1 = prev1, curr

    return prev1
```

### 如何识别 DP 题

- 问题求 **最优值**（最小代价、最大利润、最长序列）或 **计数**（方法数）。
- 问题在每一步有 **选择**（取/跳、左/右、用或不用某枚硬币），而整体最优答案依赖子问题的最优答案。
- 画出递归树会发现 **重复子问题**。
- 暴力是指数级，但 **不同状态数** 远少于递归调用数。

### DP 的分类

- **一维 DP**：状态依赖单个索引。例如：爬楼梯、House Robber、最大子数组。

- **二维 DP**：状态依赖两个索引。例如：最长公共子序列（`dp[i][j]` 对应字符串 1 的前 $i$ 个字符与字符串 2 的前 $j$ 个字符）、编辑距离、网格路径问题。

- **区间 DP**：状态是区间 `dp[i][j]`，表示 `arr[i..j]` 上的子问题。例如：矩阵链乘法、打气球。

- **背包 DP**：状态是物品索引和容量。例如：0/1 背包、找零、子集和。

- **位掩码 DP**：状态含一个表示已用元素的位掩码。例如：TSP、指派问题。状态空间 $O(2^n \cdot n)$，适用于 $n \leq 20$。

### 自顶向下 vs 自底向上

| | 自顶向下（记忆化） | 自底向上（制表） |
|--|---|---|
| 实现 | 递归 + 缓存 | 迭代 + 表 |
| 计算内容 | 只算真正需要的子问题 | 算出目标之前的所有子问题 |
| 栈溢出风险 | 有（深递归） | 无 |
| 空间优化 | 较难 | 较易（用滚动数组） |
| 编码难度 | 通常更自然（写递归、加缓存） | 需要考虑迭代顺序 |

- 面试中，自顶向下通常编码更快。生产中一般偏好自底向上，性能更好（无递归开销、更好的缓存行为）。

### 常见陷阱

| 陷阱 | 示例 | 修复 |
|---------|---------|-----|
| 状态定义错误 | `dp[i]` 信息不足以做决策 | 加维度（如 `dp[i][j]` 代替 `dp[i]`） |
| 缺少基本情况 | `dp[0]` 错 → 后续全错 | 手工验证基本情况 |
| 迭代顺序错误 | 在依赖项之前算 `dp[i]` | 画依赖箭头，按其顺序迭代 |
| `dp` 初始化错误 | 该用无穷却用 0（最小化问题） | 最小化用 `float('inf')`，最大化用 `float('-inf')` |
| 忘记考虑 "跳过" 选项 | 总是取当前元素 | 递推通常含 `max(take, skip)` |
| 可变默认参数 | `def f(memo={})` 跨调用共享缓存 | `def f(memo=None): if memo is None: memo = {}` |
| 二维 DP 差一 | `dp` 是 1 索引却写 `text1[i]` | `dp` 大小为 `(m+1) x (n+1)`，访问 `text1[i-1]` |

## 串起来

- 这四个概念构成一条递进：
    1. **Big O** 告诉你方案是否够快。
    2. **递归** 把问题拆成子问题。
    3. **回溯** = 递归 + 选择 + 撤销，用于穷举搜索。
    4. **DP** = 递归 + 缓存，用于重叠子问题上的优化。

- 看到一道新题时：
    - 估计输入规模 $n$。可接受的 Big O 是多少？
    - 若暴力是指数级且要求枚举/找配置：**回溯**（加剪枝让它可用）。
    - 若暴力是指数级且要求最优值或计数，且存在重叠子问题：**DP**。
    - 若问题结构能把搜索空间减半：**二分查找** 或 **分治**。
    - 若问题在序列上、且对子数组有约束：**滑动窗口** 或 **双指针**。
    - 若问题需要快速查找：**hash map**。
