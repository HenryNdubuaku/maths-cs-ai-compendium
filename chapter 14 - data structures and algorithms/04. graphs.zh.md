- 我们在第 12 章讲过 graph 理论（邻接矩阵、Laplacian、谱性质），第 13 章讲过 tree、平面性、着色。这里我们聚焦 **算法 pattern**：如何用代码遍历、搜索、优化 graph。

- 两种基本 graph 算法是 **BFS** 和 **DFS**。几乎每个 graph 题都能归约为其一，可能略作改动。掌握这两种，你就能解决大多数 graph 题。

## 图的表示

- **邻接表（adjacency list）**：对每个节点存一个邻居列表。空间：$O(|V| + |E|)$。最适合稀疏图（多数现实图）。

```python
# Undirected graph
graph = {
    0: [1, 2],
    1: [0, 3],
    2: [0, 3],
    3: [1, 2]
}

# From edge list
def build_graph(n, edges):
    graph = {i: [] for i in range(n)}
    for u, v in edges:
        graph[u].append(v)
        graph[v].append(u)  # omit for directed
    return graph
```

- **邻接矩阵（adjacency matrix）**：$n \times n$ 矩阵，$A[i][j] = 1$ 表示存在边 $(i, j)$。空间：$O(|V|^2)$。适合稠密图，或需要 $O(1)$ 边查询时。

- **何时用哪个**：几乎一切情况都用邻接表。只有当图稠密（$|E| \approx |V|^2$）或需要常量时间判断边是否存在时才用矩阵。

## Pattern：BFS（广度优先搜索）

- BFS 用队列 **逐层** 探索节点。它是以下场景的默认算法：
    - **无权** 图的最短路径
    - 层序遍历
    - 找连通分量
    - 任何求 "最少步数" 的题

```python
from collections import deque

def bfs(graph, start):
    visited = {start}
    queue = deque([start])

    while queue:
        node = queue.popleft()
        for neighbour in graph[node]:
            if neighbour not in visited:
                visited.add(neighbour)
                queue.append(neighbour)
```

- **关键**：在 **入队时** 标记 `visited`，而非出队时。若在出队时标记，同一节点会被多个前驱重复入队，浪费时间和可能出错。

### 简单：Number of Islands

- **问题**：给定一个由 '1'（陆地）和 '0'（水）组成的二维网格，数岛屿数量。

- **Pattern**：遍历网格。遇到 '1' 就用 BFS/DFS 把所有相连陆地标为已访问。每次发起 BFS 就是一个岛屿。

```python
from collections import deque

def num_islands(grid):
    if not grid:
        return 0

    rows, cols = len(grid), len(grid[0])
    count = 0

    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == '1':
                count += 1
                # BFS to mark entire island
                queue = deque([(r, c)])
                grid[r][c] = '0'  # mark visited
                while queue:
                    cr, cc = queue.popleft()
                    for dr, dc in [(0,1),(0,-1),(1,0),(-1,0)]:
                        nr, nc = cr + dr, cc + dc
                        if 0 <= nr < rows and 0 <= nc < cols and grid[nr][nc] == '1':
                            grid[nr][nc] = '0'
                            queue.append((nr, nc))

    return count
```

- **陷阱**：`directions = [(0,1),(0,-1),(1,0),(-1,0)]` 这个 4 连通网格邻居 pattern 几乎在每个网格题里都用。背下来。要做 8 连通，加上对角线。

- **陷阱**：直接改输入网格（`grid[r][c] = '0'`）省去单独的 `visited` 集合。面试里可接受，但要说清权衡（修改了输入）。

### 中等：Rotting Oranges

- **问题**：新鲜橘子若与腐烂橘子相邻就会腐烂。返回所有橘子腐烂的最短时间（若不可能则 -1）。

- **Pattern**：多源 BFS。把所有初始就腐烂的橘子同时入队。每层 BFS 是一个时间步。

```python
from collections import deque

def oranges_rotting(grid):
    rows, cols = len(grid), len(grid[0])
    queue = deque()
    fresh = 0

    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == 2:
                queue.append((r, c))
            elif grid[r][c] == 1:
                fresh += 1

    if fresh == 0:
        return 0

    time = 0
    while queue and fresh > 0:
        time += 1
        for _ in range(len(queue)):
            cr, cc = queue.popleft()
            for dr, dc in [(0,1),(0,-1),(1,0),(-1,0)]:
                nr, nc = cr + dr, cc + dc
                if 0 <= nr < rows and 0 <= nc < cols and grid[nr][nc] == 1:
                    grid[nr][nc] = 2
                    fresh -= 1
                    queue.append((nr, nc))

    return time if fresh == 0 else -1
```

- **关键直觉**：多源 BFS 同时处理所有源。这给出离 *任意* 源的最短距离，正是 "最后一个新鲜橘子腐烂需要多久"。

## Pattern：DFS（深度优先搜索）

- DFS 尽可能深地探索，再回溯。它用栈（显式或递归调用栈）。DFS 适用于：
    - 环检测
    - 拓扑排序
    - 连通分量
    - 回溯 / 穷举搜索
    - 带约束的寻路

```python
def dfs(graph, node, visited=None):
    if visited is None:
        visited = set()
    visited.add(node)
    for neighbour in graph[node]:
        if neighbour not in visited:
            dfs(graph, neighbour, visited)
```

### 中等：Course Schedule（环检测）

- **问题**：给定 $n$ 门课和先修关系，判断能否完成所有课（即无循环依赖）。

- **Pattern**：在有向图中检测环。用带三态的 DFS：未访问、进行中（在当前 DFS 路径上）、已完成。

```python
def can_finish(num_courses, prerequisites):
    graph = {i: [] for i in range(num_courses)}
    for course, prereq in prerequisites:
        graph[course].append(prereq)

    # 0 = unvisited, 1 = in-progress, 2 = completed
    state = [0] * num_courses

    def has_cycle(node):
        if state[node] == 1:
            return True   # back edge → cycle
        if state[node] == 2:
            return False  # already fully explored

        state[node] = 1  # mark in-progress
        for neighbour in graph[node]:
            if has_cycle(neighbour):
                return True
        state[node] = 2  # mark completed
        return False

    for course in range(num_courses):
        if has_cycle(course):
            return False
    return True
```

- **为什么三态**：两态（已访问/未访问）无法区分 "我正在探索该节点" 与 "我已探索完该节点"。发现一个正在被探索的节点（状态 = 1）意味着找到环。发现一个已完全探索的节点（状态 = 2）只是横跨边，不是环。

### 中等：Course Schedule II（拓扑排序）

- **问题**：返回一个合法的课程顺序（拓扑序）。

- **Pattern（Kahn 算法——基于 BFS）**：从入度为 0 的节点开始。处理它们，削减邻居入度。重复。

```python
from collections import deque

def find_order(num_courses, prerequisites):
    graph = {i: [] for i in range(num_courses)}
    indegree = [0] * num_courses

    for course, prereq in prerequisites:
        graph[prereq].append(course)
        indegree[course] += 1

    queue = deque([i for i in range(num_courses) if indegree[i] == 0])
    order = []

    while queue:
        node = queue.popleft()
        order.append(node)
        for neighbour in graph[node]:
            indegree[neighbour] -= 1
            if indegree[neighbour] == 0:
                queue.append(neighbour)

    return order if len(order) == num_courses else []  # empty = cycle exists
```

- **陷阱**：若结果节点数少于图节点数，说明存在环（某些节点入度永远到不了 0）。

## 最短路径

### Dijkstra 算法

- 在 **非负** 加权图中找从源点到所有其他节点的最短路径。用优先队列（最小堆）。

```python
import heapq

def dijkstra(graph, start):
    # graph: {node: [(neighbour, weight), ...]}
    dist = {node: float('inf') for node in graph}
    dist[start] = 0
    heap = [(0, start)]

    while heap:
        d, node = heapq.heappop(heap)
        if d > dist[node]:
            continue  # stale entry

        for neighbour, weight in graph[node]:
            new_dist = d + weight
            if new_dist < dist[neighbour]:
                dist[neighbour] = new_dist
                heapq.heappush(heap, (new_dist, neighbour))

    return dist
```

- 时间：用二叉堆时 $O((|V| + |E|) \log |V|)$。

- **陷阱**：`if d > dist[node]: continue` 这一行必不可少。没有它会处理陈旧堆条目，可能退化到 $O(|V|^2)$。

- **陷阱**：Dijkstra 不能处理负权。若某条边为负，贪心假设（节点一旦确定其距离就是最优）就崩了。改用 Bellman-Ford。

### 困难：Network Delay Time

- **问题**：给定 $n$ 个节点和加权有向边，找信号从源点到达所有节点的时间。若不全部可达返回 -1。

```python
def network_delay(times, n, k):
    graph = {i: [] for i in range(1, n + 1)}
    for u, v, w in times:
        graph[u].append((v, w))

    dist = dijkstra(graph, k)
    max_time = max(dist.values())
    return max_time if max_time < float('inf') else -1
```

## 强连通分量

- 在有向图中，**强连通分量（strongly connected component，SCC）** 是一个极大节点集合，其中每个节点都能到达其他每个节点。

- **Kosaraju 算法**：(1) 在原图上 DFS，记录完成顺序。(2) 转置图（反转所有边）。(3) 按完成顺序的逆序在转置图上 DFS。第 3 步中每棵 DFS 树是一个 SCC。

- **何时用**：找循环依赖、2-SAT、把有向图压缩成 SCC 的 DAG。

## 常见陷阱汇总

| 陷阱 | 示例 | 修复 |
|---------|---------|-----|
| 出队时才标记已访问 | 同一节点多次入队 | 入队时标记已访问 |
| 有向图用两态已访问 | 无法区分回边与横跨边 | 用三态：未访问/进行中/已完成 |
| Dijkstra 用于负权 | 最短路径错误 | 改用 Bellman-Ford |
| 忘写 `if d > dist[node]: continue` | 处理陈旧堆条目 | 当前距离更差时始终跳过 |
| 网格越界检查 | 索引越界 | `0 <= nr < rows and 0 <= nc < cols` |
| 没处理 time=0 边界 | 腐烂橘子：无新鲜橘子 | BFS 前检查 `fresh == 0` |
| 把有向图当无向图建 | 先修关系是单向的 | 只在一个方向加边 |
