# 图注意力网络

*图注意力网络用学习的、依赖数据的加权替代均匀的邻居 aggregation。本文件涵盖 GAT、多头 graph attention、GATv2、Graph Transformer、位置与结构编码以及可扩展性*

- 在 GCN（文件 3）中，每个 node 用由 graph 结构（归一化 adjacency）决定的固定权重 aggregation 其邻居的特征。一个有三邻居的 node 给每个邻居大致相等的权重（$\approx 1/3$）。但并非所有邻居同等重要：来自密切合作者的 message 应比来自疏远熟人的更受重视。

- **Graph Attention Networks** 通过学习**应当 attend 哪些邻居**来解决这一问题，使用与驱动 transformer 相同的 attention 机制（第 7 章）。每个 node 在其邻居上计算动态的、基于内容的 attention 分数，而非固定的、基于结构的权重。

## GAT：图注意力网络

- **GAT**（Veličković et al., 2018）计算每个 node 与其邻居之间的 attention 系数。对于 node $i$ 和邻居 $j$：

$$e_{ij} = \text{LeakyReLU}\left(\mathbf{a}^T \left[W\mathbf{h}_i \| W\mathbf{h}_j\right]\right)$$

- 其中 $W \in \mathbb{R}^{d' \times d}$ 是共享的线性变换，$\|$ 表示拼接，$\mathbf{a} \in \mathbb{R}^{2d'}$ 是可学习的 attention 向量。分数 $e_{ij}$ 衡量 node $j$ 的特征对 node $i$ 有多重要。

- 原始分数用 softmax 在所有邻居上归一化：

$$\alpha_{ij} = \text{softmax}_j(e_{ij}) = \frac{\exp(e_{ij})}{\sum_{k \in \mathcal{N}(i)} \exp(e_{ik})}$$

- 这确保 attention 权重在每个 node 的邻域内和为 1，就像 transformer 的 attention（第 7 章）。node 更新后的特征为：

$$\mathbf{h}_i' = \sigma\left(\sum_{j \in \mathcal{N}(i)} \alpha_{ij} W\mathbf{h}_j\right)$$

![GCN 给所有邻居固定相等的权重；GAT 学习依赖数据的 attention 权重](../images/gat_attention_weights.svg)

- 与 GCN 的关键区别在于：权重 $\alpha_{ij}$ 是**从数据中学到的**，而非由 graph 结构固定的。node 可以学会聚焦于最具信息量的邻居，同时忽略嘈杂或无关的邻居。

- 注意 attention 只在 edge 上计算（node $i$ 只 attend 到其邻居 $\mathcal{N}(i)$），而非所有 node 对之间。这使计算量与 edge 数成正比，而非与 node 数的平方成正比。

## 多头图注意力

- 正如在 transformer 中（第 7 章），**多头 attention** 并行运行 $K$ 个独立的 attention 机制，每个有自己的参数 $W^k$ 和 $\mathbf{a}^k$。结果在中间层拼接，在最后一层取平均：

$$\mathbf{h}_i' = \Big\|_{k=1}^{K} \sigma\left(\sum_{j \in \mathcal{N}(i)} \alpha_{ij}^k W^k \mathbf{h}_j\right)$$

- 每个头可以 attend 到邻域的不同方面：一个头可能聚焦于结构特征，另一个聚焦于语义相似性。这与 transformer 中多头 attention 的动机相同：不同的头捕捉不同类型的关系。

- 当有 $K$ 个头、每个头输出维度为 $d'$ 时，拼接后的输出维度为 $K \times d'$。最后一层通常用平均而非拼接以产生固定大小的输出。

## GATv2：修复静态注意力

- 原始 GAT 有一个微妙的局限：其 attention 函数是**静态的**（也叫基于排序的）。attention 分数依赖拼接 $[W\mathbf{h}_i \| W\mathbf{h}_j]$，但由于 attention 向量 $\mathbf{a}$ 在拼接之后施加，可以分解为两个独立分量：$\mathbf{a}^T [W\mathbf{h}_i \| W\mathbf{h}_j] = \mathbf{a}_1^T W\mathbf{h}_i + \mathbf{a}_2^T W\mathbf{h}_j$。

- 这意味着对于给定的 node $i$，其邻居的排序完全由邻居的特征 $\mathbf{h}_j$ 决定（项 $\mathbf{a}_1^T W\mathbf{h}_i$ 对 $i$ 的所有邻居都是常数）。attention 排序并不真正依赖 query node 的特征。node $i$ 和 node $k$ 会对同一组邻居给出相同的排序，这限制了表达能力。

- **GATv2**（Brody et al., 2022）通过在 attention 向量之前施加非线性来修复这一点：

$$e_{ij} = \mathbf{a}^T \text{LeakyReLU}\left(W \left[\mathbf{h}_i \| \mathbf{h}_j\right]\right)$$

- 将 LeakyReLU 移入计算意味着 attention 分数是联合特征的非线性函数，不可分解为独立项。这使得 attention 变成**动态的**：邻居的排序现在依赖于具体的 query node。GATv2 在不增加任何计算开销的情况下严格比 GAT 更具表达能力。

## Graph Transformer

- 标准 message-passing GNN 受限于 graph 拓扑：一个 node 只能 attend 到其直接邻居。经过 $k$ 层后，来自 $k$-hop 邻居的信息已经经过多次 aggregation 混合，丢失了保真度。这一局部瓶颈（加上文件 3 中的 over-smoothing）限制了对长程依赖的捕捉能力。

- **Graph Transformer** 通过对所有 node 对施加**全局 self-attention** 来打破这一瓶颈，无论它们是否共享 edge。每个 node 在单层中可以 attend 到其他所有 node，正如在标准 transformer 中（第 7 章）。

- 基本思想：将所有 node 视作 token，施加 transformer 的 self-attention：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

- 其中 $Q = XW_Q$、$K = XW_K$、$V = XW_V$ 是 node 特征 $X$ 的 query、key、value 投影（与第 7 章完全一致）。这是在一个全连接 graph（完全图 $K_n$，文件 2）上运行的 GNN。

- 问题在于：全连接 graph 忽略了实际的 graph 结构。edge 信息（谁真正与谁相连）丢失了。两种方法恢复它：

- **Graphormer**（Ying et al., 2021）通过在 attention 分数中加入**偏置项**将 graph 结构注入 transformer：

$$A_{ij} = \frac{(\mathbf{h}_i W_Q)(W_K^T \mathbf{h}_j^T)}{\sqrt{d_k}} + b_{\text{spatial}}(i, j) + b_{\text{edge}}(i, j)$$

- 空间偏置 $b_{\text{spatial}}$ 编码 node $i$ 与 $j$ 之间的最短路径距离。edge 偏置 $b_{\text{edge}}$ 编码沿最短路径的 edge 特征。此外，Graphormer 使用**中心性编码**，将 node 的度加到其输入 embedding 中，给模型关于每个 node 结构角色的信息。

- **GPS**（General, Powerful, Scalable Graph Transformer，Rampášek et al., 2022）在每一层将局部 message passing 与全局 attention 结合：

$$\mathbf{h}_i' = \text{MLP}\left(\mathbf{h}_i^{\text{MPNN}} + \mathbf{h}_i^{\text{Attention}}\right)$$

- 每层都同时应用一个标准 GNN（用于局部结构）和一个 transformer（用于全局上下文），然后合并结果。这兼得两者之长：来自 message passing 的局部结构和来自 attention 的长程依赖。

## 位置编码与结构编码

- 序列上的 transformer 使用位置编码（第 7 章）注入顺序信息。Graph 没有规范的顺序，因此需要 graph 专属的编码。

- **Laplacian 特征向量编码** 使用 graph Laplacian（文件 2）的特征向量作为位置特征。最小的 $k$ 个非平凡特征向量提供 graph 的谱 embedding：在 graph 中"邻近"的 node 有相似的特征向量值。这些被拼接到 node 特征上。

- 一个微妙之处：Laplacian 特征向量有符号歧义（若 $\mathbf{u}$ 是特征向量，$-\mathbf{u}$ 也是）。模型必须对这些符号翻转不变。解决方案包括训练期间使用随机符号翻转作为数据增强，或学习符号不变的变换。

- **随机游走编码**计算从 node $i$ 出发的随机游走经过 $k$ 步回到 node $i$ 的概率，$k = 1, 2, \ldots, K$。这些概率编码局部结构信息：密集簇中的 node 返回概率高，而稀疏区域的 node 返回概率低。落点概率 $p_{ii}^{(k)} = (A_{\text{rw}}^k)_{ii}$，其中 $A_{\text{rw}} = D^{-1}A$ 是随机游走转移矩阵。

- **度编码**只是把 node 的度作为特征加入。这出乎意料地有效，因为度是一个强结构信号：叶 node（度为 1）、桥接 node 和枢纽 node 行为不同。

- 这些编码提供了 vanilla transformer 缺乏的结构信息，使 Graph Transformer 在需要长程推理的任务上优于标准 message-passing GNN。

## 可扩展性

- GNN 的根本可扩展性挑战在于 graph 可能有数百万 node 和数十亿 edge。在完整 graph 上训练 GNN 需要将所有 node 特征和整个 adjacency matrix 存入内存，这往往不可行。

- GNN 的 **mini-batch 训练**比图像或序列更复杂，因为 node 之间相互连接。简单地采样一批 node 需要它们的邻居（第 1 层）、邻居的邻居（第 2 层），依此类推。这种**邻域爆炸**意味着一批 1000 个目标 node 可能需要在计算图中包含数百万个 node。

- **邻域采样**（GraphSAGE 风格，文件 3）通过每层对每个 node 采样固定数量的邻居来限制爆炸。当 2 层且每层 15 个样本时，每个目标 node 的子图最多有 $15^2 = 225$ 个 node，与完整 graph 大小无关。

- **Cluster-GCN**（Chiang et al., 2019）用 graph 聚类算法（如 METIS）将 graph 划分为簇，然后每次在一个簇上训练。簇内 edge 密集（大多数邻居在同一簇中），所以子图捕捉了相关结构。簇间 edge 通过偶尔包含跨簇 edge 来处理。

- **Graph Transformer 的可扩展性**更困难，因为全局 attention 是 $O(n^2)$。对于数百万 node 的 graph，全 attention 不可行。解决方案包括：
    - 稀疏 attention 模式（只 attend graph 中的 $k$ 近邻 node）
    - 线性 attention 逼近
    - 将局部 message passing（廉价，$O(|E|)$）与在粗化 graph（更少 node）上的全局 attention 结合

## 时序图与动态图

- 到目前为止我们研究的 graph 都是**静态的**：node、edge 和特征固定。但许多现实世界的 graph **随时间演化**：新用户加入社交网络，金融交易产生 edge，交通模式在一天中变化，分子相互作用也在波动。

- **时序图**（temporal graph）为每条 edge 增加一个时间戳：$(i, j, t)$ 表示 node $i$ 在时间 $t$ 与 node $j$ 互动。挑战在于学习能同时捕捉 graph 结构和时间动态的表示。

- 有两种范式：

- **离散时间动态图（DTDG）**：graph 表示为一系列快照 $G_1, G_2, \ldots, G_T$，每个时间步一个。GNN 处理每个快照，RNN 或时间 attention 机制捕捉跨快照的演化。这简单但丢失细粒度时间信息（快照之间的事件丢失），且需要选择快照频率。

- **连续时间动态图（CTDG）**：事件被建模为带时间戳互动的流。每个事件 $(i, j, t)$ 在发生的确切时刻更新 node $i$ 和 $j$ 的表示。这保留了所有时间信息。

- **Temporal Graph Network (TGN)**（Rossi et al., 2020）是领先的 CTDG 架构。每个 node 维护一个**记忆状态** $\mathbf{s}_i(t)$，每当 node 参与一次互动就更新：

$$\mathbf{s}_i(t^+) = \text{GRU}\left(\mathbf{s}_i(t^-), \; \mathbf{m}_i(t)\right)$$

- 其中 $\mathbf{m}_i(t)$ 是从互动计算的 message（结合两个 node 的特征、edge 特征和时间编码）。GRU（第 6 章）选择性地保留和遗忘过去信息，使记忆能捕捉长期模式的同时适应最近事件。

- **时间编码**将自上次互动以来的时间间隔表示为特征向量，类似于 transformer 中的位置编码（第 7 章）。一种常见方法使用可学习的 Fourier 特征：

$$\Phi(t) = \left[\cos(\omega_1 t), \sin(\omega_1 t), \ldots, \cos(\omega_d t), \sin(\omega_d t)\right]$$

- 这给模型一个关于时间间隔的丰富表示："该用户 5 分钟前最后活跃"与"3 个月前最后活跃"会被不同地 embedding。

- **Temporal Graph Attention (TGAT)** 在 node 的时序邻域上施加 self-attention：一组近期互动，每个都按特征相关性（如 GAT）和时间近度加权。来自遥远过去的互动自然被降权。

- 应用包括欺诈检测（金融 graph 中的异常交易模式）、交通预测（从历史流量模式预测拥堵）、社交网络动态（预测病毒内容传播）以及药物相互作用的时间预测。

## 编程任务（使用 CoLab 或 notebook）

1. 从头实现一个 GAT attention 头。计算一个 node 与其邻居之间的 attention 权重，并验证它们和为 1。
```python
import jax
import jax.numpy as jnp

rng = jax.random.PRNGKey(0)
k1, k2, k3 = jax.random.split(rng, 3)

n_nodes, d_in, d_out = 5, 4, 3

# Random node features
H = jax.random.normal(k1, (n_nodes, d_in))

# Learnable parameters
W = jax.random.normal(k2, (d_in, d_out)) * 0.5
a = jax.random.normal(k3, (2 * d_out,)) * 0.5

# Adjacency (node 0 connects to 1, 2, 3)
neighbours_of_0 = [1, 2, 3]

# Transform features
Wh = H @ W  # (n_nodes, d_out)

# Compute attention scores for node 0
h_i = Wh[0]
scores = []
for j in neighbours_of_0:
    h_j = Wh[j]
    e_ij = jnp.dot(a, jnp.concatenate([h_i, h_j]))
    e_ij = jax.nn.leaky_relu(e_ij, negative_slope=0.2)
    scores.append(float(e_ij))

scores = jnp.array(scores)
alpha = jax.nn.softmax(scores)

print(f"Raw scores: {scores}")
print(f"Attention weights: {alpha}")
print(f"Sum of weights: {alpha.sum():.4f}")

# Weighted aggregation
h_new = sum(alpha[k] * Wh[neighbours_of_0[k]] for k in range(len(neighbours_of_0)))
print(f"Updated node 0 features: {h_new}")
```

2. 比较 GCN（固定权重）与 GAT（学习权重）aggregation。展示 GAT 可以给邻居不同的权重，而 GCN 对它们一视同仁。
```python
import jax
import jax.numpy as jnp

# 4 nodes: node 0 connects to 1, 2, 3
A = jnp.array([[0,1,1,1],
               [1,0,0,0],
               [1,0,0,0],
               [1,0,0,0]], dtype=float)

# Features: node 1 is very relevant, node 2 is noise, node 3 is moderate
H = jnp.array([[0.0, 0.0],   # node 0
               [1.0, 0.0],   # node 1 (signal)
               [0.0, 0.0],   # node 2 (noise)
               [0.5, 0.0]])  # node 3 (moderate)

# GCN: normalised adjacency weights
A_hat = A + jnp.eye(4)
D_inv = jnp.diag(1.0 / A_hat.sum(axis=1))
gcn_weights = (D_inv @ A_hat)[0]  # weights for node 0
print(f"GCN weights for node 0: {gcn_weights}")
print("  → All neighbours get roughly equal weight")

# GAT: learned attention (simulated)
# Suppose the attention mechanism learns to focus on node 1
gat_weights = jnp.array([0.1, 0.7, 0.05, 0.15])  # learned
print(f"\nGAT weights for node 0: {gat_weights}")
print("  → Node 1 (informative) gets most attention")

gcn_output = gcn_weights @ H
gat_output = gat_weights @ H
print(f"\nGCN output: {gcn_output}  (diluted by noise)")
print(f"GAT output: {gat_output}  (focused on signal)")
```

3. 演示位置编码的益处。为一个 graph 计算 Laplacian 特征向量编码，展示结构相似的 node 会得到相似的编码。
```python
import jax.numpy as jnp
import matplotlib.pyplot as plt

# Barbell graph: two cliques connected by a bridge
n = 10
A = jnp.zeros((n, n))
# Clique 1: nodes 0-4
for i in range(5):
    for j in range(i+1, 5):
        A = A.at[i,j].set(1).at[j,i].set(1)
# Clique 2: nodes 5-9
for i in range(5, 10):
    for j in range(i+1, 10):
        A = A.at[i,j].set(1).at[j,i].set(1)
# Bridge
A = A.at[4,5].set(1).at[5,4].set(1)

D = jnp.diag(A.sum(axis=1))
L = D - A
eigenvalues, eigenvectors = jnp.linalg.eigh(L)

# Use first 3 non-trivial eigenvectors as positional encoding
pe = eigenvectors[:, 1:4]

print("Laplacian Positional Encodings:")
for i in range(n):
    group = "Clique 1" if i < 5 else "Clique 2"
    bridge = " (bridge)" if i in [4, 5] else ""
    print(f"  Node {i} ({group}{bridge}): {pe[i]}")

plt.scatter(pe[:5, 0], pe[:5, 1], c="#3498db", s=80, label="Clique 1")
plt.scatter(pe[5:, 0], pe[5:, 1], c="#e74c3c", s=80, label="Clique 2")
plt.scatter(pe[[4,5], 0], pe[[4,5], 1], c="black", s=120, marker="*",
            label="Bridge nodes", zorder=5)
plt.legend(); plt.grid(True)
plt.title("Laplacian Eigenvector Positional Encodings")
plt.xlabel("Eigenvector 1"); plt.ylabel("Eigenvector 2")
plt.show()
```
